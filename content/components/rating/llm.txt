# Rating 评分

评分组件用于收集用户对内容的星级评价，也支持只读展示模式。基于 Radix `useControllableState` 实现受控/非受控双模式，支持键盘导航和焦点管理。

本组件是复合组件，由 `Rating`（容器）和 `RatingButton`（单个星标按钮）组成，通过 Context 共享状态。无 Ant Design 直接对应。

## 何时使用 {#when-to-use}

当需要对内容进行评分时使用。

常见场景：

- 商品、文章、服务等的用户评分入口。
- 用户评价表单中的星级选择。
- 只读展示已有的评分结果。

推荐使用方式：

- 使用 `defaultValue`（非受控）或 `value`+`onValueChange`（受控）管理评分值。
- 评分范围由 children 数量决定，配合 `RatingButton` 子组件。
- 只读展示时设置 `readOnly`。

不推荐：

- 绕过 `Rating` 容器直接使用 `RatingButton`。
- 在 `RatingButton` 上自行管理选中状态——该状态由 Context 统一提供。

## 代码演示 {#examples}

### 基础用法

默认 5 颗星，初始评分 3。

```tsx
import { Rating, RatingButton } from "@/components/ui/rating"

export default function Demo() {
  return (
    <Rating defaultValue={3} className="w-full max-w-sm">
      {Array.from({ length: 5 }, (_, i) => (
        <RatingButton key={i} />
      ))}
    </Rating>
  )
}
```

### 自定义图标

可以传入自定义 SVG 图标替换默认星标。

```tsx
import { Rating, RatingButton } from "@/components/ui/rating"
import { Heart } from "lucide-react"

export default function Demo() {
  return (
    <Rating defaultValue={3}>
      {Array.from({ length: 5 }, (_, i) => (
        <RatingButton key={i} icon={<Heart />} />
      ))}
    </Rating>
  )
}
```

### 受控模式

```tsx
import { Rating, RatingButton } from "@/components/ui/rating"
import { useState } from "react"

export default function Demo() {
  const [value, setValue] = useState(2)

  return (
    <Rating value={value} onValueChange={setValue}>
      {Array.from({ length: 5 }, (_, i) => (
        <RatingButton key={i} />
      ))}
    </Rating>
  )
}
```

### 只读展示

```tsx
import { Rating, RatingButton } from "@/components/ui/rating"

export default function Demo() {
  return (
    <Rating value={4} readOnly>
      {Array.from({ length: 5 }, (_, i) => (
        <RatingButton key={i} />
      ))}
    </Rating>
  )
}
```

### 自定义数量

```tsx
import { Rating, RatingButton } from "@/components/ui/rating"

export default function Demo() {
  return (
    <Rating defaultValue={5}>
      {Array.from({ length: 10 }, (_, i) => (
        <RatingButton key={i} />
      ))}
    </Rating>
  )
}
```

## API {#api}

### Rating

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| defaultValue | `number` | `0` | 非受控模式下的默认评分值 |
| value | `number` | - | 受控模式下的评分值 |
| onChange | `(event, value: number) => void` | - | 值变化时的回调，传入原始事件和新的评分值 |
| onValueChange | `(value: number) => void` | - | 仅通知新值的回调（不含事件对象） |
| readOnly | `boolean` | `false` | 只读模式，禁止交互 |
| className | `string` | - | 自定义 CSS 类名 |
| children | `ReactNode` | - | 通常为 `RatingButton` 元素数组 |

### RatingButton

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| index | `number` | 由 Rating 注入 | 星标序号（0-based），由 Rating 容器通过 cloneElement 自动注入 |
| size | `number` | `20` | 图标像素尺寸 |
| className | `string` | - | 自定义 CSS 类名 |
| icon | `ReactElement<LucideProps>` | `<StarIcon />` | 自定义图标元素，激活时会添加 `fill-current` |
| ...LucideProps | - | - | 透传至图标的其他 Lucide icon 属性 |

## 样式模型 {#styling-model}

- 使用 Tailwind CSS 控制布局和交互样式。
- 容器使用 `inline-flex items-center gap-0.5`，各星标间隔 2px。
- 按钮聚焦环使用 `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`。
- 激活状态通过 `fill-current` 填充图标颜色。
- 只读模式移除 `cursor-pointer`，使用 `cursor-default`。
- 只读和禁用状态通过 `disabled` 属性控制交互。
- 悬停时实时预览评分效果。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/rating/index.tsx` — 完整实现
- `src/components/ui/rating.tsx` — 重导出

直接依赖摘要：

- React（useState, useCallback, useEffect, useRef, createContext, useContext, Children, cloneElement）
- `@radix-ui/react-use-controllable-state`（useControllableState）
- `lucide-react`（StarIcon 及可选的其它图标）
- Tailwind CSS

实现注意：

1. `Rating` 使用 React Context 向所有 `RatingButton` 子组件分发状态。
2. `Rating` 通过 `Children.map` + `cloneElement` 自动为每个 `RatingButton` 注入 `index`。
3. 键盘导航支持 `ArrowRight`/`ArrowLeft`，`Shift`/`Meta` + 方向键可直接跳至首尾。
4. 悬停预览使用 `hoverValue` 状态，鼠标离开容器后清除。
5. `focusedStar` 与 `useEffect` 配合确保键盘操作后焦点同步到对应按钮。
6. 受控/非受控模式由 `useControllableState` 统一管理。

## 可访问性 {#accessibility}

- 容器设置 `role="radiogroup"` 和 `aria-label="Rating"`。
- 每个 `RatingButton` 渲染为 `<button>`，通过 `tabIndex` 管理焦点——仅有当前值的按钮可获焦。
- 图标设置 `aria-hidden="true"`，避免辅助技术重复朗读。
- 键盘导航：左右方向键移动评分值，Shift/Meta 键配合方向键直接跳到首/尾。
- 聚焦样式使用 `focus-visible`，不干扰鼠标点击。
- `readOnly` 模式下按钮设置 `disabled`，移除交互能力。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/blocks/rating/index.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 评分方式 | 通过 children 数量控制星数，而非 `count` prop |
| 图标自定义 | 通过 `icon` prop 接收任意 ReactElement，而非 `character` 字符串 |
| 键盘行为 | 使用 Radix 风格焦点管理，不同于 AntD Rate 的键盘实现 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Rating 代码时，应遵守：

1. 从 `@/components/ui/rating` 导入 `Rating` 和 `RatingButton`，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码 `src/components/blocks/rating/index.tsx` 中存在的 props。
3. 不要使用 AntD Rate 的 `count`、`character`、`allowHalf`、`allowClear` 等 props——本组件不支持半星和清除。
4. 评分数量由 `Rating` 的子元素（`RatingButton`）数量决定，不是通过 prop 配置。
5. 图标自定义通过 `RatingButton` 的 `icon` prop 传入 ReactElement，不是字符串。
6. 不要生成 `antd` 专属的 `styles`、`classNames`、`ConfigProvider` 配置。
7. 受控模式使用 `value`+`onValueChange`，非受控使用 `defaultValue`。
8. 交互组件必须保留键盘和 ARIA 语义，每个 `RatingButton` 都必须是 `<button>`。
9. 如果不确定某个能力是否支持，先检查 `src/components/blocks/rating/index.tsx`，不要凭 AntD Rate 记忆生成。
