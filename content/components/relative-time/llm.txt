# RelativeTime 相对时间

RelativeTime 是一个跨时区时间展示的复合组件，通过双层 React Context 组合 `RelativeTimeZone`、`RelativeTimeZoneDisplay`、`RelativeTimeZoneDate`、`RelativeTimeZoneLabel` 等子组件。支持在单一容器内展示多个时区的日期和当前时间，未受控模式下秒级自动更新。底层使用浏览器原生 `Intl.DateTimeFormat` API 进行格式化，locale 固定为 `"en-US"`。

本组件是项目本地复合块（composite block），无直接 Ant Design 对应组件。API 以 `src/components/blocks/relative-time/index.tsx` 中的 TypeScript 类型为准。

## 何时使用

- 国际化仪表盘中展示跨时区团队当前时间。
- 全球服务状态页中显示各区域当地时间。
- 需要精确到秒的实时多时区时钟展示。
- 航班、会议、活动等场景的多时区时间对照。

不推荐：

- 仅展示单个时区的静态时间，直接使用 `Intl.DateTimeFormat` 更轻量。
- 倒计时或计时器场景（RelativeTime 展示时间点，不计算差值）。
- 在 `RelativeTime` 外部直接使用 `RelativeTimeZone` 及其子组件（缺少 Context 会导致异常）。

## 代码演示

### 基础用法

```tsx
import {
  RelativeTime,
  RelativeTimeZone,
  RelativeTimeZoneDate,
  RelativeTimeZoneDisplay,
  RelativeTimeZoneLabel,
} from "@/components/blocks/relative-time"

export default function Demo() {
  return (
    <RelativeTime>
      <RelativeTimeZone zone="America/New_York">
        <RelativeTimeZoneLabel>NYC</RelativeTimeZoneLabel>
        <RelativeTimeZoneDate />
        <RelativeTimeZoneDisplay />
      </RelativeTimeZone>
      <RelativeTimeZone zone="Asia/Shanghai">
        <RelativeTimeZoneLabel>北京</RelativeTimeZoneLabel>
        <RelativeTimeZoneDate />
        <RelativeTimeZoneDisplay />
      </RelativeTimeZone>
      <RelativeTimeZone zone="Europe/London">
        <RelativeTimeZoneLabel>London</RelativeTimeZoneLabel>
        <RelativeTimeZoneDate />
        <RelativeTimeZoneDisplay />
      </RelativeTimeZone>
    </RelativeTime>
  )
}
```

使用受控模式传入固定时间：

```tsx
<RelativeTime time={new Date("2026-07-01T14:00:00Z")}>
  <RelativeTimeZone zone="Asia/Shanghai">
    <RelativeTimeZoneLabel>上海</RelativeTimeZoneLabel>
    <RelativeTimeZoneDate />
    <RelativeTimeZoneDisplay />
  </RelativeTimeZone>
</RelativeTime>
```

> 说明：`RelativeTimeZoneDisplay` 渲染时间，`RelativeTimeZoneDate` 渲染日期，`RelativeTimeZoneLabel` 渲染时区标签。三者必须嵌套在 `RelativeTimeZone` 内，`RelativeTimeZone` 必须嵌套在 `RelativeTime` 内。受控模式传入 `time` 后停止自动更新；非受控模式每秒刷新。

## API

### RelativeTime

| 属性 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| time | 受控时间值，传入后停止自动更新 | `Date` | - |
| defaultTime | 非受控模式下的初始时间 | `Date` | `new Date()` |
| onTimeChange | 时间变化回调 | `(time: Date) => void` | - |
| dateFormatOptions | 日期格式化选项，应用于所有子时区 | `Intl.DateTimeFormatOptions` | `{ dateStyle: "long" }` |
| timeFormatOptions | 时间格式化选项，应用于所有子时区 | `Intl.DateTimeFormatOptions` | `{ hour: "2-digit", minute: "2-digit", second: "2-digit" }` |
| className | 容器自定义样式类 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。未受控模式下内部通过 `setInterval` 每 1 秒更新为 `new Date()`，使用 `@radix-ui/react-use-controllable-state` 管理受控/非受控双模式。

### RelativeTimeZone

| 属性 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| zone | IANA 时区标识符，如 `"Asia/Shanghai"` | `string` | 必填 |
| className | 容器自定义样式类 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。通过 `RelativeTimeZoneContext` 向子组件提供 `zone` 值。注：类型定义中包含 `dateFormatOptions` 和 `timeFormatOptions` 属性，但当前实现未消费它们，格式化选项统一从 `RelativeTime` 层级继承。

### RelativeTimeZoneDisplay

继承 `HTMLAttributes<HTMLDivElement>`。从 `RelativeTimeContext` 读取 `time` 和 `timeFormatOptions`，从 `RelativeTimeZoneContext` 读取 `zone`，调用 `formatTime()` 渲染格式化后的时间字符串。默认样式 `pl-8 text-muted-foreground tabular-nums`。

### RelativeTimeZoneDate

继承 `HTMLAttributes<HTMLDivElement>`。从 `RelativeTimeContext` 读取 `time` 和 `dateFormatOptions`，从 `RelativeTimeZoneContext` 读取 `zone`，调用 `formatDate()` 渲染格式化后的日期字符串。无额外默认样式。

### RelativeTimeZoneLabel

继承 `HTMLAttributes<HTMLDivElement>`。纯展示标签，无 Context 依赖。默认样式 `flex h-4 items-center justify-center rounded-xs bg-secondary px-1.5 font-mono`，通常传入时区缩写或城市名作为 children。

## 样式模型

- 样式完全由 Tailwind CSS className 控制，通过 `cn` 合并。
- `RelativeTime`：`grid gap-2`，垂直排列各时区行。
- `RelativeTimeZone`：`flex items-center justify-between gap-1.5 text-xs`，水平排列标签与时间。
- `RelativeTimeZoneDisplay`：`pl-8 text-muted-foreground tabular-nums`，等宽数字确保时间对齐。
- `RelativeTimeZoneLabel`：`h-4 rounded-xs bg-secondary px-1.5 font-mono`，Badge 风格紧凑标签。
- `RelativeTimeZoneDate`：无默认样式，继承父级排版。
- 不依赖 CVA、Radix Slot、Motion 动画库或 css-in-js 运行时。

## 依赖与实现说明

核心文件：
- `src/components/blocks/relative-time/index.tsx` — 组件实现
- `src/components/ui/relative-time.tsx` — 重新导出

直接依赖：
- `@radix-ui/react-use-controllable-state` — `useControllableState` hook，管理受控/非受控状态。
- `@/lib/utils` — `cn` 工具函数。

实现注意：
1. 双层 Context 架构：`RelativeTimeContext` 提供 `{ time, dateFormatOptions, timeFormatOptions }`；`RelativeTimeZoneContext` 提供 `{ zone }`。子组件按需消费。
2. `formatDate` 和 `formatTime` 为模块级函数，使用 `new Intl.DateTimeFormat("en-US", options).format(date)`。options 中 `timeZone` 与调用侧传入的 `zone` 合并。
3. 未受控模式：`useEffect` 中 `setInterval` 每秒更新，受控时自动清理 interval。
4. 所有格式化由浏览器原生 `Intl` API 完成，不依赖 dayjs、date-fns 等第三方日期库。
5. 组件为纯展示型复合块，不包含时间选择器、时区切换等交互功能。

## 可访问性

- `RelativeTimeZoneDisplay` 使用 `tabular-nums` 确保数字等宽显示，减少视觉跳动。
- 时区标签使用 `font-mono` 等宽字体，视觉上与标准时区标识符一致。
- 所有子组件为纯展示型 `<div>`，屏幕阅读器可自然读取文本内容。
- 建议在 `RelativeTimeZoneLabel` 中使用人类可读的时区名称（城市名或 UTC 偏移），而非仅展示裸 IANA 标识符。

## 与 Ant Design 的取舍

Ant Design 不提供多时区时间展示组件。AntD 的 DatePicker、Calendar 为输入/选择组件，与本组件的纯展示用途不同。

| 方向 | 本地决策 |
| --- | --- |
| AntD 对应 | 无直接对应组件 |
| 构建方式 | React Context + 原生 Intl.DateTimeFormat |
| API 来源 | 以 `src/components/blocks/relative-time/index.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | Tailwind CSS + 项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 无 AntD 兼容包袱，完全按本地设计体系定义 |

## LLM 使用规则

1. 从 `@/components/blocks/relative-time` 导入 `RelativeTime`、`RelativeTimeZone`、`RelativeTimeZoneDate`、`RelativeTimeZoneDisplay`、`RelativeTimeZoneLabel`。
2. 嵌套层级必须严格遵循：`RelativeTime` > `RelativeTimeZone` > (`RelativeTimeZoneLabel` / `RelativeTimeZoneDate` / `RelativeTimeZoneDisplay`)。
3. `zone` 必须使用有效 IANA 时区标识符（如 `"Asia/Shanghai"`、`"America/New_York"`、`"UTC"`），不要使用 UTC 偏移量字符串。
4. 展示实时时钟时不传 `time` prop（非受控模式，每秒自动更新）；展示指定时刻时传入 `time` prop（受控模式，停止自动更新）。
5. 不要引入 dayjs、date-fns 等日期库；组件已内置 `Intl.DateTimeFormat` 格式化。
6. 自定义格式通过 `dateFormatOptions` 和 `timeFormatOptions` 传入，遵循 `Intl.DateTimeFormatOptions` 规范。
7. 不要生成 Ant Design 专属的 `ConfigProvider`、`classNames`、`styles` 或 Design Token 配置。
8. 该组件为复合区块，并非 Ant Design 或 shadcn/ui 的一等公民组件。使用前需确认已在项目中引入。
