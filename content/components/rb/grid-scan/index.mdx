GridScan 是一个基于 Three.js 和 postprocessing 的 3D 网格扫描背景组件，模拟投影到 3D 平面的扫描线和网格效果，支持 Bloom 和色散后处理。

<h2 id="when-to-use">何时使用</h2>
当需要科幻/扫描风格的 3D 网格背景时使用。
常见场景：
- 科技/数据可视化风格页面的背景
- 安全/监控类 Dashboard 的装饰背景
- 赛博朋克风格的着陆页
不推荐：
- 传统企业官网（风格过于科幻）
- 需要高对比度文字可读性的区域（网格可能干扰阅读）
- 性能受限的移动端（Three.js + postprocessing 管线开销较大）

<h2 id="faq">常见问题</h2>
### 如何让扫描线更明显？
增大 `scanOpacity` 和 `scanGlow`，同时调整 `scanColor` 为更亮的颜色。配合 `bloomIntensity` 开启 Bloom 后处理效果更佳。

### 后处理影响性能吗？
`enablePost={true}` 时启用了 EffectComposer + Bloom + ChromaticAberration 管线，在移动端可能影响帧率。可设置 `enablePost={false}` 关闭后处理。

### 扫描方向有哪些？
`forward`（默认从上到下）、`backward`（从下到上）、`pingpong`（往返扫描）。`scanDirection` 属性传入对应的字符串即可。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-grid-scan` 导入。
2. 依赖 `three` 和 `postprocessing`，确保已安装。
3. `lineStyle` 仅支持 `"solid"`、`"dashed"`、`"dotted"` 三种值。
4. 面部追踪功能（`enableWebcam`）需要摄像头权限和 face-api.js 模型文件。
