StickerPeel 贴纸剥离组件，悬停时模拟贴纸被撕开的效果，带有 SVG 光照滤镜，支持拖拽和自定义撕开方向。

<h2 id="when-to-use">何时使用</h2>
当需要...时使用。
常见场景：
- 个性化图片展示，模拟实体贴纸的撕开交互
- 创意相册或作品集，增强触感体验
- 电商产品图的趣味化展示
不推荐：
- 需要精确布局的正式页面（拖拽会打乱位置）
- 移动端为主的应用（拖拽体验在触屏上受限）

### 自定义配置
```tsx
import { StickerPeel } from "@/components/ui/sticker-peel"

export default function Demo() {
  return (
    <div className="relative h-96 w-full">
      <StickerPeel
        imageSrc="https://images.unsplash.com/photo-1550745165-9bc0b252726f?w=400&q=80"
        rotate={15}
        peelBackHoverPct={25}
        peelBackActivePct={50}
        width={250}
        shadowIntensity={0.8}
        lightingIntensity={0.15}
        peelDirection={45}
        peelEasing="power4.out"
        initialPosition={{ x: 100, y: 50 }}
      />
    </div>
  )
}
```

<h2 id="api">API</h2>
### StickerPeel
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| imageSrc | 图片源地址（必填） | `string` | - |
| rotate | 贴纸旋转角度 | `number` | `30` |
| peelBackHoverPct | 悬停时撕开百分比 | `number` | `30` |
| peelBackActivePct | 按下时撕开百分比 | `number` | `40` |
| peelEasing | 撕开缓动函数（GSAP easing） | `string` | `"power3.out"` |
| peelHoverEasing | 悬停缓动函数 | `string` | `"power2.out"` |
| width | 贴纸宽度（像素） | `number` | `200` |
| shadowIntensity | 阴影强度（0-1） | `number` | `0.6` |
| lightingIntensity | 光照强度（0-1） | `number` | `0.1` |
| initialPosition | 初始位置 | `"center" \| { x: number; y: number }` | `"center"` |
| peelDirection | 撕开方向（角度制） | `number` | `0` |
| className | 自定义类名 | `string` | - |

<h2 id="faq">常见问题</h2>
### 撕开方向和旋转角度有什么区别？
`peelDirection` 控制撕开折叠的方向（整个容器的旋转），`rotate` 控制贴纸内部的旋转角度，两者独立生效，可组合使用创造更丰富的效果。

### 如何调整阴影和光照强度？
`shadowIntensity` 控制 `feDropShadow` 的透明度，`lightingIntensity` 控制 `feSpecularLighting` 的强度。两个值都在 0-1 范围内，越大效果越强。

### 拖拽在移动端表现如何？
移动端会禁用默认光标样式。由于使用 GSAP Draggable，在移动端拖拽仍可响应，但体验不如桌面端流畅。建议移动端仅展示悬停效果。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/sticker-peel` 导入。
2. 必须提供 `imageSrc` 属性。
3. 依赖 `gsap` 和 `gsap/Draggable`，确保项目中已安装。
4. 使用 SVG 滤镜实现光照和阴影效果，无需额外 CSS 文件。
