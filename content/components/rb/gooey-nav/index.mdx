GooeyNav 粘性粒子导航栏，点击导航项时产生液态粒子爆散动画效果。

<h2 id="when-to-use">何时使用</h2>
当需要动态、有趣的导航交互体验时使用，粒子效果增强用户点击反馈。
常见场景：
- 创意型网站的主导航栏
- 作品集页面的分类切换导航
- 需要视觉冲击力的着陆页导航
不推荐：
- 需要高可访问性（无障碍）的正式网站
- 导航项数量过多（>8个）时粒子可能重叠混乱
- 移动端性能敏感的场景（粒子动画消耗资源）

### 自定义粒子效果
```tsx
import { GooeyNav } from "@/components/navigation/react-bits/gooey-nav"

export default function Demo() {
  return (
    <GooeyNav
      items={[
        { label: "Design", href: "#design" },
        { label: "Code", href: "#code" },
        { label: "Write", href: "#write" },
      ]}
      animationTime={800}
      particleCount={20}
      particleDistances={[120, 15]}
      initialActiveIndex={1}
    />
  )
}
```

<h2 id="api">API</h2>
### GooeyNav
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| items | 导航项数组（必填） | `{ label: string; href: string }[]` | - |
| animationTime | 动画持续时间（ms） | `number` | `600` |
| particleCount | 每次点击生成的粒子数量 | `number` | `15` |
| particleDistances | 粒子扩散起始/结束距离范围 | `[number, number]` | `[90, 10]` |
| particleR | 粒子旋转半径系数 | `number` | `100` |
| timeVariance | 粒子动画时间变化范围（ms） | `number` | `300` |
| colors | 粒子颜色索引数组（对应CSS变量） | `number[]` | `[1,2,3,1,2,3,1,4]` |
| initialActiveIndex | 初始激活的导航项索引 | `number` | `0` |

<h2 id="faq">常见问题</h2>
### 如何自定义粒子颜色？
通过 `colors` 属性传入数字索引数组，组件会使用 `var(--color-1)` 到 `var(--color-4)` 等 CSS 变量。在父容器或全局样式中定义这些 CSS 变量即可。

### 导航项的 href 如何与路由集成？
组件渲染的是原生 `<a>` 标签，`href` 直接传递。对于 React Router 等 SPA 路由，可以在点击事件中自行处理，或包裹 Link 组件。

### 移动端性能如何？
粒子动画通过 DOM 操作实现，量大时建议减少 `particleCount` 并在移动端降低 `animationTime`。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/navigation/react-bits/gooey-nav` 导入。
2. `items` 为必填，每项包含 `label` 和 `href`。
3. 粒子颜色通过 CSS 变量 `--color-1` 至 `--color-4` 控制，在全局 CSS 中定义。
4. 使用 `initialActiveIndex` 设置默认高亮项。
