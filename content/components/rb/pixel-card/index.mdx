PixelCard 像素化卡片边框动画，鼠标 hover 时卡片边缘生成像素点阵扩散/收缩效果。

<h2 id="when-to-use">何时使用</h2>
当需要独特边框动效的卡片展示时使用。
常见场景：
- 功能特性卡片展示
- 作品集卡片 hover 效果
- 价格方案卡片装饰
不推荐：
- 卡片内容需要大量文字阅读（动画分散注意力）
- 对无障碍要求严格的场景（依赖 hover/focus）

```tsx

```

<h2 id="api">API</h2>

### PixelCard

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 预设色彩变体 | "default" \| "blue" \| "yellow" \| "pink" | "default" |
| gap | 像素之间的间距 | number | 5 |
| speed | 动画速度（0-100） | number | 35 |
| colors | 像素颜色（逗号分隔的 hex 值） | string | 根据 variant 预设 |
| noFocus | 是否禁用 focus 触发 | boolean | false |
| className | 额外的 CSS 类名 | string | "" |
| children | 卡片内容 | React.ReactNode | - |

<h2 id="faq">常见问题</h2>

### 如何自定义像素颜色？
传入 `colors` 字符串，多个颜色用逗号分隔，如 `colors="#ff0000,#00ff00,#0000ff"`。每个像素会随机选取一种颜色。

### pixel-card CSS 类在哪里定义？
组件依赖外部 CSS 文件 `PixelCard.css`，需确保项目中已导入对应的样式文件。

### focus 和 hover 行为有何区别？
默认同时响应 hover 和 focus 事件。设置 `noFocus={true}` 后仅响应鼠标 hover，可避免键盘 Tab 导航时的误触发。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-pixel-card` 导入。
2. 需要传入 `children` 作为卡片内容，卡片内部元素会自动居中。
3. 使用 Canvas API 实现，依赖浏览器 Canvas 支持。
4. 需要项目中已导入 `PixelCard.css` 样式文件。
