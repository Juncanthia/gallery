Aurora 基于 OGL 着色器的极光渐变背景组件，模拟北极光流动效果。

<h2 id="when-to-use">何时使用</h2>

当需要创建动态极光风格的渐变背景，色彩沿水平方向流动并带有噪声扰动时使用。

常见场景：
- 页面全屏背景、Hero 区域装饰
- 需要柔和色彩过渡和动态氛围的产品展示
- 作为卡片或区块的背景层（通过叠加定位）

不推荐：
- 需要垂直渐变或径向渐变（本组件为水平方向着色器效果）
- 需要精确色彩控制的渐变动画（颜色由噪声函数驱动）

<h2 id="examples">代码演示</h2>

### 基础用法

<ComponentShowcase id="rb/aurora" path="rb/aurora/basic" />

```tsx
import { Aurora } from "@/components/ui/rb-aurora"

export default function Demo() {
  return <Aurora />
}
```

### 自定义极光色彩

```tsx
import { Aurora } from "@/components/ui/rb-aurora"

export default function CustomDemo() {
  return (
    <Aurora
      colorStops={["#ff0000", "#00ff00", "#0000ff"]}
      amplitude={1.2}
      blend={0.7}
      speed={2.0}
    />
  )
}
```

<h2 id="api">API</h2>

### Aurora

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| colorStops | 三色渐变色阶数组 | `string[]` | `["#5227FF","#7cff67","#5227FF"]` |
| amplitude | 噪声振幅 | `number` | `1.0` |
| blend | 极光与背景混合强度 | `number` | `0.5` |
| time | 手动控制动画时间（传值后忽略内部时间） | `number` | - |
| speed | 动画速度倍率 | `number` | `1.0` |

<h2 id="faq">常见问题</h2>

### colorStops 必须是 3 个颜色吗？

是的。着色器内部硬编码了 3 个 ColorStop（位置 0.0、0.5、1.0），传入更多或更少颜色会导致着色器行为异常。

### 组件占满父容器吗？

组件本身使用 `h-full w-full`，会自动填充父容器。确保父容器有明确高度。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/rb-aurora` 导入。
2. 该组件依赖 `ogl` 库（不是 Three.js）。
3. 父容器必须有明确的宽高尺寸。
4. `colorStops` 数组固定为 3 个 CSS 颜色字符串。
5. 若需手动控制动画帧，传入 `time` prop 而非依赖组件内部 requestAnimationFrame。
