Orb 带 hue 旋转和悬停交互的 WebGL 球体背景。

<h2 id="when-to-use">何时使用</h2>
当需要炫彩的球体动态背景时使用。
常见场景：
- 页面 Hero 区域背景装饰
- 品牌视觉的动态背景
- 加载/过渡页的背景动效
不推荐：
- 包含大量文字阅读内容的区域（过于分散注意力）
- 需要精确色彩控制的品牌页面

### 自定义色相和悬停强度

```tsx
import { Orb } from "@/components/ui/rb-orb"

export default function Demo() {
  return (
    <div className="w-full h-64 overflow-hidden rounded">
      <Orb hue={180} hoverIntensity={0.4} rotateOnHover={true} backgroundColor="#111111" />
    </div>
  )
}
```

<h2 id="api">API</h2>

### Orb

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| hue | 色相旋转角度（度） | number | 0 |
| hoverIntensity | 悬停交互强度（0-1） | number | 0.2 |
| rotateOnHover | 悬停时是否旋转 | boolean | true |
| forceHoverState | 强制悬停状态 | boolean | false |
| backgroundColor | 背景颜色（hex） | string | `"#000000"` |

<h2 id="faq">常见问题</h2>

### 为什么球体不显示？
Orb 使用 WebGL 渲染，需要父容器有明确的宽高（非 0）。确保外层容器设置了 `w-full h-full` 且有实际尺寸。

### 如何让球体持续旋转而不需要悬停？
设置 `forceHoverState={true}` 并配合 `rotateOnHover={true}`。

### 这个组件性能开销大吗？
Orb 使用 requestAnimationFrame 持续渲染 WebGL，在移动端低性能设备上建议仅在 Hero 等关键区域使用。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-orb` 导入。
2. 组件需要放在一个有明确宽高的容器中，默认渲染 100% 宽高。
3. 依赖 `ogl` 库，请确保项目已安装。
4. 默认参数已经可以正常渲染，无需额外配置。
