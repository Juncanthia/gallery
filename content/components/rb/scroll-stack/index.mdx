ScrollStack 滚动堆叠组件，滚动时卡片依次固定在视口并逐层堆叠缩放，最后一张卡片释放后继续滚动，基于 Lenis 平滑滚动实现。

<h2 id="when-to-use">何时使用</h2>
需要卡片逐个堆叠并固定的滚动交互效果。
常见场景：
- 产品展示页的多层卡片揭幕
- 作品集/案例展示的沉浸式浏览
- 步骤引导页的逐层揭示
不推荐：
- 卡片内容过少（单张卡片堆叠效果不明显）
- 移动端使用（堆叠效果在窄屏上体验较差）
- 需要嵌套在已有自定义滚动的容器中且未正确配置 useWindowScroll

```tsx
      <ScrollStackItem><div className="h-64 bg-gray-700 rounded-lg p-8">第二张卡片</div></ScrollStackItem>
      <ScrollStackItem><div className="h-64 bg-gray-600 rounded-lg p-8">第三张卡片</div></ScrollStackItem>

}
```

<h2 id="api">API</h2>
### ScrollStack
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 子元素，应为 `ScrollStackItem` | `React.ReactNode` | - |
| className | 滚动容器 className | `string` | `""` |
| itemDistance | 卡片之间的间距（像素） | `number` | `100` |
| itemScale | 每张卡片相较于前一张的缩放增量 | `number` | `0.03` |
| itemStackDistance | 卡片堆叠时的垂直偏移量 | `number` | `30` |
| stackPosition | 堆叠在视口中的起始位置（百分比或像素） | `string` | `"20%"` |
| scaleEndPosition | 缩放结束位置 | `string` | `"10%"` |
| baseScale | 最底层卡片的基础缩放值 | `number` | `0.85` |
| scaleDuration | （保留参数）缩放持续时间 | `number` | `0.5` |
| rotationAmount | 每张卡片的递增旋转量（度） | `number` | `0` |
| blurAmount | 被遮挡卡片的模糊增量 | `number` | `0` |
| useWindowScroll | 是否使用 window 滚动（true）还是组件内部滚动（false） | `boolean` | `false` |
| onStackComplete | 所有卡片完成堆叠时的回调 | `() => void` | - |

### ScrollStackItem
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 卡片内容 | `React.ReactNode` | - |
| itemClassName | 卡片自定义 className | `string` | `""` |

<h2 id="faq">常见问题</h2>
### 卡片堆叠后如何退出？
最后一张卡片后会自动追加 `.scroll-stack-end` 标记元素。当滚动超越该标记时，所有卡片恢复正常流并继续滚动。确保 ScrollStack 内有足够的后续内容。

### useWindowScroll 和内部滚动有什么区别？
`useWindowScroll=true` 时组件监听 window 滚动，卡片相对于视口固定。`useWindowScroll=false`（默认）时组件内部创建 Lenis 平滑滚动容器，卡片在容器内固定。通常独立页面用 window 模式，嵌入面板用内部滚动。

### 为什么卡片出现跳动或闪烁？
检查卡片是否设置了 `transform` CSS 属性冲突。组件会直接操作卡片的 `style.transform` 和 `style.filter`，外部不要对 `.scroll-stack-card` 设置 transform。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-scroll-stack` 导入 `ScrollStack` 和 `ScrollStackItem`。
2. 子元素必须使用 `ScrollStackItem` 包裹，否则不会被识别和处理。
3. 依赖 `lenis` 实现平滑滚动，确保项目已安装该依赖。
4. 组件直接操作 DOM style 属性，不要对 `.scroll-stack-card` 使用 CSS transform/filter。
