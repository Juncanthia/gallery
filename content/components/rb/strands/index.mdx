Strands 丝线波动组件，基于 OGL/WebGL 着色器的多彩波浪线条动画，支持自定义调色板、物理参数和可选玻璃透镜折射效果。

<h2 id="when-to-use">何时使用</h2>
当需要...时使用。
常见场景：
- 页面 Hero 区域的动态视觉背景
- 品牌展示页面的多彩波浪装饰
- 音乐/艺术类网站的视觉氛围营造
不推荐：
- 低性能设备或需要频繁重绘的场景（WebGL 开销）
- 作为信息承载的主要区域（仅作背景装饰）

<h2 id="api">API</h2>
### Strands
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| colors | 色彩调色板，线条从中采样 | `string[]` | `["#FF4242","#7C3AED","#06B6D4","#EAB308"]` |
| count | 波浪线条数量（最大 12） | `number` | `3` |
| speed | 动画播放速度 | `number` | `0.5` |
| amplitude | 波浪振幅 | `number` | `1` |
| waviness | 波浪起伏程度 | `number` | `1` |
| thickness | 线条粗细 | `number` | `0.7` |
| glow | 发光强度 | `number` | `2.6` |
| taper | 线条两端渐细程度 | `number` | `3` |
| spread | 线条间的相位扩展 | `number` | `1` |
| hueShift | 全局色相偏移 | `number` | `0` |
| intensity | 整体强度 | `number` | `0.6` |
| saturation | 色彩饱和度 | `number` | `1.5` |
| opacity | 整体不透明度 | `number` | `1` |
| scale | 整体缩放 | `number` | `1.5` |
| glass | 是否启用玻璃透镜折射效果 | `boolean` | `false` |
| refraction | 透镜折射强度（仅 glass=true 时生效） | `number` | `1` |
| dispersion | 透镜色散/色差强度（仅 glass=true 时生效） | `number` | `1` |
| glassSize | 玻璃透镜尺寸系数 | `number` | `1` |
| className | 自定义类名 | `string` | - |
| style | 自定义 CSS 样式 | `CSSProperties` | - |

<h2 id="faq">常见问题</h2>
### glass 模式和非 glass 模式有什么区别？
非 glass 模式直接渲染波浪线条；glass 模式额外渲染一个圆形透镜，通过 `refraction` 和 `dispersion` 产生透过玻璃看波浪的折射和色差效果，视觉上更丰富但性能开销更大。

### 颜色调色板如何工作？
着色器将 `colors` 数组视为色板，在每条线的采样位置沿色板插值取色。若 `uColorCount` 大于 0 则使用自定义色板，否则使用默认的 spectrum 光谱函数。

### 性能方面有什么注意事项？
`count` 上限为 12（`MAX_STRANDS`），颜色数上限为 8（`MAX_COLORS`）。`glass` 模式会额外执行一次 render-to-texture + 透镜 pass，性能开销翻倍，慎用。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-strands` 导入。
2. 依赖 `ogl`（Omar's WebGL Library），确保项目中已安装。
3. 容器需要有明确的 `width` 和 `height`，组件会自动填充。
4. 所有参数可在运行时动态更新，通过 `propsRef` 实时传递给着色器。
