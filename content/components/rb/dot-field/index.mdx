DotField 基于 Canvas 2D 的点阵交互背景，鼠标悬停时点阵向外膨胀或推开，支持渐变色彩、光晕追踪和闪烁效果。

<h2 id="when-to-use">何时使用</h2>
当需要高密度点阵交互背景，鼠标移过时产生液体般的膨胀或推力效果时使用。
常见场景：
- 创意页面的沉浸式背景
- 数据可视化风格的装饰背景
- 鼠标驱动的有机交互体验
不推荐：
- 需要精确 UI 操作的区域（点阵可能分散注意力）
- 对 Canvas 性能敏感的弱设备（点阵密度取决于 dotSpacing）

<h2 id="api">API</h2>

### DotField

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| dotRadius | 点的半径（像素） | `number` | `1.5` |
| dotSpacing | 点之间的距离（不含半径） | `number` | `14` |
| cursorRadius | 鼠标影响范围半径 | `number` | `500` |
| cursorForce | 鼠标推力强度（仅在 bulgeOnly=false 时生效） | `number` | `0.1` |
| bulgeOnly | 仅膨胀模式（true=点向外推，false=点被弹开） | `boolean` | `true` |
| bulgeStrength | 膨胀强度（仅在 bulgeOnly=true 时生效） | `number` | `67` |
| glowRadius | SVG 光晕的半径 | `number` | `160` |
| sparkle | 是否启用闪烁效果（随机放大某些点） | `boolean` | `false` |
| waveAmplitude | 全局波形振幅（点随时间和位置上下波动） | `number` | `0` |
| gradientFrom | Canvas 渐变起始色 | `string` | `"rgba(168, 85, 247, 0.35)"` |
| gradientTo | Canvas 渐变结束色 | `string` | `"rgba(180, 151, 207, 0.25)"` |
| glowColor | 鼠标光晕颜色 | `string` | `"#120F17"` |

<h2 id="faq">常见问题</h2>

### bulgeOnly true 和 false 有什么区别？
`bulgeOnly=true`：鼠标附近的点向外膨胀（排挤效果），鼠标移走后点回弹到原位。`bulgeOnly=false`：点被鼠标像磁铁一样向反方向推开，带有速度和惯性（利用 `cursorForce` 和鼠标移动速度计算推力）。

### sparkle 如何工作？
使用哈希算法（`(i * 2654435761) ^ (frameCount >> 3)`），每帧随机选取约 3% 的点放大 1.8 倍绘制，模拟星空闪烁效果。不会影响点的位置数据，纯粹是绘制层面的变化。

### waveAmplitude 的作用？
叠加正弦波全局动画，使所有点在 Y/X 轴方向按正弦函数振荡。振幅越大振荡越剧烈。设为 0 则无波形效果。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/dot-field` 导入。
2. 组件使用 `React.memo` 包裹以提高性能。
3. 点阵通过 Canvas 2D 绘制，光晕使用独立 SVG `<circle>` + `<radialGradient>` 叠加层。
4. 支持通过 `{...rest}` 将额外 props 传递到外层 `div` 容器。
