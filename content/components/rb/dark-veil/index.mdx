DarkVeil 基于 OGL 的 CPPN（组合模式生成网络）着色器背景，生成有机纹理，支持色调偏移、扫描线、噪点等 CRTV 效果。

<h2 id="when-to-use">何时使用</h2>
当需要独特的有机神经网络生成纹理作为背景或视觉装饰时使用。
常见场景：
- 深色主题页面的动态背景
- 科幻、黑客风格的视觉装饰
- 需要 CRT 复古效果的叠加层
不推荐：
- 浅色背景页面（默认生成偏暗色调）
- 需要高性能的移动端（每次重绘都走完整 CPPN 计算）

<h2 id="api">API</h2>

### DarkVeil

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| hueShift | 色调旋转角度（度），0 即不旋转 | `number` | `0` |
| noiseIntensity | 画面随机噪点强度，0 为无噪点 | `number` | `0` |
| scanlineIntensity | 横向扫描线强度，0 为无扫描线 | `number` | `0` |
| speed | 动画播放速度倍率 | `number` | `0.5` |
| scanlineFrequency | 扫描线密度（控制扫描线在 Y 轴的出现频率） | `number` | `0` |
| warpAmount | 画面空间扭曲强度 | `number` | `0` |
| resolutionScale | 渲染分辨率缩放，低于 1 可降低像素以获得复古颗粒感 | `number` | `1` |

<h2 id="faq">常见问题</h2>

### scanlineIntensity 和 scanlineFrequency 如何配合？
`scanlineIntensity` 控制扫描线的暗度深度（0=无，1=最暗），`scanlineFrequency` 控制扫描线密度。典型 CRT 效果设置为 `{ scanlineIntensity: 0.3, scanlineFrequency: 800 }`。

### resolutionScale 降低会影响性能吗？
降低 `resolutionScale` 会减少渲染像素数，反而降低 GPU 负载，适合性能较弱的设备。但画面会呈现明显的像素颗粒感，这与组件风格通常是正向匹配的。

### CPPN 纹理是确定性的吗？
不是。CPPN 使用合成模式生成网络，每次渲染结果由时间 `uTime` 和诸多随机权重矩阵驱动，纹理随时间流动变化，无法预测固定状态。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/dark-veil` 导入。
2. 组件依赖 OGL（`ogl` 包），需确保项目已安装。
3. Canvas 输出会自动填入父容器，渲染类名为 `darkveil-canvas`。
4. 所有 uniform 参数变化时会实时更新 shader，无需重新创建渲染管线。
