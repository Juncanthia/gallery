ShapeGrid 基于 Canvas 2D 的图形网格背景组件，支持方形、圆形、六边形和三角形四种图形类型，网格可向指定方向移动，鼠标悬停时有填充和拖尾效果。

<h2 id="when-to-use">何时使用</h2>
需要可定制的几何图形网格背景时使用。
常见场景：
- 科技感/未来感主题的页面背景
- 数据可视化仪表盘的装饰网格
- 登录/注册页面的视觉点缀
不推荐：
- 高密度网格 + 大画布导致性能下降的场景
- 移动端低性能设备上全屏使用

```tsx
export default function Demo() {
  return (
    <div className="w-full h-80 relative bg-black">
      <ShapeGrid
        shape="hexagon"
        direction="diagonal"
        speed={0.5}

        hoverTrailAmount={5}
      />
    </div>
  )
}
```

<h2 id="api">API</h2>
### ShapeGrid
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| direction | 网格移动方向 | `"right" \| "left" \| "up" \| "down" \| "diagonal"` | `"right"` |
| speed | 网格移动速度 | `number` | `1` |
| borderColor | 图形描边颜色（支持 hex） | `string` | `"#999"` |
| squareSize | 图形大小（正方形边长/圆形直径/六边形/三角形尺寸） | `number` | `40` |
| hoverFillColor | 鼠标悬停时的填充颜色 | `string` | `"#222"` |
| shape | 图形类型 | `"square" \| "circle" \| "hexagon" \| "triangle"` | `"square"` |
| hoverTrailAmount | 鼠标拖尾数量，0 则不显示拖尾 | `number` | `0` |
| className | 容器自定义类名 | `string` | `""` |

<h2 id="faq">常见问题</h2>
### 切换 shape 类型后网格出现跳动？
不同 shape 的布局计算方式不同（六边形用蜂窝排列，三角形用交错半格排列），切换时网格偏移量会重置。这是预期行为，可视为 shape 切换的过渡效果。

### hoverTrailAmount 拖尾如何工作？
鼠标移动经过的单元格被记录到轨迹数组中，数量由 `hoverTrailAmount` 控制。拖尾透明度从最近到最远递减（`(len-i)/(len+1)`），通过 `cellOpacities` Map 做平滑插值过渡。

### Canvas 尺寸与父容器不一致？
组件在 resize 事件时以 `canvas.offsetWidth/offsetHeight` 设置画布尺寸。确保父容器有明确尺寸（非百分比依赖内容撑开），否则 canvas 尺寸可能为 0。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-shape-grid` 导入。
2. Canvas 自动填充父容器，父容器需有明确的宽高。
3. 六边形和三角形使用特殊的网格布局算法，参数含义与其他 shape 不同。
4. 悬停检测基于 Canvas 坐标计算，不依赖 DOM 事件委托。
