ScrambledText 鼠标靠近时文字随机乱码效果组件，基于 PointerEvent 实现高性能的字符级交互干扰动画。

<h2 id="when-to-use">何时使用</h2>
需要交互式的文字干扰/解密效果时使用。
常见场景：
- 悬停揭示文字内容，增强神秘感或趣味性
- 加密/解密主题的视觉展示
- 代码/终端风格的交互装饰
不推荐：
- 纯静态文字展示（无交互需求）
- 对辅助功能要求高的信息展示场景（乱码期间文字不可读）

### 自定义乱码字符集和影响范围
```tsx
import { ScrambledText } from "@/components/ui/rb-scrambled-text"

export default function Demo() {
  return (
    <div className="flex justify-center p-8">
      <ScrambledText radius={150} duration={1.5} scrambleChars="01">
        01001000 01100101 01101100 01101100 01101111
      </ScrambledText>
    </div>
  )
}
```

<h2 id="api">API</h2>
### ScrambledText
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 文本内容（必填） | `string` | - |
| radius | 鼠标影响半径（像素），在此范围内触发乱码 | `number` | `100` |
| duration | 单个字符乱码持续时间（秒） | `number` | `1.2` |
| speed | 乱码更新速度（保留参数，当前未直接使用） | `number` | `0.5` |
| scrambleChars | 乱码时使用的字符集，从该集合中随机选取 | `string` | `".:"` |
| className | 自定义样式类名，会追加到默认样式之后 | `string` | `""` |
| style | 自定义内联样式 | `React.CSSProperties` | `{}` |

<h2 id="faq">常见问题</h2>
### 乱码恢复后字符偶尔丢失，怎么办？
检查文本中是否包含特殊 Unicode 字符。组件使用 `PointerEvent` 监听，每个字符独立管理动画状态。通常确保 `duration` 不过短即可。

### 如何控制乱码的外观风格？
通过 `scrambleChars` 自定义乱码字符集，例如 `"01"` 实现二进制风格，`"!@#$%^&"` 实现符号风格。`radius` 控制鼠标多远距离触发，值越大越容易被触发。

### 组件如何处理空格和换行？
组件将 `children` 作为段落文本处理，空格正常显示。如需换行，可在文本中直接使用换行符。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-scrambled-text` 导入。
2. `children` 为必填属性，必须是纯文本字符串。
3. 组件使用 PointerEvent（而非 MouseEvent），支持触摸和笔输入。
4. 适合暗色背景使用（默认白色文字），若需浅色背景请通过 className 覆盖文字颜色。
