SplashCursor 基于 WebGL 流体模拟的全屏鼠标飞溅效果组件，鼠标移动时产生彩色流体动画，支持彩虹模式和固定颜色，使用完整的 Navier-Stokes 流体求解器。

<h2 id="when-to-use">何时使用</h2>
需要为页面添加全屏鼠标流体飞溅交互效果。
常见场景：
- 创意/作品集网站的首页特效
- 着陆页的炫酷视觉吸引
- 品牌营销页面的沉浸式体验
不推荐：
- 内容密集的页面（流体效果会干扰阅读）
- 移动端（无 hover 且流体计算性能开销大）
- 性能敏感的应用（WebGL 流体模拟持续消耗 GPU）

<h2 id="api">API</h2>
### SplashCursor
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| SIM_RESOLUTION | 流体模拟分辨率，值越大精度越高但性能消耗越大 | `number` | `128` |
| DYE_RESOLUTION | 染料（颜色）纹理分辨率 | `number` | `1440` |
| CAPTURE_RESOLUTION | （保留参数）捕获分辨率 | `number` | `512` |
| DENSITY_DISSIPATION | 密度消散速度，值越大颜色消失越快 | `number` | `3.5` |
| VELOCITY_DISSIPATION | 速度消散速度，值越大流体越快静止 | `number` | `2` |
| PRESSURE | 压力保持系数 | `number` | `0.1` |
| PRESSURE_ITERATIONS | 压力求解迭代次数 | `number` | `20` |
| CURL | 涡旋强度，控制流体的旋转卷曲程度 | `number` | `3` |
| SPLAT_RADIUS | 飞溅半径（相对于画布的比例） | `number` | `0.2` |
| SPLAT_FORCE | 飞溅力度，控制鼠标移动产生的流体推力 | `number` | `6000` |
| SHADING | 是否启用法向着色计算，增强立体感 | `boolean` | `true` |
| COLOR_UPDATE_SPEED | 彩虹模式下颜色切换速度 | `number` | `10` |
| BACK_COLOR | 背景色 | `{ r: number, g: number, b: number }` | `{ r: 0.5, g: 0, b: 0 }` |
| TRANSPARENT | 是否使用透明背景 | `boolean` | `true` |
| RAINBOW_MODE | 彩虹模式：鼠标移动颜色在 HSL 色环循环 | `boolean` | `true` |
| COLOR | 固定颜色（hex 格式），仅 `RAINBOW_MODE=false` 时生效 | `string` | `"#ff0000"` |

<h2 id="faq">常见问题</h2>
### 性能太差，页面卡顿？
降级建议：`DYE_RESOLUTION=256`（大幅降低染料分辨率）、`SIM_RESOLUTION=64`（降低模拟精度）、`SHADING=false`（关闭法向着色）、`DENSITY_DISSIPATION=8`（加速消散减少屏幕流体量）。

### 彩虹模式和固定颜色模式如何选择？
`RAINBOW_MODE=true` 时，每次 splat 都会从 HSL 色环随机取色，颜色随时间通过 `COLOR_UPDATE_SPEED` 刷新。`RAINBOW_MODE=false` 时使用 `COLOR` 指定的单一颜色。

### 组件是全屏 fixed 定位吗？
是的，组件内部使用 `position: fixed; top: 0; left: 0; width: 100%; height: 100%` 覆盖整个视口，`pointerEvents: "none"` 确保不阻止下层元素交互（鼠标事件通过 window 监听）。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-splash-cursor` 导入。
2. 组件为全屏 fixed 定位，无需额外容器样式，直接在页面根组件中使用。
3. 使用 WebGL 实现 Navier-Stokes 流体求解器，性能开销大，建议仅在创意/着陆页使用。
4. 通过 window 事件监听鼠标移动，`pointerEvents: "none"` 确保下层交互不受影响。
