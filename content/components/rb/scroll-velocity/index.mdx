ScrollVelocity 根据页面滚动速度驱动的无限滚动文字横幅组件，正反双向交替排列，滚动越快文字移动越快，基于 framer-motion 的 useVelocity 和 useSpring 实现。

<h2 id="when-to-use">何时使用</h2>
需要展示基于滚动速度动态变化的无限滚动文字横幅。
常见场景：
- 长页面的动态分隔条/装饰带
- 视差滚动的视觉增强元素
- 品牌展示页的速度感设计
不推荐：
- 静态页面中（无滚动则文字静止，效果不明显）
- 需要精确控制文字位置的布局场景

### 自定义文字和速度参数
```tsx
import { ScrollVelocity } from "@/components/ui/rb-scroll-velocity"

export default function Demo() {
  return (
    <ScrollVelocity
      texts={["FAST MOVING", "SCROLL VELOCITY", "REACT BITS"]}
      velocity={150}
      numCopies={8}
      damping={30}
      stiffness={300}
    />
  )
}
```

<h2 id="api">API</h2>
### ScrollVelocity
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| scrollContainerRef | 自定义滚动容器引用 | `React.RefObject<HTMLElement \| null>` | - |
| texts | 文字数组，每个元素渲染为一行，奇偶行反向运动 | `string[]` | `["Scroll Velocity","React Bits"]` |
| velocity | 基础滚动速度，值越大移动越快 | `number` | `100` |
| className | 每行文字 span 的 className | `string` | `""` |
| damping | 速度弹簧的阻尼系数，越小跟随越灵敏 | `number` | `50` |
| stiffness | 速度弹簧的刚度系数，越大跟随越灵敏 | `number` | `400` |
| numCopies | 每行文字的副本数量，确保无缝循环 | `number` | `6` |
| velocityMapping | 滚动速度到文字移动速度的映射范围 | `{ input: [number,number], output: [number,number] }` | `{ input: [0,1000], output: [0,5] }` |
| parallaxClassName | 视差容器 className | `string` | `"parallax"` |
| scrollerClassName | 滚动条 className | `string` | `"scroller"` |
| parallaxStyle | 视差容器内联样式 | `React.CSSProperties` | - |
| scrollerStyle | 滚动条内联样式 | `React.CSSProperties` | - |

<h2 id="faq">常见问题</h2>
### 文字出现断裂或空白间隙？
增加 `numCopies` 值。副本太少时，文字平铺不足以覆盖视口宽度加上偏移量，导致出现空白。推荐至少设置 6 个副本。

### 滚动方向反了怎么办？
奇偶行自动反向运动（奇数行正向、偶数行反向）。如果需要所有行同向，考虑只传入一个 text，或修改源码。设计上反向排列能营造更丰富的视差层次。

### 在自定义滚动容器中使用无效果？
确保 `scrollContainerRef` 正确指向目标容器元素，且容器内容高度超过容器本身以产生滚动。framer-motion 的 `useScroll` 会监听该容器的 `scrollY`。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/rb-scroll-velocity` 导入。
2. 依赖 `framer-motion`，确保项目已安装。
3. 文字行奇偶自动反向运动，传入多个 strings 可创建多层视差。
4. `numCopies` 至少设为 6，以避免无缝滚动出现空隙。
