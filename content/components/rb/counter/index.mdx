Counter 基于 Motion Spring 的数字滚动计数器，每位数字独立滚动，呈现类似老式翻页钟的 Odometer 效果。

<h2 id="when-to-use">何时使用</h2>
当需要展示数值并以逐位滚动动画呈现变化时使用。
常见场景：
- 统计数据大屏的数字展示
- 里程表或计数器风格 UI
- 游戏得分展示
不推荐：
- 需要精细控制动画时序的场景，组件内置了统一的 spring 动画
- 负数和极大数值的展示（仅支持正数）

<h2 id="api">API</h2>

### Counter

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 要展示的数值 | `number` | (必填) |
| fontSize | 字体大小（像素） | `number` | `100` |
| padding | 每位数字的内边距 | `number` | `0` |
| places | 自定义位数数组，如 `[100, 10, 1]` 表示百十个位，支持 `"."` 表示小数点 | `(number \| string)[]` | 按 value 自动推断 |
| gap | 位与位之间的间距 | `number` | `8` |
| borderRadius | 计数器容器的圆角半径 | `number` | `4` |
| horizontalPadding | 计数器容器的左右内边距 | `number` | `8` |
| textColor | 文字颜色 | `string` | `"inherit"` |
| fontWeight | 文字粗细 | `string` | `"inherit"` |
| containerStyle | 外层容器额外样式 | `React.CSSProperties` | - |
| counterStyle | 计数器内层额外样式，会与默认样式合并 | `React.CSSProperties` | - |
| digitStyle | 每位数字容器的额外样式 | `React.CSSProperties` | - |
| gradientHeight | 上下渐变遮罩的高度 | `number` | `16` |
| gradientFrom | 渐变遮罩的起始色 | `string` | `"black"` |
| gradientTo | 渐变遮罩的结束色 | `string` | `"transparent"` |
| topGradientStyle | 顶部渐变遮罩的完全自定义样式 | `React.CSSProperties` | - |
| bottomGradientStyle | 底部渐变遮罩的完全自定义样式 | `React.CSSProperties` | - |

<h2 id="faq">常见问题</h2>

### places 属性的作用是什么？
`places` 允许自定义数位结构。例如 `[1000, 100, 10, 1]` 表示千、百、十、个位，`[100, ".", 10, 1]` 则强制在十位和百位之间显示小数点。不传时组件自动从 `value` 推断。

### gradientHeight 和渐变遮罩的作用？
在数字上下两侧绘制渐变遮罩，模拟老式翻页钟的视觉深度效果。设置 `gradientHeight=0` 可完全去除。`topGradientStyle` 和 `bottomGradientStyle` 可完全自定义渐变样式。

### 支持负数吗？
源码中未对负数做特殊处理，`places` 自动推断不包含负号。如需展示负数，可通过 CSS `::before` 伪元素添加负号。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/counter` 导入。
2. `value` 为必填属性。
3. 组件依赖 `motion/react` 的 `useSpring` 和 `useTransform`。
4. 每位数字上下均有 0-9 的完整序列，通过 translateY 改变可见范围实现滚动效果，`overflow: hidden` 由外层容器控制。
