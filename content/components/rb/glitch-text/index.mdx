GlitchText 带故障抖动效果的文本展示组件，支持自动播放和悬停触发两种模式。

<h2 id="when-to-use">何时使用</h2>
当需要展示带故障艺术（Glitch Art）风格的文字效果时使用。
常见场景：
- Hero 区域的主标题，增强视觉冲击力
- 黑客/赛博朋克风格页面的文字装饰
- 活动页面的倒计时或促销文字
不推荐：
- 正文段落、长文本阅读场景，抖动效果会严重影响可读性
- 需要精准传达信息的表单标签或提示文字

### 持续动画模式
```tsx
import { GlitchText } from "@/components/ui/glitch-text-rb"

export default function Demo() {
  return (
    <GlitchText speed={0.5} enableOnHover={false} enableShadows={false}>
      ALWAYS GLITCHING
    </GlitchText>
  )
}
```

<h2 id="api">API</h2>
### GlitchText
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 显示的文本内容（必填） | `string` | - |
| speed | 动画速度倍率，值越小越快 | `number` | `1` |
| enableShadows | 是否启用红色/青色重影 | `boolean` | `true` |
| enableOnHover | 是否仅在悬停时触发故障效果 | `boolean` | `true` |
| className | 自定义样式类名 | `string` | `""` |

<h2 id="faq">常见问题</h2>
### 如何让故障效果始终播放而非仅悬停？
设置 `enableOnHover={false}`，组件将持续播放 clip-path 动画。

### 文字太长会换行吗？
组件默认 `whitespace-nowrap` 不换行。如需换行，通过 `className` 覆盖该样式。

### 能自定义故障颜色吗？
颜色通过 CSS 的 `text-shadow` 控制（`enableShadows` 启用时），红色（`red`）和青色（`cyan`）为内置颜色。如需自定义，通过 `className` 使用 CSS 变量覆盖 `::after` / `::before` 的 `text-shadow`。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/ui/glitch-text-rb` 导入。
2. `children` 是必填的 `string` 类型（非 `ReactNode`），用于设置显示文本和 `data-text` 属性。
3. 默认仅在 hover 时触发，设为 `enableOnHover={false}` 可持续播放。
4. 组件使用 `useId()` 生成唯一 CSS 动画名，支持同页多个实例。
