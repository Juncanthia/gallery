FeatureCalendar 功能日历。基于月视图的功能发布日历，支持多月切换、状态标记和条目展示，适用于发布计划和迭代排期。

<h2 id="when-to-use">何时使用</h2>

当需要以日历月视图展示带时间范围的功能项、发布计划或迭代排期时使用。

常见场景：

- 产品发布日历，按日期展示即将发布的功能。
- 迭代排期视图，标记每个功能的开始/结束日期。
- 项目里程碑跟踪，按状态颜色区分进度。

不推荐：

- 作为通用日期选择器使用——应使用 Calendar 或 DatePicker 组件。
- 展示大量条目（每月超过 10 条）——日历格子空间有限，仅显示前 3 条。
- 需要周视图或日视图的场景——本组件仅支持月视图。

<h2 id="examples">代码演示</h2>

### 基础用法

<ComponentShowcase id="feature-calendar" path="feature-calendar/basic" />

```tsx
import {
  CalendarProvider,
  CalendarHeader,
  CalendarBody,
  CalendarItem,
} from "@/components/ui/feature-calendar"

const features = [
  {
    id: "1",
    name: "首页改版",
    startAt: new Date(2025, 0, 6),
    endAt: new Date(2025, 0, 15),
    status: { id: "success", name: "已完成", color: "#22c55e" },
  },
  {
    id: "2",
    name: "API 优化",
    startAt: new Date(2025, 0, 10),
    endAt: new Date(2025, 0, 20),
    status: { id: "processing", name: "进行中", color: "#3b82f6" },
  },
]

export default function Demo() {
  return (
    <CalendarProvider>
      <CalendarHeader />
      <CalendarBody features={features}>
        {({ feature }) => <CalendarItem feature={feature} />}
      </CalendarBody>
    </CalendarProvider>
  )
}
```

### 带日期控制栏

通过 `CalendarDate`、`CalendarMonthPicker`、`CalendarYearPicker`、`CalendarDatePagination` 构建完整的日历控制栏。

```tsx
import {
  CalendarProvider,
  CalendarDate,
  CalendarDatePicker,
  CalendarMonthPicker,
  CalendarYearPicker,
  CalendarDatePagination,
  CalendarHeader,
  CalendarBody,
  CalendarItem,
} from "@/components/ui/feature-calendar"

export default function Demo() {
  return (
    <CalendarProvider locale="zh-CN" startDay={1}>
      <CalendarDate>
        <CalendarDatePicker>
          <CalendarMonthPicker />
          <CalendarYearPicker start={2020} end={2030} />
        </CalendarDatePicker>
        <CalendarDatePagination />
      </CalendarDate>
      <CalendarHeader />
      <CalendarBody features={features}>
        {({ feature }) => <CalendarItem feature={feature} />}
      </CalendarBody>
    </CalendarProvider>
  )
}
```

<h2 id="api">API</h2>

### CalendarProvider

日历上下文提供者，设置 locale 和起始星期。所有子组件必须在 CalendarProvider 内部使用。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| locale | 国际化语言环境 | `Intl.LocalesArgument` | `"en-US"` |
| startDay | 一周起始日（0=周日, 1=周一） | `number` | `0` |
| children | 子组件内容 | `ReactNode` | - |
| className | 附加 className | `string` | - |

### CalendarHeader

渲染星期标题行。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

### CalendarBody

日历主体，渲染日期网格和 feature 条目。通过 render props 模式渲染每个功能项。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| features | 要展示的功能项列表 | `Feature[]` | 必填 |
| children | render props，接收 `{ feature: Feature }` 返回 ReactNode | `(props: { feature: Feature }) => ReactNode` | 必填 |

### CalendarItem

单个 feature 条目的默认渲染组件（使用 React.memo 优化），显示状态色圆点和名称。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| feature | 功能项数据 | `Feature` | 必填 |
| className | 附加 className | `string` | - |

### CalendarMonthPicker

月份下拉选择器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

### CalendarYearPicker

年份下拉选择器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| start | 可选年份起始值 | `number` | 必填 |
| end | 可选年份结束值 | `number` | 必填 |
| className | 附加 className | `string` | - |

### CalendarDatePagination

月份前后翻页按钮。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

### CalendarDatePicker

日期选择器区域的布局容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |
| children | 子组件内容 | `ReactNode` | 必填 |

### CalendarDate

日历顶部栏的布局容器，通常放置月份/年份选择器和翻页按钮。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 子组件内容 | `ReactNode` | 必填 |

### Feature

功能项数据结构。

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| id | 唯一标识 | `string` |
| name | 功能名称 | `string` |
| startAt | 开始日期 | `Date` |
| endAt | 结束日期（决定在日历中展示的日期位置） | `Date` |
| status | 状态信息 | `Status` |

### Status

状态数据结构。

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| id | 状态标识 | `string` |
| name | 状态名称 | `string` |
| color | 状态颜色（CSS 颜色值） | `string` |

### Hooks

| Hook | 说明 | 返回值 |
| --- | --- | --- |
| `useCalendarMonth()` | 读写当前月份（Jotai atom） | `[CalendarState["month"], (m) => void]` |
| `useCalendarYear()` | 读写当前年份（Jotai atom） | `[CalendarState["year"], (y) => void]` |

### 工具函数

| 函数 | 说明 | 签名 |
| --- | --- | --- |
| `monthsForLocale` | 根据 locale 获取 12 个月份名称数组 | `(locale, format?) => string[]` |
| `daysForLocale` | 根据 locale 和起始日获取 7 天星期名称数组 | `(locale, startDay) => string[]` |

<h2 id="faq">常见问题</h2>

### Feature 在日历中的位置由哪个日期决定？

由 `endAt`（结束日期）决定。`CalendarBody` 使用 `isSameDay(feature.endAt, dateOfCell)` 匹配，而非 `startAt`。这意味着功能显示在其结束日期对应的格子里。

### 如何切换日历的起始星期（周一 vs 周日）？

设置 `CalendarProvider` 的 `startDay` 属性：`startDay={0}` 为周日开始，`startDay={1}` 为周一开始。选择器中的月份和星期名称会随 locale 自动调整。

### 如何自定义每个 feature 条目的外观？

`CalendarBody` 使用 render props 模式，不限于使用 `CalendarItem`，可在回调中返回任意自定义 JSX：

```tsx
<CalendarBody features={features}>
  {({ feature }) => (
    <div className="flex items-center gap-1">
      <span style={{ color: feature.status.color }}>●</span>
      <span className="text-xs">{feature.name}</span>
    </div>
  )}
</CalendarBody>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/feature-calendar` 按需导入所有子组件。
2. 所有子组件必须在 `CalendarProvider` 内部使用，否则无法获取 locale 和 startDay 上下文。
3. `CalendarBody` 必须同时传入 `features`（Feature 数组）和 `children`（render props 函数）。
4. Feature 的 `endAt` 决定它在日历中显示的日期位置，不是 `startAt`。
5. 每个 Feature 必须包含完整的 `status` 对象（`id`、`name`、`color`），颜色为 CSS 颜色字符串。
6. 年月状态通过 Jotai atom 全局共享，不要手动传递 month/year props 给子组件。
7. 不要引入 Ant Design 相关依赖。
