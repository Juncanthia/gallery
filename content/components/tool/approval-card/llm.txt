# ApprovalCard 审批卡片

用于展示审批类交互卡片，用户可查看审批详情并通过/驳回。支持 interactive（交互模式）和 receipt（回执模式，显示已作出的审批决定）两种展示形态。

本组件是项目内的本地基础能力，API 和行为以源码为准，不对应 Ant Design 的完整公共组件契约。

## 何时使用 {#when-to-use}

适合在需要审批交互的场景中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 请假、加班、报销等审批流程中展示审批详情，提供通过/驳回操作按钮。
- 已完成的审批以回执卡片形式展示审批结果。
- AI 助手生成需要用户确认的操作卡片。

不推荐：

- 纯粹的信息展示（无审批交互），应使用 Card 或 Descriptions 组件。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { ApprovalCard } from "@/components/ui/tool-approval-card"

export default function Demo() {
  return (
    <ApprovalCard
      id="demo"
      title="审批请假申请"
      description="张三申请 2025-07-03 至 2025-07-05 年假，共 3 天。"
      icon="calendar"
      metadata={[
        { key: "申请人", value: "张三" },
        { key: "部门", value: "技术部" },
        { key: "请假类型", value: "年假" },
      ]}
      confirmLabel="通过"
      cancelLabel="驳回"
      onConfirm={() => console.log("已通过")}
      onCancel={() => console.log("已驳回")}
    />
  )
}
```

### 回执模式（已作出审批决定）

```tsx
import { ApprovalCard } from "@/components/ui/tool-approval-card"

export default function Demo() {
  return (
    <ApprovalCard
      id="receipt-demo"
      title="请假申请已通过"
      choice="approved"
    />
  )
}
```

### 危险操作（destructive 变体）

```tsx
import { ApprovalCard } from "@/components/ui/tool-approval-card"

export default function Demo() {
  return (
    <ApprovalCard
      id="destructive-demo"
      title="确认删除项目"
      description="此操作不可撤销，将永久删除「CRM 系统重构」项目及所有关联数据。"
      icon="alert-triangle"
      variant="destructive"
      confirmLabel="确认删除"
      cancelLabel="取消"
      onConfirm={() => console.log("已删除")}
      onCancel={() => console.log("已取消")}
    />
  )
}
```

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface ApprovalCardProps extends SerializableApprovalCard {
  className?: string;
  onConfirm?: () => void | Promise<void>;
  onCancel?: () => void | Promise<void>;
}
```

```ts
interface SerializableApprovalCard {
  id: string;
  title: string;                    // 必填
  description?: string;
  icon?: string;                    // lucide-react 图标名（kebab-case）
  metadata?: MetadataItem[];
  variant?: "default" | "destructive";
  confirmLabel?: string;            // 默认 "Approve"
  cancelLabel?: string;             // 默认 "Deny"
  choice?: "approved" | "denied";   // 传入后进入回执模式
}
```

```ts
interface MetadataItem {
  key: string;
  value: string;
}
```

```ts
type ApprovalDecision = "approved" | "denied";
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `data-slot="approval-card"` — 审批卡片的语义标记。
- `data-tool-ui-id` — 组件实例的唯一标识。
- `data-receipt="true"` — 回执模式标记（仅当传入 choice 时存在）。
- `role="dialog"` — interactive 模式下使用 dialog 角色。
- `role="status"` — receipt 模式下使用 status 角色。
- `aria-labelledby` — 关联标题元素。
- `aria-describedby` — 关联描述元素（仅当有 description 时）。

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- variant 变体样式集中在组件文件内部，通过 `variant` prop 切换。
- 默认变体使用 `bg-primary/10 text-primary` 图标容器；destructive 变体使用 `bg-destructive/10 text-destructive` 图标容器。
- 主题颜色优先使用 `bg-card`、`text-foreground`、`text-muted-foreground`、`border`、`shadow-xs` 等项目级语义变量。
- receipt 模式使用 `motion-safe:animate-in` 动画效果（fade-in + blur-in-sm + zoom-in-95）。

直接依赖摘要：

- `lucide-react`（图标解析）
- `zod`（schema 校验）
- `@hyper/shadcn`（cn 工具函数、Separator 组件）

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 样式应优先通过组件 props（variant、className）表达。
3. `icon` 接收 kebab-case 格式的 lucide-react 图标名（如 `"calendar"`、`"alert-triangle"`），组件自动解析为对应图标组件。
4. 如果需要新增能力，应先修改该组件源码和相邻 schema 文件，再更新本 llm 文档。
5. 不要把第三方文档中的全量 API 视为本地已支持能力。

## 可访问性 {#accessibility}

- interactive 模式使用 `role="dialog"` + `aria-labelledby` + `aria-describedby` 标记为对话框角色。
- receipt 模式使用 `role="status"` + `aria-label` 标记为状态区域。
- 支持 `Escape` 键触发取消操作，按下 Escape 会调用 `onCancel` 回调。
- 操作按钮通过 ActionButtons 共享组件渲染，继承其可访问性约定。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 ApprovalCard 代码时，应遵守：

1. 从 `@/components/ui/tool-approval-card` 导入 `ApprovalCard`，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. `id` 和 `title` 为必填属性。
4. `icon` 属性传入 lucide-react 图标名称（kebab-case 格式，如 `"calendar"`、`"file-text"`、`"alert-triangle"`）。
5. `variant="destructive"` 用于危险操作场景，确认按钮会变为 destructive 样式。
6. `metadata` 为 `{ key: string; value: string }[]` 格式，用于展示审批关键信息。
7. 不传 `choice` 时显示操作按钮（交互模式），传入 `choice` 后显示审批结果回执（只读模式）。
8. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置。
9. 需要自定义样式时，优先使用组件暴露的 props；最后才使用 `className`。
10. 交互组件必须保留键盘和 ARIA 语义，不要移除 `role` 和 `aria-*` 属性。
11. 如果不确定某个 AntD API 是否存在，先检查 `src/components/tool/approval-card/` 源码，不要凭 AntD 记忆生成。
