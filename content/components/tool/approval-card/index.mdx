审批卡片 ApprovalCard。用于展示审批类交互卡片，用户可查看审批详情并通过/驳回。支持 interactive（交互模式）和 receipt（回执模式，显示已作出的审批决定）两种展示形态，通过 choice prop 控制。

<h2 id="when-to-use">何时使用</h2>

当需要在 AI 工作流或应用界面中展示审批操作卡片时使用。

常见场景：
- 请假、加班、报销等审批流程中展示审批详情，提供通过/驳回操作按钮。
- 已完成的审批以回执卡片形式展示审批结果（通过或拒绝）。
- AI 助手生成需要用户确认的操作卡片，如执行脚本、发送通知等。

不推荐：
- 纯粹的信息展示（无审批交互），应使用 Card 或 Descriptions 组件。
- 需要多步骤审批流程的场景，应结合 Steps 组件使用。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。

<h2 id="api">API</h2>

### ApprovalCard

审批卡片组件。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 卡片唯一标识 | `string` | - |
| title | 审批标题 | `string` | - |
| description | 审批描述/说明文本 | `string` | - |
| icon | 左侧图标名称（lucide-react 图标名，kebab-case） | `string` | - |
| metadata | 元数据键值对列表 | `{ key: string; value: string }[]` | - |
| variant | 视觉变体 | `"default"` \| `"destructive"` | `"default"` |
| confirmLabel | 确认按钮文本 | `string` | `"Approve"` |
| cancelLabel | 取消按钮文本 | `string` | `"Deny"` |
| choice | 审批决定（传入后进入回执模式） | `"approved"` \| `"denied"` | - |
| className | 附加样式类名 | `string` | - |
| onConfirm | 确认回调 | `() => void \| Promise<void>` | - |
| onCancel | 取消回调 | `() => void \| Promise<void>` | - |

<h2 id="faq">常见问题</h2>

### interactive 模式和 receipt 模式如何切换？

- 不传 `choice` 属性时为 interactive 模式，展示完整的审批卡片，包含标题、描述、图标、元数据和操作按钮。
- 传入 `choice` 属性后切换为 receipt 模式，展示简洁的回执卡片，显示审批结果（Approved/Denied）和标题，不再显示按钮。

### 如何自定义按钮文本？

通过 `confirmLabel` 和 `cancelLabel` 属性设置按钮文本，默认为 "Approve" 和 "Deny"。

```tsx
<ApprovalCard
  id="custom-labels"
  title="确认删除"
  description="此操作不可撤销"
  confirmLabel="确认删除"
  cancelLabel="取消"
  variant="destructive"
/>
```

### 如何支持键盘操作？

- interactive 模式下，按 `Escape` 键会触发 `onCancel` 回调。
- 组件使用 `role="dialog"` 标记，并通过 `aria-labelledby` 和 `aria-describedby` 提供可访问性支持。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/approval-card` 导入 `ApprovalCard`，不要从 `antd` 导入。
2. `id` 和 `title` 为必填属性。
3. `icon` 属性传入 lucide-react 图标名称（kebab-case 格式，如 `"calendar"`、`"file-text"`、`"alert-triangle"`），组件内部会自动解析。
4. `variant="destructive"` 用于危险操作（如删除确认），确认按钮会变为 destructive 样式。
5. `metadata` 为 `{ key: string; value: string }[]` 格式，用于展示审批关键信息（申请人、日期、类型等）。
6. 不传 `choice` 时显示操作按钮（可交互），传入 `choice` 后显示审批结果回执（只读）。
7. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider` 等配置。
8. 不要引入 Ant Design 相关依赖。
