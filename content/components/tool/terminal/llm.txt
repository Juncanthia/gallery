# Terminal 终端

Terminal 用于展示命令行执行结果。支持 stdout/stderr 双通道输出、ANSI 转义码彩色渲染、执行耗时格式化、退出码状态指示和输出复制功能。

## 导入路径

```tsx
import { Terminal } from "@/components/ui/tool-terminal"
```

## 组件属性

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | `string` | 是 | - | 组件唯一标识 |
| command | `string` | 是 | - | 执行的命令文本 |
| exitCode | `number` | 是 | - | 命令退出码（0 成功，非 0 红色） |
| stdout | `string` | 否 | - | 标准输出，支持 ANSI 转义码 |
| stderr | `string` | 否 | - | 标准错误，红色渲染 |
| durationMs | `number` | 否 | - | 执行耗时（毫秒） |
| cwd | `string` | 否 | - | 工作目录 |
| truncated | `boolean` | 否 | `false` | 输出是否被截断 |
| maxCollapsedLines | `number` | 否 | - | 折叠行数阈值 |
| className | `string` | 否 | - | 附加样式类名 |
| expanded | `boolean` | 否 | - | 受控展开状态 |
| defaultExpanded | `boolean` | 否 | `false` | 默认展开状态 |
| onExpandedChange | `(expanded: boolean) => void` | 否 | - | 展开状态回调 |

## 子组件

### Terminal.Header

终端头部，展示命令行（含 cwd 前缀）、退出码、耗时和复制按钮。

### Terminal.Output

终端输出区域，渲染 stdout（ANSI 彩色）和 stderr（红色），支持折叠和截断提示。

### Terminal.Empty

无输出占位，显示"No output"文本。

## 代码示例

### 基础用法

```tsx
import { Terminal } from "@/components/ui/tool-terminal"

export default function Demo() {
  return (
    <Terminal
      id="terminal-1"
      command="git log --oneline -5"
      cwd="~/project"
      exitCode={0}
      durationMs={342}
      stdout={"abc1234 feat: add terminal component\ndef5678 fix: resolve rendering bug"}
    />
  )
}
```

### 命令执行失败

```tsx
import { Terminal } from "@/components/ui/tool-terminal"

export default function Demo() {
  return (
    <Terminal
      id="terminal-error"
      command="npm run build"
      cwd="~/project"
      exitCode={1}
      durationMs={2847}
      stderr={"src/Terminal.tsx:42:11 - error TS2304: Cannot find name 'ansi'.\n\nFound 1 error."}
    />
  )
}
```

### 无输出

```tsx
<Terminal
  id="terminal-empty"
  command="touch README.md"
  cwd="~/project"
  exitCode={0}
/>
```

### 输出折叠

```tsx
<Terminal
  id="terminal-collapsed"
  command="cat package.json"
  cwd="~/project"
  exitCode={0}
  maxCollapsedLines={10}
  stdout={packageJsonContent}
/>
```

### 外部控制展开/折叠

```tsx
import { useState } from "react"

export default function Demo() {
  const [expanded, setExpanded] = useState(false)

  return (
    <Terminal
      id="terminal-controlled"
      command="find . -name '*.tsx'"
      cwd="~/project/src"
      exitCode={0}
      maxCollapsedLines={8}
      expanded={expanded}
      onExpandedChange={setExpanded}
      stdout={fileListOutput}
    />
  )
}
```

## 行为说明

- **退出码展示**：0 为正常色，非 0 为红色，始终显示在头部右侧
- **耗时格式化**：< 1000ms 显示 "Xms"，>= 1000ms 显示 "X.Xs"
- **折叠机制**：`maxCollapsedLines` 设置后，输出行数超过阈值时自动折叠为 200px 高度并显示渐隐遮罩，点击展开按钮查看全部
- **复制功能**：点击头部复制按钮将 stdout + stderr 拼接后复制到剪贴板，无输出时复制按钮禁用
- **ANSI 渲染**：stdout 和 stderr 中的 ANSI 转义码通过 `ansi-to-react` 自动渲染为彩色输出
- **截断标记**：`truncated` 为 true 时在输出底部显示 "Output truncated..." 提示

## LLM 生成约束

生成 Terminal 时必须遵守：

1. 导入路径 `@/components/ui/tool-terminal`
2. `id`、`command`、`exitCode` 为必填
3. stdout 和 stderr 至少提供一个才能展示输出区域
4. durationMs 单位为毫秒，组件自动格式化
5. 输出含 ANSI 转义码时无需手动处理，直接传入即可
6. 长输出建议设置 `maxCollapsedLines` 节省垂直空间
7. cwd 可选，传入后在命令前显示 `cwd$` 前缀
8. 不要引入 Ant Design 相关依赖
9. 不要生成 AntD 专属的 `classNames`、`styles`、ConfigProvider 等配置
