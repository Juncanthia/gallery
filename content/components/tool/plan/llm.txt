# Plan 计划

以卡片形式展示 AI 生成的执行计划，包含标题、描述、进度条和待办事项列表。每个待办项支持四种状态，带有展开描述、新品进场动画和完成庆祝动画。

支持两种形态：
- **Plan**：完整卡片形态，含外层 Card 包裹、标题、描述和进度条。
- **Plan.Compact**：紧凑模式，隐藏外层包裹和标题，仅保留待办列表和进度条。

## 导入路径

```tsx
import { Plan } from "@/components/ui/tool-plan"
```

## Props

### Plan / Plan.Compact

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | `string` | 是 | - | 组件唯一标识 |
| title | `string` | 是 | - | 计划标题 |
| description | `string` | 否 | - | 计划描述 |
| todos | `PlanTodo[]` | 是 | - | 待办事项数组（至少 1 项，id 不可重复） |
| maxVisibleTodos | `number` | 否 | `4` | 初始可见待办项数量，超出折叠 |
| className | `string` | 否 | - | 自定义类名 |

### PlanTodo

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | `string` | 是 | - | 待办项唯一标识 |
| label | `string` | 是 | - | 待办项标题 |
| status | `"pending" \| "in_progress" \| "completed" \| "cancelled"` | 是 | - | 待办项状态 |
| description | `string` | 否 | - | 待办项详情，设置后可点击展开 |

## 代码示例

### 基础用法

```tsx
import { Plan } from "@/components/ui/tool-plan"

<Plan
  id="demo"
  title="构建用户认证系统"
  todos={[
    { id: "1", label: "设计数据库模型", status: "completed" },
    { id: "2", label: "实现注册接口", status: "in_progress" },
    { id: "3", label: "实现登录接口", status: "pending" },
  ]}
/>
```


## 使用约束

1. 从 `@/components/ui/tool-plan` 导入 `Plan`，不要从 `antd` 或其他路径导入。
2. `id`、`title`、`todos` 为必填属性，`todos` 至少包含 1 个元素，且所有 `id` 唯一。
3. 每个 todo 必须设置 `status`：`"pending"` / `"in_progress"` / `"completed"` / `"cancelled"`。
4. 待办项超过 4 个时，建议设置 `maxVisibleTodos` 控制初始展示数量，避免列表过长。
5. 需要嵌入其他布局时使用 `Plan.Compact`，不建议通过 className 隐藏 Card 外壳。
6. 不要生成 AntD 专属的 `classNames`、`styles`、ConfigProvider 等配置。
7. 不要引入 Ant Design 相关依赖。
8. `todo.description` 为可选属性，仅在待办项需要展开详情时设置。
9. 进度由组件内部自动计算（completedCount / totalCount），不需要外部传入。
10. 本组件为 Tool UI，基于 shadcn/ui（Card、Accordion、Collapsible）封装，不依赖、不兼容 Ant Design。
