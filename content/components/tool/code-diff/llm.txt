# CodeDiff 代码对比

基于 @pierre/diffs 封装的代码差异展示组件。支持两种输入模式：
- **file-diff 模式**：传入 `oldCode` 和 `newCode`，组件自动计算差异。
- **patch 模式**：传入 unified diff 格式的 `patch` 字符串直接渲染。

视图模式支持 `unified`（单栏）和 `split`（左右并排），行号可控，超长内容可折叠。

## 导入路径

```tsx
import { CodeDiff } from "@/components/ui/tool-code-diff"
```

## Props

### CodeDiff / CodeDiff.Root

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | `string` | 是 | - | 组件唯一标识 |
| oldCode | `string` | 否 | - | 旧版本代码 |
| newCode | `string` | 否 | - | 新版本代码 |
| patch | `string` | 否 | - | unified diff patch 字符串 |
| language | `string` | 否 | `"text"` | 代码语言 |
| filename | `string` | 否 | - | 文件名 |
| lineNumbers | `"visible" \| "hidden"` | 否 | `"visible"` | 行号可见性 |
| diffStyle | `"unified" \| "split"` | 否 | `"unified"` | 视图样式 |
| maxCollapsedLines | `number` | 否 | - | 折叠阈值（行数） |
| className | `string` | 否 | - | 自定义类名 |
| expanded | `boolean` | 否 | - | 受控展开 |
| defaultExpanded | `boolean` | 否 | `false` | 默认展开 |
| onExpandedChange | `(v: boolean) => void` | 否 | - | 展开回调 |

### CodeDiff.Header

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| className | `string` | 否 | - | 自定义类名 |

### CodeDiff.Content

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| className | `string` | 否 | - | 自定义类名 |

### CodeDiff.CollapseToggle

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| className | `string` | 否 | - | 自定义类名 |

## 输入模式约束

- 必须提供 `patch` 或 `oldCode`/`newCode` 中的至少一个。
- `patch` 和 `oldCode`/`newCode` 不能同时提供，二选一。
- 未提供 `oldCode` 或 `newCode` 时，缺失端视为空文本。

## 代码示例

### 基础 file-diff

```tsx
import { CodeDiff } from "@/components/ui/tool-code-diff"

<CodeDiff
  id="demo"
  oldCode="const a = 1;"
  newCode="const a = 2;"
  language="typescript"
  filename="index.ts"
/>
```

### Patch 模式

```tsx
<CodeDiff
  id="demo-patch"
  patch={`--- a/index.ts
+++ b/index.ts
@@ -1 +1 @@
-const a = 1;
+const a = 2;`}
  language="typescript"
/>
```

### Split 视图

```tsx
<CodeDiff
  id="demo-split"
  oldCode="const x = 1;"
  newCode="const x = 2;"
  language="typescript"
  diffStyle="split"
/>
```

### 长文本折叠

```tsx
<CodeDiff
  id="demo-fold"
  oldCode={longOldCode}
  newCode={longNewCode}
  language="javascript"
  maxCollapsedLines={20}
/>
```

### Compound 模式

```tsx
<CodeDiff.Root id="demo" oldCode="..." newCode="..." language="typescript">
  <CodeDiff.Header />
  <CodeDiff.Content />
  <CodeDiff.CollapseToggle />
</CodeDiff.Root>
```

## 使用约束

1. 从 `@/components/ui/tool-code-diff` 导入，不要从 `antd` 或其他路径导入。
2. `id` 必须提供且唯一。
3. 不能在同一个组件实例上同时设置 `patch` 和 `oldCode`/`newCode`。
4. `language` 尽量指定实际值（如 `typescript`、`python`），以获得准确的语法高亮。
5. 默认使用 `<CodeDiff />` 简写形式即可获得完整 Header + Content + CollapseToggle。
6. 仅在需要自定义内部布局时使用 compound 模式（`<CodeDiff.Root>` + 子组件）。
7. 不要直接导入 `@pierre/diffs` 的 `PierreFileDiff` 或 `PierrePatchDiff`，始终通过本组件使用。
8. 本组件为 Tool UI，基于 `@pierre/diffs` 封装，不依赖、不兼容 Ant Design。
