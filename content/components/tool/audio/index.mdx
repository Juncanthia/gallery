Audio 音频播放组件。用于在对话中嵌入音频播放器，支持全宽（full）和紧凑（compact）两种展示形态，提供播放/暂停控制、进度条拖拽、封面图展示和来源标注。

<h2 id="when-to-use">何时使用</h2>

当 AI 需要在对话中展示音频内容时使用。Audio 提供统一的音频播放体验，支持封面图、标题、进度控制和外部链接跳转。

常见场景：

- AI 助手分享音乐、播客或有声书内容。
- 对话中嵌入来自第三方平台的音频（Spotify、Apple Music 等）。
- AI 生成或推荐的音频片段的预览播放。
- 需要统一播放器样式和交互行为的音频展示场景。

Audio 基于原生 &lt;audio&gt; 元素构建，通过 React Context 管理播放状态：

- 两种展示形态：`full`（默认，大卡片含封面和元数据）、`compact`（紧凑行内播放条）。
- 进度条拖拽控制，支持毫秒级精度跳转。
- 自动格式化时长为 `mm:ss` 格式。

### 基础用法

展示一个带封面图、标题、描述和来源信息的全宽音频卡片。

  id="tool/audio"
  path="tool/audio/basic"

```tsx
import { Audio } from "@/components/ui/audio"

export default function Demo() {
  return (
    &lt;Audio
      id="demo-audio"
      assetId="demo-audio-asset"
      src="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.mp3"
      title="SoundHelix Song 1"
      description="A sample audio track generated by SoundHelix."
      artwork="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.png"
      durationMs={240000}
      locale="zh-CN"
      source={{ label: "SoundHelix", url: "https://www.soundhelix.com" }}
    /&gt;
  )
}
```

### 紧凑模式

设置 `variant="compact"` 渲染为紧凑行内播放条，适合空间受限的场景。

```tsx
import { Audio } from "@/components/ui/audio"

export default function Demo() {
  return (
    &lt;Audio
      id="compact-audio"
      assetId="compact-audio-asset"
      src="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-2.mp3"
      variant="compact"
      title="Compact Player"
      description="A compact audio player with progress bar."
    /&gt;
  )
}
```

### 纯音频播放（无元数据）

仅展示播放器，不包含标题、描述和封面图。

```tsx
import { Audio } from "@/components/ui/audio"

export default function Demo() {
  return (
    &lt;Audio
      id="minimal-audio"
      assetId="minimal-audio-asset"
      src="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-3.mp3"
    /&gt;
  )
}
```

### 带封面图

通过 `artwork` 属性设置封面图，full 模式下封面图以 4:3 宽高比展示在卡片顶部。

```tsx
import { Audio } from "@/components/ui/audio"

export default function Demo() {
  return (
    &lt;Audio
      id="artwork-audio"
      assetId="artwork-audio-asset"
      src="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.mp3"
      title="Track with Artwork"
      description="Full player with album artwork."
      artwork="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.png"
    /&gt;
  )
}
```

## FAQ

### variant 两种形态分别适用什么场景？

- `full`（默认）：大卡片形态，封面图以 4:3 比例展示在顶部，适合需要突出视觉效果的场景，如音乐分享、专辑展示。
- `compact`：紧凑行内播放条，取消封面大图，将封面缩略图放在左侧，进度条内嵌在元数据下方，适合空间受限的场景，如列表中的音频预览。

### 如何监听播放事件？

通过 `onMediaEvent` 回调监听 `play`、`pause`、`mute`、`unmute` 四种播放事件：

```tsx
&lt;Audio
  onMediaEvent={(type) => { console.log("Media event:", type) }}
  ...
/&gt;
```

### 如何使用内部状态管理？

Audio 组件内部使用 `AudioProvider` Context 管理播放状态。如果你需要在外部控制播放，可以通过 `AudioProvider` 包裹你的组件并传入 `defaultState`：

```tsx
import { AudioProvider, Audio } from "@/components/ui/audio"

&lt;AudioProvider defaultState={{ playing: true, muted: false }}&gt;
  &lt;Audio id="controlled-audio" assetId="ca-1" src="https://example.com/song.mp3" /&gt;
&lt;/AudioProvider&gt;
```

### 进度条如何工作？

进度条基于原生 `Slider` 组件，步长为 0.1 秒。用户拖拽时组件进入 seeking 状态，避免 `timeupdate` 事件覆盖拖拽位置。松开后恢复 `timeupdate` 驱动。时长格式为 `m:ss`，不满足分钟时显示 `0:00`。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/audio` 导入 `Audio`。
2. `id`、`assetId`、`src` 为必填属性。
3. `src` 和 `artwork` 必须使用完整的 HTTPS URL。
4. `durationMs` 单位为毫秒，传入正整数。
5. `createdAt` 使用 ISO 8601 datetime 格式（如 `"2026-06-30T12:00:00.000Z"`）。
6. `source` 对象中的 `url` 和 `iconUrl` 需为合法 URL 字符串。
7. 不引入 Ant Design 或任何第三方音频播放器依赖。
8. 不要绕过 Audio 组件直接使用原生 &lt;audio&gt; 标签。
