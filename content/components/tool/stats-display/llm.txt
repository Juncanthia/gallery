# StatsDisplay 统计卡片

用于展示仪表盘、数据面板中的关键指标卡片，支持数字（含 compact 缩写）、货币、百分比四种数值格式，可选环比变化指示（delta）和迷你趋势图（sparkline）。

本组件是项目内的本地基础能力，API 和行为以源码为准，不对应 Ant Design 的完整公共组件契约。

## 何时使用 {#when-to-use}

适合在需要以卡片形式展示关键统计指标的场景中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 仪表盘或数据概览页面中展示核心业务指标（收入、用户数、订单量等）。
- 数据面板中以网格形式组织多个统计卡片。
- LLM 数据分析报告中以卡片形式呈现统计摘要。

不推荐：

- 单个统计指标直接占据整行时，卡片会变窄，如需更大展示面积应考虑其他布局方案。
- 需要复杂交互（点击展开明细、筛选联动等）的场景，StatsDisplay 仅做静态展示。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { StatsDisplay } from "@/components/ui/tool-stats-display"

export default function Demo() {
  return (
    <StatsDisplay
      id="stats-demo"
      title="Dashboard Overview"
      description="Key performance metrics for the current period"
      stats={[
        {
          key: "revenue",
          label: "Revenue",
          value: 128460,
          format: { kind: "currency", currency: "USD", decimals: 0 },
          diff: { value: 12.5, decimals: 1, label: "vs last month" },
          sparkline: {
            data: [120, 135, 128, 142, 138, 150, 145],
            color: "#10b981",
          },
        },
        {
          key: "users",
          label: "Active Users",
          value: 8472,
          format: { kind: "number", compact: true, decimals: 1 },
          diff: { value: -3.2, decimals: 1, label: "vs last week" },
          sparkline: {
            data: [8200, 8400, 8350, 8500, 8480, 8472],
            color: "#f59e0b",
          },
        },
        {
          key: "conversion",
          label: "Conversion Rate",
          value: 0.032,
          format: { kind: "percent", decimals: 1, basis: "fraction" },
          diff: { value: 0.5, decimals: 1, label: "vs last month" },
        },
        {
          key: "orders",
          label: "Total Orders",
          value: 1258,
          format: { kind: "number", decimals: 0 },
          diff: { value: 8.3, decimals: 1, label: "vs last month" },
        },
      ]}
    />
  )
}
```

### 环比方向自定义（upIsPositive）

```tsx
import { StatsDisplay } from "@/components/ui/tool-stats-display"

export default function Demo() {
  return (
    <StatsDisplay
      id="bounce-rate"
      title="Site Health"

        {

```

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface StatsDisplayProps extends SerializableStatsDisplay {
  className?: string;
  locale?: string;
}
```

```ts
interface SerializableStatsDisplay {
  id: string;
  role?: string;
  title?: string;
  description?: string;
  stats: StatItem[];           // 至少 1 项
}
```

```ts
interface StatItem {
  key: string;                 // 唯一键值
  label: string;               // 指标标签
  value: string | number;      // 指标数值
  format?: StatFormat;         // 数值格式化配置
  diff?: StatDiff;             // 环比变化指示
  sparkline?: StatSparkline;   // 迷你趋势图
}
```

```ts
type StatFormat =
  | { kind: "text" }
  | { kind: "number"; decimals?: number; compact?: boolean }
  | { kind: "currency"; currency: string; decimals?: number }
  | { kind: "percent"; decimals?: number; basis?: "fraction" | "unit" }
```

```ts
interface StatDiff {
  value: number;               // 变化数值（正值=上升）
  decimals?: number;           // 小数位数，默认 1
  upIsPositive?: boolean;      // 上升是否为正面，默认 true
  label?: string;              // 附加说明标签
}
```

```ts
interface StatSparkline {
  data: number[];              // 数据点数组，至少 2 个
  color?: string;              // 线条颜色
}
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `data-slot="stats-display"` — StatsDisplay 容器的语义标记。
- `data-tool-ui-id` — 组件实例的唯一标识。

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 组件使用 shadcn/ui Card primitive（Card、CardHeader、CardTitle、CardDescription、CardContent）。
- 多 stat 时使用 CSS Grid `repeat(auto-fit, minmax(220px, 1fr))` 自适应列数布局。
- 容器查询 `@container` 配合 `@[440px]` 断点调整内部 border 和 padding。
- dif 的颜色语义：`text-green-600 dark:text-green-400`（正向）、`text-red-600 dark:text-red-500`（负向）、`text-muted-foreground`（不变）。
- 单个 stat 时卡片限制 `max-w-sm`，数值放大至 `text-5xl`。
- Sparkline 使用本地 SVG 实现，叠加在卡片内容下方的 `inset-x-0 top-2 bottom-2` 绝对定位区域。
- 入场动画使用 Tailwind `animate-in fade-in slide-in-from-bottom` 配合 `animationDelay` 错开每项出现时间。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/tool/stats-display/stats-display.tsx` — StatsDisplay 主组件
- `src/components/tool/stats-display/sparkline.tsx` — Sparkline SVG 迷你趋势图
- `src/components/tool/stats-display/schema.ts` — Zod schema 与类型定义
- `src/components/ui/tool-stats-display.tsx` — 统一导出入口，`export { StatsDisplay } from "@/components/tool/stats-display/stats-display"`

直接依赖摘要：

- `zod`（schema 校验）
- shadcn/ui Card primitive（通过 adapter 重导出）

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 样式应优先通过组件 props 表达。
3. 如果需要新增能力，应先修改该组件源码和相邻 schema 文件，再更新本 llm 文档。
4. 不要把第三方文档中的全量 API 视为本地已支持能力。
5. `value` 为 `string` 类型时，`format` 配置会被忽略（直接展示原始文本）。
6. `percent` 格式的 `basis: "fraction"` 表示 value 为小数（如 0.032 → 3.2%），`basis: "unit"` 表示 value 已为百分比数值（如 3.2 → 3.20%）。

## 可访问性 {#accessibility}

- 外层容器使用 `<article>` 语义元素，标记为独立的内容区块。
- 数字格式化使用 `aria-label` 提供完整的语音友好版本（如货币金额的全称读法、百分比文本）。
- compact 数字的缩写部分（如 "K"、"M"）使用 `aria-hidden="true"` 隐藏，仅保留数字部分供屏幕阅读器读取。
- Sparkline SVG 使用 `aria-hidden="true"` 隐藏，纯装饰性元素。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 StatsDisplay 代码时，应遵守：

1. 从 `@/components/ui/tool-stats-display` 导入 `StatsDisplay`，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. `id` 和 `stats` 为必填属性，`stats` 数组至少包含 1 项。
4. 每个 `StatItem` 必须提供 `key`（唯一）、`label`（展示标签）和 `value`（数值）。
5. 需要格式化数值时使用 `format`，不要手动拼接货币符号或百分号。
6. 环比变化使用 `diff` 属性展示，不要在外部自行渲染 delta 标签。
7. 趋势图使用 `sparkline` 属性，不要自行在外部叠加 SVG 或 chart 组件。
8. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
9. `upIsPositive` 用于标记上升是否为正面指标：跳出率、错误率等应将此设为 `false`。
10. `percent` 格式默认 `basis: "fraction"`，即传入 0.032 显示为 3.2%；如果数据源本身已是百分比值 3.2，应设置 `basis: "unit"`。
11. `value` 支持 `string` 和 `number` 两种类型，传入 `string` 时 `format` 将被忽略（直接展示原始文本）。
12. 需要自定义样式时，优先使用组件暴露的 props；最后才使用 `className`。
13. 如果不确定某个 AntD API 是否存在，先检查 `src/components/tool/stats-display/` 源码，不要凭 AntD 记忆生成。
