# Citation 引用

用于展示网页、文档、API 等引用来源的摘要卡片，支持 default（完整卡片）、inline（内联芯片）、stacked（堆叠图标）三种展示形态。

本组件是项目内的本地基础能力，API 和行为以源码为准，不对应 Ant Design 的完整公共组件契约。

## 何时使用 {#when-to-use}

适合在需要引用卡片展示的场景中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- AI 对话或搜索结果中展示引用来源卡片。
- 文档参考列表中以卡片形式展示引用条目。
- 需要复用统一设计语言、状态样式和可访问性约定。

不推荐：

- 引用数量过多时直接平铺，应使用 CitationList 组件控制展示数量。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { Citation } from "@/components/ui/citation"

export default function Demo() {
  return (
    <Citation
      id="demo"
      href="https://example.com"
      title="Example Title"
      snippet="A brief description of the cited source."
      domain="example.com"
      type="webpage"
    />
  )
}
```

### Inline 内联模式

```tsx
import { Citation } from "@/components/ui/citation"

export default function Demo() {
  return (
    <Citation
      id="inline-demo"
      variant="inline"
      href="https://example.com"
      title="Inline Citation"
      domain="example.com"
      type="webpage"
    />
  )
}
```

### 使用 CitationList 管理引用列表

```tsx
import { CitationList } from "@/components/ui/citation-list"

export default function Demo() {
  return (
    <CitationList
      id="my-list"
      citations={[
        { id: "1", href: "https://example.com", title: "Source 1", domain: "example.com", type: "webpage" },
        { id: "2", href: "https://docs.example.com", title: "Source 2", domain: "docs.example.com", type: "document" },
        { id: "3", href: "https://api.example.com", title: "Source 3", domain: "api.example.com", type: "api" },
        { id: "4", href: "https://github.com/x/y", title: "Source 4", domain: "github.com", type: "code" },
        { id: "5", href: "https://news.example.com", title: "Source 5", domain: "news.example.com", type: "article" },
      ]}
      maxVisible={3}
      variant="default"
    />
  )
}
```

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface CitationProps extends SerializableCitation {
  variant?: CitationVariant;  // "default" | "inline" | "stacked"
  className?: string;
  onNavigate?: (href: string, citation: SerializableCitation) => void;
}
```

```ts
interface SerializableCitation {
  id: string;
  href: string;              // 绝对 URL
  title: string;
  snippet?: string;
  domain?: string;           // 不传则从 href 自动提取
  favicon?: string;          // 网站图标 URL
  author?: string;
  publishedAt?: string;      // ISO 8601 格式
  type?: CitationType;       // "webpage" | "document" | "article" | "api" | "code" | "other"
  locale?: string;           // 日期格式化语言，默认 "en-US"
}
```

```ts
interface CitationListProps {
  id: string;
  citations: SerializableCitation[];
  variant?: CitationVariant;
  maxVisible?: number;
  className?: string;
  onNavigate?: (href: string, citation: SerializableCitation) => void;
}
```

```ts
type CitationVariant = "default" | "inline" | "stacked";
```

```ts
type CitationType = "webpage" | "document" | "article" | "api" | "code" | "other";
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `data-slot="citation"` — 单个引用元素的语义标记。
- `data-slot="citation-list"` — 引用列表容器的语义标记。
- `data-tool-ui-id` — 组件实例的唯一标识。

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- variant 变体样式集中在组件文件内部，通过 `variant` prop 切换。
- 组件包装 Radix Popover primitive（inline/stacked 形态），交互状态、键盘行为和 ARIA 基础能力来自底层 primitive。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground`、`ring-ring`、`bg-muted` 等项目级语义变量。
- default 形态使用 `@container` 容器查询实现响应式布局。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/tool/citation/citation.tsx` — Citation 单引用组件
- `src/components/tool/citation/citation-list.tsx` — CitationList 引用列表组件
- `src/components/tool/citation/schema.ts` — Zod schema 与类型定义
- `src/components/ui/tool-citation.tsx` — 统一导出入口，`export { Citation } from "@/components/agent-tools/citation/citation"`
- `src/components/ui/tool-citation-list.tsx` — 统一导出入口，`export { CitationList } from "@/components/agent-tools/citation/citation-list"`

直接依赖摘要：

- `@radix-ui/react-popover`（inline/stacked 形态的 Popover）
- `lucide-react`（类型图标）
- `zod`（schema 校验）

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 样式应优先通过组件 props（variant、className）表达。
3. 如果需要新增能力，应先修改该组件源码和相邻 schema 文件，再更新本 llm 文档。
4. 不要把第三方文档中的全量 API 视为本地已支持能力。
5. `href` 属性会经过 `sanitizeHref` 安全校验，外部传入的 URL 会被规范化处理。

## 可访问性 {#accessibility}

- default 形态使用 `role="link"` + `tabIndex={0}` 标记为可聚焦的链接角色，支持键盘 Enter/Space 激活。
- inline 形态使用 `<button>` 元素，通过 `aria-label={title}` 提供可访问名称。
- 纯图标按钮（Popover 折叠状态下的 +N 按钮）使用明确的文本标签确保可访问性。
- Popover 内容区域在打开/关闭时自动管理焦点：`onOpenAutoFocus` preventDefault 避免焦点跳转，`onCloseAutoFocus` preventDefault 保持焦点稳定。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/tool/citation/` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 借鉴 AntD 使用心智，但不承诺 API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Citation / CitationList 代码时，应遵守：

1. 从 `@/components/ui/citation` 导入 `Citation`，从 `@/components/ui/citation-list` 导入 `CitationList`，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. `id`、`href`、`title` 为必填属性，`href` 必须为合法的绝对 URL（`https://` 开头）。
4. 使用 `variant` 控制展示形态，不要在外部通过 className 重新实现 variant 逻辑。
5. 多个引用项应使用 `CitationList` 组件统一管理，不要手动循环渲染 `Citation` 并自行实现折叠逻辑。
6. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
7. 需要自定义样式时，优先使用组件暴露的 props；最后才使用 `className`。
8. 交互组件必须保留键盘和 ARIA 语义，纯图标操作必须补 `aria-label`。
9. 如果不确定某个 AntD API 是否存在，先检查 `src/components/tool/citation/` 源码，不要凭 AntD 记忆生成。
10. `snippet` 建议控制在 150 字以内，超出部分组件会自动截断（line-clamp-3）。
