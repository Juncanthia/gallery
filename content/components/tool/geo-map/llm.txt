# GeoMap 地理地图

基于 Leaflet 封装的交互式地图组件，支持标记点、路线叠加、聚类和深色/浅色主题自适应。

本组件是项目内的本地基础能力，API 和行为以源码为准，不对应 Ant Design 的完整公共组件契约。

## 何时使用 {#when-to-use}

适合在需要以地图形式展示地理位置信息、路径轨迹或区域分布的场景中使用。

常见场景：

- 展示公司办公室、门店、仓库等位置的分布概览。
- 物流追踪场景中展示配送路线或运输路径。
- 数据分析场景中以地图标注数据点的地理分布。
- 行程规划中展示途经地点和路线规划。

不推荐：

- 在移动端小屏幕上展示过多标记点，建议开启 clustering 聚类功能。
- 将 GeoMap 用于精确导航——组件基于开放地图瓦片，不提供实时路况或导航功能。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 antd、@ant-design/icons、antd-style 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { GeoMap } from "@/components/ui/geo-map"

export default function Demo() {
  return (
    <GeoMap
      id="demo"
      title="办公室分布"
      description="全球主要办公地点"
      markers={[
        { id: "bj", lat: 39.9042, lng: 116.4074, label: "北京总部" },
        { id: "sh", lat: 31.2304, lng: 121.4737, label: "上海分部" },
      ]}
      viewport={{ mode: "fit", target: "markers" }}
    />
  )
}
```

## API 摘要 {#api}

以下类型来自本地源码 `src/components/tool/geo-map/schema.ts`，是生成代码时的主要依据：

```ts
interface GeoMapProps {
  id: string;                     // 组件唯一标识
  role?: string;                  // ARIA role
  receipt?: unknown;              // 收据/receipt
  title?: string;                 // 地图标题
  description?: string;           // 地图描述
  markers: GeoMapMarker[];        // 标记点数组，至少 1 项
  routes?: GeoMapRoute[];         // 路线数组
  clustering?: GeoMapClustering;  // 聚类配置
  viewport?: GeoMapViewport;      // 视口控制
  showZoomControl?: boolean;      // 是否显示缩放控件，默认 true
  theme?: "light" | "dark";       // 主题，不传则自动跟随系统
  className?: string;
  style?: GeoMapStyle;
  tooltipClassName?: string;
  popupClassName?: string;
  onMarkerClick?: (marker: GeoMapMarker) => void;
  onRouteClick?: (route: GeoMapRoute) => void;
}
```

```ts
interface GeoMapMarker {
  id?: string;                    // 标记 ID（同组件内不可重复）
  lat: number;                    // 纬度 -90 ~ 90
  lng: number;                    // 经度 -180 ~ 180
  label?: string;                 // 标记名称
  description?: string;           // 标记描述
  tooltip?: "none" | "hover" | "always";
  icon?: GeoMapMarkerIcon;        // 标记图标
}
```

```ts
type GeoMapMarkerIcon =
  | { type: "dot"; color?: string; borderColor?: string; radius?: number }   // radius: 3~16
  | { type: "emoji"; value: string; size?: number; bgColor?: string; borderColor?: string }  // size: 16~40
  | { type: "image"; url: string; width?: number; height?: number; borderRadius?: number; borderColor?: string }  // width/height: 16~64, url 必须 http/https
```

```ts
interface GeoMapRoute {
  id?: string;                    // 路线 ID（同组件内不可重复）
  points: { lat: number; lng: number }[];  // 至少 2 个点
  label?: string;
  description?: string;
  tooltip?: "none" | "hover" | "always";
  color?: string;                 // 路线颜色
  weight?: number;                // 线宽 1~12
  opacity?: number;               // 透明度 0~1
  dashArray?: string;            // 虚线样式，如 "8 4"
}
```

```ts
interface GeoMapClustering {
  enabled?: boolean;              // 是否启用聚类
  radius?: number;                // 聚类半径 20~120
  maxZoom?: number;               // 停止聚类的最大缩放 1~22
  minPoints?: number;             // 形成聚类的最小点数 2~20
}
```

```ts
type GeoMapViewport =
  | { mode: "fit"; padding?: number; maxZoom?: number; target?: "markers" | "routes" | "all" }
  | { mode: "center"; center: { lat: number; lng: number }; zoom: number }  // zoom: 1~22
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `data-slot="geo-map"` — 地图组件的语义标记。
- `data-slot="geo-map-loading"` — 加载状态占位符。
- `data-slot="geo-map-loading-label"` — 加载提示文本。
- `data-tool-ui-id` — 组件实例的唯一标识。

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 地图容器固定高度 320px，圆角边框，背景色跟随主题切换（light: muted, dark: background）。
- 标题面板使用绝对定位，带 backdrop-blur 毛玻璃效果。
- 主题色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground` 等项目级语义变量。
- 深色/浅色主题通过 CSS 变量 `--geo-map-canvas-bg` 和不同瓦片 URL 实现。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/tool/geo-map/geo-map.tsx` — GeoMap 主组件，含主题检测、标题面板、加载状态
- `src/components/tool/geo-map/geo-map-engine.tsx` — Leaflet 地图引擎，管理 map/markers/routes/clustering 生命周期
- `src/components/tool/geo-map/geo-map-overlays.tsx` — 标记和路线的 Leaflet overlay 组件
- `src/components/tool/geo-map/geo-map-icons.ts` — 标记图标渲染工具
- `src/components/tool/geo-map/schema.ts` — Zod schema 与 TypeScript 类型定义
- `src/components/tool/geo-map/geo-map-theme.module.css` — 主题 CSS Module
- `src/components/ui/tool-geo-map.tsx` — 统一导出入口

直接依赖摘要：

- `leaflet` + `react-leaflet` — 地图引擎
- `leaflet.markercluster` — 标记聚类插件
- `zod` — schema 校验

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 地图瓦片使用 CartoCDN 的 light_all / dark_all，不依赖第三方 API key。
3. 主题自动检测：优先读取 `[data-theme]` 和 `.dark` class，回退到 `prefers-color-scheme`。
4. 样式应优先通过组件 props（theme、className、style）表达。
5. 如果需要新增能力，应先修改该组件源码和相邻 schema 文件，再更新本 llm 文档。
6. 不要把第三方文档中的全量 API 视为本地已支持能力。

## 可访问性 {#accessibility}

- 地图容器使用 `role="region"` + `aria-label` 提供可访问名称。
- aria-label 组合 title 和 description 信息：`"{title}. {description}"`，缺失时回退为 `"Geographic map"`。
- 加载状态使用 `data-slot="geo-map-loading"` 语义标记，内嵌 `"Loading map..."` 提示文本。
- Leaflet 内置的缩放控件依赖原生按钮元素，键盘可操作。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/tool/geo-map/schema.ts` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 地图引擎 | 使用 Leaflet，非 Ant Design 地图组件 |
| 兼容目标 | 借鉴地图组件使用心智，但不承诺 API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 GeoMap 代码时，应遵守：

1. 从 `@/components/ui/geo-map` 导入 `GeoMap`，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. `id`、`markers` 为必填属性。`markers` 数组至少包含 1 项，每项必须含 `lat` 和 `lng`。
4. `lat` 范围 -90~90，`lng` 范围 -180~180，超出范围的经纬度会导致标记无法正确显示。
5. 使用 `viewport` 控制初始视图：`fit` 模式自适应内容，`center` 模式手动定位。推荐使用 `fit` 模式。
6. 同一组件内 `markers` 的 `id` 不可重复，`routes` 的 `id` 也不可重复。
7. `icon` 为 `image` 类型时，`url` 必须是合法的 http/https 地址。
8. 不要在外部包裹额外的 Card 或 Panel 组件，GeoMap 内部已集成标题面板和边框容器。
9. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
10. 需要自定义样式时，优先使用组件暴露的 props（`className`、`style`）；最后才覆写 CSS。
11. 交互组件必须保留键盘和 ARIA 语义。
12. 如果不确定某个 AntD API 是否存在，先检查 `src/components/tool/geo-map/` 源码，不要凭 AntD 记忆生成。
13. 不要在移动端小屏幕上展示过多标记点，建议开启 `clustering` 聚类功能。
14. `theme` 不传时自动跟随系统/文档主题，不需要手动判断或传递。
