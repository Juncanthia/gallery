# PreferencesPanel 偏好设置面板

PreferencesPanel 用于在 AI 对话界面中展示和收集用户偏好设置。支持 Switch 开关、Toggle 选项组和 Select 下拉选择三种控件类型，内置保存/取消操作按钮和已保存摘要视图（Receipt 模式）。

## 导入路径

```tsx
import { PreferencesPanel } from "@/components/ui/preferences-panel"
```

## 组件属性

### PreferencesPanel

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | `string` | 是 | - | 组件唯一标识 |
| title | `string` | 否 | - | 面板标题 |
| sections | `PreferenceSection[]` | 是 | - | 配置项分组数组 |
| value | `PreferencesValue` | 否 | - | 受控模式下的当前值 |
| onChange | `(value: PreferencesValue) => void` | 否 | - | 值变化回调 |
| actions | `ActionsProp` | 否 | `[{ id: "cancel", label: "Cancel", variant: "ghost" }, { id: "save", label: "Save Changes", variant: "default" }]` | 操作按钮配置 |
| onAction | `(actionId: string, value: PreferencesValue) => void` | 否 | - | 操作按钮点击回调 |
| onBeforeAction | `(actionId: string, value: PreferencesValue) => void` | 否 | - | 操作执行前回调 |
| className | `string` | 否 | - | 附加样式类名 |

### PreferencesPanel.Receipt

| 属性 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | `string` | 是 | - | 组件唯一标识 |
| title | `string` | 否 | - | 摘要标题 |
| sections | `PreferenceSection[]` | 是 | - | 配置项分组定义 |
| choice | `Record<string, string \| boolean>` | 是 | - | 用户最终选择的偏好值映射 |
| error | `Record<string, string>` | 否 | - | 错误信息映射 |
| className | `string` | 否 | - | 附加样式类名 |

### PreferenceSection

| 属性 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| heading | `string` | 否 | 分组标题，以 fieldset legend 渲染 |
| items | `PreferenceItem[]` | 是 | 该分组内的配置项列表 |

### PreferenceItem

| 属性 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | `string` | 是 | 配置项唯一标识，推荐英文小写短横线命名 |
| label | `string` | 是 | 配置项标签文本 |
| description | `string` | 否 | 配置项补充说明文字 |
| type | `"switch"` \| `"toggle"` \| `"select"` | 是 | 控件类型 |
| defaultChecked | `boolean` | 否 | type="switch" 时的默认选中状态，默认 false |
| options | `{ value: string; label: string }[]` | 否 | type="toggle" 时的选项列表，至少 2 个 |
| defaultValue | `string` | 否 | type="toggle" 时的默认值 |
| selectOptions | `{ value: string; label: string }[]` | 否 | type="select" 时的下拉选项，至少 5 个 |
| defaultSelected | `string` | 否 | type="select" 时的默认选中值 |

### PreferencesValue

```ts
type PreferencesValue = {
  [itemId: string]: string | boolean
}
```

## 子组件

### PreferencesPanel.Receipt

只读摘要视图，展示已确认的偏好配置，不包含交互控件。header 显示标题和 Saved/Error 状态指示，每个配置项以 label + 格式化后的值展示，有错误时显示红色错误信息和 AlertCircle 图标。

                { value: "light", label: "浅色" },

### Receipt 模式

```tsx
import { PreferencesPanel } from "@/components/ui/preferences-panel"

export default function Demo() {
  return (
    <PreferencesPanel.Receipt
      id="preferences-receipt"
      title="已保存的偏好设置"
      sections={[
        {
          heading: "通知渠道",
          items: [
            { id: "email-notifications", label: "邮件通知", type: "switch", defaultChecked: true },
            { id: "push-notifications", label: "推送通知", type: "switch", defaultChecked: false },
          ],
        },
        {
          heading: "外观",
          items: [
            {
              id: "theme", label: "主题", type: "toggle",
              options: [
                { value: "light", label: "浅色" },
                { value: "dark", label: "深色" },
                { value: "system", label: "跟随系统" },
              ],
            },
            {
              id: "language", label: "语言", type: "select",
              selectOptions: [
                { value: "zh-CN", label: "简体中文" },
                { value: "en-US", label: "English" },
              ],
            },
          ],
        },
      ]}
      choice={{
        "email-notifications": true,
        "push-notifications": false,
        theme: "dark",
        language: "zh-CN",
      }}
    />
  )
}
```

## 行为说明

- **初始值计算**：各组件的 `defaultChecked`、`defaultValue`、`defaultSelected` 定义初始值，作为判断是否 Dirty 的基准
- **Save 按钮状态**：当所有配置项的值与初始值一致时，Save 按钮自动禁用；任一值发生变化即启用
- **Cancel 行为**：点击 Cancel 将所有配置项恢复为初始值
- **受控模式**：传入 `value` + `onChange` 进入受控模式，父组件完全控制面板值；否则为内部状态管理
- **操作按钮对齐**：默认右对齐（`align: "right"`），可设置为 `"left"`
- **二次确认**：通过 `actions.confirmTimeout` 启用，单位毫秒，通常用于 destructive 操作
- **签名重置**：sections 结构变化时（如 key 变化），非受控模式下自动重置为初始值
- **Receipt 模式**：只读展示，Switch 显示 "On"/"Off"，Toggle/Select 显示对应 option 的 label 文本；有 error 时显示 AlertCircle 红色图标和错误信息
- **控件渲染**：Switch 以单行展示（标签在左，开关在右），Toggle 和 Select 在窄容器（< @sm）下标签和控件上下堆叠，宽容器并排展示
- **分组无标题处理**：无 heading 的 section 首个配置项移除顶部 padding，使其与上方内容间距紧凑

## LLM 生成约束

生成 PreferencesPanel 时必须遵守：

1. 导入路径 `@/components/ui/preferences-panel`
2. `id` 和 `sections` 为必填
3. sections 中每个分组的 items 至少包含一个配置项
4. 配置项 id 使用英文小写短横线命名（kebab-case）
5. Switch 配置项最少仅需 id、label、type 即可工作
6. Toggle 配置项 options 至少 2 项；Select 配置项 selectOptions 至少 5 项
7. Receipt 模式下 choice 必填，其 key 须与 sections 中所有配置项 id 完全一致
8. 不要在 sections 外单独渲染操作按钮，组件已内置
9. 避免单面板内配置项过多（建议不超过 8 个），防止面板过高
10. 不要引入 Ant Design 相关依赖
11. 不要生成 AntD 专属的 `classNames`、`styles`、ConfigProvider 等配置
