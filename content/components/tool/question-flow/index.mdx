QuestionFlow 用于向用户展示问题并收集选择答案，支持单步渐进、多步问卷和结果回执三种模式。

<h2 id="when-to-use">何时使用</h2>

向用户提出问题并收集答案，常见于 AI 对话界面中的交互式问答、表单引导、偏好收集等场景。

三种模式：

- **渐进模式（Progressive）**：单步问题，父组件控制步进逻辑。适合在 AI 流式对话中逐题插入。
- **多步模式（Upfront）**：预定义完整问卷，组件内部管理步进和答案收集，完成后回调 `onComplete`。
- **回执模式（Receipt）**：展示已完成选择的结果摘要，用于确认或展示用户已提交的答案。

<h2 id="examples">代码演示</h2>

### 基础用法（渐进模式）

单步问题，单选模式。通过 `step`、`title`、`options` 定义问题内容。选项支持 `label`、`description` 和 `disabled`。

  id="tool/question-flow"
  path="tool/question-flow/basic"
/>

```tsx
import { QuestionFlow } from "@/components/ui/tool-question-flow"

const options = [
  { id: "web", label: "Web 应用", description: "浏览器端运行的在线应用" },
  { id: "mobile", label: "移动应用", description: "iOS / Android 原生 App" },
  { id: "desktop", label: "桌面应用", description: "macOS / Windows / Linux 原生应用" },
  { id: "cli", label: "命令行工具", description: "终端 CLI 工具或脚本", disabled: true },
]

export default function Demo() {
  return (
    <QuestionFlow
      id="demo-question-flow"
      step={1}
      title="你想构建什么类型的项目？"
      description="选择项目类型以获取对应的模板推荐。"
      options={options}
      selectionMode="single"
    />
  )
}
```

### 多步问卷（Upfront 模式）

传入 `steps` 数组定义完整问卷。组件自动管理步进、返回和答案收集，完成后通过 `onComplete` 回调返回全部答案。

```tsx
import { QuestionFlow } from "@/components/ui/tool-question-flow"

const steps = [
  {
    id: "platform",
    title: "你的目标平台是什么？",
    options: [
      { id: "web", label: "Web" },
      { id: "mobile", label: "移动端" },
    ],
    selectionMode: "single",
  },
  {
    id: "tech",
    title: "你偏好哪些技术？",
    description: "可多选",
    options: [
      { id: "react", label: "React" },
      { id: "vue", label: "Vue" },
      { id: "svelte", label: "Svelte" },
      { id: "next", label: "Next.js" },
    ],
    selectionMode: "multi",
  },
  {
    id: "deploy",
    title: "你计划如何部署？",
    options: [
      { id: "vercel", label: "Vercel" },
      { id: "docker", label: "Docker" },
      { id: "vps", label: "自建 VPS" },
    ],
  },
]

export default function Demo() {
  return (
    <QuestionFlow
      id="multi-step-demo"
      steps={steps}
      onComplete={(answers) => console.log("Answers:", answers)}
    />
  )
}
```

### 回执模式（Receipt）

传入 `choice` 对象展示已完成选择的结果摘要。

```tsx
import { QuestionFlow } from "@/components/ui/tool-question-flow"

export default function Demo() {
  return (
    <QuestionFlow
      id="receipt-demo"
      choice={{
        title: "项目脚手架已生成",
        summary: [
          { label: "平台", value: "Web" },
          { label: "框架", value: "React + Next.js" },
          { label: "部署", value: "Vercel" },
        ],
      }}
    />
  )
}
```

<h2 id="api">API</h2>

### QuestionFlow

组件根据传入 props 自动识别模式：`choice` 存在时为回执模式，`steps` 存在时为多步模式，其余为渐进模式。

### QuestionFlowOption

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 选项唯一标识 | `string` | - |
| label | 选项文本 | `string` | - |
| description | 选项描述，显示在 label 下方 | `string` | - |
| icon | 选项图标（仅渐进模式可用） | `ReactNode` | - |
| disabled | 禁用该选项 | `boolean` | `false` |

### 渐进模式 Props（QuestionFlowProgressiveProps）

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 组件唯一标识 | `string` | - |
| step | 当前步骤序号 | `number` | - |
| title | 问题标题 | `string` | - |
| description | 问题描述 | `string` | - |
| options | 选项列表（可包含 icon） | `QuestionFlowOption[]` | - |
| selectionMode | 选择模式 | `single` \| `multi` | `single` |
| defaultValue | 默认选中项 ID 列表 | `string[]` | - |
| onSelect | 点击 Next 后的回调，接收选中 ID 列表 | `(optionIds: string[]) => void` | - |
| onBack | 返回按钮回调，存在时显示返回按钮 | `() => void` | - |
| className | 附加样式类名 | `string` | - |

### 多步模式 Props（QuestionFlowUpfrontProps）

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 组件唯一标识 | `string` | - |
| steps | 步骤定义列表 | `QuestionFlowStepDefinition[]` | - |
| onStepChange | 步骤切换回调，接收新步骤 ID | `(stepId: string) => void` | - |
| onComplete | 全部完成回调，接收所有答案 | `(answers: Record<string, string[]>) => void` | - |
| className | 附加样式类名 | `string` | - |

### QuestionFlowStepDefinition

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 步骤唯一标识 | `string` | - |
| title | 步骤标题 | `string` | - |
| description | 步骤描述 | `string` | - |
| options | 选项列表（不含 icon） | `Omit<QuestionFlowOption, "icon">[]` | - |
| selectionMode | 选择模式 | `single` \| `multi` | `single` |

### 回执模式 Props（QuestionFlowReceiptProps）

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 组件唯一标识 | `string` | - |
| choice | 结果数据 | `{ title: string; summary: { label: string; value: string }[] }` | - |
| className | 附加样式类名 | `string` | - |

<h2 id="faq">常见问题</h2>

### 渐进模式和多步模式如何选择？

- **渐进模式**：父组件完全控制步进逻辑，适合 AI 流式对话中按需插入问题。
- **多步模式**：预定义的固定问卷，组件内部管理步进和答案。

### 如何获取用户选择结果？

- 渐进模式：通过 `onSelect` 回调获取当前步骤的选中 ID 列表。
- 多步模式：通过 `onComplete` 回调获取所有步骤的答案（`Record<string, string[]>`）。

### 如何设置默认选中项？

渐进模式使用 `defaultValue` 传入默认选中的 ID 列表。

```tsx
<QuestionFlow
  id="default-demo"
  step={1}
  title="选择语言"
  options={options}
  defaultValue={["typescript"]}
/>
```

### 单步模式如何显示返回按钮？

同时传入 `step > 1` 和 `onBack`，组件自动显示返回按钮。

```tsx
<QuestionFlow
  id="back-demo"
  step={2}
  title="第二步"
  options={options}
  onBack={() => setStep(1)}
/>
```

### 多步模式的步骤如何跳转？

目前多步模式仅支持线性前进/返回，不支持任意跳转。如需复杂跳转逻辑，使用渐进模式由父组件控制。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 QuestionFlow 代码时：

1. 从 `@/components/ui/tool-question-flow` 导入 `QuestionFlow`
2. 渐进模式传入 `id`、`step`、`title`、`options`；多步模式传入 `id`、`steps`；回执模式传入 `id`、`choice`
3. 选项的 `id` 必须唯一，`label` 必填
4. 单选模式（默认）选中一个选项不会取消其它选择；多选模式需要显式设置 `selectionMode="multi"`
5. 多步模式不使用 `icon` 属性（多步模式会自动移除 icon 映射）
6. 不要同时传入 `steps` 和 `choice` 或 `step`，它们互斥
