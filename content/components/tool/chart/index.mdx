图表 Chart。基于 Recharts 封装的通用图表组件，支持 bar（柱状图）和 line（折线图）两种类型，提供标题、图例、网格线和数据点点击回调。

<h2 id="when-to-use">何时使用</h2>

当需要在界面中展示数据趋势、对比或分布时使用。Chart 接收结构化数据数组，通过 xKey 指定横轴字段，通过 series 指定纵轴系列。

常见场景：
- 仪表盘中展示销售额、流量、用户增长等统计数据的趋势图。
- 数据报告中以柱状图对比不同维度的指标。
- AI 分析结果中以折线图展示时间序列变化。

不推荐：
- 需要饼图、雷达图、散点图等非柱状/折线图类型 —— 本组件仅支持 bar 和 line。
- 数据量极大（10 万+数据点）需要高性能渲染的场景 —— Recharts DOM 渲染性能有限。
- 需要复杂交互（缩放、拖拽、框选）的高级可视化场景。

<h2 id="examples">代码演示</h2>

### 基础用法

传入 type、data、xKey、series 等必填属性，渲染一个柱状图。

<ComponentShowcase id="tool/chart" path="tool/chart/basic" />

```tsx
import { Chart } from "@/components/ui/chart-tool"

const data = [
  { month: "January", desktop: 186, mobile: 80 },
  { month: "February", desktop: 305, mobile: 200 },
  { month: "March", desktop: 237, mobile: 120 },
  { month: "April", desktop: 73, mobile: 190 },
  { month: "May", desktop: 209, mobile: 130 },
  { month: "June", desktop: 214, mobile: 140 },
]

export default function Demo() {
  return (
    <Chart
      id="chart-demo"
      type="bar"
      title="Monthly Revenue"
      description="January - June 2024"
      data={data}
      xKey="month"
      series={[
        { key: "desktop", label: "Desktop" },
        { key: "mobile", label: "Mobile" },
      ]}
      showLegend
    />
  )
}
```

### 折线图

设置 `type="line"` 渲染折线图，适用于时间序列趋势展示。

```tsx
import { Chart } from "@/components/ui/chart-tool"

const data = [
  { month: "Jan", revenue: 4000, cost: 2400 },
  { month: "Feb", revenue: 3000, cost: 1398 },
  { month: "Mar", revenue: 2000, cost: 9800 },
  { month: "Apr", revenue: 2780, cost: 3908 },
  { month: "May", revenue: 1890, cost: 4800 },
  { month: "Jun", revenue: 2390, cost: 3800 },
]

export default function Demo() {
  return (
    <Chart
      id="line-demo"
      type="line"
      title="Revenue vs Cost"
      description="H1 2024"
      data={data}
      xKey="month"
      series={[
        { key: "revenue", label: "Revenue" },
        { key: "cost", label: "Cost" },
      ]}
      showLegend
    />
  )
}
```

### 自定义颜色

通过 `colors` 数组按顺序为系列分配颜色，或通过 series 中的 `color` 属性为单个系列指定颜色。

```tsx
import { Chart } from "@/components/ui/chart-tool"

const data = [
  { name: "Page A", uv: 4000, pv: 2400, amt: 2000 },
  { name: "Page B", uv: 3000, pv: 1398, amt: 1000 },
  { name: "Page C", uv: 2000, pv: 9800, amt: 2780 },
]

export default function Demo() {
  return (
    <Chart
      id="color-demo"
      type="bar"
      title="Page Views"
      data={data}
      xKey="name"
      series={[
        { key: "uv", label: "UV", color: "#8884d8" },
        { key: "pv", label: "PV", color: "#82ca9d" },
        { key: "amt", label: "Amt", color: "#ffc658" },
      ]}
      showLegend
    />
  )
}
```

### 仅图表（无标题）


<h2 id="api">API</h2>

### Chart

图表组件。接收必填的 data/series 和可选的标题、颜色、交互回调等。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 图表唯一标识 | `string` | - |
| type | 图表类型 | `"bar"` \| `"line"` | - |
| title | 图表标题，显示在卡片头部 | `string` | - |
| description | 图表描述，显示在标题下方 | `string` | - |
| data | 图表数据数组，每行为一个数据点 | `Record&lt;string, unknown&gt;[]` | - |
| xKey | 横轴对应的 data 中的字段名 | `string` | - |
| series | 纵轴系列数组，每项包含 key/label/color | `{ key: string; label: string; color?: string }[]` | - |
| colors | 颜色调色板，按顺序分配给系列；series.color 优先 | `string[]` | 项目 CSS 变量 |
| showLegend | 是否显示图例 | `boolean` | `false` |
| showGrid | 是否显示网格线 | `boolean` | `true` |
| className | 附加样式类名 | `string` | - |
| onDataPointClick | 数据点点击回调 | `(point: ChartDataPoint) =&gt; void` | - |

### ChartDataPoint

`onDataPointClick` 回调的参数类型。

| 字段 | 说明 | 类型 |
| --- | --- | --- |
| seriesKey | 被点击系列的 key | `string` |
| seriesLabel | 被点击系列的 label | `string` |
| xValue | 数据点的 x 轴值 | `unknown` |
| yValue | 数据点的 y 轴值 | `unknown` |
| index | 数据点在数组中的索引 | `number` |
| payload | 数据点完整数据 | `Record&lt;string, unknown&gt;` |

### ChartSeries

系列配置项类型。

| 字段 | 说明 | 类型 |
| --- | --- | --- |
| key | 系列唯一标识，对应 data 行中的字段名 | `string` |
| label | 系列显示名称，用于图例和 tooltip | `string` |
| color | 系列颜色，优先于 colors 调色板 | `string`（可选） |

<h2 id="faq">常见问题</h2>

### data 数据格式有什么要求？

`data` 是一个对象数组，每个对象代表图表中的一个数据点。每个对象必须包含 `xKey` 指定的字段（值为 string 或 number），以及每个 series.key 指定的字段（值为 number 或 null）。null 值表示该数据点该系列无数据。

### 如何自定义图标颜色？

有两种方式：
1. 通过 `colors` 属性传入颜色数组，按顺序分配给 series。
2. 在 series 数组中为每个系列单独指定 `color` 属性，优先级高于 `colors`。
3. 默认颜色来自项目 CSS 变量 `--chart-1` 到 `--chart-5`。

### 如何隐藏网格线？

设置 `showGrid={false}` 即可隐藏 CartesianGrid。

### 图表数据为空时如何处理？

`data` 属性必须包含至少 1 条数据（Zod 校验 `.min(1)`）。如果数据可能为空，请在调用侧自行处理空状态展示，不要在 data 为空时渲染 Chart 组件。

### series.key 可以重复吗？

不可以。每个 series.key 必须在同一图表中唯一，Schema 层会校验重复 key 并抛出错误。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/chart-tool` 导入 `Chart`。
2. `id`、`type`、`data`、`xKey`、`series` 为必填属性。
3. `type` 仅支持 `"bar"` 和 `"line"` 两种，不要生成其他图表类型。
4. `data` 数组每行必须包含 `xKey` 和所有 `series.key` 对应字段，y 值必须为 number 或 null。
5. 不要生成 AntD 专属的 `classNames`、`styles`、ConfigProvider 等配置。
6. 不要引入 Ant Design 相关依赖。
7. 不要在外部包裹额外的 Card 组件，Chart 内部已集成 Card 结构。
8. series 中每个 key 必须唯一，不可重复。
