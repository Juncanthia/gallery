CitationList 用于展示 AI 引用的来源列表，支持三种展示变体（默认、内联、堆叠）、多类型图标、溢出折叠和悬停预览。

<h2 id="when-to-use">何时使用</h2>

在 AI 对话界面中展示模型回答所引用的来源链接，帮助用户验证信息的可靠性。

常见场景：

- AI 回答末尾展示引用的网页、文档、论文等来源。
- 聊天消息中以内联方式嵌入引用标记。
- 以紧凑的堆叠头像方式呈现多个来源。

CitationList 通过 `variant` 表达展示形式：

- `default`：纵向列表，每个引用展开完整的标题、摘要、域名等信息。
- `inline`：横向排列的标签式引用，适合嵌入对话文本中。
- `stacked`：堆叠图标的紧凑展示，点击弹出完整列表。

<h2 id="api">API</h2>

### CitationList

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 唯一标识符，用于 data-tool-ui-id 属性 | `string` | 必填 |
| citations | 引用数据数组 | `SerializableCitation[]` | 必填 |
| variant | 展示变体 | `default` \| `inline` \| `stacked` | `default` |
| maxVisible | 最大可见引用数；超出部分折叠到溢出面板 | `number` | - |
| className | 附加样式类名 | `string` | - |
| onNavigate | 点击引用时的自定义导航回调 | `(href: string, citation: SerializableCitation) => void` | - |

### SerializableCitation

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 引用唯一标识 | `string` | 必填 |
| href | 引用链接地址，必须是合法 URL | `string` | 必填 |
| title | 引用标题，溢出面板中可截断展示 | `string` | 必填 |
| snippet | 引用摘要，仅在 default 变体中展示 | `string` | - |
| domain | 来源域名，溢出面板和 default 变体中展示 | `string` | - |
| favicon | 网站图标 URL，存在时替代类型图标 | `string` | - |
| author | 作者信息 | `string` | - |
| publishedAt | 发布日期，ISO 8601 格式 | `string` | - |
| type | 引用类型，决定默认图标 | `webpage` \| `document` \| `article` \| `api` \| `code` \| `other` | `webpage` |
| locale | 语言区域 | `string` | - |

### 引用类型与图标映射

| type | 图标 | 用途 |
| --- | --- | --- |
| webpage | Globe | 通用网页 |
| document | FileText | 文档、手册 |
| article | Newspaper | 论文、文章 |
| api | Database | API 文档、数据源 |
| code | Code2 | 代码仓库、代码片段 |
| other | File | 其他来源 |

<h2 id="faq">常见问题</h2>

### 如何控制可见引用数量？

使用 `maxVisible` 属性。超出数量的引用会折叠到溢出面板中，hover 或 focus 时弹出展示：

```tsx
<CitationList
  id="citations"
  citations={manyCitations}
  maxVisible={3}
/>
```

### 如何在内联文本中使用引用？

设置 `variant="inline"`，引用会以水平排列的标签形式展示，适合嵌入对话回复中：

```tsx
<CitationList
  id="inline-citations"
  citations={citations}
  variant="inline"
/>
```

### 如何自定义点击行为？

传入 `onNavigate` 回调，可拦截默认的链接跳转行为，实现路由导航或埋点：

```tsx
<CitationList
  id="citations"
  citations={citations}
  onNavigate={(href, citation) => {
    console.log("navigate to", href, citation.title)
    window.open(href, "_blank")
  }}
/>
```

### variant 之间如何选择？

- `default`：适合独立的来源列表区域，信息展示最完整。
- `inline`：适合嵌入到对话文本流中，紧凑不打断阅读。
- `stacked`：适合空间极为有限时，以图标叠加方式呈现，hover 展开详情。

## 与 Ant Design 的差异

CitationList 为本项目 Tool UI 体系中的独立组件，Ant Design 中无直接对应组件。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 CitationList 代码时：

1. 从 `@/components/ui/citation-list` 导入 `CitationList`
2. `citations` 数组中每项的 `id` 和 `href` 为必填字段
3. `type` 字段建议明确指定，以便渲染正确的类型图标
4. 在对话消息中使用时优先选择 `variant="inline"`
5. 大量引用时使用 `maxVisible` 控制折叠阈值，避免页面过长
6. 不要引入 Ant Design 相关依赖
