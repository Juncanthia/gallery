# WeatherWidget 天气组件

以卡片形式展示指定城市的当前天气和未来预报，包含天气动画效果、温度信息、城市名称和逐日预报。

本组件是项目内的本地能力，API 和行为以源码为准，不对应 Ant Design 的完整公共组件契约。

## 何时使用 {#when-to-use}

当需要在界面中展示天气信息时使用，提供沉浸式天气卡片，支持天气动画特效和明暗主题自适应。

常见场景：

- AI 对话中展示查询到的天气结果，以可视化卡片替代纯文本回复。
- 仪表盘或工具面板中集成天气信息模块。
- 旅行规划、日程管理等场景中嵌入目的地天气预览。

不推荐：

- 需要展示逐小时详细天气数据（如小时级降水概率、紫外线指数）的场景 —— 组件仅支持逐日预报。
- 需要用户交互选择城市或切换温度单位的场景 —— 组件为只读展示，数据由外部传入。

## 代码演示 {#examples}

### 基础用法

```tsx
import { WeatherWidget } from "@/components/ui/weather-widget"

export default function Demo() {
  return (
    <WeatherWidget
      version="3.1"
      id="weather-demo"
      location={{ name: "北京" }}
      units={{ temperature: "celsius" }}
      current={{
        conditionCode: "partly-cloudy",
        temperature: 22,
        tempMin: 18,
        tempMax: 26,
        windSpeed: 12,
        precipitationLevel: "none",
        visibility: 10,
      }}
      forecast={[
        { label: "周一", conditionCode: "clear", tempMin: 19, tempMax: 27 },
        { label: "周二", conditionCode: "partly-cloudy", tempMin: 20, tempMax: 25 },
        { label: "周三", conditionCode: "rain", tempMin: 17, tempMax: 22 },
        { label: "周四", conditionCode: "cloudy", tempMin: 16, tempMax: 23 },
        { label: "周五", conditionCode: "clear", tempMin: 18, tempMax: 28 },
      ]}
      time={{ timeBucket: 5 }}
      updatedAt="2026-06-30T14:00:00Z"
      effects={{ enabled: true, quality: "medium" }}
    />
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。复杂组件应结合下方 API 摘要和源码类型补充必需 props。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface WeatherWidgetProps {
  version: "3.1";
  id: string;
  location: { name: string };
  units: {
    temperature: TemperatureUnit;
  };
  current: WeatherWidgetCurrent;
  forecast: ForecastDay[];
  time: WeatherWidgetTime;
  updatedAt?: string;
  className?: string;
  effects?: EffectSettings;
}
```

```ts
interface WeatherWidgetCurrent {
  conditionCode: WeatherConditionCode;
  temperature: number;
  tempMin: number;
  tempMax: number;
  windSpeed?: number;
  precipitationLevel?: PrecipitationLevel;
  visibility?: number;
}
```

```ts
interface ForecastDay {
  label: string;
  conditionCode: WeatherConditionCode;
  tempMin: number;
  tempMax: number;
}
```

```ts
interface WeatherWidgetTime {
  timeBucket?: number;     // 0-11，映射到一天中不同时段
  localTimeOfDay?: number;  // 0-24 的小时数
}
```

```ts
interface EffectSettings {
  enabled?: boolean;
  quality?: "low" | "medium" | "high" | "auto";
  reducedMotion?: boolean;
}
```

```ts
type WeatherConditionCode =
  | "clear"
  | "partly-cloudy"
  | "cloudy"
  | "overcast"
  | "fog"
  | "drizzle"
  | "rain"
  | "heavy-rain"
  | "thunderstorm"
  | "snow"
  | "sleet"
  | "hail"
  | "windy";

type TemperatureUnit = "celsius" | "fahrenheit";

type PrecipitationLevel = "none" | "light" | "moderate" | "heavy";
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `data-slot="weather-widget"` — 天气组件的最外层语义标记。
- `data-tool-ui-id` — 天气实例的唯一标识（对应 id prop）。
- `data-slot="card"` — 天气卡片内容区域的语义标记。

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 组件内部使用 `@container/weather` 容器查询控制响应式布局。
- 卡片宽高比为 4:3（`aspect-[4/3]`），最大宽度 `max-w-md`。
- 明暗主题自动切换：夜间/恶劣天气使用深色渐变 `from-zinc-950 via-zinc-900/70 to-zinc-950`，白天使用浅色渐变 `from-sky-50 via-sky-100/70 to-white`。
- 天气动画效果由 `EffectCompositorRuntime` 渲染，受 `conditionCode`、`windSpeed`、`precipitationLevel`、`visibility` 和 `timeOfDay` 参数驱动。
- 动画自动响应系统 `prefers-reduced-motion` 媒体查询偏好。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/tool/weather-widget/weather-widget-container.tsx` — WeatherWidget 主容器组件
- `src/components/tool/weather-widget/weather-data-overlay.tsx` — 天气数据叠加层组件
- `src/components/tool/weather-widget/schema-runtime.ts` — 运行时类型定义
- `src/components/tool/weather-widget/runtime.ts` — 模块导出入口（WeatherWidget + 所有类型）
- `src/components/tool/weather-widget/generated/weather-runtime-core.generated.ts` — 自动生成的运行时核心（EffectCompositorRuntime、天气效果逻辑）
- `src/components/ui/tool-weather-widget.tsx` — 统一导出入口

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 样式应优先通过组件 props 表达，最后才使用 `className`。
3. 如果需要新增能力，应先修改该组件源码和相邻 schema 文件，再更新本 llm 文档。
4. 不要把第三方文档中的全量 API 视为本地已支持能力。
5. `temperature`、`tempMin`、`tempMax` 数值需与 `units.temperature` 匹配，组件不自动转换。
6. `conditionCode` 仅支持本地定义的 13 种天气代码。

## 可访问性 {#accessibility}

- 组件最外层使用 `<article>` 语义标签。
- 动画效果自动响应系统 `prefers-reduced-motion` 媒体查询偏好，用户开启减少动效时自动禁用动画。
- 用户可通过 `effects.reducedMotion` 属性显式覆盖系统偏好。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 WeatherWidget 代码时，应遵守：

1. 从 `@/components/ui/weather-widget` 导入 `WeatherWidget`。
2. `version`、`id`、`location`、`units`、`current`、`forecast`、`time` 为必填属性。
3. `conditionCode` 仅支持 `"clear"`、`"partly-cloudy"`、`"cloudy"`、`"overcast"`、`"fog"`、`"drizzle"`、`"rain"`、`"heavy-rain"`、`"thunderstorm"`、`"snow"`、`"sleet"`、`"hail"`、`"windy"` 这 13 种值。
4. `temperature`、`tempMin`、`tempMax` 数值需与 `units.temperature` 匹配（摄氏用 0-50 范围，华氏用 32-122 范围）。
5. `forecast` 至少包含 1 项，建议 3-7 天以获得最佳展示效果。
6. `timeBucket` 为 0-11 的整数，常用值：0=午夜、3=清晨、5-6=正午前后、9=傍晚。
7. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置。
8. 不要在外部包裹额外的 Card 组件，WeatherWidget 内部已集成完整的卡片结构。
9. 若收到外部 API 返回的天气代码，需映射为组件支持的 `WeatherConditionCode` 后再传入。
10. 如果不确定某个 API 是否存在，先检查 `src/components/tool/weather-widget/` 源码，不要凭记忆生成。
