动效提示 TooltipMotion。带布局动画和弹性动效的提示组件，进出场伴随位移/缩放过渡，内容变更时自动触发布局变形。基于 `@/components/_primitives/animate/tooltip` 构建，叠加 `motion/react-client` 的 layout 动画能力。

<h2 id="when-to-use">何时使用</h2>

当需要提示信息并期望提示框具有流畅的进出场动效和内容布局过渡时使用。

常见场景：
- 需要提示框出现/消失时伴随弹性缩放和位移动画，提升交互感知。
- Tooltip 内容动态变化（如切换文案、加载状态）时需要平滑布局过渡。
- 需要和本项目已有的动画提示体系保持一致，共享 `TooltipProvider` 的延迟和过渡配置。

不推荐：
- 纯静态提示且不在意动效开销——使用 `@/components/ui/tooltip` 基础版。
- 在性能敏感的大批量渲染场景中频繁触发动画。
- 需要自定义颜色或箭头的提示——本组件聚焦动效能力，颜色定制应使用基础版 Tooltip。

### 基础用法

```tsx
import { Tooltip, TooltipContent, TooltipProvider, TooltipTrigger } from "@/components/ui/tooltip-motion"
import { Button } from "@/components/ui/button"

export default function Demo() {
  return (
    <TooltipProvider>
      <div className="flex items-center gap-4">
        <Tooltip>
          <TooltipTrigger asChild>
            <Button variant="outlined">悬停查看</Button>
          </TooltipTrigger>

          </TooltipContent>
        </Tooltip>
      </div>
    </TooltipProvider>
  )
}
```

### 内容动态切换

Tooltip 内容变化时自动触发布局变形动画，无需额外配置。

```tsx
import { useState } from "react"
import { Tooltip, TooltipContent, TooltipProvider, TooltipTrigger } from "@/components/ui/tooltip-motion"
import { Button } from "@/components/ui/button"

export default function Demo() {
  const [count, setCount] = useState(0)

  return (
    <TooltipProvider>
      <Tooltip>
        <TooltipTrigger asChild>
          <Button variant="outlined" onClick={() => setCount(c => c + 1)}>
            计数: {count}
          </Button>
        </TooltipTrigger>
        <TooltipContent>
          <p>当前点击次数：{count}</p>
          {count > 5 && <p className="text-xs opacity-70">点击过多啦！</p>}
        </TooltipContent>
      </Tooltip>
    </TooltipProvider>
  )
}
```

<h2 id="api">API</h2>

### TooltipProvider

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| openDelay | 打开延迟（毫秒） | `number` | `0` |
| closeDelay | 关闭延迟（毫秒） | `number` | `300` |
| transition | motion 过渡配置对象 | `Transition` | spring 配置 |
| id | 可选 ID，用于 layoutId 去重 | `string` | - |

`TooltipProvider` 必须包裹在所有 Tooltip 组件的最外层，推荐在应用根布局中放置。

### Tooltip

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| side | 弹出方向 | `'top' \| 'bottom' \| 'left' \| 'right'` | `'top'` |
| sideOffset | 距离触发器的偏移量（像素） | `number` | `10` |
| align | 对齐方式 | `'start' \| 'center' \| 'end'` | `'center'` |
| alignOffset | 交叉轴偏移量 | `number` | `0` |
| children | 内部放置 TooltipTrigger 和 TooltipContent | `ReactNode` | - |

### TooltipTrigger

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 是否将子元素作为触发器 | `boolean` | `false` |
| children | 触发器内容 | `ReactNode` | - |

### TooltipContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| layout | layout 动画模式 | `boolean \| 'position' \| 'size' \| 'preserve-aspect'` | `'preserve-aspect'` |
| children | 提示内容 | `ReactNode` | - |
| className | 自定义样式 | `string` | - |

注意：`TooltipContent` 不支持 `asChild`，不要尝试传入该 prop。内容始终渲染为独立的 `motion.div` 容器，箭头由组件自动渲染。

<h2 id="faq">常见问题</h2>

### TooltipMotion 和基础 Tooltip 的区别？
TooltipMotion 在基础气泡提示之上增加了 motion 布局动画能力：进出场伴随弹性缩放效果，内容动态变化时自动触发布局变形。基础版 Tooltip 无动画。如果不需要动效，使用 `@/components/ui/tooltip` 以减少依赖开销。

### 如何自定义动效参数？
在 `TooltipProvider` 上配置 `transition` prop，传入 motion 的 `Transition` 对象，例如 `{ type: 'spring', stiffness: 200, damping: 25 }` 可调整弹性效果。不要尝试在下层组件上直接覆盖过渡参数。

### 为什么 TooltipContent 不支持 asChild？
TooltipMotion 的 `TooltipContent` 需要内部的 `motion.div` 布局动画容器，移除 `asChild` 以确保动画结构完整。如需自定义包装元素，通过 `className` 和 CSS 调整样式而非改变 DOM 结构。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/tooltip-motion` 导入，不要从 `antd` 导入。
2. 必须用 `TooltipProvider` 包裹所有 Tooltip 实例，推荐在应用根布局中放置。
3. 每个 `Tooltip` 内部放置 `TooltipTrigger` 和 `TooltipContent` 两个子组件。
4. `TooltipContent` 不支持 `asChild`，不要尝试传入该 prop。
5. 需要自定义动效时，在 `TooltipProvider` 上配置 `transition` prop。
6. 箭头由组件自动渲染，无需手动添加。
7. 不要引入 Ant Design 相关依赖。
