# ThemeSwitcher 主题切换

三态主题切换复合组件，支持 system（跟随系统）、light（浅色）和 dark（深色）三种模式。使用 `motion/react` 的 `layoutId` 实现激活态指示器的平滑位移动画，并通过 `useControllableState` 支持受控/非受控双模式。

本组件为项目本地复合组件（Blocks），无 Ant Design 直接对应，API 和行为以 `src/components/blocks/theme-switcher/index.tsx` 源码为准。

## 何时使用 {#when-to-use}

当需要在界面中放置主题/颜色模式切换器时使用。

常见场景：

- 导航栏、设置页或页脚的主题切换入口。
- 需要三态切换（system/light/dark）而非简单 toggle 的场景。
- 偏好跟随系统但允许用户手动覆盖的应用。
- 需要与 `next-themes` 或其他主题管理方案集成的受控组件。

推荐使用方式：

- 受控模式：传入 `value` 和 `onChange`，由父组件管理主题状态，在 `onChange` 中执行实际主题变更（DOM class、CSS 变量等）。
- 非受控模式：仅传入 `defaultValue`（默认 `"system"`），组件内部管理 UI 状态。
- 通常放置在顶部导航栏右侧或设置面板中。

不推荐：

- 仅需二态切换（亮/暗）——可使用简单的 toggle 按钮。
- 需要 4 种以上主题选项——本组件仅支持三种模式。
- 需要自定义图标或主题标签——图标和 aria-label 内置，不可定制。
- 为了照搬 Ant Design API 而引入本组件——它不对应任何 AntD 组件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { ThemeSwitcher } from "@/components/ui/theme-switcher"

export default function Demo() {
  return <ThemeSwitcher />
}
```

### 受控模式

```tsx
import { useState } from "react"
import { ThemeSwitcher } from "@/components/ui/theme-switcher"

export default function Demo() {
  const [theme, setTheme] = useState<"light" | "dark" | "system">("system")

  return (
    <ThemeSwitcher
      value={theme}
      onChange={(t) => {
        setTheme(t)
        // 在此集成主题切换逻辑，如 next-themes 的 setTheme
      }}
    />
  )
}
```

> `ThemeSwitcher` 本身不包含主题切换逻辑，仅管理 UI 状态。实际主题应用（DOM class 变更、CSS 变量切换、localStorage 存储等）需在 `onChange` 回调中由调用侧实现，通常配合 `next-themes` 使用。

## 核心导出 {#exports}

- `ThemeSwitcher` — 主题切换组件
- `ThemeSwitcherProps` — 组件 Props 类型

## API {#api}

类型来自 `src/components/blocks/theme-switcher/index.tsx`。

### ThemeSwitcher

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| value | `"light"` \| `"dark"` \| `"system"` | - | 受控模式下的当前主题 |
| onChange | `(theme: "light"` \| `"dark"` \| `"system") => void` | - | 主题变化时的回调，用于执行实际的主题切换逻辑 |
| defaultValue | `"light"` \| `"dark"` \| `"system"` | `"system"` | 非受控模式下的默认主题 |
| className | `string` | - | 附加 className，与默认容器样式合并 |

组件使用 `useControllableState` 实现受控/非受控双模式：传入 `value` 时为受控模式，否则为非受控模式。

三个内置主题选项：

| key | 图标 | aria-label |
| --- | --- | --- |
| `"system"` | Monitor（lucide-react） | "System theme" |
| `"light"` | Sun（lucide-react） | "Light theme" |
| `"dark"` | Moon（lucide-react） | "Dark theme" |

## 样式模型 {#styling-model}

- 使用 Tailwind CSS 和项目语义变量。
- 外层容器：`relative isolate flex h-8 rounded-full bg-background p-1 ring-1 ring-border`，形成药丸形外壳。
- 每个按钮：`relative h-6 w-6 rounded-full`，内部放置图标。
- 图标：`relative z-10 h-4 w-4`，激活态 `text-foreground`，非激活态 `text-muted-foreground`。
- 激活态指示器：使用 motion 的 `layoutId="activeTheme"` + spring 过渡（`duration: 0.5`），`absolute inset-0 rounded-full bg-secondary`，实现跨按钮的平滑位移动画。
- 主题颜色全部使用项目级语义变量（`bg-background`、`bg-secondary`、`text-foreground`、`text-muted-foreground`、`ring-border`），不硬编码色值。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/theme-switcher/index.tsx` — 完整实现
- `src/components/ui/theme-switcher.tsx` — 重导出（barrel）

直接依赖摘要：

- `@radix-ui/react-use-controllable-state` — 受控/非受控状态管理
- `motion/react` — 激活态指示器布局动画（`layoutId`）
- `lucide-react` — Monitor、Sun、Moon 图标
- `@/lib/utils` — `cn` 样式合并工具

实现注意：

1. 组件 API 以本地 TypeScript 类型为准，`ThemeSwitcherProps` 定义了全部可配置项。
2. 使用 `useControllableState` 统一受控/非受控模式，`defaultValue` 默认为 `"system"`。
3. 通过 `mounted` 状态 + `useEffect` 防止 hydration mismatch（SSR 期间返回 `null`，客户端挂载后再渲染）。
4. 激活态指示器使用 motion 的共享布局动画（`layoutId="activeTheme"`），spring 过渡 0.5s。
5. 组件本身不包含主题切换逻辑（如修改 DOM class、操作 localStorage），仅管理 UI 选择状态。实际主题变更需在 `onChange` 回调中由调用侧实现。
6. 每个按钮设置 `type="button"` 防止表单纯提交，`aria-label` 提供可访问名称。
7. 三种主题及其对应图标/标签内置于组件，不可通过 props 扩展或替换。

## 可访问性 {#accessibility}

- 三个按钮均设置 `aria-label`（"System theme" / "Light theme" / "Dark theme"），为屏幕阅读器提供清晰的按钮语义。
- 按钮为原生 `<button type="button">`，保留键盘 focus 和 Enter/Space 激活行为。
- 纯图标按钮，依赖 `aria-label` 提供可访问名称，无可见文本标签。
- 激活态通过图标颜色对比度区分（`text-foreground` vs `text-muted-foreground`），满足 WCAG 视觉辨识要求。
- 组件在 SSR 阶段返回 `null`，避免水合不匹配导致的无障碍树异常。
- 如需增强语义，可在组件外层补充 `aria-pressed` 或当前主题的文字说明。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。Ant Design 通过 `ConfigProvider` 的 `theme` 属性控制全局主题，但不提供独立的主题切换 UI 组件。

| 方向 | 本地决策 |
| --- | --- |
| 组件来源 | 项目本地复合，非 AntD 封装或兼容层 |
| API 来源 | 以 `src/components/blocks/theme-switcher/index.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量，不引入 AntD css-in-js |
| 动画方案 | 使用 motion/react layoutId 实现跨按钮位移动画 |
| 状态管理 | 使用 @radix-ui/react-use-controllable-state |
| 主题切换逻辑 | 组件仅管 UI 状态，实际主题变更由调用侧处理（如配合 next-themes） |
| 与 AntD ConfigProvider 关系 | 互补：ThemeSwitcher 提供选择 UI，ConfigProvider 负责全局主题应用 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 ThemeSwitcher 代码时，应遵守：

1. 从 `@/components/ui/theme-switcher` 导入 ThemeSwitcher。
2. 受控模式使用 `value` + `onChange`；非受控模式使用 `defaultValue`（默认 `"system"`）。
3. `value`、`onChange` 参数和 `defaultValue` 的值类型为 `"light" | "dark" | "system"`，不要使用其他值。
4. `ThemeSwitcher` 仅管理 UI 选择状态，实际主题切换逻辑（DOM class、CSS 变量、localStorage 等）需在 `onChange` 回调中由调用侧实现。
5. 只使用本文件 API 表格和源码中存在的 props，不支持自定义主题列表、图标或标签。
6. 不要删除或绕开 motion 指示器动画——动画是组件的核心交互特征。
7. 不要自行添加 SSR hydration 处理——组件已内置 `mounted` 状态。
8. 不要生成 AntD 的 `ConfigProvider`、`theme`、`classNames`、`styles` 等属性。
9. 如果不确定某个 API 是否存在，先检查 `src/components/blocks/theme-switcher/index.tsx`，不要凭记忆生成。
