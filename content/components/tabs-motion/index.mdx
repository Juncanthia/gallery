TabsMotion 动效标签页。带高亮指示器和滑动切换动效的标签页复合组件，支持内容区高度自适应和模糊过渡。基于 `motion/react` 和自定义原语实现，不使用 Radix Tabs。

<h2 id="when-to-use">何时使用</h2>

当需要在界面中实现标签页切换并希望带有动效反馈时使用。

常见场景：
- 设置面板、详情页内的分类信息切换。
- 仪表盘中的指标分类查看。
- 登录/注册表单的 Tab 切换。
- 需要视觉层次清晰的内容分类展示。

不推荐：
- 标签数量超过 6-8 个——应改用下拉选择或垂直导航。
- 在标签页内嵌套另一层 TabsMotion。
- 需要标签位置（left/right/bottom）、可编辑标签、card 样式等——使用 `@/components/ui/tabs`。
- 无需动效的简单切换场景——使用 `@/components/ui/tabs` 更轻量。

<h2 id="examples">代码演示</h2>

### 基础用法

非受控模式下通过 `defaultValue` 设置初始激活标签。

<ComponentShowcase id="tabs-motion" path="tabs-motion/basic" />

```tsx
import {
  Tabs,
  TabsList,
  TabsTrigger,
  TabsContents,
  TabsContent,
} from "@/components/blocks/tabs-motion"

export default function Demo() {
  return (
    <Tabs defaultValue="tab1" className="w-full max-w-sm">
      <TabsList>
        <TabsTrigger value="tab1">标签一</TabsTrigger>
        <TabsTrigger value="tab2">标签二</TabsTrigger>
        <TabsTrigger value="tab3">标签三</TabsTrigger>
      </TabsList>
      <TabsContents>
        <TabsContent value="tab1">内容一</TabsContent>
        <TabsContent value="tab2">内容二</TabsContent>
        <TabsContent value="tab3">内容三</TabsContent>
      </TabsContents>
    </Tabs>
  )
}
```

### 受控模式

通过 `value` + `onValueChange` 实现受控切换。

```tsx
import { useState } from "react"
import {
  Tabs, TabsList, TabsTrigger, TabsContents, TabsContent,
} from "@/components/blocks/tabs-motion"

export default function Demo() {
  const [tab, setTab] = useState("tab1")
  return (
    <Tabs value={tab} onValueChange={setTab} className="w-full max-w-sm">
      <TabsList>
        <TabsTrigger value="tab1">账户</TabsTrigger>
        <TabsTrigger value="tab2">安全</TabsTrigger>
        <TabsTrigger value="tab3" disabled>高级</TabsTrigger>
      </TabsList>
      <TabsContents>
        <TabsContent value="tab1">账户设置内容</TabsContent>
        <TabsContent value="tab2">安全设置内容</TabsContent>
        <TabsContent value="tab3">高级设置内容</TabsContent>
      </TabsContents>
    </Tabs>
  )
}
```

<h2 id="api">API</h2>

### Tabs

根容器，管理当前激活标签值的上下文。支持受控和非受控两种模式。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 受控模式下的当前激活值 | `string` | - |
| defaultValue | 非受控模式下的初始激活值 | `string` | 第一个注册的 trigger 的 value |
| onValueChange | 切换标签时的回调 | `(value: string) => void` | - |
| className | 附加 CSS 类名 | `string` | - |
| children | 标签页内容 | `React.ReactNode` | - |

### TabsList

标签按钮容器，内部包裹高亮指示器。渲染为 `role="tablist"` 的 `div`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 CSS 类名 | `string` | - |
| children | 通常放置 `TabsTrigger` | `React.ReactNode` | - |

### TabsTrigger

标签按钮，切换当前激活标签。渲染为 `role="tab"` 的 motion `button`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 标签值（必填），需与对应 `TabsContent` 的 `value` 一致 | `string` | - |
| asChild | 使用 Slot 合并到子元素 | `boolean` | `false` |
| disabled | 禁用标签 | `boolean` | `false` |
| className | 附加 CSS 类名 | `string` | - |
| children | 标签显示内容 | `React.ReactNode` | - |

### TabsContents

内容面板滑动容器，根据当前激活值自动滑动到对应面板，并实现高度自适应动效。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| transition | 高度和位移的动效配置 | `Transition`（motion） | `{ type: 'spring', stiffness: 300, damping: 30, bounce: 0, restDelta: 0.01 }` |
| className | 附加 CSS 类名 | `string` | - |
| children | 通常放置 `TabsContent` | `React.ReactNode` | - |

### TabsContent

单一面板，仅在对应 `value` 激活时可见。非激活面板设置 `inert` 且添加模糊效果。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 面板值（必填），需与对应 `TabsTrigger` 的 `value` 一致 | `string` | - |
| asChild | 使用 Slot 合并到子元素 | `boolean` | `false` |
| className | 附加 CSS 类名 | `string` | - |
| children | 面板内容 | `React.ReactNode` | - |

<h2 id="semantic-dom">Semantic DOM</h2>

- `TabsList` 使用 `role="tablist"`。
- `TabsTrigger` 使用 `role="tab"`，通过 `data-state` 标记激活/非激活态。
- `TabsContent` 使用 `role="tabpanel"`。
- 非激活面板设置 `inert` 属性。

<h2 id="faq">常见问题</h2>

### 与 `@/components/ui/tabs` 有什么区别？

| 对比维度 | tabs-motion | tabs |
| --- | --- | --- |
| 底层实现 | 自定义原语 `@/components/_primitives/animate/tabs` | Radix Tabs |
| 动效 | motion/react 高亮指示器 + 滑动 + 模糊 | 基础高亮动画 |
| 标签位置 | 仅顶部（水平） | top/bottom/start/end |
| 样式类型 | 仅 pill 风格 | line/card |
| items 声明式 | 不支持 | 支持 |
| 适用场景 | 需要动效反馈的简单标签切换 | 功能完整的标签页 |

### 如何给 TabsTrigger 添加图标？

TabsTrigger 没有独立的 `icon` prop。图标通过 `children` 内联添加：

```tsx
<TabsTrigger value="settings">
  <SettingsIcon className="size-4" />
  设置
</TabsTrigger>
```

### 非受控模式下如何获取当前激活标签？

TabsMotion 不支持在非受控模式下获取当前值。如果需要追踪激活状态，使用受控模式：

```tsx
const [tab, setTab] = useState("tab1")
<Tabs value={tab} onValueChange={setTab}>
  ...
</Tabs>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/tabs-motion` 导入，不要从 `antd` 或 `radix-ui` 导入 Tabs。
2. `TabsTrigger` 和 `TabsContent` 的 `value` 属性必须唯一且一一对应。
3. `TabsContents` 必须包裹所有 `TabsContent`，不要将 `TabsContent` 放在其他容器中。
4. 标签数控制在 2-6 个，超出建议使用其他导航模式。
5. 受控模式使用 `value` + `onValueChange`；非受控模式只使用 `defaultValue`。
6. 不要生成 Ant Design 的 `items`、`tabPosition`、`type`、`tabBarExtraContent` 等属性——这些在此组件中不存在。
7. 图标的添加方式为在 `children` 中内联，没有独立的 `icon` prop。
8. 不要引入 Ant Design 相关依赖。
