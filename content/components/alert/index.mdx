Alert 用于在页面中展示需要用户关注的提示信息，支持多种语义变体。

<h2 id="when-to-use">何时使用</h2>

当需要在页面中向用户展示警告、错误、成功或信息提示时使用。与 Notification 不同，Alert 是页面内嵌的静态提示。

常见场景：

- 表单提交后的成功/失败反馈。
- 重要操作前的警告提示。
- 页面顶部的全局通知横幅。

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `variant` 切换语义样式。

<ComponentShowcase
  id="alert"
  path="alert/basic"
/>

```tsx
import { Alert, AlertDescription, AlertTitle } from "@/components/ui/alert"
import { Info } from "lucide-react"

export default function Demo() {
  return (
    <div className="w-full max-w-md space-y-3">
      <Alert>
        <Info className="size-4" />
        <AlertTitle>提示</AlertTitle>
        <AlertDescription>这是一条普通提示信息。</AlertDescription>
      </Alert>
      <Alert variant="destructive">
        <Info className="size-4" />
        <AlertTitle>错误</AlertTitle>
        <AlertDescription>操作失败，请重试。</AlertDescription>
      </Alert>
      <Alert variant="success">
        <Info className="size-4" />
        <AlertTitle>成功</AlertTitle>
        <AlertDescription>操作已成功完成。</AlertDescription>
      </Alert>
    </div>
  )
}
```

<h2 id="api">API</h2>

### Alert

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 语义变体 | `default` \| `destructive` \| `success` \| `warning` \| `info` | `default` |

### 子组件

| 组件 | 说明 |
| --- | --- |
| AlertTitle | 提示标题 |
| AlertDescription | 提示详细描述 |

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Alert 代码时：

1. 从 `@/components/ui/alert` 导入。
2. 使用 `variant` 表达语义，不要手动拼颜色 class。
3. 不要引入 Ant Design 的 `Alert`。


<h2 id="semantic-dom">Semantic DOM</h2>

组件通过 `data-slot` 属性暴露稳定的语义结构，可用于测试定位和样式覆写：

| data-slot | 说明 |
| --- | --- |
| `alert` | - |
| `alert-action` | - |
| `alert-description` | - |
| `alert-title` | - |
