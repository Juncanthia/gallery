Button 用于触发一个即时操作。

<h2 id="when-to-use">何时使用</h2>

标记一个操作命令，响应用户点击行为，触发相应的业务逻辑。

常见场景：

- 主操作：提交、保存、创建。
- 次操作：取消、重置、查看详情。
- 危险操作：删除、移除、禁用账号。
- 异步操作：提交中、导出中、上传中。
- 链接跳转：外部页面或内部路由。
- 图标操作：搜索、下载、刷新、更多。

Button 通过 `color` 和 `variant` 表达视觉语义：

- `color`：操作语义 — `primary`、`danger`、`success`、`warning`、`info`。
- `variant`：视觉层级 — `solid`、`outlined`、`dashed`、`filled`、`text`、`link`。

推荐组合：

- 主操作：`color="primary" variant="solid"`
- 普通操作：默认按钮，或 `color="default" variant="outlined"`
- 危险操作：`color="danger"`
- 低强调操作：`variant="text"` 或 `variant="link"`
- 异步操作：`loading`，可配合 `loadingText` 替换按钮文字

### 加载中状态

`loading` 使按钮进入加载状态，自动禁用点击，显示旋转图标并设置 `aria-busy`。`loadingText` 可在加载期间替换按钮文字。

<ComponentShowcase
  id="loading"
  path="button/loading"
/>

```tsx
import { useState } from "react"
import { Button } from "@/components/base/button"

export default function Demo() {
  const [saving, setSaving] = useState(false)

  return (
    <div className="flex flex-wrap gap-2">
      <Button color="primary" loading variant="solid">Loading</Button>
      <Button color="primary" loading loadingText="提交中" variant="outlined">提交</Button>
      <Button
        color="primary"
        loading={saving}
        loadingText="保存中"
        onClick={() => { setSaving(true); setTimeout(() => setSaving(false), 1500) }}
        variant="solid"
      >
        保存
      </Button>
    </div>
  )
}
```

### 链接与路由

传入 `href` 时，Button 渲染为链接元素，保留按钮样式。外链应显式指定 `target` 和 `rel`。使用 `asChild` 可将样式合并到路由组件（如 TanStack Router 的 `Link`）。

<ComponentShowcase
  id="link"
  path="button/link"
/>

```tsx
import { Link } from "@tanstack/react-router"
import { Button } from "@/components/base/button"

export default function Demo() {
  return (
    <div className="flex flex-wrap gap-2">
      <Button variant="link" href="/components/button">查看文档</Button>
      <Button href="https://example.com" rel="noreferrer" target="_blank">打开外链</Button>
      <Button asChild color="primary" variant="solid">
        <Link to="/components/$slug" params={{ slug: "button" }}>路由跳转</Link>
      </Button>
    </div>
  )
}
```

## 与 Ant Design 的差异

| 差异点 | 说明 |
| --- | --- |
| 无 `type` 属性 | 使用 `color + variant` 替代，如 `color="primary" variant="solid"` |
| 无 `danger` 属性 | 使用 `color="danger"` |
| 原生 `type` → `htmlType` | 避免与 AntD 的 `type` 概念冲突 |
| 无 `Button.Group` | 使用独立的 `ButtonGroup` 组件 |
| 无 ConfigProvider / Design Token | 使用 Tailwind 主题变量和 CVA 管理样式 |

<h2 id="faq">常见问题</h2>

### 如何写链接按钮？

传入 `href` 即可：

```tsx
<Button href="/components/button" variant="link">查看文档</Button>
```

配合路由组件时使用 `asChild`：

```tsx
<Button asChild>
  <Link to="/components/$slug" params={{ slug: "button" }}>查看文档</Link>
</Button>
```

### 如何关闭按钮动画？

将 `hoverScale` 和 `tapScale` 设为 `1`：

```tsx
<Button hoverScale={1} tapScale={1}>无缩放反馈</Button>
```

### 如何做加载按钮？

使用 `loading`；需要替换文案时配合 `loadingText`：

```tsx
<Button loading loadingText="提交中" color="primary" variant="solid">提交</Button>
```

### 如何做纯图标按钮？

传入 `icon` 且不传 `children`，必须补充 `aria-label`：

```tsx
<Button icon={<Search />} aria-label="搜索" />
```

### 如何在表单中使用？

提交按钮使用 `htmlType="submit"`：

```tsx
<Button htmlType="submit" color="primary" variant="solid">提交</Button>
```

### 如何自定义样式？

优先使用 `color`、`variant`、`size`、`shape` 组合。局部调整使用 `className`。频繁出现的同类样式应在 `buttonVariants` 中扩展，而非在调用处复制 `className`。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Button 代码时：

1. 从 `@/components/base/button` 导入
2. 使用 `color + variant`，不要生成 `type` 或 `danger` 属性
3. 表单按钮使用 `htmlType`，不要使用原生 `type`
4. 图标使用 `icon`，图标位置使用 `iconPlacement`
5. 纯图标按钮必须包含 `aria-label`
6. 链接按钮优先使用 `href`；路由组合使用 `asChild`
7. 加载状态使用 `loading` + 可选 `loadingText`
8. 不要引入 Ant Design 相关依赖
