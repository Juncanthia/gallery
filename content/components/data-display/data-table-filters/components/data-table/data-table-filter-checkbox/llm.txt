# DataTableFilterCheckbox 复选筛选

DataTableFilterCheckbox 是 DataTable 筛选体系中的多选筛选组件，以复选框列表形式展示可选项，支持搜索过滤、faceted 计数显示和「only」快捷单选。

本组件是 DataTable 筛选体系内的局部组件，API 和行为以源码为准。它通过 `DataTableProvider` 上下文获取表格实例和筛选状态，接收 `DataTableCheckboxFilterField` 类型 props。

## 何时使用 {#when-to-use}

当需要对表格中某一列进行多选筛选时使用。组件展示所有可选项的复选框列表，支持搜索过滤和计数显示。

常见场景：

- 对状态列（active、inactive、pending）进行多选筛选。
- 对分类列（admin、user、editor）进行多选筛选。
- 需要查看每个选项的数据行数（faceted counts）。
- 选项较多时（超过 4 个）自动展示搜索框。

不推荐：

- 用于单选筛选，可使用 Select 或 RadioGroup。
- 用于数值范围筛选，应使用 DataTableFilterSlider。
- 脱离 DataTableProvider 上下文使用。

## 代码演示 {#examples}

### 基础用法

通过 `DataTableProvider` 包裹，传入 `value`、`options` 和 `label`。组件自动从上下文获取筛选状态和 faceted 计数。

```tsx
import { useState } from "react"
import { DataTableProvider } from "@/components/data-display/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-provider"
import { DataTableFilterCheckbox } from "@/components/data-display/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-filter-checkbox"
import {
  useReactTable,
  getCoreRowModel,
  type ColumnDef,
  type ColumnFiltersState,
} from "@tanstack/react-table"
import type { DataTableFilterField } from "@/components/data-display/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-provider"

type Person = {
  id: string
  name: string
  blocks/status: string
  role: string
}

const data: Person[] = [
  { id: "1", name: "Alice", blocks/status: "active", role: "admin" },
  { id: "2", name: "Bob", blocks/status: "inactive", role: "user" },
  { id: "3", name: "Charlie", blocks/status: "active", role: "editor" },
  { id: "4", name: "Diana", blocks/status: "pending", role: "user" },
  { id: "5", name: "Eve", blocks/status: "active", role: "admin" },
]

const columns: ColumnDef&lt;Person&gt;[] = [
  { accessorKey: "name", header: "Name" },
  { accessorKey: "blocks/status", header: "Status" },
  { accessorKey: "role", header: "Role" },
]

const filterFields: DataTableFilterField&lt;Person&gt;[] = [
  {
    label: "Status",
    value: "blocks/status",
    type: "checkbox",
    options: [
      { label: "Active", value: "active" },
      { label: "Inactive", value: "inactive" },
      { label: "Pending", value: "pending" },
    ],
  },
]

export default function Demo() {
  const [columnFilters, setColumnFilters] = useState&lt;ColumnFiltersState&gt;([])

  const table = useReactTable({
    data,
    columns,
    state: { columnFilters },
    onColumnFiltersChange: setColumnFilters,
    getCoreRowModel: getCoreRowModel(),
  })

  const getFacetedUniqueValues = (
    table: ReturnType&lt;typeof useReactTable&lt;Person&gt;&gt;,
    columnId: string,
  ): Map&lt;string, number&gt; => {
    const map = new Map&lt;string, number&gt;()
    table.getPreFilteredRowModel().rows.forEach((row) => {
      const value = row.getValue(columnId) as string
      map.set(value, (map.get(value) || 0) + 1)
    })
    return map
  }

  return (
    &lt;DataTableProvider
      table={table}
      columns={columns}
      filterFields={filterFields}
      columnFilters={columnFilters}
      getFacetedUniqueValues={getFacetedUniqueValues}
    &gt;
      &lt;div className="w-full max-w-xs"&gt;
        &lt;DataTableFilterCheckbox&lt;Person&gt;
          label="Status"
          value="blocks/status"
          type="checkbox"
          options={[
            { label: "Active", value: "active" },
            { label: "Inactive", value: "inactive" },
            { label: "Pending", value: "pending" },
          ]}
        /&gt;
      &lt;/div&gt;
    &lt;/DataTableProvider&gt;
  )
}
```

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
type Option = {
  label: string;
  value: string | boolean | number | undefined;
};
```

```ts
type DataTableCheckboxFilterField&lt;TData&gt; = {
  label: string;
  value: keyof TData;
  type: "checkbox";
  options?: Option[];
  component?: (props: Option) =&gt; JSX.Element | null;
  defaultOpen?: boolean;
  commandDisabled?: boolean;
};
```

行为规则：

1. 组件依赖 `useDataTable()` 上下文，必须在 `DataTableProvider` 内部使用。
2. `value` 对应 table column id，用于查找 `columnFilters` 中的筛选值。
3. `options` 数量超过 4 个时，自动展示搜索框（`InputGroup`），按 `label` 大小写不敏感过滤。
4. 每个选项行右侧显示 faceted 计数（通过 `getFacetedUniqueValues` 获取），加载时显示 Skeleton。
5. 悬停某选项行时，右侧显示「only」按钮，点击后仅保留该选项的筛选值。
6. `isLoading` 为 true 且 `filterOptions` 为空时，显示 3 行 Skeleton 占位。
7. `component` 可自定义选项渲染，接收完整的 `Option` 对象。

## Semantic DOM {#semantic-dom}

- 源码未声明 `data-slot`，使用根元素语义和组件名定位。
- Checkbox 使用 radix `Checkbox` 组件，关联 `<Label htmlFor>`。
- 搜索框使用 `InputGroup` + `InputGroupAddon` + `InputGroupInput` 组合。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 选项列表容器使用 `border-border rounded-lg border max-h-[200px] overflow-y-auto`。
- 每个选项行：`px-2 py-2.5`，非末行 `border-b` 分隔。
- hover 时选项行背景变为 `bg-accent/50`，标签文字变为 `text-accent-foreground`。
- 「only」按钮使用 `backdrop-blur-xs` 毛玻璃效果，默认隐藏、hover 时显示。
- 搜索框使用 `h-9 rounded-lg shadow-none` 紧凑样式。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-filter-checkbox.tsx`

直接依赖：

- `@/components/core/checkbox`（radix Checkbox 封装）
- `@/components/core/input-group`（InputGroup、InputGroupAddon、InputGroupInput）
- `@/components/core/label`
- `@/components/core/skeleton`
- `@hyper/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-provider`（useDataTable）
- `@hyper/blocks/data-table-filters/lib/format`（formatCompactNumber）
- `@hyper/blocks/data-table-filters/lib/utils`（cn）
- `lucide-react`（SearchIcon）

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. filterValue 统一标准化为数组：非数组值自动包装为 `[filterValue]`。
3. isLoading 期间，若 `filterOptions` 为空则展示 Skeleton，不为空则正常渲染（支持增量加载场景）。
4. 「only」按钮通过单值数组 `[option.value]` 设置筛选，清空使用 `undefined`。
5. 样式应优先通过组件 props 或本地 className 扩展点表达。

## 可访问性 {#accessibility}

- 每个 Checkbox 关联唯一的 `<Label htmlFor={value}-${option.value}>`。
- 「only」按钮包含 `focus-visible:ring` 样式，支持键盘聚焦。
- 搜索框使用标准 `<InputGroupInput>`，支持原生键盘交互。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 DataTableFilterCheckbox 代码时，应遵守：

1. 从 `@/components/data-display/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-filter-checkbox` 导入。
2. 组件必须在 `DataTableProvider` 内部使用，否则会抛出 `useDataTable` 错误。
3. 在 `filterFields` 中配置 `type: "checkbox"` 后，由 `DataTableFilterControls` 自动管理，通常不需要手动引入。
4. 传入的 `value` 必须对应 table 中已定义的 column id。
5. `options` 为 `Option[]` 类型，每个选项包含 `label` 和 `value`。
6. 需要展示 faceted 计数时，通过 `DataTableProvider` 的 `getFacetedUniqueValues` 提供。
7. 不要引入 Ant Design 相关依赖。
8. 如果不确定某个 API 是否存在，先检查源码，不要凭 AntD 记忆生成。
