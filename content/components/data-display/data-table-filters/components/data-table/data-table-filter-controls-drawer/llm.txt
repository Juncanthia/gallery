# DataTableFilterControlsDrawer 筛选抽屉

DataTableFilterControlsDrawer 将数据表格的筛选控件收纳在抽屉（Drawer）中，通过快捷键 `⌘B` 或点击筛选图标按钮唤出。适用于移动端适配或需要节省表格工具栏空间的场景。

组件本身无 props，从 `DataTableProvider` 上下文读取 `filterFields` 并渲染对应的筛选控件（checkbox、input、slider、timerange）。

## 何时使用 {#when-to-use}

当数据表格的筛选字段较多，不便全部展示在表格工具栏时，使用筛选抽屉将筛选控件集中管理。

常见场景：

- 表格筛选字段较多（3 个以上），工具栏空间紧张。
- 移动端布局中，需要将筛选控件放入抽屉以保证表格内容的可视面积。
- 希望提供快捷键快速唤出筛选面板。
- 与 `DataTableFilterControls` 配合，将多种筛选控件以手风琴形式排列在抽屉内。

不推荐：

- 筛选字段少于 2 个时使用抽屉（直接放在工具栏即可）。
- 在非 `DataTableProvider` 上下文外使用本组件。

## 代码演示 {#examples}

### 基础用法

筛选抽屉通过底部弹出的 Drawer 展示筛选控件。点击筛选图标或按 `⌘B` 打开。

```tsx
import { DataTableFilterControlsDrawer } from "@/components/data-display/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-filter-controls-drawer"
import { DataTableProvider } from "@hyper/blocks/data-table-filters"

export default function Demo() {
  return (
    <DataTableProvider table={table} columns={columns} filterFields={filterFields}>
      <DataTableFilterControlsDrawer />
    </DataTableProvider>
  )
}
```

## API {#api}

### DataTableFilterControlsDrawer

`DataTableFilterControlsDrawer` 是无 props 组件，完全依赖 `DataTableProvider` 上下文。

组件没有显式 props，但依赖以下 Provider 层：

| 依赖 | 说明 |
| --- | --- |
| `DataTableProvider` | 提供 `filterFields`、`table`、`columns` 上下文 |
| `DataTableStoreProvider` | 提供 store adapter，用于筛选状态同步（由 `DataTableProvider` 内部的 `DataTableStoreSync` 消费） |
| `ControlsProvider` | 提供展开/折叠状态，由 `DataTableProvider` 内部提供 |

### 快捷键

| 按键 | 说明 |
| --- | --- |
| `⌘ B` | 点击触发按钮，打开筛选抽屉 |

### 触发按钮 DOM 结构


## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-filter-controls-drawer.tsx`：Drawer 外壳，Tooltip、快捷键绑定。
- `src/components/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-filter-controls.tsx`：内部筛选控件容器，根据 `filterFields` 渲染对应控件。
- `src/components/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-provider.tsx`：上下文提供者，向子组件注入 `table`、`filterFields` 等。
- `src/components/blocks/data-table-filters/lib/store/provider/DataTableStoreProvider.tsx`：BYOS adapter 提供者。
- `src/components/core/dt-blocks/data-table-filter-controls-drawer.tsx`：UI 层 re-export wrapper。

关键行为：

1. 使用 `useHotKey` hook 绑定 `⌘B` 快捷键，点击 `triggerButtonRef` 所引用的按钮。
2. 移动端（`max-width: 640px`）时将 `ref` 附加到触发按钮上，确保快捷键在移动端也能正常工作。
3. Drawer 标题/描述通过 `VisuallyHidden` 隐藏，保持可访问性但不占视觉空间。
4. Drawer 最大高度 `max-h-[calc(100dvh-4rem)]`，内容区 `overflow-y-auto` 支持滚动。
5. 筛选控件通过 `FILTER_COMPONENTS` 注册表动态渲染，支持 checkbox、input、slider、timerange 四种类型。
6. `defaultOpen` 为 `true` 的筛选字段默认展开其手风琴项。

## 主题变量 {#design-token}

本组件不引入独立主题变量，所有样式通过 Tailwind CSS class 和 shadcn 语义色变量控制。

关键样式类：

| 元素 | 样式 |
| --- | --- |
| 触发按钮 | `h-9 w-9` + `variant="ghost" size="icon"` |
| Tooltip 内容 | `text-nowrap`，Kbd 使用 `text-muted-foreground` |
| Drawer 内容 | `max-h-[calc(100dvh-4rem)]` |
| 筛选控件容器 | `flex-1 overflow-y-auto px-4` |

## 可访问性 {#accessibility}

- Drawer 标题和描述使用 `VisuallyHidden` 向屏幕阅读器暴露，视觉上不可见。
- 触发按钮是原生 `<button>`，hover 时显示 Tooltip 描述。
- 快捷键通过 `useHotKey` 绑定键盘事件。
- Drawer 底部提供显式 Close 按钮。
- 移动端适配：`max-width: 640px` 时按钮获得 `ref` 以支持快捷键触发。

## FAQ {#faq}

### 如何将筛选抽屉嵌入到表格工具栏？

将 `DataTableFilterControlsDrawer` 放置在 `DataTableToolbar` 内：

```tsx
<DataTableToolbar>
  <DataTableFilterControlsDrawer />
  <DataTableViewOptions />
</DataTableToolbar>
```

### 如何自定义快捷键？

当前快捷键硬编码为 `b`（在 Mac 上表现为 `⌘B`）。修改需编辑 `useHotKey` 调用的第二个参数：

```tsx
useHotKey(() => {
  triggerButtonRef.current?.click()
}, "f")  // 改为 ⌘F
```

### 筛选控件不显示？

确认 `DataTableProvider` 的 `filterFields` 已正确传入，且每个字段的 `type` 在 `FILTER_COMPONENTS` 注册表中存在。支持的类型：`checkbox`、`input`、`slider`、`timerange`。

### 筛选状态如何跨组件共享？

筛选状态通过 `DataTableProvider` 上下文共享。`DataTableStoreSync` 将 React Table 的 columnFilters 同步到 BYOS adapter（如 URL params 或 localStorage）。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 `DataTableFilterControlsDrawer` 代码时，应遵守：

1. 从 `@/components/data-display/blocks/data-table-filters/components/blocks/data-table/blocks/data-table-filter-controls-drawer` 导入组件。
2. 必须在 `DataTableStoreProvider` 和 `DataTableProvider` 包裹内使用。
3. `DataTableStoreProvider` 需要传入 `adapter`，展示场景使用 `useMemoryAdapter`，生产环境使用 `useNuqsAdapter`。
4. `DataTableProvider` 需要传入 `table`、`columns`、`filterFields`。
5. `filterFields` 数组定义筛选字段，每项的 `type` 对应筛选控件类型（checkbox / input / slider / timerange）。
6. 配合 `DataTableToolbar` 使用时，将本组件放在工具栏内。
7. 不要绕过 Provider 层直接使用 `DataTableFilterControls`。
8. 不要移除组件内部的 Tooltip 或快捷键行为。
9. 不要引入 Ant Design 相关依赖。
