瀑布流 Masonry。默认 `Masonry` 基于 CSS Columns 实现，支持固定列数和响应式断点列数配置，纯 CSS 无 JS 布局开销；另提供 `VirtualMasonry`/`VirtualMasonryItem` 虚拟化瀑布流，用于大列表场景，仅渲染可视区域内的卡片并按最短列优先算法精确排列。

<h2 id="when-to-use">何时使用</h2>

适合展示高度不一致的内容卡片，利用瀑布流排列减少空白间隙。

常见场景：
- 图片画廊：图片原始比例不同，瀑布流可避免留白
- 产品展示：产品卡片高度因描述长度不同而参差不齐
- 文章列表：摘要长度不一的文章卡片排列
- 仪表盘面板：高度各异的小部件网格

不推荐：
- 内容高度完全一致的场景——此时普通 grid 更合适
- 需要精确行列对齐的表格型数据
- 项目数量极少（<5）且高度一致的场景

### 虚拟化瀑布流（大列表）

数据量较大（几十上百条以上）且希望获得"最短列优先"精确排列、并且只渲染可视区域内容时，使用 `VirtualMasonry`。它需要显式的 `VirtualMasonryItem` 子项，且父容器需要有确定的宽度/高度上下文（内部使用绝对定位 + 滚动窗口裁剪）。

```tsx
import { VirtualMasonry, VirtualMasonryItem } from "@/components/ui/masonry"

export default function Demo() {
  return (
    <VirtualMasonry columnWidth={220} gap={16} itemHeight={240}>
      {items.map((item) => (
        <VirtualMasonryItem key={item.id}>
          <div className="bg-muted rounded p-4">{item.title}</div>
        </VirtualMasonryItem>
      ))}
    </VirtualMasonry>
  )
}
```

<h2 id="api">API</h2>

### Masonry

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| columns | 列数配置。传数字时为固定列数（1-6）；传对象时为响应式断点列数 | `number` \| `{ sm?: number; md?: number; lg?: number; xl?: number }` | `3` |
| gap | 间距系数，实际像素值为 `gap * 4`（默认 16px） | `number` | `4` |
| children | 子内容，每个子元素为一个瀑布流项 | `ReactNode` | - |
| className | 附加样式类名 | `string` | - |

响应式断点对应关系：

| 断点 | Tailwind 前缀 | 最小宽度 |
| --- | --- | --- |
| sm | `sm:` | 640px |
| md | `md:` | 768px |
| lg | `lg:` | 1024px |
| xl | `xl:` | 1280px |

### VirtualMasonry

大列表虚拟化瀑布流，基于最短列优先算法 + `ResizeObserver` 自动测量 + 滚动窗口裁剪渲染。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| columnWidth | 单列宽度（px） | `number` | `200` |
| columnCount | 固定列数，传入后忽略自动计算 | `number` | - |
| maxColumnCount | 自动计算列数时的上限 | `number` | - |
| gap | 列间距/行间距，数字表示两者相同，也可分别指定 | `number \| { column: number; row: number }` | `0` |
| itemHeight | 未测量前用于估算总高度的默认项高 | `number` | `300` |
| overscan | 视口外提前渲染的倍数（乘以视口高度） | `number` | `2` |
| scrollFps | 滚动位置节流帧率 | `number` | `12` |
| linear | 为 `true` 时优先按索引顺序放入对应列（更接近网格感），为 `false` 时严格放入最短列 | `boolean` | `false` |
| fallback | 首次挂载测量完成前的占位内容 | `ReactNode` | - |
| asChild | 将根节点渲染委托给子元素（Radix Slot） | `boolean` | - |

### VirtualMasonryItem

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 将节点渲染委托给子元素（Radix Slot） | `boolean` | - |

<h2 id="faq">常见问题</h2>

### CSS Columns 的列排列顺序是怎样的？

CSS Columns 采用"从上到下逐列排满"的顺序，DOM 顺序与视觉排列一致。屏幕阅读器按列宣读，符合阅读习惯。

### 为什么 `gap` 要乘以 4？

`gap` 属性值会乘以 4 转换为像素，与 Tailwind 间距体系（1=4px）保持一致。例如 `gap={4}` 产生 16px 间距。

### 如何防止卡片内容跨列断裂？

每个子元素自动包裹在 `break-inside-avoid` 容器中，组件内部已处理，无需手动添加。

### `Masonry` 和 `VirtualMasonry` 该用哪个？

默认优先用 `Masonry`（零 JS 开销，实现简单）。只有当列表很大（几十条以上持续增长）、需要精确的"最短列优先"排列、或需要虚拟化只渲染可视区域时，才使用 `VirtualMasonry`；它的子项必须是 `VirtualMasonryItem`，且不能与 `Masonry` 的 children 直接混用。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/masonry` 导入。
2. 固定列数使用 `columns={n}`（n 为 1-6）。
3. 响应式列数使用 `columns={{ sm: 1, md: 2, lg: 3 }}` 对象形式。
4. 间距使用 `gap` 属性，其值会乘以 4 转为像素。
5. 不要引入 `react-masonry-css`、`masonry-layout` 等第三方布局库。
6. 不要引入 Ant Design 相关依赖。
7. 大列表 / 虚拟化场景使用 `VirtualMasonry` + `VirtualMasonryItem`，子项必须是 `VirtualMasonryItem`，不要把 `Masonry` 的 `columns`/`gap`（数字）语义套用到 `VirtualMasonry`（`gap` 在这里可以是对象 `{ column, row }`）。
