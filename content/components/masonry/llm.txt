# Masonry 瀑布流

提供两套互补实现，均导出自 `@/components/ui/masonry`：

- `Masonry`：基于 CSS Columns 的瀑布流布局组件，通过纯 CSS 实现不等高内容的瀑布流排列，支持固定列数和响应式断点列数配置。不使用任何第三方布局库，完全基于 Tailwind CSS 的 `columns-*` 工具类和 CSS `break-inside: avoid` 实现。**默认优先使用**，零 JS 布局开销。
- `VirtualMasonry` / `VirtualMasonryItem`：基于区间树（interval tree）+ 最短列优先算法的虚拟化瀑布流，仅渲染可视区域（+ overscan）内的卡片，配合 `ResizeObserver` 自动测量每项高度、滚动节流重新计算可见范围。适合大列表（几十条以上持续增长）或需要精确"最短列优先"排列（而非 CSS Columns 的逐列排满顺序）的场景。

本组件为项目本地复合组件（Blocks），无 Ant Design 直接对应。API 和行为以 `src/components/blocks/masonry/index.tsx`（`Masonry`）与 `src/components/blocks/masonry/virtualized.tsx`（`VirtualMasonry`）源码为准。

## 何时使用 {#when-to-use}

适合展示高度不一致的内容卡片。

常见场景：

- 图片画廊：图片原始比例不同，瀑布流可避免留白。
- 产品展示：产品卡片高度因描述长度不同而参差不齐。
- 文章列表：摘要长度不一的文章卡片排列。
- 仪表盘面板：高度各异的小部件网格。

推荐使用方式：

- 固定列数：`columns={3}` 用于简单场景。
- 响应式列数：`columns={{ sm: 1, md: 2, lg: 3, xl: 4 }}` 适配不同屏幕宽度。

不推荐：

- 内容高度完全一致的场景——此时普通 grid 更合适。
- 需要精确行列对齐的表格型数据。
- 项目数量极少（<5）且高度一致的场景——CSS Columns 开销大于收益。
- 为了照搬 Ant Design API 而引入本组件——它不对应任何 AntD 组件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { Masonry } from "@/components/ui/masonry"

export default function Demo() {
  return (
    <Masonry columns={3} gap={4}>
      <div className="bg-muted rounded p-4 h-32">卡片 1</div>
      <div className="bg-muted rounded p-4 h-48">卡片 2</div>
      <div className="bg-muted rounded p-4 h-40">卡片 3</div>
      <div className="bg-muted rounded p-4 h-56">卡片 4</div>
      <div className="bg-muted rounded p-4 h-36">卡片 5</div>
      <div className="bg-muted rounded p-4 h-44">卡片 6</div>
    </Masonry>
  )
}
```

### 响应式列数

```tsx
import { Masonry } from "@/components/ui/masonry"

export default function Demo() {
  return (
    <Masonry
      columns={{ sm: 1, md: 2, lg: 3, xl: 4 }}
      gap={4}
    >
      <div className="bg-muted rounded p-4 h-32">卡片 1</div>
      <div className="bg-muted rounded p-4 h-48">卡片 2</div>
      <div className="bg-muted rounded p-4 h-40">卡片 3</div>
      <div className="bg-muted rounded p-4 h-56">卡片 4</div>
    </Masonry>
  )
}
```

### 虚拟化瀑布流（大列表）

```tsx
import { VirtualMasonry, VirtualMasonryItem } from "@/components/ui/masonry"

export default function Demo() {
  return (
    <VirtualMasonry columnWidth={220} gap={16} itemHeight={240}>
      {items.map((item) => (
        <VirtualMasonryItem key={item.id}>
          <div className="bg-muted rounded p-4">{item.title}</div>
        </VirtualMasonryItem>
      ))}
    </VirtualMasonry>
  )
}
```

## 核心导出 {#exports}

- `Masonry` — CSS Columns 瀑布流布局组件（默认推荐）
- `MasonryProps`
- `VirtualMasonry` — 虚拟化瀑布流根组件（大列表场景）
- `VirtualMasonryItem` — 虚拟化瀑布流的子项容器，`VirtualMasonry` 的 children 必须是它
- `VirtualMasonryProps`

## API {#api}

类型来自 `src/components/blocks/masonry/index.tsx`（`Masonry`）与 `src/components/blocks/masonry/virtualized.tsx`（`VirtualMasonry`）。

### Masonry

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| columns | `number` \| `{ sm?: number; md?: number; lg?: number; xl?: number }` | `3` | 列数配置。传数字时为固定列数（1-6）；传对象时为响应式断点列数 |
| gap | `number` | `4` | 间距系数，实际像素值为 `gap * 4`（即默认 16px） |
| children | `ReactNode` | - | 子内容，每个子元素为一个瀑布流项 |
| className | `string` | - | 附加 className，通过 `cn` 合并 |

列数数字支持 1-6，映射到 Tailwind 的 `columns-1` 至 `columns-6`。如果传入超出范围的值，会使用 `columns-${columns}` 作为 fallback。

响应式对象支持的断点：

| 断点 | 对应 Tailwind 前缀 | 最小宽度 |
| --- | --- | --- |
| sm | `sm:` | 640px |
| md | `md:` | 768px |
| lg | `lg:` | 1024px |
| xl | `xl:` | 1280px |

### VirtualMasonry

大列表虚拟化瀑布流根组件。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| columnWidth | `number` | `200` | 单列宽度（px） |
| columnCount | `number` | - | 固定列数，传入后忽略自动计算 |
| maxColumnCount | `number` | - | 自动计算列数时的上限 |
| gap | `number \| { column: number; row: number }` | `0` | 列间距/行间距，数字表示两者相同 |
| itemHeight | `number` | `300` | 未测量前用于估算总高度的默认项高 |
| defaultWidth | `number` | - | SSR/首屏无法测量真实宽度时的兜底宽度 |
| defaultHeight | `number` | - | SSR/首屏无法测量真实高度时的兜底高度 |
| overscan | `number` | `2` | 视口外提前渲染的倍数（乘以视口高度） |
| scrollFps | `number` | `12` | 滚动位置节流帧率 |
| linear | `boolean` | `false` | `true` 时优先按索引顺序放入对应列；`false` 时严格放入最短列 |
| fallback | `ReactNode` | - | 首次挂载测量完成前的占位内容 |
| asChild | `boolean` | - | 根节点渲染委托给子元素（Radix Slot） |

### VirtualMasonryItem

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| asChild | `boolean` | - | 节点渲染委托给子元素（Radix Slot） |

`VirtualMasonry` 的 children 会被过滤，只有类型为 `VirtualMasonryItem` 的元素才会参与布局计算。

## 样式模型 {#styling-model}

- 底层使用 CSS `columns` 多列布局，浏览器原生流式排列，无 JavaScript 布局计算。
- 每个子元素包裹在 `break-inside-avoid` 容器中，防止卡片内容跨列断裂。
- 间距通过 `gap` 属性转换为 inline style `gap: {gap * 4}px` 控制列间距。
- 列数通过 Tailwind `columns-{n}` 工具类控制，响应式断点使用 `sm:`、`md:`、`lg:`、`xl:` 前缀拼接。
- 根元素携带 `data-slot="masonry"` 语义标记。
- 主题颜色优先使用 `bg-background`、`bg-muted`、`text-foreground` 等语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/masonry/index.tsx` — `Masonry` 完整实现，同时重导出 `VirtualMasonry`
- `src/components/blocks/masonry/virtualized.tsx` — `VirtualMasonry`/`VirtualMasonryItem` 完整实现
- `src/components/ui/masonry.tsx` — 重导出全部公共 API

直接依赖摘要：

- React（`ReactNode` 类型）
- Tailwind CSS `columns-*` 和 `break-inside-*` 工具类（`Masonry`）
- `@/lib/utils`（cn，`Masonry`）
- `radix-ui`（`Slot`，`VirtualMasonry`/`VirtualMasonryItem` 的 `asChild`）
- `@/lib/compose-refs`、`@/hooks/use-isomorphic-layout-effect`（`VirtualMasonry` 复用的通用 ref/effect 工具，非 UI 组件）

实现注意（`Masonry`，CSS Columns 版）：

1. 组件基于纯 CSS columns，不依赖 JavaScript 瀑布流库。
2. `gap` 值乘以 4 映射到像素，与 Tailwind 间距体系保持一致。
3. 每个子元素自动包裹 `break-inside-avoid`，防止内容在列间断开。
4. 单个 child 时不包裹数组，直接渲染在 `break-inside-avoid` 的 div 中。
5. CSS columns 布局的列顺序为"从上到下逐列排满"，DOM 顺序与视觉一致。

实现注意（`VirtualMasonry`，虚拟化版）：

1. 内部维护一棵区间树（interval tree）按 `[top, top+height]` 索引每一项，`range()` 查询可视范围内需要渲染的项，避免线性扫描全部数据。
2. 默认按"最短列优先"（`shortestColumn`）放置新项；`linear` 为 `true` 时优先保持索引对应的列（更接近网格视觉），仅在明显失衡时才退化为最短列。
3. 每项高度通过 `ResizeObserver` 自动测量并写回 positioner，首次渲染前使用 `itemHeight` 估算总高度，避免布局抖动。
4. 滚动位置通过节流（`scrollFps`，默认 12fps）读取，减少重排频率；容器宽度变化通过防抖的 `resize`/`orientationchange`/`visualViewport resize` 监听更新。
5. 只有 `VirtualMasonry` 的直接子元素中类型为 `VirtualMasonryItem` 的部分会参与定位，其余会被过滤。
6. `fallback` 用于首次挂载（客户端测量完成前）时的占位内容，避免 SSR/CSR 不一致导致的闪烁。

## 可访问性 {#accessibility}

- 使用 `data-slot="masonry"` 语义标记根元素，便于测试和样式定位。
- CSS Columns 的阅读顺序为从上到下、从左到右（列优先），DOM 顺序与视觉排列一致，屏幕阅读器按列宣读。
- 如子元素为交互组件，由子元素各自维护可访问性。
- 推荐在瀑布流项内部确保内容有良好的语义结构。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。Ant Design 不提供瀑布流/Masonry 布局组件。

| 方向 | 本地决策 |
| --- | --- |
| 实现方式 | 纯 CSS columns，无 JavaScript 布局计算 |
| API 设计 | 本地 `columns` + `gap` + `className`，无 AntD 参照 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 与 AntD Flex/Space 关系 | 不同场景：Flex/Space 用于线性排列，Masonry 用于瀑布流 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Masonry 代码时，应遵守：

1. 从 `@/components/ui/masonry` 导入 Masonry。
2. 固定列数使用 `columns={n}`（n 为 1-6）。
3. 响应式列数使用 `columns={{ sm: 1, md: 2, lg: 3 }}` 对象形式。
4. 间距使用 `gap` 属性，其值会乘以 4 转为像素（默认 4 = 16px）。
5. 子元素直接作为 children 传入，组件会自动包裹 `break-inside-avoid`。
6. 不需要为子元素手动添加 `break-inside-avoid` 或 `mb-*` 类。
7. 不要引入 `react-masonry-css`、`masonry-layout` 等第三方布局库。
8. 不要尝试用 JavaScript 计算位置或高度——组件是纯 CSS 实现。
9. 如需自定义样式，使用 `className` 附加到 Masonry 根元素。
10. 不要生成 AntD 专属的 `styles`、`classNames`、`ConfigProvider` 配置。
11. 大列表（几十条以上持续增长）或需要精确"最短列优先"排列时，改用 `VirtualMasonry` + `VirtualMasonryItem`，不要试图用 `Masonry` 手动实现虚拟化。
12. `VirtualMasonry` 的每个 children 必须包裹在 `VirtualMasonryItem` 中，其余类型的子元素会被忽略，不参与布局。
13. `VirtualMasonry` 的 `gap` 语义与 `Masonry` 不同：可以是数字，也可以是 `{ column, row }` 对象分别指定列间距和行间距；不要照搬 `Masonry` 的 `gap * 4` 换算规则。
