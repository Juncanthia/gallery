# TimePicker 时间选择框

输入或选择时间的控件。

本组件借鉴 Ant Design TimePicker 的使用心智，但不是 AntD 兼容层。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

适合在需要选择一天中具体时间的界面中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 需要复用统一设计语言、状态样式和可访问性约定。
- 需要和本项目 Tailwind 主题变量、暗色模式、Radix/Base UI primitive 或本地交互行为保持一致。
- 需要让 AI 或业务代码稳定地从 `@/components/ui/time-picker` 引用组件，而不是直接引用第三方底层库。

不推荐：

- 为了照搬 Ant Design API 而新增兼容别名。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { TimePicker } from "@/components/ui/time-picker"

export default function Demo() {
  return <TimePicker placeholder="选择时间" />
}
```

> 说明：这是面向 LLM 的最小导入示例。TimePicker 不使用 children 作为显示值，时间值通过 `value/defaultValue/onChange` 控制。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

### TimePicker

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 当前时间值（受控） | `string` | - |
| defaultValue | 默认时间值（非受控） | `string` | - |
| onChange | 时间变化回调 | `(value: string) => void` | - |
| format | 时间显示格式 | `"HH:mm:ss"` \| `"HH:mm"` | `"HH:mm:ss"` |
| use12Hours | 是否使用 12 小时制 | `boolean` | `false` |
| open | 受控弹层打开状态 | `boolean` | - |
| defaultOpen | 默认打开状态 | `boolean` | `false` |
| onOpenChange | 弹层打开状态变化回调 | `(open: boolean) => void` | - |
| openOnFocus | 聚焦触发器时自动打开弹层 | `boolean` | `false` |
| name | 原生表单提交字段名，会渲染隐藏 input | `string` | - |
| disabled | 是否禁用 | `boolean` | `false` |
| readOnly | 是否只读，只读时保留展示但禁止修改 | `boolean` | `false` |
| required | 是否为必填表单字段 | `boolean` | `false` |
| invalid | 是否标记为校验失败 | `boolean` | `false` |
| placeholder | 未选择时的占位文本 | `string` | `"选择时间"` |
| segmentPlaceholder | 面板未选择时的分段占位文本 | `string \| { hour?: string; minute?: string; second?: string; period?: string }` | `"--"` |
| hourStep | 小时列步长 | `number` | `1` |
| minuteStep | 分钟列步长 | `number` | `1` |
| secondStep | 秒列步长 | `number` | `1` |
| allowClear | 是否显示清空按钮 | `boolean` | `true` |
| className | 自定义容器样式类名 | `string` | - |
| inputGroupClassName | 自定义触发器输入区域样式类名 | `string` | - |
| triggerClassName | 自定义触发器样式类名 | `string` | - |
| contentClassName | 自定义弹层样式类名 | `string` | - |

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 如果源码使用 CVA，则变体样式集中在组件文件或相邻 `*-variants.ts` 中。
- 如果源码包装 Radix/Base UI primitive，则交互状态、键盘行为和 ARIA 基础能力来自底层 primitive，本地组件负责视觉和 API 收敛。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground`、`ring-ring` 等项目级语义变量。

## 可访问性 {#accessibility}

- 保留原生元素或底层 primitive 的键盘语义。
- 交互型组件应提供可访问名称，例如 `aria-label`、`title` 或可见文本。
- 禁用、加载、展开、选中等状态应优先使用组件已有 props，而不是只改视觉 class。
- 纯图标触发器必须提供明确的可访问名称。

## 与 Ant Design 的取舍 {#antd-differences}

| API 来源 | 以 `src/components/ui/time-picker.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |

| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 借鉴 AntD 使用心智，但不承诺 API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 TimePicker 代码时，应遵守：

1. 从 `@/components/ui/time-picker` 导入，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. `format` 只支持 `"HH:mm:ss"` 和 `"HH:mm"`，不要生成 `mm:ss` 时长输入。
4. 需要表单提交时使用 `name`，不要手写重复 hidden input。
5. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
6. 需要自定义样式时，优先使用 `triggerClassName`、`contentClassName` 等组件暴露的 props；最后才使用 `className`。
7. 交互组件必须保留键盘和 ARIA 语义，纯图标操作必须补 `aria-label`。
8. 如果不确定某个 AntD API 是否存在，先检查 `src/components/ui/time-picker.tsx`，不要凭 AntD 记忆生成。
