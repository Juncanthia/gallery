时间选择框 TimePicker。用于选择一天中的具体时间，支持受控/非受控值、12/24 小时制、表单提交字段和步长控制。

<h2 id="when-to-use">何时使用</h2>

当需要用户输入或选择特定时间时使用。

常见场景：
- 表单中的时间字段录入，如预约时间、提醒时间
- 需要精确到秒的时间选择场景
- 与日期选择器组合使用，构成完整的日期时间录入

不推荐：
- 仅需日期选择——使用 DatePicker
- 需要时间段范围选择——使用两个 TimePicker 组合
- 需要展示时间线或时间流信息——使用 Timeline 组件
- 需要录入持续时长（如 `mm:ss`）——使用专门的时长输入组件，而不是 TimePicker

```tsx

  return <TimePicker use12Hours format="HH:mm" placeholder="选择时间" />

```

```tsx

export default function Demo() {
  return (
    <TimePicker
      name="startTime"
      required
      openOnFocus
      placeholder="开始时间"
    />
  )
}
```

### 自定义步长

```tsx
import { TimePicker } from "@/components/ui/time-picker"

export default function Demo() {
  return (
    <TimePicker
      hourStep={2}
      minuteStep={15}
      placeholder="选择时间"
      format="HH:mm"
    />
  )
}
```

<h2 id="api">API</h2>

### TimePicker

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 当前时间值（受控），格式为 `HH:mm:ss` | `string` | - |
| defaultValue | 默认时间值（非受控） | `string` | - |
| onChange | 时间变化回调 | `(value: string) => void` | - |
| format | 时间显示格式 | `"HH:mm:ss"` \| `"HH:mm"` | `"HH:mm:ss"` |
| use12Hours | 是否使用 12 小时制 | `boolean` | `false` |
| open | 受控弹层打开状态 | `boolean` | - |
| defaultOpen | 默认打开状态 | `boolean` | `false` |
| onOpenChange | 弹层打开状态变化回调 | `(open: boolean) => void` | - |
| openOnFocus | 聚焦触发器时自动打开弹层 | `boolean` | `false` |
| name | 原生表单提交字段名，会渲染隐藏 input | `string` | - |
| disabled | 是否禁用 | `boolean` | `false` |
| readOnly | 是否只读，只读时保留展示但禁止修改 | `boolean` | `false` |
| required | 是否为必填表单字段 | `boolean` | `false` |
| invalid | 是否标记为校验失败 | `boolean` | `false` |
| placeholder | 未选择时的占位文本 | `string` | `"Select time"` |
| segmentPlaceholder | 面板未选择时的分段占位文本 | `string \| { hour?: string; minute?: string; second?: string; period?: string }` | `"--"` |
| hourStep | 小时列步长 | `number` | `1` |
| minuteStep | 分钟列步长 | `number` | `1` |
| secondStep | 秒列步长 | `number` | `1` |
| allowClear | 是否显示清空按钮 | `boolean` | `true` |
| className | 自定义容器样式类名 | `string` | - |
| inputGroupClassName | 自定义触发器输入区域样式类名 | `string` | - |
| triggerClassName | 自定义触发器样式类名 | `string` | - |
| contentClassName | 自定义弹层样式类名 | `string` | - |

组件使用 `useControllableState` 实现值和弹层状态的受控/非受控双模式。触发按钮内嵌 Clock 图标，点击后弹出 Popover 面板，通过滚动列选择时、分、秒和 AM/PM。设置 `name` 后会同步隐藏 input，便于原生表单提交。

<h2 id="faq">常见问题</h2>

### 如何只选择小时和分钟，不要秒？
设置 `format="HH:mm"`，秒列将不会显示，时间值为 `HH:mm` 格式。

### 12 小时制下如何区分上午和下午？
启用 `use12Hours` 后，小时列显示 1-12，同时增加 AM/PM 时段列。组件内部仍以 24 小时字符串保存和回调，如 `14:30`。

### 如何获取选中时间的 Date 对象？
组件 `onChange` 回调返回 `string`（如 `"14:30:00"`），如需 `Date` 对象，请在回调中自行解析。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/time-picker` 导入，不要从 `antd` 导入。
2. 受控模式使用 `value` + `onChange`；非受控模式使用 `defaultValue`。
3. `format` 决定显示和值的格式，支持 `"HH:mm:ss"`、`"HH:mm"` 两种；不要把 TimePicker 用作 `mm:ss` 时长输入。
4. 需要步长控制时使用 `hourStep`、`minuteStep`、`secondStep`，值为正整数。
5. 需要原生表单提交时使用 `name`，组件会生成隐藏 input。
6. 不要引入 Ant Design 相关依赖。
