# Audio 音频

用于在 AI 对话中嵌入音频内容，支持两种展示形态和播放控制。

本组件是项目内的本地基础能力，API 和行为以源码为准，不对应 Ant Design 的完整公共组件契约。

## 何时使用 {#when-to-use}

适合在需要内嵌音频播放展示相关能力的界面中使用。优先通过组件自身 props 表达状态和语义。

## 核心导出 {#exports}

- `Audio`
- `AudioProps`
- `SerializableAudio`
- `Source`
- `AudioVariant`

## 代码演示 {#examples}

### 基础用法

```tsx
import { Audio } from "@/components/core/audio"

export default function Demo() {
  return (
    <Audio
      id="demo-audio"
      assetId="demo-audio-asset"
      src="https://example.com/audio.mp3"
    />
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。`id`、`assetId`、`src` 为必填属性，其余可选。

### 完整示例

```tsx
import { Audio } from "@/components/core/audio"

export default function Demo() {
  return (
    <Audio
      id="demo-audio"
      assetId="demo-audio-asset"
      src="https://example.com/audio.mp3"
      title="Audio Title"
      description="A detailed audio description."
      artwork="https://example.com/artwork.jpg"
      durationMs={240000}
      fileSizeBytes={4800000}
      createdAt="2026-06-30T12:00:00.000Z"
      locale="zh-CN"
      variant="full"
      source={{
        label: "Source Name",
        url: "https://example.com/audio",
      }}
      onMediaEvent={(type) => console.log("Event", type)}
    />
  )
}
```

### Compact 紧凑模式

```tsx
import { Audio } from "@/components/core/audio"

export default function Demo() {
  return (
    <Audio
      id="compact-audio"
      assetId="compact-audio-asset"
      src="https://example.com/audio.mp3"
      variant="compact"
      title="Compact Player"
      description="A compact inline player."
      artwork="https://example.com/artwork.jpg"
    />
  )
}
```

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface AudioProps extends SerializableAudio {
  variant?: "full" | "compact";  // 默认 "full"
  className?: string;
  onMediaEvent?: (type: "play" | "pause" | "mute" | "unmute") => void;
}
```

```ts
interface SerializableAudio {
  id: string;               // 必填
  assetId: string;          // 必填
  src: string;              // URL，必填
  title?: string;
  description?: string;
  artwork?: string;          // URL
  durationMs?: number;       // 毫秒，正整数
  fileSizeBytes?: number;    // 字节，正整数
  createdAt?: string;        // ISO 8601 datetime
  locale?: string;           // 默认 "en-US"
  source?: {
    label: string;
    iconUrl?: string;        // URL
    url?: string;            // URL
  };
  role?: string;
  receipt?: object;
}
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `data-slot="audio"` — 根容器
- `data-tool-ui-id` — 组件 id

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 组件使用 `group` 和 `@container` 实现容器查询。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground` 等项目级语义变量。
- full 模式最大宽度 `max-w-sm`，compact 模式 `min-w-72 max-w-md`。
- 卡片使用 `rounded-xl` 和 `shadow-xs`。

## 约束与注意事项 {#constraints}

1. `src` 必须是完整的 HTTPS URL，指向可直接播放的音频文件（如 .mp3）。
2. `artwork` 必须是完整的 HTTPS URL，指向封面图片。
3. `durationMs` 单位为毫秒，传入正整数。组件会自动格式化为 `m:ss`。
4. `createdAt` 使用 ISO 8601 datetime 格式。
5. `source.url` 和 `source.iconUrl` 需为合法 URL 字符串。
6. 组件使用内部 `AudioProvider` Context 管理播放状态，`useAudio` hook 仅在组件内部使用。
7. full 模式的封面图以 4:3 宽高比展示在卡片顶部。
8. compact 模式的封面缩略图尺寸为 48x48（size-12），背景有模糊处理效果。

## 可访问性 {#accessibility}

- 原生 `<audio>` 元素提供基础的音频播放能力。
- 播放/暂停按钮携带 `aria-label`（"Play" 或 "Pause"），自动切换。
- 进度条使用 `aria-label="Audio progress"`。
- 封面图标记 `aria-hidden="true"`，不暴露给辅助技术。
- 根元素使用 `<article>` 语义标签，`lang` 属性由 `locale` 控制。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Audio 代码时，应遵守：

1. 从 `@/components/core/audio` 导入，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置。
4. `id`、`assetId`、`src` 为必填属性，缺少会导致 schema 校验失败。
5. `src`、`artwork`、`source.url`、`source.iconUrl` 必须为完整的 HTTPS URL。
6. `durationMs` 单位为毫秒，`createdAt` 为 ISO 8601 datetime 格式。
7. 需要自定义样式时，优先使用组件暴露的 props；最后才使用 `className`。
8. 不要绕过 Audio 组件直接使用原生 `<audio>` 标签。
9. 如果不确定某个 props 是否存在，先检查 `src/components/tool/audio/audio.tsx`，不要凭记忆生成。
