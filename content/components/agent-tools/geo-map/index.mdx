GeoMap 地理地图组件。基于 Leaflet 封装的交互式地图，支持标记点（markers）和路线（routes）叠加，提供聚类（clustering）、视口控制（viewport）和深色/浅色主题自适应。

<h2 id="when-to-use">何时使用</h2>

当需要在界面中以地图形式展示地理位置信息、路径轨迹或区域分布时使用。

常见场景：

- 展示公司办公室、门店、仓库等位置的分布概览。
- 物流追踪场景中展示配送路线或运输路径。
- 数据分析场景中以地图标注数据点的地理分布。
- 行程规划中展示途经地点和路线规划。

推荐使用方式：

- 通过 `markers` 传入标记点数组，每个标记至少包含 `lat`（纬度）和 `lng`（经度）。
- 通过 `routes` 传入路线数组，每条路线至少包含 2 个路径点。
- 通过 `viewport` 控制地图初始视图，支持 `fit`（自适应）和 `center`（中心定位）两种模式。
- 使用 `title` 和 `description` 在地图左上角叠加标题面板。

不推荐：

- 在移动端小屏幕上展示过多标记点，建议开启 `clustering` 聚类功能。
- 将 GeoMap 用于精确导航——组件基于开放地图瓦片，不提供实时路况或导航功能。
- 在无网络环境下使用，组件依赖外部地图瓦片服务（CartoCDN）。

<h2 id="examples">代码演示</h2>

### 基础用法

传入 `id`、`markers` 和 `viewport` 展示基础地图。

<ComponentShowcase id="tool/geo-map" path="tool/geo-map/basic" />

```tsx
import { GeoMap } from "@/components/core/geo-map"

export default function Demo() {
  return (
    <GeoMap
      id="geo-map-demo"
      title="办公室分布"
      description="全球主要办公地点"
      markers={[
        { id: "bj", lat: 39.9042, lng: 116.4074, label: "北京" },
        { id: "sh", lat: 31.2304, lng: 121.4737, label: "上海" },
      ]}
      viewport={{ mode: "fit", target: "markers" }}
    />
  )
}
```

### 带路线

通过 `routes` 属性叠加路线展示，每条路线支持自定义颜色、线宽和虚线样式。

```tsx
import { GeoMap } from "@/components/core/geo-map"

export default function Demo() {
  return (
    <GeoMap
      id="route-demo"
      title="配送路线"
      markers={[
        { id: "wh", lat: 30.5928, lng: 114.3055, label: "武汉仓库" },
        { id: "hz", lat: 30.2741, lng: 120.1551, label: "杭州配送站" },
        { id: "nj", lat: 32.0603, lng: 118.7969, label: "南京配送站" },
      ]}
      routes={[
        {
          id: "r1",
          points: [
            { lat: 30.5928, lng: 114.3055 },
            { lat: 30.2741, lng: 120.1551 },
          ],
          label: "武汉→杭州",
          color: "#3b82f6",
          weight: 3,
        },
        {
          id: "r2",
          points: [
            { lat: 30.5928, lng: 114.3055 },
            { lat: 32.0603, lng: 118.7969 },
          ],
          label: "武汉→南京",
          color: "#ef4444",
          weight: 3,
          dashArray: "8 4",
        },
      ]}
      viewport={{ mode: "fit", target: "all" }}
    />
  )
}
```

### 深色主题

通过 `theme` 属性显式指定主题，不传则跟随系统或文档主题自动切换。

```tsx
import { GeoMap } from "@/components/core/geo-map"

export default function Demo() {
  return (
    <GeoMap
      id="dark-demo"
      theme="dark"
      title="夜间模式地图"
      markers={[
        { id: "p1", lat: 48.8566, lng: 2.3522, label: "巴黎" },
        { id: "p2", lat: 51.5074, lng: -0.1278, label: "伦敦" },
      ]}
      viewport={{ mode: "fit", target: "markers", padding: 60 }}
    />
  )
}
```

### 标记聚类

当标记点数量较多时，通过 `clustering` 开启聚类功能，相邻标记自动合并。

```tsx
import { GeoMap } from "@/components/core/geo-map"

export default function Demo() {
  return (
    <GeoMap
      id="cluster-demo"
      title="门店分布"
      markers={[
        { lat: 39.9042, lng: 116.4074, label: "北京1" },
        { lat: 39.9142, lng: 116.4174, label: "北京2" },
        { lat: 39.9242, lng: 116.3974, label: "北京3" },
        { lat: 31.2304, lng: 121.4737, label: "上海1" },
        { lat: 31.2204, lng: 121.4637, label: "上海2" },
        { lat: 22.5431, lng: 114.0579, label: "深圳1" },
        { lat: 22.5531, lng: 114.0679, label: "深圳2" },
      ]}
      clustering={{ enabled: true, radius: 60 }}
      viewport={{ mode: "fit", target: "markers" }}
    />
  )
}
```

### 三种标记图标类型

`markers` 中每个标记的 `icon` 支持 `dot`（圆点）、`emoji`（emoji 符号）和 `image`（图片）三种类型。

```tsx
import { GeoMap } from "@/components/core/geo-map"

export default function Demo() {
  return (
    <GeoMap
      id="icons-demo"
      markers={[
        {
          lat: 40.7128,
          lng: -74.006,
          label: "Dot 标记",
          icon: { type: "dot", color: "#22c55e", radius: 8 },
        },
        {
          lat: 34.0522,
          lng: -118.2437,
          label: "Emoji 标记",
          icon: { type: "emoji", value: "🌴" },
        },
        {
          lat: 41.8781,
          lng: -87.6298,
          label: "图片标记",
          icon: {
            type: "image",
            url: "https://flagcdn.com/w40/us.png",
            width: 24,
            height: 24,
          },
        },
      ]}
      viewport={{ mode: "fit", target: "markers", padding: 40 }}
    />
  )
}
```

<h2 id="api">API</h2>

### GeoMap

地理地图组件。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 组件唯一标识 | `string` | - |
| title | 地图标题，显示在左上角叠加面板中 | `string` | - |
| description | 地图描述，显示在标题下方 | `string` | - |
| markers | 标记点数组，至少包含 1 项 | `GeoMapMarker[]` | - |
| routes | 路线数组 | `GeoMapRoute[]` | - |
| clustering | 聚类配置 | `GeoMapClustering` | - |
| viewport | 视口控制，支持 fit 和 center 模式 | `GeoMapViewport` | - |
| showZoomControl | 是否显示缩放控件 | `boolean` | `true` |
| theme | 主题模式，不传则自动跟随系统 | `"light"` \| `"dark"` | 自动 |
| className | 附加样式类名 | `string` | - |
| style | 内联样式及 CSS 变量 | `GeoMapStyle` | - |
| tooltipClassName | 提示框附加样式 | `string` | - |
| popupClassName | 弹出框附加样式 | `string` | - |
| onMarkerClick | 标记点击回调 | `(marker: GeoMapMarker) => void` | - |
| onRouteClick | 路线点击回调 | `(route: GeoMapRoute) => void` | - |

### GeoMapMarker

标记点数据结构。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 标记唯一标识（同组件内不可重复） | `string` | - |
| lat | 纬度，范围 -90 ~ 90 | `number` | - |
| lng | 经度，范围 -180 ~ 180 | `number` | - |
| label | 标记名称，显示在 popup 或 tooltip 中 | `string` | - |
| description | 标记描述，显示在 label 下方 | `string` | - |
| tooltip | tooltip 显示策略 | `"none"` \| `"hover"` \| `"always"` | - |
| icon | 标记图标配置 | `GeoMapMarkerIcon` | - |

### GeoMapMarkerIcon

标记图标配置（联合类型）。

**Dot 类型：**

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 图标类型 | `"dot"` | - |
| color | 圆点颜色 | `string` | - |
| borderColor | 边框颜色 | `string` | - |
| radius | 圆点半径，范围 3~16 | `number` | - |

**Emoji 类型：**

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 图标类型 | `"emoji"` | - |
| value | emoji 字符 | `string` | - |
| size | 字号，范围 16~40 | `number` | - |
| bgColor | 背景色 | `string` | - |
| borderColor | 边框颜色 | `string` | - |

**Image 类型：**

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 图标类型 | `"image"` | - |
| url | 图片 URL（必须 http/https） | `string` | - |
| width | 宽度，范围 16~64 | `number` | - |
| height | 高度，范围 16~64 | `number` | - |
| borderRadius | 圆角 | `number` | - |
| borderColor | 边框颜色 | `string` | - |

### GeoMapRoute

路线数据结构。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 路线唯一标识（同组件内不可重复） | `string` | - |
| points | 路径点数组，至少包含 2 个点 | `{ lat: number; lng: number }[]` | - |
| label | 路线名称 | `string` | - |
| description | 路线描述 | `string` | - |
| tooltip | tooltip 显示策略 | `"none"` \| `"hover"` \| `"always"` | - |
| color | 路线颜色 | `string` | - |
| weight | 线宽，范围 1~12 | `number` | - |
| opacity | 透明度，范围 0~1 | `number` | - |
| dashArray | 虚线样式，如 `"8 4"` | `string` | - |

### GeoMapClustering

标记聚类配置。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| enabled | 是否启用聚类 | `boolean` | - |
| radius | 聚类半径（像素），范围 20~120 | `number` | - |
| maxZoom | 停止聚类的最大缩放级别，范围 1~22 | `number` | - |
| minPoints | 形成聚类的最小点数，范围 2~20 | `number` | - |

### GeoMapViewport

视口控制（联合类型）。

**Fit 模式（自适应）：**

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| mode | 视口模式 | `"fit"` | - |
| padding | 内边距（像素） | `number` | - |
| maxZoom | 适应时的最大缩放级别 | `number` | - |
| target | 适应目标 | `"markers"` \| `"routes"` \| `"all"` | - |

**Center 模式（中心定位）：**

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| mode | 视口模式 | `"center"` | - |
| center | 中心点坐标 | `{ lat: number; lng: number }` | - |
| zoom | 缩放级别，范围 1~22 | `number` | - |

<h2 id="faq">常见问题</h2>

### 地图不显示怎么办？

GeoMap 依赖外部地图瓦片服务（CartoCDN），需要网络连接。如果地图显示空白：

1. 检查是否正确设置了 `viewport`（fit 模式需要至少一个 marker 或 route）。
2. 检查 `markers` 数组是否至少包含一项。
3. 确认网络环境可以访问 `basemaps.cartocdn.com`。

### 如何控制地图初始视图？

使用 `viewport` 属性：

- `fit` 模式：地图自动缩放至适应所有标记或路线，推荐大多数场景使用。
- `center` 模式：手动指定中心点和缩放级别，适合需要精确控制的场景。

### 何时使用聚类？

当同一区域标记点数量超过 5 个时，建议开启 `clustering`，传入 `{ enabled: true }`。可进一步通过 `radius`、`maxZoom` 等参数微调聚类行为。

### 支持自定义地图样式吗？

当前使用的是 CartoCDN 的 light_all / dark_all 瓦片。如需自定义瓦片源，需要在源码层面修改 `LIGHT_TILE_URL` 和 `DARK_TILE_URL` 常量。

### 如何在服务端生成地图快照？

GeoMap 是纯客户端组件（"use client"），依赖浏览器 DOM 和 Leaflet。服务端渲染场景建议使用静态地图 API 替代。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/geo-map` 导入 `GeoMap`。
2. `id`、`markers` 为必填属性。`markers` 数组至少包含 1 项，每项必须含 `lat` 和 `lng`。
3. `lat` 范围 -90~90，`lng` 范围 -180~180，超出范围的经纬度会导致标记无法正确显示。
4. 使用 `viewport` 控制初始视图：`fit` 模式自适应内容，`center` 模式手动定位。推荐使用 `fit` 模式。
5. 同一组件内 `markers` 的 `id` 不可重复，`routes` 的 `id` 也不可重复。
6. `icon` 为 `image` 类型时，`url` 必须是合法的 http/https 地址。
7. 不要在外部包裹额外的 Card 或 Panel 组件，GeoMap 内部已集成标题面板和边框容器。
8. 不要引入 Ant Design 相关依赖。
9. 不要在移动端小屏幕上展示过多标记点，建议开启 `clustering` 聚类功能。
10. `theme` 不传时自动跟随系统/文档主题（通过 `prefers-color-scheme` 和 DOM class 检测），不需要手动判断。
11. HTML 文本中使用 `&lt;` 代替 `<` 符号。
