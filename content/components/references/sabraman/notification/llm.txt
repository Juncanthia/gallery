# LegacyNotification 通知

macOS 风格系统通知组件，支持标题、副标题、正文、图标和时间戳。

本组件是 Sabraman 体系的 macOS 风格通知组件，不是 Ant Design references/notification 兼容层。视觉上完全遵循 macOS 通知设计规范（黑色半透明毛玻璃、圆角图标、顶部高光渐变），底层函数式调用基于 sonner toast。

## 何时使用 {#when-to-use}

适合需要展示系统级操作反馈或信息提示的场景，要求用户可感知但不打断当前操作。

常见场景：

- 操作结果反馈：保存成功、删除完成、提交确认。
- 系统消息提醒：新版本发布、维护通知、权限变更。
- 状态更新通知：任务完成、审批通过、数据同步完毕。
- 即时通讯提示：新消息、未读提醒。

不推荐：

- 用 LegacyNotification 替代 Modal 做复杂交互。
- 在通知中放置过长文本——通知设计为轻量信息展示。
- 引入 `antd` 的 references/notification 或 message 来覆盖 LegacyNotification 的功能范围。

## 代码演示 {#examples}

### 基础用法

```tsx
import { LegacyNotification } from "@/components/legacy-ui"

export default function Demo() {
  return (
    <LegacyNotification
      title="提醒"
      subtitle="新消息"
      body="您有一条新的系统通知"
      time="刚刚"
    />
  )
}
```

> 说明：渲染组件从 `@/components/legacy-ui` 导入；函数式调用 `showLegacyNotification` 从 `@/components/legacy-ui` 导入。

## 核心导出 {#exports}

从 `@/components/legacy-ui`：

- `LegacyNotification`
- `LegacyNotificationProps`

从 `@/components/legacy-ui`（完整导出）：

- `showLegacyNotification`
- `ShowLegacyNotificationOptions`

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface LegacyNotificationProps extends React.HTMLAttributes<HTMLDivElement> {
  /** 正文内容。@default "Body" */
  body?: string
  /** 自定义图标。 */
  icon?: React.ReactNode
  /** 是否显示图标区域。@default true */
  showIcon?: boolean
  /** 是否显示正文。@default true */
  showBody?: boolean
  /** 是否显示副标题。@default true */
  showSubtitle?: boolean
  /** 副标题文字。@default "Subtitle" */
  subtitle?: string
  /** 时间戳文字，显示在标题右侧。@default "now" */
  time?: string
  /** 标题文字。@default "Title" */
  title?: string
}
```

```ts
interface ShowLegacyNotificationOptions extends Omit<LegacyNotificationProps, "className" | "style"> {
  /** sonner toast 配置，可覆写 duration、position 等。@default { duration: 4200, position: "top-center", unstyled: true } */
  toastOptions?: ExternalToast
}
```

```ts
function showLegacyNotification(options: ShowLegacyNotificationOptions): string | number
```

## Semantic DOM {#semantic-dom}

LegacyNotification 当前未设置 `data-slot` 属性。组件内部结构：

- 容器 div：根节点，`rounded-[8px]`、`border`、`bg-black/80`
- 图标容器 div：`size-[29px]` 圆角灰色渐变区域
- 文字容器 div：`flex-1` 纵向布局，包含 title+time、subtitle、body

## 样式模型 {#styling-model}

- 视觉风格：macOS 通知设计——黑色半透明背景 `bg-black/80`，`rounded-[8px]` 圆角，阴影和边框叠加。
- 顶部高光：绝对定位渐变层模拟 macOS 顶部光泽。
- 图标区域：`size-[29px]` 灰色渐变圆角背景 + 内阴影，`[&_svg]:size-[17px]`。
- 文字区域：白色文字 + `text-shadow`，`text-ellipsis` 溢出截断。
- 字体：固定 `"Helvetica Neue", Helvetica, Arial, sans-serif`。
- 函数式调用：sonner toast `unstyled: true`，组件样式完全自包含。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/sabraman/components/legacy-references/notification.tsx`：LegacyNotification 及 showLegacyNotification 定义。
- `src/components/core/sabraman-references/notification.tsx`：UI 层重导出（仅导出 LegacyNotification 和 LegacyNotificationProps）。

直接依赖：

- React、Tailwind CSS、sonner（函数式调用）、lucide-react（推荐图标库）。

实现注意：

1. `showLegacyNotification` 内部调用 `toast.custom()`，默认 4200ms 自动关闭，`position: "top-center"`。
2. LegacyNotification 为纯展示组件，无交互状态，所有内容通过 props 控制。
3. `showBody`、`showSubtitle`、`showIcon` 控制条件渲染，隐藏时不占位（非 `visibility: hidden`）。
4. 组件使用 `React.forwardRef` 支持 ref 透传。
5. `showLegacyNotification` 返回 sonner toast ID，可用于 `toast.dismiss(id)`。
6. 如果需要新增能力，应先修改 `legacy-references/notification.tsx` 源码，再更新本文件。

## 可访问性 {#accessibility}

- LegacyNotification 使用原生 `<div>`，当前未设置 ARIA role。调用侧可加 `role="blocks/status"` 或 `role="alert"`。
- `showLegacyNotification` 弹窗由 sonner 的 `aria-live` 区域承载。
- 图标区域为纯装饰性元素，不含交互。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/sabraman/components/legacy-references/notification.tsx` 的 TypeScript 类型为准 |
| 视觉风格 | macOS 黑色半透明毛玻璃，非 AntD 白底卡片 |
| 函数式调用 | 基于 sonner toast，非 AntD 自建 references/notification 实例 |
| 通知类型 | 无 type（info/success/warning/error），通过 icon 自由表达语义 |
| 样式运行时 | Tailwind CSS 内联，不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 借鉴通知使用心智，但不承诺 API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 LegacyNotification 代码时，应遵守：

1. 渲染组件从 `@/components/legacy-ui` 导入；函数式调用从 `@/components/legacy-ui` 导入 `showLegacyNotification`。
2. 不要从 `antd` 导入 references/notification 或 message。
3. 只使用本文件 API 摘要和源码中存在的 props。
4. 通知文案通过 `title`、`subtitle`、`body`、`time` 控制，不要拼接 HTML 或 JSX children。
5. 自定义图标使用 `icon` 属性传入 ReactNode，推荐 lucide-react 图标。
6. 函数式调用时，`toastOptions` 用于覆写 sonner 配置（duration、position 等）。
7. 不引入 Ant Design 的 ConfigProvider、Design Token 或 css-in-js。
8. 如果不确定某个 API 是否存在，先检查 `src/components/sabraman/components/legacy-references/notification.tsx`。
