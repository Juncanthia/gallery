文字提示 Tooltip。简单的文字提示气泡框，在鼠标悬停或聚焦时展示简短的提示文本。基于 Radix Tooltip 实现，支持 12 种弹出位置及自定义颜色。

<h2 id="when-to-use">何时使用</h2>

当需要为界面元素补充说明文字，且文字简短时使用。Tooltip 在用户 hover 或 focus 后短暂延迟出现，移开即消失。

常见场景：
- 为纯图标按钮提供文字说明（如工具栏中的加粗、斜体按钮）。
- 为截断文本展示完整内容（表格单元格、列表项等）。
- 为缩写、专业术语或状态标识提供解释。
- 表单字段旁的解释性提示。

不推荐：
- 需要展示大段文字——使用 Popover 或 HoverCard，Tooltip 内容建议控制在 10 字以内。
- 需要用户进行交互操作（如点击链接、填写表单）——Tooltip 鼠标移开即消失，无法承载交互。
- 需要高亮或持久展示的信息——使用 Alert 或 Banner。

<h2 id="examples">代码演示</h2>

### 基础用法

`TooltipProvider` 在应用顶层包裹一次，后续每个 `Tooltip` 只需 `TooltipTrigger` + `TooltipContent`。

<ComponentShowcase id="tooltip" path="tooltip/basic" />

```tsx
import { Tooltip, TooltipContent, TooltipProvider, TooltipTrigger } from "@/components/ui/tooltip"
import { Button } from "@/components/ui/button"

export default function Demo() {
  return (
    <TooltipProvider>
      <div className="flex flex-wrap items-center gap-4">
        <Tooltip>
          <TooltipTrigger asChild>
            <Button variant="outlined">悬停我</Button>
          </TooltipTrigger>
          <TooltipContent><p>提示文本</p></TooltipContent>
        </Tooltip>
        <Tooltip>
          <TooltipTrigger asChild>
            <Button variant="text" size="small">上方</Button>
          </TooltipTrigger>
          <TooltipContent side="top">顶部弹出</TooltipContent>
        </Tooltip>
      </div>
    </TooltipProvider>
  )
}
```

### 快捷用法

使用 `title` 和 `placement` 属性可以快速创建提示，无需手动组合 `TooltipTrigger` 和 `TooltipContent`。

```tsx
import { Tooltip, TooltipProvider } from "@/components/ui/tooltip"
import { Button } from "@/components/ui/button"

export default function Demo() {
  return (
    <TooltipProvider>
      <Tooltip title="保存当前内容" placement="top">
        <Button variant="outlined">保存</Button>
      </Tooltip>
    </TooltipProvider>
  )
}
```

### 自定义颜色与箭头

```tsx
import { Tooltip, TooltipContent, TooltipProvider, TooltipTrigger } from "@/components/ui/tooltip"

export default function Demo() {
  return (
    <TooltipProvider>
      <Tooltip>
        <TooltipTrigger asChild>
          <span className="text-sm underline cursor-help">自定义样式</span>
        </TooltipTrigger>
        <TooltipContent color="#1e293b" arrow>
          深色背景带箭头
        </TooltipContent>
      </Tooltip>
    </TooltipProvider>
  )
}
```

<h2 id="api">API</h2>

### TooltipProvider

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| delayDuration | 延迟显示时间（毫秒） | `number` | `0` |
| skipDelayDuration | 从一个提示切换到另一个时的延迟（毫秒） | `number` | `300` |
| disableHoverableContent | 是否禁止悬停至内容区保持提示 | `boolean` | `false` |

### Tooltip

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| title | 快捷提示文本，传入后自动生成 Trigger + Content | `ReactNode` | - |
| placement | 快捷弹出位置 | `TooltipPlacement` | - |
| color | 自定义背景色（十六进制值） | `string` | - |
| arrow | 是否显示箭头 | `boolean` | `false` |
| contentProps | 透传给 TooltipContent 的属性 | `TooltipContentProps` | - |
| open | 受控显隐状态 | `boolean` | - |
| defaultOpen | 非受控默认显隐 | `boolean` | `false` |
| onOpenChange | 显隐状态变化回调 | `(open: boolean) => void` | - |
| delayDuration | 覆盖 Provider 的延迟时间 | `number` | - |
| disableHoverableContent | 覆盖 Provider 的悬停内容行为 | `boolean` | - |

### TooltipTrigger

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 是否将子元素作为触发器（不额外包裹 DOM） | `boolean` | `false` |
| children | 触发器内容 | `ReactNode` | - |

### TooltipContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| side | 弹出方向 | `'top' \| 'bottom' \| 'left' \| 'right'` | `'top'` |
| align | 对齐方式 | `'start' \| 'center' \| 'end'` | `'center'` |
| sideOffset | 距离触发器的偏移量（像素） | `number` | `4` |
| alignOffset | 交叉轴偏移量 | `number` | `0` |
| arrow | 是否显示箭头 | `boolean` | `false` |
| color | 自定义背景色（十六进制值） | `string` | - |
| className | 自定义样式类名 | `string` | - |

### TooltipArrow

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| color | 箭头颜色（默认为 TooltipContent 的背景色） | `string` | - |
| className | 自定义样式 | `string` | - |

### TooltipPlacement

快捷方向枚举，内部自动映射为 `side` + `align`：

`'top'` | `'bottom'` | `'left'` | `'right'` | `'topLeft'` | `'topRight'` | `'bottomLeft'` | `'bottomRight'` | `'leftTop'` | `'leftBottom'` | `'rightTop'` | `'rightBottom'`

<h2 id="faq">常见问题</h2>

### Tooltip 和 Popover 的区别？
Tooltip 用于展示简短提示文本（建议 10 字以内），hover 后短暂延迟出现，移开即消失，不可交互。Popover 可容纳更多内容（标题、描述、按钮），通常由点击触发，支持复杂交互。两者互补，不可混用。

### 如何禁用全局的 Tooltip 延迟？
在 `TooltipProvider` 上设置 `delayDuration={0}`，所有 Tooltip 将即时显示。如需对不同 Tooltip 设置不同延迟，可在单个 `Tooltip` 上覆盖 `delayDuration`。

### title 快捷用法和 Trigger+Content 组合用法的区别？
`title` 快捷方式适合简单场景，组件自动创建 Trigger 和 Content。当需要自定义触发器（如 `asChild` 透传按钮）或自定义内容样式时，使用完整的 `TooltipTrigger` + `TooltipContent` 组合。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/tooltip` 导入，不要从 `antd` 导入。
2. 全局只需一次 `TooltipProvider`，推荐在应用根布局中放置。
3. `TooltipTrigger` 包裹触发器元素时使用 `asChild` 避免多余 DOM。
4. Tooltip 内容建议控制在 10 字以内，超出应使用 Popover 或 HoverCard。
5. 纯图标按钮作为触发器时必须提供 `aria-label` 或可见文本。
6. 需要自定义颜色时使用 `color` prop（支持十六进制值），组件会自动计算对比文字色。
7. 不要引入 Ant Design 相关依赖。
