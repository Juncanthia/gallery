DataTableToolbar 表格工具栏，提供筛选面板切换、行数统计、重置筛选、视图选项等功能集成栏，是 DataTable 筛选体验的核心操作入口。

<h2 id="when-to-use">何时使用</h2>

当需要在 DataTable 筛选场景中为用户提供统一的筛选操作入口时使用。DataTableToolbar 集成了筛选面板的展开/折叠控制、已过滤/总行数显示、筛选重置、自定义操作区和视图列选项。

常见场景：

- 数据表格顶部的筛选操作工具栏。
- 需要显示当前筛选结果的行数统计。
- 需要一键重置所有筛选条件。
- 需要自定义操作按钮（如导出、批量删除等），通过 `renderActions` 注入。
- 需要切换可见列、调整列顺序，通过内置的 DataTableViewOptions 实现。
- 移动端通过 DataTableFilterControlsDrawer 提供抽屉式筛选面板。

不推荐：

- 在 DataTableProvider 外部使用，DataTableToolbar 依赖 `useDataTable()` 获取表格上下文。
- 需要完整表格渲染（含表头、排序等）时，应配合 DataTableColumnHeader 等组件使用，DataTableToolbar 本身不渲染表格内容。

<h2 id="api">API</h2>

### DataTableToolbar

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| renderActions | 自定义操作区渲染函数 | `() => React.ReactNode` | - |

DataTableToolbar 必须在 `DataTableProvider` 内部使用，通过 `useDataTable()` 获取表格上下文（`table`、`isLoading`、`columnFilters`、`totalRows`、`filterRows`）。

### 内部组件

DataTableToolbar 内部集成了以下子组件：

| 子组件 | 说明 | 显示条件 |
| --- | --- | --- |
| 筛选面板切换按钮 | 展开/折叠 ControlsProvider 管理的筛选面板，快捷键 `⌘B` | 非移动端（`hidden sm:flex`） |
| DataTableFilterControlsDrawer | 移动端抽屉式筛选面板 | 仅移动端（`block sm:hidden`） |
| 行数统计 | 显示已过滤行数 / 总行数 | 始终显示，格式使用 `formatCompactNumber` |
| DataTableResetButton | 重置所有筛选条件，快捷键 `Esc` | 仅当 `columnFilters.length > 0` 时显示 |
| DataTableViewOptions | 列可见性切换和拖拽排序 | 始终显示 |

<h2 id="faq">常见问题</h2>

### DataTableToolbar 与 DataTableProvider 的关系？

DataTableToolbar 必须在 DataTableProvider 内部使用，通过 `useDataTable()` 获取表格上下文：

```tsx
&lt;DataTableProvider table={table} columns={columns} filterFields={filterFields}&gt;
  &lt;DataTableToolbar /&gt;
&lt;/DataTableProvider&gt;
```

### 如何自定义工具栏的操作按钮？

通过 `renderActions` 属性注入自定义 React 节点：

```tsx
&lt;DataTableToolbar
  renderActions={() =&gt; (
    &lt;div className="flex gap-2"&gt;
      &lt;Button variant="outline"&gt;导出&lt;/Button&gt;
      &lt;Button&gt;创建&lt;/Button&gt;
    &lt;/div&gt;
  )}
/&gt;
```

渲染位置在重置按钮之后、视图选项按钮之前。

### 如何在移动端使用？

DataTableToolbar 自动适配移动端：筛选面板切换按钮在移动端隐藏（`hidden sm:flex`），取而代之的是 DataTableFilterControlsDrawer（抽屉式筛选面板），通过 `block sm:hidden` 仅在移动端显示。

### 行数统计的数据来源？

行数统计优先使用 `DataTableProvider` 传入的 `totalRows` 和 `filterRows` 属性，适用于服务端分页/筛选场景。如果未传入，则 fallback 到 TanStack Table 的 `getCoreRowModel().rows.length` 和 `getFilteredRowModel().rows.length`，适用于客户端筛选场景。

### 筛选面板展开状态如何持久化？

DataTableProvider 内部包裹了 ControlsProvider，ControlsProvider 使用 `useLocalStorage` 持久化展开状态。页面刷新后自动恢复上次的展开/折叠状态。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 DataTableToolbar 代码时：
