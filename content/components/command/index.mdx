Command 命令面板，提供搜索过滤和快捷指令选择。基于 cmdk 实现，支持 CommandDialog 弹窗模式和内联模式，常用于 Cmd+K 全局快捷键面板。

<h2 id="when-to-use">何时使用</h2>

当需要提供一个快速的命令搜索和键盘导航选择交互界面时使用。

常见场景：

- Cmd+K 全局命令面板：应用级快捷键唤起命令搜索。
- 快速搜索和导航：在大量候选项中通过关键词实时过滤。
- 选项过滤和选择：搜索框 + 键盘导航 + 选中标记的组合交互。
- 内嵌搜索选择器：作为其他复杂组件（如 TreeSelect）的内部交互层。

不推荐：

- 简单下拉选择场景，使用 Select 或 Combobox 更合适。
- 不需要搜索过滤的纯菜单交互，使用 DropdownMenu 或 ContextMenu。
- 移动端全屏命令面板，键盘导航在触屏设备上体验不佳。

### CommandDialog 弹窗模式

<ComponentShowcase id="command-dialog" path="command/dialog" />

```tsx
import { useState } from "react"
import { Button } from "@/components/ui/button"
import {
  CommandDialog,
  CommandInput,
  CommandList,
  CommandEmpty,
  CommandGroup,
  CommandItem,
} from "@/components/ui/command"

export default function Demo() {
  const [open, setOpen] = useState(false)

  return (
    <>
      <Button onClick={() => setOpen(true)}>打开命令面板</Button>
      <CommandDialog open={open} onOpenChange={setOpen}>
        <CommandInput placeholder="输入命令..." />
        <CommandList>
          <CommandEmpty>无匹配结果</CommandEmpty>
          <CommandGroup heading="建议">
            <CommandItem onSelect={() => setOpen(false)}>新建文件</CommandItem>
            <CommandItem onSelect={() => setOpen(false)}>打开文件夹</CommandItem>
          </CommandGroup>
        </CommandList>
      </CommandDialog>
    </>
  )
}
```

<h2 id="api">API</h2>

### Command

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | CSS 类名 | `string` | - |

根容器，继承 cmdk `Command` 的全部属性。支持 `value`、`onValueChange`、`filter`、`shouldFilter` 等。自动管理键盘导航和选中状态。

### CommandDialog

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| title | 对话框标题（屏幕阅读器可见） | `string` | `"Command Palette"` |
| description | 对话框描述（屏幕阅读器可见） | `string` | `"Search for a command to run..."` |
| showCloseButton | 是否显示关闭按钮 | `boolean` | `false` |
| className | 对话框内容 CSS 类名 | `string` | - |

继承 `Dialog` 的所有 props（`open`、`onOpenChange` 等）。将 Command 内容嵌入 Dialog 中使用，标题和描述通过 `sr-only` 视觉隐藏。

### CommandInput

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 输入框 CSS 类名 | `string` | - |
| placeholder | 占位文本 | `string` | - |

继承 cmdk `CommandInput` 的全部属性。包裹在 `InputGroup` 中，自动附带搜索图标。

### CommandList

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | CSS 类名 | `string` | - |

继承 cmdk `CommandList` 的全部属性。可滚动选项列表容器，最大高度 72（约 18rem）。

### CommandGroup

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| heading | 分组标题（渲染为 cmdk group heading） | `React.ReactNode` | - |
| className | CSS 类名 | `string` | - |

继承 cmdk `CommandGroup` 的全部属性。自动为 `cmdk-group-heading` 子元素应用字体样式。

### CommandItem

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | CSS 类名 | `string` | - |
| disabled | 是否禁用 | `boolean` | - |
| onSelect | 选中回调 | `(value: string) => void` | - |
| children | 选项内容 | `React.ReactNode` | - |

继承 cmdk `CommandItem` 的全部属性。选中项自动显示 CheckIcon，禁用项半透明。

### CommandEmpty

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | CSS 类名 | `string` | - |

继承 cmdk `CommandEmpty` 的全部属性。无匹配结果时居中展示的内容。

### CommandSeparator

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | CSS 类名 | `string` | - |

继承 cmdk `CommandSeparator` 的全部属性。水平分隔线。

### CommandShortcut

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | CSS 类名 | `string` | - |

快捷键提示标签，渲染为 `<span>`。继承所有标准 HTML span 元素属性。选中时文字颜色变为 foreground。

<h2 id="faq">常见问题</h2>

### 如何实现 Cmd+K 全局快捷键？

使用 `useEffect` 监听键盘事件，配合 `CommandDialog` 的 `open`/`onOpenChange`：

```tsx
useEffect(() => {
  const down = (e: KeyboardEvent) => {
    if (e.key === "k" && (e.metaKey || e.ctrlKey)) {
      e.preventDefault()
      setOpen((open) => !open)
    }
  }
  document.addEventListener("keydown", down)
  return () => document.removeEventListener("keydown", down)
}, [])
```

### Command 和 Combobox 的区别？

`Command` 基于 cmdk，专为命令面板场景设计，item 支持 `onSelect` 回调，内置搜索过滤和键盘导航。`Combobox` 基于 @base-ui/react，是更通用的下拉选择组件，内部不集成 Portal/Popup 定位，需配合 `CommandDialog` 或手动定位。

### 如何关闭 CommandDialog 后执行命令？

在 `CommandItem` 的 `onSelect` 回调中调用 `setOpen(false)` 关闭对话框，同时执行具体业务逻辑。

### 如何自定义过滤逻辑？

`Command` 接受 cmdk 的 `filter` prop，可传入自定义过滤函数替代默认的模糊匹配。设置 `shouldFilter={false}` 可完全禁用内置过滤。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/command` 导入所需子组件。
2. 弹窗模式使用 `CommandDialog` + `open`/`onOpenChange`。
3. `CommandInput` 必须在 `Command` 的直接子级。
4. `CommandList` 包裹 `CommandEmpty`、`CommandGroup`、`CommandItem`。
5. 不需要额外引入 cmdk 依赖，本组件已封装好。
6. 不要引入 Ant Design 相关依赖。
