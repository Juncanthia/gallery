# CsvViewer CSV 查看器

CsvViewer 用于在浏览器中预览、搜索和操作 CSV 文件，无需后端支持。基于 Glide Data Grid 虚拟渲染，适配大表格；内置搜索、缩放、上传和下载功能。

## 何时使用 {#when-to-use}

需要在界面内直接浏览或检索 CSV / TSV 数据时使用。

常见场景：

- 数据预览：用户上传 CSV 后即时查看内容。
- 数据检索：在大表格中按列搜索并定位匹配单元格。
- 导出下载：将当前数据导出为 CSV 文件下载。

CsvViewer 接收 CSV 文本字符串作为 `data` 属性，使用 PapaParse 在浏览器端解析；表格渲染基于 Glide Data Grid 虚拟滚动，仅绘制可视区域单元格。搜索采用分批异步处理，每批 500 行，不阻塞 UI。

## 代码演示 {#examples}

### 基础用法

传入 `data` 属性即可渲染 CSV 表格。`search` 开启搜索功能，工具栏包含缩放、搜索和上传/下载操作。

```tsx
import { CsvViewer } from "@/components/document/components/ui/csv-viewer"

const SAMPLE_CSV = `Name,Age,City,Role
Alice,30,New York,Engineer
Bob,25,San Francisco,Designer
Charlie,35,Chicago,Manager`

export default function Demo() {
  return (
    <CsvViewer data={SAMPLE_CSV} search />
  )
}
```

## API {#api}

### CsvViewer

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | CSV / TSV 文本内容，传入后即刻解析渲染 | `string` | - |
| search | 是否显示搜索工具栏，开启后支持单元格搜索和导航 | `boolean` | `false` |
| className | 附加 className，会通过 cn 合并到外层容器 | `string` | - |

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/extend/components/ui/csv-viewer.tsx` — 组件完整实现，包含 CSV 解析、Glide Data Grid 渲染、搜索、缩放、上传和下载。
- `src/components/core/extend-csv-viewer.tsx` — 重导出入口，从 extend 子包导出 CsvViewer。

关键行为：

1. `data` 变化时重新解析并重置搜索状态。
2. `search` 开启时渲染搜索弹窗工具栏，否则仅显示缩放和上传/下载。
3. 搜索采用分批异步（每批 500 行），带 300ms 去抖，避免阻塞 UI。
4. 搜索结果支持键盘导航（Enter 前进，Shift+Enter 后退）。
5. 缩放级别：75%、100%、125%、150%、200%。
6. 上传支持 `.csv` 和 `.tsv` 格式。
7. 下载使用 PapaParse 序列化，自动添加 `.csv` 扩展名。
8. 表格主题自动跟随亮色/暗色模式。

## FAQ {#faq}

### 如何加载 CSV 文件？

两种方式：(1) 通过 `data` 属性传入 CSV 文本字符串；(2) 用户通过工具栏上传按钮选择本地文件。

```tsx
<CsvViewer data={csvTextContent} />
```

### 支持 TSV 吗？

支持。PapaParse 自动识别逗号或制表符分隔，上传也接受 `.tsv` 文件。

### 大表格性能如何？

基于 Glide Data Grid 虚拟渲染，只绘制可视区域内的单元格。搜索分批处理，不阻塞主线程。

### 如何自定义外观？

通过 `className` 调整外层容器样式。表格内部使用 Glide Data Grid theme，自动适配亮色/暗色主题。如需深度定制，可修改 `CsvViewer` 内部的 theme 配置。

## LLM 使用规则 {#llm-rules}

当 AI 生成 CsvViewer 代码时，应遵守：

1. 从 `@/components/document/components/ui/csv-viewer` 导入 CsvViewer。
2. CSV 数据通过 `data` 属性以字符串形式传入。
3. 如需搜索功能，设置 `search`。
4. 不需要后端服务，所有解析在浏览器端完成。
5. 不要自行实现 CSV 解析或表格渲染，使用 CsvViewer 组件。
6. 不要引入额外的 CSV 解析库，CsvViewer 已内置 PapaParse。
