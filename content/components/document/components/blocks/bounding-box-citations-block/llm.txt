# HumanReviewBlock 人类审核组件

HumanReviewBlock 是 Extend UI 人类审核组件，将 PDF 查看器与结构化审核面板并排展示，支持按字段聚焦并在 PDF 上高亮对应的边界框位置。

本组件是项目内的本地组件，API 和行为以源码为准。

## 何时使用 {#when-to-use}

当需要对 PDF 文档中的结构化提取结果进行人工审核时使用。左侧展示 PDF（带边界框高亮），右侧展示审核面板（表单视图和 JSON Diff 视图）。

常见场景：

- 文档审核：对照 PDF 原始文档审核 OCR 或 AI 提取的结构化字段。
- 数据校验：验证银行对账单、发票、合同等文档的提取结果是否准确。
- 人工标注：逐字段检查并修正自动提取的数据，标注差异点。

组件使用 `PdfBlockResizableShell` 提供可拖拽调整的左右分栏布局，右侧面板包含 Form 表单视图和 JSON Diff 对比视图。Form 视图支持编辑布尔/数字/字符串/枚举/对象/数组类型，数组使用 Glide Data Grid 电子表格编辑。JSON Diff 基于 @pierre/diffs 渲染并排对比。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
type ReviewField = {
  key: string
  schema: ReviewFieldSchema
  actual: JsonValue
  expected: JsonValue
  location?: ReviewLocation
  metadataPath?: string
}
```

```ts
type ReviewLocation = {
  page: number
  area: {
    left: number
    top: number
    width: number
    height: number
  }
}
```

```ts
type ReviewFieldSchema = {
  type: "string" | "number" | "integer" | "boolean" | "object" | "array"
  title?: string
  description?: string
  enum?: Array<string | number>
  properties?: Record<string, ReviewFieldSchema>
  items?: ReviewFieldSchema
}
```

```ts
type HumanReviewBlockProps = {
  file?: string
  fields?: ReviewField[]
  className?: string
  metadata?: Record<string, ReviewMetadataEntry>
  resolveArrayItemMetadataPath?: (
    metadataPath: string,
    rowIndex: number,
    rowValue: JsonValue
  ) => string | undefined
  resolveLocation?: (metadataPath: string) => ReviewLocation | undefined
  showExpected?: boolean
  theme?: "light" | "dark"
}
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- 用户可通过 `fields` 中每个 ReviewField 的 `key` 定位字段卡片。
- `HumanReviewHighlight` 根据 `ReviewLocation` 的百分比坐标渲染绝对定位的高亮层。

额外状态属性：

- 无额外 `data-*` 状态属性。

生成测试、样式定位或自动化脚本时，优先使用 ARIA 语义和角色属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 边界框高亮使用 `border-blue-500/70 bg-blue-500/12 shadow-[0_4px_16px_rgb(59_130_246_/_10%)]` 样式。
- 表单字段卡片支持 active 态 (`border-blue-500/60 bg-blue-500/5`) 高亮。
- Glide Data Grid 表格主题通过 `useHumanReviewGridTheme` 自动适配亮色/暗色模式。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground` 等项目级语义变量。
- JSON Diff 视图样式变量通过 `--diffs-light-bg`、`--diffs-light` 等 CSS 变量控制。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/extend/components/blocks/bounding-box-citations-block.tsx` — 块组件实现，组合 PDFViewer + HumanReviewPanel + PdfBlockResizableShell
- `src/components/extend/components/ui/bounding-box-citations.tsx` — 审核 UI 组件完整实现（约 2340 行）
- `src/components/core/extend-bounding-box-citations-block.tsx` — 路径重导出封装

直接依赖摘要：

- `@glideapps/glide-data-grid` — 表格/电子表格渲染
- `@hugeicons/core-free-icons` / `@hugeicons/react` — 图标组件
- `@pierre/diffs` — JSON Diff 虚拟化渲染
- `@hyper/extend-ui/components/ui/button` — 按钮
- `@hyper/extend-ui/components/ui/input` — 输入框
- `@hyper/extend-ui/components/ui/scroll-area` — 滚动区域
- `@hyper/extend-ui/components/ui/tabs` — 标签页
- `@hyper/extend-ui/components/ui/tooltip` — 提示
- `@hyper/extend-ui/components/ui/pdf-viewer` — PDF 查看器
- `@hyper/extend-ui/components/pdf-block-resizable-shell` — 可拖拽分栏容器
- `@hyper/extend-ui/lib/utils` — cn 工具函数

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. `location.area` 使用百分比坐标（0-100），基于 PDF 页面的百分比位置。
3. 边界框高亮在 PDF 页面上通过绝对定位 + 百分比坐标渲染。
4. 字段联动：hover/focus 字段卡片时自动滚动 PDF 到对应位置。
5. 数组编辑基于 Glide Data Grid，支持嵌套对象和数组的递归展开编辑。
6. JSON Diff 使用 @pierre/diffs 的 MultiFileDiff 并排对比，支持 Worker 线程高亮渲染。
7. 暗色模式通过 `useIsDarkTheme` 共享 MutationObserver 检测 `document.documentElement` 的 class 变化。
8. `PdfBlockResizableShell` 提供可拖拽左右分栏，默认右侧 42%，可调 30%-60%。
9. 如果需要新增能力，应先修改该组件源码，再更新本 llm 文档。

## 可访问性 {#accessibility}

- 字段卡片为可聚焦元素（`tabIndex={0}`），支持键盘 Tab 导航。
- hover 字段卡片时自动滚动 PDF 到对应位置并高亮边界框。
- Form/JSON Tab 切换通过 Tabs 组件提供键盘交互。
- Glide Data Grid 支持键盘导航（Tab/Enter/方向键）编辑单元格。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 HumanReviewBlock 代码时，应遵守：

1. 从 `@/components/core/bounding-box-citations-block` 导入 `HumanReviewBlock`。
2. `file` 为可选属性，不提供时 PDF 区域为空但不影响审核面板使用。
3. `fields` 使用 `ReviewField[]` 类型，包含 key/schema/actual/expected/location。
4. `location.area` 使用百分比坐标（left/top/width/height 均为 0-100 的百分比值）。
5. 不要手动组合 `PDFViewer` + `HumanReviewPanel`，直接使用 `HumanReviewBlock`。
6. 不要引入 Ant Design 相关依赖。
7. 数组字段的嵌套编辑使用 Glide Data Grid 组件，不要自行实现。
8. 如果不确定某个 API 是否存在，先检查源码，不要凭记忆生成。
