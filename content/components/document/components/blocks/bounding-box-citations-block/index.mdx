HumanReviewBlock 人类审核组件。将 PDF 查看器与结构化审核面板并排展示，支持按字段聚焦并在 PDF 上高亮对应的边界框位置。

<h2 id="when-to-use">何时使用</h2>

当需要对 PDF 文档中的结构化提取结果进行人工审核时使用。左侧展示 PDF（带边界框高亮），右侧展示审核面板（表单视图和 JSON Diff 视图）。

常见场景：

- 文档审核：对照 PDF 原始文档审核 OCR 或 AI 提取的结构化字段。
- 数据校验：验证银行对账单、发票、合同等文档的提取结果是否准确。
- 人工标注：逐字段检查并修正自动提取的数据，标注差异点。

不推荐：

- 仅需要展示 PDF 而不需要字段审核时，应使用 `PDFViewer` 组件。
- 仅需要 JSON 对比而不需要 PDF 定位时，应使用 `JsonDiffView` 组件。
- 需要审核非 PDF 文档（如图片、HTML）时，本组件不支持。

<h2 id="api">API</h2>

### HumanReviewBlock

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| file | PDF 文件 URL 或 data URL | `string` | - |
| fields | 审核字段列表，每个字段包含 key、schema、actual、expected、location | `ReviewField[]` | `REVIEW_FIELDS` |
| className | 根元素样式类名 | `string` | - |
| metadata | 引用元数据映射，用于通过 metadataPath 解析位置 | `Record<string, ReviewMetadataEntry>` | - |
| resolveArrayItemMetadataPath | 自定义数组项元数据路径解析函数 | `(metadataPath, rowIndex, rowValue) => string \| undefined` | - |
| resolveLocation | 自定义位置解析函数 | `(metadataPath) => ReviewLocation \| undefined` | - |
| showExpected | 是否显示期望值列并允许编辑 | `boolean` | `true` |
| theme | 主题明暗模式 | `"light"` \| `"dark"` | - |

### ReviewField

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| key | 字段唯一标识 | `string` |
| schema | JSON Schema 定义，含 type、title、description 等 | `ReviewFieldSchema` |
| actual | 实际提取值 | `JsonValue` |
| expected | 期望值（可编辑） | `JsonValue` |
| location | PDF 中的边界框位置 | `ReviewLocation` |
| metadataPath | 元数据路径，用于引用定位 | `string` |

### ReviewLocation

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| page | PDF 页码（从 1 开始） | `number` |
| area | 归一化区域（百分比） | `{ left: number, top: number, width: number, height: number }` |

组件结构：

- 左侧：`PDFViewer` 渲染 PDF，通过 `renderPageOverlay` 在当前页上渲染 `HumanReviewHighlight` 蓝色半透明高亮框。
- 右侧：`HumanReviewPanel` 审核面板，包含两个 Tab：
  - **Form 表单视图**：逐字段展示字段卡片，支持编辑实际/期望值（布尔/数字/字符串/枚举/对象/数组），对象嵌套递归展开，数组使用 Glide Data Grid 电子表格编辑。
  - **JSON Diff 视图**：基于 @pierre/diffs 的并排 JSON 对比视图，高亮 actual 与 expected 之间的差异。
- 容器：`PdfBlockResizableShell` 可拖拽调整左右分栏宽度，默认右侧占比 42%，可调范围 30%-60%。

<h2 id="faq">常见问题</h2>

### 如何加载 PDF 文件？

通过 `file` 属性传入 PDF URL 或 data URL：

```tsx
&lt;HumanReviewBlock file="https://example.com/statement.pdf" /&gt;
```

也支持通过 `PdfDropzoneBlock` 上传 PDF 后获得 blob URL 再传入。

### 如何自定义审核字段？

通过 `fields` 属性传入自定义的 `ReviewField[]`：

```tsx
import { HumanReviewBlock } from "@/components/core/bounding-box-citations-block"

const customFields = [
  {
    key: "name",
    schema: { type: "string", title: "Name" },
    actual: "John",
    expected: "John",
    location: { page: 1, area: { left: 10, top: 20, width: 30, height: 5 } },
  },
]

export default function App() {
  return (
    &lt;HumanReviewBlock fields={customFields} /&gt;
  )
}
```

### 如何实现文字位置与 PDF 的联动？

组件内置了联动机制：鼠标悬停或聚焦字段卡片时，自动计算对应 `ReviewLocation` 并调用 `viewerRef.current.scrollToPageArea()` 滚动到 PDF 对应位置，同时渲染蓝色高亮框。通过 `onFieldFocus` 和 `onLocationHover` 回调控制。

### 如何自定义位置解析？

通过 `resolveLocation` 传入自定义解析函数，从 `metadataPath` 映射到 `ReviewLocation`：

```tsx
&lt;HumanReviewBlock
  resolveLocation={(path) => myLocationMap[path]}
/&gt;
```

也可通过 `metadata` 属性传入 `ReviewMetadataEntry` 映射，组件内置的 `getMetadataLocation` 会从中提取 `citations` 的 polygon 信息并计算边界框位置。

### 如何隐藏期望值编辑功能？

设置 `showExpected={false}`，审核面板将仅显示 actual 值，不显示 expected 列，适用于只读审核场景：

```tsx
&lt;HumanReviewBlock showExpected={false} /&gt;
```

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 HumanReviewBlock 代码时：

1. 从 `@/components/core/bounding-box-citations-block` 导入 `HumanReviewBlock`。
2. `file` 为可选属性，不提供时 PDF 区域为空。
3. `fields` 使用 `ReviewField[]` 类型，默认值为 `REVIEW_FIELDS`（银行对账单示例数据）。
4. `location.area` 使用百分比坐标（0-100），基于 PDF 页面宽高的归一化坐标。
5. 不要手动组合 `PDFViewer` + `HumanReviewPanel`，直接使用 `HumanReviewBlock`。
6. 不要引入 Ant Design 相关依赖。
7. 数组字段编辑基于 Glide Data Grid 虚拟渲染，支持嵌套展开和点击联动。
8. JSON Diff 基于 @pierre/diffs 渲染并排对比，支持 dark 模式和滚动虚拟化。
