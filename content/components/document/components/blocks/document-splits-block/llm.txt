# DocumentSplitsBlock 文档分割

提供 PDF 文档页面级别切分重组的一体化组件。左侧为 PDF 查看器，支持上传与预览；右侧为可拖拽的文档分割面板，将页面自由分配到不同的分组中，支持拖拽排序和跨组移动。

本组件内部组合 PdfBlockResizableShell、PDFViewer 和 DocumentSplits，提供一站式 PDF 文档切分体验。不是 Ant Design 兼容层，实现以本地源码为准。

## 何时使用 {#when-to-use}

适合需要对 PDF 文档按章节、主题或业务单元进行页面级别拆分与重组的场景。组件自管理状态，用户可在可视化面板中拖拽操作。

常见场景：

- 学术论文、研究报告按章节拆分（摘要、方法论、结果等）。
- 合同文档按条款、附件等业务单元分拆。
- 扫描件批量处理前的页面分类与预览。

不推荐：

- 仅需查看 PDF 时，使用 PDFViewer。
- 仅需上传并预览 PDF 时，使用 PdfDropzoneBlock。
- 需要程序化自动拆分时，应在业务层调用 PDF 处理库。

## 核心导出 {#exports}

- `DocumentSplitsBlock`
- `DocumentSplits`（分割面板子组件，通常不单独使用）
- `createInitialSplits`（根据页数生成初始分割组）
- `INITIAL_SPLITS`（默认示例分割数据）
- 类型：`DocumentSplit`、`DocumentSplitPageId`

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
// DocumentSplitsBlock 定义了 PDF 查看器 + 分割面板组合的一体化接口
export function DocumentSplitsBlock({
  file,
  heightClassName,
  thumbnailImages,
}: {
  file?: string
  heightClassName?: string
  thumbnailImages?: Record<DocumentSplitPageId, string>
}): React.ReactElement

// DocumentSplits 是右侧分割面板的核心子组件
export function DocumentSplits({
  className,
  splits,
  thumbnailImages,
  withFrameDivider,
  onSelectPage,
  onSplitsChange,
}: {
  className?: string
  splits: DocumentSplit[]
  thumbnailImages?: Record<DocumentSplitPageId, string>
  withFrameDivider?: boolean
  onSelectPage: (pageNumber: number) => void
  onSplitsChange: (splits: DocumentSplit[]) => void
}): React.ReactElement

// 分割组数据结构
type DocumentSplit = {
  id: string
  title: string
  pages: DocumentSplitPageId[]
}

type DocumentSplitPageId = `page-${number}`
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `pdf-block-document-splits`

额外状态属性：

- `data-auto-save="pdf-block-document-splits"`

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 控制。
- 使用 `h-[720px]` 作为默认高度，可通过 `heightClassName` 自定义。
- 组件内部使用 `bg-background`、`bg-muted/20` 等语义色适配主题。
- 分割面板使用 `@dnd-kit` 实现拖拽交互，包含缩略图、拖拽覆盖层等视觉元素。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/extend/components/blocks/document-splits-block.tsx`
- `src/components/extend/components/ui/document-splits.tsx`

包装导出：

- `src/components/core/extend-document-splits-block.tsx`

直接依赖摘要：

- 依赖 `PdfBlockResizableShell`（左右可调分栏布局）。
- 依赖 `PDFViewer`（PDF 渲染、页码导航）。
- 依赖 `DocumentSplits`（拖拽分割面板，基于 `@dnd-kit/core` 和 `@dnd-kit/sortable`）。
- 依赖 `FileThumbnail`（页面缩略图）、`ScrollArea`（滚动容器）、`Button`（操作按钮）。
- 使用 `@hugeicons/core-free-icons`（Add01Icon、Delete02Icon、DragDropVerticalIcon）。

实现注意：

1. `DocumentSplitsBlock` 内部通过 `React.useState` 管理分割组状态，初始值为 `INITIAL_SPLITS`。
2. PDF 文件加载后通过 `handleDocumentLoadSuccess` 回调获取页数，调用 `createInitialSplits(pageCount)` 自动生成分割组。
3. 页面拖拽逻辑由 `DocumentSplits` 组件内部的 `@dnd-kit` 实现：页面在组间移动（`DragOver`）、组内排序（`DragEnd`）、分割组重新排序。
4. 上传的 PDF 文件通过 `URL.createObjectURL` 生成临时 URL，组件卸载时在 effect cleanup 中释放。
5. 左侧 PDF 查看器通过 `handlePdfUpload` 回调接收上传文件并生成新的对象 URL。
6. 页面缩略图通过 `thumbnailImages` prop 传入，未提供时显示加载占位。
7. 如需扩展分割结果输出，应在 `DocumentSplitsBlock` 源码中新增回调 prop，而非在外层 hack 内部 DOM 结构。

## 可访问性 {#accessibility}

- 所有图标按钮通过 `aria-label` 提供可访问名称（如 "Reorder {title}"、"Remove {title}"）。
- 拖拽手柄标记为 `cursor-grab`/`active:cursor-grabbing`，提供视觉与触觉反馈。
- 空分割组显示 "Drop pages here" 占位提示。
- PDF 查看器提供完整的键盘导航和 ARIA 属性。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 DocumentSplitsBlock 代码时，应遵守：

1. 从 `@/components/core/document-splits-block` 导入，不要从内部路径导入。
2. 组件为自包含组件，可选传入 `file` 指定预加载的 PDF URL。
3. 不要手动组合 PdfBlockResizableShell + PDFViewer + DocumentSplits 来模拟本组件行为。
4. 不要引入 Ant Design 相关依赖。
5. 如需自定义样式，使用外层容器的 className 或 `heightClassName` prop。
6. 如果不确定某个行为是否存在，先检查 `src/components/extend/components/blocks/document-splits-block.tsx` 和 `src/components/extend/components/ui/document-splits.tsx`，不要凭猜测生成。
7. 不要从 `@hyper/extend-ui/` 路径导入，使用项目标准的 `@/components/core/` 别名。
