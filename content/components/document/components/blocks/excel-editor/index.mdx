ExcelEditorBlock 基于 `@extend-ai/react-xlsx` 的 XLSX 编辑器组件，提供完整的电子表格编辑能力，包括单元格编辑、格式化、公式输入、边框、合并单元格、撤销/重做、工作表管理和文件上传/下载功能。

<h2 id="when-to-use">何时使用</h2>

当需要在界面中嵌入类 Excel 的电子表格编辑功能时使用。

常见场景：

- 数据录入与编辑界面。
- 报表编辑与预览。
- 电子表格文件（.xlsx）的在线查看与编辑。
- 需要富文本格式化表格数据的场景。
- 结构化数据协作编辑。

ExcelEditorBlock 的核心能力：

- 单元格编辑：支持值输入和公式输入（以 `=` 开头）。
- 格式化工具栏：字体、字号、粗体、斜体、下划线、删除线。
- 对齐控制：水平对齐（左/中/右）、垂直对齐（上/中/下）、文本换行。
- 颜色系统：文字颜色、填充颜色、边框颜色。
- 边框：全部边框、外边框、四边独立边框、无边框。
- 合并单元格：合并并居中、跨行合并、合并单元格、取消合并。
- 撤销/重做操作。
- 工作表管理：添加/删除工作表、切换工作表。
- 缩放控制：50%-400% 缩放级别。
- 文件操作：上传本地 .xlsx 文件、下载当前工作簿。
- 暗色模式：支持明暗主题切换。

### 指定 XLSX 文件

通过 `file` 属性指定远程 .xlsx 文件路径。

```tsx
import { ExcelEditorBlock } from "@/components/core/excel-editor"

export default function Demo() {
  return <ExcelEditorBlock file="/samples/my-workbook.xlsx" />
}
```

### 调整高度

通过 `heightClassName` 自定义编辑器容器高度。

```tsx
import { ExcelEditorBlock } from "@/components/core/excel-editor"

export default function Demo() {
  return <ExcelEditorBlock heightClassName="h-[500px]" />
}
```

<h2 id="faq">常见问题</h2>

### 编辑器显示"只读"怎么办？

当加载的 .xlsx 文件超过 5MB 时，编辑器会自动进入只读模式。此时可以查看数据但无法编辑。建议上传体积更小的文件，或使用 `file` 指定较小的文件。

### 如何禁止上传功能？

`ExcelEditorBlock` 的工具栏始终包含上传按钮，当前版本未暴露控制该按钮显示/隐藏的 prop。如需隐藏上传功能，可在源码中调整 `EditorFileActionsMenu` 组件。

### 如何自定义编辑器工具栏？

工具栏由 `EditorToolbar` 子组件渲染，包含字体、对齐、边框、合并、工作表管理、缩放等全套控件。如需定制，建议在 `src/components/extend/components/blocks/excel-editor.tsx` 源码基础上进行修改。

### 如何监听编辑变化？

当前 `ExcelEditorBlock` 不提供 `onChange` 回调。如需监听单元格变化，需在源码中接入 `useXlsxViewerEditing` hook 的 `setCellValue`、`setCellFormula` 等方法，并添加自定义回调逻辑。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Excel 编辑器代码时：

1. 从 `@/components/core/excel-editor` 导入 `ExcelEditorBlock`。
2. 默认不需要传任何 props 即可获得完整功能的编辑器。
3. 需要加载指定文件时使用 `file` prop。
4. 需要调整高度时使用 `heightClassName` prop，传入 Tailwind 高度类名。
5. 不要直接从 `@hyper/extend-ui` 或 `@extend-ai/react-xlsx` 导入。
6. 不要引入 Ant Design 相关依赖。
