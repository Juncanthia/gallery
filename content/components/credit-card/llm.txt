# CreditCard 信用卡

CreditCard 是一个完整的信用卡展示复合组件，由 CreditCard 外层容器、CreditCardFlipper 翻转动画、CreditCardFront/CreditCardBack 正反面、CreditCardChip 芯片、CreditCardLogo Logo 位、CreditCardMagStripe 磁条、CreditCardNumber 卡号、CreditCardName 持卡人、CreditCardExpiry 有效期、CreditCardCvv 安全码和 CreditCardServiceProvider 支付服务商标识等 13 个子组件组合而成。支持桌面端 hover 翻转和移动端点击翻转两种交互模式。

本组件为项目本地复合组件，无 Ant Design 直接对应。实现以本地源码为准。

## 何时使用 {#when-to-use}

适合在支付、钱包、账户管理等场景中展示信用卡信息，支持完整的信用卡视觉元素和正反面交互。

常见场景：

- 钱包应用中展示已绑定的信用卡。
- 支付确认页面展示卡片信息。
- 用户资料页的支付方式管理。
- 需要展示卡号、有效期、CVV 等完整卡片信息的面板。

不推荐：

- 仅需展示支付方式图标 —— 使用 `PaymentIcon` 直接渲染即可。
- 真实支付表单输入场景 —— 本组件仅供展示，不支持输入编辑。
- 在调用侧绕过子组件直接手写信用卡 UI。
- 作为通用容器 —— 组件尺寸固定为信用卡比例（85.60/53.98）。

## 代码演示 {#examples}

### 基础用法

```tsx
import {
  CreditCard,
  CreditCardFlipper,
  CreditCardFront,
  CreditCardBack,
  CreditCardChip,
  CreditCardLogo,
  CreditCardMagStripe,
  CreditCardNumber,
  CreditCardName,
  CreditCardExpiry,
  CreditCardCvv,
  CreditCardServiceProvider,
} from "@/components/blocks/credit-card"

export default function Demo() {
  return (
    <CreditCard>
      <CreditCardFlipper>
        <CreditCardFront>
          <CreditCardLogo>
            <img src="/bank-logo.svg" alt="Bank Logo" className="h-full w-full object-contain" />
          </CreditCardLogo>
          <CreditCardChip />
          <CreditCardNumber>1234 5678 9012 3456</CreditCardNumber>
          <CreditCardName>John Doe</CreditCardName>
          <CreditCardExpiry>12/28</CreditCardExpiry>
          <CreditCardServiceProvider type="Visa" />
        </CreditCardFront>
        <CreditCardBack>
          <CreditCardMagStripe />
          <CreditCardCvv>123</CreditCardCvv>
        </CreditCardBack>
      </CreditCardFlipper>
    </CreditCard>
  )
}
```

> 说明：`CreditCard` 是最外层容器，控制卡片宽高比。`CreditCardFlipper` 提供翻转动画（桌面 hover / 移动端 tap）。`CreditCardFront` 和 `CreditCardBack` 分别渲染正反面，子组件按卡片结构自由排列。`CreditCardServiceProvider` 基于 `react-svg-credit-card-payment-icons` 的 `PaymentIcon` 渲染支付服务商标识。

## API {#api}

以下类型来自 `src/components/blocks/credit-card/index.tsx` 源码，是生成代码时的主要依据。

### CreditCard

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类，与默认 `perspective-distant aspect-[8560/5398] w-full max-w-96 text-white @container` 合并 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。提供 `group/kibo-credit-card` 类名供子组件 hover 翻转使用。

### CreditCardFlipper

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 正反面内容（通常包含 CreditCardFront 和 CreditCardBack） | `React.ReactNode` | - |

继承 `HTMLAttributes<HTMLDivElement>`。桌面端通过 `group-hover/kibo-credit-card:-rotate-y-180` 响应父容器 hover 实现翻转（`duration-700 ease-in-out`）。移动端（`!supportsHover`）通过点击切换 `isFlipped` 状态，使用 `-rotate-y-180 shadow-lg` 翻转。设置了 `aria-label="Flip credit card"`。

### CreditCardFront

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| safeArea | 内容区内边距（px），控制子元素与卡片边界的距离 | `number` | `20` |
| className | 附加样式类 | `string` | - |
| children | 正面内容 | `React.ReactNode` | - |

继承 `HTMLAttributes<HTMLDivElement>`。背面隐藏（`backface-hidden`），背景为 `bg-foreground/90`，响应式圆角 `@xs:rounded-2xl rounded-lg`。

### CreditCardBack

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| safeArea | 内容区内边距（px） | `number` | `16` |
| className | 附加样式类 | `string` | - |
| children | 背面内容 | `React.ReactNode` | - |

继承 `HTMLAttributes<HTMLDivElement>`。背面隐藏（`backface-hidden`），通过内部 `CreditCardBackContext` 向 `CreditCardMagStripe` 提供 `safeArea`。当位于 `CreditCardFlipper`（`CreditCardFlipContext`）内部时自动翻转 180 度（`rotate-y-180`）。

### CreditCardChip

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 自定义芯片内容；不传时渲染默认 SVG 芯片图案 | `React.ReactNode` | 默认 SVG |

继承 `HTMLAttributes<SVGSVGElement>`（无 children 时）或作为 div（有 children 时）。默认 SVG 为金色渐变芯片图案（`#EDE5A6` 到 `#CFA255`），尺寸 `w-1/6`，定位 `absolute top-1/2 left-0 -translate-y-1/2`。

### CreditCardLogo

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。定位在卡片右上角（`absolute top-0 right-0`），尺寸 `size-1/6`。通常放置银行或发卡机构 Logo。

### CreditCardServiceProvider

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 支付服务商类型，来自 `react-svg-credit-card-payment-icons` | `PaymentIconProps['type']` | `'Visa'` |
| className | 附加样式类 | `string` | - |
| children | 自定义服务商标识；不传时使用 `PaymentIcon` 渲染 | `React.ReactNode` | 默认 `PaymentIcon` |

继承 `ComponentProps<typeof PaymentIcon>`。定位在正面右下角（`absolute right-0 bottom-0`），最大尺寸 `max-h-1/3 max-w-1/3`。

### CreditCardMagStripe

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。定位在背面上方（`top-[3%]`），高度 `h-1/4`，背景 `bg-gray-900`。宽度通过 Context 中的 `safeArea` 自动计算（`calc(100% + 2 * safeArea)`），确保磁条延伸到卡片边缘。

### CreditCardNumber / CreditCardName / CreditCardExpiry / CreditCardCvv

这四个组件均为展示型组件，继承 `HTMLAttributes<HTMLParagraphElement>`，仅暴露 `className` 属性。

| 组件 | 默认 className | 用途 |
| --- | --- | --- |
| CreditCardNumber | `font-mono @xs:text-2xl` | 卡号显示 |
| CreditCardName | `font-semibold uppercase` | 持卡人姓名 |
| CreditCardExpiry | `font-mono` | 有效期（如 "12/28"） |
| CreditCardCvv | `font-mono` | CVV 安全码 |

均设置 `lineHeight: "100%"`，渲染为 `<p>` 元素。

## 样式模型 {#styling-model}

- 样式由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 容器使用 `perspective-distant` 提供 3D 透视效果，`aspect-[8560/5398]` 固定信用卡标准宽高比（约 1.586:1）。
- 翻转动画使用 CSS `transform-3d` + `rotate-y-180`，桌面端 `group-hover` 触发，移动端 `isFlipped` 状态驱动。时长 `duration-700 ease-in-out`。
- 正反面使用 `backface-hidden` 隐藏背面，结合 `rotate-y-180` 实现 3D 翻转。
- 圆角响应式：`@xs:rounded-2xl rounded-lg`，基于 `@container` 查询。
- 子组件定位方式：`CreditCardChip`（`absolute top-1/2 left-0`）、`CreditCardLogo`（`absolute top-0 right-0`）、`CreditCardServiceProvider`（`absolute right-0 bottom-0`）、`CreditCardMagStripe`（`absolute top-[3%] left-1/2 -translate-x-1/2`）。
- 主题颜色：`bg-foreground/90`（卡片背景）、`text-white`（卡片文字）、`bg-gray-900`（磁条）。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/credit-card/index.tsx`（复合组件定义）
- `src/components/ui/credit-card.tsx`（re-export barrel）

直接依赖摘要：

- React（`useState`、`useEffect`、`createContext`、`useContext`、`window.matchMedia`）
- `react-svg-credit-card-payment-icons`（`PaymentIcon`，支付服务商 SVG 图标）
- `@/lib/utils`（`cn` 样式合并）
- Tailwind CSS（含 `@container`、`transform-3d` 插件）

实现注意：

1. `CreditCardFlipper` 通过 `useSupportsHover` hook（`window.matchMedia("(hover: hover)")`）检测设备 hover 能力，决定翻转触发方式。
2. `CreditCardBack` 检测自身是否在 `CreditCardFlipContext`（由 `CreditCardFlipper` 提供）内，是则自动添加 `rotate-y-180`。
3. `CreditCardMagStripe` 通过 `CreditCardBackContext` 获取 `safeArea` 计算宽度。
4. `CreditCardChip` 和 `CreditCardServiceProvider` 均支持 children 自定义 vs 默认渲染模式。
5. 如需新增能力，应先修改该组件源码，再更新本 llm 文档。

## 可访问性 {#accessibility}

- `CreditCardFlipper` 设置了 `aria-label="Flip credit card"` 指示翻转操作。
- 移动端点击翻转为显式交互行为；桌面端 hover 翻转为视觉增强，不影响键盘操作。
- 卡号、姓名、有效期等渲染为 `<p>` 元素，天然可被屏幕阅读器读取。
- 默认芯片 SVG 包含 `<title>Chip</title>` 提供文本描述。
- 建议为 `CreditCardLogo` 中的 `<img>` 提供有意义的 `alt` 文本。
- 组件仅供展示，不涉及表单输入或敏感数据收集。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。AntD 无内置信用卡展示组件。

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/blocks/credit-card/index.tsx` 的 TypeScript 类型为准 |
| 翻转动画 | 使用 CSS `transform-3d` + `backface-hidden`，非 JavaScript 动画 |
| 支付图标 | 使用 `react-svg-credit-card-payment-icons`，非 AntD 图标库 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 CreditCard 代码时，应遵守：

1. 从 `@/components/blocks/credit-card` 导入所有需要的子组件，不要从 `antd` 导入。
2. `CreditCard` 作为最外层容器，`CreditCardFlipper` 包裹正反面，`CreditCardFront`/`CreditCardBack` 各自包含正面/背面内容。
3. `CreditCardChip` 默认渲染 SVG 芯片图案；需要自定义时传入 `children`（此时渲染为 div 容器）。
4. `CreditCardMagStripe` 仅在 `CreditCardBack` 内部使用，会自动适配 `safeArea` 宽度。
5. `CreditCardServiceProvider` 的 `type` 使用 `react-svg-credit-card-payment-icons` 支持的类型（如 `'Visa'`、`'MasterCard'`、`'Amex'`）。
6. 卡号、姓名、有效期等使用对应的语义子组件，不要手写 `<p>` 标签替代。
7. 只使用本文件 API 摘要和源码中存在的 props，不要凭记忆生成不存在的属性。
8. 不要将信用卡组件用于收集真实支付信息 —— 它仅用于展示。
9. 不要引入 AntD ConfigProvider、AntD 图标库或 rc-component。
10. 如果不确定某个 API 是否存在，先检查 `src/components/blocks/credit-card/index.tsx`。
