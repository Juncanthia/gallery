信用卡 CreditCard。完整的信用卡展示复合组件，支持正反面翻转动画、芯片/磁条/Logo/支付服务商标识等完整视觉元素，由 13 个子组件组合而成。

<h2 id="when-to-use">何时使用</h2>

当需要在支付、钱包、账户管理等场景中展示信用卡信息，需要信用卡正反面翻转交互时使用。

常见场景：
- 钱包应用中展示已绑定的信用卡。
- 支付确认页面展示卡片信息。
- 用户资料页的支付方式管理。
- 需要展示卡号、有效期、CVV 等完整卡片信息的面板。

不推荐：
- 仅需展示支付方式图标 —— 使用 `PaymentIcon` 直接渲染即可。
- 真实支付表单输入场景 —— 本组件仅供展示，不支持输入编辑。
- 在调用侧绕过子组件直接手写信用卡 UI。
- 作为通用容器 —— 组件尺寸固定为信用卡标准比例（85.60/53.98）。

<h2 id="api">API</h2>

### CreditCard

最外层容器，控制卡片宽高比和 3D 透视。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

固定宽高比 `aspect-[8560/5398]`（约 1.586:1），最大宽度 `max-w-96`，文字白色，内设 `group/kibo-credit-card` 供子组件 hover 翻转使用。

### CreditCardFlipper

翻转容器，桌面端 hover 翻转，移动端点击翻转。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 正反面内容（通常包含 CreditCardFront 和 CreditCardBack） | `React.ReactNode` | - |

桌面端通过 CSS `group-hover/kibo-credit-card:-rotate-y-180` 响应父容器 hover，移动端通过 `window.matchMedia("(hover: hover)")` 检测后降级为点击翻转。翻转时长 `duration-700 ease-in-out`。

### CreditCardFront

卡片正面容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| safeArea | 内容区内边距（px），控制子元素与卡片边界的距离 | `number` | `20` |
| className | 附加样式类 | `string` | - |
| children | 正面内容 | `React.ReactNode` | - |

背面隐藏（`backface-hidden`），背景 `bg-foreground/90`，响应式圆角 `@xs:rounded-2xl rounded-lg`。

### CreditCardBack

卡片背面容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| safeArea | 内容区内边距（px） | `number` | `16` |
| className | 附加样式类 | `string` | - |
| children | 背面内容 | `React.ReactNode` | - |

当位于 CreditCardFlipper 内部时自动翻转 180 度（`rotate-y-180`）。通过 `CreditCardBackContext` 向 CreditCardMagStripe 提供 `safeArea`。

### CreditCardChip

芯片图案。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 自定义芯片内容；不传时渲染默认金色渐变 SVG 芯片 | `React.ReactNode` | 默认 SVG |

默认 SVG 为金色渐变芯片图案（`#EDE5A6` 到 `#CFA255`），尺寸 `w-1/6`，定位 `absolute top-1/2 left-0 -translate-y-1/2`。

### CreditCardLogo

Logo 占位区域。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

定位在卡片右上角（`absolute top-0 right-0`），尺寸 `size-1/6`。

### CreditCardServiceProvider

支付服务商标识。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 支付服务商类型（基于 react-svg-credit-card-payment-icons） | string | `"Visa"` |
| className | 附加样式类 | `string` | - |
| children | 自定义服务商标识；不传时使用 PaymentIcon 渲染 | `React.ReactNode` | 默认 PaymentIcon |

定位在右下角（`absolute right-0 bottom-0`），最大尺寸 `max-h-1/3 max-w-1/3`。

### CreditCardMagStripe

背面磁条。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

定位在背面上方（`top-[3%]`），高度 `h-1/4`，背景 `bg-gray-900`。宽度通过 Context 自动适配 safeArea。

### CreditCardNumber / CreditCardName / CreditCardExpiry / CreditCardCvv

展示型文本组件，均继承 `HTMLAttributes<HTMLParagraphElement>`，渲染为 `<p>` 元素，仅暴露 `className` 属性。

| 组件 | 默认样式 | 用途 |
| --- | --- | --- |
| CreditCardNumber | `font-mono @xs:text-2xl` | 卡号显示 |
| CreditCardName | `font-semibold uppercase` | 持卡人姓名 |
| CreditCardExpiry | `font-mono` | 有效期（如 "12/28"） |
| CreditCardCvv | `font-mono` | CVV 安全码 |

<h2 id="faq">常见问题</h2>

### 桌面端和移动端的翻转行为有何不同？

桌面端（支持 hover 的设备）通过鼠标悬停触发翻转；移动端（无 hover 能力）通过点击触发翻转。组件内部通过 `window.matchMedia("(hover: hover)")` 自动检测并切换交互模式。

### 如何自定义芯片图案？

传入 `children` 替代默认 SVG：

```tsx
<CreditCardChip>
  <img src="/custom-chip.png" alt="Custom chip" />
</CreditCardChip>
```

### 如何调整内容边距？

通过 `CreditCardFront` 和 `CreditCardBack` 的 `safeArea` 属性控制内容与卡片边界的间距（单位 px）：

```tsx
<CreditCardFront safeArea={32}>
  {/* 内容 */}
</CreditCardFront>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/credit-card` 导入所有需要的子组件。
2. CreditCard 作为最外层容器，CreditCardFlipper 包裹正反面，CreditCardFront/CreditCardBack 各含正面/背面内容。
3. CreditCardChip 默认渲染 SVG 芯片图案；需要自定义时传入 children。
4. CreditCardMagStripe 仅在 CreditCardBack 内部使用，自动适配 safeArea 宽度。
5. 不要将信用卡组件用于收集真实支付信息 —— 它仅用于展示。
6. 不要引入 Ant Design 相关依赖。
