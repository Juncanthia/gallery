树形控件 Tree。多层次的结构列表，支持展开折叠、节点选择（单选/多选）、层级连接线、图标和展开动画。采用复合组件模式，通过 `TreeProvider` 提供上下文，配合 `TreeNode`、`TreeNodeTrigger`、`TreeExpander`、`TreeIcon`、`TreeLabel`、`TreeNodeContent`、`TreeLines` 等子组件灵活组合。

<h2 id="when-to-use">何时使用</h2>

当需要展示具有层级结构的数据，并允许用户展开折叠、选择节点时使用。

常见场景：
- 文件目录浏览器，展示文件夹和文件的层级关系。
- 组织架构树，展示部门和成员的嵌套结构。
- 分类导航，展示多级分类的树形菜单。
- 权限配置界面，展示菜单/操作的权限树。

不推荐：
- 数据量极大（数千节点）但无需层级展开——使用虚拟列表或分页表格替代。
- 简单的父子选择关系——使用 Cascader 更简洁。
- 需要拖拽排序的树——本组件不支持拖拽，需扩展实现。
- 需要异步加载子节点（懒加载）——当前版本不支持，需在调用侧自行管理数据。

<h2 id="api">API</h2>

### Tree (TreeProvider)

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 树节点内容 | `ReactNode` | - |
| defaultExpandedIds | 默认展开的节点 ID 列表 | `string[]` | `[]` |
| selectedIds | 选中节点 ID 列表（受控） | `string[]` | - |
| onSelectionChange | 选择变化回调 | `(selectedIds: string[]) => void` | - |
| showLines | 是否显示层级连接线 | `boolean` | `true` |
| showIcons | 是否显示节点图标 | `boolean` | `true` |
| selectable | 是否允许节点选择 | `boolean` | `true` |
| multiSelect | 是否支持多选（Ctrl/Meta + 点击追加选择） | `boolean` | `false` |
| indent | 每级缩进距离（像素） | `number` | `20` |
| animateExpand | 是否启用展开/收起动画 | `boolean` | `true` |
| className | 自定义样式类名 | `string` | - |

### TreeNode

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| nodeId | 节点唯一标识，不传则自动生成 | `string` | 自动生成 |
| level | 节点层级（从 0 开始） | `number` | `0` |
| isLast | 是否为同级最后一个节点（影响连接线绘制） | `boolean` | `false` |
| children | 子节点内容 | `ReactNode` | - |
| className | 自定义样式 | `string` | - |

### TreeNodeTrigger

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 触发器内容（通常包含 Expander + Icon + Label） | `ReactNode` | - |
| className | 自定义样式 | `string` | - |
| onClick | 点击事件回调 | `(e: MouseEvent) => void` | - |

点击 `TreeNodeTrigger` 会同时触发展开/折叠和节点选择。`TreeLines` 会自动在 Trigger 内部渲染。

### TreeExpander

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| hasChildren | 是否有子节点，无子节点时渲染占位 | `boolean` | `false` |
| className | 自定义样式 | `string` | - |
| onClick | 点击事件回调 | `(e: MouseEvent) => void` | - |

点击仅触发展开/折叠，不会触发节点选择（通过 `e.stopPropagation()` 阻止冒泡）。

### TreeIcon

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| icon | 自定义图标，不传则使用默认文件夹/文件图标 | `ReactNode` | 自动判断 |
| hasChildren | 是否有子节点（影响默认图标：文件夹 vs 文件） | `boolean` | `false` |
| className | 自定义样式 | `string` | - |

### TreeLabel

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 标签文本 | `ReactNode` | - |
| className | 自定义样式 | `string` | - |

### TreeNodeContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| hasChildren | 是否有子节点 | `boolean` | `false` |
| children | 子节点列表 | `ReactNode` | - |
| className | 自定义样式 | `string` | - |

`TreeNodeContent` 使用 `AnimatePresence` + `motion.div` 实现展开/收起的高度动画。

### TreeLines

无需传参的内部组件，根据当前的层级信息自动绘制垂直和水平连接线。由 `TreeNodeTrigger` 自动调用，一般不需手动使用。

### TreeView

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 树内容 | `ReactNode` | - |
| className | 自定义样式，默认带 `p-2` | `string` | - |

<h2 id="faq">常见问题</h2>

### Tree 子组件如何使用？
Tree 采用复合组件模式，所有子组件通过 `Tree.<SubComponent>` 访问（如 `Tree.TreeNode`、`Tree.TreeNodeTrigger`）。或者从 `@/components/core/tree` 按需解构导入各个子组件。两者等价。

### 选择模式如何控制？
`selectable` 控制是否允许选择，设为 `false` 时点击仅展开/折叠。`multiSelect` 控制多选模式，设为 `true` 后按住 Ctrl/Meta 键点击可追加选择。受控模式下传入 `selectedIds` + `onSelectionChange` 管理选中状态。

### Tree 和 TreeSelect/Cascader 的区别？
Tree 是纯粹的展示和选择组件，适合嵌入在页面中展示完整的层级结构。TreeSelect 是下拉选择器形式的树，适合表单场景。Cascader 是级联选择器，适合逐级选择的场景。三者定位不同，不可混用。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/tree` 导入 `Tree`，不要从 `antd` 导入。
2. 必须用 `Tree` 作为根容器（即 `TreeProvider`），子组件使用 `Tree.TreeNode` 等点号语法访问。
3. 每个 `TreeNode` 内部应先放置 `TreeNodeTrigger`（包含 Expander + Icon + Label），再放置 `TreeNodeContent`（包含子级 TreeNode 列表）。
4. 有子节点的 `TreeExpander` 和 `TreeIcon` 需设置 `hasChildren`。`TreeNodeContent` 也需设置 `hasChildren` 以正确触发动画。
5. `nodeId` 必须唯一，不传时自动生成但不可控——受控选择场景必须显式传入。
6. 子级 TreeNode 必须设置 `level` 属性为父级 level + 1。
7. 需要自定义图标时通过 `TreeIcon` 的 `icon` prop 传入。
8. 不要引入 Ant Design 相关依赖。
