# ImageCrop 图片裁剪

基于 react-blocks/image-crop 的图片裁剪复合组件。通过 `ImageCrop` 提供裁剪上下文，配合 `ImageCropContent`（裁剪区域）、`ImageCropApply`（确认按钮）和 `ImageCropReset`（重置按钮）组合构建完整的图片裁剪 UI。同时保留 `Cropper` 便捷组件用于简单场景。

本组件为项目本地复合组件（Blocks），无 Ant Design 直接对应。

## 何时使用 {#when-to-use}

需要允许用户对上传的图片进行裁剪调整时使用。

常见场景：

- 头像上传裁剪。
- 封面图固定宽高比裁剪。
- 需要对裁剪结果进行文件大小限制的场景。
- 需要自定义裁剪工具栏 UI 的场景（通过组合 ImageCropApply、ImageCropReset）。

不推荐：

- 不需要用户交互的自动裁剪——使用 CSS object-fit 或服务端裁剪。
- 需要旋转、翻转等高级编辑——使用完整的图片编辑器。
- 仅展示图片而不需要裁剪功能。

## 代码演示 {#examples}

### 基础用法

使用 `ImageCrop` 包裹 `ImageCropContent`，搭配 `ImageCropApply` 和 `ImageCropReset` 操作按钮。

```tsx
import {
  ImageCrop,
  ImageCropContent,
  ImageCropApply,
  ImageCropReset,
} from "@/components/blocks/image-crop"

export default function Demo() {
  const [file, setFile] = useState<File | null>(null)

  const handleCrop = (croppedImage: string) => {
    console.log("裁剪结果:", croppedImage)
  }

  if (!file) {
    return (
      <input
        type="file"
        accept="image/*"
        onChange={(e) => setFile(e.target.files?.[0] ?? null)}
      />
    )
  }

  return (
    <ImageCrop file={file} onCrop={handleCrop} aspect={1}>
      <ImageCropContent />
      <div className="flex gap-2 mt-2">
        <ImageCropApply />
        <ImageCropReset />
      </div>
    </ImageCrop>
  )
}
```

### 使用 Cropper 便捷组件

简单场景可直接使用 `Cropper` 一步完成。

```tsx
import { Cropper } from "@/components/blocks/image-crop"

export default function Demo() {
  const [file, setFile] = useState<File | null>(null)

  if (!file) {
    return (
      <input
        type="file"
        accept="image/*"
        onChange={(e) => setFile(e.target.files?.[0] ?? null)}
      />
    )
  }

  return (
    <Cropper
      file={file}
      aspect={16 / 9}
      onCrop={(croppedImage) => console.log(croppedImage)}
    />
  )
}
```

## API {#api}

### ImageCrop

裁剪上下文容器，封装 react-blocks/image-crop 的状态管理和裁剪逻辑。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| file | `File` | 必填 | 待裁剪的图片文件 |
| maxImageSize | `number` | `1024 * 1024 * 5`（5MB） | 裁剪结果最大文件大小（bytes），超限自动缩小 |
| onCrop | `(croppedImage: string) => void` | - | 裁剪完成回调，参数为 PNG base64 data URL |
| children | `ReactNode` | 必填 | 子组件，通常为 ImageCropContent + 操作按钮 |
| onChange | `(pixelCrop: PixelCrop, percentCrop: PercentCrop) => void` | - | react-blocks/image-crop 的 onChange 回调 |
| onComplete | `(pixelCrop: PixelCrop, percentCrop: PercentCrop) => void` | - | react-blocks/image-crop 的 onComplete 回调 |

同时接受 `ReactCropProps` 的其他属性（如 `aspect`、`minWidth`、`minHeight`、`circularCrop`、`disabled`、`locked` 等），通过展开传入。`onChange`、`onComplete`、`children` 由组件内部处理，不会穿透到原生 ReactCrop props。

### ImageCropContent

裁剪区域渲染组件。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| style | `CSSProperties` | - | 附加内联样式 |
| className | `string` | - | 附加 className |

### ImageCropApply

裁剪确认按钮，点击后执行裁剪并通过 `onCrop` 返回结果。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| asChild | `boolean` | `false` | 使用 Radix Slot 将样式合并到子元素 |
| children | `ReactNode` | `<CropIcon />` | 按钮内容 |
| onClick | `(e: MouseEvent<HTMLButtonElement>) => void` | - | 裁剪完成后触发的点击回调 |
| （其他） | `ComponentProps<typeof Button>` | - | 支持 Button 组件的全部属性（variant、size 等） |

### ImageCropReset

裁剪重置按钮，将裁剪区域恢复到初始居中状态。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| asChild | `boolean` | `false` | 使用 Radix Slot 将样式合并到子元素 |
| children | `ReactNode` | `<RotateCcwIcon />` | 按钮内容 |
| onClick | `(e: MouseEvent<HTMLButtonElement>) => void` | - | 重置完成后触发的点击回调 |
| （其他） | `ComponentProps<typeof Button>` | - | 支持 Button 组件的全部属性 |

### Cropper

向后兼容的便捷组件，将 `ImageCrop` + `ImageCropContent` 合并为一步调用。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| file | `File` | 必填 | 待裁剪的图片文件 |
| maxImageSize | `number` | - | 裁剪结果最大文件大小 |
| onCrop | `(croppedImage: string) => void` | - | 裁剪完成回调 |
| onChange | `(pixelCrop: PixelCrop, percentCrop: PercentCrop) => void` | - | react-blocks/image-crop onChange 回调 |
| onComplete | `(pixelCrop: PixelCrop, percentCrop: PercentCrop) => void` | - | react-blocks/image-crop onComplete 回调 |
| className | `string` | - | 裁剪区域 className |
| style | `CSSProperties` | - | 裁剪区域内联样式 |
| （其他） | `Omit<ReactCropProps, "onChange">` | - | react-blocks/image-crop 的其他属性（如 aspect） |

## 样式模型 {#styling-model}

- 使用 Tailwind CSS 和项目主题变量。
- `ImageCropContent`：默认最大高度 `max-h-[277px]`，通过 CSS 变量 `--rc-border-color`（映射为 `--color-border`）和 `--rc-focus-color`（映射为 `--color-primary`）适配 shadcn 主题色。
- `ImageCropApply`：默认方形文本按钮（`shape="square" variant="text"`），图标为 lucide-react CropIcon。
- `ImageCropReset`：默认方形文本按钮（`shape="square" variant="text"`），图标为 lucide-react RotateCcwIcon。
- 无 `aspect` 时，默认裁剪区域覆盖图片 90% 区域（`width: 90, height: 90, unit: "%"`）。
- `ImageCropApply` 和 `ImageCropReset` 继承 Button 组件的全部样式能力。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/image-crop/index.tsx`：所有导出组件定义。

直接依赖：

- `react-blocks/image-crop`：裁剪交互核心（ReactCrop、centerCrop、makeAspectCrop）。
- `radix-ui`：Slot 用于 asChild 模式。
- `lucide-react`：CropIcon、RotateCcwIcon。
- `@/components/core/button`：Button 组件。
- `@/lib/utils`：cn 工具函数。
- `react-blocks/image-crop/dist/ReactCrop.css`：react-blocks/image-crop 必需样式。

实现要点：

1. `ImageCrop` 通过 `FileReader.readAsDataURL` 将 file 转为 data URL 作为图片源。
2. 图片加载时通过 `centerAspectCrop` 计算初始裁剪区域并居中。
3. 裁剪结果通过 Canvas API 渲染为 PNG data URL；若文件大小超限，递归缩小画布直至符合 `maxImageSize`。
4. `Cropper` 是向后兼容的便捷包装：`<ImageCrop><ImageCropContent /></ImageCrop>`。
5. `ImageCropApply` 和 `ImageCropReset` 均支持 `asChild`，可通过 Radix Slot 替换为自定义按钮。
6. `resetCrop` 将裁剪区域恢复到图片加载时的初始居中状态。

## 可访问性 {#accessibility}

- 裁剪图片使用 `alt="crop"` 标识。
- `ImageCropApply` 和 `ImageCropReset` 基于 `<button>` 元素，支持原生键盘交互和焦点管理。
- 在 `asChild` 模式下，调用侧需自行确保子元素的可访问名称（如 `aria-label`）。
- 推荐在裁剪区域外提供明确的操作提示文本。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。Ant Design 不提供图片裁剪组件。

| 方向 | 本地决策 |
| --- | --- |
| 裁剪框架 | react-blocks/image-crop，非 AntD 生态 |
| 视觉系统 | Tailwind CSS + shadcn 主题变量 |
| 组合模式 | Context + 子组件组合，支持自定义工具栏 |
| 文件输出 | Canvas API → PNG data URL，支持大小限制递归缩放 |
| 便捷模式 | Cropper 组件用于简单场景 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 ImageCrop 代码时，应遵守：

1. 从 `@/components/blocks/image-crop` 导入所需组件。
2. `ImageCrop` 必须提供 `file` 属性，通过 children 组合子组件。
3. 需要固定裁剪比例时，通过 `aspect` 属性设置（如 `aspect={1}` 为正方形）。
4. 完整用法推荐：`ImageCrop` > `ImageCropContent` + `ImageCropApply` + `ImageCropReset`。
5. 简单场景可使用 `Cropper` 便捷组件。
6. `ImageCropApply` 和 `ImageCropReset` 的 `onClick` 在裁剪/重置完成之后触发，可用于自定义副作用。
7. 通过 `maxImageSize` 控制裁剪输出文件大小（单位 bytes），默认 5MB。
8. 不要引入 Ant Design Upload 或 Image 组件替代本地的文件选择和裁剪流程。
9. 只使用本文档 API 表格和源码中存在的 props，不要凭 react-blocks/image-crop 文档推测未暴露的属性。
