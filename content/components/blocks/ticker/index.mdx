行情条 Ticker。金融资产行情展示复合组件，通过 Context 共享 Intl.NumberFormat 货币格式化器，支持图标、代码、价格和涨跌幅的组合展示。

<h2 id="when-to-use">何时使用</h2>

当需要展示金融资产的行情信息时使用，适合构建行情列表、自选股列表、资产概览等场景。

常见场景：
- 行情列表页，每个条目显示图标、代码、价格和涨跌幅
- 资产详情页的标题区域，快速展示核心行情数据
- 自选股/收藏夹的列表项
- 需要多币种、多区域格式化支持的国际金融应用

不推荐：
- 非金融场景的通用数据展示——使用普通的 flex 布局
- 仅需单个行情字段——可单独使用子组件，但仍需包裹在 `Ticker` 中提供 Context
- 实时行情推送（WebSocket 更新）——Ticker 仅负责渲染，不处理数据更新

<h2 id="api">API</h2>

### Ticker

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| currency | 货币代码（ISO 4217） | `string` | `"USD"` |
| locale | BCP 47 语言标签 | `string` | `"en-US"` |
| className | 附加样式类名 | `string` | - |

`Ticker` 渲染为 `<button type="button">`，通过 Context 向下提供 `Intl.NumberFormat` 实例。若构造失败则回退到 `en-US` + `USD`。

### TickerIcon

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| src | 图标/Logo 图片 URL | `string` | - |
| symbol | 资产代码，用于生成 fallback 首字母 | `string` | - |
| asChild | 是否透传子元素（切换为 `<div>` 容器） | `boolean` | - |
| className | 附加样式类名 | `string` | - |

默认渲染为 Avatar；`asChild` 模式下透传 `children`。

### TickerSymbol

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| symbol | 资产代码（必填，自动转大写） | `string` | - |
| className | 附加样式类名 | `string` | - |

### TickerPrice

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| price | 当前价格（必填），由 Context formatter 格式化为货币字符串 | `number` | - |
| className | 附加样式类名 | `string` | - |

### TickerPriceChange

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| change | 涨跌幅数值，正值或负值 | `number` | - |
| isPercent | 是否为百分比格式 | `boolean` | `false` |
| className | 附加样式类名 | `string` | - |

正值显示绿色（涨），负值显示红色（跌），并渲染三角箭头图标。`isPercent` 为 true 时格式化为百分比（如 `"2.35%"`），否则使用货币格式化器。

<h2 id="faq">常见问题</h2>

### 为什么 TickerPrice 或 TickerPriceChange 没有正确格式化？
这两个子组件依赖 `Ticker` 提供的 Context 来获取 `Intl.NumberFormat` 实例。确保它们被包裹在 `Ticker` 内部。如果 `currency` 或 `locale` 无效，组件会自动回退到 `en-US` + `USD`。

### 如何自定义涨跌幅的颜色？
颜色由组件内部根据 `change` 的正负自动切换（正值绿、负值红），使用 Tailwind 的 `text-green-600`/`text-red-600` 类。如需自定义，可通过 `className` 覆盖。

### 所有子组件都使用 React.memo 有什么好处？
`Ticker` 通常在行情列表中大量渲染，所有子组件使用 `React.memo` 包裹可避免不必要的重渲染，提升列表滚动性能。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/ticker` 导入，不要从 `antd` 导入。
2. `Ticker` 必须包裹所有子组件（TickerIcon、TickerSymbol、TickerPrice、TickerPriceChange），以提供格式化 Context。
3. `TickerSymbol` 的 `symbol` 为必填，自动转为大写显示。
4. `TickerPrice` 的 `price` 为必填，格式由 `Ticker` 的 `currency`/`locale` 决定。
5. 不要引入 Ant Design 相关依赖。
