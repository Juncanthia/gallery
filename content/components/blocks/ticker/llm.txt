# Ticker 滚动条

金融资产行情展示组件，提供组合式 API 用于构建股票/加密货币等的行情条目。通过 Context 共享 `Intl.NumberFormat` 货币格式化器，确保价格和涨跌幅数值在同一 Ticker 实例内格式一致。

本组件为项目本地复合组件，无 Ant Design 直接对应。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

当需要展示金融资产的行情信息时使用，适合构建行情列表、自选股列表、资产概览等场景。

常见场景：

- 行情列表页，每个条目显示图标、代码、价格和涨跌幅。
- 资产详情页的标题区域，快速展示核心行情数据。
- 自选股/收藏夹的列表项。
- 需要多币种、多区域格式化支持的国际金融应用。

不推荐：

- 非金融场景的通用数据展示——使用普通的 flex 布局。
- 仅需单个行情字段——可单独使用 `TickerSymbol`、`TickerPrice` 等子组件，但仍需包裹在 `Ticker` 中提供 Context。
- 实时行情推送（WebSocket 更新）——Ticker 本身不处理数据更新，仅负责渲染。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import {
  Ticker,
  TickerIcon,
  TickerSymbol,
  TickerPrice,
  TickerPriceChange,
} from "@/components/blocks/ticker"

export default function Demo() {
  return (
    <Ticker>
      <TickerIcon symbol="BTC" />
      <div className="flex flex-col items-start">
        <div className="flex items-center gap-1">
          <TickerSymbol symbol="BTC" />
          <TickerPrice price={67420.50} />
        </div>
        <TickerPriceChange change={1240.30} />
      </div>
    </Ticker>
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。`Ticker` 提供货币格式化 Context；`TickerPrice` 和 `TickerPriceChange` 依赖此 Context 来格式化数值。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

### Ticker

```ts
type TickerProps = HTMLAttributes<HTMLButtonElement> & {
  /** 货币代码（ISO 4217），默认 "USD" */
  currency?: string;
  /** BCP 47 语言标签，默认 "en-US" */
  locale?: string;
  children: ReactNode;
}
```

`Ticker` 渲染为 `<button type="button">`，通过 Context 向下提供 `Intl.NumberFormat` 实例。`currency` 和 `locale` 参数用于构造格式化器；若构造失败则回退到 `en-US` + `USD`。

### TickerIcon

```ts
type TickerIconProps = HTMLAttributes<HTMLImageElement> & {
  /** 图标/Logo 图片 URL */
  src?: string;
  /** 资产代码，用于生成 fallback 首字母 */
  symbol?: string;
  /** 是否透传子元素（asChild 模式） */
  asChild?: boolean;
}
```

`TickerIcon` 默认渲染为 Avatar，使用 `src` 作为图片、`symbol` 前两个字符作为 fallback。`asChild` 模式下改为渲染 `<div>` 容器并透传 `children`。

### TickerSymbol

```ts
type TickerSymbolProps = HTMLAttributes<HTMLSpanElement> & {
  /** 资产代码，必填，会自动转为大写 */
  symbol: string;
}
```

`TickerSymbol` 渲染资产代码文本，默认 `font-medium`。

### TickerPrice

```ts
type TickerPriceProps = HTMLAttributes<HTMLSpanElement> & {
  /** 当前价格，必填，由 Context 的 formatter 格式化为货币字符串 */
  price: number;
}
```

`TickerPrice` 通过 `useTickerContext` 获取格式化器来格式化 `price`。

### TickerPriceChange

```ts
type TickerPriceChangeProps = HTMLAttributes<HTMLSpanElement> & {
  /** 涨跌幅数值，正值或负值 */
  change: number;
  /** 是否为百分比，默认 false */
  isPercent?: boolean;
}
```

`TickerPriceChange` 根据 `change` 的正负自动设置颜色（绿色涨/红色跌），并渲染三角箭头图标。`isPercent` 为 true 时格式化为百分比字符串（如 `"2.35%"`），否则使用 Context 的货币格式化器。

## Semantic DOM {#semantic-dom}

源码中使用的稳定语义槽位：

- `role="img"` — 涨跌箭头 SVG 图标
- `aria-labelledby="blocks/ticker-change-icon-title"` — 箭头图标的可访问标签引用
- `<title>` — 箭头图标的语义标题（"Up icon" / "Down icon"）

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- `Ticker` 使用 `inline-flex items-center gap-1.5 whitespace-nowrap align-middle`。
- `TickerIcon` 基于 Avatar，头像 `size-7 border border-border bg-muted`。
- `TickerSymbol` 使用 `font-medium`。
- `TickerPrice` 使用 `text-muted-foreground`。
- `TickerPriceChange` 正值为 `text-green-600 dark:text-green-500`，负值为 `text-red-600 dark:text-red-500`。
- 所有子组件均使用 `React.memo` 包裹以优化渲染性能。
- 主题颜色优先使用 `bg-muted`、`text-foreground`、`text-muted-foreground`、`border-border` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/ticker/index.tsx`
- `src/components/core/blocks/ticker.tsx`（barrel 导出）

直接依赖摘要：

- `@/components/core/avatar` — TickerIcon 的头像展示
- `@/kit/utils` — `cn` 样式合并工具
- `react` — Context、memo、useMemo

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 所有子组件均使用 `React.memo` 包裹并设置 `displayName`。
3. `Ticker` 使用 Context 向下传递 `Intl.NumberFormat` 实例；`currency` 或 `locale` 变更时重建格式化器。
4. 若 `Intl.NumberFormat` 构造失败（无效 locale/currency），回退到默认 `en-US` + `USD`。
5. `TickerPriceChange` 的格式化逻辑区分 `isPercent`：true 时使用 `toFixed(2)` + `%`，false 时使用货币格式化器。
6. 涨跌箭头的 SVG 通过 `rotate-180` 控制方向，使用内联 `<title>` 提供可访问语义。
7. 如果需要新增子组件，应先修改 `src/components/blocks/ticker/index.tsx`，再更新本 llm 文档。

## 可访问性 {#accessibility}

- `Ticker` 渲染为 `<button>`，保留原生键盘交互。
- 涨跌箭头 SVG 设置 `role="img"` 和 `aria-labelledby` 引用 `<title>` 元素。
- 箭头 `<title>` 文本为 "Up icon" / "Down icon"，提供语义描述。
- 数值颜色变化（绿涨红跌）不应作为唯一的信息传达方式。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。AntD 没有行情展示类组件，其 `Statistic` 组件仅展示单一数值，不具备货币上下文共享和涨跌幅组合展示。

| 方向 | 本地决策 |
| --- | --- |
| 组件来源 | 项目本地复合，非 AntD 封装 |
| API 来源 | 以 `src/components/blocks/ticker/index.tsx` 的 TypeScript 类型为准 |
| 格式化方案 | 使用原生 `Intl.NumberFormat`，非第三方库 |
| Context 模式 | 通过 TickerContext 共享格式化器 |
| 渲染优化 | 所有子组件使用 React.memo |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Ticker 代码时，应遵守：

1. 从 `@/components/blocks/ticker` 导入，不要从 `antd` 导入。
2. `Ticker` 必须包裹所有子组件（TickerIcon、TickerSymbol、TickerPrice、TickerPriceChange），以提供格式化 Context。
3. `TickerSymbol` 的 `symbol` 为必填，会自动转为大写显示。
4. `TickerPrice` 的 `price` 为必填，格式由 `Ticker` 的 `currency`/`locale` 决定。
5. `TickerPriceChange` 的 `change` 支持正负值，颜色自动切换（正值绿、负值红）。
6. `TickerPriceChange` 的 `isPercent` 控制格式化为百分比还是货币值。
7. 只使用本文件 API 摘要和源码中存在的 props。
8. 不要生成 AntD 的 `Statistic`、`ConfigProvider`、`classNames`、`styles` 等。
9. 如果不确定某个 API 是否存在，先检查 `src/components/blocks/ticker/index.tsx`，不要凭记忆生成。
