锚点 Anchor。用于页面内锚点导航，点击后平滑滚动到目标区域，并高亮当前激活项。支持受控和非受控两种模式。

<h2 id="when-to-use">何时使用</h2>

当页面内容较长需要内部导航目录，点击后平滑滚动到对应区块。

常见场景：

- 文档页的章节目录导航。
- 单页应用的区块间跳转。
- 需要高亮当前阅读位置的侧边导航栏。
- 帮助中心或 FAQ 页面的快速定位。

不推荐：

- 页面之间的路由跳转，应使用 `Link` 或 `a` 标签。
- 仅需锚点功能但不需要导航 UI 时，直接用 `<a href="#id">` 即可。
- 多级嵌套锚点（Anchor 仅支持单层扁平列表），需要层级结构时应自行实现。

<h2 id="examples">代码演示</h2>

### 基础用法

传入 `items` 数组，每个 item 包含 `key`、`href`、`title`。受控模式下传入 `activeKey` 和 `onChange`。

/>

```tsx
import { useState } from "react"
import { Anchor } from "@/components/core/anchor"

export default function Demo() {
  const [active, setActive] = useState("about")

  return (
    <div className="flex gap-8">
      <div className="w-48 shrink-0">
        <Anchor
          items={[
            { key: "about", href: "#about", title: "关于" },
            { key: "features", href: "#features", title: "功能特性" },
            { key: "api", href: "#api", title: "API" },
            { key: "faq", href: "#faq", title: "常见问题" },
          ]}
          activeKey={active}
          onChange={setActive}
        />
      </div>
      <div className="flex-1 space-y-32 text-sm">
        <section id="about"><h3 className="text-lg font-semibold">关于</h3><p className="text-muted-foreground">产品介绍内容...</p></section>
        <section id="features"><h3 className="text-lg font-semibold">功能特性</h3><p className="text-muted-foreground">功能列表...</p></section>
        <section id="api"><h3 className="text-lg font-semibold">API</h3><p className="text-muted-foreground">接口文档...</p></section>
        <section id="faq"><h3 className="text-lg font-semibold">常见问题</h3><p className="text-muted-foreground">问题解答...</p></section>
      </div>
    </div>
  )
}
```

### 非受控模式

不传 `activeKey`，传 `defaultActiveKey` 设置初始激活项，组件自行管理激活状态。

```tsx
import { Anchor } from "@/components/core/anchor"

export default function Demo() {
  return (
    <Anchor
      defaultActiveKey="about"
      items={[
        { key: "about", href: "#about", title: "关于" },
        { key: "features", href: "#features", title: "功能特性" },
      ]}
    />
  )
}
```

### 隐藏指示条

设置 `showInk={false}` 隐藏左侧激活指示条，仅通过文字样式区分激活态。

```tsx
import { Anchor } from "@/components/core/anchor"

export default function Demo() {
  return (
    <Anchor
      showInk={false}
      items={[
        { key: "about", href: "#about", title: "关于" },
        { key: "features", href: "#features", title: "功能特性" },
      ]}
    />
  )
}
```

### 设置滚动偏移

当页面有固定顶栏时，通过 `offsetTop` 避免目标区域被遮挡。

```tsx
import { Anchor } from "@/components/core/anchor"

export default function Demo() {
  return (
    <Anchor
      offsetTop={80}
      items={[
        { key: "about", href: "#about", title: "关于" },
        { key: "features", href: "#features", title: "功能特性" },
      ]}
    />
  )
}
```

<h2 id="api">API</h2>

### Anchor

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| items | 锚点列表（必填） | `AnchorLink[]` | - |
| activeKey | 受控的当前激活 key | `string` | - |
| defaultActiveKey | 默认激活的 key | `string` | `""` |
| onChange | 激活项变化回调 | `(key: string) => void` | - |
| showInk | 是否显示激活指示条（左侧竖条） | `boolean` | `true` |
| offsetTop | 滚动时顶部偏移量（px） | `number` | `0` |
| className | 根元素自定义样式类 | `string` | - |

### AnchorLink

`items` 数组的元素类型。

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| key | 唯一标识 | `string` |
| href | 目标元素的 CSS 选择器（如 `#section-id`） | `string` |
| title | 显示的文本或节点 | `React.ReactNode` |

<h2 id="faq">常见问题</h2>

### 受控模式 vs 非受控模式？

- **受控**：传入 `activeKey` + `onChange`，由外部状态管理当前激活项，适用于需要同步锚点状态到其他 UI 的场景。
- **非受控**：只传 `defaultActiveKey`，组件内部自行管理状态，适用于简单场景。

不要同时传入 `activeKey` 和 `defaultActiveKey`——受控优先，`defaultActiveKey` 会被忽略。

### 如何自定义滚动偏移？

设置 `offsetTop`，例如有固定顶栏 64px 时传入 `offsetTop={64}`，确保滚动后目标区域不被顶栏遮挡。

### 如何在 Next.js 中使用？

Anchor 内部使用 `document.querySelector` 获取目标元素，源文件已声明 `"use client"`，可直接在客户端组件中使用。

### href 支持哪些格式？

支持任意 `document.querySelector` 可解析的 CSS 选择器。最常用的是 `#section-id`，也支持 `[data-section="xxx"]` 等更复杂的选择器。确保页面中存在对应的目标元素，否则点击无效。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/anchor` 导入。
2. `items` 中的 `href` 必须是有效 CSS 选择器（如 `#section-id`），确保页面中存在对应元素。
3. 受控场景同时传入 `activeKey` 和 `onChange`，非受控场景传入 `defaultActiveKey`，不要混用。
4. 如果有固定顶栏，务必设置 `offsetTop` 避免目标内容被遮挡。
5. 不要引入 Ant Design 相关依赖。
