对话框堆叠 DialogStack。一套组合式对话框堆叠系统，将多个对话框内容组织为栈式结构，支持步进导航、可点击回溯和受控/非受控开关。

<h2 id="when-to-use">何时使用</h2>

当需要将多个步骤组织在一个对话框栈中，用户可在步骤之间前后切换时使用。

常见场景：

- 多步骤表单（注册向导、配置向导）。
- 分步确认流程（删除确认、后果提示、最终确认）。
- 引导式 onboarding 教程。
- 包含多个子操作的复合弹窗。

不推荐：

- 单一步骤的简单确认弹窗，使用 AlertDialog 更合适。
- 纯内容展示弹窗不需要步进导航的场景，使用 Dialog 即可。
- 超过 5 步的超级向导，建议拆分为独立的多页流程。

### 可点击回溯 + 受控开启

```tsx
import { useState } from "react"
import {
  DialogStack,
  DialogStackTrigger,
  DialogStackOverlay,
  DialogStackBody,
  DialogStackContent,
} from "@/components/blocks/dialog-stack"
import { Button } from "@/components/core/button"

export default function Demo() {
  const [open, setOpen] = useState(false)

  return (
    <DialogStack open={open} onOpenChange={setOpen} clickable>
      <DialogStackTrigger asChild>
        <Button>受控开启</Button>
      </DialogStackTrigger>
      <DialogStackOverlay />
      <DialogStackBody>
        <DialogStackContent>
          <h3>第一步</h3>
          <p>点击此卡片可回到第一步。</p>
        </DialogStackContent>
        <DialogStackContent>
          <h3>第二步</h3>
          <p>点击下方第一步卡片可回溯。</p>
        </DialogStackContent>
        <DialogStackContent>
          <h3>第三步</h3>
          <p>点击前两步卡片可回溯。</p>
        </DialogStackContent>
      </DialogStackBody>
    </DialogStack>
  )
}
```

<h2 id="api">API</h2>

### DialogStack

顶层容器，管理开启/关闭状态和当前活跃索引。通过 Context 为所有子组件提供共享状态。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| open | 受控开启状态 | `boolean` | - |
| defaultOpen | 非受控默认开启状态 | `boolean` | `false` |
| onOpenChange | 开启状态变化回调 | `(open: boolean) => void` | - |
| clickable | 是否允许点击底层对话框回溯到该步骤 | `boolean` | `false` |
| className | 根容器附加 className | `string` | - |

### DialogStackTrigger

触发打开对话框堆叠的按钮。支持 `asChild` 模式包裹自定义触发器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 使用 Slot 模式渲染子元素 | `boolean` | `false` |
| className | 附加 className | `string` | - |

继承 `ButtonHTMLAttributes<HTMLButtonElement>`。

### DialogStackOverlay

背景遮罩层，点击可关闭整个对话框栈。仅在 `isOpen` 为 `true` 时渲染。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。

### DialogStackBody

对话框内容容器，通过 Portal 渲染到 `document.body` 层级。遍历子 `DialogStackContent` 按序分配 `index` 属性。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |
| children | 对话框内容，须为 `DialogStackContent` 元素 | `ReactElement<DialogStackChildProps>[]` | - |

继承 `HTMLAttributes<HTMLDivElement>`。仅在 `isOpen` 时渲染。

### DialogStackContent

单个对话框步骤的内容容器。根据与 `activeIndex` 的距离计算 Y 位移、宽度、透明度和 z-index。当前激活步骤以 `relative` 定位，后续步骤向下偏移，前序步骤向上偏移。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| index | 对话框在栈中的索引，由 Body 自动分配 | `number` | `0` |
| offset | 每层偏移距离（px） | `number` | `10` |
| className | 附加 className | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。

### DialogStackHeader / DialogStackTitle / DialogStackDescription

- **DialogStackHeader**：头部容器，包裹 Title 和 Description。继承 `HTMLAttributes<HTMLDivElement>`。
- **DialogStackTitle**：对话框标题，渲染为 `<h2>`。继承 `HTMLAttributes<HTMLHeadingElement>`。
- **DialogStackDescription**：描述文本，渲染为 `<p>`。继承 `HTMLAttributes<HTMLParagraphElement>`。

### DialogStackFooter / DialogStackNext / DialogStackPrevious

- **DialogStackFooter**：底部操作区容器。继承 `HTMLAttributes<HTMLDivElement>`。
- **DialogStackNext**：下一步按钮，到达最后一步时自动 `disabled`，默认文案 "Next"。
- **DialogStackPrevious**：上一步按钮，在第一步时自动 `disabled`，默认文案 "Previous"。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 使用 Slot 模式渲染子元素 | `boolean` | `false` |
| className | 附加 className | `string` | - |

继承 `ButtonHTMLAttributes<HTMLButtonElement>`。

<h2 id="faq">常见问题</h2>

### 如何让用户点击之前的步骤卡片来回溯？

设置 `clickable` 为 `true`。当 `clickable` 开启且用户点击索引小于 `activeIndex` 的卡片时，`activeIndex` 会回退到该步骤，同时卡片上会显示 `cursor: pointer` 样式。

### 如何控制 DialogStack 的开关状态？

使用受控模式 `open` + `onOpenChange`，或非受控模式 `defaultOpen`。组件内部使用 `@radix-ui/react-use-controllable-state` 管理状态。

### DialogStack 和 Dialog 有什么区别？

DialogStack 专为多步骤对话框设计，内置了步骤索引管理、视觉栈效果（Y 位移、透明度、z-index 堆叠）和前后步进导航。Dialog 是单一对话框，适合单步骤内容展示或表单。

### 为什么我的 DialogStackBody 内渲染不出内容？

确保 `DialogStackBody` 的子元素是 `DialogStackContent` 组件。`Body` 通过 `Children.map` 为 `DialogStackContent` 分配 `index`，其他类型的子元素不会被正确处理。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/dialog-stack` 导入所有需要的子组件。
2. 组装顺序为：DialogStack > DialogStackTrigger > DialogStackOverlay > DialogStackBody > DialogStackContent[]。
3. DialogStackContent 按步骤顺序排列，Body 会自动分配索引。
4. 每个步骤的操作区使用 DialogStackFooter 包裹 DialogStackNext / DialogStackPrevious。
5. 如需允许点击底层步骤回溯，设置 `clickable` 为 true。
6. 如需受控开关，使用 `open` + `onOpenChange`；非受控场景使用 `defaultOpen`。
7. 必须显式放置 `DialogStackOverlay`，否则无背景遮罩层。
8. 单步骤确认弹窗应使用 AlertDialog，多步骤向导使用本组件。
9. 不要引入 Ant Design 相关依赖。
