# Stories 快拍轮播

基于 Carousel primitive 构建的快拍（Stories）展示组件，常用于社交场景中的用户快拍轮播。每个快拍项支持视频预览（hover 自动播放/暂停并重置播放位置）、图片封面、作者头像与名称、标题和渐变遮罩。

本组件为项目本地复合组件，无 Ant Design 直接对应。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

当需要在界面中展示社交快拍（Stories）轮播时使用，适合类似 Instagram/Facebook Stories 的用户状态流展示。

常见场景：

- 首页、个人主页的快拍入口列表。
- 社交动态中的用户故事集合展示。
- 需要视频预览能力（hover 播放、离开复位）的内容轮播。
- 需要统一的圆角卡片、作者信息叠加和渐变遮罩的视觉风格。

不推荐：

- 通用图片轮播需求——使用 `src/components/core/carousel` 基础组件。
- 需要复杂动画切换效果的轮播——Stories 本身不包含转场动效，依赖底层 Carousel。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import {
  Stories,
  StoriesContent,
  Story,
  StoryVideo,
  StoryAuthor,
  StoryAuthorImage,
  StoryAuthorName,
  StoryTitle,
  StoryOverlay,
} from "@/components/blocks/stories"

export default function Demo() {
  return (
    <Stories className="w-full max-w-sm">
      <StoriesContent>
        <Story>
          <StoryVideo src="https://example.com/video.mp4#t=2" />
          <StoryOverlay />
          <StoryTitle>标题</StoryTitle>
          <StoryAuthor>
            <StoryAuthorImage name="User" src="https://example.com/avatar.jpg" />
            <StoryAuthorName>用户名</StoryAuthorName>
          </StoryAuthor>
        </Story>
      </StoriesContent>
    </Stories>
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。`StoryVideo` 支持 `src` 中的 `#t=秒数` 片段指定起始时间；`StoryOverlay` 的 `side` 控制渐变方向；完整用法参考下方 API。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

### Stories

```ts
type StoriesProps = ComponentProps<typeof Carousel> & {
  className?: string;
  opts?: CarouselOptions; // 默认值：align: "start", loop: false, dragFree: true
}
```

`Stories` 是 Carousel 的语义封装，默认配置为 `align="start"`、非循环、自由拖拽。传入的 `opts` 会与默认值合并（用户值优先）。

### StoriesContent

```ts
type StoriesContentProps = ComponentProps<typeof CarouselContent> & {
  className?: string;
}
```

`StoriesContent` 是 Carousel 内容容器，默认有 `-ml-2 gap-2` 的间距样式。

### Story

```ts
type StoryProps = HTMLAttributes<HTMLDivElement>;
```

`Story` 是单个快拍项容器，渲染为一个带 `role="button"` 和 `tabIndex={0}` 的可聚焦卡片。默认样式包括 `rounded-xl`、`bg-muted/40`、hover `scale-[1.02]` 和 `focus-visible:ring-2`。

### StoryVideo

```ts
type StoryVideoProps = VideoHTMLAttributes<HTMLVideoElement>;
```

`StoryVideo` 渲染视频元素。支持通过 `src` URL hash 指定起始时间，例如 `"video.mp4#t=3.5"`。默认 `loop`、`muted`、`preload="metadata"`。hover/focus 时自动播放，离开/blur 时暂停并重置到起始时间。

### StoryImage

```ts
type StoryImageProps = ComponentProps<"img"> & {
  alt: string; // 必填，需提供替代文本
}
```

`StoryImage` 渲染图片元素，使用 `absolute inset-0 object-cover` 填满卡片。`alt` 为必填属性。

### StoryAuthor

```ts
type StoryAuthorProps = HTMLAttributes<HTMLDivElement>;
```

作者信息容器，固定定位在快拍卡片底部，白色文字 `z-10`。

### StoryAuthorImage

```ts
type StoryAuthorImageProps = ComponentProps<typeof Avatar> & {
  src?: string;
  name?: string;
  fallback?: string;
}
```

作者头像，基于 `Avatar` 组件。默认 `size-6`，带 `border-white/20` 边框。若未提供 `fallback`，自动取 `name` 首字符大写。

### StoryAuthorName

```ts
type StoryAuthorNameProps = HTMLAttributes<HTMLSpanElement>;
```

作者名称文本，带有 `truncate font-medium text-sm` 样式。

### StoryTitle

```ts
type StoryTitleProps = HTMLAttributes<HTMLDivElement>;
```

标题文本容器，固定定位在快拍卡片顶部，白色文字 `z-10`。

### StoryOverlay

```ts
type StoryOverlayProps = HTMLAttributes<HTMLDivElement> & {
  side?: "top" | "bottom"; // 默认 "bottom"
}
```

渐变遮罩层，高 40px，`from-black/20 to-transparent`。`side="top"` 时从上到下渐变，`side="bottom"` 时从下到上渐变。

## Semantic DOM {#semantic-dom}

源码中使用的稳定语义槽位：

- `role="button"` — Story 卡片的交互语义
- `tabIndex={0}` — Story 卡片和 StoryVideo 的可聚焦性
- `data-slot="carousel"` — 底层 Carousel 容器
- `data-slot="carousel-content"` — 轮播内容区
- `data-slot="carousel-item"` — 单个轮播项

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 布局基于原生 Carousel 组件（Embla），`Stories` 仅覆盖默认 opts 和包裹 className。
- `Story` 卡片使用 `rounded-xl`、`bg-muted/40`、hover `scale-[1.02]` 和 `shadow-lg`。
- `StoryVideo` / `StoryImage` 使用 `absolute inset-0 object-cover` 填满卡片。
- `StoryOverlay` 根据 `side` prop 选择 `bg-gradient-to-b` 或 `bg-gradient-to-t`。
- 主题颜色使用 `bg-muted/40`、`text-white`、`ring-ring` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/stories/index.tsx`
- `src/components/core/blocks/stories.tsx`（barrel 导出）

直接依赖摘要：

- `@/components/core/avatar` — 作者头像
- `@/components/core/carousel` — 轮播容器（基于 Embla）
- `@/lib/utils` — `cn` 样式合并工具

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. `StoryVideo` 通过 `src` 的 URL hash（如 `#t=5`）指定初始播放位置，解析在 `useEffect` 中完成（正则 `/t=(\d+(?:\.\d+)?)/`）。
3. `StoryVideo` 的 hover 播放和焦点播放行为由 `onMouseOver`/`onMouseOut`/`onFocus`/`onBlur` 事件驱动。
4. `StoryImage` 使用原生 `<img>`（框架无关设计，注释标注）。
5. `Stories` 默认 opts 会在调用侧 `opts` 之后展开，调用侧可覆盖默认值。
6. 子组件应按 `Stories > StoriesContent > Story > (StoryImage|StoryVideo) + StoryOverlay + (StoryAuthor|StoryTitle)` 层级组合。
7. 如果需要新增能力，应先修改 `src/components/blocks/stories/index.tsx`，再更新本 llm 文档。

## 可访问性 {#accessibility}

- `Story` 卡片带有 `role="button"` 和 `tabIndex={0}`，支持键盘导航。
- `Story` 有 `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` 样式提供聚焦指示器。
- `StoryVideo` 带有 `tabIndex={0}` 并响应 `onFocus`/`onBlur` 事件。
- `StoryVideo` 使用 `muted` 属性防止自动播放被浏览器阻止。
- `StoryImage` 要求必填 `alt` 属性，确保图片有替代文本。
- `StoryAuthorImage` 基于 Avatar，通过 `name` / `fallback` 提供文本回退。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。其基于本地 Carousel primitive 和 Avatar 组件构建。

| 方向 | 本地决策 |
| --- | --- |
| 组件来源 | 项目本地复合，非 AntD 封装 |
| API 来源 | 以 `src/components/blocks/stories/index.tsx` 的 TypeScript 类型为准 |
| 轮播方案 | 基于本地 Carousel（Embla），非 AntD Carousel |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Stories 代码时，应遵守：

1. 从 `@/components/blocks/stories` 导入，按需引入所需子组件。
2. `StoryImage` 必须提供 `alt` 属性，这是必填项。
3. `StoryOverlay` 使用 `side` prop 控制渐变方向，默认为 `"bottom"`。
4. `StoryVideo` 可通过 `src` URL 的 `#t=秒数` 指定起始播放位置。
5. 完整快拍结构为：`Stories` > `StoriesContent` > `Story` > (视频/图片 + 遮罩 + 信息)。
6. 只使用本文件 API 摘要和源码中存在的 props。
7. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置。
8. 如果不确定某个 API 是否存在，先检查 `src/components/blocks/stories/index.tsx`，不要凭记忆生成。
