卡片堆叠 Deck。一组卡片堆叠交互组件，模拟 Tinder 风格的卡片滑动体验。卡片从堆顶逐张展示，用户可通过拖拽向左或向右滑走顶层卡片，下方卡片以缩放和偏移形成堆叠视觉深度。支持受控索引和自定义堆叠参数。

<h2 id="when-to-use">何时使用</h2>

适用于需要卡片式浏览、选择、筛选的场景。

常见场景：
- 推荐流浏览（候选人评价、内容推荐）。
- 浏览式决策界面（Tinder 式左右滑动抉择）。
- 受控步进的卡片展示向导。
- 用户通过拖拽表达"接受/拒绝"意图的交互。

不推荐：
- 简单的图片轮播 —— Deck 的堆叠视觉和拖拽复杂性属于过度设计，应使用 Carousel。
- 需要同时展示多张卡片全内容的场景 —— Deck 只展示顶层可交互卡片。
- 内容为大量不可交互数据的纯展示列表 —— 用分页或虚拟滚动更高效。

### 受控索引切换

通过 `currentIndex` 和 `onCurrentIndexChange` 实现外部控制的卡片步进。

```tsx
import { useState } from "react"
import { Deck, DeckCards, DeckItem } from "@/components/blocks/deck"
import { Button } from "@/components/core/button"

export default function Demo() {
  const [index, setIndex] = useState(0)

  return (
    <div className="space-y-4">
      <Deck className="w-full max-w-sm h-[400px]">
        <DeckCards currentIndex={index} onCurrentIndexChange={setIndex}>
          <DeckItem>Card 1</DeckItem>
          <DeckItem>Card 2</DeckItem>
          <DeckItem>Card 3</DeckItem>
        </DeckCards>
      </Deck>
      <div className="flex gap-2">
        <Button disabled={index <= 0} onClick={() => setIndex(index - 1)}>
          上一张
        </Button>
        <Button onClick={() => setIndex(index + 1)}>下一张</Button>
      </div>
    </div>
  )
}
```

### 自定义堆叠参数

调整堆叠视觉深度、滑动灵敏度和动画行为。

```tsx
import { Deck, DeckCards, DeckItem } from "@/components/blocks/deck"

export default function Demo() {
  return (
    <Deck className="w-full max-w-sm h-[400px]">
      <DeckCards
        stackSize={5}
        scale={0.08}
        perspective={800}
        threshold={100}
        onSwipe={(index, direction) => console.log(`Swiped card ${index} ${direction}`)}
        onSwipeEnd={(index, direction) => console.log(`Card ${index} ${direction} animation done`)}
      >
        {Array.from({ length: 8 }, (_, i) => (
          <DeckItem key={i}>
            <div className="flex flex-col items-center gap-4 p-6">
              <span className="text-4xl font-bold">{i + 1}</span>
              <p className="text-sm text-muted-foreground">拖动或点击按钮切换</p>
            </div>
          </DeckItem>
        ))}
      </DeckCards>
    </Deck>
  )
}
```

<h2 id="api">API</h2>

### Deck

顶层容器，提供相对定位和层级隔离。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

### DeckCards

卡片堆叠区域，管理滑动逻辑和视觉堆叠。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| currentIndex | 当前展示卡片的受控索引 | `number` | - |
| defaultCurrentIndex | 当前展示卡片的非受控索引 | `number` | `0` |
| onCurrentIndexChange | 索引变化回调 | `(index: number) => void` | - |
| onSwipe | 滑动开始回调 | `(index: number, direction: "left" \| "right") => void` | - |
| onSwipeEnd | 滑动结束回调 | `(index: number, direction: "left" \| "right") => void` | - |
| threshold | 滑动触发阈值（px） | `number` | `150` |
| stackSize | 同时可见的卡片数量 | `number` | `3` |
| perspective | CSS 3D 透视距离 | `number` | `1000` |
| scale | 每层卡片的递减缩放比例 | `number` | `0.05` |
| animateOnIndexChange | 外部索引变化时是否播放入场动画 | `boolean` | `true` |
| indexChangeDirection | 外部索引变化时的动画方向 | `"left"` \| `"right"` | `"left"` |
| className | 附加样式类 | `string` | - |

### DeckItem

单张卡片容器，提供统一的圆角、边框、阴影样式。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

默认样式 `rounded-lg border bg-card text-card-foreground shadow-lg`。

### DeckEmpty

所有卡片滑走后显示的占位状态。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 自定义空状态内容 | `ReactNode` | `"No more cards"` |
| className | 附加样式类 | `string` | - |

默认样式 `rounded-lg border border-dashed text-muted-foreground`。

<h2 id="faq">常见问题</h2>

### 如何获取滑动方向和卡片索引？

通过 `onSwipe` 和 `onSwipeEnd` 回调：

```tsx
<DeckCards
  onSwipe={(index, direction) => console.log(`Swiped card ${index} ${direction}`)}
  onSwipeEnd={(index, direction) => console.log(`Animation done for card ${index}`)}
>
```

`onSwipe` 在滑动触发时调用，`onSwipeEnd` 在退出动画完成后调用。

### 如何调整交互灵敏度？

通过 `threshold` 属性调整触发滑动的拖拽距离（默认 150px）：

```tsx
<DeckCards threshold={80}> {/* 更灵敏 */}
<DeckCards threshold={200}> {/* 需要拖得更远 */}
```

### 如何调整堆叠视觉效果？

通过 `stackSize`（可见层数）、`scale`（层间缩放差）、`perspective`（透视距离）控制：

```tsx
<DeckCards stackSize={5} scale={0.08} perspective={800}>
```

### 所有卡片滑走后如何重置？

使用受控模式，将 `currentIndex` 重置为 0：

```tsx
const [index, setIndex] = useState(0)
const handleReset = () => setIndex(0)
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/deck` 导入 Deck、DeckCards、DeckItem、DeckEmpty。
2. 卡片子元素必须作为 DeckCards 的直接子节点传入，内容使用 DeckItem 包裹。
3. 如需受控索引，使用 currentIndex + onCurrentIndexChange，不要自行管理 state 后传递给不受控 API。
4. DeckEmpty 放在 Deck 内部、DeckCards 之后，所有卡片消耗后自动显示。
5. 不要将 Deck 用于简单的图片轮播场景，考虑使用 Carousel 替代。
6. 不要引入 Ant Design 相关依赖。
