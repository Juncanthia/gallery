AvatarTooltipGroup 头像提示组，将多个 Avatar 头像以重叠方式排列，悬停时弹出自定义提示框。基于 motion/react-client 实现悬停浮起动效和提示框动画过渡，支持控制弹出方向、偏移量和延迟时间。

<h2 id="when-to-use">何时使用</h2>

当需要在紧凑空间中展示一组用户头像，并希望悬停时显示详细信息时使用。

常见场景：
- 协作者列表悬停显示用户姓名、角色等详细信息。
- 群组成员头像预览，hover 显示完整信息。
- 需要提示框内容根据数据动态变化且有平滑过渡动画的场景。
- 头像组的弹出方向、动画曲线需要自定义控制的场景。

不推荐：
- 仅需静态头像排列的场景，使用 `AvatarStack` 或普通横向 Avatar 列表即可。
- 在提示框内容中放置 Button、Input 等交互控件——提示框为只读信息展示。
- 头像数量过多（超过 10 个）时，悬停提示框交互密度过高，用户体验差。

<h2 id="examples">代码演示</h2>

### 基础用法

AvatarGroup 包裹 Avatar，每个 Avatar 内部放置 AvatarGroupTooltip 注册悬停提示内容。

<ComponentShowcase id="blocks/avatar-tooltip-group" path="blocks/avatar-tooltip-group/basic" />

```tsx
import { AvatarGroup, AvatarGroupTooltip } from "@/components/blocks/avatar-tooltip-group"
import { Avatar, AvatarImage, AvatarFallback } from "@/components/core/avatar"

export default function Basic() {
  return (
    <AvatarGroup>
      <Avatar>
        <AvatarImage src="https://github.com/user1.png" alt="A" />
        <AvatarFallback>A</AvatarFallback>
        <AvatarGroupTooltip>User A</AvatarGroupTooltip>
      </Avatar>
      <Avatar>
        <AvatarImage src="https://github.com/user2.png" alt="B" />
        <AvatarFallback>B</AvatarFallback>
        <AvatarGroupTooltip>User B</AvatarGroupTooltip>
      </Avatar>
      <Avatar>
        <AvatarFallback>C</AvatarFallback>
        <AvatarGroupTooltip>User C</AvatarGroupTooltip>
      </Avatar>
    </AvatarGroup>
  )
}
```

```tsx
import { AvatarGroup, AvatarGroupTooltip } from "@/components/blocks/avatar-tooltip-group"

```

### 自定义提示框动画

通过 `layout` 控制提示框内容变化时的 motion 布局动画模式。

```tsx
import { AvatarGroup, AvatarGroupTooltip } from "@/components/blocks/avatar-tooltip-group"
import { Avatar, AvatarFallback } from "@/components/core/avatar"

export default function CustomLayout() {
  return (
    <AvatarGroup>
      <Avatar>
        <AvatarFallback>张</AvatarFallback>
        <AvatarGroupTooltip layout="position">仅跟踪位置变化</AvatarGroupTooltip>
      </Avatar>
      <Avatar>
        <AvatarFallback>李</AvatarFallback>
        <AvatarGroupTooltip layout={false}>禁用布局动画</AvatarGroupTooltip>
      </Avatar>
    </AvatarGroup>
  )
}
```

### 禁用重叠反转

`invertOverlap={false}` 使第一个头像在最上层。

```tsx
import { AvatarGroup, AvatarGroupTooltip } from "@/components/blocks/avatar-tooltip-group"
import { Avatar, AvatarFallback } from "@/components/core/avatar"

export default function NoInvert() {
  return (
    <AvatarGroup invertOverlap={false}>
      <Avatar>
        <AvatarFallback>1</AvatarFallback>
        <AvatarGroupTooltip>第一人</AvatarGroupTooltip>
      </Avatar>
      <Avatar>
        <AvatarFallback>2</AvatarFallback>
        <AvatarGroupTooltip>第二人</AvatarGroupTooltip>
      </Avatar>
    </AvatarGroup>
  )
}
```

<h2 id="api">API</h2>

### AvatarGroup

| 属性 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| children | 子级 Avatar 元素列表 | `React.ReactElement[]` | - |
| className | 外层容器自定义样式类 | `string` | `'h-12 -space-x-3'` |
| invertOverlap | 是否反转层级——`true` 时最后一个头像在最上层 | `boolean` | `true` |
| translate | 悬停时头像的垂直位移量 | `string` \| `number` | `'-30%'` |
| transition | 头像悬停位移动画的过渡配置 | `Transition` | `{ type: 'spring', stiffness: 300, damping: 17 }` |
| openDelay | 提示框显示延迟（毫秒） | `number` | `0` |
| closeDelay | 提示框隐藏延迟（毫秒） | `number` | `0` |
| side | 提示框相对于头像的弹出方向 | `'top'` \| `'bottom'` \| `'left'` \| `'right'` | `'top'` |
| sideOffset | 提示框与头像之间的间距（像素） | `number` | `25` |
| align | 提示框相对于头像的对齐方式 | `'start'` \| `'center'` \| `'end'` | `'center'` |
| alignOffset | 提示框对齐偏移量（像素） | `number` | `0` |
| tooltipTransition | 提示框显隐动画的过渡配置 | `Transition` | `{ type: 'spring', stiffness: 300, damping: 35 }` |
| id | 提示框提供者标识 | `string` | - |

### AvatarGroupTooltip

| 属性 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| children | 提示框内显示的内容，支持文本、React 元素或复杂布局 | `React.ReactNode` | - |
| className | 提示框自定义样式类 | `string` | - |
| layout | 内容变化时 motion 布局动画模式 | `boolean` \| `'position'` \| `'size'` \| `'preserve-aspect'` | `'preserve-aspect'` |

<h2 id="faq">常见问题</h2>

### Avatar 必须以什么方式使用？

Avatar 必须以 composed mode 使用——**不传 `src` prop 给 Avatar**，改用 `AvatarImage` 子组件。这是必需的，因为 AvatarGroupTooltip 是一个"注册型"组件：它不产生可见 DOM，而是通过 `useEffect` 将自身内容注册到父级 Tooltip 上下文。只有 composed mode 下 Avatar 才会渲染其全部 children，使注册 useEffect 得以执行。

```tsx
// 正确：composed mode
<Avatar>
  <AvatarImage src="..." />
  <AvatarFallback>...</AvatarFallback>
  <AvatarGroupTooltip>...</AvatarGroupTooltip>
</Avatar>

// 错误：传 src prop
<Avatar src="...">
  <AvatarGroupTooltip>...</AvatarGroupTooltip>
</Avatar>
```

### 如何控制提示框的弹出延迟？

通过 `openDelay` 和 `closeDelay`（单位毫秒）控制：

```tsx
<AvatarGroup openDelay={200} closeDelay={150}>
  {/* ... */}
</AvatarGroup>
```

### 提示框内容支持什么格式？

`AvatarGroupTooltip` 的 `children` 即为提示框显示内容，支持纯文本、React 元素或复杂布局。与 Ant Design Tooltip 不同，提示内容通过 `children` 传入而非 `title` 属性。

### 如何禁用提示框的布局动画？

设置 `layout={false}`：

```tsx
<AvatarGroupTooltip layout={false}>无动画的提示内容</AvatarGroupTooltip>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/avatar-tooltip-group` 导入 `AvatarGroup` 和 `AvatarGroupTooltip`。
2. AvatarGroup 的子级应为 Avatar 元素，每个 Avatar 内部可选择性放置 AvatarGroupTooltip 作为提示内容注册。
3. Avatar 必须以 composed mode 使用——不传 `src` prop，改用 `AvatarImage` 子组件，以确保 AvatarGroupTooltip 的 useEffect 能正常执行。
4. AvatarGroupTooltip 的 `children` 即为提示框显示内容，可以是纯文本、React 元素或复杂布局。该组件不存在 `content` 或 `title` prop。
5. 通过 `side`、`sideOffset`、`align`、`alignOffset` 精确控制提示框弹出位置。
6. 不要在提示框内容中放置 Button、Input 等交互控件——提示框为只读信息展示。
7. 该组件为复合组件，并非 Ant Design 或 shadcn/ui 的一等公民组件。
8. 不要引入 Ant Design 相关依赖。
