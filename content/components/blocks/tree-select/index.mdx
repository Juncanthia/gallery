树选择 TreeSelect。从树状层级数据中选择一个或多个值，支持单选、多选、复选框模式以及搜索过滤。

<h2 id="when-to-use">何时使用</h2>

当需要从具有层级结构（父子关系）的数据中进行选择时使用。TreeSelect 将树形数据以可展开/收起的方式呈现，比 Select 更适合具有分类、层级、组织架构等场景。

常见场景：

- 组织架构选择（部门 > 小组 > 成员）。
- 分类目录选择（父分类 > 子分类）。
- 地区选择（国家 > 省 > 市）。
- 任何具有父子关系的数据项选择。

不推荐：

- 扁平列表选择 —— 使用 Select 或 Combobox 即可，无需树形展开。
- 数据量极大（数千节点）且不提供 search —— 大量递归渲染会造成性能问题，应考虑虚拟滚动或懒加载。
- 手动拼装 `<details>` / `<summary>` 模拟树选择 —— 丢失了统一的选中状态管理和可访问性。

### 基础用法

<ComponentShowcase id="tree-select" path="tree-select/basic" />

```tsx
import { TreeSelect, type TreeSelectOption } from "@/components/core/tree-select"

const treeData: TreeSelectOption[] = [
  {
    value: "1",
    label: "根节点",
    children: [
      { value: "1-1", label: "子节点 1" },
      { value: "1-2", label: "子节点 2" },
    ],
  },
  { value: "2", label: "节点 2" },
]

export default function Demo() {
  return (
    <TreeSelect
      options={treeData}
      placeholder="请选择"
      className="w-56"
    />
  )
}
```

### 多选模式

设置 `multiple` 启用多选，选中项以 Badge 形式显示在触发器上。

```tsx
import { TreeSelect, type TreeSelectOption } from "@/components/core/tree-select"

export default function Demo() {
  return (
    <TreeSelect
      options={treeData}
      multiple
      placeholder="多选节点"
      className="w-56"
    />
  )
}
```

### 复选框模式

`treeCheckable` 在每个节点前展示 Checkbox，选中后不会自动关闭下拉。

```tsx
import { TreeSelect, type TreeSelectOption } from "@/components/core/tree-select"

export default function Demo() {
  return (
    <TreeSelect
      options={treeData}
      treeCheckable
      showSearch
      placeholder="搜索并勾选"
      className="w-56"
    />
  )
}
```

### 搜索过滤

开启 `showSearch` 后弹出搜索框，可通过 `filterTreeNode` 自定义过滤逻辑。

```tsx
import { TreeSelect, type TreeSelectOption } from "@/components/core/tree-select"

export default function Demo() {
  return (
    <TreeSelect
      options={treeData}
      showSearch
      placeholder="搜索节点"
      className="w-56"
    />
  )
}
```

<h2 id="api">API</h2>

### TreeSelect

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| options | 树形选项数据 | `TreeSelectOption[]` | - |
| value | 当前选中值（受控） | `string` \| `string[]` | - |
| defaultValue | 默认选中值 | `string` \| `string[]` | `[]` |
| onChange | 值变化回调 | `(value: string \| string[]) => void` | - |
| multiple | 是否支持多选 | `boolean` | `false` |
| treeCheckable | 是否在节点前显示复选框 | `boolean` | `false` |
| placeholder | 未选择时的占位文本 | `string` | `"Select..."` |
| showSearch | 是否显示搜索输入框 | `boolean` | `false` |
| filterTreeNode | 自定义搜索过滤函数 | `(inputValue: string, node: TreeSelectOption) => boolean` | - |
| allowClear | 是否允许一键清空 | `boolean` | `true` |
| disabled | 是否禁用 | `boolean` | `false` |
| className | 自定义容器样式类名 | `string` | - |
| triggerClassName | 自定义触发器样式类名 | `string` | - |
| treeDefaultExpandAll | 是否默认展开全部节点 | `boolean` | `false` |

### TreeSelectOption

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 节点唯一标识 | `string` | - |
| label | 节点显示文本 | `string` | - |
| children | 子节点数组 | `TreeSelectOption[]` | - |
| disabled | 是否禁用该节点 | `boolean` | - |
| checkable | 节点是否可勾选 | `boolean` | - |

<h2 id="faq">常见问题</h2>

### multiple 和 treeCheckable 有什么区别？

`multiple` 启用后，点击任意节点即加入选中列表，以 Badge 展示在触发器内。`treeCheckable` 在每个节点前显示 Checkbox，支持勾选/取消，选中后面板不自动关闭。两者同时使用时行为一致（均为多选模式）。

### 如何让所有子节点默认展开？

设置 `treeDefaultExpandAll={true}`，所有节点在首次打开下拉时全部展开。

### 如何在搜索时只匹配 label 而非匹配整个节点数据结构？

默认的 `filterTreeNode` 为 `undefined` 时，组件内部使用 `node.label.toLowerCase().includes(inputValue.toLowerCase())` 进行简单匹配。可通过 `filterTreeNode` 自定义，例如只匹配 `value` 字段或按正则匹配。

### 选中后如何获取完整的选项对象（而不只是 value）？

组件 API 只返回 `value: string | string[]`。若需要选中选项的完整信息，在业务侧根据 `value` 从原始的 `options` 数据中查找即可，组件内部不冗余返回对象。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/tree-select` 导入 `TreeSelect`，不要从 `antd` 导入。
2. 树数据通过 `options` prop 传入（类型为 `TreeSelectOption[]`），不要使用 AntD 的 `treeData` 命名。
3. 多选用 `multiple`，复选框用 `treeCheckable`，两者语义不同。
4. 搜索功能通过 `showSearch` 开启，自定义过滤用 `filterTreeNode`。
5. 不要生成 Ant Design 专属的 `classNames`、`styles`、`configprovider`、`treeNodeLabelProp` 等 props。
6. `TreeSelectOption` 的 key 是 `value` 和 `label`，不是 AntD 的 `key` 和 `title`。
