# VideoPlayer 视频播放器

基于 Media Chrome（`media-chrome`）封装的视频播放器复合组件。使用 Web 标准 `<video>` 元素，通过 `MediaController` 提供 Material Design 风格的播放控制栏、时间显示、音量调节和快进快退按钮。

组件通过 CSS 自定义属性（`--media-*`）对接项目主题变量，自动适配亮色/暗色模式。

## 何时使用

当需要在页面中嵌入可播放视频时使用。

常见场景：

- 视频教程、产品演示、功能录屏展示。
- 媒体库中的视频预览与播放。
- 需要统一风格播放控制栏的任意视频播放场景。

不推荐：

- 使用直播流、字幕、多音轨等高级媒体能力——Media Chrome 原生支持这些，但本地封装只覆盖了基础控制子组件。
- 嵌入第三方视频平台（YouTube、Bilibili、Vimeo 等）——应使用对应平台的嵌入代码或 iframe。
- 直接使用 HTML5 原生 `<video controls>`——绕过本组件会导致播放控制栏风格与项目设计系统不一致。

## 代码演示

### 基础用法

`VideoPlayer` 是 `MediaController` 的包装组件，内部需组合控制栏子组件方可正常工作。

```tsx
import {
  VideoPlayer,
  VideoPlayerContent,
  VideoPlayerControlBar,
  VideoPlayerPlayButton,
  VideoPlayerSeekBackwardButton,
  VideoPlayerSeekForwardButton,
  VideoPlayerTimeDisplay,
  VideoPlayerTimeRange,
  VideoPlayerMuteButton,
  VideoPlayerVolumeRange,
} from "@/components/blocks/video-player"

export default function Demo() {
  return (
    <VideoPlayer className="w-full max-w-2xl">
      <VideoPlayerContent
        slot="media"
        src="https://example.com/video.mp4"
        crossOrigin=""
      />
      <VideoPlayerControlBar>
        <VideoPlayerPlayButton />
        <VideoPlayerSeekBackwardButton />
        <VideoPlayerSeekForwardButton />
        <VideoPlayerTimeDisplay />
        <VideoPlayerTimeRange />
        <VideoPlayerMuteButton />
        <VideoPlayerVolumeRange />
      </VideoPlayerControlBar>
    </VideoPlayer>
  )
}
```

## API

### VideoPlayer

包装 `MediaController`，注入项目主题 CSS 变量。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| style | 附加内联样式，与内置 CSS 变量对象合并 | `CSSProperties` | - |
| ...props | 其他 `MediaController` 属性全部透传 | `ComponentProps<typeof MediaController>` | - |

内置 CSS 变量映射：

| 变量 | 映射值 |
| --- | --- |
| `--media-primary-color` | `var(--primary)` |
| `--media-secondary-color` | `var(--background)` |
| `--media-text-color` | `var(--foreground)` |
| `--media-background-color` | `var(--background)` |
| `--media-control-hover-background` | `var(--accent)` |
| `--media-font-family` | `var(--font-sans)` |
| `--media-live-button-icon-color` | `var(--muted-foreground)` |
| `--media-live-button-indicator-color` | `var(--destructive)` |
| `--media-range-track-background` | `var(--border)` |

### VideoPlayerContent

原生 `<video>` 元素包装。`VideoPlayerContentProps` 即 `ComponentProps<"video">`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类，默认含 `mt-0 mb-0` | `string` | - |
| slot | 必须设为 `"media"` 以关联 MediaController | `string` | - |
| ...props | 所有 `<video>` 原生属性透传 | `ComponentProps<"video">` | - |

### VideoPlayerControlBar

控制栏容器，包裹所有控制按钮。`VideoPlayerControlBarProps` 即 `ComponentProps<typeof MediaControlBar>`。

所有属性透传至底层 `MediaControlBar`，无额外默认 className。

### 控制子组件

以下子组件均接受 `className`（默认含 `p-2.5`）并将所有其他属性透传至对应 Media Chrome 元素。

| 组件 | 对应元素 | 说明 |
| --- | --- | --- |
| `VideoPlayerPlayButton` | `MediaPlayButton` | 播放/暂停切换按钮 |
| `VideoPlayerSeekBackwardButton` | `MediaSeekBackwardButton` | 后退指定秒数 |
| `VideoPlayerSeekForwardButton` | `MediaSeekForwardButton` | 快进指定秒数 |
| `VideoPlayerTimeDisplay` | `MediaTimeDisplay` | 当前时间与总时长显示 |
| `VideoPlayerTimeRange` | `MediaTimeRange` | 播放进度条 |
| `VideoPlayerMuteButton` | `MediaMuteButton` | 静音/取消静音切换 |
| `VideoPlayerVolumeRange` | `MediaVolumeRange` | 音量滑块 |

每个子组件的 Props 类型均为 `ComponentProps<typeof 对应元素>`，可通过 `className` 覆盖或追加样式类。

## 样式模型

- 主题对接：`VideoPlayer` 通过 `style` prop 注入 9 个 `--media-*` CSS 自定义属性，将 media-chrome 的颜色、字体、控件背景等映射至项目 Tailwind 主题 token。
- 子组件默认间距：除 `VideoPlayerControlBar` 外，所有控制子组件默认追加 `p-2.5`（10px 内边距），`VideoPlayerContent` 默认追加 `mt-0 mb-0`。
- 暗色模式：CSS 变量值引用的是 `var(--primary)` 等主题 token，暗色模式切换时自动跟随。
- 自定义：可通过 VideoPlayer 的 `style` 覆盖任意 `--media-*` 变量；也可通过子组件的 `className` 追加 Tailwind 工具类。

## 依赖与实现说明

核心文件：

- `src/components/blocks/video-player/index.tsx`

技术依赖：

- `react` — 组件框架
- `tailwindcss` — 样式系统
- `media-chrome`（`^4.19.2`）— Web Components 媒体控制库，提供 `<media-controller>`、`<media-control-bar>` 等 Custom Elements 及其 React 绑定
- `@/kit/utils` 中的 `cn` — 类名合并工具

实现要点：

1. `VideoPlayer` 作为 `MediaController`（即 `<media-controller>` 的 React 封装）的顶层包装，负责注入主题变量。
2. 所有控制子组件为薄包装层，不额外添加业务逻辑，仅透传 props 并注入默认 Tailwind 类名。
3. `VideoPlayerContent` 的 `slot="media"` 是 Media Chrome 的强制约定——它告诉 `MediaController` 哪个元素是媒体源。
4. Media Chrome 底层使用 Web Components（Custom Elements v1），与 React 通过官方 `media-chrome/react` 绑定交互。注意 React 合成事件与原生 DOM 事件的行为差异。
5. 本组件为复合组件（Composite Block），不包含独立的业务状态管理，所有媒体状态（播放进度、音量等）由 MediaController 内部管理。

## 可访问性

- Media Chrome Web Components 内置键盘操作：空格播放/暂停，左右方向键快进/后退，上下方向键调节音量。
- 控制按钮通过 Shadow DOM 提供原生可访问名称（accessible name）。
- `<video>` 元素原生支持 `<track>` 字幕轨道；将 `<track>` 作为 `VideoPlayerContent` 的子元素即可添加字幕。
- 主题颜色系统确保控件在亮色/暗色模式下均有足够对比度。

## 与 Ant Design 的取舍

该组件为项目本地复合组件，基于 Media Chrome 构建，无 Ant Design 直接对应。Ant Design 未内置视频播放器组件。

| 方向 | 本地决策 |
| --- | --- |
| 技术基础 | Media Chrome（Web Components + React 绑定） |
| API 定义 | 以 `src/components/blocks/video-player/index.tsx` 中的类型为准 |
| 样式体系 | Tailwind CSS + CSS 自定义属性 |
| 浏览器兼容 | 依赖 Custom Elements v1（所有现代浏览器均支持） |
| 与 AntD 关系 | 不兼容 AntD（AntD 无此组件），不引入 `antd` 运行时依赖 |

## LLM 使用规则

1. 从 `@/components/blocks/video-player` 导入所有需要的子组件，按需导入。
2. `VideoPlayerContent` 必须设置 `slot="media"`，否则 MediaController 无法识别媒体源。
3. 控制按钮（PlayButton、SeekBackwardButton 等）必须放置在 `VideoPlayerControlBar` 内部，否则不会出现在控制栏中。
4. 不要使用 HTML5 原生 `<video controls>` 替代本组件——会导致播放控制栏风格与项目设计系统不一致。
5. 自定义样式优先通过 `style` prop 覆盖 `--media-*` CSS 变量，其次通过子组件 `className` 追加 Tailwind 工具类。
6. 需要字幕时，将 `<track>` 元素作为 `VideoPlayerContent` 的子元素。
7. 该组件封装的是 Web Components，React 合成事件行为与原生 DOM 事件不完全一致。不要依赖 React 事件系统处理 media-chrome 的内部事件。
8. 本组件为复合组件（Blocks），无 Ant Design 对应物，不要尝试从 `antd` 导入视频播放相关组件。
