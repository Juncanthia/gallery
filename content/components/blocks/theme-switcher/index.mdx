主题切换 ThemeSwitcher。三态主题切换复合组件，支持 system/light/dark 三种模式，通过 motion 动效实现激活态平滑切换指示。

<h2 id="when-to-use">何时使用</h2>

当需要在界面中放置主题/颜色模式切换器时使用。

常见场景：
- 导航栏、设置页或页脚的主题切换入口
- 需要三态切换（system/light/dark）而非简单 toggle 的场景
- 需要受控模式集成 next-themes 或其他主题管理方案的应用

不推荐：
- 仅需二态切换（亮/暗）——使用简单的 toggle 按钮即可
- 需要 4 种以上主题选项——本组件仅支持三种模式
- 需要自定义图标或主题标签——图标和 aria-label 内置，不可定制

### 基础用法

```tsx
import { ThemeSwitcher } from "@/components/blocks/theme-switcher"

export default function Demo() {
```

### 受控模式

```tsx
import { useState } from "react"
import { ThemeSwitcher } from "@/components/blocks/theme-switcher"

export default function Demo() {
  const [theme, setTheme] = useState<"light" | "dark" | "system">("system")

  return (
    <ThemeSwitcher
      value={theme}
      onChange={(t) => {
        setTheme(t)
        // 在此集成主题切换逻辑，如 next-themes 的 setTheme
      }}
    />
  )
}
```

> `ThemeSwitcher` 本身不包含主题切换逻辑，仅管理 UI 状态。实际主题应用（DOM class 变更、CSS 变量切换、localStorage 存储等）需在 `onChange` 回调中由调用侧实现。

<h2 id="api">API</h2>

### ThemeSwitcher

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 受控模式下的当前主题 | `"light"` \| `"dark"` \| `"system"` | - |
| onChange | 主题变化回调，用于执行实际的主题切换逻辑 | `(theme: "light"` \| `"dark"` \| `"system") => void` | - |
| defaultValue | 非受控模式下的默认主题 | `"light"` \| `"dark"` \| `"system"` | `"system"` |
| className | 附加样式类名，与默认容器样式合并 | `string` | - |

组件使用 `useControllableState` 实现受控/非受控双模式：传入 `value` 时为受控，否则为非受控。

内置三个主题选项：

| 主题 | 图标 | aria-label |
| --- | --- | --- |
| `system` | Monitor (lucide-react) | "System theme" |
| `light` | Sun (lucide-react) | "Light theme" |
| `dark` | Moon (lucide-react) | "Dark theme" |

<h2 id="faq">常见问题</h2>

### ThemeSwitcher 切换后为什么页面主题没有变化？
`ThemeSwitcher` 仅管理 UI 选择状态，不执行实际的主题切换。需要在 `onChange` 回调中自行实现 DOM class 变更、CSS 变量切换或 localStorage 存储等逻辑，通常配合 `next-themes` 使用。

### 为什么 SSR 阶段组件不可见？
组件内置了 `mounted` 状态保护，在客户端挂载前返回 `null`，避免 hydration mismatch。这是预期行为，客户端渲染后组件会正常显示。

### 能否自定义主题图标或增加第四种主题？
不支持。三个主题及其对应图标/标签内置于组件内部，不可通过 props 扩展或替换。如需更多选项，考虑基于源码自定义。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/theme-switcher` 导入。
2. 受控模式使用 `value` + `onChange`；非受控模式使用 `defaultValue`（默认 `"system"`）。
3. `value`、`onChange` 参数和 `defaultValue` 的值类型为 `"light" | "dark" | "system"`，不要使用其他值。
4. `ThemeSwitcher` 仅管理 UI 选择状态，实际主题切换逻辑需在 `onChange` 回调中由调用侧实现。
5. 不要引入 Ant Design 相关依赖。
