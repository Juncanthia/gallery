ContributionGraph 贡献图，GitHub 风格的贡献热力图复合组件。基于 SVG 渲染，支持自定义颜色级别、多语言标签、render props 模式和灵活的日期范围。

<h2 id="when-to-use">何时使用</h2>

当需要在数据看板、个人主页、统计分析等场景中展示时间序列活跃度数据，以单元格颜色深浅直观表达贡献量级时使用。

常见场景：

- 用户活跃度热力图（如 GitHub-风格的贡献图）。
- 项目提交、任务完成频率的时间分布可视化。
- 日志量、访问量等时序数据的聚合展示。
- 任何需要按日期展示量级分布的网格图表。

不推荐：

- 数据量极大（数年以上按小时粒度），组件按天聚合、周粒度展示，不适合超细粒度。
- 需要精确数值读取，热力图适合相对比较，不适合精确读数。
- 非时间维度的数据分布，应使用柱状图或折线图。

### 基础用法

<ComponentShowcase id="blocks/contribution-graph" path="blocks/contribution-graph/basic" />

```tsx
import {
  ContributionGraph,
  ContributionGraphCalendar,
  ContributionGraphBlock,
  ContributionGraphFooter,
  ContributionGraphTotalCount,
  ContributionGraphLegend,
} from "@/components/blocks/contribution-graph"

const data = [
  { date: "2025-01-01", count: 3, level: 2 },
  { date: "2025-01-02", count: 1, level: 1 },
  { date: "2025-01-05", count: 8, level: 4 },
  { date: "2025-02-14", count: 5, level: 3 },
]

export default function Demo() {
  return (
    <ContributionGraph data={data}>
      <ContributionGraphCalendar>
        {({ activity, dayIndex, weekIndex }) => (
          <ContributionGraphBlock
            key={`${weekIndex}-${dayIndex}`}
            activity={activity}
            dayIndex={dayIndex}
            weekIndex={weekIndex}
          />
        )}
      </ContributionGraphCalendar>
      <ContributionGraphFooter>
        <ContributionGraphTotalCount />
        <ContributionGraphLegend />
      </ContributionGraphFooter>
    </ContributionGraph>
  )
}
```

<h2 id="api">API</h2>

### Activity

```ts
type Activity = {
  date: string  // ISO 日期字符串，如 "2025-01-01"
  count: number // 当日贡献计数
  level: number // 颜色级别 (0 ~ maxLevel)
}
```

### ContributionGraph

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 活动数据数组 | `Activity[]` | 必填 |
| blockMargin | 单元格间距（px） | `number` | `4` |
| blockRadius | 单元格圆角半径（px） | `number` | `2` |
| blockSize | 单元格尺寸（px） | `number` | `12` |
| fontSize | 标签字号（px） | `number` | `14` |
| labels | 多语言标签配置，合并到默认英文标签 | `Labels` | 内置英文默认值 |
| maxLevel | 最大颜色级别（0 ~ maxLevel 共 maxLevel+1 级） | `number` | `4` |
| totalCount | 手动指定总计数，不传时自动从 data 求和 | `number` | 自动计算 |
| weekStart | 一周起始日（0=周日 ~ 6=周六） | `Day` (0-6) | `0`（周日） |
| style | 容器内联样式 | `CSSProperties` | - |
| className | 附加样式类 | `string` | - |
| children | 子组件内容 | `ReactNode` | 必填 |

继承 `HTMLAttributes<HTMLDivElement>`。当 `data` 为空数组时返回 `null`。自动通过 `date-fns` 计算周分组、月份标签和总计数。通过 `ContributionGraphContext` 共享所有数据。

### Labels

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| months | 12 个月份标签数组 | `string[]` | `["Jan", "Feb", ...]` |
| weekdays | 7 天星期标签数组 | `string[]` | `["Sun", "Mon", ...]` |
| totalCount | 总计数模板，支持 `{{count}}` 和 `{{year}}` | `string` | `"{{count}} activities in {{year}}"` |
| legend | 图例左右标签 | `{ less: string; more: string }` | `{ less: "Less", more: "More" }` |

### ContributionGraphCalendar

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| hideMonthLabels | 是否隐藏顶部月份标签 | `boolean` | `false` |
| className | 附加样式类 | `string` | - |
| children | render props，为每个活动渲染内容 | `(props: { activity: Activity; dayIndex: number; weekIndex: number }) => ReactNode` | 必填 |

继承 `Omit<HTMLAttributes<HTMLDivElement>, 'children'>`。内部渲染 SVG 元素，通过 `viewBox` 控制坐标系。`dayIndex` 为 0-6（周日到周六），`weekIndex` 为从左到右的周序号。

### ContributionGraphBlock

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| activity | 对应日期的活动数据 | `Activity` | 必填 |
| dayIndex | 星期序号（0-6） | `number` | 必填 |
| weekIndex | 周序号 | `number` | 必填 |
| className | 附加样式类 | `string` | - |

继承 `HTMLAttributes<SVGRectElement>`。渲染为 SVG `<rect>` 元素，通过 `data-level` 属性匹配颜色。颜色级别映射：
- level 0: `fill-muted`
- level 1: `fill-muted-foreground/20`
- level 2: `fill-muted-foreground/40`
- level 3: `fill-muted-foreground/60`
- level 4: `fill-muted-foreground/80`

### ContributionGraphFooter

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。底部区域容器，flex 横排布局。

### ContributionGraphTotalCount

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 自定义渲染，接收 `totalCount` 和 `year` | `(props: { totalCount: number; year: number }) => ReactNode` | 按 labels.totalCount 渲染 |
| className | 附加样式类 | `string` | - |

继承 `Omit<HTMLAttributes<HTMLDivElement>, 'children'>`。默认使用 `labels.totalCount` 模板，支持 `{{count}}` 和 `{{year}}` 占位符替换。

### ContributionGraphLegend

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 自定义渲染，接收 `level` | `(props: { level: number }) => ReactNode` | 默认 SVG 色块 |
| className | 附加样式类 | `string` | - |

继承 `Omit<HTMLAttributes<HTMLDivElement>, 'children'>`。渲染从 level 0 到 maxLevel 共 maxLevel+1 个色块，左右分别显示 "Less"/"More" 标签。

<h2 id="faq">常见问题</h2>

### 数据缺失的日期如何处理？

组件内部 `fillHoles` 函数自动填充数据中缺失的日期（level=0, count=0），确保日历网格连续不间断。无需手动补全 day 数据。

### 如何自定义颜色级别？

默认支持 5 个颜色级别（level 0-4），通过 `maxLevel` prop 调整级别数量。需要在 Tailwind 配置中补充对应 `data-[level]` 样式，或在 `ContributionGraphBlock` 的 `className` 中覆盖颜色映射。

### 月份标签太密集怎么办？

`getMonthLabels` 函数内置智能过滤逻辑：相邻月份标签间距至少 3 周才会同时显示，首尾月份标签也经过最小周数判断，避免标签过密。

### level 值超出范围会怎样？

组件在 `ContributionGraphBlock` 渲染时对 `activity.level` 做运行时校验（必须在 `0 ~ maxLevel` 范围内），超范围会抛出 `RangeError`。传入数据前应确保 level 值合法。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/contribution-graph` 导入所有需要的子组件。
2. `ContributionGraph` 的 `data` prop 为必填，类型为 `Activity[]`，每项包含 `date`、`count`、`level`。
3. `ContributionGraphCalendar` 使用 render props 模式，回调参数为 `{ activity, dayIndex, weekIndex }`。
4. `ContributionGraphBlock` 必须传入 `activity`、`dayIndex`、`weekIndex` 三个 prop。
5. `level` 值必须在 `0` 到 `maxLevel`（默认 4）之间，超出会抛 RangeError。
6. 底部统计和图例必须包裹在 `ContributionGraphFooter` 中。
7. 需要自定义颜色级别时调整 `maxLevel` 并补充 Tailwind 样式。
8. 不要引入 Ant Design 相关依赖。
