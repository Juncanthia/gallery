# ContributionGraph 贡献图

ContributionGraph 是一个 GitHub 风格的贡献热力图复合组件，由 ContributionGraph 容器、ContributionGraphCalendar 日历渲染器、ContributionGraphBlock 单元格、ContributionGraphFooter 底部区域、ContributionGraphTotalCount 总数统计和 ContributionGraphLegend 图例等子组件组合而成。全部基于 SVG 渲染，支持自定义颜色级别、多语言标签和灵活的 render props 模式。

本组件为项目本地复合组件，无 Ant Design 直接对应。实现以本地源码为准。

## 何时使用 {#when-to-use}

适合在数据看板、个人主页、统计分析等场景中展示时间序列的活跃度数据，以单元格颜色深浅表达贡献量级。

常见场景：

- 用户活跃度热力图（如 GitHub 贡献图）。
- 项目提交、任务完成频率的时间分布。
- 日志量、访问量等时序数据的可视化。
- 任何需要按日期展示量级分布的网格图表。

不推荐：

- 数据量极大（数年以上按小时粒度）—— 组件按天聚合，周粒度展示。
- 需要精确数值读取 —— 热力图适合相对比较，不适合精确读数。
- 不需要时间维度的数据分布 —— 使用柱状图或折线图。

## 代码演示 {#examples}

### 基础用法

```tsx
import {
  ContributionGraph,
  ContributionGraphCalendar,
  ContributionGraphBlock,
  ContributionGraphFooter,
  ContributionGraphTotalCount,
  ContributionGraphLegend,
} from "@/components/blocks/contribution-graph"

const data = [
  { date: "2025-01-01", count: 3, level: 2 },
  { date: "2025-01-02", count: 1, level: 1 },
  { date: "2025-01-05", count: 8, level: 4 },
  { date: "2025-02-14", count: 5, level: 3 },
]

export default function Demo() {
  return (
    <ContributionGraph data={data}>
      <ContributionGraphCalendar>
        {({ activity, dayIndex, weekIndex }) => (
          <ContributionGraphBlock
            key={`${weekIndex}-${dayIndex}`}
            activity={activity}
            dayIndex={dayIndex}
            weekIndex={weekIndex}
          />
        )}
      </ContributionGraphCalendar>
      <ContributionGraphFooter>
        <ContributionGraphTotalCount />
        <ContributionGraphLegend />
      </ContributionGraphFooter>
    </ContributionGraph>
  )
}
```

> 说明：`ContributionGraph` 作为顶层容器，接收 `data` 数组传入活动数据。`ContributionGraphCalendar` 通过 render props 遍历每周每天，`ContributionGraphBlock` 渲染单个日期单元格。底部使用 `ContributionGraphFooter` 包裹总数和颜色图例。

## API {#api}

以下类型来自 `src/components/blocks/contribution-graph/index.tsx` 源码，是生成代码时的主要依据。

### Activity

```ts
type Activity = {
  date: string;  // ISO 日期字符串，如 "2025-01-01"
  count: number; // 当日贡献计数
  level: number; // 颜色级别 (0 ~ maxLevel)
}
```

### ContributionGraph

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 活动数据数组 | `Activity[]` | 必填 |
| blockMargin | 单元格间距（px） | `number` | `4` |
| blockRadius | 单元格圆角半径（px） | `number` | `2` |
| blockSize | 单元格尺寸（px） | `number` | `12` |
| fontSize | 标签字号（px） | `number` | `14` |
| labels | 多语言标签配置，合并到默认英文标签 | `Labels` | 内置英文默认值 |
| maxLevel | 最大颜色级别（0 ~ maxLevel 共 maxLevel+1 级） | `number` | `4` |
| totalCount | 手动指定总计数；不传时自动从 data 求和 | `number` | 自动计算 |
| weekStart | 一周起始日（0=周日 ~ 6=周六） | `Day` (0-6) | `0`（周日） |
| style | 容器内联样式 | `CSSProperties` | - |
| className | 附加样式类 | `string` | - |
| children | 子组件内容 | `ReactNode` | 必填 |

`ContributionGraph` 继承 `HTMLAttributes<HTMLDivElement>`。当 `data` 为空数组时返回 `null`，不渲染任何内容。自动通过 `date-fns` 计算周分组、月份标签和总计数。

### ContributionGraphCalendar

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| hideMonthLabels | 是否隐藏顶部月份标签 | `boolean` | `false` |
| className | 附加样式类 | `string` | - |
| children | Render props，为每个活动渲染内容 | `(props: { activity: Activity; dayIndex: number; weekIndex: number }) => ReactNode` | 必填 |

`ContributionGraphCalendar` 继承 `Omit<HTMLAttributes<HTMLDivElement>, 'children'>`。内部渲染 SVG 元素，通过 `viewBox` 控制坐标系。`dayIndex` 为 0-6（周日到周六），`weekIndex` 为从左到右的周序号。

### ContributionGraphBlock

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| activity | 对应日期的活动数据 | `Activity` | 必填 |
| dayIndex | 星期序号（0-6） | `number` | 必填 |
| weekIndex | 周序号 | `number` | 必填 |
| className | 附加样式类 | `string` | - |

`ContributionGraphBlock` 继承 `HTMLAttributes<SVGRectElement>`。渲染为 SVG `<rect>` 元素，通过 `data-level` 属性匹配颜色级别，颜色映射为：
- level 0: `fill-muted`
- level 1: `fill-muted-foreground/20`
- level 2: `fill-muted-foreground/40`
- level 3: `fill-muted-foreground/60`
- level 4: `fill-muted-foreground/80`

### ContributionGraphFooter

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。底部容器，提供 flex 横排布局。

### ContributionGraphTotalCount

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 自定义渲染，接收 `totalCount` 和 `year` | `(props: { totalCount: number; year: number }) => ReactNode` | 默认按 `labels.totalCount` 模板渲染 |
| className | 附加样式类 | `string` | - |

继承 `Omit<HTMLAttributes<HTMLDivElement>, 'children'>`。默认使用 `labels.totalCount` 模板，支持 `{{count}}` 和 `{{year}}` 占位符替换。

### ContributionGraphLegend

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 自定义渲染，接收 `level` | `(props: { level: number }) => ReactNode` | 默认渲染 SVG 色块 |
| className | 附加样式类 | `string` | - |

继承 `Omit<HTMLAttributes<HTMLDivElement>, 'children'>`。渲染彩色图例，从 level 0 到 maxLevel 各一个色块。左右分别显示 "Less" 和 "More" 标签。

## 样式模型 {#styling-model}

- 样式由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 容器使用 `flex w-max max-w-full flex-col gap-2`，底部使用 `flex flex-wrap gap-1 whitespace-nowrap sm:gap-x-4`。
- 单元格颜色通过 `data-level` 属性匹配 Tailwind `fill-*` 类，共 5 个级别（0-4）。
- 日历容器设置 `max-w-full overflow-x-auto overflow-y-hidden` 支持横向滚动。
- SVG 使用 `block overflow-visible`，通过 `viewBox` 精确控制坐标系。
- 月份标签使用 `fill-current` 继承文本颜色，`dominantBaseline="hanging"` 顶部对齐。
- 图例色块添加 `stroke-[1px] stroke-border` 描边以增强可见性。
- 主题颜色优先使用 `text-muted-foreground`、`fill-muted`、`border-border` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/contribution-graph/index.tsx`（复合组件定义）
- `src/components/core/blocks/contribution-graph.tsx`（re-export barrel）

直接依赖摘要：

- React（`useContext`、`createContext`、`useMemo`、`Fragment`）
- `date-fns`（`differenceInCalendarDays`、`eachDayOfInterval`、`formatISO`、`getDay`、`getMonth`、`getYear`、`nextDay`、`parseISO`、`subWeeks`）
- `@/kit/utils`（`cn` 样式合并）
- Tailwind CSS

实现注意：

1. 数据通过 `ContributionGraphContext` 在子组件间共享，包括 weeks、尺寸、标签等所有计算数据。
2. `fillHoles` 自动填充数据中缺失的日期（level=0, count=0），确保日历网格连续。
3. `groupByWeeks` 按 `weekStart` 对齐日历起始日，自动补齐首尾空隙。
4. `getMonthLabels` 智能过滤显示月份标签，避免相邻月份标签过密（至少间隔 3 周）。
5. 所有计算放在 `useMemo` 中，仅在 data 变化时重新计算。
6. 组件对 `activity.level` 范围做运行时校验（`0 ~ maxLevel`），超范围抛 `RangeError`。
7. 如需新增能力，应先修改该组件源码，再更新本 llm 文档。

## 可访问性 {#accessibility}

- SVG 包含 `<title>Contribution Graph</title>` 提供整体描述。
- 每个色块设置 `data-date` 和 `data-count` 属性，可用于测试定位和辅助技术。
- 图例色块包含 `<title>{level} contributions</title>` 为每个级别提供文本描述。
- 月份标签和星期标签为纯文本 SVG `<text>` 元素，可被屏幕阅读器读取。
- 底部总计数为纯文本 div，天然可访问。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。AntD 无内置贡献热力图组件。

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/blocks/contribution-graph/index.tsx` 的 TypeScript 类型为准 |
| 渲染方式 | 基于 SVG，非 Canvas |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 数据模型 | 自定义 `Activity` 类型，非 ECharts 或第三方图表库格式 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 ContributionGraph 代码时，应遵守：

1. 从 `@/components/blocks/contribution-graph` 导入所有需要的子组件，不要从 `antd` 导入。
2. `ContributionGraph` 的 `data` prop 为必填，类型为 `Activity[]`，每项包含 `date`、`count`、`level`。
3. `ContributionGraphCalendar` 使用 render props 模式，回调参数包含 `{ activity, dayIndex, weekIndex }`。
4. `ContributionGraphBlock` 必须传入 `activity`、`dayIndex`、`weekIndex` 三个 prop。
5. `level` 值必须在 `0` 到 `maxLevel`（默认 4）之间，超出会抛 `RangeError`。
6. 底部统计和图例必须包裹在 `ContributionGraphFooter` 中。
7. 只使用本文件 API 摘要和源码中存在的 props，不要凭记忆生成不存在的属性。
8. 需要自定义颜色级别时，调整 `maxLevel` 并在 Tailwind 中补充对应的 `data-[level]` 样式。
9. 不要引入 ECharts、Chart.js 或第三方图表库替代。
10. 如果不确定某个 API 是否存在，先检查 `src/components/blocks/contribution-graph/index.tsx`。
