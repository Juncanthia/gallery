RelativeTime 跨时区时间展示复合组件，通过双层 React Context 组合 `RelativeTimeZone`、`RelativeTimeZoneDisplay`、`RelativeTimeZoneDate`、`RelativeTimeZoneLabel` 等子组件。支持在单一容器内展示多个时区的日期和当前时间，非受控模式下每秒自动更新。底层使用浏览器原生 `Intl.DateTimeFormat` API。

<h2 id="when-to-use">何时使用</h2>

当需要在同一界面展示多个时区的当前时间或指定时刻时使用。

常见场景：

- 国际化仪表盘中展示跨时区团队的当前时间。
- 全球服务状态页中显示各区域当地时间。
- 需要精确到秒的实时多时区时钟展示。
- 航班、会议、活动等场景的多时区时间对照。

不推荐：

- 仅展示单个时区的静态时间——直接使用 `Intl.DateTimeFormat` 更轻量。
- 倒计时或计时器场景——RelativeTime 展示时间点，不计算差值。
- 在 `RelativeTime` 外部直接使用 `RelativeTimeZone` 及其子组件（缺少 Context 会报错）。

<h2 id="api">API</h2>

### RelativeTime

根容器，提供时间值和格式化选项的 Context。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| time | 受控时间值，传入后停止自动更新 | `Date` | - |
| defaultTime | 非受控模式下的初始时间 | `Date` | `new Date()` |
| onTimeChange | 时间变化回调 | `(time: Date) => void` | - |
| dateFormatOptions | 日期格式化选项，应用于所有子时区 | `Intl.DateTimeFormatOptions` | `{ dateStyle: "long" }` |
| timeFormatOptions | 时间格式化选项，应用于所有子时区 | `Intl.DateTimeFormatOptions` | `{ hour: "2-digit", minute: "2-digit", second: "2-digit" }` |
| className | 容器 CSS 类名 | `string` | - |

非受控模式下通过 `setInterval` 每秒更新为 `new Date()`。继承 `HTMLAttributes<HTMLDivElement>`。

### RelativeTimeZone

时区行容器，向下提供时区信息。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| zone | IANA 时区标识符（必填） | `string` | - |
| className | 容器 CSS 类名 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。内部通过 `RelativeTimeZoneContext` 向子组件传递 `zone` 值。

### RelativeTimeZoneDisplay

时间显示组件。从 Context 读取时间、格式化选项和时区，渲染格式化后的时间字符串。默认样式 `pl-8 text-muted-foreground tabular-nums`。

### RelativeTimeZoneDate

日期显示组件。从 Context 读取时间、格式化选项和时区，渲染格式化后的日期字符串。无额外默认样式。

### RelativeTimeZoneLabel

时区标签组件。纯展示型，无 Context 依赖。默认样式 `flex h-4 items-center justify-center rounded-xs bg-secondary px-1.5 font-mono`。

<h2 id="faq">常见问题</h2>

### 如何停止实时更新？

传入 `time` prop 进入受控模式，组件不再每秒更新：

```tsx
<RelativeTime time={new Date("2026-07-01T14:00:00Z")}>
  ...
</RelativeTime>
```

### 如何自定义日期/时间格式？

通过 `dateFormatOptions` 和 `timeFormatOptions` 传入 `Intl.DateTimeFormatOptions`：

```tsx
<RelativeTime
  dateFormatOptions={{ year: "numeric", month: "short", day: "numeric" }}
  timeFormatOptions={{ hour: "2-digit", minute: "2-digit" }}
>
  ...
</RelativeTime>
```

### 组件嵌套层级有什么要求？

必须严格遵循：`RelativeTime` > `RelativeTimeZone` > (`RelativeTimeZoneLabel` / `RelativeTimeZoneDate` / `RelativeTimeZoneDisplay`)。越级使用会因缺少 Context 抛出异常。`RelativeTimeZoneLabel` 位置不限，可放在 `RelativeTimeZone` 内任意位置。

### 时区标识符应该用什么格式？

必须使用有效的 IANA 时区标识符，如 `"Asia/Shanghai"`、`"America/New_York"`、`"Europe/London"`、`"UTC"`。不要使用 UTC 偏移量字符串（如 `"+08:00"`）。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/relative-time` 导入所需子组件。
2. 嵌套层级必须遵循：`RelativeTime` > `RelativeTimeZone` > 子组件。
3. `zone` 必须使用有效 IANA 时区标识符，不要使用 UTC 偏移量。
4. 展示实时时钟时不传 `time`（非受控）；展示固定时刻时传 `time`（受控）。
5. 不要引入 dayjs、date-fns 等日期库，组件已内置 `Intl.DateTimeFormat`。
6. 不要引入 Ant Design 相关依赖。
