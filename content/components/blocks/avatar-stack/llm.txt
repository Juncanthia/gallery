# AvatarStack 头像堆叠

AvatarStack 是一个头像堆叠组件，通过 CSS mask-image 实现相邻头像间的圆形裁切重叠效果。它将多个 Avatar 子组件紧凑排列，提供静态或 hover 展开的视觉效果，适用于展示用户列表、协作者集合等场景。

本组件不是 Ant Design Avatar.Group 的兼容层，而是利用原生 CSS mask 实现的轻量方案，无需额外 JS 计算。

## 何时使用 {#when-to-use}

需要将多个头像紧凑堆叠展示时使用，例如：

- 文档协作者头像列表。
- 群组成员快速预览。
- 评论或参与者的头像聚合展示。
- 空间受限时的多头像排列。

常见场景：

- 文件/文档的共享用户列表，2-6 个头像为最佳显示范围。
- 聊天列表中的参与人头像预览。
- 看板卡片上的任务负责人集合。
- 项目详情页的团队成员展示。

推荐使用方式：

- 将 Avatar 组件作为 children 传入 AvatarStack。
- 如需 hover 展开效果，设置 `animate={true}`。
- 通过 `size` 控制每个头像的尺寸。

不推荐：

- 头像数量超过 10 个仍使用堆叠——此时应考虑"前 N 个 + 更多"的模式。
- 将非 Avatar 组件传入 children——AvatarStack 内部假设子元素含 `data-slot="avatar"`。

## 代码演示 {#examples}

### 基础用法

传入多个 Avatar 实现头像堆叠。

```tsx
import { AvatarStack } from "@/components/blocks/avatar-stack"
import { Avatar } from "@/components/core/avatar"

export default function Demo() {
  return (
    <AvatarStack>
      <Avatar name="张三" />
      <Avatar name="李四" />
      <Avatar name="王五" />
      <Avatar name="赵六" />
    </AvatarStack>
  )
}
```

### 悬停展开

设置 `animate` 为 true，hover 时头像间距展开。

```tsx
import { AvatarStack } from "@/components/blocks/avatar-stack"
import { Avatar } from "@/components/core/avatar"

export default function Demo() {
  return (
    <AvatarStack animate>
      <Avatar name="张三" />
      <Avatar name="李四" />
      <Avatar name="王五" />
    </AvatarStack>
  )
}
```

### 自定义尺寸

通过 `size` 控制每个 Avatar 的尺寸，默认为 40px。

```tsx
import { AvatarStack } from "@/components/blocks/avatar-stack"
import { Avatar } from "@/components/core/avatar"

export default function Demo() {
  return (
    <div className="space-y-4">
      <AvatarStack size={32} animate>
        <Avatar name="张三" />
        <Avatar name="李四" />
        <Avatar name="王五" />
      </AvatarStack>
      <AvatarStack size={48}>
        <Avatar name="张三" />
        <Avatar name="李四" />
        <Avatar name="王五" />
      </AvatarStack>
    </div>
  )
}
```

## API {#api}

### AvatarStack

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 子元素，应为多个 Avatar 组件 | `ReactNode` | 必填 |
| animate | 是否启用 hover 展开动画 | `boolean` | `false` |
| size | 每个头像的尺寸（宽高），同时影响 mask 裁切半径 | `number` | `40` |
| className | 附加 className | `string` | - |

## 样式模型 {#styling-model}

样式通过 Tailwind CSS 内联，使用 CSS mask-image 实现重叠效果。

- 容器使用 `-space-x-1` 产生负间距重叠，`flex items-center` 横向排列。
- animate 模式下通过 `hover:space-x-0` 和 `[&>*]:transition-all` 实现悬停展开。
- 每个子元素包裹层使用 `maskImage` 动态生成 radial-gradient 圆形裁切：第二个及之后的头像在重叠侧产生透明渐变，形成自然的圆形交叠效果。mask 半径和偏移量根据 `size` 自动计算，公式为 `circle ${size/2}px at -${size/4 + size/10}px 50%`。
- `[&_[data-slot="avatar"]]:size-full` 确保内部 Avatar 填满包裹层。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/avatar-stack/index.tsx`：AvatarStack 定义。

直接依赖：`@/kit/utils`（cn）。

关键行为：

1. 使用 React 的 `Children.map` 遍历子节点，在非首个子节点上应用 mask-image 圆形裁切。
2. `size` 同时控制包裹层宽高和 mask 渐变半径，保证裁切区域精确匹配头像边界。
3. animate 仅影响 CSS transition，无需 JavaScript 动画库。
4. 空子节点被自动过滤（返回 null）。
5. 组件不引入额外外部依赖（motion、radix 等），纯 React + Tailwind + CSS mask。

## 可访问性 {#accessibility}

- AvatarStack 为纯展示组件，使用 div 元素。
- 头像应通过 Avatar 组件自身的 `name`、`aria-label` 等属性提供可访问名称。
- 堆叠的头像不提供逐个的交互焦点——如需工具提示或可交互头像组，请使用 AvatarTooltipGroup。
- 对于屏幕阅读器，建议在 AvatarStack 容器上添加 `role="list"`，每个内部包裹层设置 `role="listitem"`（由调用侧负责）。

## 与 Ant Design 的取舍 {#antd-differences}

AvatarStack 是本地设计系统的复合组件。Ant Design 的 Avatar.Group 提供类似的 maxCount、maxPopoverPlacement 等能力，但本地不做该兼容层。

| AntD 能力 | 本地决策 | 原因 |
| --- | --- | --- |
| Avatar.Group maxCount | 不做 | 本地由调用侧控制数量，"前 N + 更多"由调用侧自行实现 |
| maxPopoverPlacement | 不做 | 本地有独立的 AvatarTooltipGroup 覆盖交互式头像组 |
| Avatar.Group size 数组 | 不做 | 本地统一 size，保持视觉一致性 |
| 点击展开 Popover | 不做 | 如有交互需求请使用 AvatarTooltipGroup |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 AvatarStack 代码时，应遵守：

1. 从 `@/components/blocks/avatar-stack` 导入 AvatarStack。
2. children 必须是 Avatar 组件（从 `@/components/core/avatar` 导入），不要传入其他类型的元素。
3. AvatarStack 可以设置 `size`、`animate`、`className`，不要添加未定义的属性。
4. `size` 与 Avatar 自身的 size 属性保持一致以呈现正确效果。
5. 头像数量建议 2-6 个；超过 10 个时应考虑截断显示。
6. 如需 hover 时展开间距，使用 `animate` 属性；不要手动写入 hover 相关的 className。
7. 不要删除或绕开 CSS mask-image 裁切逻辑——这是 AvatarStack 的核心视觉特征。
8. 不要在 AvatarStack 上添加 onClick 等交互行为——这是静态展示组件。如需可交互的头像组请使用 AvatarTooltipGroup。
