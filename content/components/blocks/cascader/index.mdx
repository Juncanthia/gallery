Cascader 级联选择，适用于选项有层级关系的数据选择。通过多列面板展示树形选项，用户逐级选择直到叶子节点或标记为可选的中间节点。

<h2 id="when-to-use">何时使用</h2>

当选项集合具有明确的层级/父子关系，用户需要从多级树形结构中逐级选择时使用。

常见场景：

- 省市区地址选择。
- 组织架构/部门选择。
- 商品类目（一级类目 / 二级类目 / 三级类目）。
- 文件目录路径选择。

Cascader 通过多列面板展示层级，选择父级后自动展开下一层。叶子节点或标记 `leaf` 的节点点击后立即提交选择并关闭面板。

<h2 id="examples">代码演示</h2>

### 基础用法

<ComponentShowcase id="cascader" path="cascader/basic" />

```tsx
import { Cascader } from "@/components/core/cascader"

const options = [
  {
    value: "zhejiang",
    label: "Zhejiang",
    children: [
      { value: "hangzhou", label: "Hangzhou", children: [
        { value: "xihu", label: "West Lake" }
      ]},
    ],
  },
]

export default function Demo() {
  return (
    <Cascader
      options={options}
      placeholder="请选择"
      className="w-48"
    />
  )
}
```

### 中间节点可选

设置 `leaf: true` 让任意层级节点成为可选节点，点击时直接完成选择。

```tsx
import { Cascader } from "@/components/core/cascader"

const options = [
  {
    value: "zhejiang",
    label: "浙江",
    leaf: true,
    children: [
      { value: "hangzhou", label: "杭州" },
      { value: "ningbo", label: "宁波" },
    ],
  },
  {
    value: "jiangsu",
    label: "江苏",
    children: [
      { value: "nanjing", label: "南京" },
      { value: "suzhou", label: "苏州" },
    ],
  },
]

export default function Demo() {
  return (
    <Cascader
      options={options}
      placeholder="可选择省份或城市"
      className="w-48"
    />
  )
}
```

### 自定义展示格式

`displayRender` 控制选中后触发器中显示的文本。

```tsx
import { Cascader } from "@/components/core/cascader"

export default function Demo() {
  return (
    <Cascader
      options={options}
      displayRender={(labels) => labels.join(" > ")}
      className="w-48"
    />
  )
}
```

<h2 id="api">API</h2>

### Cascader

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| options | 级联选项数据 | `CascaderOption[]` | 必填 |
| value | 受控选中路径（value 数组） | `string[]` | - |
| defaultValue | 默认选中路径 | `string[]` | `[]` |
| onChange | 选择完成回调，返回选中路径和路径上的选项 | `(value: string[], options: CascaderOption[]) => void` | - |
| placeholder | 占位文本 | `string` | `"Select..."` |
| disabled | 是否禁用 | `boolean` | `false` |
| displayRender | 自定义选中后的展示文本 | `(labels: string[]) => React.ReactNode` | - |
| allowClear | 是否显示清除按钮 | `boolean` | `true` |
| className | 根元素 CSS 类名 | `string` | - |
| triggerClassName | 触发器按钮 CSS 类名 | `string` | - |

### CascaderOption

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| value | 选项值 | `string` |
| label | 选项文本 | `React.ReactNode` |
| children | 子选项 | `CascaderOption[]` |
| disabled | 是否禁用该选项 | `boolean` |
| leaf | 是否叶子节点（为 `true` 时点击即选中并关闭面板） | `boolean` |

<h2 id="faq">常见问题</h2>

### 如何让任意层级节点可选？

设置 `leaf: true` 可以使该节点成为可选择叶子节点（即使它有 children），点击时立即提交选择并关闭面板。

```tsx
{ value: "hangzhou", label: "Hangzhou", leaf: true, children: [...] }
```

### 如何自定义选中后的展示？

使用 `displayRender`，它接收各级选中节点的 label 数组：

```tsx
<Cascader displayRender={(labels) => labels.join(" > ")} />
```

默认展示为 `"label1 / label2 / label3"`。

### 如何清空选择？

设置 `allowClear`（默认 `true`），选中值后触发器右侧出现 X 图标按钮，点击即可清空。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Cascader 代码时：

1. 从 `@/components/core/cascader` 导入。
2. `options` 必须为树形结构，每个节点有 `value`、`label` 和可选的 `children`。
3. `value` 和 `onChange` 的 value 是 `string[]`（路径数组），不是单个值。
4. 需要中间节点可选时设置 `leaf: true`。
5. 不要引入 Ant Design 相关依赖。
