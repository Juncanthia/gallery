Comparison 对比滑块，图片前后对比的复合组件。支持拖拽和悬停两种交互模式，基于 motion/react 弹簧动画，用户通过拖拽手柄直观对比两张等尺寸图片的差异。

<h2 id="when-to-use">何时使用</h2>

当需要让用户直观对比两张图片差异，通过拖动滑块查看边界时使用。

常见场景：

- 图像处理前后的效果对比（如修图、滤镜、AI 增强）。
- 设计稿与实现效果的视觉对比。
- 产品 A/B 版本的差异展示。
- 地图、建筑、景观等同一视角不同时期的对比。

不推荐：

- 两图并排静态对比，使用普通 flex 排版即可，Comparison 适合需要滑动查看边界的场景。
- 超过两图的对比，本组件只支持左右两张图。
- 对比内容不等尺寸，容器内图片应等尺寸以获得正确的 clip-path 裁剪效果。

<h2 id="api">API</h2>

### Comparison

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| mode | 交互模式：`drag` 为点击拖拽，`hover` 为鼠标跟随 | `"hover"` \| `"drag"` | `"drag"` |
| onDragStart | 拖拽开始回调（仅 drag 模式） | `() => void` | - |
| onDragEnd | 拖拽结束回调（仅 drag 模式） | `() => void` | - |
| className | 附加样式类 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。使用 `role="slider"` 和 `aria-valuenow`/`aria-valuemin`/`aria-valuemax` 提供可访问性语义。通过 `ImageComparisonContext` 共享滑块位置（sliderPosition、motionSliderPosition）和 mode。

### ComparisonItem

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| position | 内容区位置，决定 clip-path 动画方向 | `"left"` \| `"right"` | 必填 |
| className | 附加样式类 | `string` | - |

继承 `ComponentProps<typeof motion.div>`。`position="left"` 时 clip-path 从左侧裁剪；`position="right"` 时从右侧裁剪。使用 `useTransform` 将滑块百分比映射为 CSS clip-path 值。渲染为 `motion.div`，设置 `aria-hidden="true" role="img"`。

### ComparisonHandle

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 自定义手柄内容，覆盖默认手柄 UI | `React.ReactNode` | 默认竖线 + GripVertical 图标 |
| className | 附加样式类 | `string` | - |

继承 `ComponentProps<typeof motion.div>`。默认手柄包含一条 1px 分割竖线，drag 模式下额外显示 GripVertical 拖拽图标。手柄位置通过 `useTransform(motionSliderPosition, v => v + '%')` 跟踪。

<h2 id="faq">常见问题</h2>

### drag 和 hover 模式如何选择？

- **drag 模式**：适用于精确对比场景，用户通过拖拽手柄控制分界线位置，手柄可见并提供拖拽图标。
- **hover 模式**：适用于快速浏览，滑块跟随鼠标位置移动无需点击。手柄不显示拖拽图标，仅显示分割线。

### 为何使用 motion/react 弹簧动画？

`useSpring` 配置为 `bounce: 0, duration: 0`，确保滑块即时跟手无弹性延迟。`useTransform` 直接驱动 clip-path，性能由 motion 优化，避免 JS 直接操作 DOM。

### 对比图片尺寸不一致会怎样？

必须确保两张图片尺寸相同。组件通过 `absolute inset-0` 绝对定位两张图片，并通过 clip-path 分别裁剪左右区域。尺寸不一致会导致裁剪边界错位或显示不完整。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/comparison` 导入。
2. `Comparison` 必须作为最外层容器，所有子组件必须在其内部使用。
3. `ComparisonItem` 的 `position` prop 为必填，取值为 `"left"` 或 `"right"`。
4. 两张对比图片应有相同尺寸，以确保 clip-path 裁剪效果正确。
5. 默认使用 `mode="drag"`；如需鼠标跟随效果，使用 `mode="hover"`。
6. `ComparisonHandle` 可选，不提供时使用默认手柄 UI；提供 `children` 可完全自定义。
7. 不要删除组件内部的 motion 弹簧动画；这是本地 Comparison 的核心交互。
8. 不要引入 Ant Design 相关依赖。
