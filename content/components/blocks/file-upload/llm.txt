# FileUpload 文件上传

无头（headless）复合组件，通过 `FileUpload` 根组件管理文件状态，配合 `FileUploadDropzone`（拖放区）、`FileUploadTrigger`（触发按钮）、`FileUploadList`/`FileUploadItem`（文件列表）、`FileUploadItemPreview`/`FileUploadItemMetadata`/`FileUploadItemProgress`（文件项内部展示）、`FileUploadItemDelete`/`FileUploadClear`（删除/清空）自由组合出上传界面。

本组件为项目本地复合组件（Blocks），无 Ant Design 直接对应。API 和行为以 `src/components/blocks/file-upload/index.tsx` 源码为准。

业务层的 `@/components/document/components/ui/blocks/file-upload`（预制视觉的文档上传卡片）就是基于本组件搭建的预设，服务不同场景，见下文"与 extend FileUpload 的关系"。

## 何时使用 {#when-to-use}

当需要用户选择或拖拽文件，并对文件类型、大小、数量做前端校验，同时展示文件列表和上传进度时使用。

常见场景：

- 表单中的附件上传（简历、合同、图片素材）。
- 需要拖拽 + 点击双入口、并展示上传进度的场景。
- 需要自定义文件预览、校验规则或上传状态渲染的场景。

不推荐：

- 只需要一个开箱即用、带业务视觉（图标动画、装饰边框）的文档上传卡片——优先使用 `@/components/document/components/ui/blocks/file-upload`，不要用本组件重新拼一遍。
- 需要断点续传、分片上传等复杂传输协议——本组件只负责选择、校验和状态展示，`onUpload` 回调里自行对接传输层。
- 为了照搬 Ant Design Upload API 而引入本组件——它不对应任何 AntD 组件，API 完全不同。

## 代码演示 {#examples}

### 基础用法

```tsx
import { useState } from "react"
import {
  FileUpload,
  FileUploadDropzone,
  FileUploadTrigger,
  FileUploadList,
  FileUploadItem,
  FileUploadItemPreview,
  FileUploadItemMetadata,
  FileUploadItemDelete,
  FileUploadClear,
} from "@/components/blocks/file-upload"
import { Button } from "@/components/core/button"
import { XIcon } from "lucide-react"

export default function Demo() {
  const [files, setFiles] = useState<File[]>([])

  return (
    <FileUpload
      value={files}
      onValueChange={setFiles}
      accept="image/*,.pdf"
      maxFiles={5}
      maxSize={5 * 1024 * 1024}
      multiple
    >
      <FileUploadDropzone>
        <p>拖拽文件到此处，或点击选择</p>
        <FileUploadTrigger asChild>
          <Button variant="outline" size="sm">浏览文件</Button>
        </FileUploadTrigger>
      </FileUploadDropzone>
      <FileUploadList>
        {files.map((file) => (
          <FileUploadItem key={file.name} value={file}>
            <FileUploadItemPreview />
            <FileUploadItemMetadata />
            <FileUploadItemDelete asChild>
              <Button variant="ghost" size="icon" className="size-7">
                <XIcon />
              </Button>
            </FileUploadItemDelete>
          </FileUploadItem>
        ))}
      </FileUploadList>
      <FileUploadClear asChild>
        <Button variant="outline" size="sm">清空全部</Button>
      </FileUploadClear>
    </FileUpload>
  )
}
```

## 核心导出 {#exports}

- `FileUpload` — 根组件（store + 状态管理 + 隐藏 input）
- `FileUploadDropzone` — 拖放/点击/粘贴区域
- `FileUploadTrigger` — 独立触发按钮
- `FileUploadList` — 文件列表容器
- `FileUploadItem` — 单个文件容器
- `FileUploadItemPreview` — 缩略图/图标预览
- `FileUploadItemMetadata` — 文件名/大小/错误信息
- `FileUploadItemProgress` — 上传进度（linear/circular/fill）
- `FileUploadItemDelete` — 删除单个文件
- `FileUploadClear` — 清空全部文件
- `useFileUpload` — 读取内部 store 的选择器 hook
- `FileUploadProps`

## API {#api}

类型来自 `src/components/blocks/file-upload/index.tsx`。

### FileUpload

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| value | `File[]` | - | 受控文件列表 |
| defaultValue | `File[]` | - | 非受控初始文件列表 |
| onValueChange | `(files: File[]) => void` | - | 文件列表变化回调 |
| onAccept | `(files: File[]) => void` | - | 一批文件通过校验后触发 |
| onFileAccept | `(file: File) => void` | - | 单个文件通过校验时触发 |
| onFileReject | `(file: File, message: string) => void` | - | 单个文件被拒绝时触发 |
| onFileValidate | `(file: File) => string \| null \| undefined` | - | 自定义校验，返回非空字符串即拒绝 |
| onUpload | `(files: File[], options: { onProgress, onSuccess, onError }) => Promise<void> \| void` | - | 上传编排回调 |
| accept | `string` | - | 接受的文件类型，逗号分隔（MIME/扩展名/`type/*`） |
| maxFiles | `number` | - | 最大文件数量 |
| maxSize | `number` | - | 单文件最大体积（字节） |
| multiple | `boolean` | `false` | 是否允许多选 |
| disabled | `boolean` | `false` | 禁用整个上传器 |
| invalid | `boolean` | `false` | 外部受控的校验失败态 |
| required | `boolean` | `false` | 隐藏 input 是否必填 |
| dir | `"ltr" \| "rtl"` | - | 文字方向 |
| label | `string` | `"File upload"` | 无障碍标签（视觉隐藏） |
| name | `string` | - | 隐藏 input 的 name |
| asChild | `boolean` | - | 根节点渲染委托给子元素 |

### FileUploadDropzone / FileUploadTrigger

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| asChild | `boolean` | - | 渲染委托给子元素 |

`FileUploadDropzone` 暴露 `data-dragging`/`data-invalid`/`data-disabled` 数据属性用于样式响应；点击非 Trigger 区域、拖拽释放、粘贴文件都会走同一套校验与状态更新逻辑。

### FileUploadList

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| orientation | `"horizontal" \| "vertical"` | `"vertical"` | 排列方向 |
| forceMount | `boolean` | `false` | 无文件时也强制渲染 |
| asChild | `boolean` | - | 渲染委托给子元素 |

`children` 是普通 `ReactNode`，不是 render prop，需要自己 `map` 文件数组生成 `FileUploadItem`。

### FileUploadItem

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| value | `File` | 必填 | 对应的 File 对象 |
| asChild | `boolean` | - | 渲染委托给子元素 |

### FileUploadItemPreview

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| render | `(file: File, fallback: () => ReactNode) => ReactNode` | - | 自定义渲染，可调用 `fallback()` 拿默认内容 |
| asChild | `boolean` | - | 渲染委托给子元素 |

图片文件自动生成 blob URL 缩略图（缓存于组件内部 `WeakMap`，卸载/删除时自动 `URL.revokeObjectURL`），其他类型按扩展名/MIME 回退到对应图标。

### FileUploadItemMetadata

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| size | `"default" \| "sm"` | `"default"` | 文字尺寸 |
| asChild | `boolean` | - | 渲染委托给子元素 |

### FileUploadItemProgress

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| variant | `"linear" \| "circular" \| "fill"` | `"linear"` | 展示样式 |
| size | `number` | `40` | `circular` 变体直径（px） |
| forceMount | `boolean` | `false` | 完成后（`progress === 100`）仍然渲染 |
| asChild | `boolean` | - | 渲染委托给子元素 |

### FileUploadItemDelete / FileUploadClear

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| asChild | `boolean` | - | 渲染委托给子元素 |
| forceMount（仅 Clear） | `boolean` | `false` | 无文件时也渲染 |

### useFileUpload

```ts
function useFileUpload<T>(selector: (state: { files: Map<File, FileState>; dragOver: boolean; invalid: boolean }) => T): T
```

基于 `useSyncExternalStore` 的选择器 hook，用于在自定义子组件中读取内部 store（文件 Map、拖拽状态、校验失败态），必须在 `FileUpload` 内部使用。

## 样式模型 {#styling-model}

- 使用 Tailwind CSS 和项目主题变量（`bg-accent`、`text-destructive`、`bg-primary` 等）。
- 所有子组件根节点携带 `data-slot="blocks/file-upload-*"` 语义标记。
- `FileUploadDropzone` 通过 `data-dragging`/`data-invalid`/`data-disabled` 驱动边框和背景色变化。
- `FileUploadList` 使用 `data-[state=active|inactive]` 配合 `animate-in`/`animate-out` 做进出场动画。
- `FileUploadItemProgress` 的三种变体（linear/circular/fill）分别用条形、SVG 圆环描边、剪裁蒙层实现。
- 所有交互子组件都支持 `asChild`（基于 `radix-ui` 的 `Slot`），可以直接包裹项目的 `Button` 等组件而不引入多余包裹元素。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/file-upload/index.tsx` — 完整实现
- `src/components/core/blocks/file-upload.tsx` — 重导出

直接依赖摘要：

- `radix-ui`（`Slot` 用于 `asChild`，`Direction` 用于 RTL 继承）
- `lucide-react`（文件类型图标）
- `@/kit/utils`（cn）
- `@/hooks/use-as-ref`、`@/hooks/use-lazy-ref`（通用 ref/惰性初始化 hook，非 UI 组件，多处复用）
- `React.useSyncExternalStore`（内部 store 订阅机制）

实现注意：

1. 内部维护一个基于 `Map<File, FileState>` 的自定义 store（`getState/dispatch/subscribe`），通过 `useSyncExternalStore` 暴露给 `useFileUpload`/`useStore`，而不是 `useState` + prop drilling。
2. 受控模式（传 `value`）下，`value` 变化会通过 `SET_FILES` 同步到内部 store；非受控模式下仅在挂载时使用一次 `defaultValue`。
3. `accept` 匹配同时支持 MIME 类型（`image/png`）、扩展名（`.png`）、通配符（`image/*`）三种写法。
4. 文件校验顺序：`onFileValidate` → `accept` → `maxSize`，任意一步失败都会调用 `onFileReject` 并跳过该文件；`maxFiles` 在校验前先做数量裁剪。
5. 校验失败时 `invalid` 状态会置为 `true` 并在 2 秒后自动复位，用于短暂的错误态样式（如 `FileUploadDropzone` 的 `data-invalid`）。
6. `onUpload` 提供 `onProgress`/`onSuccess`/`onError` 三个回调用于驱动每个文件的上传状态；不传 `onUpload` 时，文件校验通过后会直接标记为 `success`（即"仅选择、不上传"模式）。
7. 图片预览使用 `URL.createObjectURL` 生成的 blob URL，缓存在 `WeakMap<File, string>` 中，文件被删除或 `FileUpload` 卸载时统一 `URL.revokeObjectURL` 释放，避免内存泄漏。
8. 支持拖拽（`onDrop`）和粘贴（`onPaste`，从剪贴板读取文件）两种方式导入文件，两者都会合成一个 `DataTransfer` 并派发到隐藏 `<input type="file">` 上，复用同一套 `onChange` 校验逻辑。

## 可访问性 {#accessibility}

- `FileUploadDropzone` 使用 `role="region"`、`aria-controls`、`aria-invalid`、`tabIndex`，支持键盘 Enter/Space 打开文件选择框。
- `FileUploadItem` 使用 `role="listitem"`、`aria-setsize`/`aria-posinset` 描述在列表中的位置，并通过隐藏的 `span` 播报当前状态（"Uploading: 40% complete"、"Upload complete"、"Error: ..." 等）。
- `FileUploadItemProgress` 使用 `role="progressbar"` 及完整的 `aria-valuemin/max/now/text`。
- 隐藏的原生 `<input type="file">` 通过 `aria-labelledby`/`aria-describedby` 与视觉隐藏的 label、Dropzone 关联。

## 与 extend FileUpload 的关系 {#relation-to-extend}

项目中还存在一个业务层组件 `@/components/document/components/ui/blocks/file-upload`（文档见 `content/components/document/components/ui/blocks/file-upload/`）。两者不是并列的两套实现，而是 primitive 与预设（preset）的关系：extend 版本内部直接组合本组件的 `FileUpload`/`FileUploadDropzone`/`useFileUpload`，复用本组件的拖拽状态机、accept/大小校验、键盘与粘贴上传逻辑，只在其上叠加业务视觉（图标动效、`BorderBeam`、缩略图文件列表）和"每次上传替换文件列表"的业务语义。

| 维度 | 本组件（`@/components/blocks/file-upload`） | extend `FileUpload` |
| --- | --- | --- |
| 定位 | 无头复合组件，负责状态和交互，不含预制视觉 | 基于本组件搭建的、预制视觉的文档上传业务卡片，开箱即用 |
| API 形态 | 复合子组件组合（Dropzone/Trigger/List/Item/...） | 单一组件 + 扁平 props |
| 视觉定制 | 完全自由（自己写 className 和布局） | 有限（`title`/`description`/图标集合等预设项） |
| 典型使用者 | 需要自定义交互或对接真实上传状态的表单 | 需要"开箱即用"文档上传区域的业务页面（如 `pdf-blocks/dropzone`） |

两者都保留，不要互相替换或合并成一个组件；生成代码时按场景二选一。如需新增底层能力（校验规则、拖拽行为等），优先改本组件（primitive），extend 版本会自动获得该能力，不要在 extend 组件里重复实现状态机逻辑。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。

| 方向 | 本地决策 |
| --- | --- |
| 组合模式 | 无头复合子组件（Dropzone/Trigger/List/Item），非 AntD 式单一 `Upload` 组件 |
| 状态管理 | 自定义 store + `useSyncExternalStore`，不依赖 AntD `fileList` 受控模型 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量，不使用 AntD token |
| 上传传输 | 只提供 `onUpload` 编排钩子，不内置任何传输实现（AntD 默认走 XHR + action URL） |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 FileUpload 代码时，应遵守：

1. 从 `@/components/blocks/file-upload` 导入，不要从 `@/components/blocks/file-upload` 或 `@/components/blocks/file-upload` 导入，也不要从 `antd` 导入 `Upload`。
2. `FileUploadList` 的 `children` 必须是普通 `ReactNode`（通常是 `files.map(...)` 的结果），不是 render prop。
3. `FileUploadItem` 必须传入 `value`（对应的 `File` 对象），否则无法关联到内部文件状态，组件会返回 `null`。
4. 优先使用受控模式（`value` + `onValueChange`）维护文件数组，而不是自己另外维护一份独立 state。
5. 需要展示进度时使用 `FileUploadItemProgress`，并在 `onUpload` 回调中调用 `onProgress` 更新进度、`onSuccess`/`onError` 标记结束状态。
6. 自定义触发按钮样式时用 `FileUploadTrigger asChild` 包裹项目的 `Button`，不要重新实现"点击打开文件选择框"的逻辑。
7. 不要引入 Ant Design 的 `Upload`/`rc-upload`，也不要引入 `react-blocks/dropzone` 等第三方拖拽库。
8. 业务专属的文档缩略图、图标动画、装饰性边框光束等视觉效果属于业务层能力，不要塞进本组件的 props；需要开箱即用的文档上传卡片时改用 `@/components/document/components/ui/blocks/file-upload`。
9. 只使用本文件 API 表格和源码中存在的 props，不要凭 AntD Upload 或其他上传库的记忆生成 API（如 `beforeUpload`、`fileList`、`customRequest` 等均不存在）。
