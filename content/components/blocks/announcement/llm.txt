# Announcement 公告条

Announcement 是一个复合公告条组件，由 Badge 作为容器包裹 AnnouncementTag 和 AnnouncementTitle。它用于在页面顶部或关键位置展示全局通知、活动公告或系统状态信息，提供简洁的一行式公告布局。

Announcement 并非 Ant Design 任何组件的兼容层。它是本地设计系统内建的复合组件，组合 Badge 语义和公告特有的排版结构，调用侧无需手动拼装样式。

## 何时使用 {#when-to-use}

需要在页面顶部、模块上方或内容区域前展示醒目但不干扰的全局性公告信息时使用。

常见场景：

- 系统维护通知、版本更新提醒。
- 活动宣传条、优惠政策横幅。
- 状态提示，例如"当前网络不稳定"。
- 需要带标签分类的公告信息（通过 AnnouncementTag）。

推荐使用方式：

- 在 Announcement 内组合 AnnouncementTag 和 AnnouncementTitle，形成"标签 + 文字"的标准公告布局。
- AnnouncementTag 用于标注公告类型（如"新功能"、"重要"、"公告"）。
- AnnouncementTitle 承载公告正文内容。

不推荐：

- 用 Announcement 替代 Banner 组件——Banner 提供完整的可关闭、带操作按钮的横幅体验；Announcement 适用于只读的、不关闭的轻量公告。
- 在 Announcement 中放置交互控件。如需操作入口，请使用 Banner。

## 代码演示 {#examples}

### 基础用法

Announcement 包裹 AnnouncementTag 和 AnnouncementTitle 构成标准公告条。

```tsx
import { Announcement, AnnouncementTag, AnnouncementTitle } from "@/components/blocks/announcement"

export default function Demo() {
  return (
    <Announcement>
      <AnnouncementTag>新功能</AnnouncementTag>
      <AnnouncementTitle>智能搜索已上线，试试用自然语言查找数据</AnnouncementTitle>
    </Announcement>
  )
}
```

### 带主题 (themed)

设置 `themed` 为 true 后，Announcement 启用主题感知的公告样式。

```tsx
import { Announcement, AnnouncementTag, AnnouncementTitle } from "@/components/blocks/announcement"

export default function Demo() {
  return (
    <Announcement themed>
      <AnnouncementTag>重要</AnnouncementTag>
      <AnnouncementTitle>系统将于今晚 22:00 进行维护升级</AnnouncementTitle>
    </Announcement>
  )
}
```

## API {#api}

### Announcement

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| themed | 是否启用主题感知样式，开启后 Tag 背景半透明、边框变淡 | `boolean` | `false` |
| variant | 继承自 Badge 的变体属性 | `Badge variant` | `"outline"` |
| className | 附加 className，通过 cn 与默认样式合并 | `string` | - |
| children | 公告内容，通常为 AnnouncementTag + AnnouncementTitle | `React.ReactNode` | - |

同时支持 Badge 的其他 `ComponentProps`（如 `count` 等）。

### AnnouncementTag

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |
| children | 标签文本 | `React.ReactNode` | - |

支持所有原生 `HTMLAttributes<HTMLDivElement>`。

### AnnouncementTitle

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |
| children | 公告标题文本 | `React.ReactNode` | - |

支持所有原生 `HTMLAttributes<HTMLDivElement>`。

## 样式模型 {#styling-model}

样式通过 Tailwind CSS 内联在组件中，通过 `cn` 工具合并。

- Announcement：`max-w-full` 保证宽度自适应，`gap-2` 控制内部间距，`rounded-full` 胶囊外形，`bg-background` 背景色配合 `shadow-sm` 产生轻微浮层感，`hover:shadow-md` 悬停反馈。
- AnnouncementTag：`-ml-2.5` 向左负边距与容器视觉对齐，`truncate` 防止溢出，`bg-foreground/5` 半透明背景。
- AnnouncementTitle：`flex items-center gap-1` 横向排列，`truncate` 处理长文本。
- themed 模式：通过 `group-[.blocks/announcement-themed]` 父子选择器联动，Tag 背景切换为 `bg-background/60`。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/announcement/index.tsx`：Announcement、AnnouncementTag、AnnouncementTitle 三者定义。

直接依赖：`@/components/core/badge`、`@/kit/utils`（cn）。

关键行为：

1. Announcement 基于 Badge 组件，继承其 `variant` 等全部属性。
2. AnnouncementTag 和 AnnouncementTitle 是纯布局组件，无状态逻辑。
3. 组件不支持关闭或展开/收起——如需可关闭的公告条，请使用 Banner。
4. 无外部依赖（除 Badge 和 cn）。

## 可访问性 {#accessibility}

- 组件为纯展示性内容，使用原生 div 元素。
- 如有紧急或重要通知，建议在父级添加适当的 `role="blocks/status"` 或 `aria-live="polite"` 属性。
- AnnouncementTag 文本应与 AnnouncementTitle 内容形成清晰的语义关系。

## 与 Ant Design 的取舍 {#antd-differences}

Announcement 是本地设计系统的复合组件，Ant Design 中无直接对应组件。AntD 的类似场景通常通过 Alert + Banner 或自定义排版实现。

| AntD 能力 | 本地决策 | 原因 |
| --- | --- | --- |
| Alert blocks/banner 模式 | 不做 Alert 兼容 | 本地有独立的 Announcement 和 Banner 两个组件分别覆盖轻量公告和可交互横幅 |
| 全局通知配置 | 不做 | Announcement 是纯展示组件，由调用侧控制可见性和位置 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Announcement 代码时，应遵守：

1. 从 `@/components/blocks/announcement` 导入 Announcement、AnnouncementTag、AnnouncementTitle。
2. Announcement 内必须包含 AnnouncementTag 和 AnnouncementTitle 子组件以形成完整公告结构。
3. 使用 `themed` 属性启用主题感知样式，而不是手动覆盖 className。
4. 不要在 Announcement 内放置 Button 或交互控件；如需操作入口请使用 Banner。
5. Announcement 的 `variant` 设为 `"outline"`（默认），不要随意更改。
6. AnnouncementTag 用于展示公告分类标签（如"新功能"、"公告"），AnnouncementTitle 用于展示正文。
7. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token。
