# MiniCalendar 迷你日历

横向展示若干连续日期，通过左右箭头翻页，支持点击选中某一天。由 `MiniCalendar`（外层容器）、`MiniCalendarNavigation`（翻页箭头）、`MiniCalendarDays`（日期列表容器）和 `MiniCalendarDay`（单个日期按钮）四个子组件组合而成。本组件是项目本地复合块，无直接 Ant Design 对应。

## 何时使用

适合在仪表盘、数据面板等需要快速切换或选中某天的界面中使用。

常见场景：仪表盘顶部日期快切栏；排班/预约系统中选择某一天；数据概览按天切换视图。不适用于需要月视图、周视图或日期范围选择的复杂日历场景。

不推荐在调用侧绕过子组件直接手写 children 布局。

## 代码演示

### 基础用法（组合模式）

```tsx
import {
  MiniCalendar,
  MiniCalendarNavigation,
  MiniCalendarDays,
  MiniCalendarDay,
} from "@/components/blocks/mini-calendar"

export default function Demo() {
  return (
    <MiniCalendar>
      <MiniCalendarNavigation direction="prev" />
      <MiniCalendarDays>
        {(date) => (
          <MiniCalendarDay key={date.toISOString()} date={date} />
        )}
      </MiniCalendarDays>
      <MiniCalendarNavigation direction="next" />
    </MiniCalendar>
  )
}
```

### 受控模式

```tsx
import {
  MiniCalendar,
  MiniCalendarNavigation,
  MiniCalendarDays,
  MiniCalendarDay,
} from "@/components/blocks/mini-calendar"

export default function Demo() {
  const [selected, setSelected] = useState<Date>()

  return (
    <MiniCalendar value={selected} onValueChange={setSelected} days={7}>
      <MiniCalendarNavigation direction="prev" />
      <MiniCalendarDays>
        {(date) => <MiniCalendarDay key={date.toISOString()} date={date} />}
      </MiniCalendarDays>
      <MiniCalendarNavigation direction="next" />
    </MiniCalendar>
  )
}
```

## API

类型来自 `src/components/blocks/mini-calendar/index.tsx`。

### MiniCalendar

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| value | `Date` | - | 受控选中日期 |
| defaultValue | `Date` | - | 默认选中日期 |
| onValueChange | `(date: Date \| undefined) => void` | - | 选中日期变化回调 |
| startDate | `Date` | - | 受控起始展示日期 |
| defaultStartDate | `Date` | `new Date()` | 默认起始展示日期（当日） |
| onStartDateChange | `(date: Date \| undefined) => void` | - | 起始日期变化回调 |
| days | `number` | `5` | 展示的连续天数 |
| className | `string` | - | 附加 className |

支持 `HTMLAttributes<HTMLDivElement>` 的其余属性。

### MiniCalendarNavigation

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| direction | `"prev" \| "next"` | 必填 | 翻页方向 |
| asChild | `boolean` | `false` | 使用 Radix Slot 渲染为子元素 |

默认渲染为 `Button`（`size="small" shape="square" variant="text"`），children 为 ChevronLeftIcon/ChevronRightIcon。支持 `ComponentProps<typeof Button>` 的其余属性。

### MiniCalendarDays

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| children | `(date: Date) => ReactNode` | 必填 | render prop，为每一天渲染内容 |

支持 `Omit<HTMLAttributes<HTMLDivElement>, "children">` 的其余属性。

### MiniCalendarDay

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| date | `Date` | 必填 | 该按钮对应的日期 |

选中态自动 `color="primary" variant="solid"`；未选中为 `color="default" variant="text"`；今天未选中时附加 `bg-accent`。支持 `ComponentProps<typeof Button>` 的其余属性。

## 样式模型

- 样式由 Tailwind CSS className 与 `cn` 合并函数控制。
- 外层容器 `flex items-center gap-2 rounded-lg border bg-background p-2`。
- `MiniCalendarDay` 最小宽度 `min-w-[3rem]`，纵向 flex-col 显示月份（10px 灰色小字）和日期（14px 半粗体）。
- 不依赖 CVA、Motion 或 Base UI。主题颜色使用 `bg-background`、`text-muted-foreground`、`bg-accent`、`text-primary-foreground` 等语义变量。

## 依赖与实现说明

核心文件：

- `src/components/blocks/mini-calendar/index.tsx` — 组件实现
- `src/components/core/blocks/mini-calendar.tsx` — 重新导出

直接依赖：`@/components/core/button`、`@/lib/utils`、`@radix-ui/react-use-controllable-state`、`date-fns`、`lucide-react`、`radix-ui`（Slot）。

实现注意：

1. 使用 React Context 在子组件间共享选中日期、起始日期和翻页状态。
2. `selectedDate` 和 `currentStartDate` 两套状态通过 `useControllableState` 实现受控/非受控双模式。
3. 导航翻页时起始日期增减 `days` 数量。
4. `MiniCalendarDay` 底层为 `Button` 组件；日期格式化使用 `date-fns`（format/isSameDay/isToday/addDays）。

## 可访问性

- `MiniCalendarDay` 和 `MiniCalendarNavigation` 底层为 `<button>`，保留原生键盘交互。
- 选中/今天日期通过 `color`/`variant` 视觉区分，建议补充 `aria-current` 增强屏幕阅读器体验。
- 纯图标导航按钮需确保有可访问名称。

## 与 Ant Design 的取舍

该组件为项目本地复合块，无直接 Ant Design 对应。

| 方向 | 本地决策 |
| --- | --- |
| AntD 对应 | 无直接对应。AntD Calendar 为完整月历，本组件为轻量横向日期条 |
| 构建方式 | 基于 Button + Context 的四子组件组合模式 |
| API 来源 | 以 `src/components/blocks/mini-calendar/index.tsx` 类型为准 |
| 视觉系统 | Tailwind CSS + 项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |

## LLM 使用规则

1. 从 `@/components/blocks/mini-calendar` 导入 `MiniCalendar` 及其全部子组件。
2. 只使用本文件 API 表格和源码中存在的 props。
3. 使用组合模式：`MiniCalendar` > `MiniCalendarNavigation` + `MiniCalendarDays` + `MiniCalendarDay`。
4. `MiniCalendarDays` 的 children 必须为 `(date: Date) => ReactNode` render prop。
5. 受控模式使用 `value`/`onValueChange`；非受控使用 `defaultValue`。
6. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token。
7. 不确定某 props 是否存在时，检查 `src/components/blocks/mini-calendar/index.tsx`。
