# CursorFollow 光标跟随

CursorFollow 是一个光标跟随效果复合组件，由 CursorProvider 容器、Cursor 光标图形和 CursorFollow 跟随标签三部分组成。它封装了底层 cursor primitive（`@/components/_internal/animate/cursor`），提供简洁的 API：CursorProvider 管理光标区域，Cursor 渲染自定义 SVG 光标，CursorFollow 在光标附近显示跟随标签（如提示文字、状态信息）。

本组件为项目本地复合组件，无 Ant Design 直接对应。实现以本地源码为准。

## 何时使用 {#when-to-use}

当需要在页面或局部区域展示自定义光标效果，并在光标附近显示跟随标签时使用。

常见场景：

- 增强页面视觉交互，用自定义 SVG 图形替换系统默认光标。
- 在光标附近展示上下文信息，如悬停元素的名称或状态。
- 全局光标效果（`global` 模式），覆盖整个页面窗口。

不推荐：

- 纯功能性提示场景 —— 应优先使用 Tooltip。
- 性能敏感的低端设备上同时使用大量全局光标实例。
- 在调用侧绕过组件 API，直接操作底层 primitive 的 DOM 结构。

## 代码演示 {#examples}

### 基础用法

```tsx
import { CursorProvider, Cursor, CursorFollow } from "@/components/blocks/cursor-follow"

export default function Demo() {
  return (
    <CursorProvider className="w-full max-w-sm">
      <div className="relative flex h-32 items-center justify-center rounded border bg-muted/20 p-4">
        <p className="text-sm text-muted-foreground">移动鼠标查看光标跟随效果</p>
        <CursorFollow>
          <Cursor />
        </CursorFollow>
      </div>
    </CursorProvider>
  )
}
```

> 说明：`CursorProvider` 必须作为最外层容器包裹光标区域。`Cursor` 渲染默认 SVG 箭头光标图形，`CursorFollow` 作为跟随标签包裹 `Cursor`。二者通过 underlying primitive 共享光标位置上下文。

### 带标签内容

```tsx
import { CursorProvider, Cursor, CursorFollow } from "@/components/blocks/cursor-follow"

export default function Demo() {
  return (
    <CursorProvider className="w-full max-w-sm">
      <div className="relative flex h-32 items-center justify-center rounded border bg-muted/20 p-4">
        <p className="text-sm text-muted-foreground">悬停查看标签</p>
        <CursorFollow side="top" sideOffset={20}>
          <span className="text-xs">查看详情</span>
          <Cursor />
        </CursorFollow>
      </div>
    </CursorProvider>
  )
}
```

## API {#api}

以下类型来自 `src/components/blocks/cursor-follow/index.tsx` 源码，是生成代码时的主要依据。

### CursorProvider

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| global | 是否全局模式，跟踪整个窗口的指针位置 | `boolean` | `false` |
| className | 容器样式类 | `string` | - |

`CursorProvider` 继承 `Omit<CursorProviderPropsPrimitive, 'children'> & CursorContainerPropsPrimitive`。全局模式下光标使用 `fixed` 定位并跟踪全窗口指针；局部模式下光标使用 `absolute` 定位，跟踪父容器指针位置。内部组合了 `CursorProviderPrimitive` 和 `CursorContainerPrimitive`。

### Cursor

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 光标容器样式类，与默认 `size-6 text-foreground` 合并 | `string` | - |

`Cursor` 继承 `Omit<CursorPropsPrimitive, 'children' | 'asChild'>`。固定渲染为 `asChild` 模式，子元素为默认 SVG 箭头光标图形（viewBox="0 0 40 40"）。如需自定义光标图形，需修改源码中的 SVG path。

### CursorFollow

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| sideOffset | 跟随标签距离光标的间距（px） | `number` | `15` |
| alignOffset | 对齐偏移量（px） | `number` | `5` |
| className | 标签容器样式类，与默认 `bg-foreground rounded-md text-background px-2 py-1 text-sm` 合并 | `string` | - |
| children | 跟随标签内容 | `React.ReactNode` | - |

`CursorFollow` 继承 `Omit<CursorFollowPropsPrimitive, 'asChild'>`。固定渲染为 `asChild` 模式，子元素为 `<div>` 包裹的跟随标签。底层 primitive 通过 motion/react 弹簧动画控制标签位置，支持 `side`（top/right/bottom/left）、`align`（start/center/end）等定位属性（来自 primitive 类型，本地组件未覆盖默认值）。

## 样式模型 {#styling-model}

- 样式由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- Cursor 默认 `size-6 text-foreground`，渲染为 SVG 箭头光标图形。
- CursorFollow 默认 `bg-foreground rounded-md text-background px-2 py-1 text-sm`，深底白字跟随标签。
- 定位和动画由底层 cursor primitive（`@/components/_internal/animate/cursor`）的 motion/react 弹簧物理控制，本地组件仅负责视觉样式收敛。
- 主题颜色优先使用 `text-foreground`、`bg-foreground`、`text-background` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/cursor-follow/index.tsx`（复合组件定义）
- `src/components/core/blocks/cursor-follow.tsx`（re-export barrel）
- `src/primitives/animate/cursor.tsx`（底层 cursor primitive，基于 motion/react 实现定位和弹簧动画）

直接依赖摘要：

- React
- `@/components/_internal/animate/cursor`（CursorProviderPrimitive、CursorPrimitive、CursorFollowPrimitive、CursorContainerPrimitive 及类型）
- `@/lib/utils`（`cn` 样式合并）
- Tailwind CSS

实现注意：

1. `CursorProvider` 通过组合 `CursorProviderPrimitive` + `CursorContainerPrimitive` 简化外部用法，避免了分别使用的繁琐。
2. `Cursor` 固定使用 `asChild`，子元素为内嵌的 SVG 光标图形；如需自定义，修改 `Cursor` 组件源码中的 SVG path。
3. `CursorFollow` 固定使用 `asChild`，子元素为 `<div>` 跟随标签，通过 `className` 自定义样式。
4. 光标位置、隐藏原生光标、弹簧动画等核心逻辑完全由底层 primitive 处理，本组件仅做 API 收敛和默认样式注入。
5. 如需新增能力，应先修改该组件源码，再更新本 llm 文档。

## 可访问性 {#accessibility}

- 组件为非交互型视觉增强组件，不影响原生元素的键盘语义。
- 底层 primitive 通过 `pointerEvents: 'none'` 确保光标不会阻挡底层元素的指针事件。
- 全局模式下 primitive 提供了 `visibilitychange` 和 `pointerout` 事件处理，确保标签页切换时光标状态正确。
- 如需在跟随标签中放置交互内容，需移除对应元素的 `pointerEvents: 'none'` 样式（在 primitive 层处理）。
- 光标元素为纯装饰性内容，不影响辅助技术体验。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/blocks/cursor-follow/index.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| AntD 对应 | 无直接 Ant Design 对应项，为项目本地复合组件 |
| 动画引擎 | 底层使用 motion/react 弹簧物理，非 CSS transition / AntD 动画 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 CursorFollow 代码时，应遵守：

1. 从 `@/components/blocks/cursor-follow` 导入，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. `CursorProvider` 必须包裹所有 `Cursor` 和 `CursorFollow` 子组件，否则它们无法获取光标位置上下文。
4. `CursorFollow` 必须包含 `children` prop，放置跟随标签的内容（可以包含 `Cursor` 或其他自定义内容）。
5. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
6. 需要自定义样式时，优先使用组件暴露的 `className` props。
7. `Cursor` 和 `CursorFollow` 的 `side`、`align` 等定位属性由底层 primitive 提供，本地组件未在 API 层面覆盖默认值。
8. 不要删除或绕过 underlying primitive 的弹簧动画；这是核心交互。
9. 不要引入 AntD ConfigProvider 或 rc-component。
10. 如果不确定某个 API 是否存在，先检查 `src/components/blocks/cursor-follow/index.tsx`，不要凭 AntD 记忆生成。
