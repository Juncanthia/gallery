# FileUploader 文件上传

FileUploader 是一个文件上传组件，提供带图标簇的拖拽上传区域。支持拖拽和点击两种选择方式，选择文件后展示文件列表，每个条目显示文件类型徽标、文件名、大小和类型信息，并支持移除操作。

本组件是 Manifest 页面模板级别的文件上传封装，使用 manifest-ui 基础组件和 lucide-react 图标，提供直观的拖拽交互体验。

## 何时使用 {#when-to-use}

当需要用户上传文件（文档、图片、表格等）时使用。上传区域提供明确的视觉引导，支持拖拽操作。

常见场景：

- 表单附件上传。
- 文档提交入口。
- 图片批量上传。
- 简历或资料文件收集。

不推荐：

- 需要上传进度条展示——本组件为前端文件选择组件，不包含实际上传逻辑。
- 需要预览文件内容——本组件只展示文件名和大小。
- 极简场景仅需一个 `<input type="file" />` 时——本组件提供完整的拖拽区风格。

## 代码演示 {#examples}

### 基础用法

默认展示拖拽上传区域，支持点击或拖拽选择文件。选择后下方显示文件列表。

```tsx
import { FileUploader } from "@/components/data-entry/blocks/file-uploader"

export default function Demo() {
  return (
    <div className="w-full max-w-xl">
      <FileUploader />
    </div>
  )
}
```

## API {#api}

### FileUploader

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| accept | 接受的文件类型（MIME 或扩展名），逗号分隔 | `string` | 常见文档和图片格式 |
| acceptedFileTypes | 可接受文件类型图标配置 | `AcceptedFileType[]` | `DEFAULT_ACCEPTED_FILE_TYPES` |
| browseLabel | 浏览按钮文字 | `string` | `"Browse files"` |
| className | 自定义样式类名 | `string` | — |
| description | 上传区描述文字 | `string` | `"PDF, DOC/DOCX, XLSX, CSV, PNG, JPG files are supported"` |
| draggingLabel | 拖拽时提示文字 | `string` | `"Drop to add"` |
| multiple | 是否允许多文件上传 | `boolean` | `true` |
| showFileList | 是否显示文件列表 | `boolean` | `true` |
| title | 上传区标题 | `string` | `"Click to upload or drop files"` |
| onFilesAccepted | 文件接受后的回调，接收原始 File 数组 | `(files: File[]) => void` | — |
| onFilesChange | 文件变更回调，接收含 id/name/type/size/url 的条目 | `(files: FileUploaderItem[]) => void` | — |

### FileUploaderItem

| 字段 | 说明 | 类型 |
| --- | --- | --- |
| id | 文件唯一标识 | `string` |
| name | 文件名 | `string` |
| type | MIME 类型 | `string` |
| size | 文件大小（字节） | `number` |
| url | 文件 Blob URL，用于预览 | `string` |

### AcceptedFileType

| 字段 | 说明 | 类型 |
| --- | --- | --- |
| label | 类型标签 | `string` |
| icon | 对应图标组件 | `typeof FileImage` |
| extensions | 扩展名列表 | `string[]` |

## Semantic DOM {#semantic-dom}

FileUploader 的上传区域使用 `role="button"` 实现键盘可操作性，支持 Enter/Space 键触发文件选择。文件列表区域使用标准 div 布局，每个文件条目包含文件名、类型、大小和移除按钮。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/core/manifest-blocks/file-uploader.tsx`：组件导出入口，re-export。
- `src/components/manifest/form/blocks/file-uploader.tsx`：FileUploader 组件完整实现。

依赖：

- `@hyper/manifest-ui/components/ui/card` — 图标簇容器。
- `@hyper/manifest-ui/components/ui/button` — 文件移除按钮。
- `@hyper/manifest-ui/lib/utils` — `cn` 工具函数。
- `lucide-react` — 文件类型图标（FileImage、FileText、FileSpreadsheet）。

关键行为：

1. 拖拽区域通过 `dragDepthRef` 追踪嵌套拖拽层级，防止子元素触发的误退出。
2. `matchesAccept` 函数同时支持扩展名（`.pdf`）、MIME 类型和通配符（`image/*`）三种匹配模式。
3. 选择新文件时会替换当前文件列表（非追加），并通过 `URL.revokeObjectURL` 清理旧的 Blob URL。
4. 每个文件条目有独立的移除按钮，点击时清理对应的 Blob URL。
5. 组件卸载时自动清理所有 Blob URL，防止内存泄漏。

## 可访问性 {#accessibility}

- 上传区域设置 `role="button"` 和 `tabIndex={0}`，支持键盘操作。
- Enter/Space 键触发文件选择对话框。
- 拖拽区域通过 `onDragOver` 阻止默认行为，确保正确的拖放体验。
- 文件列表移除按钮设置 `aria-label`。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| 拖拽上传 | 原生 Drag & Drop API，无额外依赖 |
| 图标 | lucide-react，不使用 @ant-design/icons |
| 上传进度 | 不做——本组件为纯前端文件选择，不上传 |
| 文件预览 | 不做——只显示文件名和大小 |
| beforeUpload 钩子 | 不做——通过 onFilesChange 在外层处理 |
| itemRender 自定义 | 不做——通过 showFileList 控制显隐 |

## LLM 使用规则 {#llm-rules}

当 AI 生成 FileUploader 相关代码时，应遵守：

1. 从 `@/components/data-entry/blocks/file-uploader` 导入 `FileUploader`。
2. 通过 `onFilesChange` 回调获取用户选择的文件列表，每个文件含 `id`、`name`、`type`、`size`、`url`。
3. 限制文件类型使用 `accept` 属性，字符串格式，逗号分隔。
4. 单文件上传设置 `multiple={false}`。
5. 自定义文案通过 `title`、`description`、`browseLabel`、`draggingLabel` 属性。
6. 不要引入 `antd` 的 `Upload` 或 `Dragger` 组件。
7. 不要使用 `@ant-design/icons` 图标——使用 lucide-react。
8. 上传逻辑由使用者自行实现，组件只负责文件选择和展示。
