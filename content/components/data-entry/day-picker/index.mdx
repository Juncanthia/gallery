DayPicker 日期选择器。一个带有动画切换的频率选择器，支持按天、周、月、年切换周期，选择 Weekly 时展开星期选择面板。

<h2 id="when-to-use">何时使用</h2>

当需要让用户选择时间周期频率（日/周/月/年）并辅助日期选择时使用。

常见场景：

- 任务调度器的频率设置。
- 报表周期选择。
- 提醒或通知的重复规则设置。
- 数据看板的时间粒度切换。

组件提供两种交互状态：

- **频率选择**：点击当前选项展开 Daily / Weekly / Monthly / Yearly 四个选项，带有 smooth spring 动画和 layoutId 共享元素过渡。
- **星期选择**：选择 Weekly 后下方展开 Sun-Sat 七天选择面板，每项有交错入场动画，选中项带次级背景高亮。

不推荐：

- 将本组件嵌入需要精确日期范围（起止日期）的场景——请使用 DatePicker。
- 在业务侧手写动画或自行引入 motion/react 覆盖组件内置动效。

<h2 id="api">API</h2>

DayPicker 为自包含组件，无需外部 props 即可运行。内部状态包括：

| 状态 | 类型 | 说明 |
| --- | --- | --- |
| option | `"Daily"` \| `"Weekly"` \| `"Monthly"` \| `"Yearly"` | 当前选中的频率周期 |
| day | `number`（0-6 对应 Sun-Sat） | 当 option 为 Weekly 时选中的星期 |
| isOptionOpen | `boolean` | 频率选项面板是否展开 |
| isSelectorOpen | `boolean` | 星期选择面板是否可见 |

所有交互行为由组件内部控制，调用侧无需传递外部状态。

<h2 id="faq">常见问题</h2>

### 如何在外部获取选中的频率和星期？

当前版本为演示组件，内部状态未暴露。如需外部控制，可修改源码将 `useState` 替换为受控 props（`option` / `onOptionChange` / `day` / `onDayChange`）。

### 如何修改可选频率选项？

修改源码中的 `options` 数组：

```tsx
const options: options[] = ["Daily", "Weekly", "Monthly", "Yearly"];
```

增删数组元素即可。类型别名 `type options` 也需同步更新。

### 星期名称如何国际化？

修改源码中的 `days` 数组：

```tsx
const days = ["Sun", "Mon", "Tue", "Wed", "Thu", "Fri", "Sat"];
```

替换为所需的语言即可，例如中文 `["日", "一", "二", "三", "四", "五", "六"]`。
