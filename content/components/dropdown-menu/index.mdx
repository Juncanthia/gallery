下拉菜单 DropdownMenu。在触发元素下方弹出下拉菜单列表，基于 Radix DropdownMenu 实现。支持声明式 items 配置、子菜单、复选框/单选项、快捷键标注、危险项样式和 12 种弹出位置。

<h2 id="when-to-use">何时使用</h2>

当需要为某个交互元素提供一组操作选项时使用。

常见场景：

- 用户头像/名称的下拉菜单（个人资料、设置、退出登录）。
- 表格每行的"更多操作"菜单（编辑、复制、删除）。
- 工具栏中"新建"等可展开多个选项的按钮。
- 顶部导航栏中带下拉的辅助菜单。

不推荐：

- 简单的确认弹出（如"确定删除？"），使用 Popconfirm 更轻量。
- 右键触发的上下文菜单，应使用 ContextMenu。
- 需要选择大量选项且有搜索需求时，使用 Combobox（SearchSelect）。

### 基础用法（子组件模式）

<ComponentShowcase id="dropdown-menu" path="dropdown-menu/basic" />

```tsx
import { Button } from "@/components/ui/button"
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuLabel,
  DropdownMenuSeparator,
  DropdownMenuShortcut,
  DropdownMenuTrigger,
} from "@/components/ui/dropdown-menu"

export default function Demo() {
  return (
    <DropdownMenu>
      <DropdownMenuTrigger asChild>
        <Button variant="outlined">打开菜单</Button>
      </DropdownMenuTrigger>
      <DropdownMenuContent className="w-48">
        <DropdownMenuLabel>我的账户</DropdownMenuLabel>
        <DropdownMenuSeparator />
        <DropdownMenuItem>个人资料</DropdownMenuItem>
        <DropdownMenuItem>设置</DropdownMenuItem>
        <DropdownMenuItem>
          快捷键
          <DropdownMenuShortcut>Ctrl+K</DropdownMenuShortcut>
        </DropdownMenuItem>
        <DropdownMenuSeparator />
        <DropdownMenuItem variant="destructive">退出登录</DropdownMenuItem>
      </DropdownMenuContent>
    </DropdownMenu>
  )
}
```

### items 声明式配置

使用 `items` + `trigger` 属性，一行声明完整菜单。

```tsx
import { DropdownMenu } from "@/components/ui/dropdown-menu"

export default function Demo() {
  return (
    <DropdownMenu
      trigger={<Button variant="outlined">文件操作</Button>}
      placement="bottomLeft"
      items={[
        { type: "label", label: "文件操作" },
        { type: "separator" },
        { label: "新建文件", shortcut: "Ctrl+N", onSelect: () => console.log("new") },
        { label: "打开文件", shortcut: "Ctrl+O" },
        { type: "separator" },
        { label: "删除", danger: true, onSelect: () => console.log("delete") },
      ]}
    />
  )
}
```

### 自定义弹出位置

```tsx
import { DropdownMenu } from "@/components/ui/dropdown-menu"

export default function Demo() {
  return (
    <div className="flex gap-2">
      <DropdownMenu
        trigger={<Button variant="outlined" size="sm">左下</Button>}
        placement="bottomLeft"
        items={[{ label: "选项一" }, { label: "选项二" }]}
      />
      <DropdownMenu
        trigger={<Button variant="outlined" size="sm">右下</Button>}
        placement="bottomRight"
        items={[{ label: "选项一" }, { label: "选项二" }]}
      />
      <DropdownMenu
        trigger={<Button variant="outlined" size="sm">右上</Button>}
        placement="topRight"
        items={[{ label: "选项一" }, { label: "选项二" }]}
      />
    </div>
  )
}
```

<h2 id="api">API</h2>

### DropdownMenu

顶层容器组件。提供两种使用模式：**子组件模式**（手动组合子组件）和**声明式 items 模式**。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| trigger | 触发器元素 | `React.ReactNode` | - |
| items | 声明式菜单项配置 | `DropdownMenuItemOption[]` | - |
| menu | `items` 的别名包裹对象 | `{ items?: DropdownMenuItemOption[] }` | - |
| placement | 弹出位置 | `DropdownMenuPlacement` | `"bottomLeft"` |
| contentProps | 传给 Content 的额外属性 | `DropdownMenuContentProps` | - |
| open | 受控展开状态 | `boolean` | - |
| defaultOpen | 非受控默认展开 | `boolean` | - |
| onOpenChange | 展开状态变化回调 | `(open: boolean) => void` | - |

### DropdownMenuPlacement

`"top"` | `"bottom"` | `"topLeft"` | `"topRight"` | `"bottomLeft"` | `"bottomRight"` | `"left"` | `"leftTop"` | `"leftBottom"` | `"right"` | `"rightTop"` | `"rightBottom"`

### DropdownMenuItemOption

声明式菜单项的联合类型：

| type | 关键属性 | 说明 |
| --- | --- | --- |
| `"item"`（默认） | `label`, `icon?`, `shortcut?`, `extra?`, `danger?`, `disabled?`, `onSelect?` | 普通菜单项 |
| `"label"` | `label`, `inset?` | 分组标题 |
| `"separator"` | - | 分隔线 |
| `"checkbox"` | `label`, `checked?`, `disabled?`, `onCheckedChange?` | 复选菜单项 |
| `"submenu"` | `label`, `icon?`, `extra?`, `disabled?`, `children` | 二级子菜单 |

### 子组件

| 组件 | 说明 | 关键属性 |
| --- | --- | --- |
| `DropdownMenuTrigger` | 触发器 | `asChild` |
| `DropdownMenuContent` | 菜单内容面板 | `sideOffset`, `align`, `side` |
| `DropdownMenuItem` | 普通菜单项 | `inset`, `variant` ("default" \| "destructive"), `disabled` |
| `DropdownMenuLabel` | 分组标题 | `inset` |
| `DropdownMenuSeparator` | 分隔线 | - |
| `DropdownMenuGroup` | 菜单分组容器 | - |
| `DropdownMenuShortcut` | 快捷键文字标注 | - |
| `DropdownMenuCheckboxItem` | 带勾选的复选菜单项 | `checked`, `disabled`, `onCheckedChange` |
| `DropdownMenuRadioGroup` | 单选组容器 | `value`, `onValueChange` |
| `DropdownMenuRadioItem` | 单选项 | `value`, `disabled` |
| `DropdownMenuSub` | 子菜单容器 | - |
| `DropdownMenuSubTrigger` | 子菜单触发器 | `inset`, `disabled` |
| `DropdownMenuSubContent` | 子菜单内容面板 | - |

<h2 id="faq">常见问题</h2>

### 如何实现危险操作（如删除）？

- **子组件模式**：设置 `variant="destructive"`，菜单项文字变红。
- **items 模式**：设置 `danger: true`。

两类模式均会以红色文本渲染该项，hover 时有红色背景高亮。

### DropdownMenu 和 ContextMenu 的区别？

- **DropdownMenu**：左键点击触发，用于一般操作菜单。
- **ContextMenu**：右键触发，用于上下文相关操作（复制、粘贴等）。

### 如何在菜单项上添加图标和快捷键？

- **子组件模式**：直接在 `DropdownMenuItem` 内放置图标元素和 `DropdownMenuShortcut`。
- **items 模式**：设置 `icon` 和 `shortcut` 字段。

### 如何控制弹出位置？

使用 `placement` 属性，支持 12 种位置组合。组件内部将 `placement` 映射为 Radix 的 `side` 和 `align` 属性。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/dropdown-menu` 导入所需的子组件。
2. `DropdownMenuTrigger` 使用 `asChild` 包裹 Button，避免多余的 DOM 嵌套。
3. 使用 `DropdownMenuSeparator` 分隔不同功能组的操作项。
4. 危险操作（如删除）使用 `variant="destructive"` 或 `danger: true`。
5. 简单场景优先使用 `items` + `trigger` 声明式写法。
6. 不要引入 Ant Design 的 `Dropdown` 或 `antd` 相关依赖。
