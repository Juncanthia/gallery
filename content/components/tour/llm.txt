# Tour 引导漫游

页面引导漫游（Onboarding Tour）组件，通过遮罩层高亮目标元素，并弹出步骤卡片引导用户完成多步操作熟悉流程。使用 `createPortal` 渲染至 `document.body`，结合 `motion/react` 的 `AnimatePresence` 和 Floating UI 实现卡片动画、定位、碰撞规避和滚动更新。

本组件为项目本地复合组件，功能类似于 Ant Design Tour 但实现独立。以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

当需要对用户进行产品功能引导、新功能介绍或操作流程指引时使用。

常见场景：

- 新用户首次进入页面的功能导览。
- 新功能上线后的亮点介绍。
- 复杂表单或仪表盘的分步操作指导。
- 需要视觉聚焦特定 UI 区域并附带文字说明的引导场景。

不推荐：

- 简单的单条提示——使用 `Tooltip` 或 `Popover`。
- 需要用户在引导过程中执行复杂交互（如填写表单）——Tour 仅提供线性浏览模式。
- 在移动端小屏幕上使用（卡片固定 288px 宽，遮罩效果在小屏体验不佳）。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { useState } from "react"
import { Tour, type TourStepConfig } from "@/components/blocks/tour"

const steps: TourStepConfig[] = [
  {
    title: "欢迎",
    description: "这是新功能介绍",
    placement: "center",
  },
  {
    title: "搜索框",
    description: "在这里搜索你需要的内容",
    target: "#search-input",
    placement: "bottom",
  },
  {
    title: "设置",
    description: "在这里调整你的偏好",
    target: "#settings-panel",
    placement: "right",
  },
]

export default function Demo() {
  const [open, setOpen] = useState(false)

  return (
    <>
      <button onClick={() => setOpen(true)}>开始引导</button>
      <Tour
        open={open}
        onClose={() => setOpen(false)}
        onFinish={() => setOpen(false)}
        onSkip={() => setOpen(false)}
        steps={steps}
        mask
        arrow
        gap={{ offset: 10, radius: 4 }}
      />
    </>
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。`steps` 为必填；`target` 可以是 CSS 选择器、ref、HTMLElement 或函数；完整用法参考下方 API。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

### Tour

```ts
type TourProps = {
  /** 是否显示引导 */
  open?: boolean; // 默认 false
  /** 关闭回调 */
  onClose?: () => void;
  /** 完成最后一步回调 */
  onFinish?: () => void;
  /** 跳过或提前关闭回调 */
  onSkip?: () => void;
  /** 当前步骤索引（受控） */
  current?: number;
  /** 步骤变更回调 */
  onChange?: (current: number) => void;
  /** 默认起始步骤（非受控），默认 0 */
  defaultCurrent?: number;
  /** 步骤配置数组，必填 */
  steps: TourStepConfig[];
  /** 是否显示遮罩层，默认 true */
  mask?: boolean;
  /** 遮罩 z-index，默认 1001 */
  zIndex?: number;
  /** 目标元素的间距与圆角配置 */
  gap?: { offset?: number; radius?: number }; // 默认 { offset: 10, radius: 0 }
  /** 是否显示箭头，默认 false */
  arrow?: boolean;
  /** 切换步骤时是否自动滚动到目标，默认 true */
  autoScroll?: boolean;
  /** 自动滚动时目标距离视窗边缘的最小间距，默认 80 */
  scrollOffset?: number;
  /** 目标元素滚动到视图的选项 */
  scrollIntoViewOptions?: ScrollIntoViewOptions;
  /** 是否按 Escape 关闭，默认 true */
  closeOnEscape?: boolean;
  /** 是否点击遮罩关闭，默认 false */
  closeOnOutsideClick?: boolean;
  /** 卡片的附加 className */
  className?: string;
}
```

`Tour` 本身不渲染任何可见 UI（在 `open=false` 或无有效步骤时返回 `null`）。所有可见内容通过 `createPortal` 渲染到 `document.body`。

### TourStepConfig

```ts
type TourStepConfig = {
  /** 步骤标题 */
  title?: ReactNode;
  /** 步骤描述 */
  description?: ReactNode;
  /** 目标元素，返回 null 或不传时卡片居中显示 */
  target?: string | HTMLElement | RefObject<HTMLElement | null> | (() => HTMLElement | null);
  /** 卡片相对于目标的位置，默认 "bottom" */
  placement?: "top" | "bottom" | "left" | "right" | "top-start" | "top-end" | "bottom-start" | "bottom-end" | "left-start" | "left-end" | "right-start" | "right-end" | "center";
  /** 步骤封面图/内容 */
  cover?: ReactNode;
  /** 下一步按钮的自定义属性 */
  nextButtonProps?: { children?: ReactNode; className?: string };
  /** 上一步按钮的自定义属性 */
  prevButtonProps?: { children?: ReactNode; className?: string };
  /** 本步骤关闭时的回调 */
  onClose?: () => void;
  /** 进入本步骤回调 */
  onStepEnter?: () => void;
  /** 离开本步骤回调 */
  onStepLeave?: () => void;
}
```

### 步骤导航行为

- 第一步：只显示"下一步"按钮。
- 中间步骤：显示"上一步"和"下一步"按钮。
- 最后一步：显示"上一步"和"完成"按钮，点击"完成"触发 `onFinish` 和 `onClose`。
- 步骤指示器：底部圆点，激活步为加宽的主色圆点（`bg-primary w-3`），其余为 `bg-muted-foreground/30 h-1.5 w-1.5`。

### 卡片定位

- 非 `center` placement 使用 Floating UI 的 `offset`、`shift`、`flip` 和 `arrow` middleware。
- 目标不存在或 placement 为 `center` 时，卡片居中显示。
- 窗口 resize、滚动和目标元素位置变化会自动重新计算卡片位置和高亮区域。

## Semantic DOM {#semantic-dom}

源码中使用的稳定语义槽位：

- `data-slot="tour"` — 引导漫游根容器
- `data-slot="tour-mask"` — 遮罩层
- `data-slot="tour-spotlight"` — 目标高亮区域
- `data-slot="tour-card"` — 步骤卡片
- `data-slot="tour-arrow"` — 箭头
- `data-slot="tour-title"` — 标题
- `data-slot="tour-description"` — 描述
- `data-slot="tour-footer"` — 底部导航区
- `data-slot="tour-indicator"` — 步骤指示器

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 组件通过 `createPortal` 渲染到 `document.body`，避免 z-index 和 overflow 限制。
- 遮罩层：`fixed inset-0 bg-black/40`，默认不响应外部点击，设置 `closeOnOutsideClick` 后可点击关闭。
- 镂空高亮：通过 `boxShadow` 以目标元素宽高最大值的 2 倍为扩散半径实现（`0 0 0 [max(w,h)*2]px rgba(0,0,0,0.4)`）。
- 步骤卡片：`fixed bg-popover border border-border rounded-lg shadow-xl p-5 w-72`（固定 288px 宽），非居中模式由 Floating UI 定位。
- 卡片进出场动画：`motion.div` + `AnimatePresence`，初始 `opacity: 0, scale: 0.95`，0.2s 过渡。
- 步骤指示器：未激活 `bg-muted-foreground/30 h-1.5 w-1.5 rounded-full`，激活 `bg-primary w-3`。
- 主题颜色使用 `bg-popover`、`bg-primary`、`text-foreground`、`text-muted-foreground`、`border-border` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/tour/index.tsx`
- `src/components/ui/tour.tsx`（barrel 导出）

直接依赖摘要：

- `react` / `react-dom` — `useState`、`useEffect`、`createPortal`
- `@floating-ui/react-dom` — 卡片定位、碰撞规避、自动更新和箭头坐标
- `motion/react` — `motion.div`、`AnimatePresence`
- `@/components/ui/button` — 步骤导航按钮
- `@/lib/utils` — `cn` 样式合并工具

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 受控模式下 `current` 由外部管理；非受控模式使用内部 `useState`。
3. 步骤切换时默认自动滚动目标元素，可通过 `autoScroll={false}` 关闭。
4. 定位由 Floating UI 负责，窗口 resize 和滚动会自动重新计算。
5. 镂空遮罩通过半透明遮罩层 + boxShadow 高亮区域实现。
6. 默认不支持遮罩层点击关闭；设置 `closeOnOutsideClick` 后会触发 `onSkip` 和 `onClose`。
7. `arrow` 为真实箭头渲染能力。
8. 步骤指示器数量与 `steps` 数组长度相同。
9. 如果需要新增能力，应先修改 `src/components/blocks/tour/index.tsx`，再更新本 llm 文档。

## 可访问性 {#accessibility}

- 步骤标题和描述通过 `font-semibold` + `text-muted-foreground` 区分层级。
- 导航按钮为项目本地 Button 组件，保留键盘语义。
- 遮罩层默认不拦截底层元素交互；设置 `closeOnOutsideClick` 后会接收点击。
- 步骤切换时自动 `scrollIntoView` 将目标滚动到可视区域。
- 当前版本**不包含**焦点陷阱（focus trap）——引导期间用户仍可 Tab 到遮罩后方元素。
- 步骤卡片带 `role="dialog"`，Escape 默认关闭，可通过 `closeOnEscape={false}` 禁用。

## 与 Ant Design 的取舍 {#antd-differences}

该组件功能类似 Ant Design Tour，但独立实现，基于 motion/react 和原生 Portal。

| 方向 | 本地决策 |
| --- | --- |
| 组件来源 | 项目本地复合，非 AntD 封装 |
| API 来源 | 以 `src/components/blocks/tour/index.tsx` 的 TypeScript 类型为准 |
| 动画方案 | 使用 motion/react AnimatePresence，非 AntD rc-motion |
| 定位方案 | 使用 Floating UI，非 AntD rc-trigger |
| 遮罩方案 | boxShadow 镂空技术，非 SVG mask |
| 焦点管理 | 不包含焦点陷阱 |
| 步骤指示器 | 圆点样式，非 AntD 数字指示器 |
| 组合模式 | 单一 `Tour` 组件，不可拆分子组件 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Tour 代码时，应遵守：

1. 从 `@/components/blocks/tour` 导入 `Tour` 和 `TourStepConfig`，不要从 `antd` 导入。
2. `steps` 为必填属性，每个 step 至少需提供 `title` 或 `description` 之一。
3. `target` 可以是 CSS 选择器、HTMLElement、ref，或返回 `HTMLElement | null` 的函数。
4. 受控模式下同时传入 `current` 和 `onChange`；非受控模式下使用 `defaultCurrent`，二者不可混用。
5. 组件通过 `open` / `onClose` 控制显隐，不要在组件外条件渲染 `{open && <Tour />}`。
6. `placement` 支持 `"top"`、`"bottom"`、`"left"`、`"right"` 及 `*-start`、`*-end`，也支持 `"center"`。
7. `mask` 默认为 `true`，设为 `false` 可去掉遮罩层。
8. 只使用本文件 API 摘要和源码中存在的 props。
9. 不要生成 AntD 专属的 `type`、`indicatorsRender`、`maskClassName`、`classNames`、`styles`、`ConfigProvider` 等属性。
10. 引导期间如需阻断用户其他操作，应在调用侧添加焦点陷阱或模态逻辑。
11. 如果不确定某个 API 是否存在，先检查 `src/components/blocks/tour/index.tsx`，不要凭 AntD 记忆生成。
