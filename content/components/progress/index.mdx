Progress 进度条。展示操作的当前进度，支持条形（line）、圆形（circle）、仪表盘（dashboard）和步骤条（steps）四种形态，可配置颜色、状态、渐变、段数和百分比位置。

<h2 id="when-to-use">何时使用</h2>

当需要展示操作或任务的当前进度时使用。

常见场景：
- 文件上传/下载进度展示
- 表单填写完成度指示
- 任务执行进度（如数据导入、渲染进度）
- 仪表盘样式展示（如完成率、评分）
- 分步操作进度（步骤条模式）

不推荐：
- 需要交互的进度控制——使用 `Slider`
- 加载中的不确定等待状态——使用 `Spinner` 或 `Skeleton`
- 为了照搬 Ant Design Progress API 而引入本组件——API 以本地源码为准

<h2 id="api">API</h2>

### Progress

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| percent | 进度百分比（0-100） | `number` | `0` |
| showInfo | 是否显示进度数值或状态图标 | `boolean` | `false` |
| format | 自定义进度文字内容 | `(percent: number, successPercent?: number) => ReactNode` | `percent + "%"` |
| status | 进度状态，影响颜色和图标 | `"normal" \| "exception" \| "active" \| "success"` | 自动根据 percent 判断 |
| type | 进度条类型 | `"line" \| "circle" \| "dashboard"` | `"line"` |
| strokeColor | 进度条颜色 | `string \| string[] \| ProgressGradient` | - |
| railColor | 轨道背景色 | `string` | - |
| trailColor | 轨道背景色（railColor 的别名） | `string` | - |
| strokeWidth | 进度条线宽（条形时为高度，圆形时为描边宽度） | `number` | - |
| strokeLinecap | 进度条端点样式 | `"butt" \| "square" \| "round"` | `"round"` |
| size | 尺寸预设或自定义 | `"small" \| "default" \| "middle" \| "large" \| number \| [number \| string, number] \| { width?: number; height?: number }` | - |
| steps | 步骤条配置（设置后 type 自动变为步骤条） | `number \| { count: number; gap?: number }` | - |
| success | 子进度段配置 | `{ percent?: number; strokeColor?: string }` | - |
| percentPosition | 百分比位置配置（仅 line 类型） | `{ align?: "start" \| "center" \| "end"; type?: "inner" \| "outer" }` | - |
| gapDegree | 仪表盘缺口角度（仅 dashboard 类型） | `number` | `75` |
| indicatorClassName | 进度指示器的附加 className | `string` | - |
| className | 根元素附加 CSS 类名 | `string` | - |
| children | 自定义信息节点（优先级高于 format） | `ReactNode` | - |

状态颜色映射：
| status | 颜色 |
| --- | --- |
| normal | `var(--primary)` |
| active | `var(--primary)` + `animate-pulse` |
| success | `#22c55e`（绿色）+ ✓ 图标 |
| exception | `var(--destructive)`（红色）+ × 图标 |

<h2 id="faq">常见问题</h2>

### percent 和 value 有什么区别？

两者功能相同，都是设置进度百分比。`percent` 是本组件的扩展属性，`value` 是从 Radix Progress 继承的原生属性。推荐使用 `percent`。

### 如何实现渐变色进度条？

通过 `strokeColor` 传入 `ProgressGradient` 对象（`{ from, to }` 或百分比点位映射），或传入颜色字符串数组（自动生成 `linear-gradient`）。

### 步骤条（steps）模式如何配置？

设置 `steps` 属性即可自动切换到步骤条渲染。可传入数字（指定步数，默认间距 4px）或对象 `{ count, gap }` 精细控制。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/progress` 导入，不要从 `antd` 导入。
2. 只使用本文件 API 表格和源码中存在的 props，不要凭 AntD Progress 记忆生成 API。
3. `percent` 取值范围 0-100，组件内部会自动 clamp。
4. `status` 不传时自动根据 `percent >= 100` 判断是否为 `"success"`。
5. `strokeColor` 支持三种格式：字符串（单色）、字符串数组（渐变色）、`ProgressGradient` 对象（from/to 或百分比点位）。
6. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、`strokeColor` 作为对象时的 `strokeColor["0%"]` 写法外的非标准格式。
7. 不要引入 Ant Design 相关依赖。
