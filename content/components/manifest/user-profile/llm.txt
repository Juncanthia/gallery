# UserProfile 用户资料

UserProfile 是一个用户个人资料的展示与编辑组件，适用于用户中心、账号设置等场景。组件支持查看和编辑模式切换，覆盖头像、姓名、邮箱、简介、电话、所在地、个人网站等常用资料字段。

本组件是页面模板级别的复合组件，内部组合了 Avatar、Input、Label、Button 等基础组件，提供开箱即用的个人资料页面体验。

## 何时使用 {#when-to-use}

用于展示和编辑用户个人资料。适合以下场景：

- 用户中心的个人资料页面，提供信息查看和编辑入口。
- 账号设置中的基本信息管理，支持头像上传和字段修改。
- 社交应用的用户主页，展示用户公开信息。
- SaaS 产品的个人资料设置页。

## 代码演示 {#examples}

### 基础用法

默认展示用户头像、姓名、邮箱和个人简介，点击"Edit"按钮进入编辑模式，修改后点击"Save Changes"保存。

```tsx
import { UserProfile } from "@/components/templates/manifest/form/user-profile"

export default function UserProfileDemo() {
  return <UserProfile />
}
```

### 处理保存操作

通过 `actions.onSave` 回调接收更新后的用户数据，用于调用 API 持久化。

```tsx
import { UserProfile } from "@/components/templates/manifest/form/user-profile"

export default function UserProfileSaveDemo() {
  return (
    <UserProfile
      actions={{
        onSave: (data) => console.log("Profile saved:", data),
        onCancel: () => console.log("Edit cancelled"),
      }}
    />
  )
}
```

### 异步提交与加载状态

通过 `control.isLoading` 控制保存按钮的加载状态。

```tsx
import { useState } from "react"
import { UserProfile } from "@/components/templates/manifest/form/user-profile"

export default function UserProfileLoadingDemo() {
  const [loading, setLoading] = useState(false)

  async function handleSave(data: Parameters<NonNullable<React.ComponentProps<typeof UserProfile>["actions"]>>[0]["onSave"] extends (d: infer D) => void ? D : never) {
    setLoading(true)
    await new Promise((resolve) => setTimeout(resolve, 2000))
    setLoading(false)
  }

  return (
    <UserProfile
      actions={{ onSave: handleSave }}
      control={{ isLoading: loading }}
    />
  )
}
```

### 自定义文案

通过 `data` prop 自定义标题、编辑按钮、保存按钮和取消按钮文字。

```tsx
import { UserProfile } from "@/components/templates/manifest/form/user-profile"

export default function UserProfileCustomDemo() {
  return (
    <UserProfile
      data={{
        title: "个人资料",
        editLabel: "编辑资料",
        saveLabel: "保存修改",
        cancelLabel: "放弃修改",
      }}
    />
  )
}
```

### 只读模式

设置 `appearance.editable` 为 `false`，禁用编辑功能，仅展示用户信息。

```tsx
import { UserProfile } from "@/components/templates/manifest/form/user-profile"

export default function UserProfileReadOnlyDemo() {
  return <UserProfile appearance={{ editable: false }} />
}
```

## API {#api}

### UserProfile

通过 `data`、`actions`、`appearance`、`control` 四个顶层 prop 配置组件行为。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 资料数据配置 | `object` | — |
| data.title | 页面标题 | `string` | `"Profile"` |
| data.user | 用户资料数据 | `UserData` | — |
| data.editLabel | 编辑按钮文字 | `string` | `"Edit"` |
| data.saveLabel | 保存按钮文字 | `string` | `"Save Changes"` |
| data.cancelLabel | 取消按钮文字 | `string` | `"Cancel"` |
| actions | 回调配置 | `object` | — |
| actions.onSave | 保存回调，接收更新后的 `UserData` | `(data: UserData) => void` | — |
| actions.onCancel | 取消编辑回调 | `() => void` | — |
| actions.onAvatarChange | 头像变更回调，接收文件对象 | `(file: File) => void` | — |
| appearance | 外观配置 | `object` | — |
| appearance.showTitle | 是否显示标题区域 | `boolean` | `true` |
| appearance.showAvatar | 是否显示头像区域 | `boolean` | `true` |
| appearance.editable | 是否允许编辑 | `boolean` | `true` |
| control | 控制配置 | `object` | — |
| control.isLoading | 保存按钮加载状态；为 `true` 时按钮禁用并显示加载状态 | `boolean` | `false` |

### UserData

保存时 `onSave` 回调接收的数据结构。

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| name | 用户姓名 | `string` | 否 |
| email | 电子邮箱地址 | `string` | 否 |
| bio | 个人简介 | `string` | 否 |
| avatar | 头像 URL | `string` | 否 |
| phone | 电话号码 | `string` | 否 |
| location | 所在地 | `string` | 否 |
| website | 个人网站 URL | `string` | 否 |

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/ui/user-profile.tsx`：组件导出入口，re-export。

关键行为：

1. 组件内部管理查看/编辑模式的切换，调用侧只需关心 `onSave` 回调。
2. 编辑模式下字段变为可编辑输入框，查看模式下以文本形式展示。
3. 头像支持点击更换，变更通过 `actions.onAvatarChange` 回调通知。
4. 保存按钮在 `isLoading` 为 `true` 时禁用并显示加载状态。
5. `appearance.editable` 为 `false` 时，编辑按钮不渲染，所有字段只读展示。

## FAQ {#faq}

### 如何获取编辑后的用户数据？

通过 `actions.onSave` 回调接收完整的 `UserData` 对象：

```tsx
<UserProfile
  actions={{
    onSave: (data) => {
      console.log("Name:", data.name)
      console.log("Email:", data.email)
      // 调用 API 保存数据
    }
  }}
/>
```

### 如何设置保存加载状态？

通过 `control.isLoading` 控制。典型用法是配合异步 `onSave`：

```tsx
const [loading, setLoading] = useState(false)

<UserProfile
  actions={{
    onSave: async (data) => {
      setLoading(true)
      await saveProfileAPI(data)
      setLoading(false)
    }
  }}
  control={{ isLoading: loading }}
/>
```

### 如何自定义标题和按钮文案？

通过 `data.title`、`data.editLabel`、`data.saveLabel`、`data.cancelLabel` 自定义：

```tsx
<UserProfile
  data={{
    title: "个人资料",
    editLabel: "编辑资料",
    saveLabel: "保存修改",
    cancelLabel: "放弃修改",
  }}
/>
```

### 如何隐藏头像区域？

设置 `appearance.showAvatar` 为 `false`：

```tsx
<UserProfile appearance={{ showAvatar: false }} />
```

### 如何隐藏标题区域？

设置 `appearance.showTitle` 为 `false`：

```tsx
<UserProfile appearance={{ showTitle: false }} />
```

### 如何禁用编辑功能？

设置 `appearance.editable` 为 `false`，组件将仅展示用户信息，不显示编辑按钮：

```tsx
<UserProfile appearance={{ editable: false }} />
```

### 如何处理头像上传？

通过 `actions.onAvatarChange` 回调获取用户选择的头像文件：

```tsx
<UserProfile
  actions={{
    onAvatarChange: async (file) => {
      const formData = new FormData()
      formData.append("avatar", file)
      await uploadAvatarAPI(formData)
    }
  }}
/>
```

## LLM 使用规则 {#llm-rules}

当 AI 生成 UserProfile 代码时，应遵守：

1. 从 `@/components/templates/manifest/form/user-profile` 导入 `UserProfile`。
2. 使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop，不要直接传入子组件。
3. 资料保存通过 `actions.onSave` 处理，不要手动管理组件内部的编辑状态或字段绑定。
4. 加载状态仅通过 `control.isLoading` 控制，按钮内部已处理加载文案和禁用逻辑。
5. 头像变更通过 `actions.onAvatarChange` 处理，不要自行实现头像上传 UI。
6. 不要引入 Ant Design、ConfigProvider 或 css-in-js 相关依赖。
7. 不要复刻或重新实现 UserProfile 的内部逻辑；它是一个页面模板组件，应直接使用。
