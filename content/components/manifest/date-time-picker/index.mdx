日期时间选择器 DateTimePicker。Calendly 风格的日期和时间选择组件，提供日历视图、可用时间段列表和时区选择功能，支持桌面端和移动端布局。

<h2 id="when-to-use">何时使用</h2>

当需要用户选择特定日期和时间时使用。DateTimePicker 将日历导航、日期点选和时间段选择组合为一个紧凑的复合组件，适合预约、预定和日程安排场景。

常见场景：

- 会议预约、咨询预约等日程安排页面。
- 服务预订中的时间选择步骤（如医疗、美容、咨询等）。
- 需要结合时区选择的跨国预约场景。

不推荐：

- 仅需选择日期（不需要时间）—— 使用 `Calendar` 或 `DatePicker` 组件。
- 仅需选择时间（不需要日期）—— 使用 `TimePicker` 组件。
- 需要日期区间或多日连续选择 —— DateTimePicker 一次只能选择一个日期和一个时间点。
- 仅需展示静态日期信息（非交互选择）—— 使用 `Calendar` 或纯文本展示。

<h2 id="examples">代码演示</h2>

### 基础用法

`DateTimePicker` 包含日历导航、日期点选和时间段选择，默认展示所有可用日期和时间槽位。

<ComponentShowcase id="manifest/date-time-picker" path="manifest/date-time-picker/basic" />

```tsx
import { DateTimePicker } from "@/components/ui/date-time-picker"

export default function Demo() {
  return (
    <DateTimePicker
      data={{
        title: "Select a Date & Time",
        availableDates: [
          new Date(2026, 6, 1),
          new Date(2026, 6, 7),
          new Date(2026, 6, 14),
        ],
        availableTimeSlots: [
          "9:00am",
          "10:00am",
          "2:30pm",
          "4:00pm",
        ],
        timezone: "Eastern Time - US & Canada",
      }}
      appearance={{ showTitle: true, showTimezone: true }}
    />
  )
}
```

### 隐藏标题和时区

通过 `appearance` 控制标题和时区选择器的显隐。

### 周起始日设置为周一

默认为周日（US 风格），可通过 `weekStartsOn` 设置为周一（ISO 8601 标准）。

```tsx
import { DateTimePicker } from "@/components/ui/date-time-picker"

export default function Demo() {
  return (
    <DateTimePicker
      data={{
        title: "Schedule a Meeting",
        availableDates: [
          new Date(2026, 6, 1),
          new Date(2026, 6, 7),
          new Date(2026, 6, 14),
          new Date(2026, 6, 21),
        ],
        availableTimeSlots: [
          "9:00am",
          "10:30am",
          "1:00pm",
          "3:30pm",
        ],
      }}
      appearance={{ weekStartsOn: "monday" }}
    />
  )
}
```

### 受控模式

通过 `control` prop 从外部控制选中状态，配合 `actions.onNext` 处理确认事件。

```tsx
import { useState } from "react"
import { DateTimePicker } from "@/components/ui/date-time-picker"

export default function Demo() {
  const [selectedDate, setSelectedDate] = useState<Date | null>(null)
  const [selectedTime, setSelectedTime] = useState<string | null>(null)

  return (
    <DateTimePicker
      data={{
        title: "Book an Appointment",
        availableDates: [
          new Date(2026, 6, 1),
          new Date(2026, 6, 8),
          new Date(2026, 6, 15),
        ],
        availableTimeSlots: [
          "9:00am",
          "11:00am",
          "2:00pm",
          "4:00pm",
        ],
      }}
      control={{
        selectedDate,
        selectedTime,
      }}
      actions={{
        onNext: (date, time) => {
          console.log("Selected:", date, time)
        },
      }}
    />
  )
}
```

<h2 id="api">API</h2>

### DateTimePicker

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 日期和时间数据配置 | `DateTimePickerData` | - |
| data.title | 选择器顶部标题文字 | `string` | - |
| data.availableDates | 可选择的有效日期数组 | `Date[]` | - |
| data.availableTimeSlots | 可选择的时间段字符串数组 | `string[]` | - |
| data.timezone | 默认时区名称 | `string` | `"Eastern Time - US & Canada"` |
| appearance | 外观配置 | `DateTimePickerAppearance` | `{}` |
| appearance.showTitle | 是否显示标题 | `boolean` | `true` |
| appearance.showTimezone | 是否显示时区选择器 | `boolean` | `true` |
| appearance.weekStartsOn | 每周起始日 | `"sunday"` \| `"monday"` \| `"saturday"` | `"sunday"` |
| actions | 行为回调配置 | `DateTimePickerActions` | `{}` |
| actions.onNext | 用户确认日期和时间后点击 Next 按钮的回调 | `(date: Date, time: string) => void` | - |
| control | 受控模式配置 | `DateTimePickerControl` | - |
| control.selectedDate | 受控选中日期 | `Date \| null` | - |
| control.selectedTime | 受控选中时间 | `string \| null` | - |

<h2 id="faq">常见问题</h2>

### DateTimePicker 与 DatePicker 有什么区别？

`DatePicker` 仅用于选择日期，返回一个日期值。`DateTimePicker` 同时选择日期和具体时间段，是一个完整的预约选择组件，内置日历导航、时间槽位列表和时区选择器。

### 如何修改可选时区列表？

可选时区通过组件内部的 `timezones` 常量定义，目前支持 20 个 IANA 时区。如需添加更多时区，需修改源码中的 `timezones` 数组，每个时区项需提供 `id`、`name` 和 `iana`（IANA 时区标识符）三个字段。

### 可用日期不包含周末怎么办？

在传入的 `availableDates` 数组中过滤掉周六和周日即可。组件仅标记已有日期为可选，不会自动排除周末：

```tsx
const weekdays = allDates.filter(d => d.getDay() !== 0 && d.getDay() !== 6)
<DateTimePicker data={{ availableDates: weekdays }} />
```

### 移动端布局如何工作？

移动端将日历和时间槽位拆分为两个视图。用户先选择日期后自动切换到时间选择视图，也可通过返回按钮回到日历视图。桌面端保持左右分栏布局，选中日期后时间槽位区域从右侧滑入。

### 如何使组件支持多语言？

日历 Header 的日期格式（如 "Wednesday, July 2"）和月份名称当前硬编码为英文。如需本地化，需修改源码中 `formatDateHeader` 函数和 `MONTHS` 数组，替换为对应语言的格式化逻辑。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/date-time-picker` 导入 `DateTimePicker`，不要从 `antd` 导入（AntD 无此组件）。
2. `data.availableDates` 必须是 `Date[]`，组件内部使用 `isSameDay` 比较日期，确保传入有效的 Date 对象。
3. `data.availableTimeSlots` 是字符串数组（如 `["9:00am", "2:30pm"]`），组件不自动生成时间段，需外部传入。
4. 日期不可选时组件自动置灰，无需外部处理禁用逻辑。
5. 移动端自动采用两视图布局（日历→时间），设计时无需额外响应式处理。
6. 时区选择使用 IANA 时区标识符（如 `"America/New_York"`），通过 `Intl.DateTimeFormat` 获取当前时间，确保正确处理夏令时。
7. 本组件为 Blocks 级复合组件，无 Ant Design 对应物，不要尝试从 `antd` 导入日期时间选择相关组件。
8. 受控模式需同时提供 `control.selectedDate` 和 `control.selectedTime`，组件内部使用 `useState` 初始化默认值，如需完全外部控制应使用受控模式。
9. `weekStartsOn` 影响日历网格的列排序（`sunday` → 日一二三四五六，`monday` → 一二三四五六日，`saturday` → 六日一二三四五），不影响日期计算逻辑。
