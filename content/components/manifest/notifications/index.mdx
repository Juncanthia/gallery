通知中心，提供命令式的全局通知提醒 API，在屏幕角落展示提示信息。基于 sonner 实现，支持 success、info、warning、error 四种类型。

<h2 id="when-to-use">何时使用</h2>

当需要向用户推送系统级通知消息时使用。通知从屏幕角落弹出，不打断用户当前操作。

常见场景：

- 操作成功后的反馈提示（如"保存成功"）。
- 系统级警告或错误通知。
- 需要自动消失的临时消息。
- 跨路由的全局消息推送。

通知通过 `notification.success()`、`notification.info()`、`notification.warning()`、`notification.error()` 四种方法表达语义层级。

推荐用法：

- 成功操作：`notification.success({ title: "保存成功" })`。
- 一般提示：`notification.info({ title: "系统通知" })`。
- 警告提示：`notification.warning({ title: "请注意" })`。
- 错误反馈：`notification.error({ title: "操作失败" })`。
- 不自动关闭的通知设置 `duration: 0`。

不推荐：

- 需要用户明确交互确认的提示——使用 Dialog、AlertDialog 或 Popconfirm。
- 与当前页面上下文强相关的内联提示——使用 Alert。
- 大量频繁推送消息——会导致通知堆积影响体验。
- 在 Toaster 组件未挂载时调用——通知将无法渲染。

### 基础用法

命令式调用，无需组件声明。前提是页面中已挂载 `&lt;Toaster /&gt;` 组件。

```tsx
import { notification } from "@/components/ui/notifications"
import { Button } from "@/components/ui/button"

export default function Demo() {

            title: "操作成功",
            description: "数据已保存",
          })
        }
      >
        成功通知
      </Button>
      <Button
        size="small"
        variant="outlined"
        onClick={() =>
          notification.info({
            title: "提示",
            description: "这是一条信息",
          })
        }
      >
        信息通知
      </Button>
      <Button
        size="small"
        variant="outlined"
        onClick={() =>
          notification.warning({
            title: "警告",
            description: "请注意检查",
          })
        }
      >
        警告通知
      </Button>
      <Button
        size="small"
        variant="outlined"
        onClick={() =>
          notification.error({
            title: "错误",
            description: "操作失败，请重试",
          })
        }
      >
        错误通知
      </Button>
    </div>
  )
}
```

<h2 id="api">API</h2>

### notification

命令式 API 对象，无需组件声明。前提是页面中已挂载 `&lt;Toaster /&gt;`（来自 `@/components/ui/toaster`）。

| 方法 | 说明 | 签名 |
| --- | --- | --- |
| `notification.success(args)` | 成功通知 | `(args: NotificationArgs) =&gt; NotificationClose` |
| `notification.info(args)` | 信息通知 | `(args: NotificationArgs) =&gt; NotificationClose` |
| `notification.warning(args)` | 警告通知 | `(args: NotificationArgs) =&gt; NotificationClose` |
| `notification.error(args)` | 错误通知 | `(args: NotificationArgs) =&gt; NotificationClose` |
| `notification.open(args)` | 通用通知（不指定 type） | `(args: NotificationArgs) =&gt; NotificationClose` |
| `notification.destroy(key?)` | 关闭指定或全部通知 | `(key?: string \| number) =&gt; void` |

### NotificationArgs

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| title | 通知标题（必填） | `React.ReactNode` | - |
| description | 通知内容描述 | `React.ReactNode` | - |
| type | 通知类型 | `"success" \| "info" \| "warning" \| "error"` | `"info"` |
| key | 唯一标识，用于 update/destroy | `string \| number` | - |
| duration | 自动关闭时间（秒），0 表示不关闭 | `number` | `4.5` |
| placement | 屏幕位置 | `"top" \| "topLeft" \| "topRight" \| "bottom" \| "bottomLeft" \| "bottomRight"` | `"topRight"` |
| closable | 是否显示关闭按钮 | `boolean` | `true` |
| closeIcon | 自定义关闭图标 | `React.ReactNode` | `✕` |
| icon | 自定义通知图标 | `React.ReactNode` | 根据 type 自动选择 |
| onClose | 通知关闭回调 | `() =&gt; void` | - |
| onClick | 通知点击回调 | `() =&gt; void` | - |
| className | 附加样式类名 | `string` | - |
| style | 附加内联样式 | `React.CSSProperties` | - |

<h2 id="faq">常见问题</h2>

### 为什么调用 notification.success() 后看不到通知？

确保页面根组件中已挂载 `&lt;Toaster /&gt;`，它是 sonner 的容器组件。通常放在 `App.tsx` 或 `layout.tsx` 中：

```tsx
import { Toaster } from "@/components/ui/toaster"

export default function Layout({ children }) {
  return (
    &lt;&gt;
      {children}
      &lt;Toaster /&gt;
    &lt;/&gt;
  )
}
```

### 如何手动关闭通知？

调用时保存返回的关闭函数，需要时执行：

```tsx
const close = notification.success({ title: "处理中..." })
// 稍后关闭
close()
// 或通过 key 关闭
notification.success({ title: "处理中...", key: "upload" })
notification.destroy("upload")
```

### 如何设置通知不自动关闭？

设置 `duration: 0`，通知将一直显示直到用户点击关闭按钮或手动调用 `destroy`：

```tsx
notification.warning({
  title: "重要提醒",
  description: "此通知不会自动消失",
  duration: 0,
})
```

### 自定义通知位置

通过 `placement` 属性设置通知出现位置：

```tsx
notification.info({
  title: "左上角通知",
  placement: "topLeft",
})
```

### 如何自定义图标？

通过 `icon` 属性传入自定义图标：

```tsx
import { Check } from "lucide-react"

notification.success({
  title: "自定义图标",
  icon: &lt;Check className="h-4 w-4" /&gt;,
})
```

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Notifications 代码时：

1. 从 `@/components/ui/notifications` 导入 `notification` 对象。
2. 使用 `notification.success/info/warning/error` 方法，`title` 为必填属性。
3. 确保页面中已挂载 `&lt;Toaster /&gt;`（来自 `@/components/ui/toaster`）。
4. 需要不自动关闭的通知设置 `duration: 0`。
5. 不要引入 Ant Design 相关依赖，不要使用 `useNotification` hooks。
6. 不要使用 `antd` 的 `notification` 或 `App.useApp().notification`。
