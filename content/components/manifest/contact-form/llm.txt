# ContactForm 联系表单

ContactForm 是一个完整的联系表单组件，适用于网站的"联系我们"页面或业务咨询入口，支持姓名、带国家代码选择器的电话号码、邮箱、项目描述和文件附件。

本组件是页面模板级别的复合组件，内部组合了 Input、Label、Popover、Button 等基础组件，提供开箱即用的联系表单体验。

## 何时使用 {#when-to-use}

用于收集用户的联系信息及项目需求。适合以下场景：

- 网站的"联系我们"页面，提供结构化的联系表单。
- 业务咨询入口，收集用户的基本信息与需求描述。
- 服务型页面的线索收集，支持文件附件上传。
- 需要国际化的电话输入场景，内建 200+ 国家/地区的代码选择。

## 代码演示 {#examples}

### 基础用法

默认展示标题、副标题和完整表单字段。表单字段包括：姓、名、电话号码（含国家代码选择）、邮箱、项目描述文本区域和文件附件。

```tsx
import { ContactForm } from "@/components/ui/contact-form"

export default function ContactFormDemo() {
  return (
    <div className="w-full max-w-xl">
      <ContactForm
        actions={{
          onSubmit: (data) => console.log("Submitted:", data)
        }}
      />
    </div>
  )
}
```

### 自定义文案

通过 `data` prop 自定义标题、副标题和按钮文字。

```tsx
import { ContactForm } from "@/components/ui/contact-form"

export default function ContactFormCustomDemo() {
  return (
    <div className="w-full max-w-xl">
      <ContactForm
        data={{
          title: "联系我们",
          subtitle: "请填写以下信息，我们会尽快回复您。",
          submitLabel: "提交",
        }}
      />
    </div>
  )
}
```

### 按需加载

处理表单提交时的异步加载状态。

```tsx
import { useState } from "react"
import { ContactForm } from "@/components/ui/contact-form"

export default function ContactFormLoadingDemo() {
  const [loading, setLoading] = useState(false)

  async function handleSubmit(data: Parameters<NonNullable<React.ComponentProps<typeof ContactForm>["actions"]>>[0]["onSubmit"] extends (d: infer D) => void ? D : never) {
    setLoading(true)
    // 模拟 API 请求
    await new Promise((resolve) => setTimeout(resolve, 2000))
    setLoading(false)
  }

  return (
    <div className="w-full max-w-xl">
      <ContactForm
        actions={{ onSubmit: handleSubmit }}
        control={{ isLoading: loading }}
      />
    </div>
  )
}
```

## API {#api}

### ContactForm

通过 `data`、`actions`、`appearance`、`control` 四个顶层 prop 配置组件行为。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 表单数据配置 | `object` | — |
| data.title | 表单标题 | `string` | `"Get in Touch"` |
| data.subtitle | 标题下方描述文字 | `string` | `"We'd love to hear from you. Fill out the form below."` |
| data.submitLabel | 提交按钮文字 | `string` | `"Send Message"` |
| data.initialValues | 预填充的表单初始值 | `Partial<ContactFormData>` | — |
| actions | 回调配置 | `object` | — |
| actions.onSubmit | 表单提交回调，接收完整 `ContactFormData` | `(data: ContactFormData) => void` | — |
| appearance | 外观配置 | `object` | — |
| appearance.showTitle | 是否显示标题区域 | `boolean` | `true` |
| control | 控制配置 | `object` | — |
| control.isLoading | 提交按钮加载状态；为 `true` 时按钮显示"Sending..."并禁用 | `boolean` | `false` |

### ContactFormData

表单提交时 `onSubmit` 接收的数据结构。

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| firstName | 名 | `string` | 是 |
| lastName | 姓 | `string` | 是 |
| countryId | 国家标识符，例如 `us`、`cn`、`fr` | `string` | 否 |
| countryCode | 电话国家代码，例如 `+1`、`+86`、`+33` | `string` | 否 |
| phoneNumber | 电话号码（不包含国家代码部分） | `string` | 否 |
| email | 电子邮箱地址 | `string` | 是 |
| message | 项目描述或留言内容 | `string` | 是 |
| attachment | 文件附件对象 | `File \| null` | 否 |

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/manifest/form/contact-form.tsx`：联系表单组件实现。
- `src/components/manifest/form/countries.ts`：国家/地区及其电话代码数据。
- `src/components/manifest/form/demo/form.ts`：演示用的默认数据。
- `src/components/ui/manifest-contact-form.tsx`：组件导出入口，re-export。

关键行为：

1. 表单内部管理所有字段状态，调用侧只需关心 `onSubmit` 回调。
2. `initialValues` 仅用于首次渲染，之后表单状态由组件内部管理。
3. 国家代码搜索支持按国家名称或代码进行过滤。
4. 文件附件选择后会显示文件名预览，可通过 X 按钮移除。
5. 提交按钮在 `isLoading` 为 `true` 时禁用并显示"Sending..."文案。

## FAQ {#faq}

### 如何获取表单提交数据？

通过 `actions.onSubmit` 回调接收完整的 `ContactFormData` 对象。组件内部负责表单验证和状态管理，调用侧只需关心收到数据后的处理逻辑。

```tsx
<ContactForm
  actions={{
    onSubmit: (data) => {
      console.log("Received:", data.firstName, data.lastName, data.email)
      console.log("Attachment:", data.attachment?.name)
    }
  }}
/>
```

### 如何设置提交加载状态？

通过 `control.isLoading` 控制。典型用法是配合异步 `onSubmit`：

```tsx
const [loading, setLoading] = useState(false)

<ContactForm
  actions={{
    onSubmit: async (data) => {
      setLoading(true)
      await submitToAPI(data)
      setLoading(false)
    }
  }}
  control={{ isLoading: loading }}
/>
```

### 如何隐藏标题区域？

设置 `appearance.showTitle` 为 `false`，标题和副标题将不渲染。

```tsx
<ContactForm appearance={{ showTitle: false }} />
```

### 如何预填充表单？

通过 `data.initialValues` 传入初始值。注意这仅影响首次渲染，之后由组件内部管理。

```tsx
<ContactForm
  data={{
    initialValues: {
      firstName: "张",
      lastName: "三",
      email: "zhangsan@example.com",
      countryId: "cn",
      countryCode: "+86",
    }
  }}
/>
```

### 如何自定义文案？

通过 `data.title`、`data.subtitle`、`data.submitLabel` 分别自定义标题、描述和按钮文字。

```tsx
<ContactForm
  data={{
    title: "联系我们",
    subtitle: "请填写以下信息，我们会尽快回复您。",
    submitLabel: "提交",
  }}
/>
```

### 支持哪些文件格式？

文件附件 input 的 accept 属性限制为：`.pdf`、`.doc`、`.docx`、`.txt`、`.png`、`.jpg`、`.jpeg`。用户选择文件后在表单底部显示文件名及移除按钮。

## LLM 使用规则 {#llm-rules}

当 AI 生成 ContactForm 代码时，应遵守：

1. 从 `@/components/ui/contact-form` 导入 `ContactForm`。
2. 使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop，不要直接传入子组件。
3. 表单提交通过 `actions.onSubmit` 处理，不要手动管理组件内部的表单字段状态。
4. 加载状态仅通过 `control.isLoading` 控制，按钮内部已处理加载文案和禁用逻辑。
5. 预填充数据通过 `data.initialValues` 传入。
6. 不要引入 Ant Design、ConfigProvider 或 css-in-js 相关依赖。
7. 不要复刻或重新实现 ContactForm 的内部逻辑；它是一个页面模板组件，应直接使用。
