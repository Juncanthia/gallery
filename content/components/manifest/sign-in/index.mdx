登录组件 SignIn。用户身份验证入口组件，支持邮箱/用户名和密码登录，提供"记住我"、忘记密码和注册引导等辅助功能。

<h2 id="when-to-use">何时使用</h2>

当需要为用户提供身份验证入口时使用。SignIn 将邮箱输入、密码输入、登录按钮以及辅助链接组合为一个完整的登录表单。

常见场景：

- SaaS 应用的用户登录页面。
- 会员系统的登录入口。
- 管理后台的身份验证。
- 移动端 App 的登录界面。

不推荐：

- 仅需单一字段验证（如仅输入手机号发验证码）—— 使用自定义表单。
- 需要第三方 OAuth 登录（如 Google、GitHub）—— SignIn 专注于邮箱密码登录，社交登录应单独处理。
- 仅用于展示登录表单样式（非交互）—— SignIn 为交互式表单组件。

<h2 id="examples">代码演示</h2>

### 基础用法

默认展示标题、副标题、邮箱输入框、密码输入框和登录按钮。点击登录按钮时，通过 `actions.onSubmit` 回调获取登录数据。

```tsx
import { SignIn } from "@/components/ui/sign-in"

export default function Demo() {
  return (
    <div className="w-full max-w-md">
      <SignIn />
    </div>
  )
}
```

### 处理登录提交

通过 `actions.onSubmit` 回调接收邮箱和密码，可在其中调用 API。

```tsx
import { SignIn } from "@/components/ui/sign-in"

export default function Demo() {
  return (
    <SignIn
      actions={{
        onSubmit: async (email, password) => {
          console.log("Signing in:", email)
          await signInAPI(email, password)
        },
      }}
    />
  )
}
```

### 受控模式与加载状态

通过 `control.isLoading` 控制登录按钮加载状态，适用于异步 API 调用。

```tsx
import { useState } from "react"
import { SignIn } from "@/components/ui/sign-in"

export default function Demo() {
  const [loading, setLoading] = useState(false)

  return (
    <SignIn
      control={{ isLoading: loading }}
      actions={{
        onSubmit: async (email, password) => {
          setLoading(true)
          try {
            await signInAPI(email, password)
          } finally {
            setLoading(false)
          }
        },
      }}
    />
  )
}
```

### 自定义标题与文案

通过 `data` 自定义标题、副标题、按钮文案和占位符文字。

```tsx
import { SignIn } from "@/components/ui/sign-in"

export default function Demo() {
  return (
    <SignIn
      data={{
        title: "欢迎回来",
        subtitle: "请输入您的账号和密码登录系统",
        emailLabel: "邮箱地址",
        passwordLabel: "登录密码",
        submitLabel: "登 录",
        emailPlaceholder: "请输入邮箱",
        passwordPlaceholder: "请输入密码",
      }}
    />
  )
}
```

### 隐藏辅助链接

通过 `appearance.showLinks` 控制"忘记密码"和"注册账号"链接的显示。

```tsx
import { SignIn } from "@/components/ui/sign-in"

export default function Demo() {
  return (
    <SignIn
      data={{
        title: "管理后台登录",
        subtitle: "仅限授权人员访问",
      }}
      appearance={{ showLinks: false }}
    />
  )
}
```

<h2 id="api">API</h2>

### SignIn

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 表单数据配置 | `SignInData` | — |
| data.title | 登录标题 | `string` | `"Sign In"` |
| data.subtitle | 标题下方描述文字 | `string` | `"Enter your credentials to access your account"` |
| data.emailLabel | 邮箱输入框标签 | `string` | `"Email"` |
| data.passwordLabel | 密码输入框标签 | `string` | `"Password"` |
| data.submitLabel | 登录按钮文字 | `string` | `"Sign In"` |
| data.emailPlaceholder | 邮箱输入框占位符 | `string` | `"you@example.com"` |
| data.passwordPlaceholder | 密码输入框占位符 | `string` | `"Enter your password"` |
| data.rememberMeLabel | "记住我"复选框文字 | `string` | `"Remember me"` |
| data.forgotPasswordLabel | "忘记密码"链接文字 | `string` | `"Forgot password?"` |
| data.signUpLabel | "注册账号"链接文字 | `string` | `"Don't have an account? Sign up"` |
| data.forgotPasswordHref | "忘记密码"链接地址 | `string` | `"#"` |
| data.signUpHref | "注册账号"链接地址 | `string` | `"#"` |
| data.initialValues | 预填充的初始值 | `Partial&lt;SignInFormData&gt;` | — |
| actions | 回调配置 | `SignInActions` | — |
| actions.onSubmit | 登录提交回调 | `(email: string, password: string) => void` | — |
| actions.onForgotPassword | "忘记密码"点击回调（覆盖 href 跳转） | `() => void` | — |
| actions.onSignUp | "注册账号"点击回调（覆盖 href 跳转） | `() => void` | — |
| appearance | 外观配置 | `SignInAppearance` | — |
| appearance.showTitle | 是否显示标题区域 | `boolean` | `true` |
| appearance.showRememberMe | 是否显示"记住我"复选框 | `boolean` | `true` |
| appearance.showLinks | 是否显示辅助链接（忘记密码、注册） | `boolean` | `true` |
| control | 控制配置 | `SignInControl` | — |
| control.isLoading | 登录按钮加载状态 | `boolean` | `false` |
| control.error | 表单级别错误信息 | `string` | — |

### SignInFormData

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| email | 用户邮箱地址 | `string` | 是 |
| password | 用户密码 | `string` | 是 |
| rememberMe | 是否记住登录状态 | `boolean` | 否 |

<h2 id="faq">常见问题</h2>

### 如何获取登录表单数据？

通过 `actions.onSubmit` 回调接收邮箱和密码：

```tsx
<SignIn
  actions={{
    onSubmit: (email, password) => {
      console.log("Email:", email, "Password:", password)
      // 发送 API 请求...
    },
  }}
/>
```

### 如何显示登录错误信息？

使用 `control.error` 传入错误文案，组件内部会展示错误提示：

```tsx
const [error, setError] = useState("")

<SignIn
  control={{ error }}
  actions={{
    onSubmit: async (email, password) => {
      try {
        await signInAPI(email, password)
      } catch (e) {
        setError("邮箱或密码错误，请重试")
      }
    },
  }}
/>
```

### 如何设置登录加载状态？

使用 `control.isLoading` 控制按钮加载状态：

```tsx
const [loading, setLoading] = useState(false)

<SignIn
  control={{ isLoading: loading }}
  actions={{
    onSubmit: async (email, password) => {
      setLoading(true)
      try {
        await signInAPI(email, password)
      } finally {
        setLoading(false)
      }
    },
  }}
/>
```

### 如何自定义标题和按钮文案？

通过 `data.title`、`data.subtitle`、`data.submitLabel` 等属性自定义：

```tsx
<SignIn
  data={{
    title: "欢迎回来",
    subtitle: "请输入您的账号密码",
    emailLabel: "邮箱",
    passwordLabel: "密码",
    submitLabel: "登录",
  }}
/>
```

### 如何隐藏辅助链接？

设置 `appearance.showLinks` 为 `false`：

```tsx
<SignIn appearance={{ showLinks: false }} />
```

也可以单独控制"记住我"显示：`appearance.showRememberMe`。

### 如何自定义跳转链接？

通过 `data.forgotPasswordHref` 和 `data.signUpHref` 设置链接地址：

```tsx
<SignIn
  data={{
    forgotPasswordHref: "/auth/forgot-password",
    signUpHref: "/auth/sign-up",
  }}
/>
```

如需使用回调而非链接跳转，使用 `actions.onForgotPassword` 和 `actions.onSignUp`。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/sign-in` 导入 `SignIn`，不要从 Ant Design 导入 Login 或 Form 组件。
2. 使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop 配置组件。
3. 登录提交通过 `actions.onSubmit` 处理，回调接收 `email` 和 `password` 两个参数。
4. 加载状态通过 `control.isLoading` 控制，组件内部自动禁用按钮并显示加载状态。
5. 错误信息通过 `control.error` 传入，组件内部展示错误提示。
6. 不要手动管理组件内部的表单状态，组件自行处理邮箱和密码的输入绑定。
7. 辅助链接（忘记密码、注册）可使用 `data.forgotPasswordHref` / `data.signUpHref` 设置 href，或通过 `actions.onForgotPassword` / `actions.onSignUp` 使用回调。
8. 不要引入 Ant Design 相关依赖（Form、Input、Button 等）。
