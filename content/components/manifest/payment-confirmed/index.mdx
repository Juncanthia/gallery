PaymentConfirmed 用于在支付成功后向用户展示订单确认信息。提供产品图片、价格、配送日期和订单追踪入口。

<h2 id="when-to-use">何时使用</h2>

当支付完成需要向用户展示确认摘要时使用此组件。适用于：

- 电商结账流程：用户完成支付后展示订单确认和预计送达时间。
- 订阅购买：确认订阅激活并展示计费摘要。
- 数字商品购买：确认购买成功并提供下载或访问入口。
- 服务预订：确认预约并展示服务详情。

PaymentConfirmed 支持两种展示变体：

- **default**：详细布局，包含头部确认消息、产品图片、描述和底部操作按钮。
- **compressed**：紧凑内联布局，适合空间有限的场景，如卡片列表或通知区域。

<h2 id="faq">常见问题</h2>

### 如何监听追踪订单的点击事件？

通过 `actions.onTrackOrder` 回调处理：

```tsx
<PaymentConfirmed
  data={orderData}
  actions={{
    onTrackOrder: () => {
      // 跳转到订单追踪页面或打开追踪链接
      router.push(`/orders/${orderData.orderId}/tracking`)
    },
  }}
/>
```

### 如何在紧凑布局中使用？

设置 `appearance.variant` 为 `"compressed"`：

```tsx
<PaymentConfirmed
  data={orderData}
  appearance={{ variant: "compressed" }}
/>
```

紧凑模式将确认图标、产品缩略图、名称和价格水平排列，适合卡片列表或通知等空间有限的场景。

### 如何自定义货币格式？

通过 `appearance.currency` 指定 ISO 4217 货币代码：

```tsx
<PaymentConfirmed
  data={{ price: 299 }}
  appearance={{ currency: "USD" }}
/>
```

价格通过 `Intl.NumberFormat` 格式化，自动根据货币代码显示正确的符号和格式。

### 是否支持空状态或加载状态？

PaymentConfirmed 当前不支持内置的加载状态。所有数据字段（除 `productImage`）均为可选——未传入的字段对应的 UI 区域会被隐藏。如需加载态，请在调用侧自行封装骨架屏。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 PaymentConfirmed 代码时：

1. 从 `@/components/templates/manifest/payment/payment-confirmed` 导入 PaymentConfirmed
2. 通过 `data` prop 传入订单确认数据（orderId、productName、productImage、price、deliveryDate 等）
3. 通过 `actions.onTrackOrder` 处理订单追踪操作
4. 通过 `appearance.variant` 选择 "default"（详细）或 "compressed"（紧凑）布局
5. 通过 `appearance.currency` 指定货币代码，默认 "EUR"
6. productDescription 仅在 default 变体中展示，compressed 变体忽略此字段
