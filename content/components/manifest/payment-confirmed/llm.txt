# PaymentConfirmed 支付确认

PaymentConfirmed 用于在支付成功后向用户展示订单确认信息。支持两种布局变体：default（详细布局，含头部确认消息、产品图片、描述和底部操作按钮）和 compressed（紧凑内联布局，适合空间有限的场景）。

## 何时使用 {#when-to-use}

当支付完成需要向用户展示确认摘要时使用此组件。适用于：

- 电商结账流程：用户完成支付后展示订单确认和预计送达时间。
- 订阅购买：确认订阅激活并展示计费摘要。
- 数字商品购买：确认购买成功并提供下载或访问入口。
- 服务预订：确认预约并展示服务详情。

PaymentConfirmed 的核心功能：

- 成功确认图标和"Payment confirmed"文案。
- 订单编号展示。
- 产品图片展示（支持默认和紧凑变体）。
- 价格和配送日期显示。
- 追踪订单按钮。

## 代码演示 {#examples}

### 基础支付确认

默认展示包含订单 ID、产品图片、名称、描述、价格、配送日期和追踪按钮的确认卡片。

```tsx
import { PaymentConfirmed } from "@/components/ui/manifest-payment-confirmed"

const sampleData = {
  orderId: "ORD-2024-5678",
  productName: "Air Force 1 '07",
  productDescription: "Nike · Size 10 · Black",
  productImage: "https://ui.manifest.build/demo/shoe-1.png",
  price: 299,
  deliveryDate: "Jan 20, 2024",
}

export default function Demo() {
  return (
    <div className="w-full max-w-md">
      <PaymentConfirmed
        data={sampleData}
        actions={{
          onTrackOrder: () => console.log("Track order clicked"),
        }}
        appearance={{ variant: "default", currency: "EUR" }}
      />
    </div>
  )
}
```

## API {#api}

### PaymentConfirmed

通过 `data` 传入订单数据，`actions` 处理追踪操作，`appearance` 控制布局变体和货币。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 订单确认数据 | `object` | 内置 Demo 数据 |
| data.orderId | 订单编号 | `string` | - |
| data.productName | 产品名称 | `string` | - |
| data.productDescription | 产品描述（仅 default 变体） | `string` | - |
| data.productImage | 产品图片 URL | `string` | - |
| data.price | 订单总价 | `number` | - |
| data.deliveryDate | 预计配送日期 | `string` | - |
| actions.onTrackOrder | 追踪订单回调 | `() => void` | - |
| appearance.variant | 布局变体 | `"default"` \| `"compressed"` | `"default"` |
| appearance.currency | 货币代码 | `string` | `"EUR"` |

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/manifest/payment/payment-confirmed.tsx`：支付确认卡片渲染、响应式布局、价格格式化。
- `src/components/ui/manifest-payment-confirmed.tsx`：薄包装，从 manifest 包重新导出。
- `src/components/manifest/payment/demo/payment.ts`：Demo 默认数据。

关键行为：

1. 组件为纯展示组件，无内部状态管理。
2. 所有 data 字段均为可选——未传入的对应 UI 区域会被隐藏。
3. `productDescription` 仅在 default 变体中展示，compressed 变体中忽略。
4. 价格通过 `Intl.NumberFormat` 格式化，根据 `appearance.currency` 显示正确的货币符号。
5. 支持移动端和桌面端响应式布局，移动端字段垂直堆叠，桌面端水平排列。
6. 追踪订单按钮使用 `variant="outline"` + `ExternalLink` 图标。
7. 产品图片区域在无图片时显示 `bg-muted` 占位。
8. `actions.onTrackOrder` 在点击追踪按钮时触发，无内置路由跳转逻辑。

## FAQ {#faq}

### 如何监听追踪订单的点击事件？

通过 `actions.onTrackOrder` 回调处理：

```tsx
<PaymentConfirmed
  data={orderData}
  actions={{
    onTrackOrder: () => {
      router.push(`/orders/${orderData.orderId}/tracking`)
    },
  }}
/>
```

### 如何在紧凑布局中使用？

设置 `appearance.variant` 为 `"compressed"`：

```tsx
<PaymentConfirmed
  data={orderData}
  appearance={{ variant: "compressed" }}
/>
```

### 如何自定义货币格式？

通过 `appearance.currency` 指定 ISO 4217 货币代码：

```tsx
<PaymentConfirmed
  data={{ price: 299 }}
  appearance={{ currency: "USD" }}
/>
```

### 是否支持空状态或加载状态？

PaymentConfirmed 当前不支持内置的加载状态。未传入的数据字段对应的 UI 区域会被隐藏。如需加载态，在调用侧自行封装骨架屏。

## LLM 使用规则 {#llm-rules}

当 AI 生成 PaymentConfirmed 代码时，应遵守：

1. 从 `@/components/ui/manifest-payment-confirmed` 导入 PaymentConfirmed。
2. 通过 `data` prop 传入订单确认数据。
3. 通过 `actions.onTrackOrder` 处理订单追踪操作。
4. 通过 `appearance.variant` 选择 "default" 或 "compressed" 布局。
5. 通过 `appearance.currency` 指定货币代码。
6. productDescription 仅在 default 变体中有效。
7. 所有 data 字段均为可选，缺失时对应 UI 区域自动隐藏。
