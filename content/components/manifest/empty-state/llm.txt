# EmptyState 空状态

EmptyState 是一个空状态页面模板组件，当页面或列表无数据时向用户展示友好的提示信息，包含图标、标题、描述文字和可选的操作按钮，帮助用户理解当前状态并引导下一步操作。

本组件是页面模板级别的复合组件，内部组合了 Empty 基础组件和 Button 组件，提供开箱即用的空状态体验。

## 何时使用 {#when-to-use}

当页面内容区域暂无数据需要向用户展示空状态提示时使用。适合以下场景：

- 列表页面首次加载无数据时的空状态引导，引导用户创建第一条数据。
- 筛选或搜索无结果时的友好提示，说明未找到匹配内容并提供调整建议。
- 用户尚未创建任何内容时的引导入口，如仪表盘的"创建第一个项目"按钮。
- 管理后台或内容管理系统的空白区域占位，提升用户首次使用的引导体验。

EmptyState 以垂直居中布局展示空状态信息，图标区域展示插画或自定义图标，标题和描述提供上下文说明，可选操作按钮引导用户下一步行动。

## 代码演示 {#examples}

### 基础用法

默认展示内置的空状态页面模板，包含默认图标、标题、描述和"创建新项目"按钮。

```tsx
import { EmptyState } from "@/components/data-display/manifest/empty-state"

export default function EmptyStateDemo() {
  return (
    <div className="w-full max-w-2xl">
      <EmptyState />
    </div>
  )
}
```

### 自定义文案和操作

通过 `data` 和 `actions` prop 自定义文案和操作回调。

```tsx
import { EmptyState } from "@/components/data-display/manifest/empty-state"

export default function CustomEmptyStateDemo() {
  return (
    <EmptyState
      data={{
        title: "还没有订单",
        description: "您还没有创建任何订单，点击下方按钮开始创建第一个订单。",
        actionLabel: "创建订单",
      }}
      actions={{
        onAction: () => console.log("创建订单"),
      }}
    />
  )
}
```

### 自定义图标

通过 `data.image` 传入自定义的空状态图标节点。

```tsx
import { EmptyState } from "@/components/data-display/manifest/empty-state"
import { FileText } from "lucide-react"

export default function CustomIconDemo() {
  return (
    <EmptyState
      data={{
        image: (
          <div className="flex size-20 items-center justify-center rounded-full bg-muted/60">
            <FileText className="size-10 text-muted-foreground/60" />
          </div>
        ),
        title: "暂无文档",
        description: "您还没有上传任何文档，点击下方按钮上传。",
        actionLabel: "上传文档",
      }}
    />
  )
}
```

### 隐藏元素与加载状态

通过 `appearance` 和 `control` prop 控制元素显示和加载状态。

```tsx
import { EmptyState } from "@/components/data-display/manifest/empty-state"

export default function StatesDemo() {
  return (
    <div className="flex flex-col gap-8">
      <EmptyState
        data={{ title: "搜索无结果", description: "未找到匹配的内容" }}
        appearance={{ showAction: false }}
      />
      <EmptyState
        data={{ title: "暂无通知", description: "暂时没有新的通知消息" }}
        appearance={{ showImage: false, showAction: false }}
      />
      <EmptyState control={{ isLoading: true }} />
    </div>
  )
}
```

## API {#api}

### EmptyState

通过 `data`、`actions`、`appearance`、`control` 四个顶层 prop 配置组件行为。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 数据配置 | `EmptyStateData` | 内置 Demo 数据 |
| data.title | 空状态标题 | `string` | `"暂无数据"` |
| data.description | 空状态描述文字 | `string` | `"当前没有可显示的内容..."` |
| data.actionLabel | 操作按钮文字 | `string` | `"创建新项目"` |
| data.image | 自定义空状态图标，传入 ReactNode | `React.ReactNode \| false` | 默认 inbox 图标 |
| actions | 操作回调配置 | `EmptyStateActions` | — |
| actions.onAction | 点击操作按钮回调 | `() => void` | — |
| appearance | 外观配置 | `EmptyStateAppearance` | — |
| appearance.showImage | 是否显示图标区域 | `boolean` | `true` |
| appearance.showAction | 是否显示操作按钮 | `boolean` | `true` |
| control | 控制配置 | `EmptyStateControl` | — |
| control.isLoading | 为 `true` 时显示骨架屏 | `boolean` | `false` |

### EmptyStateData

表单数据中每条空状态数据的结构。

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| image | 自定义空状态图标，传入 ReactNode 或 false 隐藏 | `React.ReactNode \| false` | 否 |
| title | 空状态标题 | `string` | 否 |
| description | 空状态描述 | `string` | 否 |
| actionLabel | 操作按钮文字 | `string` | 否 |

## Semantic DOM {#semantic-dom}

EmptyState 通过 `data-slot` 属性暴露稳定的语义结构：

| data-slot | 说明 |
| --- | --- |
| `empty` | 组件根容器（继承自 Empty） |
| `empty-header` | 标题和描述容器 |
| `empty-title` | 标题文本 |
| `empty-description` | 描述文本 |
| `empty-content` | 操作按钮容器 |

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/ui/manifest-empty-state.tsx`：组件导出入口，re-export。
- `src/components/manifest/status/empty-state.tsx`：空状态页面模板组件实现。
- `src/components/ui/empty-state.tsx`：底层 Empty 布局组件。

关键行为：

1. 组件内部使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop，调用侧不需要直接使用 Empty 子组件。
2. `control.isLoading` 为 `true` 时展示骨架屏，替代正常空状态内容。
3. 内置默认图标使用 Inbox 图标，通过 `appearance.showImage` 控制显示。
4. 内置默认操作按钮使用 Button 组件，通过 `appearance.showAction` 控制显示。
5. 操作按钮点击由 `actions.onAction` 回调处理。
6. 无 prop 时展示内置 Demo 内容，方便预览和开发。

## FAQ {#faq}

### 如何自定义空状态图标？

通过 `data.image` 传入 React 节点：

```tsx
<EmptyState
  data={{
    image: <MyCustomIcon />,
    title: "暂无数据",
  }}
/>
```

传入 `false` 可隐藏图标。

### 如何隐藏操作按钮？

设置 `appearance.showAction` 为 `false`：

```tsx
<EmptyState
  data={{ title: "搜索无结果" }}
  appearance={{ showAction: false }}
/>
```

### 如何隐藏图标区域？

设置 `appearance.showImage` 为 `false`：

```tsx
<EmptyState
  data={{ title: "暂无内容" }}
  appearance={{ showImage: false }}
/>
```

### 如何处理操作按钮点击？

通过 `actions.onAction` 回调处理：

```tsx
<EmptyState
  data={{ title: "还没有项目", actionLabel: "创建第一个项目" }}
  actions={{
    onAction: () => {
      router.push("/projects/new")
    },
  }}
/>
```

### 如何展示加载状态？

通过 `control.isLoading` 控制，组件内部自动展示骨架屏：

```tsx
const [loading, setLoading] = useState(true)

useEffect(() => {
  fetchData().finally(() => setLoading(false))
}, [])

<EmptyState control={{ isLoading: loading }} />
```

### EmptyState 与 Empty 的区别是什么？

- **EmptyState**（本组件）是页面模板组件，提供标题、描述和操作按钮的开箱即用布局，适合作为页面的空状态占位。
- **Empty**（`@/components/ui/empty-state`）是基础布局组件，提供子组件（EmptyHeader、EmptyTitle、EmptyDescription、EmptyContent、EmptyMedia）的灵活组合方式，适合需要完全自定义内部内容的场景。

## LLM 使用规则 {#llm-rules}

当 AI 生成 EmptyState 代码时，应遵守：

1. 从 `@/components/data-display/manifest/empty-state` 导入 `EmptyState`。
2. 使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop，不要直接传入子组件。
3. 空状态标题通过 `data.title` 设置，描述通过 `data.description` 设置。
4. 操作按钮文字通过 `data.actionLabel` 设置，点击回调通过 `actions.onAction` 处理。
5. 自定义图标通过 `data.image` 传入 React 节点，不要传入字符串路径。
6. 操作按钮的显示/隐藏仅通过 `appearance.showAction` 控制。
7. 图标区域的显示/隐藏仅通过 `appearance.showImage` 控制。
8. 加载状态仅通过 `control.isLoading` 控制，组件内部已处理骨架屏的渲染逻辑。
9. 不要引入 Ant Design、ConfigProvider 或 css-in-js 相关依赖。
10. 不要复刻或重新实现 EmptyState 的内部逻辑；它是一个页面模板组件，应直接使用。
11. 不要直接使用 `@/components/ui/empty-state` 的 `Empty` 组件替代本组件，除非需要完全自定义的内部布局。
