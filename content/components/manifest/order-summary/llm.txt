# OrderSummary 订单摘要

OrderSummary 是一个订单费用分解展示组件，通过 Card 容器聚合商品列表、费用明细和总计金额。它支持多商品行项目展示（含缩略图、名称、数量和行小计），以及小计、运费、税费、折扣和最终总计的分项分解。货币格式化通过 `Intl.NumberFormat` 实现，支持任意 ISO 4217 货币代码。

## 何时使用 {#when-to-use}

当需要在结算、支付确认或订单详情等页面中向用户展示订单费用的完整构成时使用。

常见场景：

- 购物车结算页：展示待支付订单的商品清单和费用明细。
- 支付确认页：用户完成支付前的最终金额确认。
- 订单详情页：历史订单的费用构成回顾。
- 发票预览：提供商品级别的费用分解供用户核对。

推荐使用方式：

- 通过 `data` prop 传入完整的订单数据对象。
- 通过 `appearance.currency` 指定目标货币代码。
- 运费为 0 时自动显示 "Free"，无需额外处理。
- 折扣仅当大于 0 时显示，传入 `discountCode` 可渲染折扣码标签。

不推荐：

- 在无商品列表时单独使用——组件核心价值在于逐项费用分解。
- 用于支付流程中的交互步骤——OrderSummary 是纯展示组件，不含任何交互逻辑。
- 用于非货币数值展示——组件内部强制使用 `Intl.NumberFormat` 格式化所有金额。

## 代码演示 {#examples}

### 基础订单摘要

多商品、含运费、税费、折扣和总计的完整订单摘要。

```tsx

const sampleData = {
  items: [
    {
      id: "1",
      name: "Wireless Headphones",
      quantity: 1,
      price: 149.99,
      image: "https://images.unsplash.com/photo-1505740420928-5e560c06d30e?w=120&h=120&fit=crop",
    },
    {
      id: "2",
      name: "USB-C Cable",
      quantity: 2,
      price: 19.99,
    },
    {
      id: "3",
      name: "Phone Case",
      quantity: 1,
      price: 29.99,
      image: "https://images.unsplash.com/photo-1601784551446-20c9e07cdbdb?w=120&h=120&fit=crop",
    },
  ],
  subtotal: 219.96,
  shipping: 5.99,
  tax: 17.60,
  discount: 20,
  discountCode: "SAVE20",
  total: 223.55,
}

export default function Demo() {
  return (
    <OrderSummary
      data={sampleData}
      appearance={{ currency: "USD" }}
    />
  )
}
```

### 免运费订单

`shipping` 为 `0` 时自动显示 "Free"。

```tsx
import { OrderSummary } from "@/components/ui/order-summary-manifest"

export default function Demo() {
  return (
    <OrderSummary
      data={{
        items: [
          { id: "1", name: "Digital Download", quantity: 1, price: 49.99 },
        ],
        subtotal: 49.99,
        shipping: 0,
        tax: 4.00,
        total: 53.99,

      appearance={{ currency: "USD" }}
    />
  )
}
```


## API {#api}

### OrderSummary

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 订单数据对象 | `object` | - |
| data.items | 订单商品列表 | `OrderItem[]` | - |
| data.subtotal | 小计（不含运费、税和折扣） | `number` | - |
| data.shipping | 运费，`0` 时显示 "Free" | `number` | - |
| data.tax | 税费 | `number` | - |
| data.discount | 折扣金额，仅 `> 0` 时显示 | `number` | - |
| data.discountCode | 折扣码，以 Badge 展示 | `string` | - |
| data.total | 最终合计金额 | `number` | - |
| appearance | 外观配置 | `object` | - |
| appearance.currency | 货币代码（ISO 4217） | `string` | `"USD"` |

### OrderItem

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| id | 商品唯一标识 | `string` | 是 |
| name | 商品名称 | `string` | 否 |
| quantity | 购买数量 | `number` | 否 |
| price | 商品单价 | `number` | 否 |
| image | 商品图片 URL | `string` | 否 |

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/manifest/payment/order-summary.tsx`：组件渲染、货币格式化、布局。
- `src/components/ui/manifest-order-summary.tsx`：薄包装，从 manifest 包重新导出。
- `src/components/manifest/payment/types.ts`：OrderItem 类型定义。
- `src/components/manifest/payment/demo/payment.ts`：Demo 默认数据。

关键行为：

1. 组件内部使用 `Intl.NumberFormat` 格式化所有金额。
2. 商品无 `image` 时显示 Package 图标占位，有 `image` 时渲染缩略图。
3. 费用分解行（subtotal、shipping、tax、discount）仅在对应字段不为 `undefined` 时渲染。
4. `shipping` 为 `0` 时显示 "Free" 而非 `$0.00`。
5. `discount` 仅当值 `> 0` 时显示，配合 `discountCode` Badge。
6. `total` 独立显示，通过 `Separator` 分隔。
7. 页脚固定显示 Terms of Service 和 Privacy Policy 免责声明。

## FAQ {#faq}

### 如何切换货币？

```tsx
<OrderSummary data={orderData} appearance={{ currency: "EUR" }} />
```

### 如何隐藏费用分解行？

不传对应字段即可。例如仅显示商品列表：

```tsx
<OrderSummary data={{ items: myItems, total: calculatedTotal }} />
```

### 如何隐藏商品占位图标？

当商品无 `image` 时组件显示 Package 图标。如需完全隐藏，请修改源码或使用 `display: none` 覆盖样式。

## LLM 使用规则 {#llm-rules}

当 AI 生成 OrderSummary 代码时，应遵守：

1. 从 `@/components/ui/order-summary-manifest` 导入 OrderSummary。
2. 通过 `data` prop 传入订单数据对象，包含 items、subtotal、shipping、tax、discount、discountCode、total。
3. 每个 item 必须包含 `id`，建议传入 `name`、`quantity`、`price`。
4. `data.total` 应等于 `subtotal + shipping + tax - discount`，确保数值一致性。
5. 通过 `appearance.currency` 指定货币代码，默认 "USD"。
6. 运费为 0 时自动显示 "Free"，无需额外处理。
7. 折扣仅当 `discount > 0` 时显示；传入 `discountCode` 展示折扣码 Badge。
8. 组件内部使用 `Intl.NumberFormat`，无需额外引入货币格式化库。
