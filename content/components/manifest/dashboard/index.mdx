Dashboard 仪表盘。用于构建数据概览页面，聚合统计卡片、图表区域和最近活动等模块。

<h2 id="when-to-use">何时使用</h2>

当需要在管理后台或数据面板中展示关键业务指标的一站式概览时使用。适用于：

- 管理后台首页，展示系统核心运营指标。
- SaaS 产品的用户工作台，提供个人或团队的数据摘要。
- 数据分析平台的概览页面，快速了解数据趋势。
- 电商后台的销售总览，包含收入、订单、用户等核心维度。

Dashboard 以模块化方式组织数据展示：

- 统计卡片区域，展示关键指标的数值和趋势变化。
- 图表区域，支持折线图、柱状图等数据可视化。
- 最近活动列表，展示最新的操作记录或事件。
- 可配置的数据源，通过 `data` prop 注入各模块的数据。

<h2 id="api">API</h2>

### Dashboard

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 仪表盘数据配置 | `object` | — |
| data.stats | 统计卡片列表 | `StatCardItem[]` | — |
| data.recentActivity | 最近活动列表 | `ActivityItem[]` | — |
| data.chartData | 图表数据配置 | `ChartData` | — |
| appearance | 外观配置 | `object` | — |
| appearance.title | 仪表盘标题 | `string` | `"Dashboard"` |
| appearance.subtitle | 标题下方描述文字 | `string` | — |
| appearance.statColumns | 统计卡片列数 | `2 \| 3 \| 4` | `4` |
| appearance.showRecentActivity | 是否显示最近活动区域 | `boolean` | `true` |
| appearance.showCharts | 是否显示图表区域 | `boolean` | `true` |
| actions | 回调配置 | `object` | — |
| actions.onActivityClick | 点击活动项的回调 | `(item: ActivityItem) => void` | — |
| actions.onRefresh | 刷新数据回调 | `() => void` | — |
| control | 控制配置 | `object` | — |
| control.isLoading | 数据加载状态 | `boolean` | `false` |
| control.error | 数据加载错误信息 | `string \| null` | `null` |

### StatCardItem

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| label | 指标名称 | `string` | 是 |
| value | 指标数值 | `string \| number` | 是 |
| change | 变化百分比 | `number` | 否 |
| trend | 变化趋势方向 | `"up" \| "down" \| "neutral"` | 否 |
| icon | 自定义图标 | `React.ReactNode` | 否 |

### ActivityItem

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| id | 活动唯一标识 | `string` | 是 |
| action | 活动描述 | `string` | 是 |
| time | 发生时间 | `string` | 否 |
| user | 关联用户 | `string` | 否 |

<h2 id="faq">常见问题</h2>

### 如何注入自定义数据？

通过 `data.stats` 和 `data.recentActivity` 注入数据：

```tsx
<Dashboard
  data={{
    stats: [
      { label: "总收入", value: "¥128,430", change: 12.5, trend: "up" },
    ],
    recentActivity: [
      { id: "1", action: "新订单", time: "2 分钟前", user: "张三" },
    ],
  }}
/>
```

不传入 `data` 时，组件使用内置的默认演示数据。

### 如何处理加载状态？

通过 `control.isLoading` 控制：

```tsx
const [loading, setLoading] = useState(true)

<Dashboard control={{ isLoading: loading }} />
```

加载中时统计卡片和最近活动区域显示骨架屏占位。

### 如何处理数据加载错误？

通过 `control.error` 传入错误信息：

```tsx
<Dashboard control={{ error: "数据加载失败，请稍后重试" }} />
```

错误状态下各区域显示错误提示，并渲染重试按钮（触发 `actions.onRefresh`）。

### 如何自定义仪表盘标题？

通过 `appearance.title` 和 `appearance.subtitle`：

```tsx
<Dashboard
  appearance={{
    title: "我的工作台",
    subtitle: "欢迎回来，以下是您的业务概览",
  }}
/>
```

### 如何调整统计卡片的列数？

通过 `appearance.statColumns` 设置：

```tsx
<Dashboard appearance={{ statColumns: 3 }} />
```

支持 `2`、`3`、`4` 列布局，在移动端自动降为 2 列。

### Dashboard 与 StatCard 的关系？

- **StatCard**：独立的统计卡片网格组件，仅展示指标卡片。
- **Dashboard**：页面模板级别的仪表盘组件，内部组合了 StatCard、最近活动列表和图表区域，提供开箱即用的仪表盘页面。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Dashboard 代码时：

1. 从 `@/components/templates/manifest/miscellaneous/dashboard` 导入 `Dashboard`。
2. 使用 `data`、`appearance`、`actions`、`control` 四个顶层 prop 配置组件。
3. 统计数据通过 `data.stats` 传入，每项包含 `label`、`value` 和可选的 `change`、`trend`。
4. 最近活动通过 `data.recentActivity` 传入，不传时使用默认演示数据。
5. 加载状态通过 `control.isLoading` 控制，错误状态通过 `control.error` 控制。
6. 标题和副标题通过 `appearance.title` 和 `appearance.subtitle` 自定义。
7. Dashboard 是页面模板组件，应直接使用，不要手动拼装 StatCard 和活动列表来重建它。
8. 不要引入 Ant Design 相关依赖。
