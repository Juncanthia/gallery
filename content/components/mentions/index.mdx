提及 Mentions。用于在文本输入中通过特殊前缀（如 `@`、`#`）触发选项列表，快速插入提及引用。支持单行 input 和多行 textarea 两种模式。

<h2 id="when-to-use">何时使用</h2>

当需要在文本输入中通过特殊前缀触发选项列表，快速插入提及引用时使用。

常见场景：
- 评论中的 `@用户名` 提及
- 文档中的 `#标签` 引用
- 邮件收件人输入
- 聊天中的成员引用

不推荐：
- 作为普通下拉选择器使用——使用 Select 或 Combobox
- 不需要前缀触发的搜索输入——使用 AutoComplete
- 固定选项列表无过滤需求——使用 Select

<h2 id="api">API</h2>

### Mentions

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 受控值 | `string` | - |
| defaultValue | 非受控默认值 | `string` | `""` |
| onChange | 值变更回调 | `(value: string) => void` | - |
| onSelect | 选中选项回调，可获知选项对象和触发前缀 | `(option: MentionsOptionType, prefix: string) => void` | - |
| prefix | 触发提及的前缀字符 | `string` \| `string[]` | `"@"` |
| options | 候选选项列表 | `MentionsOptionType[]` | `[]` |
| split | 提及值与后续文本之间的分隔符 | `string` | `" "` |
| filterOption | 自定义过滤函数 | `(input: string, option: MentionsOptionType) => boolean` | 内置大小写不敏感包含匹配 |
| placement | 下拉菜单弹出方向 | `"top"` \| `"bottom"` | `"bottom"` |
| rows | 文本框行数。`> 1` 时渲染 textarea 而非 input | `number` | `1` |
| placeholder | 占位文本 | `string` | - |
| disabled | 是否禁用 | `boolean` | `false` |
| className | 附加样式类名 | `string` | - |
| maxLength | 最大输入长度 | `number` | - |

### MentionsOptionType

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 选项值（插入到文本中的内容） | `string` | 必填 |
| label | 选项展示文本（下拉列表中显示） | `ReactNode` | 与 `value` 相同 |

<h2 id="faq">常见问题</h2>

### 如何支持多个触发前缀？

传入前缀数组即可，如同时支持 `@` 和 `#`：

```tsx
<Mentions prefix={["@", "#"]} options={users} />
```

### 如何自定义过滤逻辑？

使用 `filterOption` 回调覆盖默认的大小写不敏感匹配：

```tsx
<Mentions
  options={users}
  filterOption={(input, option) =>
    String(option.label).startsWith(input)
  }
/>
```

### 如何在选择选项后获取回调？

使用 `onSelect` 回调，可获知用户选择了哪个选项以及使用的触发前缀：

```tsx
<Mentions
  onSelect={(option, prefix) => {
    console.log(`选择了 ${option.label}，前缀为 ${prefix}`)
  }}
/>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/mentions` 导入。
2. `options` 为 `MentionsOptionType[]`，每个选项有 `value` 和可选的 `label`。
3. 多前缀使用 `prefix` 数组。
4. 多行模式设置 `rows` > 1。
5. 通过 `filterOption` 自定义过滤，通过 `onSelect` 捕获选择事件。
6. 不要引入 Ant Design 相关依赖。
