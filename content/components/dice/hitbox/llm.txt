# dice/hitbox - LLM Rules

## Import
```
import { Hitbox } from "@/components/ui/dice-hitbox"
```

## Props
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| size | `"default" \| "sm" \| "lg" \| string` | `"default"` | Hit area expansion size. Preset values: sm=8px, default=12px, lg=16px. Custom CSS length values (e.g. "20px") injected via `--size` CSS variable. |
| position | `"all" \| "top" \| "bottom" \| "left" \| "right" \| "vertical" \| "horizontal"` | `"all"` | Which sides to expand the hit area on |
| radius | `"none" \| "sm" \| "md" \| "lg" \| "full"` | `"none"` | Border radius of the expanded hit area (should match child element's border radius) |
| debug | `boolean` | `false` | Show hit area with red dashed border and semi-transparent red background for debugging |
| className | `string` | - | Additional CSS classes |
| children | `React.ReactNode` | - | Single interactive child element to expand hit area for |

## Code Examples

### Basic (expand all sides by 12px)
```tsx
import { Hitbox } from "@/components/ui/dice-hitbox"

<Hitbox>
  <button className="size-8 rounded border">×</button>
</Hitbox>
```

### With debug (visualize hit area during development)
```tsx
<Hitbox debug>
  <button className="size-8 rounded border">×</button>
</Hitbox>
```

### Different sizes
```tsx
<Hitbox size="sm">  {/* 8px */}
  <button className="size-8 rounded border">×</button>
</Hitbox>

<Hitbox size="default">  {/* 12px */}
  <button className="size-8 rounded border">×</button>
</Hitbox>

<Hitbox size="lg">  {/* 16px */}
  <button className="size-8 rounded border">×</button>
</Hitbox>
```

### Custom size (arbitrary CSS length)
```tsx
<Hitbox size="20px">
  <button className="size-8 rounded border">×</button>
</Hitbox>
```

### Directional expansion
```tsx
<Hitbox position="top">      {/* expand only above */}
  <button className="size-8 rounded border">×</button>
</Hitbox>

<Hitbox position="horizontal">  {/* expand left + right */}
  <button className="size-8 rounded border">×</button>
</Hitbox>

<Hitbox position="vertical">    {/* expand top + bottom */}
  <button className="size-8 rounded border">×</button>
</Hitbox>
```

### Matching child border-radius
```tsx
<Hitbox radius="full">
  <button className="flex size-8 items-center justify-center rounded-full border">×</button>
</Hitbox>
```

## Constraints
- DO NOT import from `antd`. Use `@/components/ui/dice-hitbox`.
- Hitbox uses Radix Slot; it must wrap a single child element and merges attributes into it.
- The hit area is created via `::after` pseudo-element — it does NOT change the child's box model or layout.
- `debug` should be removed in production code.
- `radius` should match the child element's visual border-radius for consistent touch feedback.
- Do NOT nest multiple Hitbox components.
- In `overflow: hidden` containers, the expanded hit area will be clipped at the boundary.
- The base styles use CSS custom property `--size` with defaults: `--size-default: 12px`, `--size-sm: 8px`, `--size-lg: 16px`.
- Based on Radix Slot. No `data-slot` is exposed; attributes are merged into the child element.
