# dice/client-only - LLM Rules

## Import
```
import { ClientOnly } from "@/components/ui/client-only"
```

## Props
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| children | `React.ReactNode` | - | Content rendered only on client-side |
| fallback | `React.ReactNode` | `null` | Placeholder rendered during SSR |

## Code Examples

### Basic
```tsx
import { ClientOnly } from "@/components/ui/client-only"

<ClientOnly fallback={<div>Loading...</div>}>
  <div>Client-side content</div>
</ClientOnly>
```

### Without fallback (renders null during SSR)
```tsx
import { ClientOnly } from "@/components/ui/client-only"

<ClientOnly>
  <Chart />
</ClientOnly>
```

### With skeleton placeholder
```tsx
import { ClientOnly } from "@/components/ui/client-only"

<ClientOnly fallback={<div className="animate-pulse h-32 w-full rounded bg-muted" />}>
  <DataVisualization />
</ClientOnly>
```

## How It Works
Uses `React.useSyncExternalStore` to detect client-side mounting:
- `getServerSnapshot()` returns `false` -> renders `fallback`
- `getSnapshot()` returns `true` -> renders `children`
- After hydration, the component synchronously switches to `children`

## Constraints
- DO NOT import from `antd`. Use `@/components/ui/client-only`.
- `children` is required. `fallback` defaults to `null`.
- Only wrap content that truly depends on browser-only APIs (`window`, `document`, `localStorage`, etc.).
- Do NOT use `typeof window === "undefined"` checks inside `ClientOnly` — the component already handles client detection.
- Avoid wrapping entire pages — it harms SEO and initial render performance.
- Match fallback height with expected children height to prevent layout shift on hydration.
- This component has no styling, no `data-slot` attributes, and no external dependencies beyond React.
