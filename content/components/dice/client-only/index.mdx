ClientOnly 仅客户端渲染。在 SSR 阶段渲染 fallback 占位，客户端水合后渲染 children，用于包裹依赖浏览器 API 的内容。

<h2 id="when-to-use">何时使用</h2>

当内容依赖浏览器专属 API（如 `window`、`document`、`localStorage`、`matchMedia` 等），在服务端渲染阶段无法正常执行时使用。ClientOnly 在 SSR 阶段返回 fallback，直至客户端水合后才渲染真实 children。

常见场景：
- 依赖 `window` 或 `document` 的计算逻辑
- 使用 Web API（如 `IntersectionObserver`、`ResizeObserver`）
- 第三方仅浏览器可用的库
- 需要 `localStorage` / `sessionStorage` 读取的持久化状态
- 依赖于客户端设备信息（如 `navigator.userAgent`）的内容

不推荐：
- 纯静态内容——无需包裹，直接渲染即可。
- 仅需要避免水合不匹配的样式——使用 CSS `suppressHydrationWarning` 或 `useEffect` 延迟渲染。
- 大量内容频繁切换 fallback/children——会造成 DOM 重建开销。

<h2 id="examples">代码演示</h2>

### 基础用法

`children` 为客户端渲染内容，`fallback` 为 SSR 阶段的占位内容。fallback 默认 `null`。

<ComponentShowcase id="dice/client-only" path="dice/client-only/basic" />

```tsx
import { ClientOnly } from "@/components/ui/dice-client-only"

export default function Demo() {
  return (
    &lt;ClientOnly fallback={&lt;div&gt;加载中...&lt;/div&gt;}&gt;
      &lt;div&gt;客户端内容&lt;/div&gt;
    &lt;/ClientOnly&gt;
  )
}
```

<h2 id="api">API</h2>

### ClientOnly

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 客户端渲染的内容，SSR 阶段不渲染 | `React.ReactNode` | - |
| fallback | SSR 阶段渲染的占位内容 | `React.ReactNode` | `null` |

> ClientOnly 通过 `React.useSyncExternalStore` 检测客户端挂载状态，SSR 阶段返回 `false` 以触发 fallback。

<h2 id="faq">常见问题</h2>

### ClientOnly 和 `typeof window === "undefined"` 判断有什么区别？

直接在组件内使用 `typeof window === "undefined"` 会引发水合不匹配错误（SSR 渲染 null，客户端渲染内容，DOM 不一致）。ClientOnly 使用 `useSyncExternalStore` 作为 React 认可的客户端检测方式，确保 SSR 和客户端渲染结果在第一次提交时一致。

### fallback 和 children 切换时会有闪烁吗？

仅在首次水合时进行切换，后续不会再次触发。fallback 到 children 的切换是同步完成的，不会产生可见的中间状态。如果 fallback 和 children 的高度差异较大，建议为两者设置相同的最小高度以避免布局跳动。

### 可以用 ClientOnly 包裹整个页面吗？

技术上可以，但不推荐。客户端渲染会导致 SEO 内容不可抓取。仅在页面中真正依赖浏览器 API 的最小区域内使用 ClientOnly。

### 如何实现加载动画？

将动画元素作为 `fallback` 传入：

```tsx
&lt;ClientOnly fallback={&lt;div className="animate-pulse h-8 w-full rounded bg-muted" /&gt;}&gt;
  &lt;ExpensiveChart /&gt;
&lt;/ClientOnly&gt;
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/dice-client-only` 导入 ClientOnly，不要从 `antd` 导入。
2. `children` 为必填 prop，`fallback` 可选，默认 `null`。
3. 仅包裹真正依赖浏览器 API 的内容，避免过度使用影响 SEO 和首屏渲染。
4. fallback 应尽量与 children 占据相同空间，避免水合时的布局跳动。
5. 不要在 ClientOnly 内部再嵌套另一层 `typeof window` 判断——ClientOnly 本身已处理客户端检测。
6. 不要引入 Ant Design 相关依赖。
