Portal 传送门。将子节点渲染到父组件 DOM 层级之外的指定容器中，用于突破 CSS 层叠上下文限制（如 z-index、overflow: hidden）的场景。

&lt;h2 id="when-to-use"&gt;何时使用&lt;/h2&gt;

当需要将内容渲染到当前组件树之外的 DOM 节点时使用 Portal。常见场景：

- 对话框、下拉菜单、提示框等需要突破父容器 `overflow: hidden` 或 `z-index` 限制的浮层组件。
- 将内容挂载到 `document.body` 末尾，避免被父级 CSS 属性裁剪。
- 将 React 子树挂载到由外部库或服务端渲染创建的 DOM 容器中。

Portal 本身不创建任何视觉元素，只决定子节点的 DOM 挂载位置。子节点的样式、事件、状态仍由 React 组件树管理。

### 指定容器

通过 `container` 属性将内容挂载到任意 DOM 节点。

```tsx
import { Portal } from "@/components/general/dice/headless/portal"
import { useRef, useEffect, useState } from "react"

export default function Demo() {
  const containerRef = useRef&lt;HTMLDivElement&gt;(null)
  const [mounted, setMounted] = useState(false)

  useEffect(() =&gt; {
    setMounted(true)
  }, [])

  return (
    &lt;div&gt;
      &lt;div ref={containerRef} className="rounded-md border border-dashed p-4 text-sm text-muted-foreground"&gt;
        自定义容器区域
      &lt;/div&gt;
      {mounted &amp;&amp; (
        &lt;Portal container={containerRef.current}&gt;
          &lt;span className="text-sm font-medium text-primary"&gt;
            已传送至自定义容器
          &lt;/span&gt;
        &lt;/Portal&gt;
      )}
    &lt;/div&gt;
  )
}
```

&lt;h2 id="api"&gt;API&lt;/h2&gt;

### Portal

基于 Radix Slot 原语，支持 `asChild` 用法。底层调用 `ReactDOM.createPortal`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| container | 目标容器 DOM 节点。服务端渲染时为 `null` | `Element` \| `DocumentFragment` \| `null` | `document.body` |
| asChild | 使用 Radix Slot 合并到子元素，不额外生成包裹层 | `boolean` | `false` |
| children | 需要传送的内容 | `React.ReactNode` | - |
| className | 附加 className（当不启用 `asChild` 时应用于包裹元素） | `string` | - |

支持所有 Radix Slot 的属性。

&lt;h2 id="faq"&gt;常见问题&lt;/h2&gt;

### Portal 中的事件冒泡会怎样？

Portal 中的事件冒泡遵循 **React 组件树**而非 DOM 树。这意味着在 Portal 内部触发的事件可以被子组件树中的祖先组件捕获，即使它们在 DOM 中位于不同位置。

### 和直接使用 `ReactDOM.createPortal` 有什么区别？

本 Portal 组件封装了 SSR 安全检测（`useSyncExternalStore`），确保在服务端渲染时不尝试访问 `document.body`（返回 `null`）。手动调用 `createPortal` 需自行处理 SSR 兼容，否则会导致 hydration 错误。

### `container` 为 `null` 时会发生什么？

当 `container` 为 `null`（包括 SSR 阶段）时，Portal 返回 `null`，不渲染任何内容。客户端水合后，`container` 默认为 `document.body`，子内容正常渲染。

### 什么时候需要指定 `container`？

默认挂载到 `document.body` 已满足大多数场景。当需要将内容挂载到特定容器（如第三方库创建的 DOM 节点、Shadow DOM、自定义 overlay 容器）时，使用 `container` 属性指定。

### Portal 内的内容如何受 CSS 影响？

Portal 将内容移动到目标容器的 DOM 位置，因此内容的 CSS 层叠上下文由目标容器决定，而非 React 组件在源码中的位置。这是在 `overflow: hidden` 父级中显示浮层的关键机制。

&lt;h2 id="llm-rules"&gt;LLM 使用规则&lt;/h2&gt;

1. 从 `@/components/general/dice/headless/portal` 导入 Portal。
2. Portal 仅负责 DOM 挂载位置管理，不参与样式或布局逻辑。
3. 当组件需要突破父级 `overflow: hidden`、`z-index` 等 CSS 限制时，使用 Portal 包裹浮层内容。
4. `container` 属性接受 DOM 节点引用，在 SSR 环境中确保传入 `null` 或使用条件渲染。
5. Portal 内部的事件处理遵循 React 合成事件冒泡规则，无需特殊处理。
6. 不要在 Portal 内放置复杂的全局状态管理逻辑；Portal 仅是渲染位置的转移。
7. 需要合并到子元素时使用 `asChild`，避免额外的 DOM 包裹。
