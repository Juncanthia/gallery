用于将内容从视觉呈现中隐藏，但仍保留给屏幕阅读器等辅助技术访问。

<h2 id="when-to-use">何时使用</h2>

当需要在 UI 中提供辅助信息，但又不希望它在视觉上占据空间时使用。常见场景：

- 为纯图标按钮提供可读文本标签。
- 在列表或表格中补充选中数量、筛选状态等辅助描述。
- 在语义化标题或区域标签中，补充视觉上下文以外的额外说明。
- 为交互控件提供屏幕阅读器专属的状态提示。

VisuallyHidden 通过内联 CSS（clip、position、overflow 等）将内容完全移出可视区域，同时保留其在 DOM 和可访问性树中的存在。与 `display: none` 或 `visibility: hidden` 不同，屏幕阅读器仍能正常读取其内容。

<h2 id="examples">代码演示</h2>

VisuallyHidden 包裹的内容在视觉上不可见，但对辅助技术完全可访问。支持通过 `asChild` 将隐藏样式合并到子元素上。

```tsx
import { VisuallyHidden } from "@/components/ui/dice-visually-hidden"

export default function Demo() {
  return (
    <div>
      <VisuallyHidden>屏幕阅读器可见内容</VisuallyHidden>
    </div>
  )
}
```

## API {#api}

### VisuallyHidden

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 使用 Radix Slot，将隐藏样式合并到子元素上 | `boolean` | `false` |
| children | 需要隐藏的视觉内容，屏幕阅读器仍可访问 | `React.ReactNode` | - |
| className | 附加 className | `string` | - |
| style | 附加内联样式，会与隐藏样式合并 | `React.CSSProperties` | - |

同时支持所有原生 `div` 元素的属性。

## 样式模型 {#styling-model}

VisuallyHidden 使用一套经典的可访问性隐藏方案，通过内联样式实现：

- `border: 0`、`clip: rect(0 0 0 0)`、`clipPath: inset(50%)`：将元素裁剪为零尺寸。
- `height: 1px`、`width: 1px`：保持最小尺寸，避免被某些浏览器完全移除。
- `margin: -1px`：消除对周围布局的潜在影响。
- `overflow: hidden`：防止内容溢出。
- `position: absolute`：脱离文档流。
- `whiteSpace: nowrap`：防止文本换行影响布局。
- `padding: 0`：移除内边距。

这套方案是业界标准做法，广泛用于 Bootstrap、Chakra UI、Radix UI 等主流组件库。

## 可访问性 {#accessibility}

- 隐藏的内容保留在 DOM 和可访问性树中，屏幕阅读器可正常读取。
- 不为视觉用户提供任何视觉反馈。
- 适合配合 `aria-label` / `aria-labelledby` 等属性使用，补充更丰富的上下文描述。
- 在 `asChild` 模式下，子元素的原生语义保持不变，隐藏样式仅影响视觉呈现。

## FAQ {#faq}

### 和 `display: none` 有什么区别？

`display: none` 会将元素同时从视觉渲染和可访问性树中移除，屏幕阅读器无法访问。VisuallyHidden 只对视觉用户隐藏，辅助技术仍然可以读取。

### 和 `aria-label` 有什么区别？

`aria-label` 直接提供可访问名称，不产生任何 DOM 内容。VisuallyHidden 通过在 DOM 中放置真实文本节点实现，适合需要结构化内容（如内联链接、强调文本）的场景，也适合需要文本内容被翻译工具识别的场景。

### 和 `visibility: hidden` 有什么区别？

`visibility: hidden` 的元素在屏幕阅读器中通常仍可访问，但会占据布局空间。VisuallyHidden 将元素移出文档流，不占用任何视觉空间。

### 什么时候用 `asChild`？

当你需要将隐藏样式应用到一个已有的语义元素上，而不是额外包裹一层 `div` 时使用。例如：

```tsx
<VisuallyHidden asChild>
  <span>当前搜索关键词：按钮组件</span>
</VisuallyHidden>
```

注意 `asChild` 模式下子元素的 children 结构完全由调用侧负责，VisuallyHidden 只注入隐藏样式。

### 纯图标按钮必须用 VisuallyHidden 还是 aria-label？

两种方式都可以。推荐纯图标按钮使用 `aria-label`（更简洁），VisuallyHidden 更适合需要结构化文本或有翻译需求的场景。

### 放置大量隐藏文本是否影响性能？

VisuallyHidden 渲染的是普通 DOM 节点，少量隐藏文本对性能影响微乎其微。但不应将大量冗余内容放入 VisuallyHidden；隐藏文本应简洁、有针对性。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 VisuallyHidden 代码时，应遵守：

1. 从 `@/components/ui/dice-visually-hidden` 导入 VisuallyHidden。
2. 隐藏内容应为屏幕阅读器提供有意义的辅助信息，不要放无关内容。
3. 纯图标按钮添加文本标签时，优先考虑 `aria-label`；VisuallyHidden 适合需要富文本或结构化内容的场景。
4. 需要合并到已有元素时使用 `asChild`。
5. 不要在 VisuallyHidden 内部放置复杂交互组件；它仅用于纯文本或简单内联内容。
6. 不要用 VisuallyHidden 隐藏大量内容或整块布局；它适合简短、聚焦的描述性文本。
7. VisuallyHidden 不支持 `asChild` 以外的 Radix Slot 高级用法。

## 实现说明 {#implementation-notes}

核心文件：

- `src/components/dice/components/visually-hidden.tsx`：组件实现，使用内联 CSS 实现视觉隐藏。

关键行为：

1. 默认渲染为 `<div>`，`asChild` 时使用 Radix `Slot` 合并到子元素。
2. 隐藏样式通过 `style` 属性直接内联，不依赖外部 CSS。
3. 调用侧传入的 `style` 会与隐藏样式合并，后者的优先级高于隐藏样式（用于逃生口）。
4. 不包含任何动画、状态管理或事件处理逻辑。
