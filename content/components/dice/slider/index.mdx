Slider 滑块。通过拖拽滑块在连续或离散区间内选择一个或多个数值。基于 Radix Slider 和 Tailwind CSS 实现。

<h2 id="when-to-use">何时使用</h2>

当需要在数值区间内进行选择时使用。Slider 提供直观的拖拽交互，适合调节音量、亮度、价格范围等参数。

常见场景：
- 音量、亮度、透明度等连续值调节
- 价格区间筛选（范围模式）
- 设置项中的步进式数值选择
- 表单中替代数字输入框的直观数值选择器

不推荐：
- 需要精确数值输入的场景（配合 Input 使用，或直接用 InputNumber）
- 离散选项较少时（建议使用 Radio 或 Select）
- 单个布尔值开关（使用 Switch）

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `value` / `defaultValue` 控制当前值，`max` 设定上限，`onValueChange` 监听变化。

<ComponentShowcase id="dice/slider" path="dice/slider/basic" />

```tsx
import { Slider } from "@/components/ui/slider-dice"

export default function Demo() {
  return (
    &lt;Slider defaultValue={50} max={100} step={1} /&gt;
  )
}
```

<h2 id="api">API</h2>

### Slider

基于 Radix Slider.Root，支持其全部属性。以下列出核心 API。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| defaultValue | 默认值。传入数组启用范围模式（多个滑块） | `number` \| `number[]` | - |
| value | 受控值 | `number` \| `number[]` | - |
| onValueChange | 值变化回调，在拖拽过程中持续触发 | `(value: number \| number[]) => void` | - |
| onValueCommit | 值提交回调，在拖拽结束后触发 | `(value: number \| number[]) => void` | - |
| min | 最小值 | `number` | `0` |
| max | 最大值 | `number` | `100` |
| step | 步长 | `number` | `1` |
| disabled | 是否禁用 | `boolean` | `false` |
| orientation | 方向 | `"horizontal"` \| `"vertical"` | `"horizontal"` |
| inverted | 是否反转方向 | `boolean` | `false` |
| minStepsBetweenThumbs | 范围模式下滑块之间的最小步数 | `number` | `0` |
| name | 表单字段名称 | `string` | - |
| className | 容器附加样式类 | `string` | - |

### 语义槽位

| data-slot | 元素 | 说明 |
| --- | --- | --- |
| `slider` | SliderPrimitive.Root | 滑块根容器 |
| `slider-track` | SliderPrimitive.Track | 轨道区域 |
| `slider-range` | SliderPrimitive.Range | 已填充范围 |
| `slider-thumb` | SliderPrimitive.Thumb | 滑块手柄，范围模式有多个 |

<h2 id="faq">常见问题</h2>

### 如何实现范围选择？

将 `value` 或 `defaultValue` 设为包含两个元素的数组，组件会自动渲染两个滑块手柄：

```tsx
&lt;Slider defaultValue={[20, 80]} max={100} /&gt;
```

### 如何垂直放置？

设置 `orientation="vertical"`，并确保父容器有明确的高度：

```tsx
&lt;div className="h-44"&gt;
  &lt;Slider defaultValue={50} orientation="vertical" /&gt;
&lt;/div&gt;
```

### value 和 defaultValue 有何区别？

`defaultValue` 是非受控模式，组件内部管理状态；`value` + `onValueChange` 是受控模式，由调用方管理状态。二者不可同时使用。

### onValueChange 和 onValueCommit 有何区别？

`onValueChange` 在拖拽过程中持续触发（高频），适合实时预览；`onValueCommit` 在用户松手后才触发一次，适合提交或请求场景。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/slider-dice` 导入 Slider，不要从 `antd` 导入。
2. 只使用本地源码中存在的 props（见上方 API 表），不生成 AntD 专属的 `tooltip`、`marks`、`dots` 等属性。
3. 范围选择通过 `value={[min, max]}` 或 `defaultValue={[min, max]}` 传递数组实现。
4. 受控模式下必须同时提供 `value` 和 `onValueChange`。
5. 表单场景需设置 `name` 属性以参与原生表单提交。
6. 不要引入 Ant Design 相关依赖。
