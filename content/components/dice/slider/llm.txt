# Slider 滑块

Slider 是 dice-ui 滑块组件，基于 Radix Slider 和 Tailwind CSS 实现。支持单值和范围模式，提供单选、拖拽交互。

本组件是项目内 dice-ui 库的基础组件，API 和行为以源码为准。

## 何时使用 {#when-to-use}

适合需要用户在连续或离散区间内选择数值的界面。

常见场景：
- 音量、亮度、透明度等连续值调节
- 价格区间筛选（范围模式）
- 设置项中的步进式数值选择
- 表单中替代数字输入框的直观数值选择器

不推荐：
- 需要精确数值输入的场景（配合 Input 使用）
- 离散选项较少时（建议使用 Radio 或 Select）
- 单个布尔值开关（使用 Switch）

## 代码演示 {#examples}

### 基础用法

```tsx
import { Slider } from "@/components/ui/dice-slider"

export default function Demo() {
  return (
    <Slider defaultValue={50} max={100} step={1} />
  )
}
```

### 范围选择

```tsx
import { Slider } from "@/components/ui/dice-slider"

export default function Demo() {
  return (
    <Slider defaultValue={[20, 80]} max={100} minStepsBetweenThumbs={1} />
  )
}
```

## 核心导出 {#exports}

- `Slider`

## API 摘要 {#api}

```ts
type SliderProps = React.ComponentProps<typeof SliderPrimitive.Root> & {
  className?: string;
  defaultValue?: number | number[];
  value?: number | number[];
  min?: number;
  max?: number;
};
```

核心属性：

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| defaultValue | 默认值。数组为范围模式 | `number \| number[]` | - |
| value | 受控值 | `number \| number[]` | - |
| onValueChange | 拖拽中持续触发 | `(value: number \| number[]) => void` | - |
| onValueCommit | 拖拽结束后触发 | `(value: number \| number[]) => void` | - |
| min | 最小值 | `number` | `0` |
| max | 最大值 | `number` | `100` |
| step | 步长 | `number` | `1` |
| disabled | 禁用 | `boolean` | `false` |
| orientation | 方向 | `"horizontal" \| "vertical"` | `"horizontal"` |
| inverted | 反转方向 | `boolean` | `false` |
| minStepsBetweenThumbs | 滑块最小间距（步数） | `number` | `0` |
| name | 表单字段名 | `string` | - |

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `data-slot="slider"` — SliderPrimitive.Root
- `data-slot="slider-track"` — SliderPrimitive.Track
- `data-slot="slider-range"` — SliderPrimitive.Range
- `data-slot="slider-thumb"` — SliderPrimitive.Thumb

额外状态属性：
- `data-orientation="horizontal"` | `data-orientation="vertical"` — 方向
- `data-[disabled]` — 禁用状态 opacity-50

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 滑块轨道：`bg-muted` 圆角背景，水平时 `h-1.5`，垂直时 `w-1.5`。
- 滑块手柄：`size-4` 圆形，`border border-primary bg-background shadow-sm`，hover/focus 时 `ring-4 ring-ring/50`。
- 已填充范围：`bg-primary`。
- 禁用状态：容器 `opacity-50`，手柄 `pointer-events-none opacity-50`。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-primary`、`ring-ring` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：
- `src/components/dice/ui/slider.tsx`

直接依赖：
- `radix-ui` (Slider.Root, Slider.Track, Slider.Range, Slider.Thumb)
- `@hyper/dice-ui/lib/utils` (cn)

实现注意：
1. 组件 API 以本地 TypeScript 类型为准。
2. 范围模式通过数组值 `[min, max]` 自动渲染多个 Thumb。
3. 样式应优先通过组件 props 或 `className` 扩展点表达。
4. 不要把 Radix 文档中的全量 API 视为本地已支持能力——以本地 `src/components/dice/ui/slider.tsx` 源码为准。

## 可访问性 {#accessibility}

- 基于 Radix Slider primitive，继承其键盘交互（Arrow 键调节、Home/End 跳转边界、PageUp/PageDown 大步跳转）。
- 滑块手柄为可聚焦的交互元素。
- 禁用状态通过 `disabled` 属性控制，同时移除可聚焦能力。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/dice/ui/slider.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 未实现 | `tooltip`、`marks`、`dots`、`range`（用数组值替代）、`tooltipVisible`、`getTooltipPopupContainer` |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Slider 代码时，应遵守：

1. 从 `@/components/ui/dice-slider` 导入，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. 不要生成 AntD 专属的 `tooltip`、`marks`、`dots`、`className`、`styles`、`ConfigProvider`、Design Token 配置。
4. 范围选择通过 `value={[min, max]}` 或 `defaultValue={[min, max]}` 传递数组实现，不使用 AntD 的 `range` prop。
5. 受控模式下必须同时提供 `value` 和 `onValueChange`。
6. 交互组件必须保留键盘和 ARIA 语义，Radix primitive 已内置。
7. 如果不确定某个 AntD API 是否存在，先检查 `src/components/dice/ui/slider.tsx`，不要凭 AntD 记忆生成。
