Switch 开关。用于在两种互斥状态间切换，基于 Radix Switch 和 Tailwind CSS 实现。

<h2 id="when-to-use">何时使用</h2>

当需要用户在开/关两种状态间即时切换时使用。Switch 提供直观的滑块交互，适合设置项、开关控制等场景。

常见场景：
- 设置页中的功能开关（通知、定位、暗黑模式等）
- 列表项中的快速开关操作
- 表单中的布尔值字段
- 状态切换（启用/禁用某个选项）

不推荐：
- 需要多种选项时（使用 Checkbox 或 Radio）
- 需要提交操作才能生效时（使用 Button）
- 需要显示加载状态时（使用带 loading 的 Button）

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `checked` / `defaultChecked` 控制开关状态，`onCheckedChange` 监听变化。

<ComponentShowcase id="dice/switch" path="dice/switch/basic" />

```tsx
import { Switch } from "@/components/ui/switch"

export default function Demo() {
  return (
    &lt;Switch defaultChecked /&gt;
  )
}
```

<h2 id="api">API</h2>

### Switch

基于 Radix Switch.Root，支持其全部属性。以下列出核心 API。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| checked | 受控选中状态 | `boolean` | - |
| defaultChecked | 默认选中状态（非受控） | `boolean` | - |
| onCheckedChange | 选中状态变化回调 | `(checked: boolean) => void` | - |
| disabled | 是否禁用 | `boolean` | `false` |
| required | 是否必填（表单场景） | `boolean` | `false` |
| name | 表单字段名称 | `string` | - |
| value | 选中时提交的值 | `string` | `"on"` |
| className | 附加样式类 | `string` | - |

### 语义槽位

| data-slot | 元素 | 说明 |
| --- | --- | --- |
| `switch` | SwitchPrimitive.Root | 开关根容器 |
| `switch-thumb` | SwitchPrimitive.Thumb | 滑块手柄 |

额外状态属性：
- `data-[state="checked"]` — 选中状态
- `data-[state="unchecked"]` — 未选中状态

<h2 id="faq">常见问题</h2>

### 如何受控使用？

使用 `checked` + `onCheckedChange`：

```tsx
const [checked, setChecked] = useState(false)
&lt;Switch checked={checked} onCheckedChange={setChecked} /&gt;
```

### 如何设置默认值？

使用 `defaultChecked`，组件内部自行管理状态：

```tsx
&lt;Switch defaultChecked /&gt;
```

### 如何在表单中使用？

设置 `name` 属性，选中时提交 `value`（默认 `"on"`）：

```tsx
&lt;form&gt;
  &lt;Switch name="notifications" defaultChecked /&gt;
  &lt;button type="submit"&gt;提交&lt;/button&gt;
&lt;/form&gt;
```

### Switch 和 Checkbox 有何区别？

Switch 用于即时生效的开关操作，强调状态切换的即时性；Checkbox 用于多选场景，通常配合表单提交使用。单个布尔值切换优先使用 Switch。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/switch` 导入 Switch，不要从 `antd` 导入。
2. 只使用本地源码中存在的 props（见上方 API 表），不生成 AntD 专属的 `checkedChildren`、`unCheckedChildren`、`loading`、`size` 等属性。
3. 受控模式下必须同时提供 `checked` 和 `onCheckedChange`。
4. 表单场景需设置 `name` 属性以参与表单提交。
5. 不要引入 Ant Design 相关依赖。
