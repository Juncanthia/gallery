RelativeTimeCard 相对时间卡片。以相对时间（如"5 minutes ago"）展示日期，hover 时显示详细时间与时区信息。基于 HoverCard 和 Intl API 实现。

<h2 id="when-to-use">何时使用</h2>

当需要在紧凑空间内展示时间，同时允许用户查看精确时间和不同时区时使用。

常见场景：
- 列表、表格中的时间列（如"3 分钟前"、"2 天前"）
- 聊天或评论中的消息时间戳
- 活动日志、操作记录的时间展示
- 需要跨时区对比时间（如国际团队协作场景）

不推荐：
- 需要显示完整时间格式且无 hover 交互的场景（直接使用原生 time 标签）
- 无 hover 交互能力的环境（如触摸设备上仅使用悬停可能不可用）
- 需要精确到毫秒级的时间展示

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `date` 指定时间点，`timezones` 设置 hover 卡片中展示的时区列表。组件自动计算相对时间并每秒更新。

<ComponentShowcase id="dice/relative-time-card" path="dice/relative-time-card/basic" />

```tsx
import { RelativeTimeCard } from "@/components/data-display/dice/relative-time-card"

export default function Demo() {
  return (
    &lt;RelativeTimeCard
      date={new Date(Date.now() - 1000 * 60 * 5)}
      timezones={["Asia/Shanghai", "America/New_York", "Europe/London"]}
    /&gt;
  )
}
```

<h2 id="api">API</h2>

### RelativeTimeCard

基于 HoverCard，支持其全部定位属性。以下列出核心 API。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| date | 要展示的日期/时间 | `Date` \| `string` \| `number` | 必填 |
| timezones | hover 卡片中展示的时区列表（不含用户本地时区，本地时区始终显示） | `string[]` | `["UTC"]` |
| variant | 触发文本的视觉变体 | `"default"` \| `"muted"` \| `"ghost"` | `"default"` |
| updateInterval | 相对时间文本的更新间隔（毫秒） | `number` | `1000` |
| open | 受控打开状态 | `boolean` | - |
| defaultOpen | 默认打开状态 | `boolean` | - |
| onOpenChange | 打开状态变化回调 | `(open: boolean) => void` | - |
| openDelay | 打开延迟（毫秒） | `number` | `500` |
| closeDelay | 关闭延迟（毫秒） | `number` | `300` |
| side | HoverCard 弹出方向 | `"top"` \| `"right"` \| `"bottom"` \| `"left"` | - |
| align | HoverCard 对齐方式 | `"start"` \| `"center"` \| `"end"` | - |
| sideOffset | 偏移距离（px） | `number` | - |
| alignOffset | 对齐偏移（px） | `number` | - |
| avoidCollisions | 是否避开边界碰撞 | `boolean` | - |
| collisionBoundary | 碰撞检测边界元素 | `Element` \| `Element[]` | - |
| collisionPadding | 碰撞检测内边距（px） | `number` \| `Partial<Record<"top" \| "right" \| "bottom" \| "left", number>>` | - |
| asChild | 使用 Radix Slot 合并到子元素 | `boolean` | `false` |
| className | 触发元素附加样式类 | `string` | - |

<h2 id="faq">常见问题</h2>

### 如何在不支持 hover 的移动端使用？

本组件基于 HoverCard 实现，在移动端无法使用 hover 交互。移动端场景建议使用 Popover 或直接展示完整时间。

### 如何自定义相对时间的格式？

相对时间格式由内部的 `formatRelativeTime` 函数控制。如需自定义，需直接修改 `src/components/dice/ui/relative-time-card.tsx` 源码。文件体积小，修改成本低。

### 时区列表和用户本地时区有什么区别？

用户本地时区（由系统语言自动检测）始终显示在 hover 卡片中。`timezones` 属性用于额外指定需要展示的目标时区。

### asChild 模式下有什么注意事项？

`asChild` 模式下需自行提供触发元素的内容和语义（如 `time` 标签、`dateTime` 属性）。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/data-display/dice/relative-time-card` 导入 RelativeTimeCard，不要从 antd 导入任何时间组件。
2. `date` 必须是 `Date` 对象 `string` 或 `number` 类型，三者均会被内部转换为 `Date`。
3. `timezones` 使用 IANA 时区标识符（如 `"Asia/Shanghai"`），确保时区名称有效。
4. 不要引入 Ant Design 的 `DatePicker`、`TimePicker`、`Calendar`、`RelativeTime` 等时间相关组件。
5. 移动端场景该组件不适用，应提供替代方案。
