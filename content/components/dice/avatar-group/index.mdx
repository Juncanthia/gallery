AvatarGroup 头像组。将多个头像以重叠方式排列展示，支持数量截断、方向控制、自定义溢出渲染和径向遮罩裁剪效果。

<h2 id="when-to-use">何时使用</h2>

当需要以紧凑重叠方式展示一组用户头像，并在数量过多时自动截断展示 "+N" 溢出提示时使用。

常见场景：
- 协作者列表、项目成员展示。
- 任务卡片中展示参与者头像。
- 评论区展示最近活跃用户。
- 任意需要头像堆叠展示的场景。

核心能力：
- 重叠排列头像，通过径向渐变遮罩实现裁剪效果。
- `max` 属性控制最大展示数量，超出部分自动截断。
- `renderOverflow` 自定义溢出项的渲染方式。
- 支持 `horizontal` / `vertical` 排列方向和 `ltr` / `rtl` 方向。
- `reverse` 属性反转层级顺序。
- 通过 `size` 统一控制头像尺寸。

不推荐：
- 头像数量很少（小于 3 个）——使用普通 flex 布局即可。
- 需要可点击交互的复杂成员列表——AvatarGroup 侧重展示层。
- 在无固定布局上下文中使用 `vertical` 方向——可能产生意外的垂直空间占用。

<h2 id="api">API</h2>

### AvatarGroup

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 头像子元素。通常为 Avatar 组件或其等价渲染元素 | `React.ReactNode` | - |
| orientation | 排列方向 | `"horizontal"` \| `"vertical"` | `"horizontal"` |
| dir | 排列朝向 | `"ltr"` \| `"rtl"` | `"ltr"` |
| size | 统一头像尺寸（像素），用于计算遮罩半径和偏移 | `number` | `40` |
| max | 最大显示数量。超出部分截断并以溢出元素展示 | `number` | - |
| reverse | 是否反转层级顺序。默认后一个头像覆盖在前一个之上，设为 `true` 则相反 | `boolean` | `false` |
| renderOverflow | 自定义溢出渲染，接收溢出数量。不传则使用默认 "+N" 样式 | `(count: number) => React.ReactNode` | - |
| asChild | 是否将根元素作为子元素的 Slot 渲染 | `boolean` | `false` |
| className | 附加 CSS 类名 | `string` | - |

> `max` 不传时展示所有 children；传值后最多展示 `max` 个项，其中前 `max - 1` 个为原始 children，最后 1 个为溢出元素。

<h2 id="semantic-dom">Semantic DOM</h2>

| data-slot | 说明 |
| --- | --- |
| `avatar-group` | AvatarGroup 根元素（`<div>` 或 `asChild` 时的 Slot） |
| `avatar-group-item` | 每个头像包装元素（Radix Slot），承载遮罩样式 |

额外状态属性：
- `data-orientation`：当前排列方向（`horizontal` / `vertical`）。

<h2 id="faq">常见问题</h2>

### AvatarGroup 和 Avatar 组件里的 AvatarGroup 有什么区别？

`dice-avatar` 中的 `AvatarGroup` 是一个简单的 flex 容器组件（仅提供 `-space-x-2` 重叠间距和 `ring-background`），适合头像数量固定且无需截断的场景。`dice-avatar-group` 是独立的增强版组件，额外支持 `max` 截断、`orientation` / `dir` 方向控制、`reverse` 层级反转和径向遮罩裁剪效果。两者命名相同但导入路径不同，根据场景选择。

### 如何让头像组垂直排列？

设置 `orientation="vertical"`：

```tsx
&lt;AvatarGroup orientation="vertical" size={36} max={4}&gt;
  &lt;Avatar size="sm"&gt;&lt;AvatarFallback&gt;A&lt;/AvatarFallback&gt;&lt;/Avatar&gt;
  &lt;Avatar size="sm"&gt;&lt;AvatarFallback&gt;B&lt;/AvatarFallback&gt;&lt;/Avatar&gt;
  &lt;Avatar size="sm"&gt;&lt;AvatarFallback&gt;C&lt;/AvatarFallback&gt;&lt;/Avatar&gt;
  &lt;Avatar size="sm"&gt;&lt;AvatarFallback&gt;D&lt;/AvatarFallback&gt;&lt;/Avatar&gt;
  &lt;Avatar size="sm"&gt;&lt;AvatarFallback&gt;E&lt;/AvatarFallback&gt;&lt;/Avatar&gt;
&lt;/AvatarGroup&gt;
```

### 溢出项的默认样式是什么？

不传 `renderOverflow` 时，溢出项渲染为：
```
inline-flex size-full items-center justify-center rounded-full bg-muted font-medium text-muted-foreground text-xs
```
内容为 `+{overflowCount}`。如需自定义样式或行为（如点击展开），通过 `renderOverflow` 传递渲染函数。

### reverse 属性的作用是什么？

默认行为（`reverse={false}`）下，后一个头像覆盖在前一个之上（适合 LTR 场景）。设置 `reverse={true}` 后，前面的头像覆盖后面的，同时遮罩方向反转。这通常用于需要强调第一个头像的场景。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/dice-avatar-group` 导入 AvatarGroup，不要从 `antd` 或 `@/components/ui/dice-avatar` 导入。
2. 头像 children 建议使用 `@/components/ui/dice-avatar` 的 Avatar 系列组件。
3. 只使用本地源码和本文件 API 表中存在的 props。
4. 不要使用 `flex` + `-space-x-*` 手动实现重叠排列——使用 AvatarGroup 的内置能力。
5. 溢出自定义通过 `renderOverflow` prop，不要额外包裹一层手动计数逻辑。
6. `size` 应与子 Avatar 的 `size` 视觉匹配，否则遮罩裁剪可能出现偏差。
7. 不要引入 Ant Design 相关依赖或 ConfigProvider。
