# AvatarGroup 头像组

将多个头像以重叠方式排列展示，通过径向渐变遮罩实现重叠裁剪，支持数量截断、排列方向控制和自定义溢出渲染。

本组件借鉴 Ant Design Avatar.Group 的使用心智，但不是 AntD 兼容层。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

适合需要以紧凑重叠方式展示一组用户头像，并在数量过多时自动截断展示 "+N" 的场景。优先通过组件自身 props 表达状态和语义，避免在业务侧手写重叠布局和计数逻辑。

常见场景：

- 协作者列表、项目成员展示。
- 任务卡片中展示参与者头像。
- 评论区展示最近活跃用户。
- 任意需要头像堆叠展示的场景。

不推荐：

- 为了照搬 Ant Design API 而新增兼容别名。
- 在调用侧绕过组件 API，直接手写 flex 重叠布局和溢出计数逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface AvatarGroupProps extends Omit<React.ComponentProps<"div">, "dir"> {
  /** 头像子元素 */
  children?: React.ReactNode;
  /** 排列方向。@default "horizontal" */
  orientation?: "horizontal" | "vertical";
  /** 排列朝向。@default "ltr" */
  dir?: "ltr" | "rtl";
  /** 统一头像尺寸（像素），用于遮罩半径计算。@default 40 */
  size?: number;
  /** 最大显示数量，超出截断。不传则展示全部 */
  max?: number;
  /** 反转层级顺序。@default false */
  reverse?: boolean;
  /** 自定义溢出渲染，接收溢出数量 */
  renderOverflow?: (count: number) => React.ReactNode;
  /** 是否作为子元素 Slot 渲染。@default false */
  asChild?: boolean;
  className?: string;
}
```

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `avatar-group`
- `avatar-group-item`

额外状态属性：

- `data-slot`
- `data-orientation`

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 使用 `class-variance-authority` (cva) 管理 `orientation` 和 `dir` 的组合样式变体。
- 每个头像项通过 CSS `mask-image`（径向渐变 `radial-gradient`）实现重叠裁剪效果，遮罩半径和偏移基于 `size` prop 动态计算。
- 组件包装 Radix Slot primitive，支持 `asChild` 模式。
- 默认溢出 Badge 样式：`inline-flex size-full items-center justify-center rounded-full bg-muted font-medium text-muted-foreground text-xs`。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/dice/ui/avatar-group.tsx`

直接依赖摘要：

- React、Tailwind CSS、class-variance-authority、Radix Slot primitive、本地 `cn` 工具函数。

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 样式应优先通过组件 props（`orientation`、`dir`、`reverse`、`renderOverflow`）表达。
3. 内部使用 `React.Children.toArray` 处理 children，过滤非有效元素后进行截断计算。
4. 每个子项通过 `AvatarGroupItem` 包装，应用动态计算的 `maskImage` 实现重叠裁剪。
5. 如果需要新增能力，应先修改该组件源码，再更新本 llm 文档。
6. 不要把第三方文档中的全量 API 视为本地已支持能力。

## 可访问性 {#accessibility}

- 根元素保留 `data-slot="avatar-group"` 和 `data-orientation` 语义属性。
- 子头像项保留 `data-slot="avatar-group-item"` 语义属性。
- 头像内容（图片 alt、fallback 文本）由调用方通过 Avatar 组件提供可访问性。
- 组件本身不引入额外的 ARIA 角色或键盘交互。

## 与 dice-avatar 中 AvatarGroup 的区别 {#avatar-group-comparison}

`dice-avatar` 中的 `AvatarGroup` 是简单 flex 容器（仅 `-space-x-2` 间距 + `ring-background`），适合固定数量头像场景。`dice-avatar-group` 是独立增强版，额外提供 `max` 截断、`orientation`/`dir` 方向控制、`reverse` 层级反转和径向遮罩裁剪。导入时注意区分：

```tsx
// 简单 flex 容器版（固定数量场景）
import { AvatarGroup } from "@/components/ui/avatar"

// 增强版（需要截断、方向控制、遮罩裁剪时使用）
import { AvatarGroup } from "@/components/ui/avatar-group"
```

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 AvatarGroup 代码时，应遵守：

1. 从 `@/components/ui/avatar-group` 导入 AvatarGroup，不要从 `antd` 或 `@/components/ui/avatar` 导入。
2. 头像 children 建议使用 `@/components/ui/avatar` 的 Avatar 系列组件。
3. 只使用本文件 API 摘要和源码中存在的 props。
4. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
5. 需要自定义溢出样式时，使用 `renderOverflow` prop，不要通过 className 覆盖内部默认溢出元素。
6. 不要用 `flex` + `-space-x-*` + 手动截断逻辑替代 AvatarGroup——使用组件的内置能力。
7. `size` 应与子 Avatar 的 `size` 视觉匹配，否则遮罩裁剪可能出现偏差。
8. 如果不确定某个 AntD API 是否存在，先检查 `src/components/dice/ui/avatar-group.tsx`，不要凭 AntD 记忆生成。
