# dice/checkbox - LLM Rules

## Import
```
import { Checkbox } from "@/components/ui/checkbox-dice"
```

## Props
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| checked | `boolean \| "indeterminate"` | - | Controlled checked state |
| defaultChecked | `boolean` | `false` | Uncontrolled default checked state |
| onCheckedChange | `(checked: boolean \| "indeterminate") => void` | - | Called when checked state changes |
| disabled | `boolean` | `false` | Disables the checkbox |
| required | `boolean` | `false` | Makes it required in forms |
| name | `string` | - | Form field name |
| value | `string` | `"on"` | Form submission value |
| className | `string` | - | Additional CSS classes |

## Code Examples

### Basic
```tsx
import { Checkbox } from "@/components/ui/checkbox-dice"

<Checkbox id="agree" defaultChecked />
<label htmlFor="agree">I agree</label>
```

### Controlled
```tsx
const [checked, setChecked] = useState(false)

<Checkbox
  checked={checked}
  onCheckedChange={(v) => setChecked(v === true)}
/>
```

### Disabled
```tsx
<Checkbox disabled />
<Checkbox disabled defaultChecked />
```

### Indeterminate (parent checkbox for select-all)
```tsx
<Checkbox
  checked={someChecked ? "indeterminate" : allChecked}
  onCheckedChange={(v) => {
    if (v === true) selectAll()
    else deselectAll()
  }}
/>
```

## Constraints
- DO NOT import from `antd`. Use `@/components/ui/checkbox-dice`.
- `indeterminate` is a value of the `checked` prop, not a separate boolean prop.
- Always pair `<Checkbox>` with a `<label htmlFor={id}>` for accessibility.
- For checkbox groups, compose multiple `<Checkbox>` instances inside a `<fieldset>`.
- The component accepts all `React.ComponentProps<typeof CheckboxPrimitive.Root>` additional props.
- Based on Radix Checkbox.Root. `data-slot="checkbox"` on root.
