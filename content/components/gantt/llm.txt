# Gantt 甘特图

Gantt 是一套甘特图时间线可视化组件系统。支持日视图（daily）、月视图（monthly）、季度视图（quarterly）三种时间粒度。提供侧边栏（功能列表）、时间线头部、特性条（支持拖拽排期和拖拽调整起止日期）、Today 标记线、自定义 Marker 线、无限滚动加载前后年份数据等功能。基于 `@dnd-kit` 实现拖拽交互，使用 `jotai` 管理拖拽和滚动状态。

## 何时使用 {#when-to-use}

适用于需要时间线可视化的项目管理场景：

- 项目路线图规划（Roadmap）。
- 迭代 / Sprint 排期。
- 功能发布计划。
- 多团队资源时间线。

常见场景：

- 按日/月/季度查看项目功能的时间安排。
- 拖拽调整功能起止日期。
- 查看 Today 标记了解当前进度位置。
- 自定义里程碑标记。

不推荐：

- 简单的日期范围展示——使用时间轴或日历组件更轻量。
- 需要关键路径、依赖箭头等专业甘特图功能——建议使用专门的 Gantt 库。
- 数据量极大（数千条）且无分组/分页的场景——虚拟滚动未实现。

## 代码演示 {#examples}

### 基础甘特图

```tsx
import {
  GanttProvider,
  GanttSidebar,
  GanttSidebarGroup,
  GanttSidebarItem,
  GanttTimeline,
  GanttHeader,
  GanttFeatureList,
  GanttFeatureListGroup,
  GanttFeatureRow,
  GanttToday,
} from "@/components/blocks/gantt"

const features = [
  {
    id: "1",
    name: "用户认证",
    startAt: new Date(2025, 0, 1),
    endAt: new Date(2025, 2, 15),
    status: { id: "1", name: "进行中", color: "#3b82f6" },
  },
  {
    id: "2",
    name: "数据看板",
    startAt: new Date(2025, 2, 1),
    endAt: new Date(2025, 4, 30),
    status: { id: "2", name: "计划中", color: "#f59e0b" },
  },
]

export default function Demo() {
  return (
    <GanttProvider range="monthly">
      <GanttSidebar>
        <GanttSidebarGroup name="Q1 2025">
          {features.map((f) => (
            <GanttSidebarItem key={f.id} feature={f} />
          ))}
        </GanttSidebarGroup>
      </GanttSidebar>
      <GanttTimeline>
        <GanttHeader />
        <GanttFeatureList>
          <GanttFeatureListGroup>
            <GanttFeatureRow features={features} />
          </GanttFeatureListGroup>
        </GanttFeatureList>
        <GanttToday />
      </GanttTimeline>
    </GanttProvider>
  )
}
```

### 带拖拽排期

```tsx
import { useState } from "react"
import {
  GanttProvider,
  GanttSidebar,
  GanttSidebarItem,
  GanttTimeline,
  GanttHeader,
  GanttFeatureList,
  GanttFeatureRow,
} from "@/components/blocks/gantt"

export default function Demo() {
  const [features, setFeatures] = useState(initialFeatures)

  const handleMove = (id: string, startAt: Date, endAt: Date | null) => {
    setFeatures((prev) =>
      prev.map((f) => (f.id === id ? { ...f, startAt, endAt } : f))
    )
  }

  return (
    <GanttProvider range="monthly">
      <GanttSidebar>
        {features.map((f) => (
          <GanttSidebarItem key={f.id} feature={f} />
        ))}
      </GanttSidebar>
      <GanttTimeline>
        <GanttHeader />
        <GanttFeatureList>
          <GanttFeatureRow features={features} onMove={handleMove} />
        </GanttFeatureList>
      </GanttTimeline>
    </GanttProvider>
  )
}
```

## API {#api}

### GanttProvider

甘特图顶层容器，提供 Context、CSS 变量、无限滚动逻辑。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| range | 时间粒度 | `"daily" \| "monthly" \| "quarterly"` | `"monthly"` |
| zoom | 缩放百分比 | `number` | `100` |
| onAddItem | 点击时间轴空白区域的新增回调 | `(date: Date) => void` | - |
| className | 附加 className | `string` | - |
| children | 子组件 | `ReactNode` | - |

Provider 通过 CSS 变量控制以下尺寸：
- `--gantt-column-width`：列宽 = `(zoom / 100) * columnWidth`（daily=50px, monthly=150px, quarterly=100px）。
- `--gantt-header-height`：60px。
- `--gantt-row-height`：36px。
- `--gantt-sidebar-width`：300px（动态检测侧边栏实际宽度）。

### GanttSidebar

左侧功能列表面板（sticky 定位）。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |
| children | 侧边栏内容 | `ReactNode` | - |

### GanttSidebarHeader

侧边栏表头（sticky），显示列标签 "Issues" 和 "Duration"。

无需 props 的 FC。

### GanttSidebarGroup

侧边栏分组。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| name | 分组名称 | `string` | - |
| className | 附加 className | `string` | - |
| children | 分组内条目 | `ReactNode` | - |

### GanttSidebarItem

侧边栏单个功能条目。点击或按 Enter 可滚动时间线到对应功能条位置。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| feature | 功能数据 | `GanttFeature` | 必填 |
| onSelectItem | 选中回调 | `(id: string) => void` | - |
| className | 附加 className | `string` | - |

### GanttTimeline

右侧时间线容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |
| children | 时间线内容 | `ReactNode` | - |

### GanttHeader

时间线头部，根据 `range` 自动渲染日/月/季度视图头部。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

### GanttFeatureList / GanttFeatureListGroup

功能条列表容器和分组容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |
| children | 子内容 | `ReactNode` | - |

### GanttFeatureRow

单行功能条，自动处理同时间段内多个功能的重叠展示（子行分配）。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| features | 该行包含的功能列表 | `GanttFeature[]` | 必填 |
| onMove | 拖拽移动/调整日期后的回调 | `(id: string, startAt: Date, endAt: Date \| null) => void` | - |
| children | 自定义功能条渲染函数 | `(feature: GanttFeature) => ReactNode` | - |
| className | 附加 className | `string` | - |

### GanttFeatureItem

单个可拖拽功能条（实现拖拽移动 + 边缘拖拽调整起止日期）。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 功能唯一标识 | `string` | 必填 |
| name | 功能名称 | `string` | 必填 |
| startAt | 开始日期 | `Date` | 必填 |
| endAt | 结束日期 | `Date` | 必填 |
| status | 状态对象 | `GanttStatus` | 必填 |
| lane | 泳道标识，同 lane 功能共享行 | `string` | - |
| onMove | 拖拽结束回调 | `(id: string, startDate: Date, endDate: Date \| null) => void` | - |
| children | 自定义功能条内容 | `ReactNode` | - |
| className | 附加 className | `string` | - |

### GanttToday

Today 标记线，自动定位到当前日期。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

### GanttMarker

自定义标记线。通过 `React.memo` 包裹优化渲染。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 标记唯一标识 | `string` | 必填 |
| date | 标记日期 | `Date` | 必填 |
| label | 标记标签 | `string` | 必填 |
| onRemove | 右键删除回调 | `(id: string) => void` | - |
| className | 附加 className | `string` | - |

### GanttCreateMarkerTrigger

鼠标悬停时间轴时显示"在此创建标记"按钮。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| onCreateMarker | 点击创建回调 | `(date: Date) => void` | 必填 |
| className | 附加 className | `string` | - |

### Hooks

| Hook | 说明 | 返回值 |
| --- | --- | --- |
| `useGanttDragging()` | 读写全局拖拽中状态（Jotai atom） | `[boolean, (v: boolean) => void]` |
| `useGanttScrollX()` | 读写全局横向滚动位置（Jotai atom） | `[number, (v: number) => void]` |

### 核心类型

```ts
type GanttStatus = { id: string; name: string; color: string }

type GanttFeature = {
  id: string
  name: string
  startAt: Date
  endAt: Date
  status: GanttStatus
  lane?: string
}

type GanttMarkerProps = { id: string; date: Date; label: string }

type Range = "daily" | "monthly" | "quarterly"
```

## 样式模型 {#styling-model}

全部使用 Tailwind CSS + CSS 变量（通过 inline style 注入）。

- Provider 容器：`relative isolate grid h-full w-full select-none overflow-auto rounded-sm bg-secondary`，grid 两列（sidebar + timeline）。
- Header：`sticky top-0 z-20 bg-backdrop/90 backdrop-blur-sm`。
- Sidebar：`sticky left-0 z-30 bg-background/90 backdrop-blur-md border-r`。
- 功能条：`absolute` 定位，宽高由 `getWidth`/`getOffset` 计算。
- 标记线：`absolute top-0 left-0 z-20`，通过 `translateX` 定位。
- 周末列：`bg-secondary` 背景。
- 拖拽手柄：`!cursor-col-resize absolute top-1/2 z-[3]`。
- 缩放通过 CSS 变量 `--gantt-zoom` 和 `--gantt-column-width` 控制。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/gantt/index.tsx`：全部实现。

关键依赖：

- `@dnd-kit/core`：功能条拖拽排期。
- `@dnd-kit/modifiers`：限制拖拽为水平方向。
- `date-fns`：日期计算和格式化。
- `jotai`：拖拽状态和滚动位置跨组件共享。
- `lodash.throttle`：滚动事件节流。
- `@uidotdev/usehooks`：`useMouse`、`useThrottle`、`useWindowScroll`。
- `lucide-react`：图标。
- `@/components/ui/card`、`@/components/ui/context-menu`：UI 组件。

关键行为：

1. 初始生成 3 年时间线数据（去年、今年、明年）。
2. 滚动到边界时自动扩展时间线（向前/后各追加一年）。
3. 功能条定位通过 `getOffset` 和 `getWidth` 计算像素位置。
4. 拖拽采用三个独立 `DndContext`：左侧边缘拖拽、任务条整体拖拽、右侧边缘拖拽，方向限制为水平轴。
5. `GanttFeatureRow` 通过子行算法自动检测同时间段重叠功能并分配到子行。
6. `GanttFeatureItem` 内部维护本地 `startAt`/`endAt` 状态用于拖拽实时预览，拖拽结束才通过 `onMove` 通知外部。
7. 日期计算适配三种 range 模式，通过策略函数（`getDifferenceIn`、`getStartOf`、`getAddRange` 等）统一处理。
8. 点击侧边栏条目可调用 `scrollToFeature` 滚动时间线到对应位置。

## 可访问性 {#accessibility}

- 侧边栏条目使用 `role="button"` + `tabIndex={0}` + 键盘 Enter 支持。
- 拖拽通过 `@dnd-kit` 的 `useDraggable` 提供 `role` 和 `aria-pressed` 属性。
- 右键菜单使用 `ContextMenu` 组件提供标准的右键交互支持。
- 颜色状态圆点仅作为视觉辅助，名称文本和状态对象提供等效语义信息。
- 无 ARIA 标注时间线结构，适合内部项目管理工具，公共产品需补充。

## 与 Ant Design 的取舍 {#antd-differences}

Ant Design 无甘特图组件，此为全新实现。

| AntD 能力 | 本地决策 | 原因 |
| --- | --- | --- |
| 无对应组件 | 全新实现 | 甘特图为项目特有需求 |

## LLM 使用规则 {#llm-rules}

1. 从 `@/components/blocks/gantt` 按需导入所有子组件。
2. 组装顺序：`GanttProvider` > `GanttSidebar` (含 Group/Item) + `GanttTimeline` (含 Header/FeatureList/FeatureRow)。
3. `GanttProvider` 必须设置 `range`（"daily" | "monthly" | "quarterly"）。
4. `GanttFeature` 数据必须包含完整的 `status` 对象（`id`、`name`、`color`），`color` 为 CSS 颜色字符串。
5. `GanttFeatureRow` 支持 `onMove` 回调实现拖拽排期，回调中需更新功能数据的 `startAt`/`endAt`。
6. Sidebar 和 Timeline 在 Grid 布局中左右排列，不能省略任一侧。
7. `GanttFeatureRow` 的 features 按 `startAt` 排序并自动分配子行，无需手动管理重叠。
8. 自定义标记通过 `GanttMarker` 放在 `GanttTimeline` 内、`GanttFeatureList` 之后。
9. 只使用本文件 API 表格和源码中存在的 props，不要凭 AntD 或其他甘特图库的记忆生成 API。
10. 时间轴无限滚动由 `GanttProvider` 内部自动处理，无需额外配置。
