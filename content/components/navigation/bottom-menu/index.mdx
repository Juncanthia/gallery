BottomMenu 底部菜单，一个带动画的 iOS 风格底部导航栏，点击图标展开对应的子菜单面板。

<h2 id="when-to-use">何时使用</h2>

当需要底部固定导航栏，且每个导航项可展开子菜单面板时使用。适用于移动端或桌面端底部工具栏场景。

常见场景：

- 移动端底部主导航，如首页、搜索、通知、个人中心。
- 快捷操作入口，如新建笔记、语音输入、截图。
- 主题切换面板，如亮色/暗色/跟随系统。
- 个人中心快捷菜单，如账号、设置、订阅、退出登录。

BottomMenu 通过底部图标栏 + 弹出面板的方式组织交互：

- `view` 状态控制当前展开的面板，再次点击同一图标可收起。
- 点击组件外部区域自动收起面板。
- 面板切换带有 motion 动画过渡，支持高度、宽度、透明度渐变。

不推荐：

- 将 BottomMenu 用于全局导航，应使用 Navigation Menu 或 Sidebar。
- 导航项过多（超过 6 个），应使用 TabBar 或其他布局。
- 在已存在底部安全区的页面重复叠加底部栏。

<h2 id="api">API</h2>

### BottomMenu

BottomMenu 是一个零 props 的独立组件，所有导航项和子面板内容在组件内部定义。如需定制导航项或面板内容，请直接修改 `src/components/navigation/bottom-menu.tsx` 中的 `MAIN_NAV`、`HOME_ITEMS`、`SEARCH_OPTIONS`、`NOTIFICATION_TYPES`、`PROFILE_LINKS`、`THEME_OPTIONS` 常量。

| 内部状态 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| view | 当前展开的面板 | `"default"` \| `"home"` \| `"search"` \| `"references/notifications"` \| `"profile"` \| `"theme"` | `"default"` |
| theme | 当前主题选择 | `"light"` \| `"dark"` \| `"system"` | `"light"` |

### 导航项配置

| 常量 | 说明 | 类型 |
| --- | --- | --- |
| MAIN_NAV | 底部图标栏导航项 | `{ icon: HugeiconsIcon, name: string }[]` |
| HOME_ITEMS | 首页面板快捷操作 | `{ icon: HugeiconsIcon, text: string }[]` |
| SEARCH_OPTIONS | 搜索面板筛选入口 | `{ icon: HugeiconsIcon, text: string }[]` |
| NOTIFICATION_TYPES | 通知面板分类 | `string[]` |
| PROFILE_LINKS | 个人中心链接 | `string[]` |
| THEME_OPTIONS | 主题切换选项 | `{ key: string, icon: HugeiconsIcon, text: string }[]` |

<h2 id="faq">常见问题</h2>

### 如何自定义导航项？

直接修改 `src/components/navigation/bottom-menu.tsx` 中的 `MAIN_NAV` 常量和对应面板的配置常量。

### 如何修改面板内容？

在 `src/components/navigation/bottom-menu.tsx` 中的 `content` useMemo 内，修改各 `case` 分支的 JSX 内容。

### 点击外部区域面板没有收起？

组件通过 `mousedown` 事件监听外部点击。如果外层容器阻止了事件冒泡，可能需要调整事件监听逻辑。

### 如何添加新的面板类型？

1. 在 `view` 状态类型中添加新的视图名称。
2. 在 `MAIN_NAV` 中添加对应的导航项。
3. 在 `content` useMemo 的 `switch` 中添加对应的 JSX 分支。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 BottomMenu 代码时：

1. 从 `@/components/navigation/bottom-menu` 导入 BottomMenu。
2. BottomMenu 为零 props 组件，直接渲染即可。
3. 如需要定制导航项，修改源码中的常量定义，不要通过 props 传递。
4. 图标使用 `@hugeicons/core-free-icons` 和 `@hugeicons/react` 的 `HugeiconsIcon`。
5. 不要引入 Ant Design 或 rc-component 相关依赖。
