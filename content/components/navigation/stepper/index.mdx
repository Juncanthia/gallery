Stepper 引导用户按步骤完成任务的导航组件，支持步骤指示器、前后导航、自定义渲染和完成回调。

<h2 id="when-to-use">何时使用</h2>
当需要...时使用。
常见场景：
- 多步表单（注册、配置、确认）
- 向导式流程（安装引导、新手指引）
- 需要展示进度和前后导航的线性任务
不推荐：
- 非线性流程（步骤间无顺序关系）
- 仅 1-2 步的简单场景（过度设计）

### 基础用法
<ComponentShowcase id="rb/stepper" path="rb/stepper/basic" />
```tsx
import { Stepper, Step } from "@/components/core/stepper"
export default function Demo() {
  return (
    <Stepper>
      <Step><div>Step 1 content</div></Step>
      <Step><div>Step 2 content</div></Step>
      <Step><div>Step 3 content</div></Step>
    </Stepper>
  )
}
```

### 自定义配置
```tsx
import { Stepper, Step } from "@/components/core/stepper"

export default function Demo() {
  return (
    <Stepper
      initialStep={1}
      backButtonText="上一步"
      nextButtonText="下一步"
      onStepChange={(step) => console.log("当前步骤:", step)}
      onFinalStepCompleted={() => alert("全部完成！")}
      renderStepIndicator={({ step, currentStep, onStepClick }) => (
        <button
          onClick={() => onStepClick(step)}
          className={`w-8 h-8 rounded-full text-sm ${
            step <= currentStep ? "bg-violet-600 text-white" : "bg-neutral-800 text-neutral-400"
          }`}
        >
          {step}
        </button>
      )}
    >
      <Step><div>基本信息</div></Step>
      <Step><div>详细配置</div></Step>
      <Step><div>确认提交</div></Step>
    </Stepper>
  )
}
```

<h2 id="api">API</h2>
### Stepper
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 步骤内容，每个 `<Step>` 为一个步骤 | `ReactNode` | - |
| initialStep | 初始步骤编号（从 1 开始） | `number` | `1` |
| onStepChange | 步骤变化回调 | `(step: number) => void` | - |
| onFinalStepCompleted | 最后一步完成后回调 | `() => void` | - |
| stepCircleContainerClassName | 步骤指示器容器类名 | `string` | - |
| stepContainerClassName | 步骤指示器+连线容器类名 | `string` | - |
| contentClassName | 内容区域的类名 | `string` | - |
| footerClassName | 底部按钮区域的类名 | `string` | - |
| backButtonProps | 传递给返回按钮的额外属性 | `ButtonHTMLAttributes` | - |
| nextButtonProps | 传递给下一步按钮的额外属性 | `ButtonHTMLAttributes` | - |
| backButtonText | 返回按钮文字 | `string` | `"Back"` |
| nextButtonText | 下一步按钮文字 | `string` | `"Continue"` |
| disableStepIndicators | 禁用步骤指示器点击跳转 | `boolean` | `false` |
| renderStepIndicator | 自定义步骤指示器渲染函数 | `(params: { step, currentStep, onStepClick }) => ReactNode` | - |

### Step
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 步骤内容 | `ReactNode` | - |

<h2 id="faq">常见问题</h2>
### 如何自定义步骤按钮文字？
使用 `backButtonText` 和 `nextButtonText` 属性。最后一步的按钮始终显示 "Complete"。

### 如何监听步骤变化？
使用 `onStepChange` 回调，参数为当前步骤编号。最终完成时使用 `onFinalStepCompleted`。

### 如何自定义步骤指示器外观？
使用 `renderStepIndicator` 属性，接收 `{ step, currentStep, onStepClick }` 参数，返回自定义 ReactNode。配合 `disableStepIndicators` 可完全禁用默认指示器。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/core/stepper` 导入 `Stepper` 和 `Step`。
2. 每个直接子元素需用 `<Step>` 包裹作为一个步骤。
3. 依赖 `motion/react`（`motion`、`AnimatePresence`）。
4. 步骤编号从 1 开始，内部通过 `Children.toArray` 处理子元素。
