双因素认证组件，提供验证码输入、重新发送与倒计时功能，用于登录流程中的二次身份验证。

<h2 id="when-to-use">何时使用</h2>

在登录流程中，当用户通过用户名和密码完成首次认证后，需要进行第二步身份验证时使用。

组件提供以下功能：

- 验证码输入：支持 4-8 位数字验证码，每个字符独立输入框，自动聚焦跳转。
- 重新发送：验证码过期或未收到时，可通过"重新发送"按钮获取新码，内建倒计时控制发送频率。
- 返回上一步：提供返回登录页或切换验证方式的入口。
- 验证回调：通过 `actions.onVerify` 获取用户输入的完整验证码。
- 加载与错误状态：`control.isLoading` 控制验证按钮加载态，`control.error` 展示验证失败提示。

### 基础用法

默认展示标题、描述和 6 位验证码输入框。输入完整验证码后点击验证按钮，通过 `actions.onVerify` 回调获取验证码。

<h2 id="faq">常见问题</h2>

### 如何获取用户输入的验证码？

通过 `actions.onVerify` 回调接收完整的验证码字符串：

```tsx
&lt;TwoFactor
  actions={{
    onVerify: (code) => {
      console.log("Verification code:", code)
      // 发送 API 请求验证...
    }
  }}
/>
```

### 如何控制重新发送的冷却时间？

通过 `data.resendCooldown` 设置（单位秒），默认为 60 秒：

```tsx
&lt;TwoFactor
  data={{ resendCooldown: 30 }}
  actions={{
    onResend: () => sendNewCode()
  }}
/>
```

### 如何处理验证失败？

通过 `control.error` 显示错误提示：

```tsx
const [error, setError] = useState&lt;string&gt;()

&lt;TwoFactor
  actions={{
    onVerify: async (code) => {
      const result = await verifyCode(code)
      if (!result.ok) setError("验证码错误，请重试")
    }
  }}
  control={{ error }}
/>
```

### 如何自定义文案？

通过 `data.title`、`data.subtitle`、`data.verifyLabel`、`data.resendLabel`、`data.backLabel` 分别自定义：

```tsx
&lt;TwoFactor
  data={{
    title: "双重验证",
    subtitle: "请输入发送至您设备的 6 位验证码",
    verifyLabel: "确认",
    resendLabel: "重新获取",
    backLabel: "返回",
  }}
/>
```

### 如何隐藏返回按钮？

设置 `appearance.showBack` 为 `false`：

```tsx
&lt;TwoFactor appearance={{ showBack: false }} />
```

### 如何调整验证码位数？

通过 `data.codeLength` 设置，支持 4-8 位：

```tsx
&lt;TwoFactor data={{ codeLength: 4 }} />
```

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 TwoFactor 代码时：

1. 从 `@/components/templates/form/two-factor` 导入 `TwoFactor`。
2. 使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop 配置组件。
3. 验证码通过 `actions.onVerify` 回调获取，不要手动管理组件内部的输入状态。
4. 加载状态通过 `control.isLoading` 控制，组件内部自动禁用验证按钮并显示加载动画。
5. 错误提示通过 `control.error` 传入字符串，组件内部自动展示和清除。
6. 重新发送逻辑通过 `actions.onResend` 触发，组件内部管理倒计时减少 UI 复杂度。
7. 不要引入 Ant Design 相关依赖。
