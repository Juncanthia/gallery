SignUp 注册表单。完整的用户注册组件，支持姓名、邮箱、密码、确认密码、服务条款勾选等功能，提供开箱即用的注册页面体验。

<h2 id="when-to-use">何时使用</h2>

用于新用户创建账户的场景。适用于：

- 网站或应用的注册页面，收集用户基本信息完成账户创建。
- SaaS 产品的注册入口，支持邮箱密码注册。
- 需要服务条款确认的注册流程。
- 作为认证流程（sign-in / sign-up / forgot-password）的一部分。

SignUp 组件提供结构化的注册表单：

- 姓名字段，用于标识用户身份。
- 邮箱字段，作为登录凭证和联系方式。
- 密码与确认密码字段，确保两次输入一致。
- 服务条款复选框，用户需勾选同意后方可提交。
- 提交按钮，点击后通过 `actions.onSubmit` 回调获取注册数据。

### 异步提交

处理注册提交时的异步加载状态。

```tsx
import { useState } from "react"
import { SignUp } from "@/components/templates/form/sign-up"

export default function Demo() {
  const [loading, setLoading] = useState(false)

  async function handleSubmit(data: SignUpData) {
    setLoading(true)
    await fetch("/api/auth/sign-up", {
      method: "POST",
      body: JSON.stringify(data),
    })
    setLoading(false)
  }

  return (
    <SignUp
      actions={{ onSubmit: handleSubmit }}
      control={{ isLoading: loading }}
    />
  )
}
```

<h2 id="api">API</h2>

### SignUp

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 表单数据配置 | `object` | — |
| data.title | 表单标题 | `string` | `"Create an account"` |
| data.subtitle | 标题下方描述文字 | `string` | `"Enter your details to get started."` |
| data.submitLabel | 提交按钮文字 | `string` | `"Sign Up"` |
| data.initialValues | 预填充的表单初始值 | `Partial&lt;SignUpData&gt;` | — |
| actions | 回调配置 | `object` | — |
| actions.onSubmit | 表单提交回调，接收完整注册数据 | `(data: SignUpData) => void` | — |
| appearance | 外观配置 | `object` | — |
| appearance.showTitle | 是否显示标题区域 | `boolean` | `true` |
| appearance.showSignInLink | 是否显示"已有账户？登录"链接 | `boolean` | `true` |
| appearance.signInHref | 登录页链接地址 | `string` | `"/sign-in"` |
| control | 控制配置 | `object` | — |
| control.isLoading | 提交按钮加载状态 | `boolean` | `false` |

### SignUpData

| 字段 | 说明 | 类型 | 必填 |
| --- | --- | --- | --- |
| name | 用户姓名 | `string` | 是 |
| email | 电子邮箱地址 | `string` | 是 |
| password | 登录密码 | `string` | 是 |
| confirmPassword | 确认密码，须与 password 一致 | `string` | 是 |
| agreeToTerms | 是否同意服务条款 | `boolean` | 是 |

<h2 id="faq">常见问题</h2>

### 如何获取注册数据？

通过 `actions.onSubmit` 回调接收 `SignUpData`：

```tsx
<SignUp
  actions={{
    onSubmit: (data) => {
      console.log("Name:", data.name)
      console.log("Email:", data.email)
      // 发送 API 请求...
    }
  }}
/>
```

### 如何设置提交加载状态？

使用 `control.isLoading`：

```tsx
const [loading, setLoading] = useState(false)

<SignUp
  actions={{
    onSubmit: async (data) => {
      setLoading(true)
      await signUpAPI(data)
      setLoading(false)
    }
  }}
  control={{ isLoading: loading }}
/>
```

### 如何预填充表单？

通过 `data.initialValues` 传入初始值：

```tsx
<SignUp
  data={{
    initialValues: {
      name: "张 三",
      email: "zhangsan@example.com",
    }
  }}
/>
```

### 如何自定义链接地址？

通过 `appearance.signInHref` 设置登录页链接：

```tsx
<SignUp
  appearance={{
    signInHref: "/auth/login",
  }}
/>
```

### 如何隐藏标题或登录链接？

通过 `appearance` 控制：

```tsx
<SignUp
  appearance={{
    showTitle: false,
    showSignInLink: false,
  }}
/>
```

### 密码有哪些校验规则？

SignUp 组件内置密码强度校验规则：密码长度至少 8 位，须包含至少一个大写字母、一个小写字母和一个数字。确认密码须与密码一致。校验不通过时提交按钮保持禁用状态。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 SignUp 代码时：

1. 从 `@/components/templates/form/sign-up` 导入 `SignUp`。
2. 使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop 配置组件。
3. 表单提交通过 `actions.onSubmit` 处理，不要手动管理组件内部的表单状态。
4. 加载状态通过 `control.isLoading` 控制，组件内部自动禁用按钮并显示"Creating account..."文案。
5. 组件内置密码校验和确认密码一致性检查，无需在调用侧重复实现。
6. 不要引入 Ant Design 相关依赖。
7. 不要复刻或重新实现 SignUp 的内部逻辑；它是一个页面模板组件，应直接使用。
8. 服务条款链接默认指向 `/terms`，隐私政策链接默认指向 `/privacy`。
