用户资料组件 UserProfile。展示和编辑用户个人信息的模板组件，支持头像、基本信息、联系方式等字段的查看与编辑，提供开箱即用的个人资料页面体验。

<h2 id="when-to-use">何时使用</h2>

当需要在应用中展示和编辑用户个人资料时使用。UserProfile 将头像展示、信息字段渲染、编辑/保存/取消操作组合为一个完整的个人资料页面。

常见场景：

- 用户中心的个人资料页面。
- 账号设置中的基本信息编辑。
- 社交应用的用户主页展示。
- SaaS 产品的个人资料管理。

不推荐：

- 仅需展示单个信息字段（如仅显示用户名）—— 使用自定义布局。
- 需要复杂多步骤表单流程 —— UserProfile 为单页面资料编辑。
- 管理员查看他人资料（只读视图）—— 可通过 `appearance.editable` 控制编辑权限。

<h2 id="examples">代码演示</h2>

### 基础用法

默认展示用户头像、姓名、邮箱、简介等信息，支持点击编辑按钮进入编辑模式。

<ComponentShowcase id="templates/form/user-profile" path="templates/form/user-profile/basic" />

```tsx
import { UserProfile } from "@/components/templates/form/user-profile"

export default function Demo() {
  return &lt;UserProfile /&gt;
}
```

### 处理保存操作

通过 `actions.onSave` 回调接收更新后的用户资料数据。

```tsx
import { UserProfile } from "@/components/templates/form/user-profile"

export default function Demo() {
  return (
    &lt;UserProfile
      actions={{
        onSave: (data) => {
          console.log("Profile saved:", data)
        },
      }}
    /&gt;
  )
}
```

### 异步提交与加载状态

通过 `control.isLoading` 控制保存按钮的加载状态，适用于异步 API 调用。

```tsx
import { useState } from "react"
import { UserProfile } from "@/components/templates/form/user-profile"

export default function Demo() {
  const [loading, setLoading] = useState(false)

  return (
    &lt;UserProfile
      control={{ isLoading: loading }}
      actions={{
        onSave: async (data) => {
          setLoading(true)
          try {
            await saveProfileAPI(data)
          } finally {
            setLoading(false)
          }
        },
      }}
    /&gt;
  )
}
```

### 自定义标题与文案

通过 `data` 自定义标题、编辑按钮、保存按钮和取消按钮文字。

```tsx
import { UserProfile } from "@/components/templates/form/user-profile"

export default function Demo() {
  return (
    &lt;UserProfile
      data={{
        title: "个人资料",
        editLabel: "编辑资料",
        saveLabel: "保存",
        cancelLabel: "取消",
      }}
    /&gt;
  )
}
```

### 隐藏头像

通过 `appearance.showAvatar` 控制头像区域的显示。

```tsx
import { UserProfile } from "@/components/templates/form/user-profile"

export default function Demo() {
  return (
    &lt;UserProfile
      appearance={{ showAvatar: false }}
    /&gt;
  )
}
```

### 只读模式

通过 `appearance.editable` 设置为 `false` 禁用编辑功能，仅展示用户信息。

```tsx
import { UserProfile } from "@/components/templates/form/user-profile"

export default function Demo() {
  return (
    &lt;UserProfile
      appearance={{ editable: false }}
    /&gt;
  )
}
```

<h2 id="api">API</h2>

### UserProfile

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 资料数据配置 | `UserProfileData` | — |
| data.title | 页面标题 | `string` | `"Profile"` |
| data.user | 用户资料数据 | `UserData` | — |
| data.editLabel | 编辑按钮文字 | `string` | `"Edit"` |
| data.saveLabel | 保存按钮文字 | `string` | `"Save Changes"` |
| data.cancelLabel | 取消按钮文字 | `string` | `"Cancel"` |
| actions | 回调配置 | `UserProfileActions` | — |
| actions.onSave | 保存回调，接收更新后的用户数据 | `(data: UserData) => void` | — |
| actions.onCancel | 取消编辑回调 | `() => void` | — |
| actions.onAvatarChange | 头像变更回调，接收文件对象 | `(file: File) => void` | — |
| appearance | 外观配置 | `UserProfileAppearance` | — |
| appearance.showTitle | 是否显示标题区域 | `boolean` | `true` |
| appearance.showAvatar | 是否显示头像区域 | `boolean` | `true` |
| appearance.editable | 是否允许编辑 | `boolean` | `true` |
| control | 控制配置 | `UserProfileControl` | — |
| control.isLoading | 保存按钮加载状态 | `boolean` | `false` |

<h2 id="faq">常见问题</h2>

### 如何获取编辑后的用户数据？

通过 `actions.onSave` 回调接收完整的 `UserData` 对象：

```tsx
&lt;UserProfile
  actions={{
    onSave: (data) => {
      console.log("Name:", data.name)
      console.log("Email:", data.email)
      // 发送 API 请求保存数据...
    },
  }}
/&gt;
```

### 如何设置保存加载状态？

使用 `control.isLoading` 控制保存按钮的加载状态：

```tsx
const [loading, setLoading] = useState(false)

&lt;UserProfile
  control={{ isLoading: loading }}
  actions={{
    onSave: async (data) => {
      setLoading(true)
      try {
        await saveProfileAPI(data)
      } finally {
        setLoading(false)
      }
    },
  }}
/&gt;
```

### 如何自定义标题和按钮文案？

通过 `data.title`、`data.editLabel`、`data.saveLabel`、`data.cancelLabel` 自定义：

```tsx
&lt;UserProfile
  data={{
    title: "个人资料",
    editLabel: "编辑资料",
    saveLabel: "保存修改",
    cancelLabel: "放弃修改",
  }}
/&gt;
```

### 如何隐藏头像区域？

设置 `appearance.showAvatar` 为 `false`：

```tsx
&lt;UserProfile appearance={{ showAvatar: false }} /&gt;
```

### 如何禁用编辑功能？

设置 `appearance.editable` 为 `false`，组件将仅展示用户信息，不显示编辑按钮：

```tsx
&lt;UserProfile appearance={{ editable: false }} /&gt;
```

### 如何处理头像上传？

通过 `actions.onAvatarChange` 回调获取用户选择的头像文件：

```tsx
&lt;UserProfile
  actions={{
    onAvatarChange: async (file) => {
      const formData = new FormData()
      formData.append("avatar", file)
      await uploadAvatarAPI(formData)
    },
  }}
/&gt;
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/templates/form/user-profile` 导入 `UserProfile`，不要从 Ant Design 导入 Descriptions 或 Form 组件。
2. 使用 `data`、`actions`、`appearance`、`control` 四个顶层 prop 配置组件。
3. 资料保存通过 `actions.onSave` 处理，回调接收 `UserData` 类型的完整用户数据。
4. 加载状态通过 `control.isLoading` 控制，组件内部自动禁用按钮并显示加载状态。
5. 不要手动管理组件内部的编辑状态，组件自行处理查看/编辑模式的切换。
6. 头像变更通过 `actions.onAvatarChange` 处理，不要自行实现头像上传逻辑。
7. 不要引入 Ant Design 相关依赖（Form、Input、Button、Upload 等）。
