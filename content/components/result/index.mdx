Result 结果。用于反馈一系列操作任务的处理结果，支持成功、失败、信息、警告以及 HTTP 状态码（404、403、500）对应的视觉状态。

<h2 id="when-to-use">何时使用</h2>

当操作完成需要向用户反馈处理结果时使用。

常见场景：
- 表单提交后的成功/失败反馈页面。
- 支付完成后的结果通知。
- 404 页面未找到的友好提示。
- 403 无权限访问的提示。
- 500 服务器错误的兜底展示。
- 任何需要统一结果反馈 UI 的业务流程。

不推荐：
- 简单的 toast 通知请使用 Sonner 或其他通知组件。
- 页面加载中使用，Result 是终态反馈，非加载态。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

<h2 id="api">API</h2>

### Result

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| status | 结果状态，控制默认图标和语义 | `"success"` \| `"error"` \| `"info"` \| `"warning"` \| `404` \| `"404"` \| `403` \| `"403"` \| `500` \| `"500"` | `"info"` |
| title | 结果标题 | `React.ReactNode` | - |
| subTitle | 结果副标题/描述文本 | `React.ReactNode` | - |
| icon | 自定义图标，传入 `null` 或 `false` 隐藏图标 | `React.ReactNode` | 按 status 自动 |
| extra | 操作区，通常放置 Button 等操作元素 | `React.ReactNode \| React.ReactNode[]` | - |
| children | 结果内容区（title/subTitle/extra 之外的额外内容） | `React.ReactNode` | - |
| className | 附加样式类名 | `string` | - |

`Result` 同时接受 `React.ComponentProps<"div">` 的所有属性（除 `title` 外）。

### ResultStatus 类型

```ts
type ResultStatus =
  | "success"
  | "error"
  | "info"
  | "warning"
  | 404 | "404"
  | 403 | "403"
  | 500 | "500"
```

<h2 id="faq">常见问题</h2>

### 如何隐藏默认的图标？

设置 `icon={null}` 或 `icon={false}` 可隐藏图标，此时仅展示标题和副标题。

```tsx
<Result icon={null} status="success" title="操作成功" />
```

### status 传入 404 和 "404" 有区别吗？

没有区别。`404`（数字）和 `"404"`（字符串）等价，源码内部统一转为字符串处理。建议使用字符串形式以保持一致。

### extra 和 children 有什么区别？

- `extra`：渲染在 title/subTitle 下方，通常放置操作按钮，支持数组。
- `children`：渲染在 `extra` 之后，用于更灵活的内容区域。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/result` 导入，不要从 `antd` 导入。
2. 优先使用 `status` 预设状态（success/error/info/warning）控制图标和语义。
3. `subTitle` 用于描述性文本，`title` 用于标题文本。
4. 操作按钮通过 `extra` 属性传入，支持单个或多个按钮。
5. 不要引入 Ant Design 相关依赖。

| `result-subtitle` | 副标题/描述文本 |

| `result-extra` | 操作按钮区域 |
