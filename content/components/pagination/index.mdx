Pagination 分页。用于分隔长列表，每次只加载一个页面。支持页码导航、每页条数切换、快速跳转、简洁模式和总数展示。

<h2 id="when-to-use">何时使用</h2>

当数据量较大无法在一页中完整展示时使用。

常见场景：

- 配合 Table 或 List 组件实现分页浏览。
- 服务端分页的数据列表页面。
- 需要用户切换每页显示数量时，开启 `showSizeChanger`。
- 管理后台的数据表格底部分页栏。

不推荐：

- 数据量小的静态列表（少于 10 条）——无需分页。
- 无限滚动场景——使用虚拟列表或 `loadMore` 模式。
- 前端一次性加载全部数据后做客户端分页——应使用服务端分页。

### 完整功能

```tsx
import { useState } from "react"
import { Pagination } from "@/components/ui/pagination"

export default function Demo() {
  const [current, setCurrent] = useState(1)
  const [pageSize, setPageSize] = useState(20)
  const total = 500
  return (
    <Pagination
      total={total}
      current={current}
      pageSize={pageSize}
      showSizeChanger
      showQuickJumper
      showTotal={(total, [from, to]) => `共 ${total} 条，当前 ${from}-${to}`}
      pageSizeOptions={[10, 20, 50, 100]}
      onChange={(page, size) => {
        setCurrent(page)
        setPageSize(size)
      }}
      onShowSizeChange={(current, size) => {
        console.log("pageSize changed:", size)
      }}
    />
  )
}
```

<h2 id="api">API</h2>

### Pagination

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| total | 数据总数 | `number` | `0` |
| current | 当前页码（受控） | `number` | - |
| defaultCurrent | 默认当前页码 | `number` | `1` |
| pageSize | 每页条数 | `number` | - |
| defaultPageSize | 默认每页条数 | `number` | `10` |
| onChange | 页码或每页条数变化回调 | `(page: number, pageSize: number) => void` | - |
| onShowSizeChange | 每页条数变化回调 | `(current: number, size: number) => void` | - |
| size | 分页器尺寸 | `"small"` \| `"middle"` | `"middle"` |
| align | 对齐方式 | `"start"` \| `"center"` \| `"end"` | `"center"` |
| simple | 简洁模式（readOnly 时不可编辑页码） | `boolean \| { readOnly?: boolean }` | `false` |
| showSizeChanger | 显示每页条数切换下拉框 | `boolean` | `false` |
| pageSizeOptions | 每页条数备选项 | `Array<number \| string>` | `[10, 20, 50, 100]` |
| showQuickJumper | 显示快速跳转输入框 | `boolean \| { goButton?: ReactNode }` | `false` |
| showLessItems | 显示较少页码（减少省略号跨度） | `boolean` | `false` |
| showTotal | 自定义总数展示 | `(total: number, range: [number, number]) => ReactNode` | - |
| hideOnSinglePage | 只有一页时隐藏分页器 | `boolean` | `false` |
| disabled | 禁用整个分页器 | `boolean` | `false` |
| itemRender | 自定义渲染页码/导航按钮 | `(page: number, type: PaginationItemType, originalElement: ReactNode) => ReactNode` | - |

### PaginationItemType

`"page"` | `"prev"` | `"next"` | `"jump-prev"` | `"jump-next"`，用于 `itemRender` 回调识别按钮类型。

### 子组件

| 组件 | 说明 |
| --- | --- |
| `PaginationContent` | `<ul>` 页码按钮列表容器 |
| `PaginationItem` | `<li>` 单个页码项 |
| `PaginationLink` | 页码链接，`isActive` 控制激活态，`asChild` 配合路由 |
| `PaginationPrevious` | 上一页按钮，默认显示 "Previous" |
| `PaginationNext` | 下一页按钮，默认显示 "Next" |
| `PaginationEllipsis` | 省略号，包含 `More pages` 屏幕阅读器文本 |

<h2 id="faq">常见问题</h2>

### 如何配合服务端分页？

使用受控模式，`current` 和 `pageSize` 从服务端获取，`onChange` 触发数据请求：

```tsx
const [current, setCurrent] = useState(1)
const [pageSize, setPageSize] = useState(20)
const { data, total } = useFetchData({ current, pageSize })

<Pagination
  total={total}
  current={current}
  pageSize={pageSize}
  onChange={(page, size) => {
    setCurrent(page)
    if (size !== pageSize) {
      setPageSize(size)
      setCurrent(1) // 条数变化时重置到第一页
    }
  }}
/>
```

### simple 模式的 readOnly 有什么作用？

`simple={{ readOnly: true }}` 时只显示"当前页 / 总页数"文本，不可编辑页码。适用于只读预览场景。

### showQuickJumper 的 goButton 如何自定义？

传入对象形式：`showQuickJumper={{ goButton: <span>跳转</span> }}`，将替换默认的 "Go" 按钮。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/pagination` 导入。
2. `onChange` 签名为 `(page, pageSize)` 两个参数，不仅返回 page。
3. 服务端分页务必使用受控模式（`current` + `pageSize` + `onChange`）。
4. `pageSize` 变化时需要自行处理页码重置逻辑。
5. 不要引入 Ant Design 相关依赖。
