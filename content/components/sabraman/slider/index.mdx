LegacySlider 是一个 Windows 经典风格的滑块组件，基于 Radix Slider 封装，提供简化的单值 API。

<h2 id="when-to-use">何时使用</h2>

用于在一个连续或离散区间中选择单个数值。

常见场景：

- 音量、亮度等连续值的调整。
- 数值范围的快速选择，例如价格区间、年龄范围。
- 需要还原 Windows 经典视觉风格的场景。
- 作为表单控件，替代数字输入框提供更直观的交互。

LegacySlider 将 Radix Slider 的数组值 API 简化为单数值：

- `value` / `defaultValue` 接收 `number` 而非 `number[]`。
- `onValueChange` / `onValueCommit` 回调参数为 `number`，不是数组。

<h2 id="examples">代码演示</h2>

### 基础用法

最基础的滑块。`min` 默认为 0，`max` 默认为 100。

  id="sabraman/slider"
  path="sabraman/slider/basic"
/>

```tsx
import { LegacySlider } from "@/components/ui/sabraman-slider"

export default function Demo() {
  return <LegacySlider defaultValue={50} />
}
```

### 受控滑块

传入 `value` 和 `onValueChange` 实现受控模式。

```tsx
import { useState } from "react"
import { LegacySlider } from "@/components/ui/sabraman-slider"

export default function Demo() {
  const [val, setVal] = useState(60)
  return (
    <div className="flex items-center gap-3">
      <span>{val}</span>
      <LegacySlider value={val} onValueChange={setVal} />
    </div>
  )
}
```

### 自定义范围

通过 `min` 和 `max` 设置滑动区间。

```tsx
import { LegacySlider } from "@/components/ui/sabraman-slider"

export default function Demo() {
  return <LegacySlider defaultValue={30} max={50} min={10} />
}
```

### 值提交回调

`onValueCommit` 在用户松开滑块时触发，适合需要在拖拽结束后才执行操作的场景。

```tsx
import { LegacySlider } from "@/components/ui/sabraman-slider"

export default function Demo() {
  return (
    <LegacySlider
      defaultValue={40}
      onValueCommit={(v) => console.log("committed:", v)}
    />
  )
}
```

<h2 id="api">API</h2>

### LegacySlider

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 当前值（受控） | `number` | - |
| defaultValue | 初始值（非受控） | `number` | `min` |
| min | 最小值 | `number` | `0` |
| max | 最大值 | `number` | `100` |
| onValueChange | 值变化回调 | `(value: number) => void` | - |
| onValueCommit | 值提交回调（松开时触发） | `(value: number) => void` | - |
| className | 附加样式类名 | `string` | - |

注意：`LegacySlider` 省略了 Radix Slider Root 的 `defaultValue`、`onValueChange`、`onValueCommit`、`value` 原始属性，替换为单值版本。其余 `SliderPrimitive.Root` 的原生属性可直接透传。

<h2 id="faq">常见问题</h2>

### 为什么 value 是 number 而不是 number[]？

`LegacySlider` 专门面向单值滑块场景，将 Radix 的数组 API 简化。如果你需要范围滑块（双滑块），应直接使用 Radix Slider。

### 如何获取滑块拖拽结束的值？

使用 `onValueCommit`，它在用户松开鼠标/手指时触发，参数为最终的 `number` 值。

### 可以用 step 控制步长吗？

可以。透传 Radix Slider 的 `step` 属性：

```tsx
<LegacySlider defaultValue={50} step={10} />
```

### 滑块样式可以覆盖吗？

通过 `className` 传入自定义样式。如需全局调整，修改 `src/components/sabraman/components/legacy-slider.tsx` 中的 Tailwind 类。

### 与 Radix Slider 的关系？

`LegacySlider` 是 Radix `SliderPrimitive.Root` 的封装，使用其 Track、Range、Thumb 子组件，但应用了 Windows 经典风格的渐变和阴影样式，并将数组值 API 简化为单值。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 LegacySlider 代码时：

1. 从 `@/components/ui/sabraman-slider` 导入 `LegacySlider`。
2. `value` 和 `defaultValue` 使用 `number` 类型，不是 `number[]`。
3. `onValueChange` 和 `onValueCommit` 回调参数为 `number`。
4. 不要从 `@radix-ui/react-slider` 直接导入 SliderPrimitive，使用封装好的 LegacySlider。
5. 不要将 LegacySlider 用于范围选择（双滑块），它仅支持单值。
