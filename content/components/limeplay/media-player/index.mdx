视频播放器 VideoPlayer。基于 Shaka Player 与 Media Chrome 的复合媒体播放器，提供完整的播放控制、字幕、画中画、倍速等能力。通过 `MediaProvider` 统一管理媒体状态，支持多源切换与播放列表。

<h2 id="when-to-use">何时使用</h2>

当需要在页面中嵌入功能完整的视频播放器时使用。VideoPlayer 内置错误恢复、加载海报、字幕容器和底部控制栏，开箱即用。

常见场景：

- 点播视频播放（VOD）。
- 直播流播放（HLS / DASH）。
- 需要播放列表管理的媒体应用。
- 需要自定义播放器 UI 的高级场景。

不推荐：

- 简单的视频嵌入需求 —— 使用更轻量的 `video-player` 组件。
- 嵌入第三方视频平台（YouTube、Bilibili 等）—— 使用对应平台 iframe 嵌入。
- 纯音频播放 —— 使用 `AudioPlayer` 组件。

### 基础用法

通过 `source` 传入媒体 URL，VideoPlayer 自动初始化 Shaka Player 并渲染完整播放控制。


<h2 id="api">API</h2>

### VideoPlayer

顶层容器组件，集成 `MediaProvider`、`PlaybackSourceController` 和完整播放器 UI。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| autoLoad | 是否自动加载媒体源 | `boolean` | - |
| children | 自定义子内容，渲染于播放器内部 | `ReactNode` | - |
| className | 根容器附加样式类名 | `string` | - |
| controlsHideDelay | 控制栏自动隐藏延迟（毫秒） | `number` | - |
| debug | 启用调试模式 | `boolean` | - |
| hideCursorOnIdle | 空闲时隐藏鼠标光标 | `boolean` | - |
| initialIndex | 初始播放项索引 | `number` | - |
| loading | 自定义资源加载配置 | `UseAssetOptions<VideoPlayerAsset>` | - |
| mediaProps | 透传至底层 `<video>` 元素的属性 | `Omit<VideoHTMLAttributes<HTMLVideoElement>, "as" \| "src">` | - |
| source | 媒体源 URL 或源配置 | `PlayerSource<VideoPlayerAsset>` | - |
| sourceKey | 源缓存键，用于触发重新加载 | `string` | - |

### VideoPlayerAsset

播放列表中单个资源项的类型定义。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| config | Shaka Player 配置 | `shaka.extern.PlayerConfiguration` | - |
| description | 资源描述 | `string` | - |
| id | 资源唯一标识 | `string` | - |
| poster | 海报图 URL | `string` | - |
| src | 媒体源 URL | `string` | - |
| title | 资源标题 | `string` | - |
| year | 发布年份 | `string` | - |

<h2 id="faq">常见问题</h2>

### VideoPlayer 和 video-player 组件有什么区别？

`limeplay` 下的 `VideoPlayer`（即本组件）是基于 Shaka Player 的完整媒体播放器，支持 DASH/HLS 流、DRM、自适应码率、播放列表等高级能力。`blocks` 下的 `video-player` 是更轻量的封装，基于原生 `<video>` 元素和 Media Chrome，适合简单点播场景。

### 如何传入直播流？

将 HLS（`.m3u8`）或 DASH（`.mpd`）流地址作为 `source` 传入即可：

```tsx
<VideoPlayer source="https://example.com/stream/master.m3u8" />
```

Shaka Player 会自动检测流格式并初始化对应的播放策略。

### 如何获取播放器内部状态？

使用 `usePlaybackStore` hook 访问播放状态：

```tsx
import { usePlaybackStore } from "@/components/limeplay/hooks/use-playback"

const status = usePlaybackStore((s) => s.status)
```

### 子元素会渲染在什么位置？

`children` 渲染在 `<Media>` 元素之后、控制栏叠加层之前，适合放置自定义叠加内容。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/limeplay-media-player` 导入 `VideoPlayer`，不要从 `antd` 导入（AntD 无此组件）。
2. `source` 支持字符串 URL 或 `PlayerSource<T>` 对象数组，传入 URL 即可快速启动。
3. 不要使用原生 `<video controls>` 替代本组件。
4. 本组件默认包含底部控制栏、字幕容器、错误恢复屏幕和加载海报，无需手动组合子组件。
5. 需要播放列表时，通过 `source` 传入对象数组，每个对象需包含 `src` 和可选的 `title`、`poster` 等元数据。
6. 调试播放问题时设置 `debug={true}` 启用 Shaka Player 调试日志。
7. 本组件为复合组件（Blocks），基于 Shaka Player + Media Chrome，不要引入 Ant Design 相关依赖。
