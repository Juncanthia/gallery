## 何时使用
控制视频或音频的播放速度，支持从 0.5x 到 2x 共七档速率选择。适用于在线课程、播客、视频回放等需要倍速播放的场景。

不应用于：实时直播流、无媒体上下文的 UI、需要精确帧控制的场景。

## PlaybackRateControl Props

PlaybackRateControl 是完全自包含的组合组件，无外部 Props。内部由 PlaybackRate.SelectRoot + Select.SelectTrigger + Select.SelectContent + PlaybackRate.SelectGroup 组合而成。

## PlaybackRate.SelectRoot Props

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| children | 子组件 | `ReactNode` | — |
| ...props | 继承 SelectPrimitive 全部属性 | `React.ComponentProps<typeof SelectPrimitive>` | — |

内部自动绑定 `value` 和 `onValueChange`，通过 usePlaybackRateStore 管理状态。

## PlaybackRate.SelectGroup Props

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| suffix | 速率后缀文本 | `string` | `"x"` |
| children | 分组内子组件（如 SelectLabel） | `ReactNode` | — |
| ...props | 继承 SelectGroupPrimitive 全部属性 | `ComponentProps<typeof SelectGroupPrimitive>` | — |

默认速率选项：[0.5, 0.75, 1, 1.25, 1.5, 1.75, 2]

### 读取当前速率
```tsx
import { usePlaybackRateStore } from "@/components/limeplay/hooks/use-playback-rate"

function CurrentRate() {
  const rate = usePlaybackRateStore((state) => state.value)
  return <span>当前速率: {rate}x</span>
}
```

### 程序化设置速率
```tsx
import { usePlaybackRateStore } from "@/components/limeplay/hooks/use-playback-rate"

function SetRate() {
  const setRate = usePlaybackRateStore((state) => state.setPlaybackRate)
  return <button onClick={() => setRate(1.5)}>设置 1.5x</button>
}
```

## 架构说明
- PlaybackRateControl 是 blocks 层的组合组件，封装了 PlaybackRate.SelectRoot + Select.SelectTrigger + Select.SelectContent + PlaybackRate.SelectGroup 的标准用法
- PlaybackRate.SelectRoot 继承 LimePlay 的 SelectPrimitive，内部通过 usePlaybackRateStore 绑定 value 和 onValueChange
- PlaybackRate.SelectGroup 接收 usePlaybackRateStore 的 rates 数组，自动渲染 SelectItem 列表
- 播放速率通过 media.playbackRate API 直接设置到媒体元素上
- 速率变化事件通过 MediaProvider 事件系统在 ratechange 时同步回 store

## 约束与最佳实践
1. 必须在 LimePlay MediaProvider 上下文中使用，且需注册 playbackRateFeature()
2. 默认速率选项不可删除 1x（正常速度），这是基准值
3. PlaybackRateControl 是推荐的直接使用入口，无需手动组合子组件
4. 需要自定义 UI 时使用 PlaybackRate.SelectRoot + PlaybackRate.SelectGroup 组合
5. SelectRoot 的 value 和 onValueChange 由 store 内部管理，不要在外部覆盖
6. suffix 属性修改的是显示文案，不影响实际速率值
7. 禁止在 MediaProvider 外部使用此组件，会抛出 "Missing MediaProvider in root" 错误
8. usePlaybackRateStore 只能在 MediaProvider 子树中使用
9. 速率值通过 media.playbackRate 直接设置，确保 mediaElement 已初始化
10. 组件依赖 zustand 和 @radix-ui/react-slot，确保项目已安装
