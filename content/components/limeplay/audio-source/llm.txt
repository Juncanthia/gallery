# AudioSourceProvider 音频源

向 LimePlay 播放器上下文注入音频资源数据的入口组件。内部通过 PlaybackSourceController 处理资源的解析、加载和回放控制，支持 `src` 直链和 `playbackUrls` 懒加载两种模式。

本组件是 LimePlay 播放器数据层的一部分，提供 `AudioPlayerAsset` 结构的标准音频数据注入能力。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js 或历史兼容语法糖。

## 何时使用 {#when-to-use}

适合在 LimePlay 播放器界面中加载音频资源。优先通过组件 props 表达数据语义，避免在业务侧重复手动操作 Store。

常见场景：

- 音频播放器中加载一个或多个音频文件。
- 音乐播放应用中注入带专辑、艺术家、封面等元数据的曲目。
- 在 `MediaProvider` 上下文中作为数据入口。

不推荐：

- 在非 MediaProvider 上下文之外独立使用。
- 视频播放场景 —— 应使用 VideoPlayer 或 MediaPlayer 组件的 source 属性。

## 代码演示 {#examples}

### 基础用法

```tsx
import { AudioSourceProvider } from "@/components/ui/limeplay-audio-source"

export default function Demo() {
  return (
    <AudioSourceProvider source="https://example.com/audio.mp3">
      <div>音频源已加载</div>
    </AudioSourceProvider>
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。source 支持字符串 URL、单个对象和数组三种形式。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/limeplay/blocks/audio-player/components/audio-source.tsx`（组件实现）
- `src/components/ui/limeplay-audio-source.tsx`（re-export wrapper）

直接依赖：

- `@hyper/limeplay/hooks/use-asset`（Asset, PlayerSource, UseAssetOptions, AssetRecoveryAction）
- `@hyper/limeplay/hooks/use-playback-source`（PlaybackSourceController）

实现注意：

1. `source` 支持三种形式：字符串 URL、单个 AudioPlayerAsset、AudioPlayerAsset 数组。
2. 字符串 URL 会自动转为 `[{ src: url }]`；单个对象转为 `[object]`。
3. 当同时提供 `src` 和 `playbackUrls` 时，优先使用 `src`。
4. 使用 `playbackUrls.primary` 时，组件向该端点发送 `?raw=true` 参数的 fetch 请求解析实际播放 URL。
5. 默认错误恢复策略：加载错误时 Skip 到下一首；网络错误（category === 1）时 Reload 当前资源。
6. `autoLoad` 默认 `true`，资源在挂载时自动加载到播放器。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 AudioSourceProvider 代码时，应遵守：

1. 从 `@/components/ui/limeplay-audio-source` 导入 `AudioSourceProvider`。
2. 必须在 `MediaProvider` 上下文中使用。
3. `source` 支持三种形式：字符串 URL、单个 AudioPlayerAsset 对象、AudioPlayerAsset 数组。
4. 每个 AudioPlayerAsset 至少需要 `src` 或 `playbackUrls.primary` 之一。
5. 如需元数据（标题、艺术家、封面），使用完整的 AudioPlayerAsset 对象。
6. 不要从 `antd` 导入任何音频相关组件。
7. 子组件可通过 `usePlaylistStore` / `usePlaybackStore` 读取注入的资源数据。
8. 只使用本文件 API 摘要和源码中存在的 props。
9. 如果不确定某个 API 是否存在，先检查 `src/components/limeplay/blocks/audio-player/components/audio-source.tsx`。
