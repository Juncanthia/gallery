TimelineControl 时间轴控制组件，基于 Radix Slider 的复合组件模式，用于媒体播放器的时间轴交互。通过组合 Root、Track、Progress、Buffered、Thumb 等子组件构建完整的时间轴控件，所有子组件通过 Zustand store 共享播放状态。

<h2 id="when-to-use">何时使用</h2>

当需要在媒体播放器中提供时间轴拖拽寻址、进度展示和缓冲区域显示时使用。

常见场景：

- 音频/视频播放器的进度条控制。
- 直播流的时间轴回看控制。
- 需要显示播放进度和缓冲进度的媒体界面。
- 需要 hover 预览时间位置的交互式时间轴。

不推荐：

- 纯静态进度展示（无播放器上下文），使用普通 Progress 组件即可。
- 非媒体场景的滑块控制，使用 Slider 组件。
- 在 TimelineControl 外部直接操作 Zustand store 状态。

TimelineControl 通过复合组件模式表达结构语义：

- `Root`：时间轴根容器，基于 SliderPrimitive.Root，管理拖拽寻址交互。
- `Track`：轨道容器，基于 SliderPrimitive.Track。
- `Progress`：已播放进度指示器，基于 SliderPrimitive.Indicator，宽度由播放进度动态计算。
- `Buffered`：缓冲区域指示器，显示已缓冲的时间范围。
- `Thumb`：位置指示器，支持固定位置或 hover 跟随。

```tsx
import * as TimelineControl from "@/components/ui/timeline-control"

export default function Demo() {
```

<h2 id="api">API</h2>

### Root

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| orientation | 时间轴方向 | `horizontal` \| `vertical` | `horizontal` |
| disabled | 禁用交互 | `boolean` | `false` |
| className | 附加样式类名 | `string` | - |

继承 Radix `Slider.Root` 的所有 props。自动连接播放器状态（currentTime、duration、isLive、readyState），在未加载元数据时自动禁用。处理 pointer 事件实现拖拽寻址和 hover 预览。

### Track

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

继承 Radix `Slider.Track` 的所有 props。轨道背景色为 `bg-foreground/20`，支持 focus-visible 键盘焦点样式。

### Progress

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

继承 Radix `Slider.Range` 的所有 props。宽度通过 CSS 变量 `--lp-played-width` 动态设置，背景色为 `bg-primary`。

### Buffered

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 缓冲区域渲染模式 | `combined` \| `default` \| `from-zero` | `default` |
| className | 附加样式类名 | `string` | - |

继承 `div` 元素的所有 props。`variant` 说明：
- `default`：分别显示每一段缓冲区域。
- `combined`：将多段缓冲合并为一段。
- `from-zero`：缓冲区域从零开始显示，末尾圆角。

无缓冲数据时返回 `null`。

### Thumb

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| position | 自定义位置百分比 | `number` | - |
| showWithHover | hover 时跟随鼠标位置 | `boolean` | `false` |
| className | 附加样式类名 | `string` | - |

继承 Radix `Slider.Thumb` 的所有 props。位置通过 CSS 变量 `--lp-timeline-thumb-position` 动态设置。未提供 `position` 且未启用 `showWithHover` 时，默认显示当前播放位置。duration 为 0 或无效时返回 `null`。

<h2 id="faq">常见问题</h2>

### 如何让 Thumb 跟随 hover 位置？

设置 `showWithHover`：

```tsx
<TimelineControl.Thumb showWithHover />
```

此时 Thumb 会在鼠标 hover 时跟随指针位置移动，松开后自动隐藏。

### Buffered 的三种 variant 何时使用？

- `default`：适用于大多数场景，分别显示每段缓冲。
- `combined`：适合视觉效果简洁的场景，合并所有缓冲区间。
- `from-zero`：适合强调"已缓冲总量"的场景，从零开始显示。

### TimelineControl 必须放在 Player 内部吗？

是的，Root 组件通过 `usePlayerStore` 和 `useTimelineStore` 获取播放器实例和状态。必须放在 Limeplay 播放器 Provider 的作用域内。

### 如何自定义 Thumb 样式？

通过 `className` 自定义外观。典型用法是设置 hover 时显示、默认透明：

```tsx
<TimelineControl.Thumb
  className="absolute h-6 w-1.5 rounded-full bg-primary opacity-0 transition-opacity group-hover/timeline:opacity-100"
  showWithHover
/>
```

### 如何与其他时间标签组件配合？

TimelineControl 与 `limeplay-timeline-labels` 组件协同使用，后者提供 `Elapsed`、`Duration`、`Remaining` 等时间标签。将 TimelineControl 放在 `flex` 布局中与时间标签并排即可。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/timeline-control` 导入，使用命名空间导入 `import * as TimelineControl from "..."` 或按需引入 `{ Root, Track, Progress, Buffered, Thumb }`。
2. Root、Track、Progress、Buffered、Thumb 必须按组合模式嵌套使用，不可独立使用。
3. Root 必须放在 Limeplay Player Provider 的作用域内。
4. Track 通常需要 `className="overflow-hidden"` 以限制 Progress 和 Buffered 的溢出。
5. Thumb 推荐使用 `showWithHover` 并配合 `group/timeline` 实现 hover 显示交互。
6. Buffered 的 `variant` 默认为 `"default"`，视觉简洁场景使用 `"combined"`。
7. 时间轴方向由 Root 的 `orientation` 控制，默认为 `horizontal`。
8. 不要引入 Ant Design 相关依赖。
