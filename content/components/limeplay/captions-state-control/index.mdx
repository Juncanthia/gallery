字幕状态控制按钮 CaptionsStateControl。预组装的字幕切换按钮，点击切换字幕可见性，根据可见状态自动更新图标样式。

<h2 id="when-to-use">何时使用</h2>

当需要在媒体播放器中提供字幕开关按钮时使用。CaptionsStateControl 通过 `useCaptionsStore` 读取当前字幕可见状态，点击时调用 `toggleVisibility()` 切换字幕显隐。

常见场景：

- 视频播放器的字幕开关按钮。
- 音频播放器的字幕/歌词切换入口。
- 快捷嵌入 MediaProvider 上下文内即开即用的字幕控件。

不推荐：

- 需要自定义按钮样式或行为时，应直接使用 `CaptionsControl`（ui 层）配合 `asChild` 自行组合。
- 在非 MediaProvider 上下文之外独立使用，它依赖 `useCaptionsStore` 和 `useCaptions`。
- 需要显示字幕文本容器时，必须配合 `CaptionsContainer` 使用。

<h2 id="api">API</h2>

### CaptionsStateControl

预组装的字幕开关按钮，无对外暴露的 props。内部使用 `CaptionsControl` 的 `asChild` 模式包裹 `Button`，配合 `ClosedCaptioningIcon` 展示状态。

**行为说明：**

| 行为 | 说明 |
| --- | --- |
| 点击切换 | 调用 `useCaptions().toggleVisibility()` 切换字幕显隐 |
| 图标状态 | `visible === true` 时图标为 `fill`，否则为 `regular` |
| 自动禁用 | 当无可用字幕轨道（`tracks.length === 0`）时按钮自动 disabled |
| `aria-label` | 自动生成 `"Captions"` |

### CaptionsControl（底层组件）

如需自定义按钮样式或行为，使用 `CaptionsControl` 并传入 props：

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 使用 Radix Slot 将状态和行为合并到子元素 | `boolean` | `false` |
| render | 渲染自定义子元素，与 `asChild` 互斥 | `React.ReactElement` | - |
| shortcut | 键盘快捷键提示，写入 `aria-keyshortcuts` 和 `aria-label` | `string` | - |
| disabled | 手动禁用按钮；当无可用字幕轨道时组件也会自动禁用 | `boolean` | - |
| onClick | 点击回调，在 `toggleVisibility()` 之前触发 | `(event: React.MouseEvent&lt;HTMLButtonElement&gt;) => void` | - |

组件继承 `React.ButtonHTMLAttributes&lt;HTMLButtonElement&gt;`，支持所有原生 button 属性。

**自动禁用逻辑：** 当 `textTracks` 为空或长度为 0 时，组件自动设置 `disabled`。

<h2 id="faq">常见问题</h2>

### 如何自定义字幕按钮样式？

不使用预组装的 `CaptionsStateControl`，改用底层 `CaptionsControl` 配合 `asChild` 自行组合：

```tsx
import { Button } from "@/components/ui/button"
import { CaptionsControl } from "@/components/ui/captions"
import { useCaptionsStore } from "@/components/media/hooks/use-captions"

function CustomCaptionsBtn() {
  const visible = useCaptionsStore((state) => state.visible)

  return (
    <CaptionsControl asChild shortcut="C">
      <Button size="icon" variant="outline">
        {visible ? "CC ON" : "CC OFF"}
      </Button>
    </CaptionsControl>
  )
}
```

### 为什么点击没有反应？

检查是否在 `MediaProvider` 上下文中使用。`CaptionsStateControl` 依赖 `useCaptionsStore` 和 `useCaptions`，必须在 `MediaProvider` 内部渲染。

### 如何让字幕文本显示出来？

`CaptionsStateControl` 只提供切换按钮，字幕文本需要 `CaptionsContainer` 来渲染：

```tsx
<>
  <CaptionsStateControl />
  <CaptionsContainer />
</>
```

### 如何读取字幕可见状态？

通过 `useCaptionsStore` 选择器读取：

```tsx
const visible = useCaptionsStore((state) => state.visible)
const tracks = useCaptionsStore((state) => state.tracks)
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/captions-state-control` 导入 CaptionsStateControl。
2. 必须配合 `MediaProvider` 使用，不能单独渲染。
3. 预组装组件无需传入 props，直接渲染即可。
4. 如需展示字幕文本，必须同时渲染 `CaptionsContainer`。
5. 如需自定义样式，使用底层 `CaptionsControl` 配合 `asChild`。
6. 不要引入 Ant Design 相关依赖。
