VideoPlayer 是一个基于 Shaka Player 的功能完整的视频播放器组件，支持 HLS/DASH 流媒体播放、播放列表、字幕、画中画、播放速率控制等功能。

<h2 id="when-to-use">何时使用</h2>

当需要在页面中嵌入视频播放功能时使用。VideoPlayer 封装了 Shaka Player + Media Chrome，在 `MediaProvider` 上下文中工作，提供开箱即用的播放控制和响应式 UI。

常见场景：

- 点播/直播视频播放页面。
- 需要播放列表管理的视频内容展示。
- 配合其他 limeplay 组件构建自定义播放控制界面。
- 通过 `source` 传入 URL 快速播放，或传入 `Asset[]` 构建播放列表。

不推荐：

- 在非 `MediaProvider` 上下文之外独立使用。VideoPlayer 内部已封装 `MediaProvider`，但外部如果想使用独立的控制组件需要确保在 Provider 内。
- 仅需简单 HTML5 video 标签的场景（直接用 `&lt;video&gt;` 或 `VideoPlayerContent`）。
- 音频播放场景，应使用 MediaPlayer 组件。

<h2 id="api">API</h2>

### VideoPlayer

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| source | 播放源，支持 URL 字符串、Asset 对象或 Asset 数组 | `string \| Asset \| Asset[]` | - |
| sourceKey | 用于在 source 变化时触发重新加载的标识 | `string` | - |
| autoLoad | source 变更时是否自动加载 | `boolean` | - |
| initialIndex | 播放列表初始索引 | `number` | - |
| loading | 资源加载策略配置 | `UseAssetOptions&lt;VideoPlayerAsset&gt;` | - |
| debug | 是否开启调试模式 | `boolean` | `false` |
| controlsHideDelay | 控制栏自动隐藏延迟（ms） | `number` | - |
| hideCursorOnIdle | 空闲时是否隐藏鼠标 | `boolean` | - |
| mediaProps | 传递给底层 video 元素的属性 | `Omit&lt;React.VideoHTMLAttributes&lt;HTMLVideoElement&gt;, "as" \| "src"&gt;` | - |
| className | 根容器附加样式类名 | `string` | - |
| children | 自定义子元素，渲染在视频层之上 | `React.ReactNode` | - |

### VideoPlayerAsset

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| src | 视频源 URL | `string` | - |
| id | 稳定性标识，用于播放列表和恢复状态 | `string` | - |
| title | 显示标题 | `string` | - |
| description | 描述信息 | `string` | - |
| poster | 封面海报图 URL | `string` | - |
| year | 年份 | `string` | - |
| config | Shaka Player 配置 | `shaka.extern.PlayerConfiguration` | - |

<h2 id="faq">常见问题</h2>

### 如何设置视频源？

通过 `source` 属性传入视频 URL 或 Asset 对象：

```tsx
// 直接传入 URL 字符串
<VideoPlayer source="https://example.com/video.m3u8" />

// 传入 Asset 对象，可附带元信息
<VideoPlayer source={{ src: "https://example.com/video.m3u8", title: "示例视频" }} />
```

### 如何构建播放列表？

传入 Asset 数组作为 `source`，配合 `initialIndex` 指定起始项：

```tsx
const playlist = [
  { src: "https://example.com/video1.m3u8", title: "视频一" },
  { src: "https://example.com/video2.m3u8", title: "视频二" },
]

<VideoPlayer source={playlist} initialIndex={0} />
```

### 如何自定义 VideoPlayer 内部控件？

使用 `children` 传入自定义 JSX。VideoPlayer 内部已包含 `BottomControls`（播放、进度、音量等），通过 `children` 可以在视频层之上叠加自定义 UI：

```tsx
<VideoPlayer source={src}>
  <div className="absolute left-4 top-4 text-white">自定义叠加层</div>
</VideoPlayer>
```

### 为什么视频无法播放？

检查以下几点：
1. 视频源 URL 是否可访问（CORS 配置正确）。
2. 是否在浏览器支持的格式（HLS 需 Media Source Extensions 支持）。
3. 设置 `debug={true}` 查看详细日志。

### sourceKey 有什么用？

当 `source` 引用不变但内容需要重新加载时（如同一 URL 的直播流重新连接），通过改变 `sourceKey` 触发重新加载：

```tsx
<VideoPlayer source={liveUrl} sourceKey={liveSessionId} />
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/limeplay-video-player` 导入 VideoPlayer。
2. `source` 支持 `string`（直接 URL）、`Asset` 对象或 `Asset[]`（播放列表）。
3. 组件已内置 `MediaProvider`，无需额外包裹。
4. 通过 `className` 控制尺寸和布局，推荐设置 `aspect-video`。
5. 不要引入 Ant Design 相关依赖。
