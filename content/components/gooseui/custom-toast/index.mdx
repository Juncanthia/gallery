CustomToast 提供带类型图标、进度条和手动关闭按钮的增强通知提示，基于 sonner 的 custom API 构建。

<h2 id="when-to-use">何时使用</h2>

当需要向用户反馈操作结果时使用。与原生 sonner toast 相比，CustomToast 提供更强的视觉区分和更丰富的配置。

常见场景：

- 操作成功确认：保存、提交、发布后的成功反馈。
- 操作失败提示：网络错误、校验失败、超时后的错误反馈。
- 警告提醒：数据可能丢失、配置冲突、即将超时等警告。
- 信息通知：版本更新、系统通知、状态变更等中性提示。

CustomToast 通过 `type` 区分提示语义，每种类型有独立的图标和颜色方案：

- `success`：绿色左边框 + CircleCheckIcon。
- `error`：红色左边框 + OctagonXIcon。
- `warning`：琥珀色左边框 + TriangleAlertIcon。
- `info`：蓝色左边框 + InfoIcon。

每条提示包含：类型图标、标题文本、可选描述文本、可选手动关闭按钮、底部进度条。

<h2 id="api">API</h2>

### CustomToast

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| id | 提示唯一标识，由 sonner 自动传入 | `string \| number` | - |
| type | 提示类型，决定图标和颜色 | `success` \| `error` \| `warning` \| `info` | - |
| message | 提示标题文本 | `string` | - |
| description | 提示补充描述文本，支持换行 | `string` | - |
| duration | 提示显示时长（毫秒），到期后进度条走完自动关闭 | `number` | `4000` |

### customToast

`customToast` 是创建提示的辅助对象，提供四个快捷方法：

| 方法 | 说明 | 签名 |
| --- | --- | --- |
| success | 成功提示 | `(message: string, options?: ToastOptions) => void` |
| error | 错误提示 | `(message: string, options?: ToastOptions) => void` |
| warning | 警告提示 | `(message: string, options?: ToastOptions) => void` |
| info | 信息提示 | `(message: string, options?: ToastOptions) => void` |

`ToastOptions`：

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| description | 提示补充描述文本 | `string` | - |
| duration | 提示显示时长（毫秒） | `number` | `4000` |

<h2 id="faq">常见问题</h2>

### 如何触发不同颜色的提示？

使用 `customToast` 的四个方法：

```tsx
customToast.success("保存成功")
customToast.error("删除失败")
customToast.warning("即将超时")
customToast.info("系统已更新")
```

### 如何添加描述文本？

通过 `options` 的 `description` 字段传入，支持 `\n` 换行：

```tsx
customToast.success("发布成功", {
  description: "文章已发布到线上。\n所有订阅者将收到通知。"
})
```

### 如何调整提示显示时长？

通过 `options` 的 `duration` 字段设置（单位：毫秒），默认 4000ms：

```tsx
customToast.info("短暂提示", { duration: 2000 })
customToast.warning("重要警告", { duration: 8000 })
```

### 与 sonner 原生的 toast 有何区别？

CustomToast 基于 sonner 的 `toast.custom` API 构建，增加了：

- 类型化视觉效果（图标 + 彩色左边框 + 进度条颜色）。
- 手动关闭按钮。
- 倒计时进度条。
- 封装的 `customToast` 辅助方法，减少重复配置。

同时保持与 sonner 完全兼容，Toaster 组件仍由 sonner 提供。

### 如何手动关闭提示？

每条提示右上角有关闭按钮（X 图标），点击即可关闭。也可以在代码中调用：

```tsx
import { toast } from "sonner"

// 通过 id 关闭
toast.dismiss(someId)
```

### 如何自定义提示样式？

CustomToast 使用 Tailwind CSS 构建。如需全局调整样式，修改 `src/components/gooseui/components/ui/custom-toast.tsx` 中的 class。如需调整某个 type 的颜色方案，修改 `config` 对象中的 `progressColor` 和 `iconColor`。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 CustomToast 代码时：

1. 从 `@/components/gooseui/lib/toast` 导入 `customToast`。
2. 从 `sonner` 导入 `Toaster` 并在应用根组件中渲染。
3. 从 `@/components/ui/gooseui-custom-toast` 导入 `CustomToast`（仅在需要直接使用组件时）。
4. 使用 `customToast.success()` / `.error()` / `.warning()` / `.info()` 触发提示。
5. 不要直接使用 sonner 的 `toast()` 或 `toast.custom()` 创建带类型的提示；使用 `customToast` 辅助方法。
6. `message` 为必填，`description` 为可选。
7. `duration` 单位为毫秒，默认 4000。
8. 不要在无 sonner `Toaster` 的组件中使用 `customToast`。
