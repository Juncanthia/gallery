BaselineStatus 用于展示 Web 特性的浏览器兼容状态，基于 W3C WebDX Baseline 规范。支持通过 `featureId` 获取 Web 特性的 Baseline 状态，也可直接传入 `status` 和 `year` 静态展示，还可结合 `browserCheck` 实时检测当前浏览器的特性支持情况。

<h2 id="when-to-use">何时使用</h2>

当需要在页面中展示某个 Web 平台特性的浏览器兼容性状态时使用。与普通 Badge 的区别：BaselineStatus 专注于 Web 特性兼容性语义（广泛支持 / 新可用 / 有限支持 / 无数据），Badge 用于通用状态标记。

常见场景：

- 技术文档中标注 CSS / JS 特性的浏览器支持状态。
- 组件或框架文档中说明所依赖的 Web 平台能力。
- 实时检测当前浏览器是否支持某个特性。
- 博客或教程中对新 Web API 的兼容性说明。

BaselineStatus 通过 W3C WebDX 官方的 Baseline 图标表达兼容性层级：

- `widely`：广泛可用，主流浏览器均已支持。
- `newly`：新可用，近期在各浏览器中互通。
- `limited`：有限支持，部分浏览器尚不支持。
- `no_data`：暂无数据。

不推荐：

- 将 BaselineStatus 用于通用状态标记——请使用 Badge。
- 在调用侧手动绘制 Baseline 图标绕过组件。

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `featureId` 自动获取 Web 特性的 Baseline 状态，或以 `status` + `year` 静态指定。

<ComponentShowcase id="gooseui/baseline-status" path="gooseui/baseline-status/basic" />

```tsx
import { BaselineStatus } from "@/components/data-display/gooseui/baseline-status"

export default function Demo() {
  return (
    <div className="flex flex-wrap items-center gap-4">
      <BaselineStatus featureId="container-queries" />
      <BaselineStatus featureId="has" size="sm" />
      <BaselineStatus status="widely" year={2022} />
      <BaselineStatus status="newly" year={2024} />
      <BaselineStatus status="limited" year={2024} />
      <BaselineStatus status="widely" year={2022} iconOnly />
    </div>
  )
}
```

### 浏览器实时检测

通过 `browserCheck` 实时检测当前浏览器是否支持指定特性，同时展示 Baseline 状态和浏览器支持结果。

```tsx
import { BaselineStatus } from "@/components/data-display/gooseui/baseline-status"

export default function Demo() {
  return (
    <div className="space-y-3">
      <BaselineStatus
        featureId="container-queries"
        browserCheck="container-queries"
      />
      <BaselineStatus featureId="has" browserCheck="has" />
      <BaselineStatus featureId="view-transitions" browserCheck="view-transitions" />
    </div>
  )
}
```

### 尺寸变体

支持 `sm`、`md`、`lg` 三种尺寸，默认为 `md`。

```tsx
import { BaselineStatus } from "@/components/data-display/gooseui/baseline-status"

      <BaselineStatus status="widely" year={2022} size="lg" />

```

### 纯图标模式

`iconOnly` 仅展示 Baseline 图标，不含文字标签。

```tsx
import { BaselineStatus } from "@/components/data-display/gooseui/baseline-status"

export default function Demo() {
  return (
    <div className="flex flex-wrap items-center gap-2">
      <BaselineStatus status="widely" year={2022} iconOnly />
      <BaselineStatus status="newly" year={2024} iconOnly />
      <BaselineStatus status="limited" year={2024} iconOnly />
      <BaselineStatus status="no_data" iconOnly />

```

### 服务端渲染

在服务端组件中，使用 `BaselineStatusStatic` 直接传入已获取的数据，避免客户端异步请求。

```tsx
import { BaselineStatusStatic } from "@/components/data-display/gooseui/baseline-status"

      <BaselineStatusStatic status="newly" year={2024} />
      <BaselineStatusStatic status="limited" year={2024} />
    </div>
  )
}
```

<h2 id="api">API</h2>

### BaselineStatus

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| featureId | Web 特性 ID，从 web-features 获取状态 | `string` | - |
| status | 直接指定状态（不用 featureId 时） | `"widely"` \| `"newly"` \| `"limited"` \| `"no_data"` | - |
| year | 特性可用年份 | `number` | - |
| size | 尺寸 | `"sm"` \| `"md"` \| `"lg"` | `"md"` |
| showYear | 标签中是否显示年份 | `boolean` | `true` |
| iconOnly | 仅显示图标不含文字 | `boolean` | `false` |
| browserCheck | 实时检测当前浏览器的特性支持 | `BrowserFeature` | - |
| className | 附加样式类名 | `string` | - |

### BaselineStatusStatic

与 `BaselineStatus` 相同，但不支持 `featureId`（数据由调用侧传入）。`status` 为必填。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| status | Baseline 状态（必填） | `"widely"` \| `"newly"` \| `"limited"` \| `"no_data"` | - |
| year | 特性可用年份 | `number` | - |
| size | 尺寸 | `"sm"` \| `"md"` \| `"lg"` | `"md"` |
| showYear | 标签中是否显示年份 | `boolean` | `true` |
| iconOnly | 仅显示图标不含文字 | `boolean` | `false` |
| className | 附加样式类名 | `string` | - |

### BrowserFeature

支持的浏览器特性检测项：

| 值 | 说明 |
| --- | --- |
| `"view-transitions"` | View Transitions API |
| `"popover"` | Popover API |
| `"anchor-positioning"` | CSS Anchor Positioning |
| `"container-queries"` | Container Queries |
| `"has"` | `:has()` 选择器 |
| `"scroll-driven-animations"` | Scroll-driven Animations |
| `"text-wrap-balance"` | `text-wrap: balance` |
| `"text-wrap-pretty"` | `text-wrap: pretty` |
| `"corner-shape"` | CSS `corner-shape` |

### BaselineStatusType

```ts
type BaselineStatusType = "widely" | "newly" | "limited" | "no_data"
```

## 与 Ant Design 的差异

| 差异点 | 说明 |
| --- | --- |
| 无 AntD 对应组件 | BaselineStatus 是 GooseUI 独有的 Web 特性兼容性展示组件 |
| W3C 官方图标 | 使用 W3C WebDX Community Group 授权的 Baseline 图标 |
| 浏览器检测 | 内置 `browserCheck` 实时检测当前浏览器特性支持 |
| 服务端渲染 | 提供 `BaselineStatusStatic` 用于服务端组件 |

<h2 id="faq">常见问题</h2>

### featureId 从哪里来？

`featureId` 对应 [web-platform-dx/web-features](https://github.com/web-platform-dx/web-features) 仓库中的特性 ID。组件内部通过 `getBaselineStatus()` 异步获取数据。

### featureId 和 status 有什么区别？

`featureId` 用于自动拉取 Web 特性的 Baseline 数据（异步请求）。`status` + `year` 用于直接静态指定状态，无需网络请求。两者互斥，`status` 优先。

### 什么时候用 BaselineStatusStatic？

在服务端组件（React Server Components）中使用。通过 `getBaselineStatus()` 在服务端获取数据后，将结果传给 `BaselineStatusStatic` 渲染，避免客户端额外的异步请求。

### 浏览器检测支持哪些特性？

当前支持 9 项 CSS/JS 特性检测，详见 API 文档中的 `BrowserFeature` 表格。检测基于 `CSS.supports()` 和 DOM 原型检查，无需外部依赖。

### 基线图标来源？

使用 W3C WebDX Community Group 的官方 Baseline 图标，遵循 [CC BY-ND 4.0](https://web-platform-dx.github.io/web-features/name-and-logo-usage-guidelines/) 许可。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 BaselineStatus 代码时：

1. 从 `@/components/data-display/gooseui/baseline-status` 导入 `BaselineStatus` 或 `BaselineStatusStatic`。
2. 优先使用 `featureId` 自动获取数据；静态展示使用 `status` + `year`。
3. 服务端组件使用 `BaselineStatusStatic`，不要用 `BaselineStatus`。
4. 需要浏览器检测时使用 `browserCheck`，不要手写 `CSS.supports` 逻辑。
5. `iconOnly` 用于纯图标场景，此时可省略 `showYear`。
6. 不要自行绘制 Baseline 图标绕过组件。
