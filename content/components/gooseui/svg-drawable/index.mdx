SVGDrawable 使用 CSS stroke-dasharray/stroke-dashoffset 为 SVG 路径、线条、多边形等元素添加绘制动画效果，支持关键帧、交错延迟、多种触发模式。

<h2 id="when-to-use">何时使用</h2>

当需要为 SVG 图标、Logo、插图中的路径元素添加"绘制"出现效果时使用，也可用于 hover 交互动画。

常见场景：

- 品牌 Logo 的入场动画。
- 图标系统的绘制效果。
- Hover 状态下 SVG 路径的描边动画。
- 信息图或插图的逐路径展示。

不推荐：

- 包含大量路径的复杂 SVG（性能开销大）。
- 填充型 SVG 图形（只对 stroke 路径生效）。
- 不支持 SVGGeometryElement 的浏览器环境（需 polyfill）。

<h2 id="api">API</h2>

### SVGDrawable

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 单个 SVG 子元素，必须包含可绘制的路径元素 | `React.ReactElement&lt;SVGElement&gt;` | - |
| draw | 绘制范围或关键帧数组 | `string` \| `string[]` | `"0 1"` |
| duration | 动画持续时间（ms） | `number` | `1000` |
| delay | 动画延迟（ms） | `number` | `0` |
| stagger | 路径之间的交错延迟（ms） | `number` | `0` |
| ease | 缓动函数 | `"linear"` \| `"ease"` \| `"ease-in"` \| `"ease-out"` \| `"ease-in-out"` | `"ease-in-out"` |
| trigger | 动画触发方式 | `"mount"` \| `"view"` \| `"hover"` | `"mount"` |
| animateOnMount | ⚠️ 已废弃，使用 trigger="mount" 代替 | `boolean` | `true` |
| animateOnView | ⚠️ 已废弃，使用 trigger="view" 代替 | `boolean` | `false` |
| loop | 循环播放 | `boolean` | `false` |
| direction | 动画方向 | `"normal"` \| `"reverse"` \| `"alternate"` | `"normal"` |
| className | 附加 CSS 类名 | `string` | - |

SVGDrawable 通过 `useInsertionEffect` 将 CSS 样式注入 `document.head`。子 SVG 中的可绘制元素（path、line、polyline、polygon、rect、circle、ellipse）被自动注入唯一 class，每个实例通过 `useId()` 生成唯一的 animation name，确保多个实例互不冲突。

`draw` 参数支持两种格式：
- 简单范围：`"0 1"` 表示从起点（dashoffset=length）到终点（dashoffset=0）的完整绘制。
- 关键帧数组：`["0 0", "0 1", "1 1"]` 定义多阶段动画，用于实现"绘制并擦除"或"部分绘制"效果。

`trigger` 控制动画触发时机：
- `"mount"`：组件挂载时立即播放。
- `"view"`：使用 IntersectionObserver，元素进入视口时播放。
- `"hover"`：使用 CSS `:hover` + transition，鼠标悬停时绘制。

<h2 id="faq">常见问题</h2>

### 如何实现循环绘制动画？

设置 `loop` 属性为 `true`，动画将无限循环播放。

```tsx
<SVGDrawable draw="0 1" loop>
  <svg viewBox="0 0 100 100">
    <path d="..." fill="none" stroke="currentColor" />
  </svg>
</SVGDrawable>
```

### 如何实现多阶段关键帧动画？

使用关键帧数组格式的 `draw` 属性。

```tsx
<SVGDrawable draw={["0 0", "0 1", "1 1"]} duration={3000}>
  <svg viewBox="0 0 100 100">
    <path d="..." fill="none" stroke="currentColor" />
  </svg>
</SVGDrawable>
```

### 如何让多条路径依次绘制？

设置 `stagger` 属性，为每条路径添加交错延迟。

```tsx
<SVGDrawable draw="0 1" stagger={150}>
  <svg viewBox="0 0 200 60">
    <path d="..." fill="none" stroke="currentColor" />
    <path d="..." fill="none" stroke="currentColor" />
  </svg>
</SVGDrawable>
```

### 如何在 hover 时触发绘制？

设置 `trigger="hover"`，鼠标悬停时通过 CSS transition 绘制路径。

```tsx
<SVGDrawable draw="0 1" trigger="hover" duration={500}>
  <svg viewBox="0 0 100 100">
    <path d="..." fill="none" stroke="currentColor" />
  </svg>
</SVGDrawable>
```

### 多个实例会冲突吗？

不会。每个实例通过 React `useId()` 生成唯一的 animation name 和 class 前缀，CSS 样式也通过唯一 ID 注入，确保互不冲突。

### 如何手动控制动画？

使用 `useDrawable` hook 获得命令式控制能力。

```tsx
import { useDrawable } from "@/components/ui/gooseui-svg-drawable"

function ManualControl() {
  const svgRef = useRef<SVGSVGElement>(null)
  const { animate, reset, isAnimating } = useDrawable(svgRef, {
    draw: "0 1",
    duration: 2000,
  })

  return (
    <div>
      <svg ref={svgRef} viewBox="0 0 100 100">
        <path d="..." fill="none" stroke="currentColor" />
      </svg>
      <button onClick={animate} disabled={isAnimating}>播放</button>
      <button onClick={reset}>重置</button>
    </div>
  )
}
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/gooseui-svg-drawable` 导入 `SVGDrawable`。
2. `children` 必须是单个 `<svg>` React 元素，其中包含带 `fill="none"` 和 `stroke="currentColor"` 的路径元素。
3. `draw` 使用 `"0 1"` 字符串格式或 `["0 0", "0 1", "1 1"]` 关键帧数组，不要传入其他格式。
4. `trigger` 使用 `"mount"`、`"view"`、`"hover"` 字符串，不要使用已废弃的 `animateOnMount` / `animateOnView`。
5. `ease` 使用 `"linear"`、`"ease"`、`"ease-in"`、`"ease-out"`、`"ease-in-out"` 字符串，不要传入自定义 cubic-bezier。
6. 需要命令式控制时使用 `useDrawable` hook 而非手动操作 DOM。
7. 不要引入 Ant Design 相关依赖。
