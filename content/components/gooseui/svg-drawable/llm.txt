# SVGDrawable SVG 绘制动画

使用 CSS stroke-dasharray/stroke-dashoffset 为 SVG 路径、线条、多边形等元素添加绘制动画效果。

本组件借鉴 GooseUI 的 SVG 绘制动画使用心智，但不是第三方的兼容层。实现以本地源码为准，不引入外部运行时依赖，CSS 通过 `useInsertionEffect` 注入。

## 何时使用 {#when-to-use}

当需要为 SVG 图标、Logo、插图中的路径元素添加"绘制"出现效果时使用，也可用于 hover 交互动画。

常见场景：

- 品牌 Logo 的入场动画。
- 图标系统的绘制效果。
- Hover 状态下 SVG 路径的描边动画。
- 信息图或插图的逐路径展示。

不推荐：

- 包含大量路径的复杂 SVG（性能开销大）。
- 填充型 SVG 图形（只对 stroke 路径生效）。
- 不支持 SVGGeometryElement 的浏览器环境（需 polyfill）。

## 代码演示 {#examples}

### 基础用法

```tsx
import { SVGDrawable } from "@/components/effects/interactions/gooseui/svg-drawable"

export default function Demo() {
  return (
    <SVGDrawable draw="0 1" duration={2000}>
      <svg viewBox="0 0 100 100">
        <path d="..." fill="none" stroke="currentColor" />
      </svg>
    </SVGDrawable>
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。更多配置项参见下方 API 摘要。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface SVGDrawableProps {
  /** 单个 SVG 子元素，必须包含可绘制的路径元素 */
  children: React.ReactElement<SVGElement>;
  /** 绘制范围或关键帧数组："0 1" 或 ["0 0", "0 1", "1 1"] */
  draw?: string | string[];
  /** 动画持续时间（ms） */
  duration?: number;
  /** 动画延迟（ms） */
  delay?: number;
  /** 路径之间的交错延迟（ms） */
  stagger?: number;
  /** 缓动函数 */
  ease?: "linear" | "ease" | "ease-in" | "ease-out" | "ease-in-out";
  /** 动画触发方式 */
  trigger?: "mount" | "view" | "hover";
  /** @deprecated 使用 trigger="mount" 代替 */
  animateOnMount?: boolean;
  /** @deprecated 使用 trigger="view" 代替 */
  animateOnView?: boolean;
  /** 循环播放 */
  loop?: boolean;
  /** 动画方向 */
  direction?: "normal" | "reverse" | "alternate";
  /** 附加 CSS 类名 */
  className?: string;
}
```

## Semantic DOM {#semantic-dom}

SVGDrawable 内部结构：

- 外层 `<div ref={containerRef}>` 用作 IntersectionObserver 或 hover 的监听目标。
- 子 SVG 中可绘制的路径元素（path、line、polyline、polygon、rect、circle、ellipse）被自动注入唯一的 class 名称（`drawable-{id}-{index}`）。
- CSS 样式通过 `useInsertionEffect` 注入到 `document.head`，每个实例使用 React `useId()` 生成唯一的 animation name，避免冲突。

## 样式模型 {#styling-model}

- 动画使用 CSS `stroke-dasharray` + `stroke-dashoffset` + `@keyframes` 实现。
- `trigger="hover"` 模式下使用 CSS `transition` 代替 keyframe 动画。
- 外层容器默认 `inline-block`，hover 模式下添加 `cursor-pointer`。
- 所有样式通过 `useInsertionEffect`（React 18+ CSS-in-JS 标准 hook）注入，组件卸载时自动清理。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/ui/gooseui-svg-drawable.tsx`（re-export）
- `src/components/gooseui/components/ui/svg-drawable.tsx`（实现）

直接依赖摘要：

- React（`useId`、`useInsertionEffect`、`cloneElement`）
- `cn` from `@hyper/gooseui/lib/utils`

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. `draw` 参数支持简单范围 `"0 1"` 和关键帧数组 `["0 0", "0 1", "1 1"]` 两种格式。
3. `trigger="view"` 使用 IntersectionObserver，`threshold: 0.1`。
4. `trigger="hover"` 使用 CSS `:hover` 伪类 + transition，通过外层容器的 class 选择器匹配。
5. `animateOnMount` 和 `animateOnView` 已废弃，推荐使用 `trigger` 属性代替。
6. `useDrawable` hook 提供命令式控制（`animate()` / `reset()`），适合需要手动触发动画的场景。

## 可访问性 {#accessibility}

- 动画基于 CSS，不改变 DOM 结构或文本内容。
- 尊重 `prefers-reduced-motion: reduce` 时建议业务侧自行关闭动画（组件本身不内置该检测）。
- SVG 元素本身对屏幕阅读器透明，无需额外 aria 处理。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 SVGDrawable 代码时，应遵守：

1. 从 `@/components/effects/interactions/gooseui/svg-drawable` 导入 `SVGDrawable`。
2. `children` 必须是单个 `<svg>` React 元素，其中包含带 `fill="none"` 和 `stroke="currentColor"` 的路径元素。
3. `draw` 使用 `"0 1"` 字符串格式或 `["0 0", "0 1", "1 1"]` 关键帧数组，不要传入其他格式。
4. `trigger` 使用 `"mount"`、`"view"`、`"hover"` 字符串，不要使用已废弃的 `animateOnMount` / `animateOnView`。
5. `ease` 使用 `"linear"`、`"ease"`、`"ease-in"`、`"ease-out"`、`"ease-in-out"` 字符串，不要传入自定义 cubic-bezier。
6. 需要命令式控制时使用 `useDrawable` hook 而非手动操作 DOM。
7. 不要引入 Ant Design 相关依赖。
8. 不要引入 anime.js 或其他动画库依赖。
