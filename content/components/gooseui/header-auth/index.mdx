HeaderAuth 是一个带认证状态的自适应导航头部组件。支持登录/未登录双态切换，桌面端显示水平导航栏和认证按钮，移动端折叠为汉堡菜单。

<h2 id="when-to-use">何时使用</h2>

当页面需要一个顶部导航栏，且需根据用户登录状态显示不同操作入口时使用。适用于 Web 应用的全局 header、SaaS 产品导航栏、管理后台顶部栏等场景。

常见场景：

- SaaS 应用全局导航，登录后显示用户信息和退出按钮。
- 产品落地页顶部，引导用户登录或注册。
- 管理后台或控制台的顶部工具栏。

不推荐：

- 不需要认证状态切换的纯静态导航（应使用更简单的导航组件）。
- 需要复杂用户下拉菜单（个人设置、切换账号等）的场景。

<h2 id="examples">代码演示</h2>

### 基础用法

默认 `isLoggedIn=false`，显示 "Log in" / "Sign up" 按钮。桌面端为水平导航栏，移动端收缩为汉堡菜单。

### 已登录状态

设置 `isLoggedIn={true}` 后，右侧显示用户头像（User 图标）、用户名和 Sign out 按钮。

```tsx
import { HeaderAuth } from "@/components/ui/header-auth"

export default function Demo() {
  return <HeaderAuth isLoggedIn userName="Alex" />
}
```

<h2 id="api">API</h2>

### HeaderAuth

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| isLoggedIn | 是否已登录，切换认证 UI 显示模式 | `boolean` | `false` |
| userName | 登录用户名，仅在 isLoggedIn 为 true 时显示 | `string` | `"John"` |

内部状态：

- `isOpen`（`React.useState(false)`）：控制移动端汉堡菜单的展开/收起。点击汉堡按钮切换，点击导航链接后自动关闭。

导航项 `navItems` 当前硬编码在组件内部（Dashboard / Projects / Team / Reports），通过 `next/link` 的 `Link` 组件渲染。

<h2 id="faq">常见问题</h2>

### 如何自定义导航项？

导航项 `navItems` 当前硬编码在组件源码 `src/components/gooseui/components/blocks/headers/header-auth.tsx` 中。如需修改，编辑该文件中的 `navItems` 数组即可。

```tsx
const navItems = [
  { label: "首页", href: "/" },
  { label: "产品", href: "/products" },
  { label: "关于", href: "/about" },
]
```

### 如何接入真实的登录逻辑？

组件只提供 UI 展示层，不包含认证逻辑。登录/登出按钮目前为纯展示，不触发回调。实现在 `isLoggedIn` 状态切换和按钮点击逻辑需要，在业务侧包裹组件或在源码中扩展：

1. 将 `isLoggedIn` 和 `userName` 从业务层的认证状态中获取（如 auth context）。
2. 为 Button 添加 `onClick` 处理，触发登录/登出流程。

### 移动端菜单如何工作？

屏幕宽度 &lt; 768px 时，水平导航栏隐藏，右侧出现汉堡菜单按钮。点击按钮后移动端菜单展开，显示导航链接和认证按钮。导航链接和认证按钮均设置 `w-full` 全宽，点击导航项后菜单自动关闭。

### 能否在非 Next.js 项目中使用？

组件内部使用 `next/link` 的 `Link` 组件进行客户端导航。在非 Next.js 项目中需将 `Link` 替换为对应路由库的导航组件（如 React Router 的 `<Link>`）。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 HeaderAuth 代码时：

1. 从 `@/components/ui/header-auth` 导入 `HeaderAuth`。
2. 未登录场景直接使用 `<HeaderAuth />`，显示 Log in / Sign up 按钮。
3. 已登录场景设置 `isLoggedIn` 和 `userName` 属性。
4. 不要自行拼装 header 导航 DOM 结构，使用组件内置渲染。
5. 不要从 gooseui 内部路径导入（如 `@/components/marketing-blocks/gooseui/headers/header-auth`）。
6. 不要引入 Ant Design 相关依赖或 shadcn/ui 的 navigation-menu。
7. 导航项自定义需在组件源码中修改，不要在调用处拼 nav 列表。
