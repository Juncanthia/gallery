MorphingHeader 基于 View Transitions API 实现的变形头部组件。页面滚动时，Header 在完整模式与紧凑模式之间平滑过渡，Logo 缩放、导航间距变化、辅助元素显隐均可随 Header 状态联动。

<h2 id="when-to-use">何时使用</h2>

需要顶部导航在页面滚动时动态改变形态的场景。

常见场景：

- 品牌官网的顶部导航，滚动后缩小以节省空间。
- 内容型网站（博客、文档站）的粘性头部。
- SaaS 产品的全局导航栏，完整态展示产品名，紧凑态仅保留 Logo 和核心导航。
- Landing Page 中需要固定导航但又不想遮挡内容区域的场景。

不推荐：

- 固定高度、不需要形态变化的简单导航栏。
- 对 View Transitions API 兼容性有严格要求的场景（如必须支持 Firefox/Safari 旧版本）。
- 内容极少、无需滚动的单屏页面。

MorphingHeader 使用浏览器原生 View Transitions API 实现状态切换动画，配合 CSS transition 实现尺寸和间距的平滑过渡。无需 JavaScript 动画库。

### 基础用法

MorphingHeader 通过 `compactThreshold` 控制触发阈值，子组件 MorphingLogo 和 MorphingNav 根据 `isCompact` 状态调整自身样式。

```tsx
import { MorphingHeader, MorphingLogo, MorphingNav } from "@/components/effects/interactions/gooseui/morphing-header"

export default function Demo() {
  return (
    <div className="min-h-[200vh]">
      <MorphingHeader compactThreshold={100}>
        {({ isCompact }: { isCompact: boolean }) => (
          <div className="flex items-center justify-between">
            <div className="flex items-center gap-4">
              <MorphingLogo
                src="https://ui.shadcn.com/placeholder.svg"
                alt="Logo"
                compactSize={24}
                fullSize={36}
                isCompact={isCompact}
              />
              {!isCompact && (
                <span className="text-lg font-semibold">MorphingHeader</span>
              )}
            </div>
            <MorphingNav isCompact={isCompact}>
              <a href="#features" className="hover:text-foreground/80">Features</a>
              <a href="#docs" className="hover:text-foreground/80">Docs</a>
              <a href="#about" className="hover:text-foreground/80">About</a>
            </MorphingNav>
          </div>
        )}
      </MorphingHeader>
    </div>
  )
}
```

<h2 id="api">API</h2>

### MorphingHeader

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 头部内容，支持 render prop `(isCompact: boolean) => ReactNode` | `React.ReactNode` \| `(isCompact: boolean) => React.ReactNode` | - |
| className | 附加 CSS 类名 | `string` | - |
| compactThreshold | 触发紧凑模式的滚动位置阈值（px） | `number` | `100` |

### MorphingLogo

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| src | Logo 图片地址 | `string` | - |
| alt | 图片替代文本 | `string` | `""` |
| className | 附加 CSS 类名 | `string` | - |
| compactSize | 紧凑模式下的高度（px） | `number` | `32` |
| fullSize | 完整模式下的高度（px） | `number` | `48` |
| isCompact | 当前 Header 是否为紧凑模式 | `boolean` | `false` |

### MorphingNav

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 导航内容 | `React.ReactNode` | - |
| className | 附加 CSS 类名 | `string` | - |
| isCompact | 当前 Header 是否为紧凑模式 | `boolean` | `false` |

MorphingHeader 使用 View Transitions API 管理状态切换动画。`document.startViewTransition` 可用时自动启用过渡动画，不可用时直接切换状态（无动画）。组件通过 `passive: true` 的 scroll 事件监听优化滚动性能。`MorphingHeader` 的 `children` 支持 render prop 模式，在类型为 function 时传入当前 `isCompact` 状态，调用侧可据此控制子元素的显隐和样式。

<h2 id="faq">常见问题</h2>

### 如何在非 Chromium 浏览器中降级？

View Transitions API 当前主要在 Chromium 125+ 中可用。MorphingHeader 内部已做了降级处理：当 `document.startViewTransition` 不可用时，状态切换直接生效（无过渡动画），但 CSS transition 仍然有效。如需更完善的降级方案，可通过 `className` 传入自定义过渡样式。

### children 的 render prop 和普通 ReactNode 有什么区别？

当 `children` 类型为 function 时，MorphingHeader 会在每次渲染时调用该函数并传入当前 `isCompact` 布尔值。调用侧可利用此值控制子元素的显隐（如隐藏标题文字）或样式变化。当 `children` 为普通 ReactNode 时直接渲染，不传入额外参数。

### 如何调整紧凑触发的滚动距离？

通过 `compactThreshold` 属性设置，单位为 px。默认值为 `100`，即页面滚动超过 100px 时切换到紧凑模式。

### MorphingLogo 和 MorphingNav 可以单独使用吗？

这两个子组件设计为配合 MorphingHeader 使用，通过 `isCompact` 属性接收 Header 的状态。理论上可独立使用，但 `isCompact` 需要由外部状态管理。推荐在 MorphingHeader 的 render prop 内使用它们，确保状态同步。

### 如何自定义 Header 的高度和样式？

通过 `className` 传入自定义 Tailwind 类名。紧凑/完整模式下的垂直内边距分别由 `py-2` 和 `py-4` 控制。如需修改默认内边距，可通过 className 覆盖，如 `className="[&.py-2]:py-3 [&.py-4]:py-6"` 或直接替换 padding 相关类名。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 MorphingHeader 代码时：

1. 从 `@/components/effects/interactions/gooseui/morphing-header` 导入 `MorphingHeader`、`MorphingLogo`、`MorphingNav`。
2. `MorphingHeader` 的 `children` 推荐使用 render prop 模式 `({ isCompact }) => (...)` 以便子组件根据 Header 状态联动。
3. `MorphingLogo` 必须传入 `src` 和 `isCompact` 属性。
4. `MorphingNav` 的 `isCompact` 属性应与 `MorphingHeader` 的状态保持同步。
5. 不要引入 `framer-motion`、`@react-spring` 等第三方动画库。
6. 不要引入 Ant Design 相关依赖。
7. 如需滚动演示效果，页面内容区域需设置足够高度（如 `min-h-[200vh]`）。
