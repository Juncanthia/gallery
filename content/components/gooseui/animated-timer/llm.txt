# AnimatedTimer 数字翻页时钟

带渐变色边框的数字翻页时钟，通过垂直滚动动画切换数字，支持 12/24 小时制和秒显控制。

本组件借鉴 GooseUI 的时钟动画使用心智，但不是第三方的兼容层。实现以本地源码为准，不引入外部运行时依赖或 CSS-in-JS。

## 何时使用 {#when-to-use}

适合在需要动态翻页时钟效果的场景使用。优先通过组件自身 props 表达配置，避免在业务侧重复拼接 className。

常见场景：

- 仪表盘、监控面板的实时时钟。
- 品牌首页 Hero 区域的动态时间展示。
- 需要视觉吸引力的时间展示区域。

不推荐：

- 需要精确到毫秒的高频刷新场景。
- 对性能极其敏感的 SSR 渲染列表。

## 核心导出 {#exports}

- `AnimatedTimer`
- `AnimatedTimerProps`

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface AnimatedTimerProps {
  /** 是否显示秒 */
  showSeconds?: boolean;
  /** 是否使用 24 小时制 */
  use24Hour?: boolean;
  /** 附加 CSS 类名 */
  className?: string;
}
```

默认值：`showSeconds=true`，`use24Hour=true`。

## Semantic DOM {#semantic-dom}

AnimatedTimer 渲染为外层渐变色边框包裹内层深色背景的数字时钟。

DOM 结构：

- 外层容器：`rounded-xl bg-gradient-to-br from-blue-600 to-red-500 p-1`（渐变色边框）。
- 内层容器：`bg-zinc-900 rounded-lg`（深色背景，包裹所有数字位）。
- 每个数字位 `TimerChar`：`relative overflow-hidden` 容器，内部 `absolute` 定位的数字列。通过 `top` 样式驱动垂直滚动，`transition-[top] duration-200 ease-out` 实现平滑过渡。
- 冒号分隔符：`flex items-center justify-center`，静态渲染 `<span>` 显示冒号。
- 当前数字：`opacity-100` + 大字号；非当前数字：`opacity-5` + 小字号。
- 骨架态（SSR / 未挂载）：数字显示 `opacity-20`，无动画。

响应式尺寸通过 Tailwind 断点控制（`h-[60px] md:h-[100px] lg:h-[150px]`）。

## 样式模型 {#styling-model}

- 渐变色边框由外层 `div` 的 `bg-gradient-to-br from-blue-600 to-red-500 p-1` + 内层 `bg-zinc-900` 实现。
- 数字滚动动画由 `transition-[top] duration-200 ease-out` 驱动。
- 数字位使用 `overflow-hidden` 裁剪视口外的数字。
- 所有样式使用 Tailwind CSS，不依赖 CSS-in-JS 或运行时 token。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/ui/gooseui-animated-timer.tsx`（re-export）
- `src/components/gooseui/components/ui/animated-timer.tsx`（实现）

直接依赖摘要：

- React（`useState`、`useEffect`、`useRef`）
- `cn` 工具函数（`@hyper/gooseui/lib/utils`）

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 挂载前渲染骨架占位，SSR 阶段不启动计时器。
3. 通过 `setInterval` 每秒更新，`useEffect` cleanup 清理定时器。
4. `TimerChar` 子组件通过 `offsetHeight` 计算单字符高度驱动滚动位置。
5. 冒号分隔符为静态渲染，不参与滚动动画。

## 可访问性 {#accessibility}

- 数字位通过 `span` 元素渲染，内容为纯数字文本。
- 冒号分隔符使用 `span` 渲染，对屏幕阅读器可见。
- 骨架态与挂载态内容结构一致，无突兀的 DOM 替换。
- 无闪烁或高频动画，不会触发 `prefers-reduced-motion` 兼容问题。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/gooseui/components/ui/animated-timer.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS + CSS transition 动画 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 非 AntD 组件，无兼容需求 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 AnimatedTimer 代码时，应遵守：

1. 从 `@/components/ui/gooseui-animated-timer` 导入 `AnimatedTimer`。
2. 使用 `showSeconds` 控制秒显（`boolean`），`use24Hour` 控制小时制（`boolean`）。
3. 不要自行实现数字滚动动画，使用组件内置的 `TimerChar` 动画机制。
4. 不要在 SSR 阶段依赖组件输出精确时间，挂载后才会启动计时。
5. 不要引入 Ant Design 相关依赖。
6. 自定义样式通过 `className` 传入，不要在调用处覆写核心渐变边框。
