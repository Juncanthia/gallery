ThemeToggle 基于 next-themes 和 framer-motion 实现的亮色/暗色主题切换按钮，点击可在 light 和 dark 模式间切换，带有 SVG 形变动画。

&lt;h2 id="when-to-use"&gt;何时使用&lt;/h2&gt;

当需要为页面或应用提供亮色/暗色模式切换入口时使用。该组件自动读取当前主题状态，通过形变动画在太阳图标（亮色模式）和月亮图标（暗色模式）之间平滑过渡。

常见场景：

- 网站导航栏或页脚中的主题切换按钮。
- 设置面板或偏好页面中的主题切换控件。
- 文档站点、博客或个人主页的主题切换入口。

不推荐：

- 需要多种主题色（如 zinc、red、blue 等）切换的场景，请使用 ThemeCustomizer。
- 需要 system / light / dark 三态切换的场景，请使用 CSSThemeSwitcher。

ThemeToggle 依赖 next-themes 的 `ThemeProvider`，通过 `useTheme` hook 读取并切换主题。首次渲染时显示占位骨架（圆角脉冲方块），客户端挂载后切换到实际图标，避免 hydration 不匹配。

- `next-themes`：提供 `useTheme` hook，需在应用根节点包裹 `ThemeProvider`。
- `framer-motion`（motion）：驱动 SVG 路径形变动画和光芒射线的交错动画。

&lt;h2 id="faq"&gt;常见问题&lt;/h2&gt;

### ThemeToggle 和 ThemeCustomizer 有什么区别？

ThemeToggle 仅提供 light/dark 两态切换，动画为太阳/月亮形变。ThemeCustomizer 提供 7 种主题色（zinc、red、orange、green、blue、violet、pink）+ light/dark 模式，有 6 种布局变体（Pill、Bar、Sidebar、Dock、Corner、Toolbar）。如果只需要简单的亮暗切换，使用 ThemeToggle；如果需要丰富的主题定制，使用 ThemeCustomizer。

### 应用根节点需要什么配置？

ThemeToggle 依赖 next-themes 的 `ThemeProvider`，需在应用根组件中包裹：

```tsx
import { ThemeProvider } from "next-themes"

export default function RootLayout({ children }) {
  return (
    &lt;ThemeProvider attribute="class" enableSystem={false}&gt;
      {children}
    &lt;/ThemeProvider&gt;
  )
}
```

注意 `enableSystem` 设为 `false` 时，切换仅在 light/dark 之间进行，与 ThemeToggle 的二态切换逻辑一致。

### SVG 动画依赖什么库？

ThemeToggle 使用 framer-motion 的 `motion` 组件驱动 SVG 动画：
- 太阳/月亮主体路径（`d` 属性）形变动画，duration 0.2s。
- 太阳光芒射线（8 条）使用 `Variants` 的交错动画（staggerChildren 0.03s），在暗色模式时反向消失（staggerDirection -1）。

### 如何自定义动画行为？

如需修改动画参数（duration、staggerChildren 等），可 fork 组件源码并调整 `rayVariant` 和 `raysVariants` 中的 transition 配置。
