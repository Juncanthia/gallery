ScrollToTop 返回页面顶部组件，在用户向下滚动超过阈值后显示浮动按钮，点击可平滑回到页面顶部。

<h2 id="when-to-use">何时使用</h2>

当页面内容较长，用户需要快速返回顶部时使用。常见于长文章、产品列表、搜索结果等页面。

- 长文章或文档页面。
- 产品列表或瀑布流页面。
- 大量内容的仪表盘页面。
- 搜索结果页面。

ScrollToTop 使用固定定位浮于页面右下角，默认在滚动超过 300px 后出现，点击后以 `smooth` 方式平滑返回顶部。提供多种视觉变体以适应不同设计风格。

<h2 id="faq">常见问题</h2>

### 为什么采用 JavaScript 而非纯 CSS 方案？

ScrollToTop 需要在按钮上绑定点击事件以触发 `window.scrollTo()`，因此必须使用 JavaScript。滚动监听通过 `passive` 事件优化性能，不会阻塞主线程。

### 如何仅导入某个变体？

默认变体 `ScrollToTop` 从 `@/components/navigation/gooseui/scroll-to-top` 导入。其他变体（Pill、Minimal、Progress、Text）需从源码路径 `@/components/navigation/gooseui/scroll-to-top` 导入。

### 按钮遮挡了页面关键内容怎么办？

通过 `className` 属性传入自定义类名调整位置。

```tsx
<ScrollToTop className="bottom-20 right-4" />
```

### 如何让按钮只在特定页面显示？

在父组件中按条件渲染即可。

```tsx
{showScrollButton && <ScrollToTop />}
```

### 点击动画异常怎么办？

按钮点击时有内置的涟漪和弹跳动画，持续时间 500-600ms。如需禁用，可通过 `className` 覆盖动画相关样式。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 ScrollToTop 代码时：

1. 默认变体从 `@/components/navigation/gooseui/scroll-to-top` 导入 `ScrollToTop`。
2. 默认无需任何属性，直接 `<ScrollToTop />` 即可在页面右下角显示圆形返回顶部按钮。
3. `size` 属性支持 `sm`（40px）、`md`（48px）、`lg`（56px）三种尺寸，默认 `md`。
4. `threshold` 调整按钮出现阈值，单位像素，默认 300。
5. `behavior` 控制滚动动画：`smooth`（默认）、`instant`、`auto`。
6. Pill、Minimal、Progress、Text 变体从 `@/components/navigation/gooseui/scroll-to-top` 导入。
7. `ScrollToTopPill` 的 `label` 属性用于自定义文字，默认值为 `Scroll to top`，建议中文页面改为 `回到顶部`。
8. `ScrollToTopProgress` 的 `progressColor` 和 `progressBgColor` 接收 CSS 颜色值（HEX、RGB 等），不接收 Tailwind 类名。
9. 组件使用 `fixed` 定位 + `z-50`，如果页面有其他固定元素需注意层级冲突。
10. 不要用 `window.scrollTo` + `requestAnimationFrame` 自己实现，组件内部已处理。
11. 不要引入任何第三方"返回顶部"库。
