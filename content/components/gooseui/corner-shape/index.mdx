CornerShape 圆角塑形组件，基于 CSS `corner-shape: squircle` 原生语法与 SVG clip-path 回退方案，将矩形元素切割为超椭圆（superellipse）圆角形状，呈现 iOS 风格的连续平滑曲线。

<h2 id="when-to-use">何时使用</h2>

当需要在界面中呈现连续曲线的圆角形状时使用，例如图片裁剪、卡片形状、头像框、按钮样式等。

常见场景：

- 图片以 squircle 形状展示，替代传统的 `border-radius` 圆角。
- 卡片、面板需要更柔和的圆角曲线。
- 头像框需要与 Apple 系统风格一致。
- 按钮、徽章需要区别于常规圆角的视觉强调。

CornerShape 在 Chrome 139+ 中使用原生 CSS `corner-shape: squircle`，无需 JavaScript 计算。其他浏览器自动降级为 SVG clip-path，视觉差异极小。

超椭圆参数 `n` 控制圆角程度：`n=2` 为椭圆，`n=5` 为 Apple squircle 标准，`n` 越大越接近方形。

### Hook 用法

`useSuperellipsePath` 生成 SVG 路径字符串，`useCornerShapeSupport` 检测浏览器是否支持 CSS `corner-shape`。

```tsx
import { useSuperellipsePath, useCornerShapeSupport } from "@/components/effects/interactions/gooseui/corner-shape"

export default function Demo() {
  const path = useSuperellipsePath(5, 120)
  const supported = useCornerShapeSupport()

  return (
    <div>
      <svg viewBox="0 0 120 120" width="120" height="120">
        <path d={path} fill="url(#grad)" />
        <defs>
          <linearGradient id="grad" x1="0%" y1="0%" x2="100%" y2="100%">
            <stop offset="0%" stopColor="#3B82F6" />
            <stop offset="100%" stopColor="#8B5CF6" />
          </linearGradient>
        </defs>
      </svg>
      <p className="text-xs text-muted-foreground mt-2">
        浏览器 corner-shape 支持: {supported ? "是" : "否（使用 SVG 降级）"}
      </p>
    </div>
  )
}
```

<h2 id="api">API</h2>

### Squircle

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| n | 超椭圆指数。2=椭圆，5=squircle，更大值更接近方形 | `number` | `5` |
| radius | 圆角比例，CSS 值如 "20%" 或 "24px" | `string` | `"20%"` |
| className | 附加样式类名 | `string` | - |
| children | 子元素内容 | `React.ReactNode` | - |

Squircle 接受所有标准 HTML `div` 元素属性。

语义属性：`data-squircle`、`data-native`、`data-n`。

### SquircleImage

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| n | 超椭圆指数 | `number` | `5` |
| radius | 圆角比例 | `string` | `"20%"` |
| containerClassName | 容器附加样式 | `string` | - |
| className | img 元素附加样式 | `string` | - |

SquircleImage 接受所有标准 HTML `<img>` 元素属性。

### SquircleComparison

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |
| children | 子元素内容 | `React.ReactNode` | - |

### useSuperellipsePath

```ts
(n: number = 5, size: number = 100): string
```

生成指定尺寸的超椭圆 SVG path 数据。

### useCornerShapeSupport

```ts
(): boolean
```

检测当前浏览器是否支持 CSS `corner-shape` 属性。返回 `true` 表示可使用原生 squircle 渲染。

<h2 id="faq">常见问题</h2>

### 什么浏览器支持原生 corner-shape？

CSS `corner-shape: squircle` 目前仅在 Chrome 139+（Chromium 内核）中原生支持。Firefox 和 Safari 会自动降级为 SVG clip-path 方案，视觉效果几乎无差异。

可通过浏览器地址栏访问 `chrome://flags/#enable-experimental-web-platform-features` 在更早版本中启用实验性支持。

### 降级方案的性能如何？

SVG clip-path 方案每创建一个 `Squircle` 实例都会渲染一个零尺寸 `<svg>` 元素及其 `<clipPath>`。对于页面中存在少量 squircle（&lt; 50 个）的场景，性能开销可忽略。若页面存在上百个 squircle 元素，建议使用 `useCornerShapeSupport` 检测后在业务层做数量控制。

### n 参数如何影响形状？

- `n=4`：产生较柔和的圆角过渡，介于标准圆角和 squircle 之间。
- `n=5`：Apple 标准 squircle，连续曲率，视觉最平滑。
- `n=6,8,...`：圆角逐渐向标准 `border-radius` 靠拢，更大的 n 值更接近方形圆角。

### radius 参数如何理解？

`radius` 在原生模式下直接设置 `border-radius`，决定圆角比例。在 SVG 降级模式下不使用（SVG clip-path 本身决定了形状）。

```tsx
<Squircle radius="20%">...</Squircle>   // 圆角占尺寸 20%
<Squircle radius="24px">...</Squircle> // 圆角 24px
```

### 如何自定义 Squircle 的样式？

通过 `className` 设置背景色、尺寸等样式。与普通 HTML 元素一样使用 Tailwind CSS。

```tsx
<Squircle
  className="w-32 h-32 bg-gradient-to-br from-pink-500 to-orange-500 shadow-lg"
/>
```

### Squircle 和 SquircleImage 有什么区别？

`SquircleImage` 是 `Squircle` 的封装，内部自动放置 `<img>` 并设置 `object-cover`，专为图片裁剪场景简化 API。如需更复杂的子元素结构（如叠加文字、遮罩），使用 `Squircle`。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 CornerShape 代码时：

1. 从 `@/components/effects/interactions/gooseui/corner-shape` 导入 `Squircle`、`SquircleImage`、`SquircleComparison`。
2. 使用 `n` 控制超椭圆曲率，默认使用 `n=5`（Apple 标准 squircle）。
3. 使用 `radius` 控制圆角比例，默认 `"20%"`。
4. 图片裁剪优先使用 `SquircleImage`，复杂子元素使用 `Squircle`。
5. 浏览器兼容性无需手动处理，组件内部已完成 CSS 原生 / SVG 自动降级。
6. `useSuperellipsePath` 用于需要原始 SVG 路径的场景，`useCornerShapeSupport` 用于需要知道当前降级状态的场景。
7. 不要引入 `border-radius` + `overflow: hidden` 模拟 squircle，应直接使用本组件。
8. 不要为 squircle 效果手动编写 SVG clip-path，应使用本组件或 hook。
