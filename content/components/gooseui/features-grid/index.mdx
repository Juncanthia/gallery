FeaturesGrid 是一个响应式特性展示网格块组件，使用图标 + 标题 + 描述的卡片布局，适用于落地页、产品介绍或功能亮点展示区域。

<h2 id="when-to-use">何时使用</h2>

当需要在页面中以网格形式展示产品特性、服务亮点或功能列表时使用。适合用于首页、定价页、产品介绍页等营销场景。

常见场景：

- 落地页的功能亮点展示区域。
- SaaS 产品特性介绍。
- 服务列表或优势说明。

不推荐：

- 内容需要复杂交互或动态加载的场景（静态展示组件）。
- 需要嵌套子层级的特性分类（平铺展示，无层级）。
- 单列或列表形式更适合内容密集型展示时。

FeaturesGrid 接收可选的 title、subtitle 和 features 数组，不传入则使用内置示例数据。每个 feature 项包含 icon（Lucide 图标组件）、title 和 description。

### 基础用法

默认展示内置的 6 个示例特性卡片，无需传递任何 props。

### 自定义标题和副标题

通过 `title` 和 `subtitle` 自定义区块标题和描述文案。

```tsx
import { FeaturesGrid } from "@/components/ui/gooseui-features-grid"

export default function Demo() {
  return (
    <FeaturesGrid
      title="为什么选择我们"
      subtitle="专业的开发工具，助你高效构建现代化产品"
    />
  )
}
```

### 自定义特性列表

通过 `features` 传入自定义的特性数据，包含图标、标题和描述。

```tsx
import { Sparkles, Shield, Zap } from "lucide-react"
import { FeaturesGrid } from "@/components/ui/gooseui-features-grid"

export default function Demo() {
  return (
    <FeaturesGrid
      title="核心能力"
      subtitle="为开发者打造的现代化组件库"
      features={[
        {
          icon: Sparkles,
          title: "精美动画",
          description: "内置多种动画效果，让界面更生动",
        },
        {
          icon: Shield,
          title: "类型安全",
          description: "完整的 TypeScript 类型定义",
        },
        {
          icon: Zap,
          title: "极致性能",
          description: "轻量级实现，零冗余依赖",
        },
      ]}
    />
  )
}
```

<h2 id="api">API</h2>

### FeaturesGrid

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| title | 区块主标题 | `string` | `"Everything for Modern Development"` |
| subtitle | 区块副标题/描述 | `string` | `"Components that save time and help you build quality products"` |
| features | 特性数据数组 | `Feature[]` | 内置 6 项示例数据 |
| className | 附加 CSS 类名 | `string` | - |

### Feature

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| icon | 图标组件（Lucide 图标） | `React.ComponentType<{ className?: string }>` | - |
| title | 特性标题 | `string` | - |
| description | 特性描述 | `string` | - |

FeaturesGrid 内部使用 CSS Grid 响应式布局，移动端单列、平板双列、桌面三列。每个 FeatureCard 带有 hover 状态 (`hover:bg-muted/50`)，图标区域使用 `bg-primary/10` 浅色背景配合 `text-primary` 主色图标。

<h2 id="faq">常见问题</h2>

### 如何自定义单个卡片的样式？

通过 `className` 传入额外的 CSS 类名可覆盖区块整体样式。如需修改单个卡片样式，需直接修改组件源码（组件设计为"复制到项目中自由修改"模式）。

### 图标可以使用其他图标库吗？

`icon` 属性类型为 `React.ComponentType<{ className?: string }>`，任何接受 `className` 的 React 图标组件均可使用，如 `react-icons`、`@tabler/icons-react` 等。

### 支持多少列布局？

默认使用 `grid gap-6 sm:grid-cols-2 lg:grid-cols-3`：移动端 1 列，平板（sm）2 列，桌面（lg）3 列。feature 数量不限，自动换行。

### 可以不显示标题区域吗？

传入空字符串 `title=""` 和 `subtitle=""` 可使标题区域内容为空，但标题容器仍会渲染（保留 `text-center` 区域的高度影响）。如需完全移除，修改组件源码或通过 `className` 隐藏。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 FeaturesGrid 代码时：

1. 从 `@/components/ui/gooseui-features-grid` 导入 `FeaturesGrid`。
2. `features` 数组中的 `icon` 必须是 React 组件（如 `import { Sparkles } from "lucide-react"`），不能是字符串或 URL。
3. 不传入任何 props 时展示内置的 6 项默认特性。
4. 每个 feature 的 `title` 和 `description` 建议简洁，适应卡片内有限空间。
5. 组件为营销/展示型区块，不应在特性卡内嵌入表单或交互控件。
6. 不要引入 Ant Design 或 `antd` 相关依赖。
