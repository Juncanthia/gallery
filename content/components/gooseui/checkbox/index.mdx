Checkbox 多选框，基于 SVG 动画实现选中/未选中状态的平滑过渡。

<h2 id="when-to-use">何时使用</h2>

在一组可选项中进行多项选择。

常见场景：

- 表单中的多项选择，如协议勾选、功能开关。
- 列表项多选，如批量操作。
- 设置项启用/禁用。

Checkbox 使用 SVG path 的 stroke-dasharray 和 stroke-dashoffset 动画实现选中的勾选形态变换，无需外部图标库。

### 基础用法

Checkbox 支持默认、已选、禁用等基础状态。通过 `label` 设置文字标签。

```tsx

export default function Demo() {
```

### 语义变体

Checkbox 支持 `default`、`destructive`、`success` 三种颜色变体。

```tsx

      <Checkbox label="危险变体" variant="destructive" defaultChecked />
      <Checkbox label="成功变体" variant="success" defaultChecked />

```

### 尺寸

Checkbox 支持 `sm`、`default`、`lg` 三种尺寸，默认为 `default`。

### 受控模式

Checkbox 支持受控和非受控两种使用方式。受控模式通过 `checked` + `onChange` 管理状态。

```tsx
import { useState } from "react"
import { Checkbox } from "@/components/ui/checkbox"

export default function Demo() {
  const [checked, setChecked] = useState(false)

  return (
    <div className="flex flex-col gap-3">
      <Checkbox
        label={`受控复选框：${checked ? "已选中" : "未选中"}`}
        checked={checked}
        onChange={(e) => setChecked(e.target.checked)}
      />
      <Checkbox label="非受控复选框" defaultChecked />
    </div>
  )
}
```

### 分组用法

多个 Checkbox 组合使用以支持多选场景。

```tsx
import { Checkbox } from "@/components/ui/checkbox"

export default function Demo() {

      <Checkbox label="选项 C" />
      <Checkbox label="选项 D（禁用）" disabled />

```

<h2 id="api">API</h2>

### Checkbox

通过设置 Checkbox 的属性产生不同的样式和交互行为。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 颜色变体 | `default` \| `destructive` \| `success` | `default` |
| size | 复选框尺寸 | `sm` \| `default` \| `lg` | `default` |
| label | 文字标签；不传则为纯复选框 | `string` | - |
| id | 输入元素 id；未传时自动生成 | `string` | auto-generated |
| className | 附加样式类名，作用于 `<label>` | `string` | - |

Checkbox 继承原生 `<input type="checkbox">` 的所有属性，包括 `checked`、`defaultChecked`、`disabled`、`onChange`、`required`、`name`、`value` 等。

<h2 id="faq">常见问题</h2>

### 如何获取复选框的选中状态？

非受控模式使用 `defaultChecked` 设置初始值。受控模式使用 `checked` + `onChange`：

```tsx
const [checked, setChecked] = useState(false)
<Checkbox checked={checked} onChange={(e) => setChecked(e.target.checked)} />
```

### 如何实现全选/反选功能？

通过受控模式管理一组 Checkbox 的状态，结合 `checked` 和 `onChange` 实现。

```tsx
const [selected, setSelected] = useState<string[]>([])
function toggle(value: string) {
  setSelected((prev) =>
    prev.includes(value) ? prev.filter((v) => v !== value) : [...prev, value]
  )
}
```

### 如何自定义复选框样式？

通过 `variant` 和 `size` 控制预设样式变体。如需深度定制，可通过 Tailwind CSS 变量覆盖：

- `--checkbox-checked`：选中时勾选线条颜色
- `--checkbox-unchecked`：未选中时边框颜色
- `--checkbox-size`：复选框尺寸

通过 `className` 传入自定义 CSS 变量即可：

```tsx
<Checkbox
  className="[--checkbox-checked:theme(colors.blue.500)] [--checkbox-size:1.25rem]"
  label="自定义颜色"
/>
```

### 为什么复选框使用 SVG 而不是 CSS border？

SVG path 动画实现了选中时从边框到勾选形态的平滑过渡，这是纯 CSS border + checkbox 伪元素无法实现的。该实现无需引入外部图标库，纯通过 stroke-dasharray / stroke-dashoffset 控制动画。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Checkbox 代码时：

1. 从 `@/components/ui/checkbox` 导入 Checkbox。
2. 使用 `label` 设置标签文字；不需要 label 时可省略。
3. 颜色语义通过 `variant` 控制：`default`、`destructive`、`success`。
4. 尺寸通过 `size` 控制：`sm`、`default`、`lg`。
5. 使用原生 input 属性：`checked`、`defaultChecked`、`disabled`、`onChange`、`required`。
6. Checkbox 是纯展示+交互组件，不包含表单验证逻辑；表单验证应在父级 Form 组件中处理。
7. 不要为勾选图标引入任何外部图标库（lucide-react 等）；勾选由 SVG 动画实现。
8. 样式定制优先使用 CSS 变量覆盖，而非在调用处堆叠 className。
9. 不要引入 Ant Design Checkbox 或 rc-checkbox。
