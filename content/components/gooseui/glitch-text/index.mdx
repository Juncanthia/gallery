GlitchText 为文本添加故障抖动效果。支持色差偏移、扫描线和辉光，通过 intensity 控制故障强度。

<h2 id="when-to-use">何时使用</h2>

当需要为标题、标语或关键文本赋予故障艺术风格时使用，适用于赛博朋克、科技感、黑客等主题场景。

常见场景：

- 首页 Hero 区域的标题动画。
- 黑客松、科技活动页面的标语。
- 游戏化界面的特效文本。
- 品牌标识的视觉强化。

不推荐：

- 正文或长文本内容，故障效果会严重影响可读性。
- 对无障碍要求严格的场景，闪烁动画可能引发不适。
- 表单标签、按钮文案等功能性文本。

<h2 id="examples">代码演示</h2>

### 基础用法

GlitchText 接收 `text` 属性渲染故障效果文本，默认 `intensity="medium"`，开启扫描线和辉光。

<h2 id="api">API</h2>

### GlitchText

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| text | 要渲染的文本内容 | `string` | - |
| intensity | 故障效果强度 | `"low"` \| `"medium"` \| `"high"` | `"medium"` |
| showScanlines | 是否显示扫描线叠加层 | `boolean` | `true` |
| showGlow | 是否显示文字背后辉光效果 | `boolean` | `true` |
| color | 自定义文字颜色（CSS 颜色值） | `string` | `"currentColor"` |
| className | 附加 CSS 类名 | `string` | - |

GlitchText 使用 CSS clip-path 动画和 mix-blend-mode 实现故障效果。`intensity` 控制故障偏移量、倾斜角度和动画周期：`low` 偏移 2px/倾斜 5deg/8s，`medium` 偏移 3px/倾斜 10deg/5s，`high` 偏移 5px/倾斜 20deg/3s。通过 `useId()` 生成唯一标识符，确保多个实例的 keyframe 动画互不冲突。

<h2 id="faq">常见问题</h2>

### 如何调整故障效果的强度？

通过 `intensity` 属性设置，可选 `low`、`medium`、`high` 三档，分别对应不同的偏移量、倾斜角度和动画速度组合。

```tsx
<GlitchText text="HELLO" intensity="low" />
<GlitchText text="WORLD" intensity="medium" />
<GlitchText text="CYBER" intensity="high" />
```

### 如何去除扫描线或辉光效果？

将 `showScanlines` 或 `showGlow` 设为 `false` 即可关闭对应效果。

```tsx
<GlitchText text="CLEAN" showScanlines={false} showGlow={false} />
```

### 如何在暗色背景上使用？

GlitchText 默认使用 `currentColor`，继承父元素的文字颜色。在暗色背景上可直接传入亮色 `color` 属性，或通过 Tailwind class 设置父容器的文字颜色。

```tsx
<div className="text-cyan-400">
  <GlitchText text="DARK MODE" />
</div>
```

### 多个实例会冲突吗？

不会。每个实例通过 React `useId()` 生成唯一的 keyframe 名称，确保 CSS 动画互不影响。

### 如何自定义文字颜色？

通过 `color` 属性传入任意 CSS 颜色值（hex、rgb、hsl 等），色差层的 cyan/magenta 偏移色不受此属性影响。

```tsx
<GlitchText text="NEON" color="#00ff88" />
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/glitch-text-gooseui` 导入 `GlitchText`。
2. 必须传入 `text` 属性，组件依赖此属性渲染内容。
3. `intensity` 使用 `"low"`、`"medium"`、`"high"` 三档字符串，不要传入数字。
4. `color` 传入 CSS 颜色字符串（如 `"#00ff88"`），非 Tailwind class。
5. 不要为正文或长文本使用 GlitchText。
6. 不要引入 Ant Design 相关依赖。
