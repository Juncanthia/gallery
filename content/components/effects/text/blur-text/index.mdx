BlurText 基于 Motion 的模糊渐入文字动画组件，支持逐字或逐词显示，可自定义关键帧。

<h2 id="when-to-use">何时使用</h2>

当需要文字从模糊状态渐入显示，营造富有层次感的文本入场动画时使用。

常见场景：
- Hero 标题、Slogan 的逐词渐入展示
- 段落文本的滚动触发入场效果
- 需要多关键帧过渡的复杂文字动画（模糊 -> 半模糊 -> 清晰）

不推荐：
- 需要即读即显的信息型文本（动画有延迟）
- 极长文本（每个 word/letter 都是独立的 motion.span，DOM 节点数较多）

<h2 id="examples">代码演示</h2>

### 基础用法

<ComponentShowcase id="rb/blur-text" path="rb/blur-text/basic" />

```tsx
import { BlurText } from "@/components/core/blur-text"

export default function Demo() {
  return (
    <BlurText
      text="Hello World, this is a blur text animation demo."
      className="text-4xl font-bold text-white"
    />
  )
}
```

### 逐字母入场与自定义关键帧

```tsx
import { BlurText } from "@/components/core/blur-text"

export default function CustomDemo() {
  return (
    <BlurText
      text="逐字母渐入效果"
      animateBy="letters"
      direction="bottom"
      delay={100}
      stepDuration={0.5}
      animationFrom={{ filter: "blur(20px)", opacity: 0, y: -100 }}
      animationTo={[
        { filter: "blur(10px)", opacity: 0.3, y: -30 },
        { filter: "blur(0px)", opacity: 1, y: 0 },
      ]}
      easing={(t) => t * (2 - t)}
    />
  )
}
```

<h2 id="api">API</h2>

### BlurText

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| text | 要显示的文本 | `string` | `""` |
| delay | 每个单元之间的延迟（ms） | `number` | `200` |
| className | 根元素的自定义类名 | `string` | `""` |
| animateBy | 动画粒度 | `"words" \| "letters"` | `"words"` |
| direction | 动画起始方向 | `"top" \| "bottom"` | `"top"` |
| threshold | IntersectionObserver 可见阈值 | `number` | `0.1` |
| rootMargin | IntersectionObserver 边距 | `string` | `"0px"` |
| animationFrom | 自定义起始状态 | `{ filter: string; opacity: number; y: number }` | 自动根据 direction 生成 |
| animationTo | 自定义结束态关键帧数组 | `Array<{ filter: string; opacity: number; y: number }>` | 自动生成两帧过渡 |
| easing | 关键帧缓动函数 | `(t: number) => number` | `(t) => t`（线性） |
| onAnimationComplete | 最后一个单元动画完成回调 | `() => void` | - |
| stepDuration | 每个步骤的持续时间（秒） | `number` | `0.35` |

<h2 id="faq">常见问题</h2>

### animationFrom 和 animationTo 如何配合？

`animationFrom` 定义起始状态，`animationTo` 定义中间关键帧数组。总步骤数 = `animationTo.length + 1`，关键帧时间均匀分布。例如 2 个 to 帧 + 1 个起始 = 3 步，时间节点为 [0, 0.5, 1]。

### 为什么最后一个单元才触发 onAnimationComplete？

因为每个 span 独立进行动画，只有到最后一个元素的动画完成回调才调用 `onAnimationComplete`，确保整个文本动画已全部结束。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/blur-text` 导入。
2. 依赖 `motion`（framer-motion）库。
3. `text` 为空字符串时不渲染任何内容。
4. 自动基于 IntersectionObserver 触发，无需手动控制可见性。
5. 每个 word/letter 为独立的 `motion.span`，会生成 `will-change-[transform,filter,opacity]` 提示。
