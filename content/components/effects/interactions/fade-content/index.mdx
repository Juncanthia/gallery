FadeContent 基于 IntersectionObserver 的淡入显示组件，支持模糊过渡和自动消失动画。

<h2 id="when-to-use">何时使用</h2>
当需要页面滚动到元素时触发淡入动画时使用。
常见场景：
- 长页面的逐段内容入场动画
- 需要从模糊到清晰的视觉过渡
- 内容出现一段时间后自动淡出的提示/通知
不推荐：
- 需要重复触发入场动画的场景（observer 触发一次后即取消观察）
- 对 `opacity: 0` 元素有 SEO 要求的场景

<h2 id="examples">代码演示</h2>
### 基础用法

<ComponentShowcase id="rb/fade-content" path="rb/fade-content/basic" />

```tsx
import { FadeContent } from "@/components/core/fade-content"
export default function Demo() {
  return (
    <FadeContent duration={1000}>
      <div className="rounded-lg bg-neutral-800 p-8 text-white">Content</div>
    </FadeContent>
  )
}
```

### 模糊淡入 + 自动消失

```tsx
import { FadeContent } from "@/components/core/fade-content"

export default function Demo() {
  return (
    <FadeContent
      blur
      duration={1200}
      delay={200}
      disappearAfter={3000}
      disappearDuration={800}
      onComplete={() => console.log("出现完毕")}
      onDisappearanceComplete={() => console.log("消失完毕")}
    >
      <div className="rounded bg-blue-900 p-6 text-white">Fade in with blur, then disappear</div>
    </FadeContent>
  )
}
```

<h2 id="api">API</h2>
### FadeContent
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 需要淡入的内容节点 | `React.ReactNode` | - |
| container | 指定滚动的容器元素或选择器 | `string \| HTMLElement \| null` | `null` |
| blur | 是否启用模糊过渡（从 blur(10px) 到 blur(0)） | `boolean` | `false` |
| duration | 淡入动画持续时间（ms，>10 时当作 ms，<=10 时当作秒） | `number` | `1000` |
| ease | CSS 缓动函数名称 | `string` | `"power2.out"` |
| delay | 动画延迟时间（单位规则同 duration） | `number` | `0` |
| threshold | IntersectionObserver 可见阈值 | `number` | `0.1` |
| initialOpacity | 元素初始透明度 | `number` | `0` |
| disappearAfter | 出现后多少毫秒开始消失，0 表示不自动消失 | `number` | `0` |
| disappearDuration | 消失动画持续时间（单位规则同 duration） | `number` | `0.5` |
| disappearEase | 消失动画的缓动函数 | `string` | `"power2.in"` |
| onComplete | 出现动画完成后的回调 | `() => void` | - |
| onDisappearanceComplete | 消失动画完成后的回调 | `() => void` | - |
| className | 包裹元素的附加样式类名 | `string` | `""` |
| style | 包裹元素的内联样式 | `React.CSSProperties` | - |

<h2 id="faq">常见问题</h2>
### duration/disappearDuration 的单位是秒还是毫秒？
组件内部会判断：值 > 10 时当作毫秒处理，<= 10 时当作秒处理。例如 `duration={1000}` 等价于 `duration={1}`，都是 1 秒。

### IntersectionObserver 触发后还会再次触发吗？
不会。一旦元素进入可视区域并启动动画后，observer 会立即取消观察该元素。如需重复触发，需要在组件卸载后重新挂载。

### 自定义 ease 需要什么格式？
直接传入 CSS transition-timing-function 的合法值即可，如 `"ease"`, `"ease-in-out"`, `"cubic-bezier(0.4, 0, 0.2, 1)"`。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/core/fade-content` 导入。
2. 无额外第三方依赖，仅使用原生 IntersectionObserver API 和 CSS transition。
3. `blur` 和 `disappearAfter` 可以组合使用，实现 模糊入场 -> 内容展示 -> 模糊消失 的完整动画序列。
4. `container` 属性用于指定滚动容器，默认为 null 时使用 viewport 作为 root。
