StarBorder 星光边框组件，带有渐变动画边框的容器，产生旋转星光边框效果。

<h2 id="when-to-use">何时使用</h2>
当需要...时使用。
常见场景：
- 按钮或卡片的动态渐变星光边框装饰
- 需要吸引用户注意的 CTA 区域
- 暗色背景下的高亮边框效果
不推荐：
- 频繁闪烁可能引起不适的场合（可通过 `speed` 调慢）
- 纯白背景（星光效果不明显）

<h2 id="api">API</h2>
### StarBorder
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| as | 渲染的 HTML 元素类型 | `React.ElementType` | `"button"` |
| className | 外层容器自定义类名 | `string` | - |
| color | 星光渐变颜色 | `string` | `"white"` |
| speed | 星光旋转动画速度（CSS 时间值） | `string` | `"6s"` |
| thickness | 边框厚度，控制内边距 | `number` | `1` |
| children | 内容（文本或其他元素） | `ReactNode` | - |

StarBorder 泛型支持 `as` 属性的具体类型推导，例如 `StarBorder<"a">` 可接受 `<a>` 的合法属性。

<h2 id="faq">常见问题</h2>
### 如何改变星光动画速度？
通过 `speed` 属性传入 CSS 时间值，如 `"4s"`（更快）或 `"10s"`（更慢）。

### 能否渲染为链接？
可以。设置 `as="a"` 并传入 `href` 属性，组件会渲染为 `<a>` 标签并继承所有对应 HTML 属性。

### 星光效果在亮色背景下不明显怎么办？
星光效果使用 `radial-gradient` 实现，在暗色或黑色 (`#222` 边框) 背景下效果最佳。亮色背景建议使用较深的 `color` 值。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/core/star-border` 导入。
2. 默认为 `button` 元素，可通过 `as` 属性修改为其他标签。
3. 内容通过 `children` 传入，内层自动包裹在深色背景容器中。
4. 无需额外依赖，仅使用 CSS 动画和内联样式。
