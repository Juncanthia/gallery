OrbitImages 轨道图片组件，图片沿指定形状（椭圆、圆形、星形等）的轨道持续运动，支持自定义路径。

<h2 id="when-to-use">何时使用</h2>
当需要展示图片沿预定轨道旋转运动时使用。
常见场景：
- Logo 墙和品牌展示
- 技能/技术栈图标轮播
- Hero 区域动态装饰
不推荐：
- 图片数量过多（超过 20 张会显得拥挤）
- 需要精确控制每张图片位置的场景

<h2 id="api">API</h2>

### OrbitImages

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| images | 图片源数组 | string[] | [] |
| altPrefix | 图片 alt 前缀 | string | `"Orbiting image"` |
| shape | 轨道形状 | "ellipse" \| "circle" \| "square" \| "rectangle" \| "triangle" \| "star" \| "heart" \| "infinity" \| "wave" \| "custom" | "ellipse" |
| customPath | 自定义 SVG 路径（shape 为 custom 时生效） | string | - |
| baseWidth | 设计稿基础宽度（响应式缩放基准） | number | 1400 |
| radiusX | X 轴半径（椭圆、矩形、无限、波浪） | number | 700 |
| radiusY | Y 轴半径（椭圆、矩形、无限、波浪） | number | 170 |
| radius | 圆形/星形/三角形半径 | number | 300 |
| starPoints | 星形顶点数 | number | 5 |
| starInnerRatio | 星形内径与外径之比 | number | 0.5 |
| rotation | 轨道整体旋转角度（度） | number | -8 |
| duration | 完整一圈的时长（秒） | number | 40 |
| itemSize | 图片尺寸（px） | number | 64 |
| direction | 运动方向 | "normal" \| "reverse" | "normal" |
| fill | 是否沿轨道均匀分布 | boolean | true |
| width | 容器宽度 | number | 100 |
| height | 容器高度 | number | 100 |
| className | 额外的 CSS 类名 | string | "" |
| showPath | 是否显示轨道路径 | boolean | false |
| pathColor | 路径的颜色 | string | `"rgba(0,0,0,0.1)"` |
| pathWidth | 路径线条宽度 | number | 2 |
| easing | 动画缓动函数名 | string | "linear" |
| paused | 是否暂停动画 | boolean | false |
| centerContent | 轨道中心的 React 节点 | React.ReactNode | - |
| responsive | 是否启用响应式缩放 | boolean | false |

<h2 id="faq">常见问题</h2>

### 如何让图片在圆形轨道上运动？
设置 `shape="circle"` 并调整 `radius` 控制圆的大小。

### 图片不显示怎么办？
检查 `images` 数组中的路径是否正确。图片使用 `img` 标签渲染，确保路径相对于 public 目录。

### 响应式模式下图片位置偏移？
使用 `responsive={true}` 时，组件会根据容器宽度自动缩放。确保 `baseWidth` 与设计稿的实际宽度匹配。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/core/orbit-images` 导入。
2. 传入 `images` 字符串数组作为轨道上的图片源。
3. 该组件依赖 `motion`（motion/react），确保项目已安装。
4. 需要 `centerContent` 时直接传入 React 节点即可。
