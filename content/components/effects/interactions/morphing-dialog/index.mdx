MorphingDialog 基于 View Transitions API 实现元素变形过渡的对话框组件。点击触发元素时，源元素平滑变形为对话框；关闭对话框时，对话框反向收缩回源元素位置。适用于图片放大预览、卡片内容展开等场景。

<h2 id="when-to-use">何时使用</h2>

需要在页面内将一个元素平滑变形展开为更大的容器（对话框）时使用。

常见场景：

- 图片列表点击放大预览，保持视觉连续性。
- 卡片内容从摘要展开为详情，无需页面跳转。
- 仪表盘数据项从缩略形式展开为完整信息面板。
- 画廊或产品展示中，缩略图到详情的无缝过渡。

MorphingDialog 使用浏览器原生 View Transitions API 实现变形动画，通过 `viewTransitionName` 在源元素和对话框之间建立动画关联。无需 JavaScript 动画库，无需 `@radix-ui/react-dialog` 等第三方对话框库。

<h2 id="api">API</h2>

### MorphingDialog

根容器组件，通过 Context 管理对话框的开关状态、唯一标识符和变形动画。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 子组件内容 | `React.ReactNode` | - |

MorphingDialog 使用 `useId()` 生成唯一 `viewTransitionName`，确保页面内多个实例的动画互不冲突。`openDialog(triggerEl)` 方法通过 `document.startViewTransition()` 触发源元素到对话框的变形；`closeDialog()` 反向从对话框变形回源元素。

### MorphingDialogTrigger

触发变形对话框的源元素容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 触发元素内容 | `React.ReactNode` | - |
| className | 附加样式类名 | `string` | - |
| style | 内联样式 | `React.CSSProperties` | - |

触发元素在展开动画后自动隐藏（`[data-expanded=true]:invisible`），关闭动画结束后恢复显示。

### MorphingDialogContainer

原生 `<dialog>` 元素的包装容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 对话框内容 | `React.ReactNode` | - |

使用 `dialog.showModal()` 和 `dialog.close()` 管理对话框生命周期。支持 Escape 键关闭和点击遮罩层关闭。

### MorphingDialogContent

对话框内容区域。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 内容 | `React.ReactNode` | - |
| className | 附加样式类名 | `string` | - |
| style | 内联样式 | `React.CSSProperties` | - |

挂载时自动聚焦，确保键盘导航可用。

### MorphingDialogImage

对话框内的图片展示组件。

| --- | --- | --- | --- |
| src | 图片地址 | `string` | - |
| alt | 图片描述 | `string` | - |
| className | 附加样式类名 | `string` | - |
| style | 内联样式 | `React.CSSProperties` | - |

### MorphingDialogTitle

对话框标题。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 标题内容 | `React.ReactNode` | - |
| className | 附加样式类名 | `string` | - |

### MorphingDialogSubtitle

对话框副标题。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 副标题内容 | `React.ReactNode` | - |
| className | 附加样式类名 | `string` | - |

### MorphingDialogDescription

对话框描述文本区域。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 描述内容 | `React.ReactNode` | - |
| className | 附加样式类名 | `string` | - |

### MorphingDialogClose

关闭按钮。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 自定义关闭按钮内容 | `React.ReactNode` | `<X />` 图标 |
| className | 附加样式类名 | `string` | - |

关闭按钮支持移动端 44x44px 最小触摸区域，通过 `after:absolute after:inset-0 after:-m-2` 扩展不可见触摸目标。

MorphingDialog 基于浏览器原生 View Transitions API 和 `<dialog>` 元素实现，不依赖 `@radix-ui/react-dialog`、`@framer-motion` 或任何第三方动画/对话框库。

<h2 id="faq">常见问题</h2>

### 和 Radix Dialog 或传统对话框有什么区别？

MorphingDialog 使用浏览器原生 View Transitions API 实现源元素到对话框的平滑变形，提供了传统对话框不具备的视觉连续性。同时使用原生 `<dialog>` 元素替代第三方对话框实现，代码更精简。

### 哪些浏览器支持变形效果？

View Transitions API 目前在 Chromium 111+ 可用。在不支持的浏览器中，MorphingDialog 会自动降级为即时显示/隐藏，不会报错。组件内部通过 `prefersReducedMotion()` 和 `document.startViewTransition` 特性检测实现降级。

### 用户在系统中开启了"减少动效"会怎样？

组件内部检查 `prefers-reduced-motion: reduce` 媒体查询，匹配时关闭所有变形动画，对话框直接显示/隐藏。

### 多个 MorphingDialog 实例会冲突吗？

不会。每个实例通过 React `useId()` 生成唯一的 `viewTransitionName`（格式为 `morph-<uniqueId>`），确保 CSS view transition 互不干扰。

### 如何自定义对话框尺寸？

通过 `MorphingDialogContent` 的 `className` 或 `style` 属性覆盖宽高。默认 `max-w-lg`（最大宽度 32rem），高度限制为 `max-h-[calc(100vh-2rem)]`。

### 如何在移动端使用？

MorphingDialog 内置移动端触摸优化：关闭按钮通过 `after` 伪元素扩展 44x44px 触摸目标，对话框最大高度限制为视口高度减 2rem，确保内容不超出屏幕。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 MorphingDialog 代码时：

1. 从 `@/components/effects/interactions/morphing-dialog` 导入所需子组件。
2. 必须按 `MorphingDialog` > `MorphingDialogTrigger` + `MorphingDialogContainer` > `MorphingDialogContent` 的层级结构使用。
3. `MorphingDialogClose` 应放置在 `MorphingDialogContent` 内部，提供关闭交互。
4. 触发元素（children）的内容由调用侧控制，组件不限制触发器的外观。
5. 不要引入 `@radix-ui/react-dialog`、`@framer-motion` 或其他对话框/动画库。
6. 注意 View Transitions API 仅 Chromium 111+ 支持；非 Chromium 环境自动降级为无动画切换。
7. 不要修改组件内部的 `viewTransitionName` 生成逻辑（`morph-<uniqueId>` 格式）。
