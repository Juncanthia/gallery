BounceCards 弹性弹跳卡片展示组件，支持图片卡片的弹性入场动画和悬停推开交互。

<h2 id="when-to-use">何时使用</h2>

当需要展示一组图片卡片，入场时带有弹性弹跳动画，悬停某张卡片时相邻卡片被推开的交互效果时使用。

常见场景：
- 团队成员/作品集展示
- 产品图片画廊
- 需要活泼交互感的卡片列表

不推荐：
- 卡片数量超过 5 张（预设 transformStyles 只有 5 组位置）
- 需要触摸交互的场景（hover 效果在移动端不适用）

### 禁用悬停推开效果

```tsx
import { BounceCards } from "@/components/core/bounce-cards"

export default function CustomDemo() {
  return (
    <BounceCards
      images={["/img1.jpg", "/img2.jpg", "/img3.jpg"]}
      transformStyles={[
        "rotate(10deg) translate(-120px)",
        "rotate(0deg)",
        "rotate(-10deg) translate(120px)",
      ]}
      enableHover={false}
      containerWidth={350}
      containerHeight={350}
    />
  )
}
```

<h2 id="api">API</h2>

### BounceCards

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| images | 图片 URL 数组 | `string[]` | `[]` |
| containerWidth | 容器宽度（px） | `number` | `400` |
| containerHeight | 容器高度（px） | `number` | `400` |
| animationDelay | 入场动画延迟（秒） | `number` | `0.5` |
| animationStagger | 每张卡片的交错延迟（秒） | `number` | `0.06` |
| easeType | GSAP 缓动类型 | `string` | `"elastic.out(1, 0.8)"` |
| transformStyles | 每张卡片的初始 CSS transform 字符串 | `string[]` | 5 种预设旋转+平移 |
| enableHover | 是否启用悬停推开相邻卡片 | `boolean` | `true` |
| className | 容器额外样式类名 | `string` | - |

<h2 id="faq">常见问题</h2>

### 如何让卡片数量超过 5 张？

`transformStyles` 数组控制每张卡片的初始位置，需传入与 `images` 长度一致的 transform 字符串数组。每多一张卡片需增加一个 transform 样式。

### 悬停推开效果的距离是多少？

悬停某张卡片时，左侧卡片向右推 160px，右侧卡片向左推 160px，推开的距离与悬停卡片的索引距离有关，距离越远延迟越大（distance * 0.05s）。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/bounce-cards` 导入。
2. 依赖 `gsap` 库。
3. `transformStyles` 数组长度需与 `images` 长度匹配。
4. 附带样式表 `bounce-cards.css`。
5. 卡片通过绝对定位叠加，入场动画从 scale(0) 弹跳到 scale(1)。
