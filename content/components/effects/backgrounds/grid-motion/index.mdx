GridMotion 是一个基于 GSAP 的网格项鼠标平移交互组件，多行网格内容根据鼠标位置产生不同速度的横向移动。

<h2 id="when-to-use">何时使用</h2>
当需要网格布局项目随鼠标位置平移交互时使用。
常见场景：
- 作品集展示的网格画廊
- 产品列表的交互式浏览
- 着陆页的视觉层次展示
不推荐：
- 需要精确点击交互的网格（平移可能干扰点击定位）
- 内容密集的仪表盘或数据表格
- 需要无障碍访问的场景

```tsx
import { GridMotion } from "@/components/core/grid-motion"

  "Design System",
  "Web App",
  "Mobile",
  "Branding",
  "Illustration",
  "Motion",
]

export default function Demo() {
  return (
    <GridMotion
      items={items}
      gradientColor="#1a1a2e"
    />
  )
}
```

<h2 id="api">API</h2>
### GridMotion
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| items | 网格项数组（支持文本或图片URL） | `string[]` | `[]` |
| gradientColor | 渐变背景色 | `string` | `"black"` |

<h2 id="faq">常见问题</h2>
### 网格项数量如何计算？
组件固定为 4 行 x 7 列 = 28 项。如果 `items` 少于 28 项，会自动用默认文本 `Item 1` 至 `Item 28` 补全。如果 `items` 多于 28 项，只取前 28 项。

### 如何传入图片？
`items` 数组中的字符串如果以 `http` 开头，会被识别为图片 URL 并渲染为背景图；否则渲染为纯文本内容。

### 性能表现如何？
依赖 GSAP 的 blocks/ticker 驱动，所有动画由 GSAP 时间线管理。4 行 DOM 元素的同步平移性能开销较小，但应避免在移动端低端设备上叠加过多其他动画。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/core/grid-motion` 导入。
2. 依赖 `gsap`，确保已安装。
3. `items` 为空数组时自动使用默认占位项（28个）。
4. 以 `http` 开头的字符串渲染为图片背景图，其他为文本内容。
