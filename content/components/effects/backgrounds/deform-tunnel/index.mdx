DeformTunnel 变形隧道。基于 Three.js 和 React Three Fiber 实现的片段着色器隧道效果。通过鼠标 hover 和移动产生局部形变与光效响应，适用于页面背景或沉浸式视觉装饰。

<h2 id="when-to-use">何时使用</h2>

当需要全屏或区域性的隧道形变视觉效果时使用。

常见场景：
- 首页 Hero 区域的动态背景，提升视觉冲击力。
- 作品集或创意页面的沉浸式装饰，配合鼠标交互增强参与感。
- 加载过渡页面的动态背景。

不推荐：
- 低性能设备或移动端大量使用 shader 效果，可能影响流畅度。
- 作为信息承载区域使用，shader 效果会干扰可读性。
- 多个 shader 组件同时渲染，GPU 负载过高。

### 基础用法

默认配置下，DeformTunnel 渲染全屏隧道效果，支持鼠标 hover 交互。

```tsx
import { DeformTunnel } from "@/components/effects/backgrounds/deform-tunnel"

export default function Demo() {
  return (
    <div className="relative w-full h-[400px] rounded overflow-hidden">
      <DeformTunnel />
    </div>
  )
}
```

<h2 id="api">API</h2>

### DeformTunnel

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |
| imageSrc | 隧道纹理图片 URL | `string` | `"https://images.unsplash.com/photo-1508349937151-22b68b72d5b1"` |
| speed | 动画速度倍率 | `number` | `1.0` |
| wobble | 晃动强度 [x, y] | `[number, number]` | `[2.5, 1.2]` |
| depth | 隧道深度 | `number` | `5.0` |
| scale | 纹理缩放 | `number` | `0.3` |
| decay | 光衰减系数 (0.0 ~ 1.0) | `number` | `0.8` |
| exposure | 曝光度 / 整体亮度 | `number` | `8.5` |

<h2 id="faq">常见问题</h2>

### 如何调整隧道速度？

通过 `speed` 属性控制动画速度，默认 `1.0`。增大值加速，减小值减速。

```tsx
<DeformTunnel speed={0.5} />
```

### 如何更换隧道纹理？

通过 `imageSrc` 传入自定义图片 URL。纹理会在隧道中重复平铺。

```tsx
<DeformTunnel imageSrc="https://example.com/my-texture.jpg" />
```

### 如何控制隧道深度和形变强度？

`depth` 控制隧道纵深，`wobble` 控制扭曲幅度 [x, y]，`decay` 控制光线衰减。三者可组合调整视觉效果。

```tsx
<DeformTunnel depth={3.0} wobble={[1.5, 0.8]} decay={0.6} />
```

### 如何在固定区域（非全屏）使用？

外层容器设置相对定位和固定宽高即可，DeformTunnel 默认撑满父容器。

```tsx
<div className="relative w-full h-[400px] rounded overflow-hidden">
  <DeformTunnel />
</div>
```

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 DeformTunnel 代码时：

1. 从 `@/components/effects/backgrounds/deform-tunnel` 导入 `DeformTunnel`。
2. 外层容器必须设置 `relative` 定位，DeformTunnel 通过 `absolute inset-0` 撑满。
3. 容器需指定明确的宽高（如 `w-full h-[400px]`），否则 Canvas 无法正确渲染。
4. 纹理 URL 必须是有效的网络图片地址，不支持本地相对路径。
5. 多个 DeformTunnel 实例不建议同时渲染，shader 循环对 GPU 负载较高。
6. 不要引入 Three.js 或 @react-three/fiber 作为使用该组件的依赖，组件内部已封装。
