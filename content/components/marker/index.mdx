Marker 标记。用于文本标记、分隔提示和信息展示。通过 MarkerIcon 和 MarkerContent 子组件组合，支持默认（带图标提示）、分隔线（两侧横线）和带下边框三种变体。

<h2 id="when-to-use">何时使用</h2>

适合在列表、卡片或页面中插入带图标的分隔标注，或在两个内容区块之间放置文本分隔线。

常见场景：
- 列表中的"没有更多了"、"以上为最新消息"等提示。
- 区块间的文本分隔线（`variant="separator"`）。
- 带图标的提示信息（如"暂无数据"、"加载失败"）。
- 带下边框的段落标记（`variant="border"`）。

不推荐：
- 用 `<hr>` 或手写 border class 代替 `variant="separator"`，Marker 提供一致的间距和语义结构。
- 在 Marker 外部手动拼接图标和文本，应使用 MarkerIcon + MarkerContent 子组件。

### 基础用法

三种变体：默认（带图标提示）、`separator`（两侧横线）、`border`（下边框）。

```tsx
import { Info } from "lucide-react"
import { Marker, MarkerIcon, MarkerContent } from "@/components/ui/marker"

export default function Demo() {
  return (
    <div className="w-full space-y-4">
      <Marker>
        <MarkerIcon><Info /></MarkerIcon>
        <MarkerContent>
          提示信息，可包含 <a href="#">链接</a>。
        </MarkerContent>
      </Marker>

      <Marker variant="separator">
        <MarkerContent>我是分隔文本</MarkerContent>
      </Marker>

      <Marker variant="border">
        <MarkerContent>下方有分隔线的标记</MarkerContent>
      </Marker>
    </div>
  )
}
```

### 纯分隔线

`variant="separator"` 不传 children 时渲染为纯横线。

```tsx
import { Marker } from "@/components/ui/marker"

export default function Demo() {
  return (
    <div className="w-full space-y-4">
      <p>上面的内容</p>
      <Marker variant="separator" />
      <p>下面的内容</p>
    </div>
  )
}
```

### asChild 用法

传入 `asChild` 可将样式合并到子元素，适合与 Link 等组件配合。

```tsx
import { Marker, MarkerContent } from "@/components/ui/marker"

export default function Demo() {
  return (
    <Marker asChild>
      <a href="/more" className="no-underline">
        <MarkerContent>查看更多</MarkerContent>
      </a>
    </Marker>
  )
}
```

<h2 id="api">API</h2>

### Marker

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 变体样式 | `"default"` \| `"separator"` \| `"border"` | `"default"` |
| asChild | 将样式合并到直接子元素，使用 Radix Slot | `boolean` | `false` |
| className | 附加样式类名 | `string` | - |
| children | 标记内容，通常为 MarkerIcon + MarkerContent | `React.ReactNode` | - |

接受所有标准 HTML `div` 元素属性。根元素带有 `data-slot="marker"` 和 `data-variant`。
- `default`：带图标的提示信息模式，图标和文本水平排列。
- `separator`：两侧横线 + 中间文本，内容居中对齐且不折行。
- `border`：下方带边框，适合段落标注。

### MarkerIcon

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |
| children | 图标元素 | `React.ReactNode` | - |

接受所有标准 HTML `span` 元素属性。带有 `aria-hidden="true"`，图标为纯装饰元素。容器固定 `16px` 正方形，内部 SVG 自动适配尺寸。

### MarkerContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |
| children | 文本或 HTML 内容 | `React.ReactNode` | - |

接受所有标准 HTML `span` 元素属性。在 `separator` 变体中自动居中对齐且不折行。内置对内部 `<a>` 标签的样式覆盖：下划线 + hover 高亮。

<h2 id="faq">常见问题</h2>

### 如何只显示分隔线不要文字？

使用 `variant="separator"` 且不传 `children`：

```tsx
<Marker variant="separator" />
```

### Marker 支持 asChild 吗？

支持。传入 `asChild` 时，样式会合并到直接子元素，适合与路由 Link 等组件配合：

```tsx
<Marker asChild>
  <Link to="/more">查看更多</Link>
</Marker>
```

### MarkerContent 中的链接如何统一样式？

MarkerContent 内置了对内部 `<a>` 标签的样式覆盖：下划线 + hover 时高亮为 foreground 颜色。无需手动为每个链接添加 class。

### 三种 variant 的典型使用场景？

- `default`：带图标的提示信息，如"暂无数据"、"加载失败"等。
- `separator`：内容区块之间的文本分隔线，如"以上为最新消息"。
- `border`：下方需要视觉分隔的标记区域，如段落标注。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/marker` 导入 Marker、MarkerIcon、MarkerContent。
2. 图标使用 `MarkerIcon` 包裹，文本使用 `MarkerContent` 包裹，不要混用。
3. 分隔线使用 `variant="separator"`，带下划线使用 `variant="border"`，默认为带图标提示模式。
4. 只使用本地源码中存在的 props，不生成 AntD 专属配置。
5. 不要引入 Ant Design 相关依赖。
