Tags 标签组。多选标签组件，基于 Command + Popover 构建，支持搜索过滤、分组展示、创建新标签、Badge 展示已选值和逐个移除操作。实际实现位于 `src/components/blocks/combobox/index.tsx`（历史上曾与一个通用 Combobox primitive 共享文件；该 primitive 与 `@/components/ui/combobox` 完全重复，已删除，现在这个文件只包含 Tags 实现），`@/components/blocks/tags` 作为对外的 barrel 重导出。

<h2 id="when-to-use">何时使用</h2>

当需要在表单或筛选区域中选择多个标签值时使用。

常见场景：
- 内容标签选择，如文章分类、用户标签。
- 多选下拉筛选器。
- 需要搜索过滤选项的标签选择。
- 需要创建新标签的内联新增场景。
- 需要 Badge 样式展示已选值并可逐个移除。

不推荐：
- 单选场景——使用 `@/components/ui/select` 或 `@/components/ui/combobox`。
- 标签仅用于展示（不可选）——使用 `@/components/ui/tag` 或 `@/components/ui/badge`。
- 大量选项（>1000）且无搜索——考虑虚拟列表或其他方案。

### 完整状态管理

结合 `value`/`setValue` 实现受控标签管理。

```tsx
import { useState } from "react"
import {
  Tags, TagsTrigger, TagsValue, TagsContent,
  TagsInput, TagsList, TagsEmpty, TagsGroup, TagsItem,
} from "@/components/ui/tags"

const allTags = [
  { value: "react", label: "React", group: "前端" },
  { value: "vue", label: "Vue", group: "前端" },
  { value: "go", label: "Go", group: "后端" },
  { value: "python", label: "Python", group: "后端" },
]

export default function Demo() {
  const [selected, setSelected] = useState("react")

  const handleRemove = (value: string) => {
    setSelected(prev => prev.split(",").filter(v => v !== value).join(","))
  }

  return (
    <Tags value={selected} setValue={setSelected} className="w-full max-w-sm">
      <TagsTrigger>
        {selected.split(",").filter(Boolean).map(tag => (
          <TagsValue key={tag} onRemove={() => handleRemove(tag)}>
            {allTags.find(t => t.value === tag)?.label ?? tag}
          </TagsValue>
        ))}
      </TagsTrigger>
      <TagsContent>
        <TagsInput placeholder="搜索标签..." />
        <TagsList>
          <TagsEmpty>未找到标签</TagsEmpty>
          <TagsGroup heading="前端">
            {allTags.filter(t => t.group === "前端").map(tag => (
              <TagsItem key={tag.value} value={tag.value}>{tag.label}</TagsItem>
            ))}
          </TagsGroup>
          <TagsGroup heading="后端">
            {allTags.filter(t => t.group === "后端").map(tag => (
              <TagsItem key={tag.value} value={tag.value}>{tag.label}</TagsItem>
            ))}
          </TagsGroup>
        </TagsList>
      </TagsContent>
    </Tags>
  )
}
```

<h2 id="api">API</h2>

### Tags

根容器，管理已选标签值和下拉开关状态。内部包裹 `Popover`，通过 ResizeObserver 跟踪触发器宽度以同步下拉宽度。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 当前已选标签值（以逗号或分隔符连接的字符串） | `string` | - |
| setValue | 设置已选标签值 | `(value: string) => void` | - |
| open | 受控打开状态 | `boolean` | - |
| onOpenChange | 打开状态变更回调 | `(open: boolean) => void` | - |
| className | 附加 CSS 类名 | `string` | - |
| children | 子组件 | `React.ReactNode` | - |

### TagsTrigger

触发器按钮，渲染为 `PopoverTrigger asChild`，`role="combobox"`。默认包含 "Select a tag..." 占位文本。

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| 继承 `Button` 所有 props | 所有 Button 组件的属性均可用 | `ComponentProps<typeof Button>` |

### TagsValue

已选标签 Badge，表示单个已选标签值。当提供 `onRemove` 时会显示 X 关闭图标。

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| onRemove | 移除标签的回调 | `() => void` |
| 继承 `Badge` 所有 props | 所有 Badge 组件的属性均可用 | `ComponentProps<typeof Badge>` |

### TagsContent

下拉面板容器，内部包裹 `Command` 组件，宽度通过 ResizeObserver 与触发器同步。

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| 继承 `PopoverContent` 所有 props | 所有 PopoverContent 组件的属性均可用 | `ComponentProps<typeof PopoverContent>` |

### TagsInput

搜索输入框，用于过滤标签列表。

| --- | --- | --- |
| 继承 `CommandInput` 所有 props | 所有 CommandInput 组件的属性均可用 | `ComponentProps<typeof CommandInput>` |

### TagsList

<h2 id="faq">常见问题</h2>

### Tags 和 Combobox 有什么区别？

Tags 的实现基于同一套 `Command` + `Popover` 组合方式，但直接封装为多选标签选择器（Badge 展示已选值、逐个移除），不依赖 `@/components/ui/combobox`。`Combobox` 更通用，用于单值搜索选择；Tags 专注于多选标签场景。两者是同类交互模型的不同产物，不是继承关系。

### 已选标签的增删逻辑由谁管理？

标签的增删逻辑完全由调用侧管理。组件通过 `value`/`setValue` 提供读写接口，`TagsValue` 的 `onRemove` 回调用于移除单个标签。调用侧负责维护已选标签字符串（通常以逗号分隔）和处理创建/移除逻辑。

### 如何获取完整的已选标签列表？

`value` 是一个字符串，需要用分隔符解析：

```tsx
const selectedList = value.split(",").filter(Boolean)
```

可以配合受控模式实现完整的状态管理（见上方"完整状态管理"示例）。

### TagsTrigger 默认显示 "Select a tag..." 如何自定义？

通过 `TagsTrigger` 的 `children` 自定义触发器内容。已选标签通过 `TagsValue` 展示，默认占位文本可通过传入自定义 `children` 覆盖。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/tags` 导入，不要从 `antd` 导入。
2. 使用 `Tags` > `TagsTrigger` + `TagsContent` 的标准结构。
3. `TagsValue` 需要 `onRemove` 回调才能显示移除按钮。
4. `TagsContent` 内部放置 `TagsInput`、`TagsList`、`TagsGroup`、`TagsItem`。
5. 标签的增删逻辑由调用侧管理（`value`/`setValue`/`onRemove`），组件本身不维护已选列表。
6. 不要生成 Ant Design 的 `mode="tags"`、`tokenSeparators`、`maxTagCount` 等属性——这些在本组件中不存在。
7. 不要引入 Ant Design 相关依赖。
