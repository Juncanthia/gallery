排版 Typography。提供文本排版能力，包含四个子组件：Text（行内文本）、Title（标题 h1-h5）、Paragraph（段落）和 Link（链接）。支持语义类型、尺寸字重、文本装饰、可复制和溢出省略等交互能力。

<h2 id="when-to-use">何时使用</h2>

当需要对文本进行统一的语义化排版时使用。Typography 提供类型、尺寸、字重、装饰等标准化样式，以及复制、溢出省略等交互能力。

常见场景：

- 页面标题层级（H1-H5）的语义化输出。
- 正文段落的语义区分（主文本、次要文本、成功、警告、危险）。
- 行内代码、键盘按键、高亮标记等特殊文本样式的快速应用。
- 可复制的标题或段落，自动附带复制按钮。
- 单行/多行文本溢出省略，支持展开/收起。

不推荐：

- 手写 `<h1>`-`<h5>` 替代 `Title level={N}`，丢失统一的标题样式。
- 手动拼接 `text-green-600` 等颜色 class 来区分文本语义，应使用 `type` 属性。
- 在 Text 中嵌套复杂的块级元素，Text 设计为行内元素（span）。
- 为了一次性生效通过 `className` 覆盖组件默认样式，优先使用组件暴露的 props。

### 溢出省略

`ellipsis` 支持单行截断和多行展开/收起。

```tsx
export default function Demo() {
  return (
    <div className="max-w-xs space-y-6">
      <div>
        <Text ellipsis className="block w-full">

        </Text>

        </Paragraph>
      </div>
    </div>
  )
}
```

<h2 id="api">API</h2>

### Text

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 文本语义类型，覆盖 variant 的颜色设置 | `"secondary"` \| `"success"` \| `"warning"` \| `"danger"` | - |
| variant | CVA 变体，与 type 互斥（type 优先生效） | `"default"` \| `"secondary"` \| `"success"` \| `"warning"` \| `"danger"` \| `"disabled"` | `"default"` |
| size | 文本尺寸 | `"xs"` \| `"sm"` \| `"base"` \| `"lg"` \| `"xl"` | `"base"` |
| weight | 字重 | `"normal"` \| `"medium"` \| `"semibold"` \| `"bold"` | `"normal"` |
| disabled | 禁用状态 | `boolean` | `false` |
| underline | 下划线 | `boolean` | `false` |
| italic | 斜体 | `boolean` | `false` |
| delete / del | 删除线（两个属性等效） | `boolean` | `false` |
| mark | 高亮标记，黄色背景 | `boolean` | `false` |
| code | 代码样式，等宽字体 + 灰色背景 | `boolean` | `false` |
| keyboard | 键盘按键样式，等宽字体 + 边框 + 阴影 | `boolean` | `false` |
| strong | 加粗（等同于 weight="semibold"） | `boolean` | `false` |
| copyable | 是否可复制，传入 CopyableConfig 可自定义图标和提示 | `boolean` \| `CopyableConfig` | - |
| ellipsis | 溢出省略配置，true 为单行截断 | `boolean` \| `EllipsisConfig` | - |
| asChild | 合并样式到子元素 | `boolean` | `false` |

渲染为 `<span>` 元素（asChild 时为 Slot）。接受所有标准 HTML span 属性。

| type | 文本语义类型 | `"secondary"` \| `"success"` \| `"warning"` \| `"danger"` | - |

### Paragraph

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 文本语义类型 | `"secondary"` \| `"success"` \| `"warning"` \| `"danger"` | - |
| disabled | 禁用状态 | `boolean` | `false` |
| strong | 加粗 | `boolean` | `false` |
| copyable | 是否可复制 | `boolean` \| `CopyableConfig` | - |
| ellipsis | 溢出省略 | `boolean` \| `EllipsisConfig` | - |

### Link

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| href | 链接地址 | `string` | - |
| disabled | 禁用状态 | `boolean` | `false` |

渲染为 `<a>` 元素。接受所有标准 HTML a 元素属性。禁用时自动移除 href 并应用灰化样式。

### CopyableConfig

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| text | 自定义复制文本，支持同步/异步函数 | `string` \| `(() => string \| Promise<string>)` | 自动取 children 文本 |
| onCopy | 复制回调 | `(event?) => void` | - |
| icon | 自定义图标，数组 [未复制图标, 已复制图标] | `ReactNode` \| `[ReactNode, ReactNode]` | 复制/对勾 |
| tooltips | 提示文本，数组 [未复制提示, 已复制提示]，false 禁用 | `ReactNode` \| `[ReactNode, ReactNode]` \| `false` | - |
| tabIndex | 复制按钮的 tabIndex | `number` | - |

### EllipsisConfig

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| rows | 省略行数，大于 1 使用多行截断 | `number` | `1` |
| expandable | 是否可展开，`"collapsible"` 支持收起 | `boolean` \| `"collapsible"` | `false` |
| suffix | 省略时末尾显示的附加内容 | `React.ReactNode` | - |
| symbol | 展开/收起操作的符号或文字 | `ReactNode` \| `(expanded: boolean) => ReactNode` | `"More"` / `"Collapse"` |
| defaultExpanded | 默认展开状态 | `boolean` | `false` |
| expanded | 受控展开状态 | `boolean` | - |
| onExpand | 展开/收起回调 | `(event, { expanded: boolean }) => void` | - |
| tooltip | 省略时是否显示完整文本的 tooltip | `boolean` \| `React.ReactNode` | - |

<h2 id="faq">常见问题</h2>

### Text、Paragraph、Title 什么时候用哪个？

- **Text**：行内文本（`<span>`），不产生块级换行。适合一段话中的文本标注、代码片段、键盘按键等。
- **Paragraph**：块级段落（`<p>`），自带上下间距 `[&:not(:first-child)]:mt-4`。适合正文内容。
- **Title**：页面或区块标题（`<h1>`-`<h5>`），映射到语义标签。适合层级标题。

### 如何禁用 copyable 的提示气泡？

传入 `copyable={{ tooltips: false }}` 即可禁用复制前后的提示。也可传入自定义 ReactNode 替换默认提示文本。

### ellipsis 的 expandable 与 collapsible 有何区别？

`expandable: true` 点击展开后不可收起，适合"查看更多"场景。`expandable: "collapsible"` 点击可在展开/收起间切换，适合"展开/收起"场景。

### Text code 和 Text keyboard 的样式区别？

- `code`：灰色背景 (`bg-muted`)，适合展示行内代码片段。
- `keyboard`：灰色背景 + 边框 + 阴影 (`shadow-xs`)，适合展示键盘按键（如 `⌘K`）。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/typography` 导入 Text、Title、Paragraph、Link，不要从 `antd` 导入。
2. 文本语义使用 `type` 属性（`secondary`/`success`/`warning`/`danger`），不要手动拼颜色 class。
3. 标题使用 `Title level={N}` 而非手动写 `<h1>`-`<h5>`，确保统一的标题样式。
4. 行内代码用 `Text code`，键盘按键用 `Text keyboard`，删除线用 `Text delete`（或 `Text del`）。
5. 只使用本地源码中存在的 props，不生成 AntD 专属配置（如 `classNames`、`styles`、`ConfigProvider`）。
6. 复制功能使用 `copyable` 属性而非手动添加 CopyButton，禁止引入 Ant Design 相关依赖。
