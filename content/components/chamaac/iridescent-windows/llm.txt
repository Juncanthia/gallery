# IridescentWindows 彩虹窗口

基于 Three.js 和 GLSL 着色器的彩虹色动态纹理背景。通过复数运算和分形映射生成流动的彩虹窗口效果。

本组件来自 chamaac UI 库，以源码为准。

## 何时使用 {#when-to-use}

适合需要彩虹色动态纹理、液体金属光泽等氛围型视觉效果的场景。

常见场景：

- 品牌展示页 Hero 区域背景。
- 深色主题页面的动态装饰层。
- 科技、艺术、前卫风格页面的氛围组件。

不推荐：

- 低端移动设备的性能敏感页面。
- 文本密集型的信息展示页。
- 需要精确视觉一致性的设计系统组件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { IridescentWindows } from "@/components/ui/iridescent-windows"

export default function Demo() {
  return (
    <div className="w-full rounded overflow-hidden">
      <IridescentWindows />
    </div>
  )
}
```

> 说明：IridescentWindows 自带 `min-h-[400px]` 和 `bg-black`，直接使用即可。

## 样式模型 {#styling-model}

- 组件根元素使用 `cn` 合并 `className`，默认类为 `relative w-full h-full min-h-[400px] bg-black overflow-hidden`。
- 内部渲染 Three.js Canvas，视觉完全由 GLSL 着色器控制。
- 着色器核心算法：复数有理函数 `polyA / polyB` 取对数虚部，结合 band/depth/glow 三层混合。
- 不依赖 CVA 变体系统。
- 不包装 Radix/Base UI primitive。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/chamaac/iridescent-windows/iridescent-windows.tsx`
- `src/components/ui/chamaac-iridescent-windows.tsx`（barrel 导出）

直接依赖摘要：

- `@react-three/fiber`
- `three`
- `clsx`

实现注意：

1. 着色器使用复数有理函数映射生成彩虹色纹理，核心为 GLSL `cx_div(polyA, polyB)` 后取对数虚部。
2. Canvas DPR 限制为 `[1, 2]`，降低 GPU 开销。
3. 组件默认 `min-h-[400px]` 确保 Canvas 有足够渲染区域。
4. 通过 `themeColors` 或 `color` 派生三层颜色，控制暗部、中间调和亮部。

## 可访问性 {#accessibility}

- 纯装饰性组件，不参与交互。
- 不需要 ARIA 标签或键盘焦点。
- 对于 prefers-reduced-motion 用户，建议在业务层禁用或替换为静态背景。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 IridescentWindows 代码时，应遵守：

1. 从 `@/components/ui/iridescent-windows` 导入，使用命名导出 `{ IridescentWindows }`，不要从 `@react-three/fiber` 或 `three` 直接引用。
2. 组件自带 `bg-black` 和 `min-h-[400px]`，确保父容器具有明确宽度。
3. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider` 配置。
4. 不要引入 Ant Design 相关依赖。
5. 动画效果由着色器内部驱动，不需要传递 `animate` 或 `autoPlay` 等 props。
6. 如果不确定某个 API 是否存在，先检查 `src/components/chamaac/iridescent-windows/iridescent-windows.tsx`，不要凭记忆生成。
