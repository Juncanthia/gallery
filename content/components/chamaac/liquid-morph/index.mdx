LiquidMorph 液态变形。基于 Three.js 和 GLSL 着色器实现，通过经典 Perlin 3D 噪声算法生成流动的液态变形视觉效果，支持鼠标交互。适合用作页面背景或氛围装饰。

<h2 id="when-to-use">何时使用</h2>

当需要为页面或区块添加液态流动、色彩变形等氛围型视觉效果时使用。

常见场景：
- 全屏品牌展示页 Hero 区域背景。
- 深色主题页面的动态装饰层。
- 创意型、艺术类页面的氛围组件。

不推荐：
- 性能敏感的低端设备页面，WebGL 着色器开销较高。
- 需要精确视觉一致性的场景，着色器效果受 GPU 差异影响。
- 纯信息展示的文本密集型页面，动态背景会分散注意力。

### 基础用法

LiquidMorph 默认铺满父容器（`absolute inset-0`），需确保父容器 `relative` 且有明确尺寸。指针事件默认禁用（`pointer-events-none`），确保不阻挡前景交互。

```tsx
import { LiquidMorph } from "@/components/ui/liquid-morph"

export default function Demo() {
  return (
    &lt;div className="relative w-full h-64 rounded overflow-hidden"&gt;
      &lt;LiquidMorph /&gt;
      &lt;div className="absolute inset-0 flex items-center justify-center"&gt;
        &lt;p className="text-white/70 text-lg font-medium"&gt;Liquid Morph&lt;/p&gt;
      &lt;/div&gt;
    &lt;/div&gt;
  )
}
```

### 自定义配色

通过 `color1`、`color2`、`color3` 自定义三色渐变，默认为 Indigo、Pink、Amber 组合。

```tsx
import { LiquidMorph } from "@/components/ui/liquid-morph"

export default function Demo() {
  return (
    &lt;div className="relative w-full h-64 rounded overflow-hidden"&gt;
      &lt;LiquidMorph
        color1="#0ea5e9"
        color2="#8b5cf6"
        color3="#f97316"
      /&gt;
    &lt;/div&gt;
  )
}
```

### 调整动画速度

通过 `speed` 控制动画流动速度，默认为 `0.5`。数值越大流动越快。

```tsx
import { LiquidMorph } from "@/components/ui/liquid-morph"

export default function Demo() {
  return (
    &lt;div className="relative w-full h-64 rounded overflow-hidden"&gt;
      &lt;LiquidMorph speed={1.5} /&gt;
    &lt;/div&gt;
  )
}
```

<h2 id="api">API</h2>

### LiquidMorph

液态变形背景组件。基于 WebGL 着色器实现，铺满父容器。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |
| speed | 动画速度倍率 | `number` | `0.5` |
| color1 | 第一层颜色（hex） | `string` | `"#4338ca"` |
| color2 | 第二层颜色（hex） | `string` | `"#db2777"` |
| color3 | 第三层颜色（hex） | `string` | `"#f59e0b"` |

<h2 id="faq">常见问题</h2>

### LiquidMorph 为什么没有显示？

确保父容器设置了 `position: relative`（或 `absolute`/`fixed`）且有明确的宽高。LiquidMorph 使用 `absolute inset-0` 铺满父容器，如果父容器没有尺寸或定位上下文，Canvas 不会渲染。

### 如何减少性能开销？

默认 DPR 限制为 `[1, 2]`，关闭了抗锯齿以优化性能。如需进一步优化，可以：
- 仅在用户可见时渲染（通过条件渲染或 IntersectionObserver）。
- 减小父容器尺寸，降低 GPU 填充率。
- 在低端设备上降级为静态背景色。

### 颜色格式支持什么？

`color1`、`color2`、`color3` 接受 6 位 hex 颜色字符串（如 `"#4338ca"`），内部通过正则解析为 RGB 传递给着色器。

### 鼠标交互如何工作？

LiquidMorph 内置鼠标交互，鼠标移动时会影响噪声采样的偏移量，产生液态跟随效果。组件自带 `pointer-events-none`，交互通过 Three.js 的 `state.pointer` 驱动。

### 如何调整视觉效果？

- `speed`：控制动画流动速度，默认 `0.5`。
- `color1`、`color2`、`color3`：三色渐变，着色器通过 smoothstep 在三种颜色之间混合。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/liquid-morph` 导入 LiquidMorph，不要从 `@react-three/fiber` 或 `three` 直接引用。
2. 确保父容器有 `position: relative` 和明确尺寸。
3. LiquidMorph 自带 `pointer-events-none`，前景内容需要独立的点击层。
4. 不要生成 AntD 专属的 `classNames`、`styles`、ConfigProvider 等配置。
5. 不要引入 Ant Design 相关依赖。
