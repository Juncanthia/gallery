# Waves 波浪

基于 Three.js 和 GLSL 着色器的 3D 单纯形噪声波浪动态背景。通过对平面网格进行顶点位移，生成流动的波浪效果。

本组件来自 chamaac UI 库，以源码为准。

## 何时使用 {#when-to-use}

适合需要动态波浪、海洋流动等氛围背景的场景。

常见场景：

- 品牌展示页 Hero 区域背景。
- 深色主题页面的动态装饰层。
- 科技感、海洋主题页面的氛围组件。

不推荐：

- 低端移动设备的性能敏感页面。
- 文本密集型的信息展示页。
- 需要精确视觉一致性的设计系统组件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { Waves } from "@/components/effects/backgrounds/chamaac/waves/waves"

export default function Demo() {
  return (
    <div className="relative w-full h-64 rounded overflow-hidden bg-black">
      <Waves />
    </div>
  )
}
```

> 说明：Waves 铺满父容器，使用 `absolute inset-0` 定位。确保父容器 `relative` 且有尺寸。

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
interface WavesProps {
  className?: string
  backgroundColor?: string  // Canvas 背景色，默认 "#000000"
  waveColor1?: string       // 波浪中层颜色，默认 "#071697"
  waveColor2?: string       // 波浪高峰颜色，默认 "#00d4ff"
  waveColor3?: string       // 波浪谷底颜色，默认 "#000000"
  waveSpeedX?: number       // X 轴流动速度，默认 0.0125
  waveSpeedY?: number       // Y 轴流动速度，默认 0.005
  waveAmpX?: number         // 波浪振幅，默认 32
}
```

## 样式模型 {#styling-model}

- 组件根元素使用 `cn` 合并 `className`，默认类为 `absolute inset-0 w-full h-full pointer-events-none z-0`。
- 内部渲染 Three.js Canvas，视觉完全由 GLSL 着色器控制。
- 使用 3D 单纯形噪声（Simplex Noise）对平面网格进行顶点位移。
- 不依赖 CVA 变体系统。
- 不包装 Radix/Base UI primitive。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/chamaac/waves/waves.tsx`
- `src/components/ui/chamaac-waves.tsx`（barrel 导出）

直接依赖摘要：

- `@react-three/fiber`
- `@react-three/drei`
- `three`
- `clsx`

实现注意：

1. 着色器采用 3D 单纯形噪声（Simplex Noise）对平面网格进行顶点位移，生成波浪效果。
2. 颜色映射：噪声值 [0, 0.5) 在 waveColor3 和 waveColor1 之间混合，[0.5, 1.0] 在 waveColor1 和 waveColor2 之间混合。
3. 组件完全覆盖父容器（`absolute inset-0`），需父容器提供定位上下文。
4. `pointer-events-none` 确保不阻挡前景交互。
5. Canvas camera 位置为 `[0, 0, 4]`，平面网格带 -0.2 弧度 X 轴旋转以产生透视感。

## 可访问性 {#accessibility}

- 纯装饰性组件，不参与交互。
- 不需要 ARIA 标签或键盘焦点。
- 对于 prefers-reduced-motion 用户，建议在业务层禁用或替换为静态背景。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Waves 代码时，应遵守：

1. 从 `@/components/effects/backgrounds/chamaac/waves/waves` 导入，不要从 `@react-three/fiber` 或 `three` 直接引用。
2. 确保父容器具有 `position: relative` 和明确的宽高尺寸。
3. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider` 配置。
4. 不要引入 Ant Design 相关依赖。
5. 动画效果由着色器内部驱动（`useFrame` + `u_time`），不需要传递 `animate` 或 `autoPlay` 等 props。
6. 如果不确定某个 API 是否存在，先检查 `src/components/chamaac/waves/waves.tsx`，不要凭记忆生成。
