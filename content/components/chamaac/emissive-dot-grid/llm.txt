# EmissiveDotGrid 发光点阵

基于 Three.js 和 GLSL 着色器的程序化发光点阵背景。通过 Perlin 噪声驱动点阵动画，结合 Bloom 后处理产生辉光效果。

本组件来自 chamaac UI 库，以源码为准。

## 何时使用 {#when-to-use}

适合需要动态点阵、发光粒子效果的场景。

常见场景：

- 品牌展示页 Hero 区域背景。
- 深色主题页面的动态装饰层。
- 科技、数据主题页面的氛围组件。

不推荐：

- 低端移动设备的性能敏感页面。
- 文本密集型的信息展示页。
- 需要精确视觉一致性的设计系统组件。

import { BurntOrangeDotGrid } from "@/components/effects/backgrounds/chamaac/emissive-dot-grid/emissive-dot-grid"

## 样式模型 {#styling-model}

- 组件根元素使用 `fixed inset-0 w-screen h-screen bg-[#010101] z-0` 覆盖全视口。
- 内部渲染 Three.js Canvas（orthographic 相机），视觉完全由 GLSL 着色器控制。
- 着色器实现 Perlin 噪声驱动点阵缩放动画，颜色为橙色调（#df5000）。
- 使用 EffectComposer + Bloom 后处理产生辉光效果。
- 点阵使用 InstancedMesh，网格尺寸 140x160，AdditiveBlending 混合。
- 不依赖 CVA 变体系统。
- 不包装 Radix/Base UI primitive。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/chamaac/emissive-dot-grid/emissive-dot-grid.tsx`
- `src/components/ui/chamaac-emissive-dot-grid.tsx`（barrel 导出）

直接依赖摘要：

- `@react-three/fiber`
- `@react-three/drei`
- `@react-three/postprocessing`
- `three`
- `clsx`

实现注意：

1. 使用 InstancedMesh 渲染 140x160 个平面点，避免逐个实例的性能开销。
2. Perlin 噪声标准实现（snoise），驱动每颗点的缩放动画。
3. Canvas DPR 使用默认值，antialias/stencil/depth 均关闭以降低 GPU 开销。
4. 组件完全覆盖视口（`fixed inset-0`），z-index 为 `0`，前景内容需更高层级。
5. Bloom post-processing 产生橙色辉光效果，intensity=1.8, radius=0.4。
