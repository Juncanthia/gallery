EmissiveDotGrid 发光点阵。基于 Three.js 实例化渲染和 GLSL 着色器实现的程序化发光点阵背景。通过 Perlin 噪声驱动每颗点的缩放动画，结合 Bloom 后处理产生辉光效果。适合用作全屏背景或氛围装饰。

<h2 id="when-to-use">何时使用</h2>

当需要为页面添加动态点阵、发光粒子等氛围型视觉效果时使用。

常见场景：

- 全屏背景装饰，如 Hero 区域、品牌展示页。
- 深色主题页面的动态背景层。
- 科技、数据、星空主题的营销页面。

不推荐：

- 性能敏感的低端设备页面，WebGL 实例化渲染开销较高。
- 需要精确视觉一致性的场景，着色器效果受 GPU 差异影响。
- 纯信息展示的文本密集型页面，动态背景会分散注意力。

### 基础用法

EmissiveDotGrid 默认铺满整个视口（`fixed inset-0 w-screen h-screen`），适合作为页面全局背景层。

```tsx
import { BurntOrangeDotGrid } from "@/components/ui/chamaac-emissive-dot-grid"

export default function Demo() {
  return (
    &lt;div className="relative w-full h-96 rounded overflow-hidden bg-[#010101]"&gt;
      &lt;BurntOrangeDotGrid /&gt;
    &lt;/div&gt;
  )
}
```

<h2 id="api">API</h2>

### EmissiveDotGrid

发光点阵组件。基于 Three.js 实例化渲染，使用 140x160 点阵和 Perlin 噪声驱动动画。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

组件无外部配置参数，视觉和动画参数由内部着色器控制。

<h2 id="faq">常见问题</h2>

### EmissiveDotGrid 为什么没有显示？

确保父容器有明确的尺寸。组件使用 `fixed inset-0 w-screen h-screen` 覆盖视口，在局部预览中需要包裹在 `relative` 容器内并设置高度。

### 如何减少性能开销？

- 组件内部已使用实例化渲染（InstancedMesh）和 orthographic 相机以降低 GPU 开销。
- 如需进一步优化，可在低端设备上降级为静态背景色。
- 仅在用户可见时渲染（通过条件渲染或 IntersectionObserver）。

### 如何调整视觉效果？

当前版本的颜色、点阵密度、Bloom 强度等参数硬编码在着色器中。如需定制，需直接修改 `src/components/chamaac/emissive-dot-grid/emissive-dot-grid.tsx` 中的着色器代码和配置值。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/chamaac-emissive-dot-grid` 导入 BurntOrangeDotGrid，不要从 `@react-three/fiber` 或 `three` 直接引用。
2. 组件默认覆盖全视口（`fixed inset-0 w-screen h-screen`），作为页面全局背景层使用。
3. 确保 z-index 层级规划正确，前景内容需要高于组件的 `z-0`。
4. 不要生成 AntD 专属的 `classNames`、`styles`、ConfigProvider 等配置。
5. 不要引入 Ant Design 相关依赖。
