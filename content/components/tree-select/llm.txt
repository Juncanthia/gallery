# TreeSelect 树选择

树型选择控件。

本组件借鉴 Ant Design TreeSelect 的使用心智，但不是 AntD 兼容层。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

适合在需要 树选择 相关能力的界面中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 需要复用统一设计语言、状态样式和可访问性约定。
- 需要和本项目 Tailwind 主题变量、暗色模式、Radix/Base UI primitive 或本地交互行为保持一致。
- 需要让 AI 或业务代码稳定地从 `@/components/ui/tree-select` 引用组件，而不是直接引用第三方底层库。

不推荐：

- 为了照搬 Ant Design API 而新增兼容别名。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

import { TreeSelect } from "@/components/ui/tree-select"

export default function Demo() {

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

### TreeSelect

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| options | 树形选项数据 | `TreeSelectOption[]` | - |
| value | 当前选中值（受控） | `string` \| `string[]` | - |
| defaultValue | 默认选中值 | `string` \| `string[]` | `[]` |
| onChange | 值变化回调 | `(value: string \| string[]) => void` | - |
| multiple | 是否支持多选 | `boolean` | `false` |
| treeCheckable | 是否在节点前显示复选框 | `boolean` | `false` |
| placeholder | 未选择时的占位文本 | `string` | `"Select..."` |
| showSearch | 是否显示搜索输入框 | `boolean` | `false` |
| filterTreeNode | 自定义搜索过滤函数 | `(inputValue: string, node: TreeSelectOption) => boolean` | - |
| allowClear | 是否允许一键清空 | `boolean` | `true` |
| disabled | 是否禁用 | `boolean` | `false` |
| className | 自定义容器样式类名 | `string` | - |
| triggerClassName | 自定义触发器样式类名 | `string` | - |
| treeDefaultExpandAll | 是否默认展开全部节点 | `boolean` | `false` |

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- 源码未声明 `data-slot`，使用根元素语义和组件名定位。

额外状态属性：

- 无额外 `data-*` 状态属性。

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 如果源码使用 CVA，则变体样式集中在组件文件或相邻 `*-variants.ts` 中。
- 如果源码包装 Radix/Base UI primitive，则交互状态、键盘行为和 ARIA 基础能力来自底层 primitive，本地组件负责视觉和 API 收敛。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground`、`ring-ring` 等项目级语义变量。

## 可访问性 {#accessibility}

- 保留原生元素或底层 primitive 的键盘语义。
- 交互型组件应提供可访问名称，例如 `aria-label`、`title` 或可见文本。
- 禁用、加载、展开、选中等状态应优先使用组件已有 props，而不是只改视觉 class。
- 纯图标触发器必须提供明确的可访问名称。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/ui/tree-select.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 借鉴 AntD 使用心智，但不承诺 API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 TreeSelect 代码时，应遵守：

1. 从 `@/components/ui/tree-select` 导入，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
4. 需要自定义样式时，优先使用组件暴露的 props；最后才使用 `className`。
5. 交互组件必须保留键盘和 ARIA 语义，纯图标操作必须补 `aria-label`。
6. 如果不确定某个 AntD API 是否存在，先检查 `src/components/ui/tree-select.tsx`，不要凭 AntD 记忆生成。
