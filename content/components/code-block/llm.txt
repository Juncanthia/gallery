# CodeBlock 代码块

CodeBlock 是一个功能完整的代码展示组件，基于 Shiki 实现语法高亮，支持多文件 Tab 切换、行号显示、diff/高亮/聚焦/词高亮标注、文件图标自动匹配和复制功能。它使用 `@radix-ui/react-use-controllable-state` 管理文件 Tab 切换状态，通过 Shiki transformers 提供丰富的代码标注能力。

本组件不是 Ant Design 的兼容层。它是本地设计系统内建的复合代码展示组件。

## 何时使用 {#when-to-use}

需要在页面中展示带语法高亮的代码块时使用，例如：

- 文档站点中的代码示例展示。
- 组件文档中的用法代码。
- 技术博客中的代码片段。
- 多文件代码对比展示（通过文件切换）。

常见场景：

- 组件文档页面的代码演示区域。
- API 文档中的请求/响应示例。
- 教程中的多步骤代码展示。
- Storybook 或组件库的代码预览。

推荐使用方式：

- 使用 CodeBlock 作为最外层容器，传入 `data` 数组定义多个代码文件。
- 在 CodeBlock 内组合 CodeBlockHeader（含 CodeBlockFiles + CodeBlockCopyButton）、CodeBlockBody（含 CodeBlockItem + CodeBlockContent）。
- 使用 `defaultValue` 或 `value` + `onValueChange` 控制当前显示的代码文件。

不推荐：

- 将 CodeBlock 用于单行内联代码——请使用 `<code>` 标签或 Snippet 组件。
- 在 CodeBlockContent 中传入非代码文本——CodeBlock 专为代码展示设计。
- 不设置 `data` 或传递空数组——CodeBlock 必须有至少一个文件。
- 引入 antd 或 Prism 等外部高亮库替代 Shiki。

## 代码演示 {#examples}

### 基础用法

展示单文件代码块，带语法高亮和复制按钮。

```tsx
import {
  CodeBlock,
  CodeBlockHeader,
  CodeBlockBody,
  CodeBlockItem,
  CodeBlockContent,
  CodeBlockCopyButton,
  CodeBlockFilename,
} from "@/components/ui/code-block"

const code = `import { useState } from "react"

export function Counter() {
  const [count, setCount] = useState(0)
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  )
}`

export default function Demo() {
  return (
    <CodeBlock data={[{ language: "tsx", filename: "counter.tsx", code }]}>
      <CodeBlockHeader>
        <CodeBlockFilename>counter.tsx</CodeBlockFilename>
        <CodeBlockCopyButton />
      </CodeBlockHeader>
      <CodeBlockBody>
        {(item) => (
          <CodeBlockItem value={item.language}>
            <CodeBlockContent language="tsx">{item.code}</CodeBlockContent>
          </CodeBlockItem>
        )}
      </CodeBlockBody>
    </CodeBlock>
  )
}
```

### 多文件切换

通过 data 数组定义多个文件，配合 CodeBlockFiles 和 CodeBlockSelect 实现切换。

```tsx
import {
  CodeBlock,
  CodeBlockHeader,
  CodeBlockFiles,
  CodeBlockFilename,
  CodeBlockSelect,
  CodeBlockSelectTrigger,
  CodeBlockSelectValue,
  CodeBlockSelectContent,
  CodeBlockSelectItem,
  CodeBlockBody,
  CodeBlockItem,
  CodeBlockContent,
  CodeBlockCopyButton,
} from "@/components/ui/code-block"

const files = [
  { language: "tsx", filename: "app.tsx", code: `export default function App() {\n  return <div>Hello</div>\n}` },
  { language: "css", filename: "styles.css", code: `body {\n  margin: 0;\n  padding: 0;\n}` },
]

export default function Demo() {
  return (
    <CodeBlock data={files} defaultValue="tsx">
      <CodeBlockHeader>
        <CodeBlockFiles>
          {(item) => (
            <CodeBlockFilename key={item.language} value={item.language}>
              {item.filename}
            </CodeBlockFilename>
          )}
        </CodeBlockFiles>
        <CodeBlockSelect>
          <CodeBlockSelectTrigger>
            <CodeBlockSelectValue />
          </CodeBlockSelectTrigger>
          <CodeBlockSelectContent>
            {(item) => (
              <CodeBlockSelectItem key={item.language} value={item.language}>
                {item.filename}
              </CodeBlockSelectItem>
            )}
          </CodeBlockSelectContent>
        </CodeBlockSelect>
        <CodeBlockCopyButton />
      </CodeBlockHeader>
      <CodeBlockBody>
        {(item) => (
          <CodeBlockItem key={item.language} value={item.language}>
            <CodeBlockContent language={item.language as BundledLanguage}>
              {item.code}
            </CodeBlockContent>
          </CodeBlockItem>
        )}
      </CodeBlockBody>
    </CodeBlock>
  )
}
```

## API {#api}

### CodeBlock

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| data | 代码数据数组，每项包含 language、filename、code | `CodeBlockData[]` | 必填 |
| defaultValue | 非受控模式下的默认激活 language | `string` | `""` |
| value | 受控模式下的激活 language | `string` | - |
| onValueChange | 激活 language 变化的回调 | `(value: string) => void` | - |
| className | 附加 className | `string` | - |

### CodeBlockData

| 字段 | 说明 | 类型 |
| --- | --- | --- |
| language | 代码语言标识（对应 Shiki BundledLanguage） | `string` |
| filename | 文件名，用于文件图标匹配和展示 | `string` |
| code | 源代码文本 | `string` |

### CodeBlockHeader

标准 `HTMLAttributes<HTMLDivElement>` 容器，内含 `flex h-9 items-center justify-between border-b` 布局。

### CodeBlockFiles

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 渲染函数，接收 CodeBlockData 每项，返回 ReactNode | `(item: CodeBlockData) => ReactNode` | 必填 |
| className | 附加 className | `string` | - |

### CodeBlockFilename

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| icon | 自定义文件图标组件 | `IconType` | 自动匹配 |
| value | 仅在当前激活文件与此 value 匹配时显示 | `string` | - |
| children | 文件名文本 | `React.ReactNode` | - |
| className | 附加 className | `string` | - |

文件名图标自动根据文件名模式匹配（如 `*.tsx` -> SiReact, `*.css` -> SiCss, Dockerfile -> SiDocker 等 80+ 种文件类型）。

### CodeBlockSelect / CodeBlockSelectTrigger / CodeBlockSelectValue / CodeBlockSelectItem

分别对应 Select、SelectTrigger、SelectValue、SelectItem 组件的封装，自动从 CodeBlockContext 获取 value 和 onValueChange。

### CodeBlockSelectContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 渲染函数，接收 CodeBlockData 每项 | `(item: CodeBlockData) => ReactNode` | 必填 |

### CodeBlockCopyButton

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| onCopy | 复制成功回调 | `() => void` | - |
| onError | 复制出错回调 | `(error: Error) => void` | - |
| timeout | 复制后图标状态保持时长（ms） | `number` | `2000` |
| className | 附加 className | `string` | - |

自动获取当前激活文件的代码内容作为复制目标。content 已由组件内部处理。

### CodeBlockBody

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 渲染函数，接收 CodeBlockData 每项 | `(item: CodeBlockData) => ReactNode` | 必填 |

### CodeBlockItem

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 文件标识（language），仅匹配当前激活值时渲染 | `string` | 必填 |
| lineNumbers | 是否显示行号 | `boolean` | `true` |
| className | 附加 className | `string` | - |

### CodeBlockContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| children | 源代码文本 | `string` | 必填 |
| language | Shiki 语言标识 | `BundledLanguage` | - |
| syntaxHighlighting | 是否启用语法高亮 | `boolean` | `true` |
| themes | 主题配置（已预留，当前固定使用 CSS 变量主题） | `CodeOptionsMultipleThemes["themes"]` | - |
| className | 附加 className | `string` | - |

## 样式模型 {#styling-model}

CodeBlock 通过 CSS 变量主题 + Tailwind CSS 构建。

- 外层容器：`bg-[var(--code-bg-color)] text-[var(--text-color)]` 使用 CSS 变量支持明暗主题，`rounded border` 提供边框和圆角。
- Header：`h-9 flex items-center` 横向排列，`border-b` 底部分隔，`bg-muted/20` 弱背景。
- 代码区域：`text-[13px] leading-relaxed` 紧凑但可读的字号，`[&_.shiki]:!bg-transparent` 确保 Shiki 背景透明。
- 行号：通过 CSS counter 实现（`counter-reset: line` / `counter-increment: line`），`[&_.line]:before:content-[counter(line)]`。
- 行高亮：高亮行左侧带颜色竖线标记（`w-0.5`），带浅色背景。
- diff 标注：add 行绿色背景 + 绿色左边线，remove 行红色背景 + 红色左边线。
- 聚焦：非聚焦行 `blur-[2px]`，聚焦行 `blur-none`。
- 词高亮：`highlighted-word` 类带浅色背景。
- Shiki 使用 `createCssVariablesTheme` 通过 `--shiki-` 前缀变量注入颜色，自动适配暗色模式。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/code-block/index.tsx`：全部子组件定义。

直接依赖：

- `shiki`（`codeToHtml`、`BundledLanguage`、`createCssVariablesTheme`）
- `@shikijs/transformers`（notation-diff、notation-highlight、notation-word-highlight、notation-focus、notation-error-level）
- `react-icons/si`（80+ 文件类型图标）
- `@radix-ui/react-use-controllable-state`（受控/非受控状态管理）
- `@/components/ui/copy-button`（复制按钮）
- `@/components/ui/select`（下拉选择器）
- `@/lib/utils`（cn 工具函数）

关键行为：

1. Shiki 语法高亮在 useEffect 中异步执行，完成后通过 dangerouslySetInnerHTML 注入。
2. 语法高亮失败时回退到 CodeBlockFallback 纯文本 `<pre><code>` 结构。
3. 文件图标通过正则匹配 filenameIconMap，按文件名模式自动选择 Simple Icons。
4. CodeBlockContext 向下传递 value、onValueChange、data，子组件通过 useContext 获取。
5. CodeBlockItem 仅在 `value === activeValue` 时渲染，实现 Tab 切换。
6. CSS 变量主题 (`--shiki-*`) 使得高亮颜色自动适配明暗模式。
7. themes prop 已预留但当前固定使用 CSS 变量主题。

## 可访问性 {#accessibility}

- 代码块为只读展示内容，不参与 Tab 顺序（除 CopyButton 和 Select）。
- CodeBlockCopyButton 基于 CopyButton，继承其键盘可访问性。
- CodeBlockSelect 基于 Select 组件，继承其 ARIA 语义和键盘导航。
- 语法高亮禁用时回退到标准 `<pre><code>` 结构。
- 行号和标注通过 CSS 实现，不影响 DOM 语义。
- 语义化 `data-slot` 属性用于测试定位。
- 代码内容通过 `dangerouslySetInnerHTML` 渲染，需确保输入的代码来源可信。

## 与 Ant Design 的取舍 {#antd-differences}

CodeBlock 是本地复合组件，Ant Design 中无对应组件。Shiki 是独立的语法高亮引擎。

| 方向 | 本地决策 |
| --- | --- |
| AntD 对应 | 无直接对应。AntD 不提供代码高亮展示组件 |
| 语法高亮 | 使用 Shiki + CSS 变量主题，非 Prism 或 AntD |
| 视觉系统 | Tailwind CSS + 项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 组合模式 | 基于 React Context + render props 的复合组件模式 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 CodeBlock 代码时，应遵守：

1. 从 `@/components/ui/code-block` 导入 CodeBlock 及所需子组件，不要从 antd 导入。
2. CodeBlock 的 `data` 属性为必填，每项必须包含 `language`、`filename`、`code`。
3. CodeBlockFiles、CodeBlockBody、CodeBlockSelectContent 使用 render props 模式（`(item: CodeBlockData) => ReactNode`），回调中需为每项生成 key。
4. CodeBlockItem 和 CodeBlockSelectItem 的 `value` 必须与 data 中的 `language` 字段匹配。
5. CodeBlockContent 的 `language` 必须与 data 中的 language 一致，`children` 必须是代码字符串（非 HTML）。
6. 不要用 `<pre><code>` 手动替代 CodeBlockContent——语法高亮、行号、diff 标注等能力仅在 CodeBlockContent 中可用。
7. 禁用语法高亮时设置 `syntaxHighlighting={false}`，不要直接移除 CodeBlockContent。
8. 代码行内标注使用 Shiki transformer 注释语法（如 `// [!code highlight]`、`// [!code ++]`），不要手动添加 CSS 类。
9. 不要引入 antd 或 Prism 等外部高亮库。
10. 如果需新增能力或不确定某 API，先检查 `src/components/blocks/code-block/index.tsx`。
