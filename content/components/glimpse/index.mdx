Glimpse 悬停预览卡片组件，基于 Radix HoverCard 封装。鼠标悬停在触发器上时弹出浮层显示额外的内容预览，支持标题、描述、图片等布局。包含 Glimpse、GlimpseTrigger、GlimpseContent、GlimpseTitle、GlimpseDescription 和 GlimpseImage 六个子组件。

<h2 id="when-to-use">何时使用</h2>

当需要悬停展示补充信息而不离开当前页面时使用。

常见场景：

- 链接悬停预览（文章标题悬停显示摘要和封面图）。
- 用户头像悬停显示用户卡片。
- 产品名称悬停显示缩略信息。
- 术语悬停显示定义。

不推荐：

- 需要复杂交互的弹出内容（如表单、操作按钮）——使用 Popover。
- 移动端主要交互路径——HoverCard 无 touch 事件支持。
- 需要点击才能打开的场景——使用 Dialog 或 Popover。

<h2 id="api">API</h2>

### Glimpse

悬停卡片顶层容器，等同于 `HoverCard`。继承 `ComponentProps<typeof HoverCard>`，所有 Radix HoverCard 属性均可用。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| open | 受控开启状态 | `boolean` | - |
| defaultOpen | 默认开启状态 | `boolean` | - |
| onOpenChange | 开启状态变化回调 | `(open: boolean) => void` | - |
| openDelay | 悬停到打开的延迟（ms） | `number` | - |
| closeDelay | 移开到关闭的延迟（ms） | `number` | - |

### GlimpseTrigger

触发器，悬停在此元素上弹出预览内容。继承 `ComponentProps<typeof HoverCardTrigger>`。通常使用 `asChild` 模式包裹链接或文本。

### GlimpseContent

弹出浮层的内容容器。继承 `ComponentProps<typeof HoverCardContent>`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| side | 弹出方向 | `"top"` \| `"bottom"` \| `"left"` \| `"right"` | - |
| align | 对齐方式 | `"start"` \| `"center"` \| `"end"` | - |
| className | 附加 className | `string` | - |

### GlimpseTitle

预览卡片标题，渲染为 `<p>`。继承 `ComponentProps<"p">`。默认样式：`truncate font-semibold text-sm`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

### GlimpseDescription

预览卡片描述文本，渲染为 `<p>`，最多显示 2 行。继承 `ComponentProps<"p">`。默认样式：`line-clamp-2 text-muted-foreground text-sm`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加 className | `string` | - |

### GlimpseImage

预览卡片缩略图，渲染为 `<img>`。继承 `ComponentProps<"img">`。默认样式：`mb-4 aspect-[120/63] w-full rounded-md border object-cover`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| src | 图片地址 | `string` | 必填 |
| alt | 图片替代文本 | `string` | 必填 |
| className | 附加 className | `string` | - |

<h2 id="faq">常见问题</h2>

### Glimpse 和 HoverCard 有什么区别？

Glimpse 是对 HoverCard 的语义化封装，所有 Glimpse/GlimpseTrigger/GlimpseContent 均直接透传至 HoverCard/HoverCardTrigger/HoverCardContent。额外提供了 GlimpseTitle、GlimpseDescription 和 GlimpseImage 三个预设样式的子组件，便于快速构建悬停预览卡片。

### 如何在移动端使用？

HoverCard 依赖 mouseenter/mouseleave 事件，无 touch 支持。如需在移动端使用类似弹出效果，建议改用 Popover 组件的 click 触发模式。

### 触发器必须使用链接吗？

不需要。GlimpseTrigger 使用 `asChild` 模式可以包裹任意元素（按钮、文本、图标等），只要该元素接受 ref 即可。

### 如何控制延迟时间？

通过 Glimpse 的 `openDelay` 和 `closeDelay` 属性控制。`openDelay` 为鼠标悬停后到卡片出现的延迟（ms），`closeDelay` 为鼠标移开后到卡片消失的延迟（ms）。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/glimpse` 导入所需的子组件。
2. Glimpse 必须包含 GlimpseTrigger 和 GlimpseContent 作为直接子节点。
3. GlimpseContent 内部推荐顺序：GlimpseImage（可选）-> GlimpseTitle -> GlimpseDescription。
4. GlimpseTrigger 通常使用 `asChild` 包裹触发元素（链接、按钮、文本等）。
5. 所有组件透传底层 Radix 属性，type 与 HoverCard 一致，可直接查阅 Radix HoverCard 文档。
6. 不要用于移动端主要交互路径，HoverCard 无 touch 事件支持。
7. 不要引入 Ant Design 相关依赖。
