播放控制 PlaybackControl。响应用户点击，切换媒体播放/暂停状态，自动根据播放状态更新无障碍标签。

<h2 id="when-to-use">何时使用</h2>

当需要在媒体播放器中提供播放、暂停、重播控制按钮时使用。PlaybackControl 通过 `usePlaybackStore` 读取当前播放状态（playing / paused / ended / buffering），并在点击时调用 `togglePaused()` 切换播放状态。

常见场景：

- 音频播放器的主播放/暂停按钮。
- 视频播放器的播放控制按钮。
- 配合 `asChild` 将状态行为注入自定义按钮组件。
- 通过 `shortcut` 提示键盘快捷键（如 Space）。

不推荐：

- 在非 MediaProvider 上下文之外独立使用 PlaybackControl，它依赖 `usePlaybackStore`。
- 需要同时展示上/下曲切换按钮时，应使用 `PlaybackControls`（blocks 层），而非单独组合 PlaybackControl。
- 在不需要状态驱动的纯静态 Toggle 场景中使用，应直接用 Button 的 `onClick` 切换。

<h2 id="examples">代码演示</h2>

### 基础用法

使用 `asChild` 将播放状态控制注入自定义 Button，根据 `blocks/status` 展示不同图标。

<ComponentShowcase id="limeplay/playback-control" path="limeplay/playback-control/basic" />

```tsx
import { Button } from "@/components/core/button"
import { PlaybackControl } from "@/components/core/playback-control"
import { usePlaybackStore } from "@/components/media/hooks/use-playback"

export default function Demo() {
  const blocks/status = usePlaybackStore((state) => state.blocks/status)

  return (
    <PlaybackControl asChild shortcut="Space">
      <Button size="icon" variant="ghost">
        {blocks/status === "playing" ? (
          <PauseIcon weight="fill" />
        ) : blocks/status === "ended" ? (
          <RepeatIcon />
        ) : blocks/status === "buffering" ? (
          <CircleNotchIcon className="animate-spin" weight="bold" />
        ) : (
          <PlayIcon weight="fill" />
        )}
      </Button>
    </PlaybackControl>
  )
}
```

<h2 id="api">API</h2>

### PlaybackControl

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 使用 Radix Slot 将状态和行为合并到子元素 | `boolean` | `false` |
| render | 渲染自定义子元素，与 `asChild` 互斥 | `React.ReactElement` | - |
| shortcut | 键盘快捷键提示，写入 `aria-keyshortcuts` 和 `aria-label` | `string` | - |
| disabled | 手动禁用按钮；当 `readyState` &lt; `HAVE_CURRENT_DATA` 时组件也会自动禁用 | `boolean` | - |
| onClick | 点击回调，在 `togglePaused()` 之前触发 | `(event: React.MouseEvent&lt;HTMLButtonElement&gt;) => void` | - |

组件继承 `React.ButtonHTMLAttributes&lt;HTMLButtonElement&gt;`，支持所有原生 button 属性。

**状态驱动的 aria-label 自动生成规则：**

| blocks/status | aria-label |
| --- | --- |
| playing | "Pause" + shortcut 提示 |
| ended | "Replay" + shortcut 提示 |
| 其他 | "Play" + shortcut 提示 |

**自动禁用逻辑：** 当 `readyState < MediaReadyState.HAVE_CURRENT_DATA`（即 `readyState < 2`）时，组件自动设置 `disabled`。当媒体数据尚未加载到可播放程度时，播放按钮不可用。

<h2 id="faq">常见问题</h2>

### 如何在自定义按钮中使用？

使用 `asChild` 将 PlaybackControl 的行为注入到子组件中。子组件会继承 PlaybackControl 的 `onClick`（调用 `togglePaused`）、`aria-label`、`aria-keyshortcuts` 和 `disabled` 状态：

```tsx
<PlaybackControl asChild shortcut="Space">
  <Button size="icon" variant="ghost">
    <PlayIcon />
  </Button>
</PlaybackControl>
```

### 如何监听播放状态变化？

通过 `usePlaybackStore` 选择器读取状态：

```tsx
const blocks/status = usePlaybackStore((state) => state.blocks/status)
// blocks/status: "init" | "loading" | "playing" | "paused" | "buffering" | "ended" | "error" | ...
```

### 如何在 render 模式下使用？

传入 `render` 属性指定要渲染的子元素，PlaybackControl 会通过 `React.cloneElement` 注入 props：

```tsx
<PlaybackControl render={<CustomButton />}>
  <PlayIcon />
</PlaybackControl>
```

### 为什么点击后没有反应？

检查是否在 `MediaProvider` 上下文中使用。PlaybackControl 依赖 `usePlaybackStore`，该 hook 只能在 `MediaProvider` 内部使用。另外检查 `readyState` 是否已到达 `HAVE_CURRENT_DATA`，否则组件会被自动禁用。

### shortcut 属性如何使用？

传入键盘快捷键标识（如 `"Space"`），组件会自动：
1. 写入 `aria-keyshortcuts="Space"`
2. 在 `aria-label` 中追加 `(keyboard shortcut Space)` 提示

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/playback-control` 导入 PlaybackControl。
2. 必须配合 `MediaProvider` 使用，不能单独渲染。
3. 推荐使用 `asChild` 模式包裹自定义 Button。
4. 如需展示状态相关图标，通过 `usePlaybackStore` 读取 `blocks/status`。
5. 不要引入 Ant Design 相关依赖。
