# TimelineControl 时间轴控制

媒体播放器时间轴交互控件。基于 @radix-ui/react-slider 的复合组件模式，通过 Zustand store 连接播放器状态实现拖拽寻址、进度展示和缓冲区域显示。Root 为根容器，内部组合 Track、Progress、Buffered、Thumb 构成完整时间轴控件。

本组件面向 Limeplay 播放器生态，不引入 Ant Design 的 Slider 或 ConfigProvider。

## 何时使用 {#when-to-use}

适合在需要 时间轴控制 相关能力的界面中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 需要复用统一设计语言、状态样式和可访问性约定。
- 需要和本项目 Tailwind 主题变量、Radix primitive 或本地交互行为保持一致。
- 需要让 AI 或业务代码稳定地从 `@/components/core/timeline-control` 引用组件，而不是直接引用第三方底层库。

不推荐：

- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons` 或 `antd-style` 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import * as TimelineControl from "@/components/core/timeline-control"

export default function Demo() {
  return (

```

## API {#api}

### Root

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| orientation | 时间轴方向 | `horizontal` \| `vertical` | `horizontal` |
| disabled | 禁用交互 | `boolean` | `false` |
| className | 根元素 CSS 类名 | `string` | - |

> 继承 Radix `Slider.Root` 的所有 props。自动连接播放器状态，在 readyState 低于 HAVE_METADATA 时自动禁用。处理 pointer 事件实现拖拽寻址。

### Track

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 根元素 CSS 类名 | `string` | - |

> 继承 Radix `Slider.Track` 的所有 props。推荐添加 `overflow-hidden` 限制子元素溢出。

### Progress

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 根元素 CSS 类名 | `string` | - |

> 继承 Radix `Slider.Range` 的所有 props。宽度通过 CSS 变量 `--lp-played-width` 由播放进度动态控制。

### Buffered

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 缓冲区域渲染模式 | `combined` \| `default` \| `from-zero` | `default` |
| className | 根元素 CSS 类名 | `string` | - |

> 继承标准 `div` 元素属性。variant 控制缓冲区域的渲染方式：`default` 分别显示每段；`combined` 合并为一段；`from-zero` 从零开始显示。无缓冲数据时返回 null。

### Thumb

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| position | 自定义位置百分比 | `number` | - |
| showWithHover | hover 时跟随光标 | `boolean` | `false` |
| className | 根元素 CSS 类名 | `string` | - |

> 继承 Radix `Slider.Thumb` 的所有 props。位置通过 CSS 变量 `--lp-timeline-thumb-position` 动态设置。duration 为 0 或无效时返回 null。

## Semantic DOM {#semantic-dom}

组件通过 Radix Slider primitive 暴露语义结构：

- Root 渲染为 Radix `Slider.Root`，`aria-label` 设为 "Timeline Slider"。
- Track 渲染为 Radix `Slider.Track`，`tabIndex={0}` 支持键盘操作。
- Progress 渲染为 Radix `Slider.Range`。
- Buffered 渲染为包含多个范围片段的 `div` 容器。
- Thumb 渲染为 Radix `Slider.Thumb`。

额外状态属性：

- `disabled` 属性在 readyState 不足或手动设置时启用。
- `data-disabled` 在 Thumb 禁用时设置。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/core/limeplay-timeline-control.tsx`：ui 层 re-export。
- `src/components/limeplay/ui/timeline-control.tsx`：核心实现。

直接依赖摘要：

- `@radix-ui/react-slider`
- `@hyper/limeplay/hooks/use-playback`
- `@hyper/limeplay/hooks/use-player`
- `@hyper/limeplay/hooks/use-timeline`
- `@hyper/limeplay/hooks/use-track-events`
- `@hyper/limeplay/lib/utils`

实现注意：

1. Root 使用 `useImperativeHandle` 转发 ref 到内部 SliderPrimitive.Root 的 ref。
2. Progress 的宽度由 `useTimelineStore` 的 `progress` 状态通过 CSS 变量动态控制。
3. Buffered 使用 `processBufferedRanges` 处理原始缓冲数据，按 variant 归一化为百分比区间。
4. Thumb 位置优先级：`position` prop > `showWithHover` 时的 hover 位置 > 当前播放位置。
5. 直播模式（isLive）下，seek 行为会自动映射到 seekRange 区间。
6. 所有 pointer 事件交互通过 `useTrackEvents` hook 统一处理。

## 可访问性 {#accessibility}

- Root 设置 `aria-label="Timeline Slider"`。
- Track 设置 `tabIndex={0}`，支持键盘焦点和 focus-visible 样式。
- disabled 状态通过原生 disabled 属性传递，自动禁用键盘和 pointer 交互。
- 键盘操作和 ARIA 基础能力来自 Radix Slider primitive。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/limeplay/ui/timeline-control.tsx` 的 TypeScript 类型为准 |
| primitive | 使用 Radix Slider，不使用 AntD Slider 或 rc-slider |
| 状态管理 | 使用 Zustand store，不依赖 AntD ConfigProvider |
| 视觉系统 | 使用 Tailwind CSS 和 CSS 变量 |
| 兼容目标 | Limeplay 生态专用，不承诺 AntD API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 TimelineControl 代码时，应遵守：

1. 从 `@/components/core/timeline-control` 导入，推荐使用命名空间 `import * as TimelineControl from "..."`。
2. Root、Track、Progress、Buffered、Thumb 必须按复合模式嵌套使用。
3. Root 必须放在 Limeplay Player Provider 作用域内。
4. Track 推荐添加 `className="overflow-hidden"`。
5. Thumb 推荐使用 `showWithHover` 配合 `group/timeline` 实现 hover 交互。
6. Buffered 默认 variant 为 `"default"`，简洁场景使用 `"combined"`。
7. 不要引入 Ant Design 相关依赖。
8. 如果不确定某个能力是否存在，先检查源码，不要凭 AntD Slider 记忆生成。
