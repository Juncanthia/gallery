# VideoPlayer 视频播放器

基于 Shaka Player 与 Media Chrome 的完整视频播放器复合组件。集成 `MediaProvider` 状态管理、`PlaybackSourceController` 源切换、错误恢复屏幕、加载海报和底部控制栏，开箱即用。

支持 DASH、HLS 流媒体格式，自适应码率切换，DRM 内容保护，以及播放列表管理。

## 何时使用

当需要功能完整的视频播放器时使用。点播视频、直播流、需要播放列表管理的场景均适用。

不适用于简单视频嵌入（使用更轻量的 `blocks/video-player`）或第三方平台嵌入（YouTube 等）。

## 代码演示

### 基础用法

```tsx
import { VideoPlayer } from "@/components/core/media-player"

export default function Demo() {
  return (
    <div className="w-full max-w-3xl">
      <VideoPlayer className="aspect-video" source="https://example.com/video.mp4" />
    </div>
  )
}
```

## API

### VideoPlayer

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| autoLoad | 是否自动加载媒体源 | `boolean` | - |
| children | 自定义子内容 | `ReactNode` | - |
| className | 根容器附加样式类名 | `string` | - |
| controlsHideDelay | 控制栏自动隐藏延迟（毫秒） | `number` | - |
| debug | 启用调试模式 | `boolean` | - |
| hideCursorOnIdle | 空闲时隐藏鼠标光标 | `boolean` | - |
| initialIndex | 初始播放项索引 | `number` | - |
| loading | 自定义资源加载配置 | `UseAssetOptions<VideoPlayerAsset>` | - |
| mediaProps | 透传至底层 `<video>` 元素的属性 | `Omit<VideoHTMLAttributes<HTMLVideoElement>, "as" \| "src">` | - |
| source | 媒体源 URL 或源配置 | `PlayerSource<VideoPlayerAsset>` | - |
| sourceKey | 源缓存键，用于触发重新加载 | `string` | - |

### VideoPlayerAsset

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| config | Shaka Player 配置 | `shaka.extern.PlayerConfiguration` | - |
| description | 资源描述 | `string` | - |
| id | 资源唯一标识 | `string` | - |
| poster | 海报图 URL | `string` | - |
| src | 媒体源 URL | `string` | - |
| title | 资源标题 | `string` | - |
| year | 发布年份 | `string` | - |

## 技术实现

核心文件：

- `src/components/limeplay/blocks/video-player/components/media-player.tsx`

技术依赖：

- `react` — 组件框架
- `tailwindcss` — 样式系统
- `shaka-player` — DASH/HLS 流媒体播放引擎
- `media-chrome` — Web Components 媒体控制 UI
- `@hyper/limeplay/hooks/use-asset` — 资源加载状态管理
- `@hyper/limeplay/hooks/use-playback` — 播放状态管理
- `@hyper/limeplay/hooks/use-playback-source` — 源切换控制器

实现要点：

1. `VideoPlayer` 作为顶层复合组件，内部组合 `MediaProvider`、`PlaybackSourceController`、`RootContainer`、`Media`（video）和完整 UI 层。
2. Shaka Player 负责媒体解码、自适应码率、DRM 解密等核心播放能力。
3. Media Chrome 提供 Material Design 风格的控制栏 UI（通过 Web Components）。
4. 内置 `FallbackPoster`（加载前展示）、`PlayerErrorScreen`（错误恢复）和 `CaptionsContainer`（字幕渲染）。
5. 播放状态通过 Zustand store（`usePlaybackStore`）暴露，可在外部监听和响应。

## LLM 使用规则

1. 从 `@/components/core/media-player` 导入 `VideoPlayer`。
2. `source` 支持字符串 URL 或对象数组，传入 URL 即可快速启动。
3. 不要使用原生 `<video controls>` 替代本组件。
4. 本组件默认包含底部控制栏、字幕容器、错误恢复和加载海报，无需手动组合子组件。
5. 播放列表场景通过 `source` 传入对象数组，每个对象需包含 `src` 字段。
6. 调试时设置 `debug={true}` 启用 Shaka Player 日志。
7. 本组件为复合组件（Blocks），不依赖 Ant Design。
