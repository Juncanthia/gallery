画中画控制 PictureInPictureControl。响应用户点击，切换画中画（Picture-in-Picture）模式，自动根据画中画状态更新无障碍标签。

<h2 id="when-to-use">何时使用</h2>

当需要在媒体播放器中提供画中画模式切换按钮时使用。PictureInPictureControl 通过 `usePictureInPictureStore` 读取当前画中画状态（active / supported），并在点击时调用 `toggle()` 切换画中画模式。

常见场景：

- 视频播放器的画中画控制按钮。
- 配合 `asChild` 将状态行为注入自定义按钮组件。
- 通过 `shortcut` 提示键盘快捷键（如 P）。

不推荐：

- 在非 MediaProvider 上下文之外独立使用 PictureInPictureControl，它依赖 `usePictureInPictureStore`。
- 需要同时展示播放/暂停/音量等控制时，应使用 `BottomControls`（blocks 层），而非单独组合 PictureInPictureControl。
- 在不需要状态驱动的纯静态 Toggle 场景中使用，应直接用 Button 的 `onClick` 切换。

<h2 id="examples">代码演示</h2>

### 基础用法

使用 `asChild` 将画中画状态控制注入自定义 Button，根据 `active` 状态展示不同图标。

<ComponentShowcase id="limeplay/pip-control" path="limeplay/pip-control/basic" />

```tsx
import { PictureInPictureIcon } from "@phosphor-icons/react"
import { Button } from "@/components/core/button"
import { PictureInPictureControl } from "@/components/core/pip-control"
import { usePictureInPictureStore } from "@/components/media/hooks/use-picture-in-picture"

export default function Demo() {
  const isPictureInPictureActive = usePictureInPictureStore(
    (state) => state.active
  )

  return (
    <PictureInPictureControl asChild shortcut="P">
      <Button size="icon" variant="ghost">
        <PictureInPictureIcon
          weight={isPictureInPictureActive ? "fill" : "regular"}
        />
      </Button>
    </PictureInPictureControl>
  )
}
```

<h2 id="api">API</h2>

### PictureInPictureControl

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 使用 Radix Slot 将状态和行为合并到子元素 | `boolean` | `false` |
| render | 渲染自定义子元素，与 `asChild` 互斥 | `React.ReactElement` | - |
| shortcut | 键盘快捷键提示，写入 `aria-keyshortcuts` 和 `aria-label` | `string` | - |
| disabled | 手动禁用按钮；当 `readyState` &lt; `HAVE_METADATA` 或浏览器不支持画中画时组件也会自动禁用 | `boolean` | - |
| onClick | 点击回调，在 `toggle()` 之前触发 | `(event: React.MouseEvent&lt;HTMLButtonElement&gt;) => void` | - |

组件继承 `React.ButtonHTMLAttributes&lt;HTMLButtonElement&gt;`，支持所有原生 button 属性。

**状态驱动的 aria-label 自动生成规则：**

| active | aria-label |
| --- | --- |
| true | "Exit Picture-in-Picture" + shortcut 提示 |
| false | "Enter Picture-in-Picture" + shortcut 提示 |

**自动禁用逻辑：**

1. 当 `readyState < MediaReadyState.HAVE_METADATA`（即元数据未加载）时，组件自动设置 `disabled`。
2. 当 `!isPictureInPictureSupported`（浏览器不支持画中画）时，组件自动设置 `disabled`。
3. 手动传入 `disabled={true}` 时始终禁用。

**数据属性：**

组件渲染的元素带有 `data-label="lp-picture-in-picture-control"` 属性，用于测试定位和样式覆写。

<h2 id="faq">常见问题</h2>

### 如何在自定义按钮中使用？

使用 `asChild` 将 PictureInPictureControl 的行为注入到子组件中。子组件会继承 PictureInPictureControl 的 `onClick`（调用 `toggle`）、`aria-label`、`aria-keyshortcuts` 和 `disabled` 状态：

```tsx
<PictureInPictureControl asChild shortcut="P">
  <Button size="icon" variant="ghost">
    <PictureInPictureIcon />
  </Button>
</PictureInPictureControl>
```

### 如何监听画中画状态变化？

通过 `usePictureInPictureStore` 选择器读取状态：

```tsx
const isActive = usePictureInPictureStore((state) => state.active)
const isSupported = usePictureInPictureStore((state) => state.supported)
```

### 如何在 render 模式下使用？

传入 `render` 属性指定要渲染的子元素，PictureInPictureControl 会通过 `React.cloneElement` 注入 props：

```tsx
<PictureInPictureControl render={<CustomButton />}>
  <PictureInPictureIcon />
</PictureInPictureControl>
```

### 为什么点击后没有反应？

检查是否在 `MediaProvider` 上下文中使用。PictureInPictureControl 依赖 `usePictureInPictureStore` 和 `usePlaybackStore`，这些 hook 只能在 `MediaProvider` 内部使用。另外检查 `readyState` 是否已到达 `HAVE_METADATA` 且浏览器是否支持画中画，否则组件会被自动禁用。

### shortcut 属性如何使用？

传入键盘快捷键标识（如 `"P"`），组件会自动：
1. 写入 `aria-keyshortcuts="P"`
2. 在 `aria-label` 中追加 `(keyboard shortcut P)` 提示

### 如何与 VideoPlayer 配合使用？

PictureInPictureControl 设计为 VideoPlayer 的子组件，放置在 `MediaProvider` 上下文内即可正常工作。如需与其他播放器控制组合，可将其放入 `BottomControls` 区域：

```tsx
<VideoPlayer source={src}>
  <PictureInPictureControl asChild shortcut="P">
    <CustomButton />
  </PictureInPictureControl>
</VideoPlayer>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/pip-control` 导入 PictureInPictureControl。
2. 必须配合 `MediaProvider` 使用，不能单独渲染。
3. 推荐使用 `asChild` 模式包裹自定义 Button。
4. 如需展示状态相关图标，通过 `usePictureInPictureStore` 读取 `active`。
5. 不要引入 Ant Design 相关依赖。
