Dialog 模态对话框，在页面正中打开浮层承载内容并锁定背景交互。基于 Radix Dialog 实现，支持标题、描述、操作按钮区和受控显隐。

<h2 id="when-to-use">何时使用</h2>

需要用户处理事务，又不希望跳转页面以致打断工作流程时，使用 Dialog 在当前页面打开浮层承载相应操作。

- 需要用户确认的操作（删除、提交等）—— 使用 Dialog 或 AlertDialog。
- 需要用户在浮层中填写表单（如新增/编辑）。
- 需要展示详细信息但不想跳页。
- 当只是一个简单的确认询问时，使用 Popconfirm 或 AlertDialog 更轻量。

<h2 id="examples">代码演示</h2>

### 基础用法

`DialogTrigger` 触发打开，`DialogContent` 容纳头部、正文和底部操作区。

<ComponentShowcase
  id="dialog"
  path="dialog/basic"
/>

```tsx
import { useState } from "react"
import { Button } from "@/components/ui/button"
import {
  Dialog, DialogContent, DialogDescription,
  DialogFooter, DialogHeader, DialogTitle, DialogTrigger,
} from "@/components/ui/dialog"

export default function Demo() {
  const [open, setOpen] = useState(false)

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>打开对话框</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>确认操作</DialogTitle>
          <DialogDescription>
            确定要执行此操作吗？此操作不可撤销。
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button variant="outlined" onClick={() => setOpen(false)}>取消</Button>
          <Button color="primary" variant="solid" onClick={() => setOpen(false)}>确认</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  )
}
```

<h2 id="api">API</h2>

### Dialog

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| open | 对话框是否可见（受控） | `boolean` | - |
| defaultOpen | 默认是否可见（非受控） | `boolean` | `false` |
| onOpenChange | 显隐状态变化回调 | `(open: boolean) => void` | - |
| modal | 是否为模态（锁定背景交互） | `boolean` | `true` |

### 子组件

| 组件 | 说明 | 建议用法 |
| --- | --- | --- |
| DialogTrigger | 触发打开的元素 | `asChild` 包裹 Button |
| DialogContent | 对话框主体面板 | 包裹 Header + Body + Footer |
| DialogHeader | 头部区域 | 包含 Title + Description |
| DialogTitle | 对话框标题 | 简洁描述操作目的 |
| DialogDescription | 补充描述文本 | 说明操作后果或细节 |
| DialogFooter | 底部操作按钮区 | `flex gap-2 justify-end` |
| DialogClose | 关闭按钮 | 渲染为 X 图标按钮 |

<h2 id="faq">常见问题</h2>

### Dialog 与 AlertDialog 的区别？

- **Dialog**：通用模态对话框，内容灵活，适合表单、详情等自定义场景。
- **AlertDialog**：专注确认/警告型对话框，内置取消和确认按钮，用于破坏性操作（删除等）。

### 如何实现嵌套对话框？

不推荐嵌套 Dialog。如需在对话框中再次确认，考虑在当前 Dialog 中使用 Popconfirm 或将确认流程合并为一个表单步骤。

### 如何自定义关闭按钮的行为？

使用受控模式 `open` + `onOpenChange`，在 `onOpenChange` 中加入自定义逻辑（如表单脏数据提示）。

```tsx
<Dialog open={open} onOpenChange={(next) => {
  if (isDirty && !confirm("有未保存的更改，确定关闭？")) return
  setOpen(next)
}}>
```

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Dialog 代码时：

1. 从 `@/components/ui/dialog` 导入所有子组件。
2. 使用受控模式管理 `open` 状态，确保可在操作完成后关闭。
3. `DialogTrigger` 使用 `asChild` 包裹 Button，避免多余的 DOM 嵌套。
4. 破坏性确认操作优先使用 AlertDialog，不要用 Dialog 模拟确认框。
5. 不要引入 Ant Design 的 `Modal`。
