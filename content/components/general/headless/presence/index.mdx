根据 `present` 属性控制子元素的挂载与卸载，支持 CSS 动画进出过渡。基于 Radix UI Presence 原语实现。

<h2 id="when-to-use">何时使用</h2>

当需要根据某个状态条件性地挂载 / 卸载 DOM 节点，并且希望在卸载前播放退出动画时使用。常见场景：

- 对话框、抽屉等浮层组件的开 / 关动画。
- 提示信息、通知的淡入淡出。
- Tooltip、Popover 的延迟显示与隐藏。
- 列表项的添加 / 删除过渡动画。
- 任何需要 "先播退出动画，再移除 DOM" 的场景。

Presence 的核心价值在于：普通的条件渲染（`{show && &lt;Component /&gt;}`）会在状态切换时**立即移除 DOM**，导致退出动画无法播放。Presence 会检测子元素的 CSS 动画，在动画结束后再真正卸载，确保退出动画完整执行。

<h2 id="examples">代码演示</h2>

通过 `present` 属性控制子元素的挂载与卸载。当 `present` 从 `true` 变为 `false` 时，如果子元素定义了退出动画（animation-name），Presence 会等待动画完成后再移除 DOM。

<ComponentShowcase id="general/headless/presence" path="general/headless/presence/basic" />

```tsx
import { Presence } from "@/components/general/headless/presence"

export default function Demo() {
  const [present, setPresent] = useState(true)

  return (
    <div className="space-y-4">
      <button onClick={() => setPresent(p => !p)}>
        {present ? "卸载" : "挂载"}内容
      </button>
      <Presence present={present}>
        <div className="data-[state=open]:animate-in data-[state=closed]:animate-out">
          内容区域
        </div>
      </Presence>
    </div>
  )
}
```

<h2 id="api">API</h2>

### Presence

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| present | 是否挂载子元素，`false` 时等待退出动画完成后卸载 | `boolean` | - |
| children | 子元素，支持 render props `({ present })` 获取当前 presence 状态 | `React.ReactElement \| (props: { present: boolean }) => React.ReactElement` | - |

> 本组件直接包装 Radix UI `Presence.Root`，接受其所有 props。

<h2 id="faq">常见问题</h2>

### 和条件渲染 `{show && &lt;Component /&gt;}` 有什么区别？

条件渲染在状态变为 `false` 时**立即从 DOM 中移除元素**，退出动画无法播放。Presence 会检测元素的 CSS `animation-name`，等待动画完成后才卸载，确保退出动画完整展示。

### Presence 如何检测退出动画？

Presence 通过读取子元素的 `getComputedStyle()` 中的 `animation-name` 来判断。当 `present` 从 `true` 变为 `false` 时，如果子元素有非 `none` 的 `animation-name`，Presence 会进入 `unmountSuspended` 状态，监听 `animationend` 事件，待动画结束后再真正卸载。

### 子元素没有动画会怎样？

如果子元素的 `animation-name` 为 `none` 或 `display` 为 `none`，Presence 会在 `present` 变为 `false` 时**立即卸载**，不等待动画。行为等价于条件渲染。

### 子元素如何定义进入 / 退出动画？

使用 Tailwind CSS 动画工具类或自定义 CSS `@keyframes`。推荐做法是结合 `data-state` 属性：

```css
/* 进入动画 */
[data-state="open"] { animation: fadeIn 0.2s ease-out; }
/* 退出动画 */
[data-state="closed"] { animation: fadeOut 0.2s ease-in; }
```

Presence 会自动设置 `data-state="open"`（挂载中）和 `data-state="closed"`（退出中）。

### render props 模式何时使用？

当你需要在卸载过程中仍然渲染内容（`forceMount`），让子组件根据 `present` 状态自行决定渲染逻辑时使用：

```tsx
<Presence present={show}>
  {({ present }) =>
    present ? <DialogContent /> : null
  }
</Presence>
```

这种模式下，Presence 永远不会返回 `null`，而是始终渲染子元素，由调用侧控制显示逻辑。

<h2 id="llm-rules">LLM 使用规则</h2>

当 AI 生成本项目 Presence 代码时，应遵守：

1. 从 `@/components/general/headless/presence` 导入，不要从 `radix-ui` 直接导入。
2. `present` 是必需属性，传入 `boolean` 值控制挂载 / 卸载。
3. 需要退出动画的场景必须使用 Presence，不要用条件渲染替代。
4. 子元素的动画通过 CSS `animation-name` 定义，Presence 自动检测并等待动画完成。
5. 使用 `data-[state=open]` 和 `data-[state=closed]` 属性选择器定义动画样式。
6. 不要在 Presence 外部嵌套条件渲染（如 `{show && <Presence>}`），这会使 Presence 失去作用。
7. 不要引入 Ant Design 相关依赖。
