# Portal 传送门

Portal 用于将子节点渲染到父组件 DOM 层级之外的指定容器中。它封装了 `ReactDOM.createPortal` 并内置 SSR 安全检测，确保在服务端渲染时不尝试访问浏览器 DOM。

本组件基于 Radix Slot 原语，支持 `asChild` 将传送能力合并到已有元素上。

## 何时使用 {#when-to-use}

当需要突破 CSS 层叠上下文限制（如父级 `overflow: hidden`、`z-index`）时使用。常见于对话框、下拉菜单、工具提示等浮层组件。

Portal 只决定 DOM 挂载位置；子节点的样式、事件、React 上下文仍由组件树管理。

## 导入

```tsx
import { Portal } from "@/components/general/headless/portal"
```

## 代码演示 {#examples}

### 基础用法

```tsx
import { Portal } from "@/components/general/headless/portal"

export default function Demo() {
  return (
    <Portal>
      <div className="fixed bottom-4 right-4 z-50 rounded-md border bg-background px-4 py-3 text-sm shadow">
        Portal 内容 — 已传送至 document.body
      </div>
    </Portal>
  )
}
```

### 指定容器

```tsx
import { Portal } from "@/components/general/headless/portal"
import { useRef } from "react"

export default function Demo() {
  const containerRef = useRef<HTMLDivElement>(null)

  return (
    <div>
      <div ref={containerRef} className="rounded-md border border-dashed p-4">
        自定义容器区域
      </div>
      <Portal container={containerRef.current}>
        <span className="text-sm font-medium text-primary">
          已传送至自定义容器
        </span>
      </Portal>
    </div>
  )
}
```

### asChild 模式

```tsx
import { Portal } from "@/components/general/headless/portal"

export default function Demo() {
  return (
    <Portal asChild>
      <span className="fixed bottom-4 right-4 rounded-md border bg-background px-3 py-2 text-sm shadow">
        合并模式 — 包裹元素被省略
      </span>
    </Portal>
  )
}
```

## API {#api}

### Portal

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| container | 目标容器 DOM 节点。SSR 时为 `null` | `Element` \| `DocumentFragment` \| `null` | `document.body` |
| asChild | 使用 Radix Slot 合并到子元素 | `boolean` | `false` |
| children | 需要传送的内容 | `React.ReactNode` | - |
| className | 附加 className | `string` | - |

支持所有 Radix Slot 属性。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Portal 代码时，应遵守：

1. 从 `@/components/general/headless/portal` 导入 Portal。
2. Portal 仅负责 DOM 挂载位置管理，不参与样式或布局逻辑。
3. 当组件需要突破父级 CSS 限制时，使用 Portal 包裹浮层内容。
4. `container` 属性接受 DOM 节点引用。SSR 环境中使用条件渲染或确保传入 `null`。
5. Portal 内部的事件处理遵循 React 合成事件冒泡规则，无需特殊处理。
6. 不要在 Portal 内放置复杂的全局状态管理逻辑。
7. 需要合并到子元素时使用 `asChild`，避免额外的 DOM 包裹。
