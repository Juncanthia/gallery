删除按钮是一个带有撤销倒计时的危险操作交互组件。点击"Delete Account"后展开撤销区域，显示 10 秒倒计时和撤销按钮，使用 motion/react 实现流畅的状态切换动画。

本组件是 UseLayouts 集合中的交互组件，展示危险操作二次确认的视觉与动效方案。

<h2 id="when-to-use">何时使用</h2>

当需要为危险操作（如删除账号、清空数据）提供撤销窗口时使用。点击主按钮后，按钮膨胀为包含撤销图标、取消文案和倒计时的确认区域。

常见场景：

- 账号删除、数据清空等不可逆操作前的缓冲确认。
- 需要几秒钟冷静期的危险操作流程。
- 用流畅动画替代传统弹窗二次确认的交互方案。

交互流程：

- **初始状态**：红色"Delete Account"按钮。
- **确认状态**：点击后按钮展开，左侧出现撤销图标，中间显示"Cancel Deletion"，右侧显示 10 秒倒计时。
- **撤销操作**：点击取消按钮或倒计时按钮，组件还原为初始状态。
- **动画锁**：状态切换期间 400ms 内禁止再次点击，防止动画冲突。

<h2 id="api">API</h2>

### DeleteButton

DeleteButton 是一个自包含组件，所有交互状态（删除确认、倒计时、动画锁）由组件内部管理，无需外部 props。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| - | 无需外部 props | - | - |

### 内部状态

| 状态 | 类型 | 说明 |
| --- | --- | --- |
| `isDeleting` | `boolean` | 是否处于确认删除状态，控制 AnimatePresence 中两个 motion.button 的显隐 |
| `count` | `number` | 倒计时秒数，初始 10，每秒递减，归零后停止 |
| `isAnimating` | `boolean` | 动画锁，状态切换后 400ms 内阻止重复点击 |

### 动画参数

| 参数 | 说明 | 值 |
| --- | --- | --- |
| layout transition | 布局切换动画 | `{ duration: 0.4, ease: [0.77, 0, 0.175, 1] }` |
| backgroundColor | 背景色过渡 | `{ duration: 0.4, ease: "easeInOut" }` |
| 文字逐字入场 | 每个字符的入场动画 | `{ duration: 0.3, ease: [0.785, 0.135, 0.15, 0.86] }`，延迟 `i * 0.005s` |
| count 数字滚动 | 倒计时数字切换动画 | `{ duration: 0.2, ease: [0.33, 1, 0.68, 1] }` |

### 视觉结构

| 元素 | 说明 |
| --- | --- |
| `AnimatePresence mode="popLayout"` | 管理两个按钮状态的进出动画，`popLayout` 确保退出元素立即脱离布局流 |
| `key="delete"` 按钮 | 初始态红色按钮，显示"Delete Account"文字，字符逐个 animated |
| `key="cancel"` 按钮 | 确认态粉色按钮，包含撤销图标（Undo03Icon）、"Cancel Deletion"文字和倒计时计数 |
| `motion.span key={count}` | 倒计时数字，通过 AnimatePresence 实现数字滚动切换效果 |

<h2 id="faq">常见问题</h2>

### 如何修改文案？

修改组件内部的 `deleteText` 和 `cancelText` 常量：

```tsx
const deleteText = "Delete Account";
const cancelText = "Cancel Deletion";
```

### 如何修改倒计时时长？

修改 `useState(10)` 的初始值。注意同时调整 `useEffect` 中 `count === 0` 的终止条件如需保留。

### 如何调整动画速度？

修改 `motion.button` 和 `motion.span` 的 `transition` 属性：

- 布局切换动画：调整 `layout transition` 的 `duration` 和 `ease` 曲线。
- 文字逐字动画：调整字符动画的 `duration` 和 `delay` 乘数（`i * 0.005`）。

### 动画锁的作用是什么？

`isAnimating` 状态在按钮点击后 400ms 内置为 `true`，并在 `onClick` 中检查。当 `isAnimating` 为 `true` 时，`pointerEvents: "none"` 阻止按钮接收点击事件，避免 AnimatePresence 动画进行中用户重复点击导致状态错乱。

### 如何自定义颜色？

当前按钮使用内联样式 `style` 设置背景色：
- 删除态：`#FE322A`（红色）和 `#FFEDF1`（浅粉）
- 确认态：`#FFEDF1`（浅粉）和 `#FE322A`（红色）

如需修改，直接调整 `motion.button` 的 `animate`、`initial`、`exit` 中的 `backgroundColor` 值。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 DeleteButton 代码时：

1. 从 `@/components/general/delete-button` 导入 `DeleteButton`（default export）
2. 组件自包含，无需传入任何 props
3. 不要移除或修改组件内部的 `motion/react` 动画逻辑，包括 AnimatePresence、layoutId、transition 曲线
4. 修改文案时，保持 `deleteText` 和 `cancelText` 为字符串常量
5. 修改倒计时时长时，同步检查 `useEffect` 终止条件和 `handleClick` 中的 `setCount(10)`
6. 保留 `isAnimating` 动画锁机制，防止动画进行中重复触发状态切换
7. 图标使用 `@hugeicons/core-free-icons` 的 `Undo03Icon` + `@hugeicons/react` 的 `HugeiconsIcon`
8. 不要引入 Ant Design 或 rc-component 相关依赖
