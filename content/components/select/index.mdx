Select 选择器。下拉选择器，基于 Radix Select 和 Base UI Combobox 实现。支持单选、多选、标签创建、搜索过滤、选项分组、自定义渲染和多种尺寸变体。

<h2 id="when-to-use">何时使用</h2>

弹出一个下拉菜单给用户选择操作，用于代替原生的选择器，或者需要一个更优雅的多选器时。

常见场景：
- 表单中选择单一选项（如性别、状态、城市）。
- 筛选面板中选择多个标签。
- 邮件系统中输入多个收件人（tags 模式）。
- 可搜索+可输入的选择框（combobox 模式）。
- 大量选项需要通过搜索过滤的场景。

不推荐：
- 当选项少于 5 项时，建议直接将选项平铺，使用 RadioGroup 是更好的选择。
- 需要级联层级选择时请使用 Cascader。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

### 基础用法

单选、多选和搜索模式。`mode="multiple"` 启用多选，`showSearch` 启用搜索过滤。

```tsx
import { useState } from "react"
import { Select } from "@/components/ui/select"

export default function Demo() {
  const [value, setValue] = useState<string>()
  const [multi, setMulti] = useState<string[]>()

  return (
    <div className="flex flex-wrap items-start gap-4">
      <Select
        options={[
          { label: "选项一", value: "1" },
          { label: "选项二", value: "2" },
          { label: "选项三（禁用）", value: "3", disabled: true },
        ]}
        placeholder="请选择"
        value={value}
        onValueChange={setValue}
        className="w-44"
      />
      <Select
        mode="multiple"
        allowClear
        options={[
          { label: "苹果", value: "apple" },
          { label: "香蕉", value: "banana" },
          { label: "橘子", value: "orange" },
        ]}
        placeholder="多选"
        value={multi}
        onValueChange={setMulti}
        className="w-48"
      />
      <Select
        showSearch
        options={[
          { label: "北京", value: "beijing" },
          { label: "上海", value: "shanghai" },
          { label: "广州", value: "guangzhou" },
        ]}
        placeholder="搜索城市"
        className="w-44"
      />
    </div>
  )
}
```

<h2 id="api">API</h2>

### Select

主组件，同时支持声明式 API（`options` 模式）和 Legacy Radix 组合模式。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| options | 选项列表，支持 `SelectOption` 和 `SelectOptionGroup` | `SelectOptionItem[]` | - |
| value | 受控值；单选/combobox 为 `string`，多选/标签为 `string[]` | `string \| string[]` | - |
| defaultValue | 非受控默认值 | 同 `value` | - |
| onValueChange | 选中值变化回调；单选/combobox 返回 `string \| undefined`，多选/标签返回 `string[]` | `(value) => void` | - |
| mode | 选择模式 | `"multiple"` \| `"tags"` \| `"combobox"` | 单选 |
| placeholder | 未选中时的占位文本 | `React.ReactNode` | - |
| showSearch | 是否显示搜索输入框，大量选项时建议开启 | `boolean` | `false` |
| searchPlaceholder | 搜索输入框的占位文本 | `string` | - |
| allowClear | 是否显示清除按钮，允许清空已选值 | `boolean` | `false` |
| disabled | 是否禁用 | `boolean` | `false` |
| size | 尺寸 | `"small"` \| `"middle"` \| `"large"` | `"middle"` |
| variant | 视觉变体 | `"outlined"` \| `"filled"` | `"outlined"` |
| status | 校验状态 | `"error"` \| `"warning"` | - |
| placement | 下拉菜单弹出位置 | `"bottomLeft"` \| `"bottomRight"` \| `"topLeft"` \| `"topRight"` | `"bottomLeft"` |
| maxCount | 多选模式下的最大选中数量 | `number` | - |
| tokenSeparators | tags 模式下的分隔符数组，用于批量粘贴 | `string[]` | - |
| filterOption | 自定义搜索过滤逻辑，`false` 关闭过滤 | `boolean \| ((input, option) => boolean)` | 内置过滤 |
| filterSort | 搜索结果自定义排序 | `(a, b, input) => number` | - |
| notFoundContent | 搜索无匹配时的展示内容 | `React.ReactNode` | `"No results found."` |
| optionRender | 自定义下拉选项中每一项的渲染 | `(option, info) => React.ReactNode` | - |
| popupRender | 自定义整个下拉面板的渲染 | `(originNode) => React.ReactNode` | - |
| labelRender | 自定义选中后在触发器中展示的标签 | `(option) => React.ReactNode` | - |
| prefix | 前缀图标 | `React.ReactNode` | - |
| suffixIcon | 后缀图标，传入 `null` 隐藏 | `React.ReactNode \| null` | 下滑箭头 |
| onSearch | 搜索文本变化回调，用于远程搜索 | `(value: string) => void` | - |
| onSelect | 选中选项时的回调 | `(value, option) => void` | - |
| onDeselect | 取消选中选项时的回调（多选） | `(value, option) => void` | - |
| onClear | 点击清除按钮时的回调 | `() => void` | - |
| open | 受控展开下拉菜单 | `boolean` | - |
| defaultOpen | 默认展开 | `boolean` | - |
| onOpenChange | 下拉菜单展开/收起回调 | `(open: boolean) => void` | - |
| className | 触发器附加样式类名 | `string` | - |
| contentClassName | 下拉面板附加样式类名 | `string` | - |
| listClassName | 选项列表附加样式类名 | `string` | - |
| name | 表单字段名称 | `string` | - |
| id | HTML id 属性 | `string` | - |
| required | 是否必填 | `boolean` | - |
| aria-label | 无障碍标签 | `string` | - |

<h2 id="faq">常见问题</h2>

### 单选、多选、标签和搜索模式如何选择？

- **单选（默认）**：适合从有限选项中选一个，如表单中的性别、状态。
- `mode="multiple"`：从预设选项中选多个，如筛选面板。
- `mode="tags"`：用户可自由输入任意文本并回车创建标签，如邮件收件人。
- `mode="combobox"`：单选 + 可搜索 + 可自由输入，如搜索式选择框。
- `showSearch`：不改变模式，仅增强单选/多选的搜索过滤体验。

### 大量选项时如何优化体验？

1. 开启 `showSearch`，利用内置的 `filterOption` 按 label 搜索过滤。
2. 如果选项超过 1000 条，考虑在 `onSearch` 中触发远程搜索。
3. 使用 `notFoundContent` 自定义搜索无结果的兜底内容。

### 如何处理单选/多选的回调类型差异？

- 单选/combobox 模式：`onValueChange` 返回 `string | undefined`。
- 多选/tags 模式：`onValueChange` 返回 `string[]`。

TypeScript 会根据 `mode` 属性自动推断返回类型。

### 如何自定义选项展示？

使用 `optionRender` 自定义下拉选项，使用 `labelRender` 自定义触发器中的标签显示。

```tsx
<Select
  options={data}
  optionRender={(option, { index }) => (
    <div className="flex items-center gap-2">
      <img src={option.avatar} className="size-5 rounded-full" />
      <span>{option.label}</span>
    </div>
  )}
/>
```

### 如何实现级联选择？

Select 本身不支持级联。请使用 Cascader（级联选择器）或 TreeSelect（树型选择器）。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/select` 导入，不要从 `antd` 导入。
2. 优先使用 `options` 声明式配置，而非 Legacy 子组件拼接。
3. 多选用 `mode="multiple"`；标签输入用 `mode="tags"`；搜索单选用 `mode="combobox"`。
4. `onValueChange` 回调：单选返回 `string | undefined`，多选/标签返回 `string[]`。
5. 大量选项时配合 `showSearch`，不要将所有选项平铺在 DOM 中。
6. 不要引入 Ant Design 相关依赖。
