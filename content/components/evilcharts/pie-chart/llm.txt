# EvilPieChart 可组合饼图

EvilPieChart 是 EvilCharts 图表库中的饼图组件。它采用可组合架构——`<Pie />`、`<Tooltip />`、`<Legend />`、`<Label />`、`<Background />` 作为子组件自由拼装，底层基于 Recharts 渲染，上层提供 EvilCharts 统一的 config、渐变色、发光、加载骨架等能力。目标是让调用侧只写业务配置，不接触 Recharts 底层细节。

## 导入路径

```tsx
// 主组件
import { EvilPieChart } from "@/components/charts/evilcharts"
// 子组件
import { Pie, Tooltip, Legend, Label, Background } from "@/components/charts/evilcharts/components/pie-chart"
// 类型
import { type ChartConfig } from "@/components/charts/evilcharts/components/chart"
```

## 何时使用 {#when-to-use}

使用 EvilPieChart 展示数据的占比分布。

常见场景：

- 浏览器、设备、来源等维度的占比对比。
- 预算分配、市场份额等百分比场景。
- 需要点击选择扇区以驱动外部状态或联动过滤的场景。
- 需要 loading 骨架屏在数据加载期间占位的场景。

不推荐：

- 数据项超过 7 个（优先考虑柱状图）。
- 数值差距过小难以肉眼区分的对比。
- 需要精确数值比较的场景。

EvilPieChart 核心概念：

- `config`：`ChartConfig` 类型，定义每个扇区的 `label` 和 `colors: { light: string[], dark: string[] }`。键名与数据行的 `nameKey` 字段值一一对应。
- `data` / `dataKey` / `nameKey`：数据行数组、数值字段名、名称字段名。
- 子组件自由组合：核心为 `<Pie />`（扇区）+ `<Tooltip />`（hover 提示）+ `<Legend />`（图例）；可选 `<Label />`（扇区标签）+ `<Background />`（背景纹理）。

### 环形图 + 标签 + 圆角扇区

### 发光扇区 + 选中回调

```tsx
import { useState } from "react"
import { EvilPieChart } from "@/components/charts/evilcharts"
import { Pie, Tooltip, Legend } from "@/components/charts/evilcharts/components/pie-chart"

export default function Demo() {
  const [selection, setSelection] = useState(null)

  return (
    <div className="space-y-2">
      <EvilPieChart
        className="h-full w-full p-4"
        data={data}
        dataKey="visitors"
        nameKey="browser"
        config={chartConfig}
        onSelectionChange={setSelection}
        defaultSelectedSector="chrome"
      >
        <Legend isClickable />
        <Tooltip />
        <Pie isClickable glowingSectors={["chrome"]} />
      </EvilPieChart>
      {selection && (
        <p className="text-sm text-muted-foreground">
          选中：{selection.dataKey} — {selection.value}
        </p>
      )}
    </div>
  )
}
```

## API {#api}

### EvilPieChart

饼图根容器，管理数据、选中状态和加载状态，为子组件提供共享上下文。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| config | 扇区颜色和标签配置 | `ChartConfig` | 必填 |
| data | 数据行 | `TData[]` | 必填 |
| dataKey | 数值字段 | `string` | 必填 |
| nameKey | 名称字段 | `string` | 必填 |
| children | 子组件组合 | `ReactNode` | 必填 |
| className | 容器附加样式 | `string` | - |
| chartProps | Recharts PieChart 透传属性 | `ComponentProps<typeof RechartsPieChart>` | - |
| defaultSelectedSector | 初始选中扇区名称 | `string \| null` | `null` |
| onSelectionChange | 选中变化回调 | `(selection: { dataKey: string; value: number } \| null) => void` | - |
| isLoading | 加载状态，显示脉冲骨架 | `boolean` | `false` |

### Pie

饼图扇区渲染，自动生成径向渐变色和发光滤镜。`isLoading` 时显示动画骨架。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 填充样式 | `gradient` | `gradient` |
| innerRadius | 内半径，大于 0 时为环形图 | `number \| string` | `0` |
| outerRadius | 外半径 | `number \| string` | `"80%"` |
| cornerRadius | 扇区圆角，单位 px | `number` | `0` |
| paddingAngle | 扇区间隙度数；负数则重叠 | `number` | `0` |
| startAngle | 起始绘制角度 | `number` | `0` |
| endAngle | 终止绘制角度 | `number` | `360` |
| isClickable | 扇区是否可点击选择 | `boolean` | `false` |
| glowingSectors | 需要发光效果的扇区名称列表 | `string[]` | `[]` |
| children | 可组合 `<Label />` | `ReactNode` | - |
| pieProps | Recharts Pie 透传属性 | `Omit<ComponentProps<typeof RechartsPie>, "data" \| "dataKey" \| "nameKey">` | - |

### Label

在 `<Pie />` 内声明扇区标签。自身不渲染 DOM，由父 `<Pie />` 解析并注入 Recharts LabelList。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| dataKey | 标签对应的数据字段 | `string` | 饼图的 `dataKey` |
| labelListProps | Recharts LabelList 透传属性 | `Omit<LabelListProps, "dataKey">` | - |

### Tooltip

Hover 悬浮提示。`isLoading` 时自动隐藏。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 提示框视觉样式 | `TooltipVariant` | - |
| roundness | 提示框圆角 | `TooltipRoundness` | - |
| defaultIndex | 默认显示的扇区索引 | `number` | - |

### Legend

扇区图例。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 图例标识样式 | `ChartLegendVariant` | - |
| align | 水平对齐 | `left` \| `center` \| `right` | `center` |
| verticalAlign | 垂直对齐 | `top` \| `middle` \| `bottom` | `bottom` |
| isClickable | 图例条目是否可点击切换选中 | `boolean` | `false` |

### Background

可选装饰背景，绘制在饼图后方。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 背景纹理样式 | `BackgroundVariant` | `dots` |

## 实现说明 {#implementation-notes}

核心文件：

- `src/components/evilcharts/components/evilcharts/charts/pie-chart.tsx` — 完整的组件实现：根容器、Pie、Tooltip、Legend、Label、Background、渐变色定义、发光滤镜、加载骨架。
- `src/components/evilcharts/components/pie-chart.ts` — 重导出入口。
- `src/components/ui/evilcharts-pie-chart.tsx` — 项目级封装，统一导入路径。

关键行为：

1. `EvilPieChart` 通过 React Context 管理 `config`、`data`、`dataKey`、`nameKey`、`isLoading`、`selectedSector` 等共享状态。
2. 每个 `<Pie />` 使用 `useId()` 生成唯一 ID，所有 SVG gradient 和 filter 都在该 ID 下定义，避免多实例冲突。
3. `config` 中 `colors.light` / `colors.dark` 通过 CSS 变量 (`--color-{key}-{index}`) 注入，组件自动生成 `<linearGradient>` 填充扇区。
4. `isClickable` 模式下，点击扇区设置 `selectedSector`；未选中扇区 `opacity: 0.3`；选中扇区再次点击取消选中。
5. `glowingSectors` 为每个指定的扇区生成 `<filter>` 包含 `feGaussianBlur` + `feColorMatrix` + `feMerge` 实现 SVG 外发光。
6. `isLoading` 时渲染 5 个等分扇区，使用 `motion/react` 实现逐个延迟的脉冲动画。Tooltip 自动隐藏，Legend 保留（但扇区无数据）。
7. `paddingAngle < 0` 时扇区启用 `stroke` 以清晰区分重叠区域。
8. 所有子组件（Pie、Tooltip、Legend、Background）通过 `usePieChart()` hook 读取共享上下文，不存在直接 prop drilling。

## FAQ {#faq}

### 如何做环形图？

设置 `<Pie />` 的 `innerRadius` 大于 0：

```tsx
<Pie isClickable innerRadius={60} />
```

### 如何选中扇区并联动外部状态？

```tsx
<EvilPieChart ... onSelectionChange={(s) => setSelection(s)}>
  <Pie isClickable />
</EvilPieChart>
```

### 如何高亮特定扇区？

```tsx
<Pie isClickable glowingSectors={["chrome"]} />
```

### 如何在扇区上显示数值？

在 `<Pie />` 内组合 `<Label />`：

```tsx
<Pie isClickable>
  <Label />
</Pie>
```

### 如何配置渐变色？

在 `config` 中每个扇区的 `colors` 数组传入多个色值。单色值时渲染纯色。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 EvilPieChart 代码时，应遵守：

1. `EvilPieChart` 从 `@/components/charts/evilcharts` 导入；`Pie`、`Tooltip`、`Legend`、`Label`、`Background` 从 `@/components/charts/evilcharts/components/pie-chart` 导入；`ChartConfig` 类型从 `@/components/charts/evilcharts/components/chart` 导入。
2. `config` 必须满足 `ChartConfig` 类型——每个键对应数据行中的名称值，包含 `label` 和 `colors: { light: string[], dark: string[] }`。
3. 至少组合 `<Pie />` + `<Tooltip />` + `<Legend />` 以获得完整的图表交互。
4. 需要点击选择时使用 `<Pie isClickable />` 和 `<Legend isClickable />`。
5. `innerRadius` 大于 0 时为环形图；`paddingAngle` 大于 0 时扇区有间隙；`cornerRadius` 大于 0 时扇区有圆角。
6. 加载状态使用 `isLoading`，不要在外部自行实现加载 UI。
7. 不要从 `recharts` 直接导入 `PieChart`、`Pie`、`Cell`、`Tooltip`、`Legend` 等——使用 EvilCharts 提供的可组合 API。
8. 不要引入 Ant Design Charts、BizCharts、ECharts 或其他图表库。
