# EvilBrush 缩放刷选控件

EvilBrush 是 EvilCharts 图表库中的缩放刷选控件。它在图表下方渲染一组缩略图，提供可拖拽的左右手柄和中间平移区域，用于在主图表中联动缩放可见数据区间。底层基于 Recharts 渲染缩略图，手柄交互使用 motion/react 的 spring 动画。

## 导入路径

```tsx
// 主组件
import { EvilBrush } from "@/components/ui/brush"
// Hook（管理刷选状态与数据切片）
import { useEvilBrush } from "@/components/charts/evilcharts/components/brush"
// 类型
import { type ChartConfig } from "@/components/charts/evilcharts/components/chart"
```

## 何时使用

- 时序数据浏览：大量数据中快速定位到特定区间。
- 联动图表：一个刷选控件控制多个图表的可见区间。
- 数据探索：拖拽手柄调整范围，拖拽中间区域平移窗口。

## Props Reference

### EvilBrush

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| data | `Record<string, unknown>[]` | required | Full dataset (mini chart always renders all data) |
| chartConfig | `ChartConfig` | required | Series color and label config |
| dataKeys | `string[]` | `Object.keys(chartConfig)` | Data keys to plot on the mini chart |
| xDataKey | `string` | - | X-axis key used for handle labels |
| variant | `"line" \| "area" \| "bar"` | `"area"` | Mini chart visual variant |
| height | `number` | `56` | Brush pixel height |
| className | `string` | - | Extra container classes |
| stacked | `boolean` | `false` | Stack areas/bars in the mini chart |
| strokeVariant | `"solid" \| "dashed" \| "animated-dashed"` | `"solid"` | Stroke style for mini chart |
| connectNulls | `boolean` | `false` | Connect null data points in line/area variants |
| barRadius | `number` | `3` | Bar corner radius for bar variant |
| startIndex | `number` | - | Controlled start index |
| endIndex | `number` | - | Controlled end index |
| defaultStartIndex | `number` | `0` | Initial start index (uncontrolled) |
| defaultEndIndex | `number` | `data.length - 1` | Initial end index (uncontrolled) |
| onChange | `(range: EvilBrushRange) => void` | - | Fired when visible range changes |
| formatLabel | `(value: unknown, index: number) => string` | - | Format handle label from xDataKey value |
| curveType | `CurveType` | `"monotone"` | Curve type for line/area variants |
| minSpan | `number` | `2` | Minimum number of data points that must remain selected |
| showLabels | `boolean` | `true` | Render labels on handles |
| skipStyle | `boolean` | `false` | Skip ChartStyle (when outer container already provides CSS vars) |

### useEvilBrush

Hook that manages brush state and returns sliced visible data.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| data | `TData[]` | required | Full dataset |
| defaultStartIndex | `number` | `0` | Initial start index |
| defaultEndIndex | `number` | `data.length - 1` | Initial end index |

**Return value:**

| Prop | Type | Description |
|------|------|-------------|
| range | `{ startIndex: number; endIndex: number }` | Current brush range |
| visibleData | `TData[]` | Sliced data from startIndex to endIndex (via `useDeferredValue` for smooth drag) |
| brushProps | `{ startIndex; endIndex; onChange }` | Controlled props to spread onto EvilBrush |

## Code Examples

### Basic: EvilBrush with EvilAreaChart

```tsx
import { EvilBrush } from "@/components/ui/brush"
import { useEvilBrush } from "@/components/charts/evilcharts/components/brush"
import { EvilAreaChart } from "@/components/ui/area-chart-evilcharts"
import { Area, XAxis, Grid, Tooltip } from "@/components/charts/evilcharts/components/area-chart"
import { type ChartConfig } from "@/components/charts/evilcharts/components/chart"

const chartConfig = {
  sales: {
    label: "销售额",
    colors: { light: ["#2563eb"], dark: ["#3b82f6"] },
  },
  orders: {
    label: "订单数",
    colors: { light: ["#d946ef"], dark: ["#e879f9"] },
  },
} satisfies ChartConfig

const data = Array.from({ length: 90 }, (_, i) => ({
  day: `${i + 1}日`,
  sales: Math.round(300 + Math.sin(i / 7) * 150 + Math.sin(i / 20) * 80),
  orders: Math.round(80 + Math.cos(i / 9) * 40 + Math.cos(i / 25) * 20),
}))

export default function Demo() {
  const { visibleData, brushProps } = useEvilBrush({ data })

  return (
    <div className="flex flex-col gap-2 h-full w-full p-4">
      <EvilAreaChart
        data={visibleData}
        config={chartConfig}
        className="flex-1 w-full"
        xDataKey="day"
      >
        <Grid />
        <XAxis dataKey="day" />
        <Tooltip />
        <Area dataKey="sales" variant="default" />
        <Area dataKey="orders" variant="default" />
      </EvilAreaChart>

      <EvilBrush
        data={data}
        chartConfig={chartConfig}
        dataKeys={["sales", "orders"]}
        xDataKey="day"
        variant="area"
        height={56}
        {...brushProps}
      />
    </div>
  )
}
```

### Controlled Mode

```tsx
import { useState } from "react"
import { EvilBrush } from "@/components/ui/brush"

export default function Demo() {
  const [range, setRange] = useState({ startIndex: 0, endIndex: 29 })
  const visibleData = data.slice(range.startIndex, range.endIndex + 1)

  return (
    <div className="flex flex-col gap-2">
      <EvilAreaChart data={visibleData} config={chartConfig}>
        {/* ... */}
      </EvilAreaChart>
      <EvilBrush
        data={data}
        chartConfig={chartConfig}
        startIndex={range.startIndex}
        endIndex={range.endIndex}
        onChange={setRange}
        variant="bar"
        minSpan={5}
      />
    </div>
  )
}
```

### Bar Variant with Custom Labels

```tsx
<EvilBrush
  data={data}
  chartConfig={chartConfig}
  dataKeys={["sales"]}
  xDataKey="day"
  variant="bar"
  barRadius={4}
  curvedType="linear"
  showLabels
  formatLabel={(v) => (v as string).replace("日", "")}
/>
```

## Constraints

1. `data` must be the full dataset — EvilBrush renders all data points in the mini chart.
2. `chartConfig` must satisfy `ChartConfig` — each key needs `label` and `colors: { light: string[], dark: string[] }`.
3. Every key in `dataKeys` must exist in both `data` rows and `chartConfig` object.
4. Use `useEvilBrush` to manage state — do not manually slice data with `Array.slice()` unless in controlled mode.
5. Spread `brushProps` onto EvilBrush — never pass them as individual props.
6. `minSpan` must be >= 2 to avoid degenerate zero-width selections.
7. When data length changes at runtime (e.g., new data loaded), the range resets to full span.
8. The mini chart uses `isAnimationActive={false}` to avoid animation jank during drag.
9. Drag updates use `React.startTransition` to defer parent re-renders — the brush handles stay responsive while the parent chart can lag by a frame.
10. Do NOT use Recharts' native `Brush` component — use EvilBrush instead.
