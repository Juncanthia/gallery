EvilRadarChart 用于展示多维数据在多个轴上的分布，以雷达图形状直观对比不同序列的覆盖范围。

<h2 id="when-to-use">何时使用</h2>

雷达图（Radar Chart）适合展示多变量数据在数个维度上的表现，便于对比不同系列的优劣势分布。

常见场景：

- 多维能力评估：技能雷达、能力模型对比。
- 竞品对比：多个产品的多维指标对比。
- 性能分析：系统各维度性能评分。
- 多维度评测：多候选方案的综合评分。

EvilRadarChart 采用组合式 API，通过 &lt;EvilRadarChart&gt; 作为容器，内部自由组合 &lt;Radar&gt;、&lt;PolarGrid&gt;、&lt;PolarAngleAxis&gt;、&lt;PolarRadiusAxis&gt;、&lt;Tooltip&gt;、&lt;Legend&gt; 等子组件。

<h2 id="faq">常见问题</h2>

### 如何配置系列颜色？

在 `config` 中为每个 dataKey 定义 `colors` 对象，至少包含一个主题键（`light` / `dark`）：

```tsx
const chartConfig = {
  desktop: {
    label: "Desktop",
    colors: { light: ["#3b82f6"], dark: ["#60a5fa"] },
  },
}
```

使用多色数组可生成渐变色：

```tsx
colors: { light: ["#6366f1", "#a855f7", "#ec4899"], dark: ["#818cf8", "#c084fc", "#f472b6"] }
```

### 如何实现点击选中系列？

为 &lt;Radar&gt; 添加 `isClickable`，配合 &lt;Legend isClickable&gt; 可同时通过图例切换选中：

```tsx
&lt;Radar dataKey="desktop" isClickable&gt;...&lt;/Radar&gt;
&lt;Legend isClickable /&gt;
```

点击已选中的系列会清除选择（恢复全部可见）。`onSelectionChange` 回调可监听选中变化。

### 如何处理加载状态？

设置 `isLoading={true}` 并将 `data` 设为空数组：

```tsx
&lt;EvilRadarChart data={isLoading ? [] : data} config={chartConfig} isLoading={isLoading}&gt;
  ...
&lt;/EvilRadarChart&gt;
```

加载骨架自动随机动画，可通过 `loadingPoints` 控制骨架数据点数。

### 如何自定义图表背景？

使用 `backgroundVariant` 属性添加背景图案：

```tsx
&lt;EvilRadarChart ... backgroundVariant="dots"&gt;
```

支持 11 种背景图案：`dots`、`grid`、`cross-hatch`、`diagonal-lines`、`plus`、`falling-triangles`、`4-pointed-star`、`tiny-checkers`、`overlapping-circles`、`wiggle-lines`、`bubbles`。

### 如何透传 Recharts 原生属性？

每个子组件都提供了透传 escape hatch：
- &lt;EvilRadarChart&gt; 的 `chartProps` 透传给 Recharts RadarChart
- &lt;Radar&gt; 的 `radarProps` 透传给 Recharts Radar
- &lt;PolarGrid&gt;、&lt;PolarAngleAxis&gt;、&lt;PolarRadiusAxis&gt; 直接接受所有 Recharts 对应组件的属性

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 EvilRadarChart 代码时：

1. 从 `@/components/ui/evilcharts/radar-chart` 导入所有子组件
2. `config` 必须包含每个 dataKey 的 `label` 和 `colors`（至少一个主题键 `light`/`dark`）
3. 使用组合式 API：&lt;EvilRadarChart&gt; 作为容器，内部自由组合子组件
4. 每个 &lt;Radar&gt; 必须指定 `dataKey`，该 key 必须同时存在于 `data` 和 `config` 中
5. 数据点标记使用 &lt;Dot&gt; 和 &lt;ActiveDot&gt; 组合在 &lt;Radar&gt; 内部
6. 所有子组件必须位于 &lt;EvilRadarChart&gt; 内部，否则会抛出错误
7. 加载时传递空数据数组 + `isLoading={true}`
8. 不应直接导入 Recharts 的 RadarChart、PolarGrid 等组件，使用 EvilCharts 封装
