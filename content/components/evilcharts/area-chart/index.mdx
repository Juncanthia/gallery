EvilAreaChart 是一个可组合的面积图组件，通过组合 &lt;Area /&gt;、&lt;XAxis /&gt;、&lt;Grid /&gt;、&lt;Tooltip /&gt;、&lt;Legend /&gt; 等子组件构建图表，支持多种填充变体、曲线类型、堆叠模式、进场动画与加载骨架屏。

<h2 id="when-to-use">何时使用</h2>

用于展示数值随时间或其他连续维度的变化趋势，强调数据总量与趋势面积。

常见场景：

- 趋势分析：展示单系列或多系列数据随时间的累积变化。
- 堆叠比较：比较各分类对总量贡献的同时展示总量趋势。
- 占比分布：`stackType="expanded"` 将面积转换为百分比视图。
- 范围交互：通过 `showBrush` 提供刷选工具，实现时间范围缩放。
- 加载占位：通过 `isLoading` 在数据加载期间展示骨架屏。

EvilAreaChart 采用 **组合模式**（Compound Pattern），图表本体提供数据、主题和上下文；子组件负责视觉呈现：

- `EvilAreaChart`：根容器，持有 data 和 config，管理选中、加载、刷选状态。
- `Area`：单个面积系列，自包含填充和描边样式定义。
- `XAxis` / `YAxis`：横纵坐标轴。
- `Grid`：背景网格线。
- `Tooltip`：悬停提示框。
- `Legend`：系列图例，支持点击交互。
- `Dot` / `ActiveDot`：数据点标记。

通过 `config` 属性传入 `ChartConfig` 对象，每个系列键对应一组深浅主题的颜色：

```tsx
const chartConfig = {
  desktop: {
    label: "Desktop",
    colors: { light: ["#047857"], dark: ["#10b981"] },
  },
} satisfies ChartConfig
```

多色数组时从左到右渐变分布。`ChartConfig` 使用 `satisfies` 确保类型安全。

### 如何实现系列选中交互？

为 `Area` 设置 `isClickable`，同时为 `Legend` 设置 `isClickable`：

```tsx
&lt;Legend isClickable /&gt;
&lt;Area dataKey="desktop" variant="gradient" isClickable&gt;...&lt;/Area&gt;
```

未选中的系列自动降低透明度并切换为条纹填充。

### 如何格式化坐标轴标签？

通过 `XAxis` 的 `tickFormatter` 实现：

```tsx
&lt;XAxis dataKey="month" tickFormatter={(value) => value.substring(0, 3)} /&gt;
```

`expanded` 模式下 Y 轴自动使用百分比格式化，无需手动指定。

### 如何添加刷选缩放？

设置 `showBrush` 并提供 `xDataKey`：

```tsx
&lt;EvilAreaChart data={data} config={chartConfig} showBrush xDataKey="month"&gt;...&lt;/EvilAreaChart&gt;
```

`brushFormatLabel` 可格式化刷选轴的标签文字。

### 如何展示加载状态？

设置 `isLoading`，同时传空 `data` 或保持原有数据：

```tsx
&lt;EvilAreaChart data={isLoading ? [] : data} config={chartConfig} isLoading={isLoading}&gt;...&lt;/EvilAreaChart&gt;
```

加载期间坐标轴、提示框和图例自动隐藏。`loadingPoints` 控制骨架屏的数据点数（默认 14）。

### 如何关闭进场动画？

将 `animationType` 设为 `"none"`：

```tsx
&lt;EvilAreaChart data={data} config={chartConfig} animationType="none"&gt;...&lt;/EvilAreaChart&gt;
```

系统开启"减弱动态效果"（prefers-reduced-motion）时自动回退为 `none`。

### 如何选择曲线类型？

图表级别通过 `curveType` 设置，所有 Area 继承。单个 Area 可通过自己的 `curveType` 覆盖：

```tsx
&lt;EvilAreaChart data={data} config={chartConfig} curveType="bump"&gt;
  &lt;Area dataKey="desktop" variant="gradient" curveType="step" /&gt;
&lt;/EvilAreaChart&gt;
```

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 EvilAreaChart 代码时：

1. 从 `@/components/ui/evilcharts-area-chart` 导入 `EvilAreaChart`
2. 从 `@/components/ui/evilcharts/components/area-chart` 导入 `Area`, `XAxis`, `YAxis`, `Grid`, `Tooltip`, `Legend`, `Dot`, `ActiveDot`
3. `config` 使用 `satisfies ChartConfig` 确保类型安全；每个系列必须提供 `light` 和 `dark` 颜色
4. `Area` 的 `dataKey` 必须同时存在于 `data` 和 `config` 中
5. 不要跳过坐标轴组件 — 至少需要 `XAxis`
6. 堆叠模式下自动共享 `stackId`，无需手动设置
7. 图表采用组合模式 — 所有视觉元素作为 `EvilAreaChart` 的子组件使用
8. 加载状态传递空数组或原数据均可，`isLoading` 控制骨架屏显示
9. 不要引入 Recharts 原生组件，使用 EvilCharts 的组合 API
10. 不要手动设置 `isAnimationActive` — 组件内部已关闭 Recharts 动画并用 motion.dev 替代
