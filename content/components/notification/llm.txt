# Notification 通知提醒

全局通知提醒 API，在屏幕角落展示提示信息。基于 `sonner` 实现，提供与 Ant Design `notification` 类似的命令式调用 API：`notification.success()`、`notification.info()`、`notification.warning()`、`notification.error()`。

与 AntD 不同，本地 `notification` 不包含 `useNotification` hooks 和 `ConfigProvider` 全局配置；而是直接通过 `toaster` 组件订阅 sonner 实例来控制系统级配置。

## 何时使用 {#when-to-use}

当需要向用户推送系统级通知消息时使用。通知从屏幕角落弹出，不打断用户当前操作。

常见场景：

- 操作成功后的反馈提示（如"保存成功"）。
- 系统级警告或错误通知。
- 需要自动消失的临时消息。
- 跨路由的全局消息推送。

不推荐：

- 需要用户明确交互确认的提示——使用 `Alert`、`Dialog` 或 `Popconfirm`。
- 与当前页面上下文强相关的内联提示——使用 `Alert` 的 banner 模式。
- 大量频繁推送消息——会导致通知堆积影响体验。
- 在 `toaster` 组件未挂载时调用——通知将无法渲染。

## 代码演示 {#examples}

### 基础用法

命令式调用，无需组件声明。

```tsx
import { notification } from "@/components/ui/notification"

export default function Demo() {
  return (
    <div className="flex flex-wrap gap-2">
      <button onClick={() => notification.success({ title: "操作成功", description: "数据已保存" })}>
        成功通知
      </button>
      <button onClick={() => notification.info({ title: "提示", description: "这是一条信息" })}>
        信息通知
      </button>
      <button onClick={() => notification.warning({ title: "警告", description: "请注意检查" })}>
        警告通知
      </button>
      <button onClick={() => notification.error({ title: "错误", description: "操作失败，请重试" })}>
        错误通知
      </button>
    </div>
  )
}
```

### 带操作与配置

```tsx
import { notification } from "@/components/ui/notification"

export default function Demo() {
  return (
    <button
      onClick={() =>
        notification.success({
          title: "文件已导出",
          description: "报表已生成，点击下载",
          duration: 0,
          placement: "bottomRight",
          onClick: () => window.open("/download"),
        })
      }
    >
      导出并通知
    </button>
  )
}
```

## API {#api}

### notification

命令式 API 对象，无需组件声明。

| 方法 | 说明 | 签名 |
| --- | --- | --- |
| `notification.success(args)` | 成功通知 | `(args: NotificationArgs) => NotificationClose` |
| `notification.info(args)` | 信息通知 | `(args: NotificationArgs) => NotificationClose` |
| `notification.warning(args)` | 警告通知 | `(args: NotificationArgs) => NotificationClose` |
| `notification.error(args)` | 错误通知 | `(args: NotificationArgs) => NotificationClose` |
| `notification.open(args)` | 通用通知（不指定类型） | `(args: NotificationArgs) => NotificationClose` |
| `notification.destroy(key?)` | 关闭指定通知（不传 key 则关闭全部） | `(key?: string \| number) => void` |

### NotificationArgs

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| title | 通知标题（必填） | `React.ReactNode` | - |
| description | 通知内容描述 | `React.ReactNode` | - |
| type | 通知类型 | `"success" \| "info" \| "warning" \| "error"` | `"info"` |
| key | 唯一标识，用于后续 update/destroy | `string \| number` | - |
| duration | 自动关闭时间（秒）。0 表示不自动关闭 | `number` | `4.5` |
| placement | 屏幕位置 | `"top" \| "topLeft" \| "topRight" \| "bottom" \| "bottomLeft" \| "bottomRight"` | `"topRight"` |
| closable | 是否显示关闭按钮 | `boolean` | `true` |
| closeIcon | 自定义关闭图标 | `React.ReactNode` | `✕` |
| icon | 自定义通知图标 | `React.ReactNode` | 根据 type 自动选择 |
| onClose | 通知关闭回调 | `() => void` | - |
| onClick | 通知点击回调 | `() => void` | - |
| className | 附加样式类名 | `string` | - |
| style | 附加内联样式 | `React.CSSProperties` | - |

### NotificationClose

返回值是关闭函数：`() => void`。调用后关闭该通知。

## 样式模型 {#styling-model}

- 使用 Tailwind CSS，通知内容布局为 `flex gap-3`。
- 标题：`text-sm font-medium text-foreground`。
- 描述：`text-sm text-muted-foreground mt-1`。
- 关闭按钮：`text-muted-foreground opacity-70 hover:opacity-100`。
- 通知外观（颜色、圆角、阴影等）由 sonner 的默认样式和 `toaster` 组件配置控制。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/ui/notification-api.tsx`

依赖：

- `sonner` — 底层 toast 库，提供 `toast` 方法及类型变体。
- `@/components/ui/toaster` — sonner 的 `Toaster` 组件，控制系统级配置。

实现注意：

1. `notification` 是纯命令式 API，不依赖 React Context 或 Provider。
2. `duration` 为 0 时，通知不会自动关闭（内部设为 `Infinity`）。
3. `placement` 映射：`top`→`top-center`、`bottom`→`bottom-center`，其余与 sonner position 对应。
4. 每个方法返回 `NotificationClose` 函数，调用即可关闭该通知。
5. 需要确保页面中挂载了 `<Toaster />`（来自 `@/components/ui/toaster`），否则通知不可见。

## 可访问性 {#accessibility}

- sonner 内置 ARIA 通知角色和屏幕阅读器播报。
- 关闭按钮设置 `aria-label="Close"`。
- 通知自动消失的 duration 默认 4.5 秒，给用户足够阅读时间。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| 底层实现 | sonner（Svelte/React toast 库） |
| API 风格 | 命令式 `notification.success()`，与 AntD 一致 |
| useNotification hooks | 不做——本地不需要 hooks 模式 |
| ConfigProvider | 不做——通过 `<Toaster />` 组件配置 |
| 关闭所有通知 | `notification.destroy()`（不传 key），与 AntD 一致 |
| 语义 DOM 插槽 | 不做——本地通过 className/style 自定义 |
| Design Token | 不做——使用 Tailwind 主题变量 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目通知相关代码时，应遵守：

1. 从 `@/components/ui/notification` 导入 `notification` 对象。
2. 使用 `notification.success/info/warning/error` 方法，传入 `NotificationArgs` 对象。
3. 确保页面中已挂载 `<Toaster />`（来自 `@/components/ui/toaster`）。
4. `title` 为必填属性，`description` 为可选。
5. 需要不自动关闭的通知，设置 `duration: 0`。
6. 不要引入 `antd` 的 `notification` 或 `App.useApp().notification`。
7. 不要使用 `useNotification` hooks——本地不支持。
8. 自定义样式使用 `className` 和 `style`，不要依赖 AntD 的 `classNames` / `styles` 对象 API。
