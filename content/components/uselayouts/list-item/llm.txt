# ListItem 列表项筛选器

ListItem 是 UseLayouts 系列布局组件，提供一个带动画的过滤选择组件。

本组件是自定义设计实现，不基于 Ant Design。使用 motion/react 驱动面板展开动画和选项入场动画，hugeicons 提供图标。

## 何时使用 {#when-to-use}

适合需要紧凑筛选器入口的场景，点击触发按钮展开选项列表供用户单选。选择后自动收起面板，选中项带对勾标记。

常见场景：

- 任务列表按类型筛选。
- 工具栏快捷过滤入口。
- 移动端或桌面端分类筛选控件。

不推荐：

- 多选场景，应使用 Checkbox 或 Select `mode="multiple"`。
- 选项过多需要搜索，应使用 Select 或 Combobox。
- 作为主导航控件。

## API 摘要 {#api}

ListItem 为零 props 组件，内部管理所有状态和筛选配置。

内部状态：

- `selectedFilterKey: FilterKey` — 当前选中的筛选项，默认为 `filterKeys[0]`。
- `isOpened: boolean` — 选项面板是否展开。

筛选选项通过 `filterKeys` 常量定义，修改时直接编辑源码：

- `filterKeys` — `{ name: string; Icon: ({ size }: { size: number }) => ReactNode }[]`，默认包含 tasks、events、reminders、appointments、meetings、celebrations 六项。

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- 组件未声明 `data-slot`，使用组件 DOM 结构和 Tailwind class 定位。
- 触发按钮容器：`relative left-2.5 w-20 h-20`，包含 FilterHorizontalIcon 和 layoutId="wrapper" 的 motion.div。
- 选中图标容器：`relative right-2.5 w-20 h-20 border border-border rounded-full`。
- 选项面板：`absolute z-20 w-72 px-1 py-1 bg-card border border-border rounded-[20px]`。
- 选项条目：`px-3 py-2 rounded-2xl flex justify-between items-center`，带 `hover:bg-accent`。
- 选中标记：`w-6 h-6 rounded-full`，选中时显示 `bg-primary` 背景 + Tick02Icon 对勾。

## 样式模型 {#styling-model}

- 样式由 Tailwind CSS className 直接控制，不使用 CVA。
- 面板展开动画由 motion/react 的 `layoutId` 共享布局动画驱动。
- 选项入场动画使用 motion.div 的 `initial`/`animate`，带递增 `delay`（公式 `(index + 8) * 0.025`）。
- 主题颜色使用 `bg-background`、`text-foreground`、`text-muted-foreground`、`bg-accent`、`border-border`、`bg-primary`、`text-primary-foreground` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/uselayouts/list-item.tsx`
- `src/components/ui/uselayouts-list-item.tsx`

直接依赖：

- `motion/react` — 面板和选项动画。
- `clsx` — 条件 className 拼接。
- `@hugeicons/core-free-icons` — 图标库。
- `@hugeicons/react` — React 图标组件。

实现注意：

1. 组件使用 `MotionConfig` 统一动画配置，`transition={{ type: "spring", duration: 0.85, bounce: 0.35 }}`。
2. 面板展开使用 `layoutId="wrapper"`，触发按钮和展开面板共享同一 layoutId，实现平滑过渡。
3. 选中项通过 `filterKey.name` 字符串比较判断，非引用比较，支持从 `filterKeys` 数组中安全查找。
4. 选项入场 delay 使用 `(index + 8) * 0.025` 公式，确保第一个选项有 0.2s 的基础延迟。
5. 选择后通过 `setTimeout` 延时 150ms 关闭面板，确保用户能看到选中反馈。

## 可访问性 {#accessibility}

- 触发按钮和选项条目为 `<div>` 元素，具备 `cursor-default` 样式提示可交互。
- 当前未使用 `<button>` 元素，无法通过键盘聚焦或操作。
- 建议扩展时补充 `role="button"`、`tabIndex={0}`、`aria-pressed` 和键盘事件（Enter/Space）支持。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 自定义设计，AntD 无对应组件 |
| 视觉系统 | Tailwind CSS + motion/react 动画 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 css-in-js |
| 图标系统 | 使用 hugeicons，不使用 @ant-design/icons |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 ListItem 代码时，应遵守：

1. 从 `@/components/ui/uselayouts-list-item` 导入 `ListItem`（named export）。
2. ListItem 为零 props 组件，直接渲染即可。
3. 如需定制筛选项，修改 `src/components/uselayouts/list-item.tsx` 中的 `filterKeys` 数组。
4. 图标使用 `@hugeicons/core-free-icons`，不要使用 `@ant-design/icons`。
5. 不要引入 `antd`、`antd-style` 或 rc-component。
