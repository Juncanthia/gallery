# FolderInteraction 文件夹交互

文件夹交互是一个点击展开动画组件，通过点击文件夹区域触发生动的页面分层弹出效果，展示多层页面从文件夹中展开和收回的过渡动画。

本组件是 UseLayouts 集合中的交互展示组件，使用 motion/react 驱动弹性动画。

## 何时使用 {#when-to-use}

当需要以文件夹形式展示内容分层，通过点击切换展开/闭合状态时使用。组件提供三层页面同时弹出的弹性过渡效果。

常见场景：

- 文件管理类界面，用文件夹动画增强操作反馈。
- 内容分层展示，通过展开动画揭示隐藏信息。
- 创意着陆页，用交互式文件夹吸引用户注意力。

不推荐：

- 需要展示实际文件内容的场景（本组件为视觉演示型，页面层内为占位线条）。
- 需要自定义页面内容的场景，应直接修改组件源码中的 Page 子组件。

## 代码演示 {#examples}

### 基础用法

组件无需任何 props，点击文件夹区域即可切换展开和闭合状态。

```tsx
import { FolderInteraction } from "@/components/ui/uselayouts-folder-interaction"

export default function Demo() {
  return <FolderInteraction />
}
```

## 核心导出 {#exports}

- `FolderInteraction` (named export)

## API 摘要 {#api}

组件为自包含交互组件，无需传递任何 props。

```ts
// 组件内部通过 useState 管理 isOpen 状态
// 点击 wrapper 区域 toggle 展开/闭合
```

所有交互行为（点击切换、弹性动画）均由组件内部的 `useState` 和 `motion/react` 管理。

## Semantic DOM {#semantic-dom}

源码中未声明 `data-slot`，使用根元素语义和组件名定位。

## 样式模型 {#styling-model}

- 样式通过 Tailwind CSS className 和内联 style 共同表达。
- 文件夹封套使用 SVG 路径绘制，配合 `linearGradient` 和 `clipPath` 实现毛玻璃效果（`backdropFilter: blur(6.5px)`）。
- 三层页面使用 `motion.div` 驱动弹性动画，每层预设不同的初始位置和展开位置：
  - 第一层：初始 `rotate: -3, x: -38, y: 2`，展开 `rotate: -8, x: -70, y: -55`
  - 第二层（中间层）：初始 `rotate: 0, x: 0, y: 0`，展开 `rotate: 1, x: 2, y: -75`
  - 第三层：初始 `rotate: 3.5, x: 42, y: 1`，展开 `rotate: 9, x: 75, y: -60`
- 文件夹前盖通过 `rotateX` 旋转实现翻开效果（`-40deg`）。
- 颜色使用硬编码值（`#18151B`、`#2D2535`、`#E8E7F0`），不依赖项目主题变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/uselayouts/folder-interaction.tsx`
- `src/components/ui/uselayouts-folder-interaction.tsx`（re-export）

实现注意：

1. 组件内部定义 `Page` 子组件，渲染 8 层占位线条模拟文档内容。
2. 三层页面使用 `motion/react` 的 `spring` 类型过渡，各自独立配置 `stiffness`、`damping`、`bounce`。
3. 点击 `.wrapper` 触发 `setIsOpen(!isOpen)` 切换状态。
4. 文件夹容器宽度 `w-80`、高度 `h-52`，页面层宽 `w-87.5%`。
5. 展开后页面层 z-index：`10`（两侧）、`20`（中间层）。

## 可访问性 {#accessibility}

- 组件为纯视觉交互组件，点击区域无键盘可操作性。
- 如果需要在生产环境使用，建议补充 `role="button"`、`tabIndex={0}` 和键盘事件处理。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 FolderInteraction 代码时，应遵守：

1. 从 `@/components/ui/uselayouts-folder-interaction` 导入 `FolderInteraction`（named export）。
2. 组件无需 props，直接渲染即可获得完整的文件夹交互效果。
3. 如需自定义页面内容，直接修改 `src/components/uselayouts/folder-interaction.tsx` 中的 `Page` 子组件。
4. 如需调整动画参数，修改各层的 `initial`、`open`、`transition` 配置。
5. 不要引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。
