# TwentyTwelveOne 平滑下拉菜单

TwentyTwelveOne 是 UseLayouts 系列布局组件，提供一个带动画的弹出式下拉菜单。

本组件是自定义设计实现，不基于 Ant Design。使用 motion/react 驱动展开/收起动画和活跃指示器过渡，hugeicons 提供图标，react-use-measure 测量内容高度。

## 何时使用 {#when-to-use}

适合需要紧凑弹出式操作菜单的场景。点击三点图标触发展开，支持鼠标悬停高亮和活跃项指示器。

常见场景：

- 用户头像旁边的更多操作入口。
- 卡片右上角的上下文菜单。
- 工具栏中的快捷操作面板。

不推荐：

- 用于长列表导航。
- 菜单项超过 10 个。
- 在已有弹出层的场景中重复叠加。

## API 摘要 {#api}

TwentyTwelveOne 为零 props 组件，内部管理展开状态、活跃项和悬停状态。

内部状态：

- `isOpen: boolean` — 菜单是否展开，默认 `false`。
- `activeItem: string` — 当前活跃的菜单项 ID，默认 `"profile"`。
- `hoveredItem: string | null` — 当前悬停的菜单项 ID，默认 `null`。

菜单项通过内部 `menuItems` 数组定义，修改时直接编辑源码：

```ts
const menuItems = [
  { id: "profile", label: "Profile", icon: UserIcon },
  { id: "upgrade", label: "Upgrade", icon: CreditCardIcon },
  { id: "projects", label: "Projects", icon: FolderIcon },
  { id: "documentation", label: "Documentation", icon: File01Icon },
  { id: "divider", label: "", icon: null },
  { id: "settings", label: "Settings", icon: SettingsIcon },
  { id: "help", label: "Get Help", icon: HelpCircleIcon },
  { id: "logout", label: "Logout", icon: LogoutIcon },
];
```

- `id: "divider"` 项渲染为分隔线，不需要 icon 和 label。
- `id: "logout"` 项点击后会关闭菜单，且样式使用红色主题。
- 活跃指示器使用 motion `layoutId` 实现平滑过渡。

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- 触发器容器：`relative h-10 w-10`，绝对定位在父容器右上角。
- 弹出面板：`bg-popover border border-border shadow-lg overflow-hidden`，通过 motion 控制圆角从 12px 到 14px。
- 菜单项容器：`p-2`，内部为 `<ul>` 列表。
- 分隔线：`<hr>` 元素，`border-border my-1.5`。
- 活跃指示器：`motion.div` 使用 `layoutId="activeIndicator"` 和 `layoutId="leftBar"` 实现联动动画。
- Logout 样式：文本 `text-red-600`，背景 `bg-red-50`，左侧指示器 `bg-red-500`。

## 样式模型 {#styling-model}

- 样式由 Tailwind CSS className 直接控制，不使用 CVA。
- 容器展开宽度 220px，收起宽度 40px，高度 40px。
- 动画由 motion/react 的 `motion.div` + `layout` prop 驱动。
- 展开/收起使用 spring 动画（damping: 34, stiffness: 380, mass: 0.8）。
- 菜单项入场使用 `easeOutQuint` 缓动曲线 `[0.23, 1, 0.32, 1]`。
- 内容高度测量使用 `react-use-measure`。
- 外部点击检测使用原生 `mousedown` 事件监听。
- 主题颜色使用 `bg-popover`、`text-foreground`、`text-muted-foreground`、`bg-muted`、`border-border` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/uselayouts/smooth-dropdown.tsx`
- `src/components/ui/uselayouts-smooth-dropdown.tsx`

直接依赖：

- `motion/react` — 展开/收起动画，活跃指示器过渡。
- `react-use-measure` — 内容高度测量。
- `@hugeicons/core-free-icons` — 图标库。
- `@hugeicons/react` — React 图标组件。

实现注意：

1. 组件为零 props 设计，所有配置通过修改源码 `menuItems` 数组完成。
2. 点击外部区域通过容器 ref + `mousedown` 事件监听实现。
3. 内容高度通过 `useMeasure` 动态测量，确保展开动画高度自适应。
4. 活跃指示器使用 motion `layoutId` 实现联动过渡（背景 + 左侧竖条）。
5. 菜单项入场使用错开延迟（`0.06 + index * 0.02`），营造瀑布流效果。

## 可访问性 {#accessibility}

- 触发器为 `<motion.div>`，未使用原生 `<button>`，不具备键盘焦点语义。
- 当前未提供 `aria-expanded`、`aria-haspopup` 等 ARIA 属性。
- 菜单项为 `<motion.li>`，非 `<button>` 元素，键盘操作受限。
- 建议在实际使用中补充键盘导航和 ARIA 属性。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 自定义设计，AntD Dropdown 功能更丰富但样式定制受限 |
| 视觉系统 | Tailwind CSS + motion/react 动画 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 css-in-js |
| 图标系统 | 使用 hugeicons，不使用 @ant-design/icons |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 TwentyTwelveOne 代码时，应遵守：

1. 从 `@/components/navigation/uselayouts/smooth-dropdown` 导入 TwentyTwelveOne。
2. TwentyTwelveOne 为零 props 组件，直接渲染即可。
3. 如需定制菜单项，修改 `src/components/uselayouts/smooth-dropdown.tsx` 中的 `menuItems` 数组。
4. 图标使用 `@hugeicons/core-free-icons`，不要使用 `@ant-design/icons`。
5. 不要引入 `antd`、`antd-style` 或 rc-component。
