EmptyTestimonial 是一个带有动画的空状态推荐组件，展示文件夹翻开动画和添加推荐引导，适合在无数据时引导用户参与互动。

<h2 id="when-to-use">何时使用</h2>

当页面暂无 testimonial 数据时使用，通过生动的文件夹翻开动画和引导按钮鼓励用户添加第一条推荐内容。

常见场景：

- 推荐墙/评价页面无数据时的空状态占位。
- 用户反馈列表为空时的引导入口。
- UGC 内容区（评论、留言等）的数据初始空态。

组件内部管理两种交互状态：

- **悬停动画**：光标悬停在 "Love" 标签上时触发心形图标的弹性脉冲动画。
- **文件夹翻开**：点击文件夹容器后，三张内页卡片以 spring 动画向外飞出，封面保持原位，模拟文件夹打开效果。再次点击关闭。

不推荐：

- 用于已有数据的列表下方展示（非空状态场景）。
- 替代标准的 Empty 组件用于一般性无数据提示（本组件具有特定的 animation 和 testimonial 语境）。

<h2 id="api">API</h2>

### EmptyTestimonial

该组件为自包含交互组件，无需传递任何 props。所有交互行为（悬停检测、文件夹翻开/关闭、spring 动画）均由组件内部管理。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| - | 无需传递任何 props | - | - |

内部状态：

| 状态 | 类型 | 说明 |
| --- | --- | --- |
| isOpen | `boolean` | 文件夹是否处于翻开状态 |
| isHovered | `boolean` | 光标是否悬停在 "Love" 标签上 |

视觉结构：

- 标题区：包含 "Wall of" 文字和带心形图标的 "Love" 标签，支持 hover 动画。
- 文件夹容器：SVG 绘制的文件夹封面和背面，内有三张可翻开的页面卡片（motion.div），点击切换开合。
- 空状态文本：显示 "Oops..There are no testimonials" 和 "Be the first one to add a testimonial" 提示。
- CTA 按钮：链接至 Twitter intent，引导用户添加推荐内容。

<h2 id="faq">常见问题</h2>

### 如何修改推荐提交的目标链接？

修改源码中 `&lt;a&gt;` 标签的 `href` 属性：

```tsx
href="https://twitter.com/intent/tweet?text=..."
```

替换为你自己的提交入口 URL 即可。

### 如何修改空状态文案？

修改源码中的两处文本：

- `&lt;p&gt;` 标签内的 `Oops..There are no testimonials` 和 `Be the first one to add a testimonial`。
- `&lt;span&gt;` 标签内的按钮文字 `Add Testimonial`。

### 如何调整文件夹动画参数？

修改源码中 `pages` 数组中每页的 `open`、`initial` 和 `transition` 配置。调整 `stiffness`（刚度）和 `damping`（阻尼）可改变 spring 动画的速度和弹性。

### 如何在有数据时替换此组件？

在业务代码中使用条件渲染：

```tsx
{testimonials.length === 0 ? <EmptyTestimonial /> : <TestimonialList />}
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/uselayouts-empty-testimonial` 导入 `EmptyTestimonial`（default export）。
2. 组件无需任何 props，直接渲染即可获得完整的空状态交互体验。
3. 组件内部使用 `motion/react` 和 `@hugeicons/react`，不要在外部重复引入。
4. 组件为全宽 (`w-full`) 布局，内边距为 `py-24 px-4`，如需调整外观通过 `className` 覆写外层容器。
5. 不要引入 Ant Design 相关依赖。
