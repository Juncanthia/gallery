# VerticalTabs 垂直标签

VerticalTabs 是一个垂直标签页切换组件，左侧为服务标签列表（含自动播放进度条），右侧为对应图片卡片的垂直滑动切换动画。它是一个无 props 的自包含组件，内置 3 个服务展示项。

本组件来自 UseLayouts 集合，使用 motion/react 驱动卡片垂直切换动画，@hugeicons 提供导航图标，风格现代。

## 何时使用 {#when-to-use}

当需要以视觉化方式展示服务项目或内容分类时使用，尤其适合：

- 个人作品集或服务展示页面。
- SaaS 服务的能力介绍板块。
- 需要自动轮播和手动选择结合的内容切换场景。
- 图文搭配的垂直标签页布局。

VerticalTabs 的核心交互：

- **自动轮播**：默认每 5 秒自动切换到下一个服务项。
- **手动选择**：点击左侧任意标签，卡片和描述立即切换到对应项。
- **悬停暂停**：鼠标悬停在图片卡片上时暂停自动轮播，移开后恢复。
- **进度指示**：左侧激活标签有垂直进度条动画，直观展示当前项播放进度。
- **卡片动画**：motion 驱动的垂直滑动切换（spring 动画），方向感知（向上/向下滑动匹配操作方向）。
- **导航按钮**：图片卡片区域底部有前/后切换按钮，支持手动浏览。

不推荐：

- 服务项目数量过少（&lt; 3 项）的场景，轮播效果不明显。
- 需要展示大量文字描述的场景（每项 description 建议控制在 50 字以内）。
- 移动端首屏核心区域（组件设计偏向桌面布局，移动端体验可能受限）。

## 代码演示 {#examples}

### 基础用法

VerticalTabs 无需传入任何 props，直接渲染即可获得完整的垂直标签页面板。

```tsx
import { VerticalTabs } from "@/components/ui/vertical-tabs"

export default function Demo() {
  return <VerticalTabs />
}
```

## API {#api}

VerticalTabs 为自包含组件，不接受外部 props。所有内部状态（activeIndex、direction、isPaused 等）由组件自行管理。

内部关键常量：

| 常量 | 说明 | 默认值 |
| --- | --- | --- |
| SERVICES | 服务数据数组，每项含 id、title、description、image | 3 项预设数据（Web Design / Framer Development / Branding） |
| AUTO_PLAY_DURATION | 自动播放间隔（毫秒） | 5000 |

如需扩展，在 `src/components/uselayouts/vertical-tabs.tsx` 中修改以上常量和内部函数即可。

## Semantic DOM {#semantic-dom}

VerticalTabs 未使用 `data-slot` 语义属性暴露结构。组件由 `section`、`button`、`motion.div` 和 `img` 元素组合而成。标签按钮和图片卡片通过 React state（activeIndex）和 motion 的 animate 属性实现联动，不依赖语义标记。

## 样式模型 {#styling-model}

VerticalTabs 的所有样式直接通过 Tailwind CSS class 在组件内部定义，不依赖 CVA 或外部变体文件。

核心样式策略：

- 外层容器：`w-full bg-background py-8 md:py-16 lg:py-24`，全宽背景。
- 内容区域：`grid grid-cols-1 lg:grid-cols-12 gap-12 lg:gap-16`，桌面端 12 列网格布局。
- 左侧标签区：`lg:col-span-5`，标签按钮垂直排列，每项之间有顶部分割线。
- 标签按钮：左侧有竖向进度条指示器（`motion.div`，height 动画从 0% 到 100%）。
- 标签文字：激活态 `text-foreground`，非激活态 `text-muted-foreground/60 hover:text-foreground`。
- 描述文字：通过 `AnimatePresence` 实现展开/折叠动画（opacity + height）。
- 右侧图片区：`lg:col-span-7`，圆角容器 `rounded-3xl md:rounded-[2.5rem]`，aspect 比例。
- 图片动画：motion.div 通过 `y` 属性垂直滑动，spring 动画（stiffness: 260, damping: 32）。
- 导航按钮：`rounded-full` 圆形，玻璃态 `bg-background/80 backdrop-blur-md`，绝对定位在图片区右下角。
- 图片遮罩：`bg-linear-to-t from-black/20`，底部渐变遮罩。

## 主题变量 {#design-token}

VerticalTabs 消费 Tailwind 主题变量和语义色 class。所有颜色通过 Tailwind class 引用，不依赖 css-in-js 或 ConfigProvider。

常用变量来源：

| 语义 | Tailwind / CSS token |
| --- | --- |
| 页面背景 | `bg-background` |
| 前景文本 | `text-foreground` |
| 次级文本 | `text-muted-foreground` |
| 次级文本降低透明度 | `text-muted-foreground/60` |
| 边框 | `border-border` / `border-border/40` / `border-border/50` |
| 次级背景 | `bg-muted` / `bg-muted/30` |

## 可访问性 {#accessibility}

- 标签按钮使用原生 `<button>` 元素，支持键盘导航和点击。
- 标签按钮有明确的 hover、激活和非激活视觉状态。
- 导航按钮有 `aria-label`（Previous / Next）。
- `onMouseEnter` / `onMouseLeave` 控制自动播放暂停和恢复。
- 当前版本标签按钮未提供 ARIA 标签，如需生产环境使用建议补充。
- 图片使用 `alt` 属性，提供基本的图像替代文本。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/uselayouts/vertical-tabs.tsx`：组件主体，包含 VerticalTabs 函数、SERVICES 数据、动画逻辑。
- `src/components/ui/uselayouts-vertical-tabs.tsx`：re-export 桥接文件，将 default export 转为命名 export `VerticalTabs`。

关键行为：

1. VerticalTabs 不接受外部 props，初始 activeIndex 为 0。
2. `activeIndex` 驱动标签和图片切换，通过 `handleNext` / `handlePrev` / `handleTabClick` 控制。
3. `direction`（1 或 -1）记录切换方向，用于 motion 的方向感知动画。
4. 自动播放通过 `useEffect` + `setInterval` 实现，`isPaused` 控制暂停。
5. 进度条动画通过 `motion.div` 的 `height` 属性从 0% 到 100%，时长与 `AUTO_PLAY_DURATION` 同步。
6. 图片切换动画使用 motion 的 `variants`（enter/center/exit），y 轴方向由 `direction` 决定。
7. 导航箭头图标使用 @hugeicons/core-free-icons（ArrowLeft01Icon / ArrowRight01Icon），通过 HugeiconsIcon 组件渲染。
8. 图片使用 Unsplash 外链，生产环境建议替换为本地或 CDN 资源。

## FAQ {#faq}

### 如何修改标签页的服务内容？

在 `src/components/uselayouts/vertical-tabs.tsx` 中修改 `SERVICES` 数组。每项需要提供 `id`（唯一标识）、`title`（显示名称）、`description`（描述文字）和 `image`（图片 URL）。数组长度可任意调整，组件会自动适配。

### 如何调整自动播放速度？

修改 `AUTO_PLAY_DURATION` 常量（单位毫秒）。默认 5000（5 秒）。设为较大值可减慢播放，设为较小值可加快。如需禁用自动播放，可将该值设为 `Infinity` 或注释掉 `useEffect` 中的 `setInterval` 逻辑。

### 如何禁用自动播放？

将 `AUTO_PLAY_DURATION` 设为 `Infinity` 或极大值，或移除 `useEffect` 中的 `setInterval` 逻辑。手动点击标签或导航按钮仍然可用。

### VerticalTabs 可以展示多少项服务？

`SERVICES` 数组长度无硬限制，但建议 3-8 项。过多（&gt; 10 项）左侧标签区域会变得拥挤，需要调整布局或标签间距来适配。

### VerticalTabs 的图片需要什么尺寸？

组件内部图片使用 `object-cover` 填充，容器宽高比在移动端为 `aspect-4/5`，平板端 `aspect-4/3`，桌面端 `aspect-16/11`。建议使用 16:11 或 4:3 比例的图片以获得最佳显示效果。生产环境建议使用本地或 CDN 图片替换 Unsplash 外链。

### VerticalTabs 适合移动端吗？

组件内置响应式布局（`lg:` 断点），移动端会自动切换为上下布局（标签列表在上，图片卡片在下）。导航按钮和图片卡片在移动端仍然可用，但由于 12 列网格在窄屏退化为单列，建议在 `>=1024px` 宽度下使用以获得最佳体验。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 VerticalTabs 相关代码时，应遵守：

1. 从 `@/components/ui/vertical-tabs` 导入 `VerticalTabs`（named export）。
2. 不要给 VerticalTabs 传递任何 props，组件自身管理全部状态。
3. 如需引入其他依赖（motion、@hugeicons），从组件源码处参考正确的导入路径。
4. VerticalTabs 依赖 motion/react 的 AnimatePresence 和 spring 动画，不要移除这些动画逻辑。
5. 图标使用 @hugeicons/core-free-icons（ArrowLeft01Icon、ArrowRight01Icon），通过 HugeiconsIcon 组件渲染，不要替换为其他图标库。
6. 样式使用 Tailwind CSS class 内联定义，不引入 CVA 或其他样式管理方案。
7. 不要引入 Ant Design 相关依赖（ConfigProvider、antd-style、@ant-design/icons 等）。
