# StackedList 堆叠列表

StackedList 是 UseLayouts 系列布局组件，提供一个双层团队列表展示：上层展示在线成员列表，下层为可展开的完整成员目录面板。使用 motion/react 驱动面板展开/折叠和列表项入场动画，hugeicons 提供图标。

本组件是自定义设计实现，不基于 Ant Design。

## 何时使用 {#when-to-use}

适合团队协作场景，需要同时展示在线成员概览和完整成员目录。上层列表保持始终可见，下层目录面板默认折叠，点击后以 spring 动画展开并覆盖整个容器。

常见场景：

- 团队协作应用的成员管理面板。
- 仪表盘侧边栏的团队状态展示。
- 项目成员目录，支持按角色分类浏览。
- 需要快速搜索和浏览的列表场景。

StackedList 的双层布局：

- 上层：在线成员列表，每项包含头像、姓名、在线状态和颜色编码的角色标签。
- 下层：可展开的成员目录面板，点击展开后覆盖上层，展示全量成员并支持搜索过滤。
- 在线成员显示绿色指示器和 "Online" 文本，离线成员显示相对时间。
- 角色标签：PM（黄色）、Designer（蓝色）、Data Specialist（绿色）、Creator（紫色）。

不推荐：

- 用于超大量成员的场景，当前为静态成员列表。
- 用于需要分页或虚拟滚动的成员列表。
- 用于需要对成员进行增删改查的管理后台。

## API 摘要 {#api}

StackedList 为零 props 组件，内部管理展开状态和搜索过滤。

内部状态：

- `isExpanded: boolean` — 成员目录面板是否展开。
- `searchQuery: string` — 搜索过滤文本，按姓名和角色匹配。

内部数据 `ALL_MEMBERS` 为 `Member[]` 类型的硬编码数组，包含 7 个示例成员。如需定制成员数据，请直接修改 `src/components/uselayouts/stacked-list.tsx`。

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- 组件未声明 `data-slot`，使用组件 DOM 结构定位。
- 外层容器：`div`，`max-w-[440px]`、`rounded-[40px]`、`border`。
- 上层活跃成员区域：`div.flex-1.overflow-y-auto.px-8`。
- 成员列表项：`motion.div`，包含头像（img）、信息区（h3 + p）、角色标签（RoleBadge）。
- 在线指示器：绝对定位的绿色圆点 `bg-green-500`。
- 底部面板：绝对定位的 `motion.div`，使用 `layout` 动画控制位置和尺寸。
- 面板折叠态：头像预览环（3 个头像 + "+N" 计数）。
- 面板展开态：搜索框 + 全量成员列表。

## 样式模型 {#styling-model}

- 样式由 Tailwind CSS className 直接控制，不使用 CVA。
- 动画由 motion/react 的 `motion.div`、`AnimatePresence`、`layout` 动画驱动。
- 图标使用 `@hugeicons/core-free-icons`（ProfileIcon、Search01Icon、Cancel01Icon、Add01Icon 及角色图标）。
- Input 组件来自主线 `@/components/ui/input`。
- Button 组件来自主线 `@/components/ui/button`。
- class 合并使用 `cn` 工具函数。

动画细节：

- 列表项使用 `variants: { hidden, visible }` 控制 opacity、x、y、rotate，通过 `staggerChildren` 实现依次入场。
- 底部面板使用 `layout` + spring transition 实现从紧凑条到全屏的平滑展开/折叠。
- 面板内的头像使用 `layoutId` 实现共享布局动画。
- `AnimatePresence` 控制搜索框的入场/退场。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/uselayouts/stacked-list.tsx`
- `src/components/ui/uselayouts-stacked-list.tsx`

直接依赖：

- `motion/react` — 面板展开动画、列表项入场动画、layoutId 共享动画。
- `@hugeicons/core-free-icons` — ProfileIcon、Search01Icon、Cancel01Icon、Add01Icon、Briefcase01Icon、PaintBoardIcon、Database01Icon、QuillWrite01Icon。
- `@hugeicons/react` — `HugeiconsIcon` 组件。
- `@/components/ui/input` — Input 搜索框。
- `@/components/ui/button` — Button 图标按钮。
- `@hyper/uselayouts/lib/cn` — cn 工具函数。

实现注意：

1. 组件为零 props 设计，内部管理 `isExpanded`、`searchQuery` 状态。
2. `ALL_MEMBERS` 为硬编码静态数据，在线成员通过 `.filter(m => m.online)` 派生。
3. 搜索过滤使用 `useMemo` 缓存过滤结果，匹配姓名和角色。
4. 底部面板使用绝对定位 + `motion.div layout` 实现从底部条到全屏覆盖的展开动画。
5. 面板折叠时点击任意位置触发展开；展开后点击关闭按钮折叠。
6. 面板内的头像使用 `layoutId={avatar-${m.id}}` 与折叠态预览头像共享布局动画。
7. `AnimatePresence` 控制面板内搜索框的入场/退场。
8. 成员列表项使用 `staggerChildren` 实现批量入场，`staggerDirection: -1` 实现反向退场。
9. 自定义滚动条样式通过 `custom-scrollbar` 和 `scroll-visible` className 控制。

## 可访问性 {#accessibility}

- 面板折叠态为可点击区域，展开后关闭按钮为可点击元素。
- 搜索框使用 `Input` 组件，支持键盘输入。
- 当前面板未使用原生 `<button>` 包裹点击区域，缺少键盘焦点和 ARIA 属性，建议扩展时补充。
- 建议扩展时添加 `role`、`tabIndex`、`aria-label` 和键盘 Enter/Space 激活支持。
- 头像使用 `alt` 属性提供成员名称。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 自定义设计，AntD 无直接对应的堆叠列表组件 |
| 视觉系统 | Tailwind CSS + motion/react 动画 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 css-in-js |
| 图标系统 | 使用 hugeicons，不使用 @ant-design/icons |
| 数据管理 | 内部硬编码静态数据，不依赖外部数据源 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 StackedList 代码时，应遵守：

1. 从 `@/components/ui/stacked-list` 导入 StackedList。
2. StackedList 为零 props 组件，直接渲染即可。
3. 如需定制成员数据或角色，修改 `src/components/uselayouts/stacked-list.tsx` 中的 `ALL_MEMBERS` 和 `styles`。
4. 图标使用 `@hugeicons/core-free-icons`，不要使用 `@ant-design/icons`。
5. 不要引入 `antd`、`antd-style` 或 rc-component。
6. 如有新增动画需求，使用 motion/react，不要引入其他动画库。
