StackedList 堆叠列表组件，展示团队成员的层次化列表，支持展开式成员目录面板与搜索过滤，带动画过渡。

<h2 id="when-to-use">何时使用</h2>

当需要展示团队成员列表，并同时提供紧凑的在线成员概览和可展开的完整成员目录时使用。

常见场景：

- 团队协作应用的成员管理面板。
- 仪表盘侧边栏的团队状态展示。
- 项目成员目录，支持按角色分类浏览。
- 需要快速搜索和浏览的列表场景。

StackedList 通过双层布局组织交互：

- 上层：在线成员列表，以列表形式垂直展示，每项包含头像、姓名、在线状态和角色标签。
- 下层：展开式成员目录面板，默认折叠为紧凑条，点击展开后覆盖整个容器，展示全量成员并支持搜索过滤。
- 在线成员显示绿色状态指示器和 "Online" 文本。
- 离线成员显示 "Xm ago" / "Xh ago" 等相对时间。
- 角色标签使用颜色编码：PM（黄色）、Designer（蓝色）、Data Specialist（绿色）、Creator（紫色）。

不推荐：

- 用于超大量成员的场景，当前数据为硬编码的静态列表。
- 用于需要分页或虚拟滚动的成员列表。
- 用于需要对成员数据进行增删改查的管理后台。

<h2 id="api">API</h2>

### StackedList

StackedList 为零 props 组件，内部管理展开状态和搜索过滤。

| 内部状态 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| isExpanded | 成员目录面板是否展开 | `boolean` | `false` |
| searchQuery | 搜索过滤文本 | `string` | `""` |

内部数据 `ALL_MEMBERS` 为硬编码的成员列表，包含 7 个示例成员。如需定制成员数据，请直接修改 `src/components/uselayouts/stacked-list.tsx` 中的 `ALL_MEMBERS` 数组。

### Member 数据模型

| 字段 | 说明 | 类型 |
| --- | --- | --- |
| id | 成员唯一标识 | `string` |
| name | 成员名称 | `string` |
| status | 状态文本（"Online" 或相对时间） | `string` |
| online | 是否在线 | `boolean` |
| role | 角色名称 | `string` |
| roleType | 角色类型 | `"pm"` \| `"designer"` \| `"data"` \| `"creator"` |
| avatar | 头像 URL | `string` |

<h2 id="faq">常见问题</h2>

### 如何修改成员数据？

直接修改 `src/components/uselayouts/stacked-list.tsx` 中的 `ALL_MEMBERS` 数组：

```tsx
const ALL_MEMBERS: Member[] = [
  { id: "01", name: "...", status: "...", online: true, role: "...", roleType: "pm", avatar: "..." },
  // 添加更多成员...
];
```

### 如何修改角色标签的样式？

修改 `RoleBadge` 组件中 `styles` 对象的颜色和图标配置：

```tsx
const styles = {
  pm: { bg: "bg-[#FFFCEB]", text: "text-[#856404]", border: "border-[#FFEBA5]", icon: Briefcase01Icon },
  // 修改或新增角色类型...
};
```

### 如何在展开目录时保留搜索状态？

搜索状态 `searchQuery` 在展开/折叠面板时保持不变，用户在目录面板中的搜索词在面板折叠后依然保留。

### 如何调整动画速度？

修改 `sweepSpring` 对象或面板容器的 `transition` 属性：

```tsx
const sweepSpring = {
  type: "spring" as const,
  stiffness: 400,  // 增大值加快动画
  damping: 35,     // 增大值减少弹跳
  mass: 0.5,
};
```

### 如何添加新增成员按钮的行为？

`Add01Icon` 按钮当前无点击回调。在 `Button` 上添加 `onClick` 处理函数即可。如需集成表单或模态框，在对应回调中实现。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 StackedList 代码时：

1. 从 `@/components/ui/stacked-list` 导入 StackedList。
2. StackedList 为零 props 组件，直接渲染即可。
3. 如需定制成员数据或角色，修改源码中的 `ALL_MEMBERS` 和 `styles`。
4. 图标使用 `@hugeicons/core-free-icons`，不要使用 `@ant-design/icons`。
5. 动画使用 motion/react，不要引入其他动画库。
6. 不要引入 Ant Design 或 rc-component 相关依赖。
