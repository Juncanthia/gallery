# Input 输入框

Input 输入框，用于接收用户的文本输入。组件基于原生 `<input>` 元素，通过 Tailwind CSS 提供统一的视觉样式，支持所有 HTML input 属性和 `ref` 转发。

本组件是 UseLayouts 集合中的基础表单组件，提供项目统一的输入框样式。

## 何时使用 {#when-to-use}

适合在需要用户输入文本内容的界面中使用。优先通过原生 HTML input 属性表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 表单中的文本输入、密码输入、数字输入。
- 搜索框、筛选条件输入。
- 登录、注册等表单页面。

不推荐：

- 多行文本输入，应使用 Textarea 组件。
- 需要选择器、级联选择等复合交互，应使用 Select、Cascader 等专用组件。

## 代码演示 {#examples}

### 基础用法

```tsx
import Input from "@/components/ui/uselayouts-input"

export default function Demo() {
  return <Input placeholder="请输入内容" />
}
```

> 说明：这是面向 LLM 的最小导入示例。Input 支持所有原生 HTML input 属性，复杂场景请结合下方 API 摘要使用 `type`、`disabled`、`value`、`onChange` 等属性。

## 核心导出 {#exports}

- `Input` (default)

## API 摘要 {#api}

以下类型来自本地源码，是生成代码时的主要依据：

```ts
type InputProps = React.InputHTMLAttributes<HTMLInputElement>
```

组件通过 `React.forwardRef` 转发 ref，接受所有原生 HTML input 属性。无额外自定义 props。

默认样式（通过 `cn` 合并，className 可覆写）：

- 布局：`flex h-10 w-full`
- 外观：`rounded-md border border-input bg-background`
- 内边距与字体：`px-3 py-2 text-sm`
- 聚焦态：`ring-offset-background focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`
- 禁用态：`disabled:cursor-not-allowed disabled:opacity-50`
- 占位符：`placeholder:text-muted-foreground`
- file 类型：`file:border-0 file:bg-transparent file:text-sm file:font-medium`

## Semantic DOM {#semantic-dom}

源码中未声明 `data-slot`。组件渲染为原生 `<input>` 元素，使用原生 HTML 语义定位。

## 样式模型 {#styling-model}

- 样式完全通过 Tailwind CSS className 调用 `cn()` 合并。
- 无 CVA variant，无 variant 变体系统。
- 不包装 Radix 或 Base UI primitive。
- 主题颜色使用 `border-input`、`bg-background`、`text-muted-foreground`、`ring-ring` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/uselayouts/input.tsx`

直接依赖摘要：

- `react`（`forwardRef`）
- `clsx`（`cn` 合并 className）

实现注意：

1. 组件使用 `React.forwardRef` 转发 ref，可直接获取原生 `<input>` 引用。
2. 自定义 className 通过 `cn()` 与默认样式合并，随后续的 className 可覆写提前的类。
3. 无额外状态管理或事件拦截，`onChange` 等事件直接透传至原生 input。

## 可访问性 {#accessibility}

- 渲染为原生 `<input>` 元素，保留所有原生键盘和 ARIA 语义。
- 建议配合 `<label>` 或 `aria-label` 提供可访问名称。
- `disabled` 属性设置原生的 `disabled` 属性和 `cursor-not-allowed` 样式。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Input 代码时，应遵守：

1. 从 `@/components/ui/uselayouts-input` 导入 `Input`（default export）。
2. 受控模式使用 `value` + `onChange`，非受控模式使用 `defaultValue`。
3. 需要输入框引用时使用 `ref`，不要通过 DOM 查询直接操作。
4. 禁用态使用 `disabled` 属性。
5. 不要引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。
6. 自定义样式通过 `className` 覆写，不要通过外部 CSS 或内联 style 覆盖内部默认样式。
