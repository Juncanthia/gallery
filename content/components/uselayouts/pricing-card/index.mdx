PricingCard 是一个交互式定价卡片组件，支持月付/年付切换、多方案选择、用户数量调整和流畅的动画过渡。

<h2 id="when-to-use">何时使用</h2>

当需要展示产品定价方案供用户选择时使用。组件内置月付/年付切换、方案对比和用户数量调节，适用于 SaaS 订阅类产品的定价页面。

常见场景：

- SaaS 产品定价页面，展示不同订阅层级的方案对比。
- 服务型产品的价格展示，支持灵活的计费周期切换。
- 团队版产品的定价选择，附带用户数量配置。

不推荐：

- 单一定价的简单产品展示（使用更简洁的卡片或标签即可）。
- 需要复杂价格计算逻辑的场景（组件仅支持基础的价格展示）。

组件内置三档方案（Plus / Standard / Advanced），支持月付/年付切换和用户数量调整。无需传递任何 props 即可使用。

<h2 id="api">API</h2>

### PricingCard

组件为自包含组件，无需传递任何 props。所有方案数据、计费周期和用户数量均由组件内部状态管理。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| - | 无需传递任何 props | - | - |

组件内部数据结构：

- `plans`：三档定价方案数组，每项包含 id、name、description、monthlyPrice、yearlyPrice 和 features。
- `billingCycle`：计费周期状态，`"monthly"` 或 `"yearly"`，默认 `"monthly"`。
- `selectedPlan`：当前选中的方案 ID，默认 `"standard"`。
- `userCount`：用户数量，默认 `3`。

动画依赖 `motion/react` 的 `LayoutGroup`、`AnimatePresence` 和 `motion.div`，实现方案切换时的展开折叠动画。价格数字使用 `@number-flow/react` 实现数字滚动效果。图标使用 `@hugeicons/core-free-icons` 和 `@hugeicons/react`。

<h2 id="faq">常见问题</h2>

### 如何自定义定价方案？

方案数据定义在组件的 `plans` 数组中。直接修改 `src/components/uselayouts/pricing-card.tsx` 中的 `plans` 数组，调整 id、name、description、价格和 features 即可。

### 如何修改默认选中的方案？

修改组件内 `useState("standard")` 的初始值，将 `"standard"` 替换为 `"plus"` 或 `"advanced"`。

### 组件使用的动画库是什么？

使用 `motion/react`（Framer Motion 的 React 版本）实现布局动画和展开折叠过渡，使用 `@number-flow/react` 实现价格数字的滚动动画效果。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/pricing-card` 导入 `PricingCard`（named export）。
2. 组件无需任何 props，直接渲染即可获得完整交互。
3. 组件默认最大宽度为 `max-w-[450px]`，如需调整通过 `className` 覆写容器样式。
4. 组件依赖 `motion/react`、`@number-flow/react` 和 `@hugeicons/react`，确保项目中已安装这些依赖。
5. 不要引入 Ant Design 相关依赖。
