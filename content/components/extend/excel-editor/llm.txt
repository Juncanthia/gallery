# ExcelEditorBlock Excel 编辑器

内嵌 XLSX 表格编辑器，支持单元格编辑、格式化、公式、边框、合并、工作表管理和文件上传/下载。

本组件是 Extend 系列中的 block 级组件，基于 `@extend-ai/react-xlsx` 构建完整编辑工具栏，API 和行为以源码为准。

## 何时使用 {#when-to-use}

适合需要嵌入式 Excel 编辑能力的场景。通过 `file` 和 `heightClassName` props 控制文件源和布局尺寸。

常见场景：

- 数据录入与编辑界面。
- 报表在线编辑预览。
- .xlsx 文件查看、编辑、下载。
- 需要完整格式化工具栏的数据表格组件。

不推荐：

- 仅需轻量级表格展示——使用项目中的 Table 组件。
- 仅需 CSV 查看——使用 `CsvViewer`。
- 需要自定义编辑行为或外部状态同步——当前版本不暴露编辑事件回调。

## 代码演示 {#examples}

### 基础用法

```tsx
import { ExcelEditorBlock } from "@/components/ui/excel-editor"

export default function Demo() {
  return <ExcelEditorBlock />
}
```

> 说明：默认加载内置示例文件 `/samples/crazy-chart-zoo.xlsx`，包含完整的编辑工具栏。

## 核心导出 {#exports}

- `ExcelEditorBlock`

## API {#api}

### ExcelEditorBlock

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| file | 远程 .xlsx 文件 URL | `string` | `/samples/crazy-chart-zoo.xlsx` |
| heightClassName | 编辑器容器 Tailwind 高度类名 | `string` | `h-[720px]` |

> `ExcelEditorBlock` 内部封装了 `XlsxEditorPreview`（来自 `@hyper/extend-ui`），后者基于 `@extend-ai/react-xlsx` 的 `XlsxViewerProvider`、`XlsxViewer` 和全套 hooks 构建。

## 编辑能力 {#editing-capabilities}

组件工具栏提供以下编辑功能：

### 文本格式化
- 字体选择：Arial, Times New Roman, Georgia, Helvetica, Courier New
- 字号：8-48pt
- 粗体、斜体、下划线、删除线

### 颜色
- 文字颜色（ColorPicker）
- 填充颜色（ColorPicker）
- 无填充（清除填充）
- 边框颜色（ColorPicker）

### 对齐
- 水平对齐：左对齐、居中对齐、右对齐
- 垂直对齐：顶部对齐、居中对齐、底部对齐
- 文本换行、不换行、自适应缩小

### 数字格式
- General（通用）
- Number（0.00）
- Currency（$#,##0.00）
- Percent（0.00%）
- Date（m/d/yyyy）
- 自定义格式字符串

### 边框
- 四边边框、外边框、无边
- Bottom / Left / Top / Right 独立边框
- 边框颜色可配置

### 合并与工作表
- 合并并居中、跨行合并、合并单元格、取消合并
- 添加/删除工作表
- 工作表切换选择

### 操作
- 撤销/重做
- 公式栏：`fx` 前缀的公式输入框，以 `=` 开头触发公式模式
- 缩放：50%-400%
- 暗色模式切换
- 上传本地 .xlsx / 下载当前工作簿

### 编辑限制
- 文件超过 5MB 自动切换只读模式
- 只读模式下工具栏编辑控件禁用

## Semantic DOM {#semantic-dom}

组件内部使用 `@extend-ai/react-xlsx` 的 `XlsxViewer` 渲染表格网格。DOM 结构由第三方库控制，不暴露稳定的本地语义槽位。

额外状态属性：

- `data-pressed`：工具栏按钮激活态
- `data-disabled`：禁用态
- `aria-label`：所有工具栏按钮提供可访问名称

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与项目 `cn` 合并函数控制。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground` 等项目级语义变量。
- 编辑器工具栏采用 Ghost/Secondary 变体按钮，与项目整体设计语言一致。
- 编辑器网格由第三方库 `@extend-ai/react-xlsx` 渲染，样式扩展性有限。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/ui/extend-excel-editor.tsx`（重导出）
- `src/components/extend/components/blocks/excel-editor.tsx`（实现）
- `src/components/extend/components/ui/xlsx-editor.tsx`（XlsxEditorPreview）

直接依赖摘要：

- `@extend-ai/react-xlsx`
- `@hyper/extend-ui`
- `@hugeicons/core-free-icons`
- `@hugeicons/react`

实现注意：

1. 组件 API 以 `ExcelEditorBlock` 的 TypeScript props 类型为准。
2. 编辑器默认加载 `/samples/crazy-chart-zoo.xlsx` 示例文件。
3. 如果需要新增编辑能力或修改工具栏，应修改 `src/components/extend/components/blocks/excel-editor.tsx`，然后更新本 llm 文档。
4. 不要把 `@extend-ai/react-xlsx` 或 `@hyper/extend-ui` 文档中的全量 API 视为本地已支持能力。

## 可访问性 {#accessibility}

- 所有工具栏按钮均提供 `aria-label` 属性。
- 编辑、禁用、激活等状态通过组件 props 和 `data-*` 属性表达。
- 键盘操作由底层 `@extend-ai/react-xlsx` 处理。
- 纯图标操作均通过 `ToolbarTooltip` 提供文本提示。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/extend/components/blocks/excel-editor.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | Extend 独立组件，不与 AntD 对齐 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 ExcelEditorBlock 代码时，应遵守：

1. 从 `@/components/ui/excel-editor` 导入，不要从 `@hyper/extend-ui` 或 `@extend-ai/react-xlsx` 导入。
2. 只使用本文件 API 摘要中列出的 props：`file` 和 `heightClassName`。
3. 默认无需任何 props 即可渲染完整编辑器。
4. 需要自定义样式时使用 `heightClassName` prop。
5. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置。
6. 不要在业务代码中直接使用 `@extend-ai/react-xlsx` 的 hooks（`useXlsxViewerEditing` 等），除非修改组件源码本身。
7. 如果不确定某个功能是否存在，先检查 `src/components/extend/components/blocks/excel-editor.tsx`，不要凭第三方库文档生成。
