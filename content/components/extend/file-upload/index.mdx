FileUpload 文件上传组件，支持点击上传和拖拽上传，带可视化拖拽反馈和文件列表展示。

<h2 id="when-to-use">何时使用</h2>

当需要用户上传文件时使用。支持点击选择文件和拖拽文件到指定区域两种方式。

常见场景：

- 文档上传：上传 PDF、Word、Excel 等办公文档。
- 图片上传：上传 PNG、JPG 等图片文件。
- 批量上传：一次选择或拖入多个文件。
- 拖拽上传：将文件从文件管理器直接拖入上传区域。

不推荐：

- 用于超大文件（&gt;100MB）上传，应使用分片上传方案。
- 在移动端使用拖拽上传，移动端拖拽体验受限应优先使用点击上传。

<h2 id="examples">代码演示</h2>

### 基础用法

点击上传区域或拖拽文件进入虚线区域即可上传。支持 `.pdf`、`.doc`、`.docx`、`.xlsx`、`.csv`、`.png`、`.jpg`、`.jpeg` 格式。

import { FileUpload } from "@/components/document/components/ui/file-upload"

export default function Demo() {
  return (
    &lt;FileUpload

  )

<h2 id="api">API</h2>

### FileUpload

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| accept | 接受的文件类型（MIME 或扩展名，逗号分隔） | `string` | `.pdf,.doc,.docx,.xlsx,.csv,.png,.jpg,.jpeg,...` |
| acceptedFileTypes | 上传区域展示的文件类型图标 | `AcceptedFileType[]` | Image / PDF / Sheet |
| borderBeamTheme | 拖拽时边框光束主题 | `"light"` \| `"dark"` | `"light"` |
| browseLabel | 浏览按钮文本 | `string` | `"Browse files"` |
| className | 附加样式类名 | `string` | - |
| description | 上传区域描述文本 | `string` | `"PDF, DOC/DOCX, XLSX, CSV, PNG, or JPG"` |
| draggingLabel | 拖拽中提示文本 | `string` | `"Drop to add"` |
| multiple | 是否支持多文件上传 | `boolean` | `true` |
| showBorderBeam | 是否显示拖拽边框光束效果 | `boolean` | `true` |
| showFileList | 是否显示已上传的文件列表 | `boolean` | `true` |
| title | 上传区域标题 | `string` | `"Click to upload or drop files"` |
| onFilesAccepted | 文件通过格式校验后的回调 | `(files: File[]) => void` | - |
| onFilesChange | 文件列表变化回调（含 id/name/type/size/url） | `(files: FileUploadItem[]) => void` | - |

### FileUploadItem

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| id | 文件唯一标识 | `string` |
| name | 文件名 | `string` |
| type | MIME 类型 | `string` |
| size | 文件大小（字节） | `number` |
| url | 文件预览 URL（blob URL） | `string` |

### AcceptedFileType

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| label | 文件类型标签 | `string` |
| icon | 文件类型图标（@hugeicons/core-free-icons） | `HugeiconsIcon['icon']` |

<h2 id="faq">常见问题</h2>

### 如何限制文件类型？

通过 `accept` 属性指定允许的文件类型：

```tsx
&lt;FileUpload accept=".pdf,.docx" /&gt;
```

支持文件扩展名（`.pdf`）和 MIME 类型（`application/pdf`）。

### 如何限制单文件上传？

设置 `multiple={false}`：

```tsx
&lt;FileUpload multiple={false} /&gt;
```

### 如何获取上传的文件数据？

使用 `onFilesChange` 获取带有元信息的文件列表，或使用 `onFilesAccepted` 获取原生 File 对象：

```tsx
&lt;FileUpload
  onFilesChange={(files) => {
    files.forEach((f) => console.log(f.name, f.size))
  }}
/&gt;
```

### 如何自定义上传区域的图标？

通过 `acceptedFileTypes` 传入自定义文件类型图标数组：

```tsx
import { FilePdfIcon } from "@hugeicons/core-free-icons"

&lt;FileUpload
  acceptedFileTypes={[{ label: "PDF", icon: FilePdfIcon }]}
/&gt;
```

### 如何关闭拖拽光束效果？

设置 `showBorderBeam={false}`：

```tsx
&lt;FileUpload showBorderBeam={false} /&gt;
```

### 如何隐藏文件列表？

设置 `showFileList={false}`，只保留上传区域：

```tsx
&lt;FileUpload showFileList={false} /&gt;
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/document/components/ui/file-upload` 导入 `FileUpload`。
2. 使用 `accept` 限制文件类型，使用 `multiple` 控制单/多文件。
3. 通过 `onFilesChange` 获取完整文件信息，`onFilesAccepted` 获取原生 File 对象。
4. `onFilesChange` 回调中的文件带有 blob URL，注意在数据同步或持久化前将其转换为 File 或 ArrayBuffer。
5. 不要引入 Ant Design Upload 或其 rc-upload 依赖。
