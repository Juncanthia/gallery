SchemaBuilderPanel 是一个可视化的 JSON Schema 编辑器，支持通过拖拽和表单编辑的方式构建和预览 JSON Schema 结构。

<h2 id="when-to-use">何时使用</h2>

需要在可视化界面中构建、编辑或预览 JSON Schema 时使用。

常见场景：

- Schema 设计：通过拖拽方式快速搭建 JSON Schema 的结构定义。
- API 文档生成：可视化编辑字段类型、描述和枚举值，实时预览 JSON 输出。
- 数据建模：定义嵌套对象、数组项和枚举约束。
- 提取字段配置：为数据提取任务配置目标字段的 Schema 定义。

SchemaBuilderPanel 提供 Form 和 JSON 两个 Tab：Form 标签页通过表格形式拖拽编辑 Schema 属性（支持嵌套对象、数组和枚举）；JSON 标签页实时展示序列化后的 JSON Schema，支持语法高亮。

<h2 id="api">API</h2>

### SchemaBuilderPanel

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |
| defaultSchema | 非受控模式下的初始 Schema | `SchemaBuilderSchema` | `SAMPLE_SCHEMA` |
| schema | 受控模式下的 Schema，传入后组件进入受控模式 | `SchemaBuilderSchema` | - |
| onSchemaChange | Schema 变更回调 | `(schema: SchemaBuilderSchema) => void` | - |
| theme | 强制指定 JSON 预览主题，不传则跟随系统 | `"light"` \| `"dark"` | - |

### SchemaBuilderSchema

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| properties | Schema 的顶层属性列表 | `SchemaBuilderProperty[]` |

### SchemaBuilderProperty

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| id | 属性唯一标识 | `string` |
| key | 属性键名 | `string` |
| type | 属性类型 | `"string"` \| `"number"` \| `"integer"` \| `"boolean"` \| `"null"` \| `"object"` \| `"array"` \| `"enum"` |
| description | 属性描述 | `string` |
| enumValues | type 为 enum 时的枚举值列表 | `SchemaBuilderEnumValue[]` |
| properties | type 为 object 时的嵌套属性列表 | `SchemaBuilderProperty[]` |
| items | type 为 array 时的数组项定义 | `{ type, properties?, enumValues? }` |

<h2 id="faq">常见问题</h2>

### 如何受控使用 SchemaBuilderPanel？

传入 `schema` 和 `onSchemaChange` 即可进入受控模式：

```tsx
import { SchemaBuilderPanel, type SchemaBuilderSchema } from "@/components/ui/extend-schema-builder"

function App() {
  const [schema, setSchema] = useState&lt;SchemaBuilderSchema&gt;()

  return (
    &lt;SchemaBuilderPanel
      schema={schema}
      onSchemaChange={setSchema}
    /&gt;
  )
}
```

### 如何自定义初始 Schema？

通过 `defaultSchema` 传入自定义的初始 Schema 结构：

```tsx
import { SchemaBuilderPanel } from "@/components/ui/extend-schema-builder"

const MY_SCHEMA = {
  properties: [
    { id: "1", key: "name", type: "string", description: "用户名" },
    { id: "2", key: "age", type: "integer", description: "年龄" },
  ],
}

export default function Demo() {
  return (
    &lt;SchemaBuilderPanel defaultSchema={MY_SCHEMA} /&gt;
  )
}
```

### 如何序列化 Schema？

使用导出的 `serializeSchema` 函数将 `SchemaBuilderSchema` 转换为标准 JSON Schema 格式：

```tsx
import { serializeSchema } from "@/components/ui/extend-schema-builder"

const jsonSchema = serializeSchema(schema)
console.log(JSON.stringify(jsonSchema, null, 2))
```

### 支持哪些字段类型？

支持 string、number、integer、boolean、null、object、array、enum 八种类型。array 支持嵌套 object 和 enum 作为数组项类型。

### 如何修改 JSON 预览的主题？

通过 `theme` 属性强制指定亮色或暗色主题：

```tsx
&lt;SchemaBuilderPanel theme="dark" /&gt;
```

不传时自动跟随系统主题设置。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 SchemaBuilderPanel 代码时：

1. 从 `@/components/ui/extend-schema-builder` 导入
2. 默认用法不需要任何 props，组件内置示例 Schema
3. 使用 `schema` + `onSchemaChange` 进入受控模式
4. 通过 `defaultSchema` 自定义初始 Schema 结构
5. 使用 `serializeSchema` 导出标准 JSON Schema
6. 不要自行实现 Schema 编辑器或拖拽排序，使用 SchemaBuilderPanel 组件
7. 不要引入额外的 Schema 编辑库（如 @rjsf/core、react-jsonschema-form），SchemaBuilderPanel 已内置完整的拖拽编辑和 JSON 预览能力
