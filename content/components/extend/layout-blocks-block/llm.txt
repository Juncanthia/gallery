# OcrBlocksBlock OCR 区块

左侧展示内嵌 PDF 查看器并叠加 OCR 识别区块的高亮标注，右侧展示 OCR 区块列表，支持双向联动定位。组件内置 Attention Is All You Need 论文示例数据。

本组件内部组合 PDFViewer、OcrBlocksPanel 和 PdfBlockResizableShell，提供一站式 OCR 区块审查体验。不是 Ant Design 兼容层，实现以本地源码为准。

## 何时使用 {#when-to-use}

适合需要在同一界面展示 PDF 文档及其 OCR 识别结果，并支持区块与页面双向联动定位的场景。

常见场景：

- 文档 OCR 处理工具的识别结果展示与校对。
- 合同、论文等 PDF 文档的版面分析与内容提取。
- 需要逐区块审查 OCR 识别质量的人工审核工作流。

不推荐：

- 仅需展示 PDF 时，使用 PDFViewer。
- 仅需 OCR 文本内容时，直接使用 OcrBlocksPanel。
- 需要自定义 OCR 流程时，应在业务层编排，而非依赖组件内部数据。

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `pdf-block-ocr` — PdfBlockResizableShell 根元素的 `data-slot` 值

额外状态属性：

- `data-slot="pdf-block-ocr"` — 可调整面板根容器标记

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 控制。
- 使用 `PdfBlockResizableShell` 提供左右分栏布局，支持拖拽调整宽度比例。
- 左侧 PDF 查看器通过 `renderPageOverlay` 叠加 OCR 区块高亮矩形（激活态蓝色，非激活态半透明灰色）。
- 右侧 OcrBlocksPanel 使用虚拟滚动 (`@tanstack/react-virtual`)，区块列表项带类型图标和 Markdown 渲染。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`text-muted-foreground` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/extend/components/blocks/layout-blocks-block.tsx` — OcrBlocksBlock 主组件
- `src/components/extend/components/ui/layout-blocks.tsx` — OcrBlocksPanel、OcrBlockOverlay、getOcrBlocks 等底层实现
- `src/components/ui/extend-layout-blocks-block.tsx` — 路径重导出封装

直接依赖摘要：

- 依赖 `PDFViewer`（PDF 渲染）和 `PdfBlockResizableShell`（分栏布局）。
- 使用 `@tanstack/react-virtual` 实现虚拟滚动。
- 使用 `react-markdown` + `rehypeRaw` + `rehypeSanitize` + `remarkGfm` 渲染区块 Markdown 内容。
- 使用 `@hugeicons/core-free-icons` 提供区块类型图标。
- 内置 `ATTENTION_OCR_OUTPUT` 示例数据和 `/samples/attention.pdf` 示例 PDF。

实现注意：

1. `OcrBlocksBlock` 的 `file` 和 `output` 均有默认值，无 props 也可直接使用。
2. 组件内部通过 `React.useMemo` 对 OCR 输出进行转换，通过 `React.useCallback` 缓存 overlay 渲染函数。
3. 区块点击联动通过 `scrollToPageArea` 实现页面滚动定位，使用 `activeBlockIdRef` 防止重复滚动。
4. OCR 区块叠加层通过 `renderPageOverlay` 注入到 PDFViewer 中，与页面尺寸匹配渲染。
5. 分栏宽度比例自动保存到 localStorage（key: `pdf-block-ocr`）。
6. 如需扩展功能（如自定义 OCR 处理逻辑），应修改源码而非在外层 hack 内部 DOM 结构。

## 可访问性 {#accessibility}

- OCR 区块列表中的每个区块按钮支持键盘导航和点击。
- 区块按钮使用 `tabIndex={0}` 和 `role="button"`。
- PDFViewer 提供完整的键盘导航和 ARIA 属性。
- 区块类型图标通过 `aria-hidden="true"` 和 `focusable="false"` 标记为装饰性元素。

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 OcrBlocksBlock 代码时，应遵守：

1. 从 `@/components/ui/extend-layout-blocks-block` 导入，不要从内部路径导入。
2. `file` 和 `output` 均有默认值，无 props 时展示内置示例。
3. 不要手动组合 PDFViewer + OcrBlocksPanel + PdfBlockResizableShell 来模拟本组件行为。
4. 不要引入 Ant Design 相关依赖。
5. 如需自定义样式，使用外层容器的 className。
6. 如果不确定某个行为是否存在，先检查源码文件，不要凭猜测生成。
7. 如需获取 OCR 区块数据，使用 `getOcrBlocks(output)` 工具函数。
