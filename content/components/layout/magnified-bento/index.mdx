MagnifiedBento 是一个带放大镜交互的便当卡片布局组件，通过拖拽透镜揭示隐藏的高亮标签内容。

<h2 id="when-to-use">何时使用</h2>

当需要一个带有交互趣味的技能 / 标签展示面板时使用，尤其适合团队介绍页的能力展示、首页服务覆盖范围的可视化呈现、需要轻微趣味动效的展示型卡片。

常见场景：

- 团队技能栈展示，以标签云方式呈现全栈能力。
- 服务介绍卡片，自动滚动标签配合透镜交互吸引用户注意。
- 首页 Banner 下方能力展示区，增加页面交互趣味性。
- 项目管理面板中的标签分类可视化。

MagnifiedBento 是一个无 props 的自包含组件，渲染完整面板：

- 三行自动滚动的标签云，奇数行和偶数行反向滚动产生视觉差异。
- 可拖拽放大镜透镜，揭示透镜下方隐藏的高亮标签版本。
- 底部标题 "Intelligent Workflows" 和说明文案。
- 左右渐变遮罩边缘，避免标签截断突兀感。

<h2 id="api">API</h2>

MagnifiedBento 为自包含组件，不接受外部 props。所有内部状态（透镜位置、标签配置、动画参数）由组件自行管理。

如需扩展，在 `src/components/layout/magnified-bento.tsx` 中修改：

- `TAG_ROWS`：三行标签配置数组。
- `CONFIG`：全局配置（`title`、`description`、`containerHeight`、`lensSize`）。

<h2 id="faq">常见问题</h2>

### 如何自定义标签内容？

在 `src/components/layout/magnified-bento.tsx` 中修改 `TAG_ROWS` 数组，每项包含 `id`、`icon`、`label`。图标来自 `@hugeicons/core-free-icons`。

### 如何修改标题和描述？

修改 `CONFIG.title` 和 `CONFIG.description`。

### 如何调整透镜大小？

修改 `CONFIG.lensSize`（默认 92），同时需同步调整透镜内部白色覆盖层的尺寸。

<h2 id="llm-rules">LLM 使用规则</h2>

5. 样式使用 Tailwind CSS class 内联定义。
6. 不要引入 Ant Design 相关依赖。
