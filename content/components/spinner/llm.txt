# Spinner 加载旋转

Spinner 是一个加载动画指示器组件，提供 8 种预置的视觉变体。它使用 lucide-react 的图标作为基础，配合 Tailwind CSS 的 `animate-spin` 动画和自定义 SVG 动画。

本组件不是 Ant Design 兼容层。Ant Design 的 Spin 组件是内嵌在容器中的加载方案（`Spin` + `Spin.spin()`），本地 Spinner 是更轻量的独立指示器，仅供内联使用。设计目标是无依赖的纯展示加载动画。

## 何时使用 {#when-to-use}

在需要指示内容正在加载、处理中或等待响应时使用。

常见场景：

- 数据加载中的占位指示。
- 表单提交按钮内的加载状态。
- 页面/区块的初始加载。
- 无限滚动底部的加载更多指示。
- 任何需要视觉反馈的异步操作等待期。

推荐使用方式：

- 默认变体（`default`，基于 lucide Loader2Icon）适用于大多数场景。
- `throbber` 适用于与 Lucide 图标集风格统一的内联加载。
- `pinwheel` 适用于需要更明显旋转效果的场景。
- `circle-filled` 适用于需要强调品牌色的加载状态。
- `ellipsis` / `ring` / `bars` / `infinite` 提供更多视觉选择，适用于特定的 UI 风格。

不推荐：

- 在一个页面内同时使用多种变体（应在项目内统一 1-2 种）。
- 在非异步操作的上下文中使用（会误导用户）。

## 代码演示 {#examples}

### 所有变体

```tsx
import { Spinner } from "@/components/blocks/spinner"

const variants = [
  "default",
  "throbber",
  "pinwheel",
  "circle-filled",
  "ellipsis",
  "ring",
  "bars",
  "infinite",
] as const

export default function Demo() {
  return (
    <div className="flex flex-wrap items-center gap-4">
      {variants.map((variant) => (
        <div key={variant} className="flex flex-col items-center gap-1">
          <Spinner variant={variant} />
          <span className="text-xs text-muted-foreground">{variant}</span>
        </div>
      ))}
    </div>
  )
}
```

### 自定义尺寸和颜色

```tsx
import { Spinner } from "@/components/blocks/spinner"

export default function Demo() {
  return (
    <div className="flex flex-wrap items-center gap-6">
      <Spinner size={16} className="text-blue-500" />
      <Spinner variant="circle-filled" size={32} className="text-emerald-500" />
      <Spinner variant="ellipsis" size={40} className="text-red-500" />
      <Spinner variant="ring" size={48} className="text-purple-500" />
    </div>
  )
}
```

## API {#api}

### Spinner

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 视觉变体 | `"default"` \| `"throbber"` \| `"pinwheel"` \| `"circle-filled"` \| `"ellipsis"` \| `"ring"` \| `"bars"` \| `"infinite"` | `"default"` |
| size | 图标尺寸（像素），仅对部分变体生效 | `number` | `24` |
| className | 附加 className | `string` | - |

继承 `LucideProps`（来自 lucide-react），因此也支持 `color`、`strokeWidth` 等 Lucide 图标属性。

变体说明：

| 变体 | 实现 | 动画 |
| --- | --- | --- |
| `default` | `Loader2Icon` (lucide) | `animate-spin`，默认 `size-6` |
| `throbber` | `LoaderIcon` (lucide) | `animate-spin` |
| `pinwheel` | `LoaderPinwheelIcon` (lucide) | `animate-spin` |
| `circle-filled` | 双层 `LoaderCircleIcon` (lucide) | 上层 `animate-spin`，下层固定半透明 |
| `ellipsis` | 自定义 SVG（3 个圆点） | SVG `<animate>` 上下弹跳 |
| `ring` | 自定义 SVG（圆环扩展） | SVG `<animate>` 圆环半径从 1 到 20 + 透明度渐变 |
| `bars` | 自定义 SVG（3 个竖条） | CSS `@keyframes` 条形高度变化，三根条不同延迟 |
| `infinite` | 自定义 SVG（无限路径） | SVG `<animate>` stroke-dashoffset 移动 |

## 样式模型 {#styling-model}

Spinner 的样式由 Tailwind CSS + SVG 内联动画组合而成，无独立 CVA 配置。

核心样式策略：

- **lucide 变体**（default / throbber / pinwheel / circle-filled）：继承 `LucideProps`，使用 `animate-spin` 实现旋转动画。
- **SVG 变体**（ellipsis / ring / bars / infinite）：自定义 SVG 结构，内嵌 `<animate>` 或 `<style>` + CSS keyframes 标签驱动动画。
- 颜色继承 `currentColor`，通过 `className="text-*"` 控制。
- 尺寸通过 `size` 属性控制（映射到 SVG 的 width/height 和 viewBox）。
- `circle-filled` 使用双层结构：底部层 `rotate-180` 且 `opacity-20`，上层正常旋转，形成"追逐"效果。
- 所有变体统一设置 `role="status"` 和 `aria-label="Loading"`。自定义 SVG 变体使用 `<title>Loading...</title>` 提供可访问名称。

| 变体 | 尺寸来源 | 颜色来源 | 动画引擎 |
| --- | --- | --- | --- |
| default | `size` prop（默认通过 className `size-6`） | `currentColor` + className | Tailwind `animate-spin` |
| throbber | `size` prop | `currentColor` + className | Tailwind `animate-spin` |
| pinwheel | `size` prop | `currentColor` + className | Tailwind `animate-spin` |
| circle-filled | `size` prop（默认 24） | `currentColor` + className | Tailwind `animate-spin` |
| ellipsis | `size` prop（默认 24） | `currentColor` + className | SVG `<animate>` |
| ring | `size` prop（默认 24） | `currentColor` + className | SVG `<animate>` |
| bars | `size` prop（默认 24） | `currentColor` + className | CSS `@keyframes` |
| infinite | `size` prop（默认 24） | `currentColor` + className | SVG `<animate>` |

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/spinner/index.tsx`：全部变体定义。

依赖：

- `lucide-react`：Loader2Icon、LoaderCircleIcon、LoaderIcon、LoaderPinwheelIcon。
- `@/lib/utils`：`cn` 工具函数。

关键行为：

1. `Spinner` 组件根据 `variant` 属性分发到对应的内部变体组件。
2. 未指定 `variant` 或值为 `"default"` 时，使用 `Loader2Icon`，默认 `size-6`。
3. `SpinnerProps` 继承 `LucideProps`，意味着所有变体都接受 Lucide 图标的通用属性（如 `size`、`color`、`strokeWidth`）。自定义 SVG 变体通过解构 `size` 和 `...props` 兼容这些属性。
4. 每个变体通过 `data-slot="spinner"`、`role="status"` 和 `aria-label="Loading"` 提供无障碍语义。
5. `circle-filled` 特殊处理：先渲染一个 `rotate-180` 的半透明底层圆，再渲染正常旋转的上层圆，叠加形成追逐效果。
6. 自定义 SVG 变体的动画完全由内嵌的 `<animate>` 元素驱动，不依赖 JavaScript 或外部 CSS 文件。
7. 组件无状态、无副作用，纯渲染。

## 可访问性 {#accessibility}

- Default 变体设置 `data-slot="spinner"`、`role="status"` 和 `aria-label="Loading"`。
- 所有自定义 SVG 变体包含 `<title>Loading...</title>` 提供可访问名称。
- `role="status"` 告诉屏幕阅读器这是一个状态指示器，内容变化时会自动通知用户（但不打断当前操作）。
- 动画使用纯 CSS/SVG，不触发 layout 重排，对性能友好。
- 建议在异步操作的容器上额外设置 `aria-busy="true"`，提供更完整的加载状态语义。

## 与 Ant Design 的取舍 {#antd-differences}

Ant Design 的 Spin 组件是一个功能更完整的加载方案，支持嵌入容器（`<Spin>` wrapper）、全屏加载（`Spin.spin()`）、自定义指示器和延迟显示。本地 Spinner 定位为更轻量的纯展示型指示器。

| AntD 能力 | 本地决策 | 本地写法 / 原因 |
| --- | --- | --- |
| `<Spin>` 容器包裹 | 不做 | 本地 Spinner 是独立指示器，容器包裹由调用侧自行实现 |
| `Spin.spin()` 全屏加载 | 不做 | 全屏加载属于应用级行为，不在组件范围内 |
| `delay` 延迟显示 | 不做 | 简化 API；调用侧可在逻辑层实现延迟 |
| `tip` 加载文案 | 不做 | 调用侧可自行在 Spinner 下方放置文本 |
| `indicator` 自定义指示器 | 不做 | 提供 8 种预置变体覆盖常见需求 |
| `size` small/default/large | 保留但简化为 number | 直接传像素值，更精确 |
| 嵌套模式（`spinning`） | 不做 | 不包裹子内容 |

| 考量 | 决策 | 原因 |
| --- | --- | --- |
| 变体数量 | 8 种 | 覆盖主流加载动画风格，从极简的 throbber 到复杂的 infinite |
| 动画引擎 | CSS animation + SVG animate | 纯声明式，无 JavaScript 动画依赖 |
| 图标来源 | lucide-react + 自定义 SVG | 首选 lucide 图标减少自定义代码，4 种 lucide 变体提供基础覆盖 |
| 尺寸控制 | `size` number prop | 比 small/default/large 三档更精确 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Spinner 代码时，应遵守：

1. 从 `@/components/blocks/spinner` 导入 Spinner。
2. 使用 `variant` 指定变体：`"default"`、`"throbber"`、`"pinwheel"`、`"circle-filled"`、`"ellipsis"`、`"ring"`、`"bars"`、`"infinite"`。
3. 在项目内统一使用 1-2 种变体，不要混用多种。
4. 通过 `className="text-*"` 控制颜色（组件使用 `currentColor`）。
5. 通过 `size` 控制尺寸（数字像素值），不要用 className 覆盖宽高。
6. 不要在 Spinner 上设置 `role` 或 `aria-label`（组件已内置）。
7. 不要在 Spinner 外部包裹额外的动画容器。
8. 异步容器（如 Button、Card）应设置 `aria-busy="true"` 配合 Spinner 使用。
9. 不要尝试从 Ant Design 导入 Spin 组件来替代本地 Spinner。
10. 默认变体已经够用，无特殊原因不要指定 variant。
