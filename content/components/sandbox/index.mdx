Sandbox 沙箱预览。基于 `@codesandbox/sandpack-react` 的实时代码编辑与预览复合组件，封装了自建的 Tab 切换系统，在同一个容器中切换代码编辑器、文件浏览器、预览和控制台面板。设计目标是服务文档站点的代码演示。

<h2 id="when-to-use">何时使用</h2>

在文档或页面中内嵌可编辑的实时代码示例时使用。

常见场景：
- 组件文档站点的交互式代码演示。
- 技术博客中的可运行代码示例。
- Playground 式的代码试验环境。

推荐层级结构：
SandboxProvider（最外层，配置 files、template） > SandboxLayout > SandboxTabs > SandboxTabsList + SandboxTabsTrigger + SandboxTabsContent（内嵌 SandboxCodeEditor / SandboxPreview / SandboxConsole / SandboxFileExplorer）

不推荐：
- 在生产环境用户界面中使用（Sandpack 加载较重）。
- 在 SandboxTabs 外部使用 TabsTrigger 或 TabsContent。
- 混用受控/非受控模式（value 和 defaultValue 二选一）。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

### 基础用法

<ComponentShowcase id="sandbox" path="sandbox/basic" />

```tsx
import {
  SandboxCodeEditor,
  SandboxLayout,
  SandboxPreview,
  SandboxProvider,
  SandboxTabs,
  SandboxTabsContent,
  SandboxTabsList,
  SandboxTabsTrigger,
} from "@/components/ui/sandbox"

const files = { "/App.tsx": `export default () => <h1>Hello Sandbox!</h1>` }

export default function Demo() {
  return (
    <SandboxProvider files={files} template="react-ts">
      <SandboxLayout>
        <SandboxTabs defaultValue="preview">
          <SandboxTabsList>
            <SandboxTabsTrigger value="code">Code</SandboxTabsTrigger>
            <SandboxTabsTrigger value="preview">Preview</SandboxTabsTrigger>
          </SandboxTabsList>
          <SandboxTabsContent value="code">
            <SandboxCodeEditor />
          </SandboxTabsContent>
          <SandboxTabsContent value="preview">
            <SandboxPreview />
          </SandboxTabsContent>
        </SandboxTabs>
      </SandboxLayout>
    </SandboxProvider>
  )
}
```

```tsx
import {

  SandboxTabsList,
  SandboxTabsTrigger,

export default function Demo() {
  return (
    <SandboxProvider template="react-ts" files={{ "/App.tsx": `export default () => <p>Code Playground</p>` }}>
      <SandboxLayout>
        <SandboxTabs defaultValue="editor">
          <SandboxTabsList>
            <SandboxTabsTrigger value="files">Files</SandboxTabsTrigger>

            <SandboxTabsTrigger value="console">Console</SandboxTabsTrigger>

```

<h2 id="api">API</h2>

### SandboxProvider

继承 `SandpackProviderProps`（来自 `@codesandbox/sandpack-react`）。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| files | 文件映射，键为路径，值为文件内容 | `Record<string, string>` | 必填 |
| template | Sandpack 模板类型 | `SandpackTemplate` | 必填 |
| className | 附加样式类名 | `string` | - |

### SandboxLayout

继承 `SandpackLayoutProps`。默认强制 `!rounded-none !border-none !bg-transparent !h-full`，移除 Sandpack 默认圆角和边框。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

### SandboxTabs

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| defaultValue | 默认激活的 Tab 值（非受控） | `string` | - |
| value | 当前激活的 Tab 值（受控） | `string` | - |
| onValueChange | Tab 切换回调 | `(value: string) => void` | - |
| className | 附加样式类名 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。

### SandboxTabsList

继承 `HTMLAttributes<HTMLDivElement>`。`role="tablist"`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

### SandboxTabsTrigger

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | Tab 唯一标识（必填） | `string` | 必填 |
| className | 附加样式类名 | `string` | - |

继承 `Omit<ButtonHTMLAttributes<HTMLButtonElement>, "onClick">`。`role="tab"`，`data-state` 为 `"active"`/`"inactive"`，`aria-selected` 自动绑定。

### SandboxTabsContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 对应 Tab 的唯一标识（必填） | `string` | 必填 |
| className | 附加样式类名 | `string` | - |

继承 `HTMLAttributes<HTMLDivElement>`。`role="tabpanel"`，非激活态通过 `pointer-events-none absolute h-0 w-0 opacity-0` 隐藏（非条件渲染，保持 Sandpack 实例存活）。

### SandboxCodeEditor

继承 `CodeEditorProps`。默认 `showTabs={false}`（隐藏 Sandpack 内部文件 Tab）。

### SandboxConsole

继承 `Parameters<typeof SandpackConsole>[0]`。默认 `className="h-full"`。

### SandboxPreview

继承 `PreviewProps`，额外添加 `className`。默认 `showOpenInCodeSandbox={false}`。

### SandboxFileExplorer

继承 `ComponentProps<typeof SandpackFileExplorer>`。默认 `autoHiddenFiles={true}`。

<h2 id="faq">常见问题</h2>

### SandboxTabs 的内容切换和普通 Tabs 有什么不同？

SandboxTabsContent 使用绝对定位隐藏而非条件渲染（`pointer-events-none absolute h-0 w-0 opacity-0`），这样切换 Tab 时 Sandpack 实例不会销毁重建，代码编辑状态得以保持。

### template 支持哪些模板？

继承自 Sandpack 的模板系统。常用：`"react-ts"`（React + TypeScript）、`"vue-ts"`（Vue + TypeScript）、`"static"`（静态 HTML/CSS/JS）、`"vanilla-ts"`（原生 TypeScript）。

### 可以自定义 Sandpack 的主题吗？

Sandpack 主题通过 SandpackProvider 的 `theme` 属性配置。Sandbox 组件本身不封锁 Sandpack 的原生 props。

### 为什么这个组件不用于生产环境？

Sandpack 在浏览器内编译代码，加载体积较大，适合文档站点和 demo 场景，不适合面向终端用户的生产界面。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/sandbox` 导入所需子组件。
2. SandboxProvider 必须提供 `files` 和 `template`，包裹 SandboxLayout，SandboxLayout 包裹 SandboxTabs。
3. SandboxTabs 内部按顺序放置 SandboxTabsList 和 SandboxTabsContent，TabsTrigger 的 `value` 必须与对应 TabsContent 的 `value` 一致。
4. 代码编辑器用 SandboxCodeEditor，预览用 SandboxPreview，文件浏览用 SandboxFileExplorer，控制台用 SandboxConsole。
5. 不要在 SandboxTabs 外部使用 SandboxTabsTrigger 或 SandboxTabsContent。
6. 不要为 SandboxCodeEditor 设置 `showTabs={true}`，不要为 SandboxPreview 设置 `showOpenInCodeSandbox={true}`。
7. 不要引入 Ant Design 相关依赖。

<h2 id="semantic-dom">Semantic DOM</h2>

组件通过 `data-state` 属性暴露 Tab 激活状态（`"active"`/`"inactive"`），通过 `role` 属性暴露 ARIA 语义（`tablist`、`tab`、`tabpanel`），`aria-selected` 和 `aria-hidden` 自动绑定。
