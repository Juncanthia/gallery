文件上传 FileUpload。无头复合组件，通过 Dropzone/Trigger/List/Item 等子组件自由组合，内置拖拽上传、粘贴上传、文件校验、进度展示与可访问性播报。

<h2 id="when-to-use">何时使用</h2>

当需要用户选择或拖拽文件，并对文件类型、大小、数量做前端校验，同时展示文件列表和上传进度时使用。

常见场景：
- 表单中的附件上传（简历、合同、图片素材）。
- 需要拖拽 + 点击双入口、并展示上传进度的场景。
- 需要自定义文件预览、校验规则或上传状态渲染的场景。

不推荐：
- 只需要一个开箱即用、带业务视觉（图标动画、装饰边框）的文档上传卡片——优先查看业务层是否已有类似封装（如 `@/components/document/components/ui/file-upload`），不要重复造轮子。
- 需要断点续传、分片上传等复杂传输协议——本组件只负责选择、校验和状态展示，`onUpload` 回调里自行对接传输层。

### 基础用法

<ComponentShowcase id="file-upload" path="file-upload/basic" />

```tsx
import { useState } from "react"
import {
  FileUpload,
  FileUploadDropzone,
  FileUploadTrigger,
  FileUploadList,
  FileUploadItem,
  FileUploadItemPreview,
  FileUploadItemMetadata,
  FileUploadItemDelete,
  FileUploadClear,
} from "@/components/ui/file-upload"
import { Button } from "@/components/ui/button"
import { XIcon } from "lucide-react"

export default function Demo() {
  const [files, setFiles] = useState<File[]>([])

  return (
    <FileUpload
      value={files}
      onValueChange={setFiles}
      accept="image/*,.pdf"
      maxFiles={5}
      maxSize={5 * 1024 * 1024}
      multiple
    >
      <FileUploadDropzone>
        <p>拖拽文件到此处，或点击选择</p>
        <FileUploadTrigger asChild>
          <Button variant="outline" size="sm">浏览文件</Button>
        </FileUploadTrigger>
      </FileUploadDropzone>
      <FileUploadList>
        {files.map((file) => (
          <FileUploadItem key={file.name} value={file}>
            <FileUploadItemPreview />
            <FileUploadItemMetadata />
            <FileUploadItemDelete asChild>
              <Button variant="ghost" size="icon" className="size-7">
                <XIcon />
              </Button>
            </FileUploadItemDelete>
          </FileUploadItem>
        ))}
      </FileUploadList>
      <FileUploadClear asChild>
        <Button variant="outline" size="sm">清空全部</Button>
      </FileUploadClear>
    </FileUpload>
  )
}
```

### 服务端上传与进度

```tsx
<FileUpload
  onUpload={async (files, { onProgress, onSuccess, onError }) => {
    await Promise.all(
      files.map(async (file) => {
        try {
          for (let progress = 0; progress <= 100; progress += 20) {
            await new Promise((resolve) => setTimeout(resolve, 150))
            onProgress(file, progress)
          }
          onSuccess(file)
        } catch (error) {
          onError(file, error instanceof Error ? error : new Error("上传失败"))
        }
      })
    )
  }}
>
  {/* ...同上，FileUploadItem 内部加 FileUploadItemProgress */}
</FileUpload>
```

<h2 id="api">API</h2>

### FileUpload

根组件，管理受控/非受控文件列表、拖拽状态、校验和上传编排。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 受控文件列表 | `File[]` | - |
| defaultValue | 非受控初始文件列表 | `File[]` | - |
| onValueChange | 文件列表变化回调（新增或删除后触发） | `(files: File[]) => void` | - |
| onAccept | 一批文件通过校验后触发，携带本批被接受的文件 | `(files: File[]) => void` | - |
| onFileAccept | 单个文件通过校验时触发 | `(file: File) => void` | - |
| onFileReject | 单个文件被拒绝时触发，携带拒绝原因 | `(file: File, message: string) => void` | - |
| onFileValidate | 自定义校验，返回非空字符串表示拒绝并作为提示信息 | `(file: File) => string \| null \| undefined` | - |
| onUpload | 上传编排回调，通过 `onProgress/onSuccess/onError` 驱动每个文件的状态 | `(files: File[], options) => Promise<void> \| void` | - |
| accept | 接受的文件类型，逗号分隔（MIME、扩展名或 `type/*` 通配） | `string` | - |
| maxFiles | 最大文件数量，超出的文件会被拒绝 | `number` | - |
| maxSize | 单文件最大体积（字节） | `number` | - |
| multiple | 是否允许选择多个文件 | `boolean` | `false` |
| disabled | 禁用整个上传器 | `boolean` | `false` |
| invalid | 外部受控的校验失败态（用于展示错误样式） | `boolean` | `false` |
| required | 隐藏的原生 `<input type="file">` 是否必填 | `boolean` | `false` |
| dir | 文字方向，未传时自动继承最近的 Radix `Direction` | `"ltr" \| "rtl"` | - |
| label | 无障碍标签文案（视觉隐藏） | `string` | `"File upload"` |
| name | 隐藏 input 的 `name` 属性，用于原生表单提交 | `string` | - |
| asChild | 根节点渲染委托给子元素 | `boolean` | - |

### FileUploadDropzone

可点击、可拖拽、可粘贴的拖放区域，点击非 Trigger 区域也会打开文件选择框。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 渲染委托给子元素 | `boolean` | - |

暴露的状态数据属性：`data-dragging`（拖拽悬停中）、`data-invalid`（本次校验失败，2 秒后自动清除）、`data-disabled`。

### FileUploadTrigger

独立的"浏览文件"触发器按钮，可与 `FileUploadDropzone` 组合，也可单独使用。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 渲染委托给子元素（如包一层 `Button`） | `boolean` | - |

### FileUploadList

文件列表容器，默认只在有文件时渲染，带进入/离开动画。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| orientation | 排列方向 | `"horizontal" \| "vertical"` | `"vertical"` |
| forceMount | 无文件时也强制渲染 | `boolean` | `false` |
| asChild | 渲染委托给子元素 | `boolean` | - |

### FileUploadItem

单个文件的容器，需要传入对应的 `File` 对象作为 `value`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 对应的 `File` 对象 | `File` | 必填 |
| asChild | 渲染委托给子元素 | `boolean` | - |

### FileUploadItemPreview

文件预览：图片文件自动生成 blob URL 缩略图，其他类型回退到按扩展名/MIME 选择的图标。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| render | 自定义渲染，`fallback()` 可拿到默认预览内容 | `(file: File, fallback: () => ReactNode) => ReactNode` | - |
| asChild | 渲染委托给子元素 | `boolean` | - |

### FileUploadItemMetadata

展示文件名、大小与错误信息；不传 `children` 时使用内置布局。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| size | 文字尺寸 | `"default" \| "sm"` | `"default"` |
| asChild | 渲染委托给子元素 | `boolean` | - |

### FileUploadItemProgress

上传进度指示器，上传完成（`progress === 100`）后自动隐藏，除非 `forceMount`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 展示样式 | `"linear" \| "circular" \| "fill"` | `"linear"` |
| size | `circular` 变体的直径（px） | `number` | `40` |
| forceMount | 完成后仍然渲染 | `boolean` | `false` |
| asChild | 渲染委托给子元素 | `boolean` | - |

### FileUploadItemDelete / FileUploadClear

`FileUploadItemDelete` 删除单个文件；`FileUploadClear` 清空全部文件（默认仅在有文件时渲染）。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 渲染委托给子元素 | `boolean` | - |
| forceMount（仅 Clear） | 无文件时也渲染 | `boolean` | `false` |

### useFileUpload

选择器 hook，读取 `FileUpload` 内部 store（基于 `useSyncExternalStore`），用于在自定义子组件中访问文件状态。

```tsx
const fileCount = useFileUpload((state) => state.files.size)
```

<h2 id="faq">常见问题</h2>

### 如何渲染文件列表？

`FileUploadList` 的 `children` 是普通 `ReactNode`，不是 render prop。需要自己维护文件数组（推荐用受控 `value`/`onValueChange`）并 `map` 成 `FileUploadItem`：

```tsx
<FileUploadList>
  {files.map((file) => (
    <FileUploadItem key={file.name} value={file}>...</FileUploadItem>
  ))}
</FileUploadList>
```

### 如何限制文件类型和大小？

使用 `accept`（逗号分隔的 MIME/扩展名/通配符）和 `maxSize`（字节）：

```tsx
<FileUpload accept="image/*,.pdf" maxSize={5 * 1024 * 1024} />
```

### 如何对接真实的上传接口？

通过 `onUpload` 回调，在其中调用你的上传 API，并通过 `onProgress`/`onSuccess`/`onError` 驱动每个文件的进度和状态：

```tsx
<FileUpload
  onUpload={async (files, { onProgress, onSuccess, onError }) => {
    for (const file of files) {
      try {
        await uploadToServer(file, (percent) => onProgress(file, percent))
        onSuccess(file)
      } catch (error) {
        onError(file, error as Error)
      }
    }
  }}
/>
```

### 和 `@/components/document/components/ui/file-upload` 是什么关系？

两者不是两套互相独立的实现，而是 primitive 与预设（preset）的关系：`extend` 的 `FileUpload` 内部就是用本组件的 `FileUpload`/`FileUploadDropzone`/`useFileUpload` 搭建的，拖拽状态机、accept/大小校验、键盘与粘贴上传等逻辑全部复用本组件，`extend` 只额外补充了业务视觉（图标动效、边框光束、缩略图文件列表）和"每次上传替换文件列表"的业务语义。按需要的自由度选：需要自由组合 UI 或对接真实上传进度，直接用本组件搭；需要开箱即用的文档上传卡片，用 `extend` 的封装。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/ui/file-upload` 导入，不要从 `@/components/_shared/dice/ui/file-upload` 或 `@/components/blocks/file-upload` 导入。
2. `FileUploadList` 的 children 必须是普通 `ReactNode`（通常是 `files.map(...)` 的结果），不是 render prop。
3. `FileUploadItem` 必须传入 `value`（对应的 `File` 对象），否则无法关联到内部文件状态。
4. 优先使用受控模式（`value` + `onValueChange`）维护文件数组。
5. 需要展示进度时使用 `FileUploadItemProgress`，并在 `onUpload` 回调中调用 `onProgress` 更新进度。
6. 需要自定义触发按钮样式时，用 `FileUploadTrigger asChild` 包裹项目的 `Button` 组件，不要重新实现点击打开文件选择框的逻辑。
7. 不要引入 Ant Design 的 `Upload`/`rc-upload`，也不要引入 `react-dropzone` 等第三方拖拽库。
8. 业务专属的文档缩略图、图标动画、边框光束等视觉效果属于业务层能力，不要塞进本组件；如需要开箱即用的文档上传卡片，改用 `@/components/document/components/ui/file-upload`。
