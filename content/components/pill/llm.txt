# Pill 药丸标签

胶囊形标签复合组件套件，提供 `Pill`（基础标签）、`PillAvatar`（头像）、`PillButton`（操作按钮）、`PillStatus`（状态区块）、`PillIndicator`（状态圆点）、`PillDelta`（增量箭头）和 `PillIcon`（图标）以及 `PillAvatarGroup`（头像组）等子组件。这些组件可自由组合，构建紧凑的信息标记和状态展示。

本组件为项目本地复合组件（Blocks），无 Ant Design 直接对应。Pill 的设计目标与 AntD `Tag` 不同：Tag 是有边框的矩形标签，Pill 是圆角胶囊形，更适合承载头像、状态等高密度信息。API 和行为以 `src/components/blocks/pill/index.tsx` 源码为准。

## 何时使用 {#when-to-use}

当需要以胶囊形标签展示紧凑信息或状态时使用。

常见场景：

- 用户标签（头像 + 名称 + 状态指示）。
- 数据指标（标签 + 增量 + 数值）。
- 过滤器/选项标签（可关闭的胶囊标签）。
- 状态标记（在线/离线/警告）。
- 紧凑属性展示（带图标的指标项）。

不推荐：

- 需要矩形边框的传统标签——使用 `Badge` 或 `Tag` 组件。
- 需要计数气泡——使用 `Badge` 组件的 dot/count 模式。
- 为了照搬 Ant Design Tag API 而引入本组件——Pill 与 Tag 的设计语言不同。

## 代码演示 {#examples}

### 基础用法

```tsx
import { Pill } from "@/components/blocks/pill"

export default function Demo() {
  return (
    <div className="flex gap-2">
      <Pill>默认标签</Pill>
      <Pill variant="solid">实心标签</Pill>
      <Pill color="primary" variant="solid">主色标签</Pill>
      <Pill color="destructive" variant="soft">危险标签</Pill>
    </div>
  )
}
```

### 状态指示组合

```tsx
import { Pill, PillIndicator, PillDelta, PillButton } from "@/components/blocks/pill"

export default function Demo() {
  return (
    <div className="flex gap-2">
      <Pill>
        <PillIndicator variant="success" pulse />
        在线
      </Pill>
      <Pill>
        <PillIndicator variant="warning" />
        繁忙
        <PillButton>×</PillButton>
      </Pill>
      <Pill>
        销售额
        <PillDelta delta={12} />
      </Pill>
    </div>
  )
}
```

### 带头像的标签

```tsx
import { Pill, PillAvatar, PillStatus } from "@/components/blocks/pill"

export default function Demo() {
  return (
    <Pill>
      <PillAvatar src="/avatar.jpg" fallback="张" />
      <PillStatus>
        <PillIndicator variant="success" />
        张三
      </PillStatus>
      管理员
    </Pill>
  )
}
```

## 核心导出 {#exports}

- `Pill` — 基础胶囊标签（包装 Badge）
- `PillAvatar` — 小型头像（包装 Avatar）
- `PillButton` — 标签内操作按钮（包装 Button）
- `PillStatus` — 状态区块（左侧带分隔线）
- `PillIndicator` — 状态圆点（支持颜色和脉冲动画）
- `PillDelta` — 增量箭头（正/负/零三种状态）
- `PillIcon` — 图标包装（统一尺寸和颜色）
- `PillAvatarGroup` — 头像组（重叠排列 + 径向遮罩）
- 所有对应 `*Props` 类型

## API {#api}

### Pill

基础的胶囊形标签，内部使用 `Badge` 组件渲染，添加 `rounded-full` 和标准内边距。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| variant | `BadgeVariant` | `"secondary"` | 标签视觉变体，透传给 Badge |
| themed | `boolean` | `false` | 是否使用主题色（对应 Badge 的 themed 模式） |
| className | `string` | - | 附加 className |
| children | `ReactNode` | - | 标签内容 |
| （其他） | `ComponentProps<typeof Badge>` | - | 支持 Badge 组件的全部属性（如 `color`） |

### PillAvatar

标签内的小型头像，默认尺寸 16px × 16px（`h-4 w-4`），带有负左边距（`-ml-1`）与标签边缘对齐。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| fallback | `string` | - | 头像加载失败时的回退文本（通常为姓名首字母） |
| className | `string` | - | 附加 className |
| （其他） | `ComponentProps<typeof AvatarImage>` | - | 支持 AvatarImage 的全部属性（如 `src`、`alt`） |

### PillButton

标签内的小型操作按钮，默认尺寸 24px × 24px（`size-6`），圆形。负外边距（`-my-2 -mr-2`）使其在标签内对齐。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| className | `string` | - | 附加 className |
| children | `ReactNode` | - | 按钮内容（通常为 × 图标） |
| （其他） | `ComponentProps<typeof Button>` | - | 支持 Button 组件的全部属性 |

默认设置为 `shape="square" variant="text"`，hover 时显示微弱背景。

### PillStatus

标签内的状态区块，带有右侧分隔线。通常用于放置状态指示器 + 文本。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| children | `ReactNode` | - | 状态内容（通常为 PillIndicator + 文本） |
| className | `string` | - | 附加 className |

### PillIndicator

状态圆点指示器，支持四种颜色和可选的脉冲动画。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| variant | `"success" \| "error" \| "warning" \| "info"` | `"success"` | 颜色变体 |
| pulse | `boolean` | `false` | 是否显示脉冲动画（`animate-ping`） |

颜色映射：

| variant | 颜色 | CSS 类 |
| --- | --- | --- |
| success | 翠绿色 | `bg-emerald-500` / `bg-emerald-400`（脉冲） |
| error | 玫瑰红色 | `bg-rose-500` / `bg-rose-400`（脉冲） |
| warning | 琥珀色 | `bg-amber-500` / `bg-amber-400`（脉冲） |
| info | 天蓝色 | `bg-sky-500` / `bg-sky-400`（脉冲） |

### PillDelta

增量方向指示图标，根据 delta 值显示上箭头（正）、下箭头（负）或横线（零）。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| delta | `number` | 必填 | 增量数值。>0 绿色上箭头，<0 红色下箭头，=0 灰色横线 |
| className | `string` | - | 附加 className |

### PillIcon

统一尺寸的图标包装，默认 `size-3`（12px），颜色为 `text-muted-foreground`。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| icon | `typeof ChevronUpIcon` | 必填 | lucide-react 图标组件类型 |
| className | `string` | - | 附加 className |

### PillAvatarGroup

头像组容器，将多个 PillAvatar 重叠排列（`-space-x-1`），并使用径向渐变遮罩（`mask-image`）实现平滑的重叠边界过渡效果。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| children | `ReactNode` | - | 头像列表（通常为多个 PillAvatar） |
| className | `string` | - | 附加 className |

## 样式模型 {#styling-model}

- 使用 Tailwind CSS 和项目主题变量。
- `Pill`：基于 `Badge` 组件，添加 `rounded-full`（完全圆角胶囊形）、`px-3 py-1.5`（标准内边距）、`gap-2`（子元素间距）和 `font-normal`（正常字重）。
- `PillAvatar`：`h-4 w-4`（16px 正方形），`-ml-1`（向左偏移与标签圆角对齐）。
- `PillButton`：`size-6 rounded-full`（24px 圆形），`-my-2 -mr-2`（负外边距消除标签内边距影响），hover 时 `bg-foreground/5`。
- `PillStatus`：`flex items-center gap-2 border-r pr-2 font-medium`，右侧带分隔线。
- `PillIndicator`：`relative flex size-2`（8px 圆点），脉冲层使用 `animate-ping`，颜色通过 variant 映射。
- `PillDelta`：`size-3`（12px 图标），正数翠绿色、负数玫瑰红色、零灰色。
- `PillIcon`：`size-3 text-muted-foreground`（12px 统一尺寸和颜色）。
- `PillAvatarGroup`：`-space-x-1`（重叠排列），非第一个头像使用 `mask-image: radial-gradient(circle 9px at -4px 50%, transparent 99%, white 100%)` 实现平滑的相交边界。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/pill/index.tsx`

直接依赖摘要：

- `@/components/ui/avatar`（Avatar、AvatarImage、AvatarFallback）
- `@/components/ui/badge`（Badge）
- `@/components/ui/button`（Button）
- `lucide-react`（ChevronDownIcon、ChevronUpIcon、MinusIcon）
- `@/lib/utils`（cn）

实现注意：

1. `Pill` 是对 `Badge` 的样式增强，添加了胶囊形圆角和标准内边距。所有 Badge 的属性（`color`、`variant`、`size` 等）均可透传。
2. `PillAvatar` 使用 `Avatar` + `AvatarImage` + `AvatarFallback` 组合，尺寸缩小为 16px。
3. `PillButton` 使用负外边距技巧，使按钮在标签内紧密贴合右侧边缘。
4. `PillStatus` 使用 `border-r` 分隔线，将状态区域与标签其余内容视觉分离。
5. `PillIndicator` 的脉冲动画使用两层 `<span>`：外层 `animate-ping` 带透明度，内层为实色圆点。
6. `PillAvatarGroup` 使用 CSS `mask-image` 径向渐变实现头像重叠时的平滑遮罩效果。
7. 所有子组件均为纯样式包装，无状态管理，可自由组合。

## 可访问性 {#accessibility}

- `Pill` 基于 `Badge` 组件，渲染为 `<span>` 或 `<div>`，不具备交互语义。
- `PillButton` 基于 `Button` 组件，渲染为 `<button>`，支持键盘聚焦和点击。
- `PillAvatar` 中的 `AvatarFallback` 提供图像加载失败时的文本回退。
- `PillIndicator` 的颜色语义通过 variant 属性映射，非仅依赖颜色传达信息——建议配合文本标签使用。
- `PillAvatarGroup` 的重叠排列是纯视觉的，DOM 顺序与视觉顺序一致。
- 推荐在可关闭的标签中为 `PillButton` 提供 `aria-label`（如 `aria-label="移除"`）。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。Ant Design 提供 `Tag` 组件，但 Pill 的设计目标和外观不同。

| 方向 | 本地决策 |
| --- | --- |
| 形状 | 胶囊形（`rounded-full`），与 AntD Tag 的矩形圆角不同 |
| 组合模式 | 原子化子组件自由组合，非单一 Tag 组件的预设模式 |
| 状态指示 | 独立 `PillIndicator` 组件，支持脉冲动画 |
| 头像集成 | 内置 `PillAvatar` 和 `PillAvatarGroup` |
| API 来源 | 以 `src/components/blocks/pill/index.tsx` 为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 与 AntD Tag 关系 | 不同设计语言，不做 API 对齐 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Pill 代码时，应遵守：

1. 从 `@/components/blocks/pill` 导入所需的子组件。
2. `Pill` 是基础容器，支持 Badge 的全部属性（`color`、`variant` 等）。
3. 子组件可自由组合：`PillAvatar` + `PillStatus`（头像+状态）、`PillIndicator` + `PillDelta`（状态指示+增量）、`Pill` + `PillButton`（可关闭标签）。
4. `PillIndicator` 的 `pulse` 属性仅用于需要强调的实时状态（如"在线"），不要在静态信息上使用。
5. `PillDelta` 的 `delta` 为 0 时显示灰色横线，为正时绿色上箭头，为负时红色下箭头。
6. `PillAvatarGroup` 内应放置 `PillAvatar`，利用径向遮罩实现平滑重叠效果。
7. 只使用本文件 API 表格和源码中存在的 props，不要凭 AntD Tag 的记忆生成 API。
8. 不要将 `Pill` 与 `Tag` 或 `Badge` 混淆——Pill 是胶囊形信息标签，Tag 是矩形属性标签。
