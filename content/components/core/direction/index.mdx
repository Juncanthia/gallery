文本方向 Direction。提供 RTL/LTR 文本方向切换能力，基于 Radix Direction 实现。包含 `DirectionProvider` 组件提供方向上下文和 `useDirection` hook 获取当前方向。

<h2 id="when-to-use">何时使用</h2>

当需要支持从右到左（RTL）的文本方向，或需要在组件树中根据方向条件性地调整布局和样式时使用。

常见场景：

- 国际化应用中阿拉伯语、希伯来语等 RTL 语言的布局适配。
- 条件性地根据文本方向调整箭头、图标等方向性元素的朝向。
- 在应用顶层或局部区域切换文本方向。
- 需要感知当前方向以正确渲染 UI 元素的组件。

不推荐：

- 手动给元素添加 `dir="rtl"` 属性来绕过 DirectionProvider，会丢失 Context 传递能力。
- 在不需要 RTL 支持的应用中引入，增加不必要的复杂度。
- 混淆 `direction` 和 `dir` 两个 prop（`direction` 是 `dir` 的别名，同时传递时 `direction` 优先）。

<h2 id="examples">代码演示</h2>

### 基础用法

`DirectionProvider` 设置 `dir` 属性，子组件通过 `useDirection` 获取当前方向。

<ComponentShowcase id="direction" path="direction/basic" />

```tsx
import { DirectionProvider, useDirection } from "@/components/core/direction"

function DirDisplay() {
  const dir = useDirection()
  return (
    <span className="rounded bg-muted px-2 py-1 font-mono text-sm">
      当前方向：{dir}
    </span>
  )
}

export default function Demo() {
  return (
    <div className="space-y-4">
      <div>
        <p className="mb-2 text-sm text-muted-foreground">LTR（默认）</p>
        <DirectionProvider dir="ltr">
          <div className="flex items-center gap-2 rounded border p-3">
            <span className="text-sm">-></span>
            <DirDisplay />
            <span className="text-sm">-></span>
          </div>
        </DirectionProvider>
      </div>
      <div>
        <p className="mb-2 text-sm text-muted-foreground">RTL</p>
        <DirectionProvider dir="rtl">
          <div className="flex items-center gap-2 rounded border p-3">
            <span className="text-sm"><-</span>
            <DirDisplay />
            <span className="text-sm"><-</span>
          </div>
        </DirectionProvider>
      </div>
    </div>
  )
}
```

### 条件性布局

根据当前方向调整组件渲染。

```tsx
import { DirectionProvider, useDirection } from "@/components/core/direction"

function Arrow() {
  const dir = useDirection()
  return <span>{dir === "rtl" ? "<-" : "->"}</span>
}

export default function Demo() {
  return (
    <DirectionProvider dir="rtl">
      <div className="flex items-center gap-2">
        <span>返回</span>
        <Arrow />
      </div>
    </DirectionProvider>
  )
}
```

<h2 id="api">API</h2>

### DirectionProvider

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| dir | 文本方向，透传至 Radix Direction.DirectionProvider | `"ltr" \| "rtl"` | - |
| direction | `dir` 的别名，同时传递时优先级高于 `dir` | `"ltr" \| "rtl"` | - |
| children | 子组件树 | `React.ReactNode` | - |

包裹在需要方向感知的组件树外层。内部使用 Radix `Direction.DirectionProvider`，浏览器会基于 HTML `dir` 属性自动应用对应的 CSS `direction` 和文本布局规则。

### useDirection

Hook，返回当前 Direction 上下文的 `dir` 值（`"ltr"` 或 `"rtl"`），无参数。基于 Radix `Direction.useDirection`。

<h2 id="faq">常见问题</h2>

### `dir` 和 `direction` 有什么区别？

`direction` 是 `dir` 的别名。同时传递两者时 `direction` 优先。推荐使用 `dir` 与 HTML 标准属性保持一致。

### 支持嵌套 DirectionProvider 吗？

支持。子级 `DirectionProvider` 会覆盖父级的方向设置，不影响兄弟节点。

### `useDirection` 在 `DirectionProvider` 外部调用会怎样？

返回 Radix Direction 上下文的默认值。建议确保调用 `useDirection` 的组件在 `DirectionProvider` 子树内。

### Direction 与 CSS 的 `direction` 属性有何关系？

`DirectionProvider` 设置的是 HTML `dir` 属性。浏览器会基于此自动应用对应的 CSS `direction` 和文本布局规则（文字对齐、内联方向等），无需额外设置 CSS。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/direction` 导入 `DirectionProvider` 和 `useDirection`。
2. 在应用顶层或需要 RTL 的局部区域使用 `DirectionProvider` 包裹。
3. 子组件中通过 `useDirection()` 获取当前方向，用于条件样式或布局。
4. 基于 Radix Direction，不依赖 Ant Design 的 `ConfigProvider.direction`。
5. 不要手动设置 `dir` HTML 属性来绕过 Provider。
6. 不要引入 Ant Design 相关依赖。
