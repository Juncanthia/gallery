按钮组 ButtonGroup。将多个按钮紧凑排列为一个视觉整体，自动处理首尾圆角和边框重叠。配合 ButtonGroupSeparator 和 ButtonGroupText 可构建含分隔线和文本标签的复合操作区。

<h2 id="when-to-use">何时使用</h2>

当需要将多个关联按钮组合在一起形成视觉整体时使用。ButtonGroup 通过 CVA 变体自动管理子元素的圆角和边框，避免手动编写 `rounded-l-none` 等样式类。

常见场景：
- 编辑工具栏（复制、粘贴、剪切）。
- 表单操作区（提交 + 重置）。
- 带状态文本的复合操作按钮组。
- 纵向排列的选项按钮组。

不推荐：
- 将无关联或语义无关的按钮强行放入 ButtonGroup。
- 在 ButtonGroup 内部使用非标准按钮元素破坏圆角合并逻辑。
- 用多个独立按钮手写 `rounded-l-none` 等 class 来模拟组合效果。

<h2 id="api">API</h2>

### ButtonGroup

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| orientation | 排列方向 | `"horizontal"` \| `"vertical"` | `"horizontal"` |
| className | 根元素 CSS 类名 | `string` | - |
| children | 组合内容，通常为 Button、ButtonGroupSeparator、ButtonGroupText | `React.ReactNode` | - |

接受所有标准 HTML `div` 属性。根元素带有 `role="group"`、`data-slot="button-group"` 和 `data-orientation`。水平模式下自动处理子元素首尾圆角（中间元素无圆角，首尾保留对应侧圆角）。

### ButtonGroupText

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| asChild | 将样式合并到子元素，使用 Radix Slot | `boolean` | `false` |
| className | 根元素 CSS 类名 | `string` | - |
| children | 文本内容 | `React.ReactNode` | - |

接受所有标准 HTML `div` 属性。ButtonGroupText 是纯展示性的文本标签，**不是按钮**，无点击交互行为。样式为圆角边框 + 浅色背景（`border bg-muted`），适合展示状态文本如"草稿"、"已发布"等。

### ButtonGroupSeparator

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| orientation | 分隔线方向 | `"horizontal"` \| `"vertical"` | `"vertical"` |
| className | 根元素 CSS 类名 | `string` | - |

接受 `Separator` 组件的所有 props。在横向按钮组中默认渲染为竖线（`orientation="vertical"`），在纵向按钮组中改为 `orientation="horizontal"` 渲染为横线。

<h2 id="faq">常见问题</h2>

### ButtonGroupText 和 Button 的区别？

ButtonGroupText 是一个纯展示性的文本标签，不是按钮。它渲染为 `<div>`，无点击交互、hover 效果等按钮行为。适合在按钮组中展示状态文本如"草稿"、"已发布"等。需要交互时请使用 Button。

### 如何使用纵向按钮组？

设置 `orientation="vertical"`，按钮组内部的圆角和边框会自动处理：

```tsx
<ButtonGroup orientation="vertical">
  <Button variant="outlined">选项一</Button>
  <Button variant="outlined">选项二</Button>
  <Button variant="outlined">选项三</Button>
</ButtonGroup>
```

### ButtonGroupSeparator 在横向和纵向按钮组中的表现？

在横向按钮组（默认）中，Separator 渲染为竖线分割左右按钮。在纵向按钮组中，`orientation` 默认仍为 `"vertical"`，但由于父容器的 flex-col 布局，渲染效果为横线。可手动传入 `orientation="horizontal"` 明确指定。

### 可以在 ButtonGroup 中放非 Button 元素吗？

ButtonGroup 依赖 CVA 变体样式处理子元素的圆角合并（`[&>*:not(:first-child)]:rounded-l-none` 等），非标准按钮元素可能导致样式异常。如需嵌入输入框，建议使用 `ButtonGroupText` 或外层自行包裹。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/button-group` 导入 ButtonGroup、ButtonGroupSeparator、ButtonGroupText。
2. 相关按钮用 ButtonGroup 包裹，不要手动写 `rounded-l-none` 等 class。
3. ButtonGroupText 用于在按钮组中嵌入文本标签，不是按钮，无交互行为。
4. 纵向排列设置 `orientation="vertical"`。
5. 不要引入 Ant Design 相关依赖。
