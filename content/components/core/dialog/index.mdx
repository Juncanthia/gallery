对话框 Dialog。在页面正中打开浮层承载内容并锁定背景交互。基于 Radix Dialog 实现，提供完整的子组件体系和便捷 API 层，支持标题、描述、操作按钮区、遮罩控制、受控显隐和自定义宽高。

<h2 id="when-to-use">何时使用</h2>

当需要用户在模态浮层中处理事务，又不希望跳转页面打断工作流程时使用。

常见场景：

- 需要用户确认的操作（删除、提交等）。
- 在浮层中填写表单（新增/编辑数据）。
- 展示详细信息但不想跳页。
- 引导式多步骤操作。

不推荐：

- 简单的确认询问（如"确定删除？"），使用 AlertDialog 更轻量。
- 移动端大段内容展示，优先考虑 Drawer。
- 嵌套 Dialog（在对话框中再弹对话框），应合并流程或使用 Popconfirm。

<h2 id="examples">代码演示</h2>

### 基础用法

<ComponentShowcase id="dialog" path="dialog/basic" />

```tsx
import { useState } from "react"
import { Button } from "@/components/core/button"
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from "@/components/core/dialog"

export default function Demo() {
  const [open, setOpen] = useState(false)

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>打开对话框</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>确认操作</DialogTitle>
          <DialogDescription>
            确定要执行此操作吗？此操作不可撤销。
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button variant="outlined" onClick={() => setOpen(false)}>取消</Button>
          <Button variant="solid" onClick={() => setOpen(false)}>确认</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  )
}
```

### 使用便捷 API

Dialog 组件支持通过 props 直接传入 title、description、okText 等，无需手动组合子组件。

```tsx
import { useState } from "react"
import { Button } from "@/components/core/button"
import { Dialog } from "@/components/core/dialog"

export default function Demo() {
  const [open, setOpen] = useState(false)

  return (
    <Dialog
      open={open}
      onOpenChange={setOpen}
      trigger={<Button>便捷 API</Button>}
      title="确认操作"
      description="确定要执行此操作吗？"
      okText="确认"
      cancelText="取消"
      onOk={() => { console.log("ok"); setOpen(false) }}
      onCancel={() => setOpen(false)}
    />
  )
}
```

### 自定义宽度和无遮罩

```tsx
import { useState } from "react"
import { Button } from "@/components/core/button"
import { Dialog, DialogContent, DialogHeader, DialogTitle, DialogTrigger } from "@/components/core/dialog"

export default function Demo() {
  const [open, setOpen] = useState(false)

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>宽对话框</Button>
      </DialogTrigger>
      <DialogContent className="sm:max-w-2xl">
        <DialogHeader>
          <DialogTitle>宽对话框示例</DialogTitle>
        </DialogHeader>

        </div>
      </DialogContent>
    </Dialog>
  )
}
```

<h2 id="api">API</h2>

### Dialog

顶层容器组件。提供两种使用模式：**子组件模式**（手动组合子组件）和**便捷 API 模式**（通过 props 声明）。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| open | 受控显隐状态 | `boolean` | - |
| defaultOpen | 非受控默认显隐状态 | `boolean` | - |
| onOpenChange | 显隐状态变化回调 | `(open: boolean) => void` | - |
| modal | 是否为模态 | `boolean` | `true` |
| trigger | 触发器元素（便捷 API） | `React.ReactNode` | - |
| title | 对话框标题（便捷 API） | `React.ReactNode` | - |
| description | 补充描述（便捷 API） | `React.ReactNode` | - |
| footer | 底部内容，传入 `null` 隐藏底部（便捷 API） | `React.ReactNode` | - |
| okText | 确认按钮文案（便捷 API） | `React.ReactNode` | `"OK"` |
| cancelText | 取消按钮文案（便捷 API） | `React.ReactNode` | `"Cancel"` |
| confirmLoading | 确认按钮加载态（便捷 API） | `boolean` | - |
| closable | 是否显示关闭按钮（便捷 API） | `boolean` | - |
| closeIcon | 自定义关闭图标（便捷 API） | `React.ReactNode` | `<X />` |
| mask | 是否显示遮罩层（便捷 API） | `boolean` | - |
| maskClosable | 点击遮罩是否关闭（便捷 API） | `boolean` | - |
| width | 对话框宽度（便捷 API） | `number \| string` | - |
| showCloseButton | 是否显示关闭按钮（便捷 API，与 closable 等效） | `boolean` | - |
| okButtonProps | 确认按钮额外属性（便捷 API） | `ButtonProps` | - |
| cancelButtonProps | 取消按钮额外属性（便捷 API） | `ButtonProps` | - |
| contentProps | 传给 DialogContent 的额外属性（便捷 API） | `Omit<DialogContentProps, 'children'>` | - |
| onOk | 点击确认回调（便捷 API） | `React.MouseEventHandler<HTMLButtonElement>` | - |
| onCancel | 点击取消回调（便捷 API） | `React.MouseEventHandler<HTMLButtonElement>` | - |

### DialogTrigger

触发打开对话框的元素。常用 `asChild` 包裹 Button。

继承 `DialogTriggerPrimitiveProps`。

### DialogContent

对话框主体面板。固定居中定位，默认最大宽度 `sm:max-w-lg`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| closable | 是否显示关闭按钮 | `boolean` | `true` |
| closeIcon | 自定义关闭图标 | `React.ReactNode` | `<X />` |
| mask | 是否显示遮罩层 | `boolean` | `true` |
| maskClosable | 点击遮罩是否关闭 | `boolean` | `true` |
| showCloseButton | 是否显示关闭按钮（与 closable 等效） | `boolean` | - |
| width | 对话框宽度 | `number \| string` | - |
| className | 附加 className | `string` | - |

### DialogHeader / DialogTitle / DialogDescription

- **DialogHeader**：头部容器，包裹 Title 和 Description。默认 `text-center sm:text-left`。
- **DialogTitle**：对话框标题，`text-lg font-semibold`。
- **DialogDescription**：补充描述文本，`text-sm text-muted-foreground`。

### DialogFooter

底部操作区容器。默认 `flex-col-reverse sm:flex-row sm:justify-end`。

### DialogClose / DialogOverlay / DialogPortal

- **DialogClose**：关闭按钮，渲染 X 图标，位于内容区右上角。
- **DialogOverlay**：背景遮罩层，`fixed inset-0 z-50 bg-black/80`。
- **DialogPortal**：Portal 容器，将内容渲染到 body 层级。

<h2 id="faq">常见问题</h2>

### Dialog 和 AlertDialog 有什么区别？

- **Dialog**：通用模态对话框，内容灵活，适合表单、详情等自定义场景。
- **AlertDialog**：专注确认/警告型对话框，内置取消和确认按钮，用于破坏性操作（删除等）。

### 如何在关闭前拦截（如表单脏数据提示）？

使用受控模式 `open` + `onOpenChange`，在回调中加入自定义逻辑：

```tsx
<Dialog open={open} onOpenChange={(next) => {
  if (isDirty && !confirm("有未保存的更改，确定关闭？")) return
  setOpen(next)
}}>
```

### 便捷 API 和子组件模式如何选择？

便捷 API 适合标准确认/表单对话框（有标题、描述、确定的确认/取消按钮对）。子组件模式适合需要完全自定义内容布局的场景。

### 如何实现点击遮罩不关闭？

设置 `maskClosable={false}`，或在 `DialogContent` 上设置。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/dialog` 导入所需的子组件。
2. 使用受控模式管理 `open` 状态，确保可在操作完成后关闭。
3. `DialogTrigger` 使用 `asChild` 包裹 Button，避免多余 DOM 嵌套。
4. 破坏性确认操作优先使用 AlertDialog，不要用 Dialog 模拟确认框。
5. 标准确认场景推荐使用便捷 API（`trigger` + `title` + `description` + `okText` + `cancelText`）。
6. 不要引入 Ant Design 的 `Modal` 或 `antd` 相关依赖。
