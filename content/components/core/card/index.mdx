Card 通用卡片容器，用于信息分组和展示。提供标题、描述、内容、操作区、封面图和内嵌 Tab 等结构化的子组件，支持加载骨架、悬浮效果和尺寸/变体控制。

<h2 id="when-to-use">何时使用</h2>

当需要在页面中将一组相关信息聚合展示时使用。Card 是最常用的信息容器之一。

常见场景：

- 列表页的条目卡片（如商品卡片、文章卡片）。
- 仪表盘的数据指标卡片。
- 设置页面的分组面板。
- 详情页的信息区块。
- 内容流的 Feed 卡片。

与单纯用 `div` + `border` 的区别：Card 提供统一间距（`--card-spacing` 变量）、结构语义（`data-slot`）和暗色模式适配。

### 基础用法

Header 放标题和描述，Content 放主体，Footer 放操作按钮。

/>

```tsx
import { Card, CardContent, CardDescription, CardFooter, CardHeader, CardTitle } from "@/components/core/card"
import { Button } from "@/components/core/button"

export default function Demo() {
  return (
    <Card className="w-full max-w-sm">
      <CardHeader>
        <CardTitle>卡片标题</CardTitle>
        <CardDescription>卡片的描述信息，用于补充说明。</CardDescription>
      </CardHeader>
      <CardContent>
        <p className="text-sm text-muted-foreground">卡片主体内容区域。</p>
      </CardContent>
      <CardFooter className="flex gap-2">
        <Button size="small" variant="outlined">取消</Button>
        <Button size="small" color="primary" variant="solid">确认</Button>
      </CardFooter>
    </Card>
  )
}
```

### 带封面的卡片

`cover` 在卡片顶部展示封面图。

```tsx
import { Card, CardContent, CardHeader, CardTitle, CardDescription, CardFooter } from "@/components/core/card"
import { Button } from "@/components/core/button"

export default function Demo() {
  return (
    <Card className="w-64" cover={<img src="https://picsum.photos/320/160" alt="" className="aspect-video w-full object-cover" />}>
      <CardHeader>
        <CardTitle>风景照片</CardTitle>
        <CardDescription>拍摄于 2024 年</CardDescription>
      </CardHeader>
      <CardContent>这是一张自然风光照片。</CardContent>
      <CardFooter>
        <Button size="small" variant="text">查看详情</Button>
      </CardFooter>
    </Card>

```

### 加载状态

`loading` 显示骨架屏。

```tsx
import { Card, CardHeader, CardTitle, CardDescription, CardContent } from "@/components/core/card"

export default function Demo() {
  return (
    <Card loading className="w-64">
      <CardHeader>
        <CardTitle>标题</CardTitle>
        <CardDescription>描述</CardDescription>
      </CardHeader>
      <CardContent>内容</CardContent>
    </Card>
  )
}
```

### 带 Tab 的卡片

`tabList` 在卡片头部添加选项卡切换。

```tsx
import { Card, CardContent } from "@/components/core/card"

export default function Demo() {
  return (
    <Card
      className="w-80"
      title="设置"
      tabList={[
        { key: "general", label: "通用" },
        { key: "security", label: "安全" },
        { key: "billing", label: "账单" }
      ]}
    >
      <CardContent>选项卡内容区域。</CardContent>
    </Card>
  )
}
```

### CardMeta 元信息

`CardMeta` 在内容区展示头像+标题+描述的元信息。

```tsx
import { Card, CardContent, CardMeta } from "@/components/core/card"
import { Avatar } from "@/components/core/avatar"

export default function Demo() {
  return (
    <Card className="w-64">
      <CardContent>
        <CardMeta
          avatar={<Avatar size="large">JD</Avatar>}
          title="张三"
          description="前端开发工程师"
        />
      </CardContent>
    </Card>
  )
}
```

### Card

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| title | 卡片标题（API 模式） | `React.ReactNode` | - |
| description | 卡片描述（API 模式） | `React.ReactNode` | - |
| extra | 头部右侧额外内容 | `React.ReactNode` | - |
| cover | 封面图 | `React.ReactNode` | - |
| actions | 底部操作按钮数组 | `React.ReactNode[]` | - |

| tabBarExtraContent | Tab 栏右侧内容 | `React.ReactNode` | - |
| activeTabKey | 当前激活 Tab（受控） | `string` | - |
| defaultActiveTabKey | 默认激活 Tab | `string` | 第一项 |
| onTabChange | Tab 切换回调 | `(key: string) => void` | - |
| loading | 加载骨架屏 | `boolean` | `false` |
| hoverable | 悬浮阴影效果 | `boolean` | `false` |
| size | 尺寸 | `default` \| `sm` \| `small` | `default` |
| variant | 变体 | `outlined` \| `borderless` | `outlined` |
| type | 类型 | `inner` | - |
| bordered | 是否显示边框（`borderless` 快捷方式） | `boolean` | - |
| headStyle / bodyStyle | 头部/主体内联样式 | `CSSProperties` | - |
| headerClassName / bodyClassName | 头部/主体附加 className | `string` | - |
| actionsClassName | 操作区附加 className | `string` | - |
| className | 附加样式 | `string` | - |

### CardHeader

头部容器，`grid` 布局自动处理标题+操作。

### CardTitle

卡片标题，使用 `font-heading text-base font-medium`。

### CardDescription

卡片描述，`text-sm text-muted-foreground`。

### CardContent

主体内容区，自动应用 `--card-spacing` 横向内边距。

### CardFooter

底部操作区，支持 `border-t` 分隔线。

### CardMeta

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| avatar | 头像 | `React.ReactNode` | - |
| title | 元信息标题 | `React.ReactNode` | - |
| description | 元信息描述 | `React.ReactNode` | - |

### CardGrid

网格子项，用于 `Card` 内部的网格布局。

<h2 id="faq">常见问题</h2>

### Card 和纯 div 的区别？

Card 提供结构化子组件和统一间距变量 `--card-spacing`，而 div 需要手动处理间距、边框、暗色模式适配。

### 如何让卡片可点击？

直接给 Card 添加 `onClick`，或使用 `asChild` 包裹为 `<a>` 或路由 Link。

### 如何使用 API 模式 vs 子组件模式？

两种方式等价：

```tsx
// API 模式
<Card title="标题" description="描述">
  <CardContent>内容</CardContent>
</Card>

// 子组件模式
<Card>
  <CardHeader>
    <CardTitle>标题</CardTitle>
    <CardDescription>描述</CardDescription>
  </CardHeader>
  <CardContent>内容</CardContent>
</Card>
```

### 如何控制卡片间距？

卡片使用 CSS 变量 `--card-spacing` 统一控制内边距。`size="sm"` 时自动缩小。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 Card 代码时：

1. 从 `@/components/core/card` 导入所需子组件。
2. 遵循 `CardHeader > CardContent > CardFooter` 的结构顺序。
3. 标题使用 `CardTitle`，描述使用 `CardDescription`，不要用裸 `<h3>` 或 `<p>`。
4. 加载状态使用 `loading` 属性，不要手动渲染骨架屏。
5. 底部操作使用 `actions` 属性或 `CardFooter`。
6. 不要引入 Ant Design 相关依赖。
