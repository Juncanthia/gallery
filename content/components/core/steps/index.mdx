Steps 步骤条，引导用户按照流程完成多步任务，支持多种展示类型和方向，动画由 motion 驱动。

<h2 id="when-to-use">何时使用</h2>

当任务需要多个步骤依次完成，且需要向用户展示当前进度时使用。

常见场景：
- 多步骤表单引导（填写信息 -> 验证身份 -> 完成注册）。
- 订单物流进度展示（已下单 -> 已发货 -> 已送达）。
- 向导式配置流程。
- 任务审核进度（提交 -> 审核中 -> 通过/驳回）。
- 安装向导的进度指示。

不推荐：
- 步骤间无先后依赖关系时使用——此时应为并列 Tab 或列表。
- 步骤数量超过 8 个时全部展示——应考虑使用 `maxCount` 折叠或改用分页表单。
- 将 Steps 用于纯信息展示（如 TL;DR 摘要）——此时使用 `Descriptions` 或普通列表更合适。

<h2 id="examples">代码演示</h2>

### 基础用法

`items` 定义步骤列表，`current` 控制当前步骤（从 0 开始）。配合 Button 实现前进/后退。

<ComponentShowcase id="steps" path="steps/basic" />

```tsx
import { useState } from "react"
import { Steps } from "@/components/core/steps"
import { Button } from "@/components/core/button"

const stepItems = [
  { title: "填写信息", description: "输入基本信息" },
  { title: "验证身份", description: "手机号或邮箱验证" },
  { title: "完成注册", description: "注册成功" },
]

export default function Demo() {
  const [current, setCurrent] = useState(1)
  return (
    <div className="w-full space-y-6">
      <Steps current={current} items={stepItems} />
      <div className="flex gap-2">
        <Button
          size="small"
          disabled={current <= 0}
          onClick={() => setCurrent(current - 1)}
        >
          上一步
        </Button>
        <Button
          size="small"
          color="primary"
          variant="solid"
          disabled={current >= 2}
          onClick={() => setCurrent(current + 1)}
        >
          下一步
        </Button>
      </div>
    </div>
  )
}
```

### 导航模式

设置 `type="navigation"` 并传入 `onChange`，步骤变为可点击跳转。

```tsx
import { useState } from "react"
import { Steps } from "@/components/core/steps"

export default function Demo() {

      type="navigation"
      current={current}
      onChange={setCurrent}

        { title: "基本信息" },
        { title: "详细信息" },
        { title: "确认提交" },
      ]}

  )
```

<h2 id="api">API</h2>

### Steps

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| items | 步骤配置列表（必填） | `StepItem[]` | - |
| current | 当前激活步骤索引（从 0 开始） | `number` | `0` |
| initial | 起始偏移量，步骤编号 = initial + index + 1 | `number` | `0` |
| direction | 排列方向 | `"horizontal"` \| `"vertical"` | `"horizontal"` |
| orientation | `direction` 的别名 | `"horizontal"` \| `"vertical"` | - |
| size | 尺寸，`"small"` 和 `"sm"` 等效 | `"default"` \| `"small"` \| `"sm"` | `"default"` |
| type | 展示类型 | `"default"` \| `"navigation"` \| `"dot"` \| `"inline"` \| `"panel"` | `"default"` |
| blocks/status | 当前步骤的默认状态 | `"wait"` \| `"process"` \| `"finish"` \| `"error"` | `"process"` |
| percent | 当前步骤进度百分比（0-100），显示环形进度 | `number` | - |
| progressDot | 圆点模式，`true` 或自定义渲染函数 | `boolean \| ProgressDotRender` | `false` |
| titlePlacement | 标题相对图标的位置 | `"horizontal"` \| `"vertical"` | `"vertical"` |
| labelPlacement | `titlePlacement` 的别名 | `"horizontal"` \| `"vertical"` | - |
| maxCount | 最大显示步骤数，超出折叠 | `number` | - |
| onChange | 步骤点击回调（type="navigation" 时有效） | `(index: number) => void` | - |

### StepItem

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| key | 唯一标识 | `string \| number` |
| title | 步骤标题 | `React.ReactNode` |
| description | 步骤描述（显示在标题下方） | `React.ReactNode` |
| content | 步骤额外内容（与 description 互斥，content 优先） | `React.ReactNode` |
| subTitle | 副标题 | `React.ReactNode` |
| icon | 自定义步骤图标（process 状态下显示） | `React.ReactNode` |
| blocks/status | 单独指定该步骤的状态，优先级高于位置推算 | `StepStatus` |
| disabled | 是否禁用点击交互 | `boolean` |

### StepStatus

`"wait"` | `"process"` | `"finish"` | `"error"`

### StepsType

`"default"` | `"navigation"` | `"dot"` | `"inline"` | `"panel"`

<h2 id="faq">常见问题</h2>

### 如何标记某步骤为失败？

在对应 `StepItem` 中设置 `blocks/status: "error"`，该步骤图标会变为 X 图标且文字显示为红色。

### `type` 属性的各模式有什么区别？

| 类型 | 说明 |
| --- | --- |
| `"default"` | 标准步骤条，圆形数字图标，只读展示 |
| `"navigation"` | 步骤可点击导航，配合 `onChange` 实现点击跳转 |
| `"dot"` | 点状步骤条（等价于 `progressDot: true`），适合空间紧凑场景 |
| `"inline"` | 内联模式，步骤以带边框卡片形式展示 |
| `"panel"` | 面板模式，步骤带有背景面板 |

### `current` 是从 0 还是 1 开始？

`current` 从 0 开始，`current={0}` 表示第一个步骤为当前步骤。步骤图标上的数字默认从 1 开始编号，可通过 `initial` 调整偏移量。

### 如何在"进行中"步骤显示环形进度？

使用 `percent` 属性（0-100）。当前步骤图标变为 conic-gradient 环形进度圈。

### `maxCount` 折叠后如何自定义省略号样式？

折叠后的省略步骤使用 `title: "..."` 和 `disabled: true` 渲染。可通过传入自定义 StepItem 覆盖，但省略号步骤始终位于折叠区域。

### 水平模式和垂直状态下连接线如何处理？

水平模式下步骤之间通过 `border-t-2` 横线连接，已完成步骤连接线为 `border-primary` 主色。垂直模式下通过 `w-0.5` 竖线连接，已完成步骤为 `bg-primary`。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/steps` 导入。
2. `current` 从 0 开始计数，不是从 1 开始。
3. `items` 中每个 item 至少需提供 `title`。
4. 步骤状态优先级：item 自身 `blocks/status` > 由 `current` 和 `blocks/status` prop 推断。
5. 需要可点击导航时设置 `type="navigation"` 并传入 `onChange`。
6. 不要引入 Ant Design 相关依赖。
