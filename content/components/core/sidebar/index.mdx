Sidebar 侧边栏。提供完整的侧边栏系统，包含展开/折叠、图标模式、移动端抽屉、快捷键切换、菜单高亮动画等能力。基于 Radix Slot、motion/react 和 Tailwind CSS 实现。

<h2 id="when-to-use">何时使用</h2>

当需要为应用提供侧边导航栏时使用。Sidebar 封装了布局行为、响应式适配、状态持久化和动画。

常见场景：
- 管理后台的侧边导航栏
- 文档站点的目录导航
- 需要折叠/展开的辅助信息面板
- 带搜索和用户信息的复合侧边栏

不推荐：
- 将 Sidebar 仅作为视觉装饰而不提供导航功能
- 在 Sidebar 外部使用 `SidebarTrigger` 而不在 `SidebarProvider` 内部（会找不到 Context）
- 用固定宽度的 `<aside>` 手动实现类似功能，应使用 Sidebar 系统提供的状态管理和响应式能力

### 基础用法

`SidebarProvider` 包裹整个布局，内部使用 `Sidebar` + `SidebarTrigger` 组合。

```tsx
import { Home, Inbox, Settings, Users } from "lucide-react"
import {
  Sidebar,
  SidebarContent,
  SidebarGroup,
  SidebarGroupContent,
  SidebarGroupLabel,
  SidebarMenu,
  SidebarMenuButton,
  SidebarMenuItem,
  SidebarProvider,
  SidebarTrigger,
} from "@/components/core/sidebar"

function NavSidebar() {
  return (
    <Sidebar>
      <SidebarContent>
        <SidebarGroup>
          <SidebarGroupLabel>导航</SidebarGroupLabel>
          <SidebarGroupContent>
            <SidebarMenu>
              <SidebarMenuItem>
                <SidebarMenuButton isActive>
                  <Home /><span>首页</span>
                </SidebarMenuButton>
              </SidebarMenuItem>
              <SidebarMenuItem>
                <SidebarMenuButton>
                  <Inbox /><span>收件箱</span>
                </SidebarMenuButton>
              </SidebarMenuItem>
              <SidebarMenuItem>
                <SidebarMenuButton>
                  <Users /><span>用户</span>
                </SidebarMenuButton>
              </SidebarMenuItem>
              <SidebarMenuItem>
                <SidebarMenuButton>
                  <Settings /><span>设置</span>
                </SidebarMenuButton>
              </SidebarMenuItem>
            </SidebarMenu>
          </SidebarGroupContent>
        </SidebarGroup>
      </SidebarContent>
    </Sidebar>
  )
}

export default function Demo() {
  return (
    <SidebarProvider defaultOpen className="h-64 overflow-hidden rounded border">
      <NavSidebar />
      <main className="flex flex-1 items-center justify-center p-4">
        <SidebarTrigger />
      </main>
    </SidebarProvider>
  )
}
```

<h2 id="api">API</h2>

### SidebarProvider

顶层容器，提供 SidebarContext。展开状态通过 Cookie 持久化（`sidebar_state`，有效期 7 天）。内置 `⌘B` / `Ctrl+B` 键盘快捷键切换。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| defaultOpen | 默认展开状态 | `boolean` | `true` |
| open | 受控展开状态 | `boolean` | - |
| onOpenChange | 展开状态变化回调 | `(open: boolean) => void` | - |

### Sidebar

侧边栏主体。移动端自动使用 Sheet 抽屉模式，无需额外处理。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| side | 侧边位置 | `"left"` \| `"right"` | `"left"` |
| variant | 视觉变体：`sidebar` 标准侧栏，`floating` 浮动圆角，`inset` 内嵌模式 | `"sidebar"` \| `"floating"` \| `"inset"` | `"sidebar"` |
| collapsible | 折叠模式：`offcanvas` 完全隐藏，`icon` 收起为图标，`none` 禁用折叠 | `"offcanvas"` \| `"icon"` \| `"none"` | `"offcanvas"` |
| animateOnHover | 菜单项 hover 时是否启用 Highlight 高亮动画 | `boolean` | `true` |
| containerClassName | Highlight 动画容器的样式类名 | `string` | - |
| transition | Highlight 动画的 motion transition 配置 | `Transition` | `{ type: 'spring', stiffness: 350, damping: 35 }` |

### SidebarTrigger

展开/折叠切换按钮，渲染为 `variant="text" shape="square"` 的 Button，图标为 `PanelLeft`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| 继承 Button | 所有 Button props | `React.ComponentProps<typeof Button>` | - |

### SidebarRail

折叠后的窄条触发区，hover 时拖拽光标提示可展开。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| 继承 button | 所有 button 属性 | `React.ComponentProps<'button'>` | - |

### SidebarInset

主内容区域容器，渲染为 `<main>`，支持 `inset` 变体的圆角和阴影。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| 继承 main | 所有 main 属性 | `React.ComponentProps<'main'>` | - |

### SidebarHeader

侧边栏顶部区域，渲染为 `<div>`，默认 `flex flex-col gap-2 p-2`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |

### SidebarContent

侧边栏主体滚动区域，`collapsible=icon` 时隐藏溢出内容。

| 属性 | 说明 | 类型 | 默认值 |

<h2 id="faq">常见问题</h2>

### 如何切换折叠模式？

设置 `collapsible` 属性：
- `"offcanvas"`（默认）：折叠时完全隐藏，展开时覆盖内容。
- `"icon"`：折叠时缩小为图标条，展开时显示完整侧边栏。`SidebarMenuButton` 的 `tooltip` 属性会在折叠时自动显示提示。
- `"none"`：不折叠，始终完整显示。

### 如何自定义侧边栏宽度？

通过 CSS 变量 `--sidebar-width` 和 `--sidebar-width-icon` 控制。默认值分别为 `16rem` 和 `3rem`。在 SidebarProvider 的 style 中覆盖：

```tsx
<SidebarProvider style={{ '--sidebar-width': '20rem', '--sidebar-width-icon': '4rem' } as React.CSSProperties}>
```

### SidebarMenuButton 如何配合路由 Link 使用？

使用 `asChild` 模式将 Link 作为子元素：

```tsx
<SidebarMenuButton asChild isActive={pathname === '/home'}>
  <Link to="/home"><Home /><span>首页</span></Link>
</SidebarMenuButton>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/sidebar` 导入所有需要的组件，不要从 `antd` 导入。
2. 顶层必须使用 `SidebarProvider` 包裹，否则 `useSidebar` 和 `SidebarTrigger` 将无法工作。
3. `SidebarMenuButton` 结合路由时使用 `asChild` + `isActive`，不要手动拼接 active class。
4. 移动端自动使用 Sheet 抽屉模式，无需额外编写移动端适配代码。
5. 只使用本地源码中存在的 props，不生成 AntD 专属的 `Layout.Sider` 配置。
6. 不要引入 Ant Design 相关依赖。
