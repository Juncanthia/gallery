Item 列表项。提供结构化的列表项子组件体系，包括 Item（根容器）、ItemMedia（媒体区域）、ItemContent（内容区）、ItemTitle（标题）、ItemDescription（描述）、ItemActions（操作区）、ItemHeader（头部）、ItemFooter（底部）、ItemGroup（分组容器）、ItemSeparator（分隔线）。

<h2 id="when-to-use">何时使用</h2>

当需要构建具有统一布局结构的列表项时使用，通常配合 List 组件。

常见场景：
- 带图标/头像的列表项（ItemMedia + ItemContent）。
- 带有操作按钮的列表项（ItemActions）。
- 带标题和描述的列表项（ItemTitle + ItemDescription）。
- 多层级分组的列表（ItemGroup）。

不推荐：
- 用原始 div 手动拼接列表项结构，Item 组件提供一致的间距、边框和无障碍支持。
- 在 Item 外部包裹额外的容器来控制布局，应使用 List 或 ItemGroup。

<h2 id="api">API</h2>

### Item

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 列表项样式变体 | `"default"` \| `"outline"` \| `"muted"` | `"default"` |
| size | 列表项尺寸 | `"default"` \| `"sm"` \| `"xs"` | `"default"` |
| asChild | 是否渲染为子元素（Radix Slot） | `boolean` | `false` |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。带 `data-slot="item"`、`data-variant`、`data-size`。当 `asChild=true` 时使用 Radix Slot 渲染。

### ItemMedia

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 媒体区域样式变体 | `"default"` \| `"icon"` \| `"image"` | `"default"` |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。`"icon"` 变体自动设置图标尺寸，`"image"` 变体提供固定容器和图片裁切。

### ItemContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。自动占满剩余空间（`flex-1`）。

### ItemTitle

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。单行截断（`line-clamp-1`），带 `data-slot="item-title"`。

### ItemDescription

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `<p>` 元素属性。最多两行（`line-clamp-2`），带 `data-slot="item-description"`。

### ItemActions

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。操作按钮区域，带 `data-slot="item-actions"`。

### ItemHeader

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。占满一行，内部可左右分布内容。

### ItemFooter

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。占满一行，通常放置元信息。

### ItemGroup

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有标准 HTML `div` 元素属性。`role="list"`，带 `data-slot="item-group"`。

### ItemSeparator

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类名 | `string` | - |

接受所有 `Separator` 组件的属性。默认 `orientation="horizontal"`，带 `data-slot="item-separator"`。

<h2 id="faq">常见问题</h2>

### Item 的三种 variant 有什么区别？

- `default`：透明边框，适合简洁的纯文本列表。
- `outline`：可见边框，适合需要视觉边界的卡片式列表。
- `muted`：灰色背景，适合需要弱化视觉权重的次级信息。

### asChild 何时使用？

当需要将 Item 渲染为 `<a>` 或路由 Link 等非 div 元素时使用。传入 `asChild` 后，样式合并到第一个子元素：

```tsx
<Item asChild>
  <a href="/detail">可点击的列表项</a>
</Item>
```

### ItemMedia 的 variant="image" 如何处理图片尺寸？

`variant="image"` 自动给容器设置固定尺寸（默认 `10`，`sm` 时 `8`，`xs` 时 `6`），内部 `<img>` 使用 `object-cover` 裁切填充。图片需自行设置 `src` 和 `alt`。

### 如何在 Item 中实现右侧对齐的操作区？

将 `ItemActions` 放在 `ItemContent` 之后，它会自动靠右对齐。通过 Item 的 flex 布局，ItemContent 占用剩余空间：

```tsx
<Item>
  <ItemContent>...</ItemContent>
  <ItemActions>...</ItemActions>
</Item>
```

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/list-item` 导入 Item、ItemMedia、ItemContent 等子组件。
2. 列表项使用 `variant` 控制边框样式，使用 `size` 控制间距，不要手写 className 覆盖。
3. 图标放在 ItemMedia（`variant="icon"`）中，头像放在 ItemMedia（`variant="image"`）中。
4. 使用 ItemHeader/ItemFooter 组织多行布局，ItemActions 放置操作按钮。
5. 不要引入 Ant Design 相关依赖。
