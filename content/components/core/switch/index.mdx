Switch 开关。用于在两个互斥状态间即时切换，如开/关、启用/禁用。相比 Checkbox，Switch 更适合无需二次确认即生效的设置项。

<h2 id="when-to-use">何时使用</h2>

当需要表示开关状态并期望切换后立即生效时使用。

常见场景：
- 系统设置项的启用/禁用切换（如通知开关、暗色模式）。
- 列表项的行内快速开关（如用户启用/禁用状态）。
- 需要加载态的异步切换（如开启某个服务需要等待后端响应）。

不推荐：
- 操作需要用户二次确认时——使用 Checkbox 或带确认的交互。
- 多选项场景——使用 Checkbox.Group 或 RadioGroup。

<h2 id="examples">代码演示</h2>

### 基础用法

支持 `small`/`middle` 尺寸、`loading` 加载态、`checkedChildren`/`unCheckedChildren` 自定义内容、`thumbIcon` 自定义滑块图标和 `pressedWidth` 自定义按压宽度。

<ComponentShowcase id="switch" path="switch/basic" />

```tsx
import { useState } from "react"
import { Switch } from "@/components/core/switch"

export default function Demo() {
  const [on, setOn] = useState(true)
  return (
    <div className="flex flex-wrap items-center gap-4">
      <Switch checked={on} onChange={setOn} />
      <Switch defaultChecked />
      <Switch disabled />
      <Switch defaultChecked disabled />
      <Switch size="small" />
      <Switch loading />
    </div>
  )
}
```

### 带内联内容

通过 `checkedChildren` 和 `unCheckedChildren` 在开关两侧显示文字或图标。

```tsx
import { Switch } from "@/components/core/switch"

export default function Demo() {
  return (
    <div className="flex flex-wrap items-center gap-4">
      <Switch checkedChildren="开" unCheckedChildren="关" defaultChecked />
      <Switch checkedChildren="ON" unCheckedChildren="OFF" size="small" />
    </div>
  )
}
```

<h2 id="api">API</h2>

### Switch

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| checked | 开关状态（受控） | `boolean` | - |
| defaultChecked | 默认开关状态（非受控） | `boolean` | `false` |
| onCheckedChange | Radix 原生状态变化回调 | `(checked: boolean) => void` | - |
| onChange | 状态变化回调（与 onCheckedChange 等效，便利别名） | `(checked: boolean) => void` | - |
| size | 尺寸 | `"small"` \| `"middle"` | `"middle"` |
| loading | 加载中状态，开关不可操作，滑块显示旋转图标 | `boolean` | `false` |
| disabled | 是否禁用 | `boolean` | `false` |
| checkedChildren | 开启时显示的内联内容（开关右侧） | `React.ReactNode` | - |
| unCheckedChildren | 关闭时显示的内联内容（开关左侧） | `React.ReactNode` | - |
| thumbIcon | 滑块上的自定义图标（loading 时自动覆盖为旋转动画） | `React.ReactElement` | - |
| pressedWidth | 滑块按压时的动画宽度（px） | `number` | `small: 15; middle: 19` |
| className | 附加 CSS 类名 | `string` | - |

### SwitchSize

| 值 | 尺寸 | 开关宽x高 |
| --- | --- | --- |
| `"small"` | 小号 | `28px x 16px` |
| `"middle"` | 中号 | `32px x 20px` |

<h2 id="faq">常见问题</h2>

### loading 与 disabled 有什么区别？

`loading` 表示正在执行异步操作（如调用 API 切换状态），滑块会显示旋转图标。`disabled` 表示开关不可用（如无权限操作）。两者均会使开关不可点击。当 `loading` 为 `true` 时，`disabled` 的效果会被覆盖——组件内部使用 `mergedDisabled = disabled || loading`。

### 如何在表单中使用 Switch？

Switch 本身不提供表单绑定能力。配合 React Hook Form 或 Formik 使用时，将 `checked`/`onChange` 绑定到表单库的 value/onChange：

```tsx
const { field } = useController({ name: "enabled" })
<Switch checked={field.value} onChange={field.onChange} />
```

### checkedChildren/unCheckedChildren 太长导致布局异常？

当存在内联内容时，Switch 宽度会自动扩展（`small: min-w-9`、`middle: min-w-11`）。如果内容仍然过长，可以通过调整字号或缩短文案解决，不建议通过 `className` 覆盖宽度。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/switch` 导入。
2. `onChange` 回调参数是 `boolean`（不是事件对象），与 Ant Design 的 `onChange(checked)` 一致。
3. 异步操作使用 `loading` 属性，而非手动添加旋转图标。
4. 禁用状态使用 `disabled`，不要通过 `className="pointer-events-none opacity-50"` 模拟。
5. 不要引入 Ant Design 相关依赖。
