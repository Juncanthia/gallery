Table 表格。展示结构化行列数据，支持声明式 columns/dataSource 配置、排序、分页、行选择、行展开、加载态和空状态。基于原生 `<table>` 元素封装，提供语义化子组件和统一样式。

<h2 id="when-to-use">何时使用</h2>

当需要展示大量结构化数据，并支持排序、分页、选择等交互时使用。

常见场景：
- 后台管理系统的数据列表（用户列表、订单列表）。
- 报表和数据分析页面的数据展示。
- 需要行选择、行展开等交互的数据表格。

不推荐：
- 简单的键值对展示——使用 Descriptions 组件。
- 字段极少的列表——使用 List 组件。
- 高度定制的复杂表格布局——直接使用原生 `<table>` 加 Tailwind。

<h2 id="examples">代码演示</h2>

### 子组件模式

`Table` 搭配 `TableHeader`、`TableBody`、`TableRow`、`TableHead`、`TableCell` 构建表格。

<ComponentShowcase id="table" path="table/basic" />

```tsx
import {
  Table, TableBody, TableCaption, TableCell,
  TableHead, TableHeader, TableRow,
} from "@/components/core/table"

const data = [
  { id: "1", name: "张三", department: "技术部", blocks/status: "在职" },
  { id: "2", name: "李四", department: "产品部", blocks/status: "在职" },
  { id: "3", name: "王五", department: "设计部", blocks/status: "休假" },
]

export default function Demo() {
  return (
    <Table className="w-full max-w-md">
      <TableCaption>员工列表</TableCaption>
      <TableHeader>
        <TableRow>
          <TableHead>姓名</TableHead>
          <TableHead>部门</TableHead>
          <TableHead>状态</TableHead>
        </TableRow>
      </TableHeader>
      <TableBody>
        {data.map((row) => (
          <TableRow key={row.id}>
            <TableCell>{row.name}</TableCell>
            <TableCell>{row.department}</TableCell>
            <TableCell>{row.blocks/status}</TableCell>
          </TableRow>
        ))}
      </TableBody>
    </Table>
  )
}
```

### columns/dataSource 声明式

使用 `columns` + `dataSource` 配置式 API，类似 Ant Design。

```tsx
import { Table } from "@/components/core/table"
```

### 排序、分页与行选择

`columns` 中设置 `sorter` 启用排序，`pagination` 启用分页，`rowSelection` 启用多选。

```tsx
import { Table, type TableColumn } from "@/components/core/table"

interface Record {
  id: string; name: string; age: number; address: string;
}

const columns: TableColumn<Record>[] = [
  { title: "姓名", dataIndex: "name", sorter: true },
  { title: "年龄", dataIndex: "age", sorter: (a, b) => a.age - b.age },
  { title: "地址", dataIndex: "address" },
]

const dataSource: Record[] = [
  { id: "1", name: "张三", age: 32, address: "北京市朝阳区" },
  { id: "2", name: "李四", age: 28, address: "上海市浦东新区" },
]

export default function Demo() {
  return (
    <Table
      columns={columns}
      dataSource={dataSource}
      rowKey="id"
      pagination={{ pageSize: 5 }}
      rowSelection={{
        onChange: (keys, rows) => console.log(keys, rows),
      }}
    />
  )
}
```

### 行展开

通过 `expandable` 属性启用行展开，点击展开按钮显示额外内容。

```tsx
import { Table } from "@/components/core/table"

const columns = [
  { title: "姓名", dataIndex: "name" },
  { title: "部门", dataIndex: "department" },
]

const dataSource = [
  { id: "1", name: "张三", department: "技术部", detail: "负责前端架构设计" },
  { id: "2", name: "李四", department: "产品部", detail: "负责需求分析与规划" },
]

export default function Demo() {
  return (
    <Table
      columns={columns}
      dataSource={dataSource}
      rowKey="id"
      expandable={{
        expandedRowRender: (record) => (
          <p className="text-muted-foreground">{record.detail}</p>
        ),
      }}
    />
  )
}
```

| loading | 加载中状态，显示骨架屏 | `boolean` | `false` |

<h2 id="faq">常见问题</h2>

### 子组件模式 vs columns/dataSource 模式如何选择？

- **子组件模式**：适合简单表格，列和样式完全由开发者控制，更灵活。
- **columns/dataSource 模式**：适合需要排序、分页、行选择、行展开等交互能力的表格，API 风格类似 Ant Design。

两种模式互斥，传入 `columns`/`dataSource` 后 `children` 会被忽略。

### 排序如何工作？

在 `columns` 中为需要排序的列设置 `sorter` 属性。`sorter: true` 使用默认字符串比较，`sorter: (a, b) => number` 可自定义比较逻辑。点击列标题可在升序、降序、无排序之间切换。排序状态通过 `onChange` 回调的第三个参数 `sorter` 获取。

### 如何使用受控分页和排序？

```tsx
const [current, setCurrent] = useState(1)
const [pageSize, setPageSize] = useState(10)

<Table
  columns={columns}
  dataSource={data}
  pagination={{ current, pageSize, onChange: (p, ps) => { setCurrent(p); setPageSize(ps) } }}
  onChange={(pagination, filters, sorter) => {
    // sorter.order: "ascend" | "descend" | null
    // sorter.columnKey: 排序列的 key
  }}
/>
```

### 如何展示加载、空数据、错误等状态？

- **加载中**：设置 `loading={true}`，组件自动显示骨架屏。
- **空数据**：通过 `emptyText` 或 `locale.emptyText` 自定义空状态文案。
- **错误状态**：Table 不内置错误状态，应在 Table 外层处理，如显示 Result 或 Alert 组件。

### 如何固定表头？

Table 不内置粘性表头。给 `table-container` 设置 `max-height` + `overflow-auto`，`TableHead` 设置 `sticky top-0 bg-background` 即可实现。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/table` 导入所需子组件和类型。
2. 声明式模式使用 `columns` + `dataSource` + `rowKey`，不要生成 Ant Design 的 `column` 单数拼写。
3. 子组件模式使用 `map` 遍历数据生成 `TableRow`，始终设置 `key`。
4. 状态、标签等字段使用 Badge 或 Tag 组件展示，不要裸写原始值文本。
5. 排序使用 `columns[].sorter`，分页使用 `pagination`，不要自己在数据层手动处理。
6. `dataIndex` 支持嵌套路径如 `"user.name"` 或数组 `["user", "name"]`。
7. 不要引入 Ant Design 相关依赖。
