Alert 警告提示。用于在页面中向用户展示警告、错误、成功或信息提示，支持多种语义变体和可关闭模式。

<h2 id="when-to-use">何时使用</h2>

当需要在页面中向用户展示警告、错误、成功或信息提示时使用。与 Notification 不同，Alert 是页面内嵌的静态提示，不会自动消失。

常见场景：

- 表单提交后的成功/失败反馈。
- 重要操作前的警告提示。
- 页面顶部的全局通知横幅（设置 `blocks/banner` 属性）。
- 需要用户手动关闭的提示信息。
- 系统状态变更通知。

不推荐：

- 需要自动消失的轻量反馈，应使用 Toast / Notification。
- 需要用户确认的阻断式提示，应使用 AlertDialog。
- 将 Alert 用于纯装饰性信息展示（无实际语义），会降低用户对警告的敏感度。

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `type` 切换语义样式，通过 `variant` 切换外观风格。

  id="alert"
  path="alert/basic"
/>

```tsx
import { Alert, AlertDescription, AlertTitle } from "@/components/core/alert"
import { Info } from "lucide-react"

export default function Demo() {
  return (
    <div className="w-full max-w-md space-y-3">
      <Alert type="info">
        <Info className="size-4" />
        <AlertTitle>提示</AlertTitle>
        <AlertDescription>这是一条普通提示信息。</AlertDescription>
      </Alert>
      <Alert type="error">
        <Info className="size-4" />
        <AlertTitle>错误</AlertTitle>
        <AlertDescription>操作失败，请重试。</AlertDescription>
      </Alert>
      <Alert type="success">
        <Info className="size-4" />
        <AlertTitle>成功</AlertTitle>
        <AlertDescription>操作已成功完成。</AlertDescription>
      </Alert>
      <Alert type="warning">
        <Info className="size-4" />
        <AlertTitle>警告</AlertTitle>
        <AlertDescription>请注意操作风险。</AlertDescription>
      </Alert>
    </div>
  )
}
```

### 可关闭

设置 `closable` 显示关闭按钮，用户可手动关闭提示。

```tsx
import { Alert, AlertDescription, AlertTitle } from "@/components/core/alert"

export default function Demo() {
  return (
    <div className="w-full max-w-md space-y-3">
      <Alert type="info" closable>
        <AlertTitle>可关闭提示</AlertTitle>
        <AlertDescription>点击右侧 X 按钮关闭此提示。</AlertDescription>
      </Alert>
      <Alert
        type="success"
        closable={{ closeIcon: <span className="text-xs">关闭</span> }}
      >
        <AlertTitle>自定义关闭按钮</AlertTitle>
        <AlertDescription>通过 closable 对象自定义关闭图标。</AlertDescription>
      </Alert>
    </div>
  )
}
```

### 横幅模式

设置 `blocks/banner` 属性，Alert 会去掉左右边框和圆角，适用于页面顶部的全局通知。

```tsx
import { Alert, AlertDescription, AlertTitle } from "@/components/core/alert"

export default function Demo() {
  return (
    <Alert type="warning" blocks/banner closable showIcon>
      <AlertTitle>系统维护通知</AlertTitle>
      <AlertDescription>
        系统将于今晚 22:00-24:00 进行维护升级，期间部分功能可能不可用。
      </AlertDescription>
    </Alert>
  )
}
```

### filled 风格

`variant="filled"` 使用背景色填充，适用于需要更强调视觉的场景。

```tsx
import { Alert, AlertDescription, AlertTitle } from "@/components/core/alert"

export default function Demo() {
  return (
    <div className="w-full max-w-md space-y-3">
      <Alert type="success" variant="filled">
        <AlertTitle>操作成功</AlertTitle>
        <AlertDescription>数据已保存。</AlertDescription>
      </Alert>
      <Alert type="error" variant="filled">
        <AlertTitle>操作失败</AlertTitle>
        <AlertDescription>请检查网络后重试。</AlertDescription>
      </Alert>
    </div>
  )
}
```

### 带操作区

通过 `action` 属性添加操作按钮或链接。

```tsx
import { Alert, AlertDescription, AlertTitle } from "@/components/core/alert"
import { Button } from "@/components/core/button"

export default function Demo() {
  return (
    <Alert
      type="warning"
      showIcon
      action={
        <Button size="small" variant="outlined" color="secondary">
          查看详情
        </Button>
      }
    >
      <AlertTitle>存储空间不足</AlertTitle>
      <AlertDescription>当前已使用 85% 的存储空间，建议清理。</AlertDescription>
    </Alert>
  )
}
```

### children 模式（无 API props）

当不传 `type`/`title`/`description` 等高层 props 时，Alert 退化为纯容器，仅应用基础样式，children 完全自由。

```tsx
import { Alert } from "@/components/core/alert"

export default function Demo() {
  return (
    <Alert variant="destructive" className="w-full max-w-md">
      <div className="flex items-center gap-2">
        <span className="i-lucide-alert-triangle size-4" />
        <span className="text-sm font-medium">完全自定义内容布局</span>
      </div>
    </Alert>
  )
}
```

| --- | --- | --- | --- |
| type | 语义类型，控制颜色主题 | `success` \| `info` \| `warning` \| `error` | 无 type 且无 variant="destructive" 时默认 `info` |
| variant | 外观风格 | `outlined` \| `filled` \| `default` \| `destructive` | `outlined` |
| title | 标题内容（message 别名） | `React.ReactNode` | - |
| message | 标题内容（title 别名） | `React.ReactNode` | - |
| description | 描述内容 | `React.ReactNode` | - |
| icon | 自定义图标 | `React.ReactNode` | 根据 type 自动选择 |

| className | 附加样式 | `string` | - |

<h2 id="faq">常见问题</h2>

### variant 的 `outlined` / `filled` 与 `destructive` 有什么区别？

- `outlined`：边框+浅色文字，适合信息型提示。
- `filled`：背景色+边框，视觉更强，适合需要强调的提示。
- `destructive`：使用 shadcn 原始的 destructive 样式变体，仅红/灰色系。
- `default`：shadcn 原始默认样式。

`type` 控制的是语义色彩（success/info/warning/error），`variant` 控制的是外观表现手法。两者组合使用。

### closable 三种形态如何选择？

- `closable`（boolean）：显示默认 X 图标关闭按钮。
- `closable={{ closeIcon: <Custom /> }}`：自定义关闭图标。
- `closable={{ onClose: (e) => { e.preventDefault(); ... } }}`：阻止默认关闭行为，手动控制。

### 如何控制 Alert 的显示/隐藏？

Alert 提供内部关闭状态管理（通过 `closable`），点击关闭后组件返回 `null`。如需外部控制，应在父组件中条件渲染：

```tsx
{showAlert && <Alert type="error" closable onClose={() => setShowAlert(false)}>...</Alert>}
```

### blocks/banner 模式和普通模式的布局差异？

blocks/banner 模式下 Alert 去掉左右 border 和 border-radius（`rounded-none border-x-0`），适合横跨页面全宽的顶部通知栏。普通模式保留圆角边框，适合卡片式嵌入布局。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/alert` 导入 Alert、AlertTitle、AlertDescription、AlertAction。
2. 使用 `type` 表达语义（success/info/warning/error），不要手动拼颜色 class。
3. `type` + `variant` 组合使用：用 `type` 选颜色，用 `variant` 选风格。
4. 页面顶部全局通知使用 `blocks/banner` 属性。
5. 不要引入 Ant Design 相关依赖。
