# Timeline 时间轴

垂直或水平展示的时间流信息。

本组件借鉴 Ant Design Timeline 的使用心智，但不是 AntD 兼容层。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

适合在需要 时间轴 相关能力的界面中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 需要复用统一设计语言、状态样式和可访问性约定。
- 需要和本项目 Tailwind 主题变量、暗色模式、Radix/Base UI primitive 或本地交互行为保持一致。
- 需要让 AI 或业务代码稳定地从 `@/components/core/timeline` 引用组件，而不是直接引用第三方底层库。

不推荐：

- 为了照搬 Ant Design API 而新增兼容别名。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { Timeline } from "@/components/core/timeline"

export default function Demo() {
  return (
    <Timeline
      activeIndex={1}
      items={[
        { time: "09:00", title: "已提交", content: "表单已进入审批流程" },
        { time: "10:30", title: "审批中", content: "等待主管确认" },
        { time: "11:00", title: "待归档", content: "完成后自动归档" },
      ]}
    />
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。推荐使用 `items` 配置式 API；不要从 dice 的 compound timeline 导入子组件。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 如果源码使用 CVA，则变体样式集中在组件文件或相邻 `*-variants.ts` 中。
- 如果源码包装 Radix/Base UI primitive，则交互状态、键盘行为和 ARIA 基础能力来自底层 primitive，本地组件负责视觉和 API 收敛。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground`、`ring-ring` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/core/timeline.tsx`

直接依赖摘要：

- `motion/react`

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 推荐通过 `items` 声明时间线条目；`children` 只用于少量自定义补充内容。
3. `activeIndex` 会派生 `completed/active/pending` 状态，并同步到 `data-blocks/status` 和连接线。
4. `time` 会渲染为 `<time>`，适合时间语义；普通标签可继续使用 `label`。
5. 如果需要新增能力，应先修改该组件源码，再更新本 llm 文档。
6. 不要把第三方文档中的全量 API 视为本地已支持能力。

## 可访问性 {#accessibility}

- 保留原生元素或底层 primitive 的键盘语义。
- 交互型组件应提供可访问名称，例如 `aria-label`、`title` 或可见文本。
- 禁用、加载、展开、选中等状态应优先使用组件已有 props，而不是只改视觉 class。
- 纯图标触发器必须提供明确的可访问名称。

| 方向 | 本地决策 |

| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |

| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 借鉴 AntD 使用心智，但不承诺 API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Timeline 代码时，应遵守：

1. 从 `@/components/core/timeline` 导入，不要从 `antd` 导入。
2. 优先使用 `items` 属性声明式配置时间线条目，每个条目至少提供 `content`、`children`、`title` 或 `time`。
3. 使用 `activeIndex` 表示流程进度；需要覆盖单项状态时使用 item 的 `blocks/status`。
4. 不要生成 `TimelineDot`、`TimelineConnector`、`TimelineHeader` 等 dice compound 子组件。
5. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
6. 需要自定义样式时，优先使用组件暴露的 props；最后才使用 `className`。
7. 如果不确定某个 AntD API 是否存在，先检查 `src/components/core/timeline.tsx`，不要凭 AntD 记忆生成。
