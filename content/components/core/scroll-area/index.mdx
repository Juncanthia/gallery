ScrollArea 滚动区域。为内容区域提供统一样式的滚动容器，基于 Radix ScrollArea 实现。内容溢出时自动显示滚动条，支持水平和垂直滚动。

<h2 id="when-to-use">何时使用</h2>

当需要在固定高度/宽度的容器内展示可滚动内容时使用。相比原生滚动条，ScrollArea 提供一致的外观和更好的可定制性。

常见场景：
- 侧边栏导航菜单的滚动区域。
- 对话框/抽屉中的长内容。
- 数据列表或表格的固定高度滚动容器。
- 代码块或日志面板。
- 需要复用统一设计语言和可访问性约定的滚动容器。

不推荐：
- 不设置容器高度（`h-*` 或 className）导致滚动条永不出现。
- 在页面主体使用 ScrollArea 替代 `overflow: auto`，通常页面主体使用原生滚动更自然。
- 用 `overflow: auto` + 自定义 CSS 模拟 ScrollArea 的样式。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

### 基础用法

```tsx
import { ScrollArea } from "@/components/core/scroll-area"

export default function Demo() {
  return (
    <ScrollArea className="h-48 w-full max-w-sm rounded border">
      <div className="space-y-2 p-4">
        {Array.from({ length: 30 }, (_, i) => (
          <p key={i} className="text-sm text-muted-foreground">
            第 {i + 1} 行：ScrollArea 在内容溢出时自动显示滚动条。
          </p>
        ))}
      </div>
    </ScrollArea>
  )

```

<h2 id="api">API</h2>

### ScrollArea

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| type | 滚动条显示模式 | `"auto"` \| `"always"` \| `"scroll"` \| `"hover"` | `"hover"` |
| dir | 文本方向 | `"ltr"` \| `"rtl"` | `"ltr"` |
| scrollHideDelay | 滚动条隐藏延迟（毫秒） | `number` | `600` |
| className | 附加样式类名，用于设置容器尺寸 | `string` | - |
| children | 滚动内容 | `React.ReactNode` | - |

包装 Radix `ScrollArea.Root`，接受其所有 props。带 `data-slot="scroll-area"`。

### ScrollBar

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| orientation | 滚动条方向 | `"horizontal"` \| `"vertical"` | `"vertical"` |
| forceMount | 强制挂载至 DOM（用于动画） | `true` \| `undefined` | - |
| className | 附加样式类名 | `string` | - |

包装 Radix `ScrollArea.ScrollAreaScrollbar`，接受其所有 props。带 `data-slot="scroll-area-scrollbar"` 和 `data-orientation`。

ScrollArea 内部已包含默认的 ScrollBar，一般无需手动使用 ScrollBar 组件。

<h2 id="faq">常见问题</h2>

### 如何控制滚动条显隐？

ScrollArea 默认 `type="hover"`，悬停时显示滚动条。可设置为 `"always"` 始终显示、`"auto"` 自动判断、`"scroll"` 滚动时显示。

### 为什么我的 ScrollArea 不出现滚动条？

最常见的原因是容器没有设置高度。必须通过 `className="h-48"` 或父容器约束给 ScrollArea 一个有限的高度。

### ScrollArea 支持水平滚动吗？

支持。当内容宽度超出容器时，底部会自动出现水平滚动条。可通过 ScrollBar 的 `orientation` 属性单独控制。

### 如何配合虚拟列表使用？

将虚拟列表渲染为 ScrollArea 的 children 即可，ScrollArea 提供外层滚动的容器和样式。确保虚拟列表的滚动容器与 ScrollArea 的 Viewport 保持一致。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/scroll-area` 导入，不要从 `antd` 导入。
2. 必须为 ScrollArea 设置高度（`h-*` 或 className 约束），否则不会出现滚动条。
3. ScrollArea 内部已包含 ScrollBar，通常无需手动使用 ScrollBar 组件。
4. 基于 Radix ScrollArea，不依赖 Ant Design 的实现。
5. 不要引入 Ant Design 相关依赖。

<h2 id="semantic-dom">Semantic DOM</h2>

组件通过 `data-slot` 属性暴露稳定的语义结构，可用于测试定位和样式覆写：

| data-slot | 说明 |
| --- | --- |
| `scroll-area` | 滚动区域根容器 |
| `scroll-area-viewport` | 滚动视口，可聚焦且有 focus-visible 环 |
| `scroll-area-scrollbar` | 滚动条，带 `data-orientation` 状态 |
| `scroll-area-thumb` | 滚动条滑块 |
