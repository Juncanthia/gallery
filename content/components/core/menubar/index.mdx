菜单栏 Menubar。提供桌面应用风格的顶部菜单栏，支持多级菜单、子菜单、分隔线、快捷键和选中状态，基于 Radix Menubar 实现。支持子组件手动组合和 `items` 数组声明式两种使用方式。

<h2 id="when-to-use">何时使用</h2>

当需要为应用提供桌面风格的菜单栏时使用。

常见场景：
- 编辑器、IDE 类应用的顶部菜单（文件、编辑、视图等）
- 管理后台的功能导航栏
- 需要多级下拉菜单的复杂操作入口
- 替代传统水平导航栏的桌面应用场景

不推荐：
- 点击按钮弹出的单次操作菜单——使用 DropdownMenu
- 侧边栏式导航——使用 NavigationMenu
- 移动端底部导航

### items 声明式配置

```tsx
import { Menubar } from "@/components/core/menubar"

export default function Demo() {
  return (
    <Menubar
      items={[
        {
          type: "group",
          label: "文件",
          children: [
            { label: "新建", shortcut: "Ctrl+N" },
            { label: "打开", shortcut: "Ctrl+O" },
            { type: "divider" },
            { label: "保存", shortcut: "Ctrl+S" },
            { label: "退出", danger: true },
          ],
        },
        {
          type: "group",
          label: "编辑",
          children: [
            { label: "撤销", shortcut: "Ctrl+Z" },
            { label: "重做", shortcut: "Ctrl+Shift+Z" },
            { type: "divider" },
            { label: "剪切", shortcut: "Ctrl+X" },
            { label: "复制", shortcut: "Ctrl+C" },
            { label: "粘贴", shortcut: "Ctrl+V" },
          ],
        },
      ]}
    />
  )
}
```

<h2 id="api">API</h2>

### Menubar

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| items | 声明式菜单配置数组 | `MenubarItemOption[]` | - |
| selectedKeys | 受控选中项 keys | `React.Key[]` | - |
| defaultSelectedKeys | 默认选中项 keys | `React.Key[]` | `[]` |
| onSelect | 选中回调，包含 key、keyPath、selectedKeys、item 和 domEvent | `(info: MenubarSelectInfo) => void` | - |
| contentProps | 传给 Content 的额外属性 | `ComponentProps<typeof MenubarPrimitive.Content>` | - |
| className | 附加样式类名 | `string` | - |

### MenubarItemOption

声明式配置的菜单项类型。

| 类型 | 关键属性 | 说明 |
| --- | --- | --- |
| `"item"`（默认） | `label`, `icon?`, `extra?`, `shortcut?`, `disabled?`, `danger?`, `onSelect?` | 普通菜单项 |
| `"divider"` | - | 分隔线 |
| `"group"` | `label`, `children` | 顶级菜单组（渲染为 Trigger + Content） |
| `"submenu"` | `label`, `icon?`, `disabled?`, `danger?`, `children` | 二级子菜单 |

### MenubarTrigger

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| selected | 是否选中 | `boolean` | - |
| danger | 是否危险样式 | `boolean` | - |
| className | 附加样式类名 | `string` | - |

### MenubarItem

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| variant | 样式变体 | `"default"` \| `"destructive"` | `"default"` |
| inset | 是否内缩进 | `boolean` | - |
| selected | 是否选中 | `boolean` | - |
| className | 附加样式类名 | `string` | - |

<h2 id="faq">常见问题</h2>

### Menubar 和 DropdownMenu 有什么区别？

- **Menubar**：顶层始终可见的菜单栏，点击展开，适合桌面应用风格（如 VS Code、Figma）。
- **DropdownMenu**：点击按钮后弹出，适合单次操作菜单。

### 如何在 items 模式中实现嵌套子菜单？

使用 `"submenu"` 类型，其 `children` 可继续嵌套更多 item 或 submenu：

```tsx
items={[
  {
    type: "submenu",
    label: "视图",
    children: [
      { label: "放大" },
      { label: "缩小" },
    ],
  },
]}
```

### 如何实现危险操作（红色）菜单项？

子组件模式使用 `variant="destructive"`，items 模式使用 `danger: true`。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/menubar` 导入。
2. 顶级菜单有两种方式：`"group"` 类型（有 Trigger+Content）和 `"submenu"` 类型。
3. `"item"` 类型在顶级时渲染为纯 Trigger（无下拉面板）。
4. 危险操作使用 `variant="destructive"` 或 `danger: true`。
5. 简单场景优先使用 `items` 声明式写法。
6. 不要引入 Ant Design 的 `Menu`（horizontal 模式）。
