FileTree 文件树。基于 Radix 的目录和文件层级结构展示组件，支持展开折叠、选中、Git 状态标记和连线样式。

<h2 id="when-to-use">何时使用</h2>

当需要展示目录和文件的层级结构，并提供展开/折叠和选中交互时使用。

常见场景：

- 代码编辑器的文件浏览器侧边栏。
- 文档目录结构导航。
- 项目管理工具中的文件列表。
- 带 Git 状态标记的文件变更视图（新建/修改/删除）。

不推荐：

- 作为通用树形控件使用——FileTree 专为文件/目录语义设计，通用树应使用 Tree 组件。
- 数据量极大的深层嵌套场景——Radix 基于 DOM 渲染，深层递归可能影响性能。

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `treeData` 属性声明式传入树数据。

<ComponentShowcase id="file-tree" path="file-tree/basic" />

```tsx
import { Files } from "@/components/core/file-tree"

export default function Demo() {
  return (
    <Files
      treeData={[
        {
          key: "src",
          title: "src",
          children: [
            { key: "app", title: "app.tsx" },
            { key: "index", title: "index.ts" },
          ],
        },
        { key: "package", title: "package.json" },
      ]}
      className="w-48"
    />
  )
}
```

### 使用子组件手动构建

可以通过 `Files`、`FolderItem`、`FolderTrigger`、`FolderContent`、`FileItem` 系列子组件手动构建文件树。

```tsx
import { Files, FolderItem, FolderTrigger, FolderContent, FileItem } from "@/components/core/file-tree"

export default function Demo() {
  return (
    <Files className="w-56">
      <FolderItem value="src">
        <FolderTrigger>src</FolderTrigger>
        <FolderContent>
          <FileItem>app.tsx</FileItem>
          <FileItem>index.ts</FileItem>
        </FolderContent>
      </FolderItem>
      <FileItem>package.json</FileItem>
    </Files>
  )
}
```

### 带 Git 状态标记

通过 `gitStatus` 属性为文件/文件夹添加 Git 状态颜色标记。

```tsx
import { Files } from "@/components/core/file-tree"

export default function Demo() {
  return (
    <Files
      treeData={[
        {
          key: "src",
          title: "src",
          children: [
            { key: "app", title: "app.tsx", gitStatus: "modified" },
            { key: "index", title: "index.ts", gitStatus: "untracked" },
            { key: "utils", title: "utils.ts", gitStatus: "deleted" },
          ],
        },
        { key: "package", title: "package.json" },
      ]}
      className="w-56"
    />
  )
}
```

<h2 id="api">API</h2>

### Files

顶层文件树容器，包装 Radix Files primitive。支持声明式 `treeData` 和手动子组件两种构建方式。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| treeData | 声明式文件树数据 | `FileTreeDataNode[]` | - |
| children | 手动构建的子组件内容 | `ReactNode` | - |
| defaultExpandedKeys | 默认展开的 key 列表 | `React.Key[]` | - |
| expandedKeys | 受控展开的 key 列表 | `React.Key[]` | - |
| onExpandedKeysChange | 展开变更回调 | `(keys: React.Key[]) => void` | - |
| defaultSelectedKeys | 默认选中的 key 列表 | `React.Key[]` | - |
| selectedKeys | 受控选中的 key 列表 | `React.Key[]` | - |
| onSelect | 选中回调 | `(selectedKeys: React.Key[], info: FileTreeSelectInfo) => void` | - |
| selectable | 是否可选择 | `boolean` | `true` |
| showIcon | 是否显示图标 | `boolean` | `true` |
| showLine | 是否显示缩进连线 | `boolean` | `true` |
| className | 附加 className | `string` | - |

### FileTreeDataNode

声明式树节点数据结构。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| key | 唯一标识 | `string` | 必填 |
| title | 节点标题 | `ReactNode` | 必填 |
| children | 子节点列表 | `FileTreeDataNode[]` | - |
| icon | 自定义图标 | `React.ElementType \| ReactNode` | - |
| gitStatus | Git 状态标记 | `"untracked"` \| `"modified"` \| `"deleted"` | - |
| disabled | 禁用 | `boolean` | - |
| selectable | 是否可选中 | `boolean` | - |
| isLeaf | 是否叶子节点（即使有 children 也渲染为文件） | `boolean` | - |
| className | 节点附加 className | `string` | - |

### FileTreeSelectInfo

选中事件信息。

| 属性 | 说明 | 类型 |
| --- | --- | --- |
| selected | 当前节点是否被选中 | `boolean` |
| selectedKeys | 选中后的 key 列表 | `React.Key[]` |
| node | 当前操作的节点数据 | `FileTreeDataNode` |

### FolderItem

文件夹项容器，包裹 FolderTrigger 和 FolderContent。

### FolderTrigger

文件夹触发器，支持点击展开/折叠和选中。扩展了 `FileLabelPrimitiveProps`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| gitStatus | Git 状态标记 | `"untracked"` \| `"modified"` \| `"deleted"` | - |
| selected | 是否选中 | `boolean` | - |
| disabled | 是否禁用 | `boolean` | - |
| selectable | 是否可选中 | `boolean` | - |
| onSelect | 选中回调 | `() => void` | - |

### FolderContent

文件夹内容容器，包裹子文件/文件夹。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| showLine | 是否显示缩进连线 | `boolean` | `true` |
| className | 附加 className | `string` | - |

### FileItem

文件项组件。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| icon | 自定义图标，`false` 隐藏图标 | `React.ElementType \| ReactNode \| false` | `FileIcon` |
| gitStatus | Git 状态标记 | `"untracked"` \| `"modified"` \| `"deleted"` | - |
| selected | 是否选中 | `boolean` | - |
| disabled | 是否禁用 | `boolean` | - |
| selectable | 是否可选中 | `boolean` | - |
| onSelect | 选中回调 | `() => void` | - |

### SubFiles

子级文件树容器，类型等同于 `FilesProps`，用于手动构建嵌套结构。

<h2 id="faq">常见问题</h2>

### 如何实现文件树的受控展开/折叠？

通过 `expandedKeys` + `onExpandedKeysChange` 实现受控模式：

```tsx
const [expanded, setExpanded] = useState<React.Key[]>([])
<Files
  treeData={data}
  expandedKeys={expanded}
  onExpandedKeysChange={setExpanded}
/>
```

### treeData 模式和手动子组件模式有什么区别？

`treeData` 模式通过声明式数据自动渲染树结构，适合大多数场景。手动子组件模式需要手动组合 `FolderItem`/`FileItem` 等，适合需要完全自定义渲染的场景。

### Git 状态的颜色含义是什么？

`untracked`（新建）为绿色，`modified`（修改）为橙色，`deleted`（删除）为红色。文件夹显示圆点标记，文件显示字母标记（U/M/D）。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/core/file-tree` 导入 `Files` 及所需的子组件。
2. 优先使用 `treeData` 声明式模式构建文件树。
3. 需要受控展开时使用 `expandedKeys` + `onExpandedKeysChange`。
4. 需要选择功能时使用 `selectedKeys` + `onSelect`，单选模式下选中新节点会自动取消旧节点。
5. 隐藏缩进连线设置 `showLine={false}`。
6. 不要引入 Ant Design 相关依赖。
