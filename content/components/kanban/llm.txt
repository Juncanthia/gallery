# Kanban 看板

基于 @dnd-kit 的看板复合组件，通过 `KanbanProvider` 提供拖拽上下文，配合 `KanbanBoard`（列容器）、`KanbanHeader`（列标题）、`KanbanCards`（卡片列表）和 `KanbanCard`（可拖拽卡片）组合构建完整的看板界面。

本组件为项目本地复合组件（Blocks），无 Ant Design 直接对应，API 和行为以 `src/components/blocks/kanban/index.tsx` 源码为准。

## 何时使用 {#when-to-use}

当需要在界面中实现看板式拖拽管理时使用。

常见场景：

- 任务管理面板（如 Todo / In Progress / Done）。
- CRM 销售管道、候选人招聘流程等阶段拖拽。
- 需要卡片在列间自由移动并实时更新数据源的任何看板场景。
- 需要屏幕阅读器无障碍播报（announcements）的拖拽交互。

不推荐：

- 简单的列表拖拽排序（不需要列维度时）——使用 @dnd-kit/sortable 即可。
- 需要同时选中多个卡片批量拖拽的场景——组件仅支持单卡片拖拽。
- 为了照搬 Ant Design API 而引入本组件——它不对应任何 AntD 组件。

## 代码演示 {#examples}

### 基础用法

使用 `KanbanProvider` 包裹，通过 render prop 渲染各列。

```tsx
import {
  KanbanProvider,
  KanbanBoard,
  KanbanHeader,
  KanbanCards,
  KanbanCard,
} from "@/components/ui/kanban"

const columns = [
  { id: "todo", name: "待办" },
  { id: "in-progress", name: "进行中" },
  { id: "done", name: "已完成" },
]

const initialData = [
  { id: "1", name: "设计稿评审", column: "todo" },
  { id: "2", name: "API 联调", column: "in-progress" },
  { id: "3", name: "发布上线", column: "done" },
]

export default function Demo() {
  const [data, setData] = useState(initialData)

  return (
    <KanbanProvider
      columns={columns}
      data={data}
      onDataChange={setData}
    >
      {(column) => (
        <KanbanBoard key={column.id} id={column.id}>
          <KanbanHeader>{column.name}</KanbanHeader>
          <KanbanCards id={column.id}>
            {(item) => (
              <KanbanCard key={item.id} {...item} />
            )}
          </KanbanCards>
        </KanbanBoard>
      )}
    </KanbanProvider>
  )
}
```

## API {#api}

### KanbanProvider

拖拽上下文容器，封装 @dnd-kit 的 DndContext、传感器和 announcements。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| columns | `KanbanColumnProps[]` | 必填 | 列定义数组 |
| data | `KanbanItemProps[]` | 必填 | 卡片数据数组 |
| children | `(column: KanbanColumnProps) => ReactNode` | 必填 | render prop，为每列渲染内容 |
| onDataChange | `(data: KanbanItemProps[]) => void` | - | 数据变更回调，拖拽结束后触发 |
| onDragStart | `(event: DragStartEvent) => void` | - | 拖拽开始回调 |
| onDragOver | `(event: DragOverEvent) => void` | - | 拖拽经过回调 |
| onDragEnd | `(event: DragEndEvent) => void` | - | 拖拽结束回调 |
| className | `string` | - | 根容器 className |

`KanbanProvider` 同时接受 `DndContextProps` 的其他属性（如 `collisionDetection`、`modifiers` 等），可通过展开传入覆盖默认值。

### KanbanBoard

列的 droppable 容器。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| id | `string` | 必填 | 列唯一标识，对应 data 中的 `column` 字段 |
| children | `ReactNode` | - | 列内容（通常为 KanbanHeader + KanbanCards） |
| className | `string` | - | 额外样式类名 |

### KanbanHeader

列标题。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| className | `string` | - | 额外样式类名 |
| （其他） | `HTMLAttributes<HTMLDivElement>` | - | 支持 div 原生属性 |

### KanbanCards

卡片列表容器，自动过滤当前列的数据并包裹 `SortableContext`。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| id | `string` | 必填 | 列 ID，用于过滤 data 中 `column` 匹配的卡片 |
| children | `(item: KanbanItemProps) => ReactNode` | 必填 | render prop，为每个卡片渲染内容 |
| className | `string` | - | 额外样式类名 |
| （其他） | `HTMLAttributes<HTMLDivElement>` | - | 支持 div 原生属性（children 和 id 除外） |

### KanbanCard

可拖拽卡片。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| id | `string` | 必填 | 卡片唯一标识 |
| name | `string` | 必填 | 卡片显示名称（无 children 时作为默认内容） |
| column | `string` | 必填 | 所属列 ID |
| children | `ReactNode` | - | 自定义卡片内容，不传则显示 name |
| className | `string` | - | 额外样式类名 |

`KanbanCard` 支持泛型 `<T extends KanbanItemProps>`，可在数据项上携带任意额外字段。

### KanbanItemProps

```ts
type KanbanItemProps = {
  id: string;
  name: string;
  column: string;
} & Record<string, unknown>;
```

### KanbanColumnProps

```ts
type KanbanColumnProps = {
  id: string;
  name: string;
} & Record<string, unknown>;
```

## 样式模型 {#styling-model}

- 使用 Tailwind CSS 和项目主题变量（`bg-secondary`、`text-foreground`、`ring-primary` 等）。
- `KanbanBoard`：带边框圆角的列容器，拖拽悬停时显示 `ring-primary` 高亮环。
- `KanbanHeader`：半粗体小字标题。
- `KanbanCards`：纵向 flex 列表，包裹在 `ScrollArea` 中支持溢出滚动。
- `KanbanCard`：圆角小卡片，拖拽时半透明（opacity-30）且 pointer-events-none；拖拽预览（DragOverlay）通过 `tunnel-rat` portal 渲染到 body，保持完整不透明和蓝色选中环。
- 根网格容器使用 `grid auto-cols-fr grid-flow-col` 实现等宽列水平排列。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/blocks/kanban/index.tsx`

直接依赖摘要：

- `@dnd-kit/core`（DndContext、DragOverlay、sensors、closestCenter）
- `@dnd-kit/sortable`（SortableContext、useSortable、arrayMove）
- `@dnd-kit/utilities`（CSS.Transform）
- `tunnel-rat`（DragOverlay portal 隧道）
- `@/components/ui/card`（Card 组件）
- `@/components/ui/scroll-area`（ScrollArea、ScrollBar）
- `react-dom`（createPortal）

实现注意：

1. `KanbanProvider` 内部管理 `activeCardId` 状态，用于控制 DragOverlay 中显示拖拽预览。
2. 传感器默认使用 MouseSensor、TouchSensor、KeyboardSensor，支持鼠标、触屏和键盘拖拽。
3. 列间移动逻辑：在 `handleDragOver` 中检测跨列拖拽，更新卡片 `column` 字段并调用 `arrayMove` 重排。
4. announcements 对象提供屏幕阅读器播报，覆盖拖拽全生命周期（开始、经过、结束、取消）。
5. 组件通过泛型支持扩展数据字段，可在 `KanbanItemProps` 和 `KanbanColumnProps` 上附加任意属性。
6. DragOverlay 通过 `tunnel-rat` + `createPortal` 渲染到 `document.body`，确保拖拽预览不受父容器 overflow/clip 影响。

## 可访问性 {#accessibility}

- 使用 `@dnd-kit` 的 `accessibility.announcements` 提供完整的屏幕阅读器播报。
- 播报内容覆盖拖拽各阶段：拾取卡片、经过列上方、放入列中、取消拖拽。
- 支持键盘拖拽（KeyboardSensor），符合 WAI-ARIA 拖拽交互模式。
- `KanbanCard` 渲染为带 `cursor-grab` 样式的 div，拖拽中变为 `cursor-grabbing`。
- 推荐在使用时设置列标题的可见文本（通过 `KanbanHeader`），确保列有明确语义标识。

## 与 Ant Design 的取舍 {#antd-differences}

该组件为项目本地复合组件，无 Ant Design 直接对应。Ant Design 不提供看板/Kanban 组件。

| 方向 | 本地决策 |
| --- | --- |
| 拖拽框架 | 使用 @dnd-kit，非 AntD 生态 |
| API 来源 | 以 `src/components/blocks/kanban/index.tsx` 为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 组合模式 | render prop + 复合子组件，非 AntD 式单一组件 |
| 数据管理 | 受控模式，通过 `onDataChange` 回调驱动数据源 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Kanban 代码时，应遵守：

1. 从 `@/components/ui/kanban` 导入全部子组件，不要从 `antd` 导入。
2. `KanbanProvider` 必须提供 `columns`、`data` 和 `onDataChange`，使用 render prop 模式渲染各列。
3. 每个 `KanbanBoard` 的 `id` 必须与数据项的 `column` 字段以及 `columns` 中的列 ID 对应。
4. `KanbanCards` 的 `id` 用于过滤数据，`children` 必须是 render prop `(item) => <KanbanCard />`。
5. 数据驱动：在 `onDataChange` 中更新 state，组件本身不持有数据。
6. 只使用本文件 API 表格和源码中存在的 props，不要凭 AntD 或其他看板库的记忆生成 API。
7. 需要自定义卡片内容时，通过 `KanbanCard` 的 `children` 渲染，不要在外部包装。
8. 确保 `KanbanCard` 的每个 item 有唯一的 `id`，否则拖拽排序行为异常。
