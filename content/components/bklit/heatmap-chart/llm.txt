## 何时使用
以颜色强度矩阵展示二维数据密度与分布。适合贡献日历、时间分布矩阵、多维交叉分析。采用 GitHub 风格单元格渲染，内置单元格悬停高亮、图例联动、加载骨架屏和流畅入场动画。稀疏数据或单维度趋势场景考虑使用折线图/柱状图替代。

## 核心数据模型

```ts
// 每一列代表一个周期（如一周）
interface HeatmapColumn {
  bin: number   // 列索引
  bins: HeatmapBin[]  // 该列内每行（如每天）的数据
}

// 每一个 bin 代表一个单元格
interface HeatmapBin {
  count: number   // 数值（决定颜色深浅，0-4 映射至 5 级色阶）
  bin: number     // 行索引
  date: Date      // 日期（用于 X 轴月份标签和 tooltip 格式化）
}
```

## 约束

- 依赖 @visx/heatmap、@visx/responsive、@visx/group、@visx/scale 及 motion (framer-motion)
- 数据须符合 `HeatmapColumn[]` 格式：每列包含 `bins` 数组，每个 bin 含 `count`/`bin`/`date`
- `count` 值按阈值（0, 1, 2, 3, 4+）映射至 5 级色阶；如需自定义分段逻辑，使用 `colorScale` 回调
- `HeatmapChart` 必须包含至少一个子组件（`HeatmapCells`），否则只渲染空白 SVG
- `layout="fluid"` 时高度随数据行数自适应（GitHub 风格）；`layout="fill"` 时需父容器有明确高度
- Y 轴标签固定显示 Mon/Wed/Fri，行按 0=Sun 至 6=Sat 索引
- 每个 `HeatmapColumn` 的 `bins` 长度应一致（建议 7），长度不一致时取首个 column 的 bins 长度

## 最佳实践

- 颜色选择：使用 5 级渐变色阶（低→高），避免使用不可区分的色相；提供 `lessLabel`/`moreLabel` 标注
- 动画：默认启用入场动画（1600ms），在仪表盘等需要即时显示的场景可通过 `animate={false}` 关闭
- 性能：超过 100 列时考虑使用 `xDomain` 限制可见范围，避免渲染过多 DOM 节点
- 交互：`HeatmapTooltip` 自动监听单元格悬停；如需自定义格式化，传入 `formatLabel`
- 加载态：通过 `status="loading"` + `loadingLabel` 渲染骨架屏；`HeatmapChartLoading` 提供开箱即用的加载预设
- 响应式：组件内置 `ParentSize` 自适应容器宽度；通过 `aspectRatio` 或父容器高度控制纵向空间
- 无障碍：X/Y 轴标签通过 `createPortal` 渲染至容器 DOM，使用 `aria-hidden="true"` 装饰性 SVG 与可见标签分离
