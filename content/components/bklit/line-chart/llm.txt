## 何时使用

折线图适合展示**时序数据的趋势变化与对比**。支持多条折线叠加、平滑曲线插值、盈亏分段着色、边缘渐隐、hover 高亮、散点标记、投影区间、加载骨架屏及动画入场效果。

常见场景：时序指标监控、多系列趋势对比、财务盈亏可视化、实时流式数据、历史+预测投影。

不适用场景：离散类别对比（用 BarChart）、占比展示（用 PieChart）、维度超过 5 条（拆分为多图或限制展示）、离散点分布（用 ScatterChart）、极少数数据点（<3 个）。

## 示例代码

```tsx
import { LineChart } from "@/components/ui/bklit-line-chart"

const data = [
  { date: new Date(2024, 0, 1), value: 100 },
  { date: new Date(2024, 0, 2), value: 200 },
  { date: new Date(2024, 0, 3), value: 150 },
  { date: new Date(2024, 0, 4), value: 300 },
  { date: new Date(2024, 0, 5), value: 250 },
  { date: new Date(2024, 0, 6), value: 350 },
  { date: new Date(2024, 0, 7), value: 280 },
]

export default function Demo() {
  return (
    <LineChart data={data}>
      <Grid />
      <Line dataKey="value" stroke="var(--chart-line-primary)" />
    </LineChart>
  )
}
```

### 投影区间（历史+预测虚线）

```tsx
import { LineChart } from "@/components/ui/bklit-line-chart"
import { Line, Grid, ProjectionLine } from "@/components/bklit/charts"

const historicalData = [
  { date: new Date(2024, 0, 1), value: 100 },
  { date: new Date(2024, 0, 2), value: 200 },
  { date: new Date(2024, 0, 3), value: 150 },
  { date: new Date(2024, 0, 4), value: 300 },
  { date: new Date(2024, 0, 5), value: 250 },
]
const projectionData = [
  { date: new Date(2024, 0, 6), value: 350 },
  { date: new Date(2024, 0, 7), value: 380 },
]

const allData = [...historicalData, ...projectionData]

export default function Demo() {
  return (
    <LineChart data={allData}>
      <Grid />
      <Line
        dataKey="value"
        dashFromIndex={historicalData.length}
        dashArray="6,4"
        stroke="var(--chart-line-primary)"
      />
    </LineChart>
  )
}
```

## 架构说明

- LineChart 内部通过 `ParentSize`（@visx/responsive）自适应容器尺寸
- 使用 `scaleTime` + `scaleLinear`（@visx/scale）计算时间轴映射和数值映射
- 线条渲染基于 `LinePath`（@visx/shape），支持任意 d3 curve 插值函数
- 动画基于 `motion/react`，reveal clipPath 控制线条按时间方向逐步显现
- y-domain 自动从所有 Line 子组件的 dataKey 聚合计算，支持多系列共享或独立 Y 轴
- 加载骨架屏通过 `status="loading"` 触发，自动渲染 shimmer 标签和脉冲动画
- Line 组件通过 `ChartProvider` context 获取父级 scale 和布局参数
- 配合 `ChartBrush` / `ChartBrushLayout` 实现时间区间缩放联动

## 约束与最佳实践

1. 导入路径：主组件 `@/components/ui/bklit-line-chart`，子组件 `@/components/bklit/charts`
2. 支持从 `@/components/bklit/charts` 导入的常用子组件：`Line`、`Grid`、`ChartTooltip`、`TooltipIndicator`、`TooltipBox`、`TooltipContent`、`XAxis`、`YAxis`、`ChartBrush`、`ChartBrushLayout`、`ChartLegend`、`ReferenceArea`、`ProfitLossLine`、`ProjectionLine`、`ProjectionLineEndMarker`、`SeriesPointMarker`、`generateChartSkeletonData`
3. data 中日期字段必须为 Date 对象，不能是字符串或时间戳
4. 每个 data 项的数值字段值必须为 number 类型
5. 多系列时每个 Line 必须指定唯一的 dataKey
6. 颜色推荐使用 CSS 变量：`var(--chart-line-primary)`、`var(--chart-1)`～`var(--chart-5)`、`var(--chart-grid)`、`var(--foreground)`
7. 数据点建议 5-100 个；超过 500 个数据点默认开启 decimation（降采样）
8. 始终搭配 Grid 渲染网格线，增强数值可读性
9. 无数据时使用 `status="loading"` + `loadingLabel` 展示骨架屏，而非空白或 "No Data" 文本
10. 需要预测区间时使用 `dashFromIndex` 从实际数据末尾起将折线转为虚线
11. 需要边缘渐隐效果时使用 `fadeEdges` 属性，而非手动控制 opacity
12. 多系列对比时使用不同的 CSS 颜色变量区分
13. 配合 `ChartBrush`/`ChartBrushLayout` 实现缩放时，通过 `xDomain` 和 `tweenYDomainOnXDomainChange` 控制
14. 盈亏可视化使用 `ProfitLossLine` 替代 `Line`，自动按正负值分段着色
15. 加载状态使用 `status="loading"`，不要自行实现骨架屏逻辑
16. 动画通过 `animationDuration`、`animationEasing`、`enterTransition` 控制，不要写自定义动画覆盖
17. 不要引入 Recharts、Ant Design Charts、ECharts 等第三方图表库
18. LineChart 内部使用 ParentSize 自适应容器尺寸，无需手动传 width/height
19. 所有子组件（Line、Grid、ChartTooltip 等）必须通过 children 传入，不要在外层包裹
20. 折线条数建议控制在 5 条以内，超出时考虑拆分多图或使用图例交互
