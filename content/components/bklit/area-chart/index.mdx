面积图用于展示随时间变化的连续数值趋势，通过填充区域直观呈现数据的体量与波动范围。

<h2 id="when-to-use">何时使用</h2>

面积图适合展示**随时间序列变化的连续数据**，填充区域能增强数值体量的视觉感知，让趋势与数据规模一目了然。基于 BKLit Charts 库（底层 @visx），支持流畅的入场动画、加载骨架屏、渐变填充、边缘渐隐以及缩放联动。

常见场景：

- 时序指标监控：展示一段周期内指标（如 DAU、营收、缓存命中率）的变化趋势与总体规模。
- 堆叠对比分析：多系列面积叠加，对比各部门/渠道的收入贡献变化，突出总量与占比。
- 加载骨架屏：通过 `status="loading"` 与 `loadingLabel` 渲染加载骨架，配合骨架数据实现无跳动的内容过渡。
- 缩放联动：配合 `ChartBrush` 或 `xDomain` 实现时间范围缩放，聚焦特定时间段的数据细节。
- 渐变填充与视觉增强：通过 `fill`、`fillOpacity`、`gradientSpan`、`gradientToOpacity` 自定义填充渐变，突出数据范围感。

不适用场景：

- 离散类别对比：类别差异大且无需关注趋势变化时，优先使用柱状图（`BarChart`）。
- 占比展示：各部分占整体的比例关系，使用饼图（`PieChart`）或环形图（`RingChart`）。
- 非时序数据：x 轴为非时间序列的连续数据，考虑使用折线图（`LineChart`）或散点图（`ScatterChart`）。
- 对比维度超过 5 个：过多系列叠加会导致填充区域相互遮挡、难以辨认，建议拆分为多个图表或使用折线图。

<h2 id="faq">常见问题</h2>

### 如何自定义颜色？

通过 `Area` 子组件的 `fill` 和 `stroke` 属性设置，支持 CSS 变量、十六进制颜色、渐变 URL 和 pattern 填充：

```tsx
<AreaChart data={data}>
  <Area
    dataKey="value"
    fill="var(--chart-1)"
    fillOpacity={0.25}
    stroke="var(--chart-1)"
    strokeWidth={2}
  />
</AreaChart>
```

使用 pattern 图案填充：

```tsx
import { PatternArea } from "@/components/bklit/charts"

<AreaChart data={data}>
  <PatternArea dataKey="value" pattern="circles" stroke="var(--chart-1)" />
</AreaChart>
```

### 如何展示加载骨架屏？

设置 `status="loading"` 并提供 `loadingLabel`。配合 `generateChartSkeletonData` 生成骨架数据可实现无缝过渡：

```tsx
import { AreaChart } from "@/components/ui/bklit-area-chart"
import { Area, Grid, generateChartSkeletonData } from "@/components/bklit/charts"

const skeletonData = generateChartSkeletonData({ dataKey: "value", pointCount: 8 })

export default function LoadingDemo() {
  return (
    <AreaChart data={skeletonData} status="loading" loadingLabel="加载中...">
      <Grid />
      <Area dataKey="value" fill="var(--chart-line-primary)" fillOpacity={0.3} />
    </AreaChart>
  )
}
```

### 如何处理多系列数据？

在同个 `AreaChart` 内放置多个 `Area` 子组件，每个对应不同的 `dataKey`：

```tsx
const multiSeriesData = [
  { date: new Date(2024, 0, 1), revenue: 500, costs: 300 },
  { date: new Date(2024, 0, 2), revenue: 600, costs: 250 },
  // ...
]

<AreaChart data={multiSeriesData}>
  <Grid />
  <Area dataKey="revenue" fill="var(--chart-1)" fillOpacity={0.3} stroke="var(--chart-1)" />
  <Area dataKey="costs" fill="var(--chart-2)" fillOpacity={0.3} stroke="var(--chart-2)" />
</AreaChart>
```

### 如何添加 Tooltip？

使用 `ChartTooltip` 子组件，配合 `TooltipIndicator`、`TooltipBox` 和 `TooltipContent`：

```tsx
import { ChartTooltip, TooltipIndicator, TooltipBox, TooltipContent } from "@/components/bklit/charts"

<AreaChart data={data}>
  <Grid />
  <Area dataKey="value" fill="var(--chart-line-primary)" fillOpacity={0.3} stroke="var(--chart-line-primary)" />
  <ChartTooltip>
    <TooltipIndicator />
    <TooltipBox>
      <TooltipContent />
    </TooltipBox>
  </ChartTooltip>
</AreaChart>
```

### 如何配合 ChartBrush 实现缩放？

通过 `ChartBrush` 与 `ChartBrushLayout` 联动设置 `xDomain`：

```tsx
import { ChartBrushLayout, useChartInteraction } from "@/components/bklit/charts"

function ZoomableChart() {
  const { selection, clearSelection } = useChartInteraction()
  return (
    <ChartBrushLayout
      selection={selection}
      onClearSelection={clearSelection}
      renderChart={(xDomain) => (
        <AreaChart data={data} xDomain={xDomain}>
          <Grid />
          <Area dataKey="value" fill="var(--chart-line-primary)" fillOpacity={0.3} stroke="var(--chart-line-primary)" />
        </AreaChart>
      )}
    />
  )
}
```

<h2 id="llm-rules">LLM 使用规则</h2>

- 导入: `import { AreaChart } from "@/components/ui/bklit-area-chart"`
- 子组件: `import { Area, Grid, ChartTooltip, TooltipIndicator, TooltipBox, TooltipContent } from "@/components/bklit/charts"`
- 来自 BKLit Charts 库（基于 @visx/xychart 和 @visx/shape）
- 必须提供 `data` 属性（`Record<string, unknown>[]`，每项含 Date 和数值字段）
- `xDataKey` 默认值为 `"date"`，数据中日期字段为 `Date` 对象
- 推荐搭配 `Grid` 子组件渲染网格线，提升可读性
- 多系列时每个 `Area` 必须指定 `dataKey`
- 加载状态使用 `status="loading"` 配合 `loadingLabel`，不要自行实现骨架屏
- 颜色优先使用 CSS 变量（`var(--chart-line-primary)`、`var(--chart-1)`～`var(--chart-5)`）
- 不要引入 Recharts、Ant Design Charts 等第三方图表库
