## 何时使用

面积图适合展示**随时间序列变化的连续数据**，填充区域能增强数值体量的视觉感知。支持入场动画、加载骨架屏、渐变填充、边缘渐隐、缩放联动等能力。

常见场景：时序指标监控、堆叠对比分析、加载骨架屏、缩放联动、渐变填充增强。

不适用场景：离散类别对比（用 BarChart）、占比展示（用 PieChart）、对比维度超过 5 个（拆分为多图或改用 LineChart）。

## 示例代码

### 基础面积图

import { AreaChart } from "@/components/ui/bklit-area-chart"
import { Area, Grid } from "@/components/bklit/charts"

const data = [
  { date: new Date(2024, 0, 1), value: 100 },
  { date: new Date(2024, 0, 2), value: 220 },
  { date: new Date(2024, 0, 3), value: 150 },
  { date: new Date(2024, 0, 4), value: 380 },
  { date: new Date(2024, 0, 5), value: 260 },
  { date: new Date(2024, 0, 6), value: 310 },
  { date: new Date(2024, 0, 7), value: 420 },
]

<AreaChart data={data}>
  <Grid />
  <Area dataKey="value" fill="var(--chart-line-primary)" fillOpacity={0.3} stroke="var(--chart-line-primary)" />
</AreaChart>

### 加载骨架屏

import { AreaChart } from "@/components/ui/bklit-area-chart"

const skeletonData = generateChartSkeletonData({ dataKey: "value", pointCount: 8 })

  <Grid />
  <Area dataKey="value" fill="var(--chart-line-primary)" fillOpacity={0.3} />
</AreaChart>

## 约束

- 需要 @visx/* 系列依赖（@visx/responsive、@visx/shape、@visx/curve、@visx/gradient、@visx/scale、@visx/grid）
- 需要 motion（原 framer-motion）依赖
- 需要 d3-array 依赖
- data 中日期字段必须为 Date 对象，不能是字符串或时间戳
- xDataKey 对应字段值会被转为 Date 进行时间尺度映射
- 每个 data 项的数据字段值必须为 number 类型
- 多系列时每个 Area 必须指定唯一的 dataKey
- 建议在容器中设置 aspectRatio 保证渲染比例
- 颜色推荐使用 CSS 变量：var(--chart-line-primary)、var(--chart-1)～var(--chart-5)、var(--chart-grid)、var(--foreground)
- 加载状态用 status="loading"，不要自行实现骨架屏逻辑
- 动画通过 animationDuration、animationEasing、enterTransition 控制，不要写自定义动画覆盖
- 不要引入 Recharts、Ant Design Charts、ECharts 等第三方图表库
- AreaChart 内部使用 ParentSize 自适应容器尺寸，无需手动传 width/height
- 所有子组件（Area、Grid、ChartTooltip 等）必须通过 children 传入，不要在外层包裹
- 从 @/components/ui/bklit-area-chart 导入 AreaChart
- 从 @/components/bklit/charts 导入 Area、Grid、ChartTooltip、PatternArea 等子组件

## 最佳实践

- 始终搭配 Grid 渲染网格线，增强数值可读性
- 数据点建议 5-100 个，超过 500 个数据点图表默认开启 decimation（降采样）
- 无数据时用 status="loading" + loadingLabel 展示骨架屏，而非空白或 "No Data" 文本
- 需要预测区间时用 dashFromIndex 从实际数据末尾起将折线转为虚线
- 需要部分透明效果时用 fadeEdges 控制边缘渐隐，而非手动控制 opacity
- 多系列时使用不同的 CSS 颜色变量区分：var(--chart-1)、var(--chart-2) 等
- 配合 ChartBrush/ChartBrushLayout 实现缩放时，通过 xDomain 和 tweenYDomainOnXDomainChange 控制
- pattern 图案填充用 PatternArea 子组件替代 Area，pattern 属性支持 circles、hexagons、lines、waves
