类型导入: import type { FunnelChartProps, FunnelStage, FunnelGradientStop } from "@/components/ui/bklit-funnel-chart"

组件类型: 图表 / BKLit Charts

## 何时使用

展示分阶段转化率数据，直观呈现用户从初始阶段到最终目标各步骤的流失与留存情况。适用于用户增长分析、销售漏斗监控、电商转化分析和运营活动评估等场景。

## Props 速查

### FunnelChartProps

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| data | 漏斗阶段数据，数组第一项为最大值（漏斗入口） | `FunnelStage[]` | — (必填) |
| orientation | 漏斗方向 | `"horizontal"` \| `"vertical"` | `"horizontal"` |
| color | 默认分段填充色，支持 CSS 变量 | `string` | `"var(--chart-1)"` |
| layers | 每个分段的视觉层级（光晕环数） | `number` | `3` |
| className | 附加样式类名 | `string` | — |
| style | 内联样式 | `CSSProperties` | — |
| showPercentage | 是否显示转化百分比 | `boolean` | `true` |
| showValues | 是否显示阶段数值 | `boolean` | `true` |
| showLabels | 是否显示阶段名称 | `boolean` | `true` |
| hoveredIndex | 受控 hover 状态，hover 的分段索引 | `number \| null` | — |
| onHoverChange | hover 状态变化回调 | `(index: number \| null) => void` | — |
| formatPercentage | 百分比格式化函数 | `(pct: number) => string` | `(p) => \`${Math.round(p)}%\`` |
| formatValue | 数值格式化函数 | `(value: number) => string` | `intFmt`（千分位整数） |
| staggerDelay | 分段入场动画的间隔延迟（秒） | `number` | `0.12` |
| enterTransition | Framer Motion 入场动画配置 | `Transition` | — |
| gap | 分段之间的间距（px） | `number` | `4` |
| renderPattern | 自定义 visx 图案渲染函数 | `(id: string, color: string) => ReactNode` | — |
| edges | 分段边缘样式 | `"curved"` \| `"straight"` | `"curved"` |
| labelLayout | 标签布局模式 | `"spread"` \| `"grouped"` | `"spread"` |
| labelOrientation | 标签组排列方向（`labelLayout="grouped"` 时生效） | `"vertical"` \| `"horizontal"` | 水平漏斗默认 `"vertical"`，垂直漏斗默认 `"horizontal"` |
| labelAlign | 标签组对齐位置 | `"center"` \| `"start"` \| `"end"` | `"center"` |
| grid | 网格配置，传 `true` 开启默认网格，传对象精细控制 | `boolean \| { bands?, bandColor?, lines?, lineColor?, lineOpacity?, lineWidth? }` | `false` |

### FunnelStage

| 字段 | 说明 | 类型 | 必填 |
|------|------|------|------|
| label | 阶段名称 | `string` | 是 |
| value | 阶段数值，必须递减 | `number` | 是 |
| displayValue | 自定义显示值，覆盖 formatValue | `string` | 否 |
| color | 覆盖该分段的默认颜色 | `string` | 否 |
| gradient | 渐变色停靠点数组 | `FunnelGradientStop[]` | 否 |

### FunnelGradientStop

| 字段 | 说明 | 类型 |
|------|------|------|
| offset | 渐变位置 | `string \| number` |
| color | 颜色值 | `string` |

## 示例代码

### 基础用法

```tsx
import { FunnelChart } from "@/components/ui/bklit-funnel-chart"

const data = [
  { label: "访问", value: 5000 },
  { label: "注册", value: 3000 },
  { label: "下单", value: 1500 },
  { label: "支付", value: 800 },
]

export default function Demo() {
  return <FunnelChart data={data} />
}
```

### 自定义颜色与渐变

```tsx
import { FunnelChart } from "@/components/ui/bklit-funnel-chart"
import type { FunnelStage } from "@/components/ui/bklit-funnel-chart"

const data: FunnelStage[] = [
  { label: "曝光", value: 10000, color: "#8B5CF6" },
  {
    label: "点击",
    value: 6000,
    gradient: [
      { offset: "0%", color: "#3B82F6" },
      { offset: "100%", color: "#06B6D4" },
    ],
  },
  { label: "转化", value: 2000, color: "#10B981" },
]

export default function Demo() {
  return (
    <FunnelChart
      data={data}
      layers={4}
      showPercentage={true}
      showValues={true}
    />
  )
}
```

### 垂直漏斗 + 直边 + 网格

```tsx
import { FunnelChart } from "@/components/ui/bklit-funnel-chart"

const data = [
  { label: "线索", value: 8000 },
  { label: "商机", value: 4000 },
  { label: "报价", value: 2000 },
  { label: "签约", value: 600 },
]

export default function Demo() {
  return (
    <FunnelChart
      data={data}
      orientation="vertical"
      edges="straight"
      grid={{ bands: true, lines: true }}
      gap={2}
    />
  )
}
```

## 约束

- 需要 `motion/react` 依赖，Framer Motion >= 11。
- 数据数组 `data` 的第一项必须为最大值（漏斗入口），后续值应递减。
- 当 `data` 为空数组时，组件返回 `null` 不渲染任何内容。
- 组件容器使用 `aspectRatio` 自适应宽高（水平 `2.2/1`，垂直 `1/1.8`），无需手动设置固定尺寸。
- 每个分段的 `label` 作为 React key，必须唯一。
- `hoveredIndex` 和 `onHoverChange` 为受控 hover 模式，不传时组件内部自行管理 hover 状态。
- 仅在 `labelLayout="grouped"` 时 `labelOrientation` 和 `labelAlign` 才生效；`"spread"` 模式下标签自动分布到分段两端和中心。

## 最佳实践

- 漏斗阶段建议控制在 3-8 个，过多阶段会降低可读性。
- 数据值必须递减，非递减数据应改用柱状图或条形图。
- 优先使用 `color` 和 `gradient` 表达阶段语义，而非仅靠标签文字。
- `layers` 建议 2-5，过少缺乏视觉层次，过多造成渲染性能浪费。
- 移动端考虑使用 `orientation="vertical"` 改善窄屏阅读体验。
- 需要精确数值时配合数据表格，漏斗图本身侧重趋势表达。
- 网格 (`grid`) 适合多阶段场景，帮助用户对齐视觉参考线。
- 受控 hover 模式下，可结合外部状态实现联动高亮或详情面板。
