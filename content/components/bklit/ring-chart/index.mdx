RingChart 用于展示环形进度或占比数据，支持多层嵌套环、动画进入效果和 hover 交互。

<h2 id="when-to-use">何时使用</h2>

环形图（RingChart）是 BKLit Charts 图表库中的环形进度组件，基于 @visx 底层图形库构建。每个环的弧度精确反映 value 与 maxValue 的占比关系，支持多层嵌套环同时展示多组指标的完成进度。组件内置进入动画（展开 + 进度填充）和 hover 高亮交互，环与环之间自动处理视觉层次（hover 放大、非 hover 淡化、外层推开）。

常见场景：

- 进度展示：任务完成度、目标达成率、容量使用率、预算消耗比例
- 占比对比：多维度数据在整体中的占比关系（如各渠道流量占比）
- 仪表盘 KPI：结合 RingCenter 在环中心展示核心数字和标签文案，支持 hover 切换显示单项数据
- 多层指标：使用多个 Ring 子组件展示不同层级的指标（如年度/季度/月度完成率）
- 实时数据看板：通过 geometryScrubbing 模式支持高频数据更新，适用于 Studio 编辑器的拖拽调参场景

不适用的场景：

- 精确数值对比：环图难以精确比较多个值的大小差异，此时应使用 BarChart
- 时序趋势：展示数据随时间的变化趋势，应使用 LineChart 或 AreaChart
- 超过 5 层嵌套：环形层数过多会导致视觉拥挤，建议不超过 5 层
- 分类占比（非环形）：如需展示分类在整体中的占比，应使用 PieChart

<h2 id="examples">代码演示</h2>

### 基础用法

<ComponentShowcase id="bklit/ring-chart" path="bklit/ring-chart/basic" />

```tsx
import { RingChart } from "@/components/ui/bklit-ring-chart"
import { Ring } from "@/components/bklit/charts"

const data = [
  { label: "Q1", value: 85, maxValue: 100, color: "var(--chart-1)" },
  { label: "Q2", value: 65, maxValue: 100, color: "var(--chart-2)" },
  { label: "Q3", value: 45, maxValue: 100, color: "var(--chart-3)" },
]

export default function Demo() {
  return (
    <RingChart data={data} baseInnerRadius={50} strokeWidth={14}>
      <Ring index={0} />
      <Ring index={1} />
      <Ring index={2} />
    </RingChart>
  )
}
```

<h2 id="api">API</h2>

### RingChart Props

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| data | 环数据数组，每项代表一个嵌套环 | `RingData[]` | — |
| size | 固定尺寸 (px)，不传则自适应父容器 | `number` | — |
| strokeWidth | 环的描边宽度 | `number` | `12` |
| ringGap | 环与环之间的间距 | `number` | `6` |
| baseInnerRadius | 最内层环的内半径 | `number` | `60` |
| animationDuration | 动画持续时间 (ms) | `number` | `1100` |
| className | 容器附加样式类名 | `string` | — |
| hoveredIndex | 受控 hover 索引 | `number \| null` | — |
| onHoverChange | hover 状态变化回调 | `(index: number \| null) => void` | — |
| startAngle | 起始角度 (弧度)，默认顶部 | `number` | `-Math.PI / 2` |
| endAngle | 结束角度 (弧度)，默认完整圆 | `number` | `3 * Math.PI / 2` |
| enterTransition | Framer Motion 进入动画过渡配置 | `Transition` | — |
| enterStaggerScale | 环交错延迟倍率 (1 = 默认) | `number` | `1` |
| geometryScrubbing | 高频几何更新模式，跳过 Motion 路径变形 | `boolean` | `false` |
| children | 子组件 (Ring, RingCenter 等) | `ReactNode` | — |

### RingData

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| label | 环的显示标签 | `string` | — |
| value | 当前值 | `number` | — |
| maxValue | 最大值（决定进度百分比） | `number` | — |
| color | 颜色覆盖，不传则使用默认调色板 | `string` | — |

### Ring Props

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| index | 环在 data 数组中的索引（必填） | `number` | — |
| color | 环的颜色覆盖 | `string` | — |
| animate | 是否启用进入动画 | `boolean` | `true` |
| showGlow | hover 时是否显示发光效果 | `boolean` | `true` |
| lineCap | 端点样式 | `"round" \| "butt"` | `"round"` |

### RingCenter Props

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| defaultLabel | 非 hover 状态下显示的标签 | `string` | `"Total"` |
| formatOptions | NumberFlow 格式化选项 | `ChartStatFlowFormat` | 标准记数法 |
| prefix | 数值前缀（如 "$"） | `string` | — |
| suffix | 数值后缀（如 "%"） | `string` | — |
| className | 容器附加样式类名 | `string` | — |
| valueClassName | 数值文本样式类名 | `string` | — |
| labelClassName | 标签文本样式类名 | `string` | — |
| children | 自定义渲染函数，完全控制中心内容 | `(props) => ReactNode` | — |

<h2 id="faq">常见问题</h2>

### 如何自定义环的颜色？

通过 data 数组中每项的 `color` 属性设置，支持 CSS 变量和任意 CSS 颜色值：

```tsx
const data = [
  { label: "完成", value: 75, maxValue: 100, color: "var(--chart-1)" },
  { label: "进行中", value: 45, maxValue: 100, color: "#3b82f6" },
]
```

也可以在 Ring 子组件上通过 `color` 属性覆盖：

```tsx
<Ring index={0} color="var(--chart-4)" />
```

### 如何在环中心显示内容？

使用 RingCenter 子组件，默认识别 hover 状态自动切换显示总值或单项值：

```tsx
<RingChart data={data}>
  <Ring index={0} />
  <RingCenter defaultLabel="总进度" suffix="%" />
</RingChart>
```

如需完全自定义中心内容，使用 children 渲染函数：

```tsx
<RingCenter>
  {({ value, label, isHovered }) => (
    <div>
      <span>{isHovered ? label : "总计"}</span>
      <span>{value}%</span>
    </div>
  )}
</RingCenter>
```

### 如何控制环的大小？

不传 `size` 时，RingChart 自动占满父容器（aspect-square），适用于响应式布局。传入 `size` 时使用固定尺寸：

```tsx
<RingChart data={data} size={200}>
  <Ring index={0} />
</RingChart>
```

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 RingChart 代码时：

1. 从 `@/components/ui/bklit-ring-chart` 导入 RingChart
2. 从 `@/components/bklit/charts` 导入 Ring、RingCenter 等子组件
3. data 数组中每项必须包含 `label`、`value`、`maxValue`，`color` 可选
4. Ring 子组件必须提供 `index` 属性，对应 data 数组的索引
5. 使用 `baseInnerRadius`（非 `innerRadius`）、`strokeWidth`、`ringGap` 控制环的尺寸
6. 颜色推荐使用 CSS 变量 `var(--chart-1)` 至 `var(--chart-5)`
7. 环数建议不超过 5 层，避免视觉拥挤
8. 不要引入 Ant Design 相关依赖
