雷达图（Radar Chart）以多边形在放射状坐标轴上展示多维数据，直观对比各项指标的强弱。

<h2 id="when-to-use">何时使用</h2>

当需要同时展示多个维度的量化指标，并对不同数据集进行横向比较时使用雷达图。雷达图将数据点映射到以中心为原点的放射轴上，通过封闭的多边形面积表现综合水平，适合在仪表盘、竞品分析、能力评估等场景中快速传递相对关系。

常见场景：

- 能力评估：展示个人或团队在技能、素质等维度上的得分。
- 竞品对比：在不同指标上同时比较多个产品的表现。
- 多维评分：展示同一实体在多个评价维度上的得分分布。
- 性能雷达：展示系统或服务在不同资源维度上的使用率。

不适用场景：

- 指标之间量纲不同且无法归一化时，不推荐使用雷达图。
- 数据集超过 3 组时，多边形会严重重叠，应考虑拆分为多张图或使用平行坐标图。
- 指标数量超过 10 个时，图形密集难以阅读，建议缩减为最具代表性的维度。

<h2 id="examples">代码演示</h2>

### 基础用法

通过 `data` 定义数据集，`metrics` 定义维度轴。值域统一归一化到 0-100。

<ComponentShowcase id="bklit/radar-chart" path="bklit/radar-chart/basic" />

```tsx
import { RadarChart } from "@/components/ui/radar-chart-bklit"
import { RadarArea, RadarGrid, RadarAxis, RadarLabels } from "@/components/charts/bklit/charts"

const metrics = [
  { key: "speed", label: "速度" },
  { key: "strength", label: "力量" },
  { key: "endurance", label: "耐力" },
  { key: "agility", label: "敏捷" },
  { key: "intelligence", label: "智力" },
]

const data = [
  {
    label: "当前能力",
    values: { speed: 80, strength: 65, endurance: 90, agility: 70, intelligence: 55 },
  },
]

export default function Demo() {
  return (
    <RadarChart data={data} metrics={metrics}>
      <RadarGrid />
      <RadarAxis />
      <RadarArea index={0} />
      <RadarLabels />
    </RadarChart>
  )
}
```

<h2 id="faq">常见问题</h2>

### 如何自定义颜色？

通过 `RadarData` 的 `color` 字段为每个数据系列指定颜色。支持 CSS 变量和十六进制值：

```tsx
const data = [
  {
    label: "当前能力",
    values: { speed: 80, strength: 65, endurance: 90 },
    color: "var(--chart-1)",
  },
  {
    label: "目标能力",
    values: { speed: 90, strength: 80, endurance: 85 },
    color: "var(--chart-2)",
  },
]
```

### 如何显示多组数据对比？

在 `data` 数组中传入多条数据，并为每组添加 `RadarArea` 并指定对应的 `index`：

```tsx
const data = [
  { label: "当前能力", values: { speed: 80, strength: 65 } },
  { label: "目标能力", values: { speed: 90, strength: 80 } },
]

<RadarChart data={data} metrics={metrics}>
  <RadarGrid />
  <RadarAxis />
  <RadarArea index={0} />
  <RadarArea index={1} />
  <RadarLabels />
</RadarChart>
```

### 如何控制尺寸？

两种方式：

- 传 `size` 属性指定固定像素尺寸。
- 不传 `size`，组件通过 `ParentSize` 自适应父容器尺寸，父容器需有确定的宽高或 `aspect-square`。

### 数据值应该如何归一化？

`RadarData.values` 的每个值应在 0-100 范围内。原始数据需要在外层归一化后再传入。组件不会自动归一化。

<h2 id="llm-rules">LLM 使用规则</h2>

- 导入: `import { RadarChart } from "@/components/ui/radar-chart-bklit"`
- 子组件: `import { RadarArea, RadarGrid, RadarAxis, RadarLabels } from "@/components/charts/bklit/charts"`
- 必须同时提供 `data` 和 `metrics` 两个属性
- `data` 格式为 `{ label, values: Record<string, number>, color? }[]`
- `metrics` 格式为 `{ key: string, label: string }[]`
- `RadarArea` 使用 `index` 指定数据索引，不使用 `dataKey`/`fill`/`fillOpacity`
- 数据值统一归一化到 0-100
- 来自 BKLit Charts 库（基于 @visx 和 motion）
