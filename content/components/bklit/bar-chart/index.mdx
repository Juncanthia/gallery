BarChart 柱状图用于比较不同分类数据的数值大小，支持垂直/水平布局、分组对比和堆叠展示。

<h2 id="when-to-use">何时使用</h2>

当需要在不同类别之间直观地比较数值差异时，使用柱状图是最佳选择。柱状图通过柱子的高度（或宽度）直观表达数值量级，让读者在一眼扫过时即可感知各分类之间的相对大小和排名关系。适用于业绩对比、排名展示、多系列分组分析以及部分与整体关系的堆叠拆解等场景。

常见场景：

- 不同时间段（月度、季度、年度）的销售额对比，直观反映增长或下滑
- 各部门、各产品线的业绩指标对比，便于快速定位优势与短板
- 排名 Top N 数据展示，如热门商品、高转化渠道
- 同一分类下的多组数据分组对比（多系列柱状图），如预算 vs 实际
- 部分与整体关系的堆叠分析，如销售额按渠道、地域、品类拆分

不推荐：

- 数据分类数超过 20 个时，柱状图会变得拥挤难以阅读，改用 `orientation="horizontal"` 横向布局或横向条形图
- 展示连续数据的趋势变化时，柱状图不如折线图直观，应使用 LineChart
- 占比分析（如市场份额分布），优先使用 PieChart 或 RingChart
- 双变量相关性分析（如身高与体重），应使用 ScatterChart

const data = [

<h2 id="faq">常见问题</h2>

### 如何设置柱的颜色？

通过 Bar 的 `fill` 属性设置。可以使用 CSS 变量（如 `var(--chart-line-primary)`）、渐变 URL（`url(#gradient)`）、或图案 URL（`url(#pattern)`）。

### 如何堆叠柱状图？

设置 `BarChart` 的 `stacked={true}`。

### 如何加载中显示骨架？

设置 `BarChart` 的 `status="loading"`，组件会自动渲染加载骨架。

- 必须提供 `data` 和至少一个 `Bar` 子组件
- 在容器中设置 `aspectRatio` 或固定高度确保渲染
