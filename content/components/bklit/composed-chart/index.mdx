组合图用于在共享时间轴上同时展示多种数据系列，将折线图、面积图和柱状图融为一体，帮助用户在单一视图中比较不同量纲或不同维度的趋势变化。

<h2 id="when-to-use">何时使用</h2>

当需要在同一个时间轴上对比展示多组关联数据时，组合图是理想选择。它将折线的精确趋势、面积的范围感知和柱状图的离散对比能力整合在一个坐标系中，适用于仪表盘、数据报告和监控面板等场景。

常见场景：

- 趋势与构成对比：折线展示收入趋势，柱状图展示按渠道拆分的构成。
- 目标与实际对比：折线展示目标值，柱状图展示实际达成，面积展示两者差距。
- 多指标监控：在同一个时间轴下同时监控 PV、UV、转化率等指标的走势。
- 加载态展示：骨架屏加载动画配合脉冲效果，优化大数据加载体验。

不应使用的场景：

- 单一指标展示：仅需要一个折线或柱状图时，使用 LineChart 或 BarChart 更简洁。
- 无时间维度的分类对比：适合使用 BarChart 或分组柱状图。
- 数据量级差距过大（超过 10 倍）：不同系列共用 Y 轴会导致部分系列不可见，考虑使用双 Y 轴或分开展示。

<h2 id="faq">常见问题</h2>

### 如何自定义颜色？

通过子组件的 `fill` 或 `stroke` 属性设置，支持 CSS 变量和渐变 URL。例如：

```tsx
<Line dataKey="revenue" stroke="var(--chart-line-secondary)" />
<Area dataKey="volume" fill="var(--chart-line-primary)" fillOpacity={0.2} />
<SeriesBar dataKey="count" fill="var(--chart-fill-tertiary)" />
```

### 如何堆叠柱状图？

在 ComposedChart 上设置 `stacked` 属性，多个 `SeriesBar` 子组件将在同一 x 位置堆叠。

```tsx
<ComposedChart data={data} stacked stackGap={1}>
  <SeriesBar dataKey="organic" fill="var(--chart-fill-primary)" />
  <SeriesBar dataKey="paid" fill="var(--chart-fill-secondary)" />
  <Line dataKey="total" stroke="var(--chart-line-primary)" />
</ComposedChart>
```

### 如何给子组件添加加载态？

Line 和 Area 内置加载脉冲动画，通过 `loading` 属性控制。

```tsx
<ComposedChart data={data}>
  <Area dataKey="area" fill="var(--chart-line-secondary)" loading />
  <Line dataKey="line" stroke="var(--chart-line-primary)" loading />
</ComposedChart>
```

### 量级差距大的系列如何共存？

通过 `yAxisId` 将不同系列绑定到不同 Y 轴，实现双轴图表。

```tsx
<ComposedChart data={data}>
  <Line dataKey="conversion" yAxisId="right" stroke="var(--chart-line-secondary)" />
  <Area dataKey="visits" yAxisId="left" fill="var(--chart-line-primary)" fillOpacity={0.2} />
</ComposedChart>
```

<h2 id="llm-rules">LLM 使用规则</h2>

- 导入: `import { ComposedChart } from "@/components/charts/bklit/charts"`
- 子组件导入: `import { Line, Area, SeriesBar } from "@/components/charts/bklit/charts"`
- 来自 BKLit Charts 库（基于 @visx）
- 必须提供 `data` 属性，每行至少包含一个日期字段和若干数值字段
- 日期字段默认 key 为 `"date"`，可通过 `xDataKey` 自定义
- 子组件中 Line 适合精确趋势线，Area 适合范围填充，SeriesBar 适合离散对比
- 颜色通过子组件的 `fill` 或 `stroke` 属性设置，优先使用 CSS 变量（如 `var(--chart-line-primary)`）
- 需要 @visx/* 系列依赖（@visx/responsive, @visx/shape, @visx/curve, @visx/gradient 等）
