组件类型: 图表 / BKLit Charts / 时间序列组合图

## 何时使用

当需要在同一个时间轴上对比展示多组关联数据时使用组合图。它将折线的精确趋势、面积的范围感知和柱状图的离散对比能力整合在一个坐标系中。

适用场景：
- 趋势与构成对比（折线+柱状）
- 目标与实际对比（折线+面积）
- 多指标监控（多折线+多面积）
- 加载态展示（骨架屏脉冲动画）

不适用场景：
- 单一指标展示 → 使用 LineChart 或 BarChart
- 无时间维度的分类对比 → 使用 BarChart
- 数据量级差距过大（超过 10 倍）→ 使用双 Y 轴或分开展示

## 示例代码

### 基础组合图

```tsx
import { ComposedChart } from "@/components/ui/bklit-composed-chart"
import { Line, Area, SeriesBar } from "@/components/bklit/charts"

const data = [
  { date: new Date(2024, 0, 1), line: 100, area: 80, bar: 200 },
  { date: new Date(2024, 0, 2), line: 150, area: 120, bar: 250 },
  { date: new Date(2024, 0, 3), line: 130, area: 100, bar: 180 },
  { date: new Date(2024, 0, 4), line: 200, area: 170, bar: 300 },
  { date: new Date(2024, 0, 5), line: 180, area: 160, bar: 220 },
]

export default function Demo() {
  return (
    <ComposedChart data={data}>
      <Area dataKey="area" fill="var(--chart-line-secondary)" fillOpacity={0.2} />
      <SeriesBar dataKey="bar" fill="var(--chart-line-primary)" fillOpacity={0.3} />
      <Line dataKey="line" stroke="var(--chart-line-primary)" />
    </ComposedChart>
  )
}
```

### 堆叠柱状图 + 折线

```tsx
import { ComposedChart } from "@/components/ui/bklit-composed-chart"
import { Line, SeriesBar } from "@/components/bklit/charts"

export default function StackedDemo() {
  return (
    <ComposedChart data={revenueData} stacked stackGap={1}>
      <SeriesBar dataKey="organic" fill="var(--chart-fill-primary)" />
      <SeriesBar dataKey="paid" fill="var(--chart-fill-secondary)" />
      <Line dataKey="total" stroke="var(--chart-line-primary)" />
    </ComposedChart>
  )
}
```

### 双 Y 轴组合图

```tsx
import { ComposedChart } from "@/components/ui/bklit-composed-chart"
import { Line, Area } from "@/components/bklit/charts"

export default function DualAxisDemo() {
  return (
    <ComposedChart data={analyticsData}>
      <Area dataKey="visits" yAxisId="left" fill="var(--chart-line-primary)" fillOpacity={0.15} />
      <Line dataKey="conversion" yAxisId="right" stroke="var(--chart-line-secondary)" />
    </ComposedChart>
  )
}
```

## 约束

- **依赖**：需要 @visx/* 系列依赖（@visx/responsive, @visx/shape, @visx/curve, @visx/gradient, @visx/scale 等）
- **数据格式**：`data` 数组每项必须包含日期字段（默认为 `date`，Date 对象或可被 `new Date()` 解析的值）和数值字段
- **日期字段**：默认 x 轴 key 为 `"date"`，可通过 `xDataKey` 自定义
- **渲染容器**：组件通过 `aspectRatio`（默认 `"2 / 1"`）和 `ParentSize` 自适应容器宽度，须在外部设置容器宽度
- **颜色规范**：优先使用 CSS 变量（`var(--chart-line-primary)`, `var(--chart-line-secondary)` 等），其次使用 Tailwind 颜色 token，避免硬编码 hex 值
- **子组件限制**：ComposedChart 仅接受 Line、Area、SeriesBar 作为直接子组件，其他类型的子组件将被忽略
- **动画**：入场动画默认 1100ms，通过 `animationDuration` 调整；可通过 `animate={false}` 关闭子组件动画

## 最佳实践

- 折线层级最高，优先放在子组件列表末尾，确保渲染在最上层
- 面积放在折线之前，避免遮挡折线
- 柱状图适合数量较少的离散对比，建议不超过 3 组 SeriesBar
- 多系列时使用 Legend 组件辅助图例识别
- 加载态使用 `loading` 属性配合 `loadingStyle="sweep"` 实现扫光效果
- 大数据量（>100 点）时考虑关闭 `showMarkers` 以减少渲染负担
