Choicebox 选择框，基于 RadioGroup 封装的卡片式单选组件，支持标题、副标题和描述信息。提供组合式子组件 API，适用于列表式互斥选择场景。

<h2 id="when-to-use">何时使用</h2>

当需要以卡片列表形式展示互斥选项时使用，例如套餐选择、支付方式选择、模板选择等。

常见场景：

- 定价页面中的套餐选择（基础版、专业版、企业版）。
- 设置页面中的方案切换。
- 注册流程中的账户类型选择。
- 商品配置中的规格选择。

推荐使用方式：
- Choicebox 包裹多个 ChoiceboxItem，每个 Item 内组合 ChoiceboxItemHeader + ChoiceboxItemTitle + ChoiceboxItemSubtitle + ChoiceboxItemDescription。
- ChoiceboxIndicator 放在 item 末尾作为选中状态指示器。

不推荐：

- 选项超过 6 个仍使用卡片式——此时应考虑 Select 下拉或普通 Radio 列表。
- 在选项卡片中放置过多文本——ChoiceboxItem 设计为紧凑卡片，描述文字宜精简。
- 用 Choicebox 替代 Select 的下拉行为——Choicebox 始终平铺展示所有选项。

### 受控模式

使用 `value` 和 `onValueChange` 实现受控选择。

```tsx
import { useState } from "react"
import {
  Choicebox,
  ChoiceboxItem,
  ChoiceboxItemHeader,
  ChoiceboxItemTitle,
  ChoiceboxItemSubtitle,
  ChoiceboxItemDescription,
  ChoiceboxIndicator,
} from "@/components/blocks/choicebox"

export default function Demo() {
  const [value, setValue] = useState("basic")

  return (
    <div className="space-y-2">
      <p className="text-sm text-muted-foreground">当前选择: {value}</p>
      <Choicebox value={value} onValueChange={setValue}>
        <ChoiceboxItem value="basic">
          <ChoiceboxItemHeader>
            <ChoiceboxItemTitle>基础版</ChoiceboxItemTitle>
            <ChoiceboxItemSubtitle>免费</ChoiceboxItemSubtitle>
          </ChoiceboxItemHeader>
          <ChoiceboxItemDescription>适合个人用户</ChoiceboxItemDescription>
          <ChoiceboxIndicator />
        </ChoiceboxItem>
        <ChoiceboxItem value="pro">
          <ChoiceboxItemHeader>
            <ChoiceboxItemTitle>专业版</ChoiceboxItemTitle>
            <ChoiceboxItemSubtitle>￥99/月</ChoiceboxItemSubtitle>
          </ChoiceboxItemHeader>
          <ChoiceboxItemDescription>适合小团队</ChoiceboxItemDescription>
          <ChoiceboxIndicator />
        </ChoiceboxItem>
      </Choicebox>
    </div>
  )
}
```

<h2 id="api">API</h2>

### Choicebox

本质是 RadioGroup 的卡片样式封装，继承 `ComponentProps<typeof RadioGroup>`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| defaultValue | 非受控模式下的默认选中值 | `string` | - |
| value | 受控模式下的选中值 | `string` | - |
| onValueChange | 选中值变化的回调 | `(value: string) => void` | - |
| className | 附加样式类（默认含 `w-full`） | `string` | - |
| children | ChoiceboxItem 子元素 | `React.ReactNode` | - |

### ChoiceboxItem

每个卡片选项，继承 `ComponentProps<typeof RadioGroupItem>`。内部通过 ChoiceboxItemContext 向子组件传递 value。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| value | 选项值（必填） | `string` | - |
| id | 选项 ID，用于 FieldLabel 的 htmlFor 关联 | `string` | - |
| className | 附加样式类 | `string` | - |
| children | 子组件（Header、Description、Indicator） | `React.ReactNode` | - |

### ChoiceboxItemHeader

包裹标题和副标题的容器，继承 `ComponentProps<typeof FieldContent>`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 通常为 ChoiceboxItemTitle + ChoiceboxItemSubtitle | `React.ReactNode` | - |

### ChoiceboxItemTitle

选项标题，继承 `ComponentProps<typeof FieldTitle>`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 标题文本 | `React.ReactNode` | - |

### ChoiceboxItemSubtitle

选项副标题（如价格），继承 `HTMLAttributes<HTMLSpanElement>`，默认样式 `text-muted-foreground text-xs font-normal`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 副标题文本 | `React.ReactNode` | - |

### ChoiceboxItemDescription

选项描述文本，继承 `ComponentProps<typeof FieldDescription>`。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 附加样式类 | `string` | - |
| children | 描述文本 | `React.ReactNode` | - |

### ChoiceboxIndicator

选中状态指示器，内部通过 ChoiceboxItemContext 自动获取 ChoiceboxItem 的 value，无需调用侧手动传入。继承 `Partial<ComponentProps<typeof RadioGroupItem>>`。

<h2 id="faq">常见问题</h2>

### Choicebox 和 RadioGroup 的区别？

Choicebox 是 RadioGroup 的卡片式封装，每个选项以卡片形式展示，包含标题、副标题和描述信息。RadioGroup 是普通单选列表。当选项需要展示较多信息时使用 Choicebox，简单文本选项使用 RadioGroup。

### ChoiceboxIndicator 需要传 value 吗？

不需要。ChoiceboxIndicator 内部通过 ChoiceboxItemContext 自动获取父级 ChoiceboxItem 的 value 属性。只需在 ChoiceboxItem 内放置 `<ChoiceboxIndicator />` 即可。

### 子组件排列顺序有要求吗？

推荐顺序：ChoiceboxItemHeader（含 Title + Subtitle）-> ChoiceboxItemDescription -> ChoiceboxIndicator。实际顺序自由，但 Header 放在最前、Indicator 放在最后是最佳实践。

### 选项数量过多怎么办？

Choicebox 始终平铺展示所有选项。当选项超过 6 个时，建议改用 Select 下拉组件或普通 RadioGroup，避免页面过长。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/blocks/choicebox` 导入全部子组件。
2. Choicebox 的 children 必须是 ChoiceboxItem，不要混入其他组件。
3. 每个 ChoiceboxItem 必须设置唯一的 `value` 属性。
4. ChoiceboxItem 内部推荐顺序：ChoiceboxItemHeader -> ChoiceboxItemDescription -> ChoiceboxIndicator。
5. 不要引入 Ant Design 相关依赖。
