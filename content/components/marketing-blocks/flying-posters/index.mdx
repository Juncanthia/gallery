FlyingPosters 基于 OGL（WebGL）的飞行海报组件，图片以 3D 扭曲平面形式垂直滚动飞过视口，支持鼠标和触摸滚动。

<h2 id="when-to-use">何时使用</h2>
当需要以垂直滚动飞行的 3D 视觉效果展示图片海报时使用。
常见场景：
- 作品集/画廊的视觉展示
- 产品图片的创意滑动浏览
- 电影/活动海报宣传页
不推荐：
- 需要精确滚动定位的图片浏览（滚动缓动有惯性）
- 大量图片的列表展示（每个 item 一个 WebGL Plane）

### 自定义配置

```tsx
import { FlyingPosters } from "@/components/marketing-blocks/flying-posters"

export default function Demo() {
  return (
    <FlyingPosters
      items={["/images/1.jpg", "/images/2.jpg", "/images/3.jpg", "/images/4.jpg"]}
      planeWidth={400}
      planeHeight={500}
      distortion={5}
      scrollEase={0.02}
      cameraFov={30}
      cameraZ={15}
      className="h-[600px] w-full"
    />
  )
}
```

<h2 id="api">API</h2>
### FlyingPosters
| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| items | 海报图片 URL 数组 | `string[]` | `[]` |
| planeWidth | 每个海报平面的宽度 | `number` | `320` |
| planeHeight | 每个海报平面的高度 | `number` | `320` |
| distortion | 3D 扭曲形变强度 | `number` | `3` |
| scrollEase | 滚动缓动系数（0-1，越小惯性越强） | `number` | `0.01` |
| cameraFov | 相机视场角（FOV） | `number` | `45` |
| cameraZ | 相机在 Z 轴的位置 | `number` | `20` |
| className | 外层容器自定义类名 | `string` | - |

<h2 id="faq">常见问题</h2>
### 滚动是如何实现的？
组件监听 `wheel` 事件、鼠标拖拽和触摸拖拽，将滚动增量累加到 `scroll.target`，通过 `lerp` 缓动插值平滑过渡到 `scroll.current`。多个海报沿 Y 轴排列并无限循环滚动。

### 海报图片需要什么尺寸？
图片会自动适应 planeWidth/planeHeight 的比例（通过 aspect-ratio 保持缩放）。建议使用相同比例的图片以避免裁剪变形。

### 如何禁用特定交互方式？
组件的滚动、鼠标拖拽和触摸事件在内部处理，没有暴露禁用开关。如需限制，可在外层通过 CSS `pointer-events` 控制。

<h2 id="llm-rules">LLM 使用规则</h2>
1. 从 `@/components/marketing-blocks/flying-posters` 导入。
2. 依赖 `ogl`（OGL WebGL 框架），确保项目中已安装。
3. `items` 为必填属性，传入图片 URL 数组，图片需支持跨域访问（设置 `crossOrigin="anonymous"`）。
4. 组件通过 `className` 接收容器样式，需设置固定高度以确保 Canvas 有渲染区域。
