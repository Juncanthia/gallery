HeroClassic 是一个经典的 Hero 区域组件，包含徽章、标题、副标题、双 CTA 按钮、社交证明和 Hero 配图占位区域。采用左右双栏布局，右侧包含装饰性浮动卡片。

<h2 id="when-to-use">何时使用</h2>

用于 Landing Page 首屏、产品首页、营销页面顶部等需要展示核心价值主张和引导用户行动的 Hero 区域。

常见场景：

- SaaS 产品 Landing Page 的首屏 Hero。
- 开源项目首页的品牌展示区域。
- 营销活动页的核心转化区域。

不推荐：

- 需要动态内容的 Hero（如 A/B 测试 variant），建议基于此组件扩展。
- 纯文字无配图的 Hero 场景，应使用更简化的布局。
- 需要复杂动画或视频背景的场景。

HeroClassic 是一个零配置的展示型块组件（block component），无需传递任何 props 即可渲染完整的 Hero 区域。自定义需通过修改源码实现。

<h2 id="api">API</h2>

HeroClassic 为展示型块组件，当前无对外暴露的 props。组件内部使用 Tailwind CSS 类名控制所有样式，包括响应式断点（`sm:`、`md:`、`lg:`）。

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| N/A | 当前版本无 props | - | - |

如需自定义内容（标题、描述、按钮文案、图片等），请直接修改源文件 `src/components/gooseui/components/blocks/hero/hero-classic.tsx`。

### 内部结构

组件渲染一个 `<section>`，内部包含：

- **Badge**：圆角边框 + 脉冲动画圆点 + "New Release v2.0" 文案。
- **Headline**：多行标题，关键词高亮为 `text-primary`。
- **Description**：灰色副标题，最大宽度 `max-w-xl`。
- **CTA Buttons**：两个按钮 — "Get Started Free"（`variant="solid"`）和 "Watch Demo"（`variant="outline"`），带图标动画。
- **Social Proof**：4 个占位头像 + "2,000+ developers building with us"。
- **Hero Image**：占位图区域含渐变装饰、网格纹理叠加层、两张浮动状态卡片。

<h2 id="faq">常见问题</h2>

### 如何修改标题和按钮文案？

HeroClassic 是零配置展示组件，文案硬编码在组件内部。修改需直接编辑源文件 `hero-classic.tsx` 中的对应文本内容。

### 如何替换占位图为真实图片？

在源文件中找到 `{/* Main image placeholder */}` 注释区域，将 `div` 占位内容替换为 `<img>` 或 `<Image>` 组件。

### 移动端适配如何？

组件已内置响应式断点：`py-16 sm:py-24 md:py-32`、标题字号 `text-4xl sm:text-5xl lg:text-6xl`、按钮 `flex-col sm:flex-row`。浮动卡片在移动端隐藏（`hidden md:block`）。

### 与 hero-split 的区别？

`hero-classic` 采用左右均分双栏布局（`md:grid-cols-2`），右侧为带装饰的完整图片区域。`hero-split` 布局更灵活，详见其文档。

### 可以添加动画吗？

当前版本仅含基础的 hover 动画（按钮图标位移）和 Badge 脉冲动画。如需入场动画，建议在源码中组合 `motion` 或 CSS animation。

<h2 id="llm-rules">LLM 使用规则</h2>

1. 从 `@/components/marketing-blocks/hero-classic` 导入 `HeroClassic`。
2. `HeroClassic` 为零配置组件，直接渲染即可，无需传 props。
3. 如需自定义内容，修改源文件 `src/components/gooseui/components/blocks/hero/hero-classic.tsx`。
4. 不要自行用 Tailwind 拼装 Hero 区域替代本组件。
5. 不要引入 Ant Design 相关依赖。
