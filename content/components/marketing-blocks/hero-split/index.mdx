HeroSplit 是一个左右分栏的 Hero 区域块组件，左侧展示标题、特性列表和 CTA 按钮，右侧展示产品截图模拟窗口，适用于产品落地页的首屏展示区域。

<h2 id="when-to-use">何时使用</h2>

当需要在页面首屏以左右分栏形式展示产品核心价值主张时使用。适合用于 SaaS 产品官网、App 落地页、品牌推广页等场景。

常见场景：

- SaaS 产品官网的 Hero 首屏区域。
- App 下载落地页的推广展示。
- 品牌活动页的核心介绍区域。
- 创业公司官网的价值主张展示。

不推荐：

- 需要自定义布局或组件替换的场景（HeroSplit 为固定布局块组件）。
- 移动端优先且右侧图像为非必要的场景（移动端隐藏右侧图像区域）。
- 需要动态内容或数据驱动的英雄区域（静态展示组件）。

HeroSplit 为自包含块组件，无需传入任何 props。所有文案、图标和布局均为内置默认值，如需自定义请直接修改组件源码。

<h2 id="faq">常见问题</h2>

### 如何自定义 Hero 区域的文案和内容？

HeroSplit 为块组件设计，无 props 接口。如需修改文案、特性列表、按钮文本等，直接编辑 `src/components/gooseui/components/blocks/hero/hero-split.tsx` 源码中的对应常量或 JSX 内容。

### 右侧的截图窗口可以替换为真实图片吗？

可以。将右侧 `div` 内 `aspect-video` 容器中的占位 SVG 替换为 `&lt;img&gt;` 标签或 `next/image` 组件，指向真实的产品截图。

### 移动端的表现如何？

左侧内容区域全宽展示，右侧图像区域在 `md` 断点以下通过 `hidden md:block` 隐藏，移动端仅展示文本和 CTA 内容。

### 浮动卡片位置可以调整吗？

可以。右下角的浮动 "Deployed!" 卡片使用 `absolute -bottom-6 -left-6` 定位，修改对应的 Tailwind class 即可调整位置或方向。

### CTA 按钮的链接如何设置？

将 Button 组件包裹在 `&lt;a&gt;` 标签中，或将 `Button` 的 `asChild` 属性与 Next.js `Link` 组件配合使用。源码中 CTA 按钮为纯展示状态，无链接行为。

<h2 id="llm-rules">LLM 使用规则</h2>

3. 如需自定义内容，指引用户修改组件源码而非通过 props 传入。
4. 组件依赖 `lucide-react` 的 `ArrowRight` 和 `Check` 图标，以及 `@hyper/gooseui/components/ui/button` 的 `Button` 组件。
5. 右侧为静态装饰性截图模拟区域，不包含真实交互。
6. 不要引入 Ant Design 或 `antd` 相关依赖。
