组件类型: 图表 / BKLit Charts
底层依赖: @visx (shape, scale, group, responsive), motion/react

## 何时使用
在多维放射轴上展示和对比数据的综合表现。适合能力评估、竞品对比、多维评分等场景。所有数值需预先归一化到 0-100。

## Props 详解

| 参数 | 说明 | 类型 | 默认值 | 必填 |

| size | 固定像素尺寸，不传则自适应父容器 | `number` | — | 否 |

### RadarData (数据类型)

```ts
interface RadarData {
  label: string                    // 数据系列名称
  values: Record<string, number>  // 各维度值 0-100，key 对应 metrics[].key
  color?: string                  // 自定义颜色，默认使用 Chart 调色板 var(--chart-1)~var(--chart-5)
}
```

### RadarMetric (维度类型)

```ts
interface RadarMetric {
  key: string    // 唯一标识，与 RadarData.values 的 key 对应
  label: string  // 轴标签显示文本
}
```

## 示例代码

### 基础单组数据

```tsx
import { RadarChart } from "@/components/charts/chart-kit"
import { RadarArea, RadarGrid, RadarAxis, RadarLabels } from "@/components/charts/chart-kit"

const metrics = [
  { key: "speed", label: "速度" },
  { key: "strength", label: "力量" },
  { key: "endurance", label: "耐力" },
  { key: "agility", label: "敏捷" },
  { key: "intelligence", label: "智力" },
]

const data = [
  {
    label: "当前能力",
    values: { speed: 80, strength: 65, endurance: 90, agility: 70, intelligence: 55 },
  },
]

export default function Demo() {
  return (
    <RadarChart data={data} metrics={metrics}>
      <RadarGrid />
      <RadarAxis />
      <RadarArea index={0} />
      <RadarLabels />
    </RadarChart>
  )
}
```

### 多组数据对比（带自定义颜色）

```tsx
const metrics = [
  { key: "price", label: "价格" },
  { key: "quality", label: "质量" },
  { key: "service", label: "服务" },
  { key: "design", label: "设计" },
  { key: "ecosystem", label: "生态" },
]

const data = [
  {
    label: "产品 A",
    values: { price: 75, quality: 85, service: 70, design: 90, ecosystem: 60 },
    color: "var(--chart-1)",
  },
  {
    label: "产品 B",
    values: { price: 60, quality: 70, service: 85, design: 65, ecosystem: 90 },
    color: "var(--chart-2)",
  },
]

<RadarChart data={data} metrics={metrics}>
  <RadarGrid />
  <RadarAxis />
  <RadarArea index={0} />
  <RadarArea index={1} />
  <RadarLabels interactive />
</RadarChart>
```

### 固定尺寸与自定义网格层数

```tsx
<RadarChart data={data} metrics={metrics} size={400} levels={4} margin={50}>
  <RadarGrid showLabels={false} stroke="var(--chart-grid)" strokeOpacity={0.3} />
  <RadarAxis stroke="var(--chart-grid)" strokeOpacity={0.4} />
  <RadarArea index={0} showGlow={false} />
  <RadarLabels offset={20} fontSize={12} />
</RadarChart>
```

## 约束

- 需要 @visx/* 和 motion/react 依赖（已内置于 BKLit Charts）。
- `data` 数组长度不超过 5（超出时多边形过度重叠，难以阅读）。
- `metrics` 数组长度建议 3-8 个，超过 10 个时标签和轴线密集，可读性严重下降。
- 每个 `values` 的值须为 0-100 的数字，组件不会自动归一化。
- `metrics[].key` 必须与所有 `data[].values` 的 key 一致，否则对应维度的值为 0。
- 不传 `size` 时，父容器必须有确定尺寸或 `aspect-square`，否则渲染高度为 0。
- 子组件必须嵌套在 `<RadarChart>` 内部，依赖 Provider 注入的上下文。

## 最佳实践

- 每个 `RadarArea` 通过 `index` 对应 `data[index]`，顺序和数量需一致。
- 多组对比时，每组指定不同的 `color` 以区分，默认使用 Chart 内置的五色调色板。
- 自定义颜色推荐使用 CSS 变量（如 `var(--chart-1)`），自动适配主题切换。
- 标签较多的场景可增大 `RadarLabels.offset` 避免与数据点重叠。
- 父容器设置固定宽高或使用 `size` 属性，避免布局抖动。
- 对于包含敏感比较的场景（如人物能力评估），建议添加图例说明各多边形含义。
- 动画可通过 `animate={false}` 关闭，或通过 `motionReplayKey` 触发重播。
