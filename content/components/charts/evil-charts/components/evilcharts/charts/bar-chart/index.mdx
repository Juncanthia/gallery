EvilBarChart 是一个基于 Recharts 的可组合柱状图组件，采用 config + data 模式，通过组合子组件（Bar、XAxis、YAxis、Grid、Tooltip、Legend）构建图表。

<h2 id="when-to-use">何时使用</h2>

需要以柱状图形式展示分类数据的数值对比时使用 EvilBarChart。

常见场景：

- 周期性数据对比：月度营收、季度成本、年度增长。
- 多系列对比：不同产品线的销售额对比。
- 堆叠数据：展示部分与整体的构成关系。
- 百分比堆叠：展示各部分的占比变化趋势。
- 水平柱状图：类别名称较长或需要横向阅读时。
- 带缩放刷的图表：数据量较大，需要用户自由缩放区间。

EvilBarChart 通过可组合的子组件实现灵活配置：

- `EvilBarChart`：根容器，持有数据、配置和共享上下文。
- `Bar`：单个柱状系列，支持 6 种视觉变体和动画。
- `XAxis` / `YAxis`：坐标轴，自动适配布局方向。
- `Grid`：背景网格线，跟随布局自动对齐。
- `Tooltip`：悬浮提示框。
- `Legend`：图例，支持点击选中/取消系列。

### Bar 变体

`Bar` 支持 6 种视觉变体：`default`、`hatched`、`duotone`、`duotone-reverse`、`gradient`、`stripped`。

```tsx
import { EvilBarChart } from "@/components/charts/evil-charts"
import { Bar, XAxis } from "@/components/charts/evil-charts/components/bar-chart"
import { type ChartConfig } from "@/components/charts/evil-charts/components/chart"

const chartConfig = {
  sales: {
    label: "销售额",
    colors: { light: ["#2563eb"], dark: ["#60a5fa"] },
  },
} satisfies ChartConfig

const variants = ["default", "hatched", "duotone", "duotone-reverse", "gradient", "stripped"] as const

export default function Demo() {
  return (
    <div className="grid grid-cols-2 gap-4">
      {variants.map((variant) => (
        <EvilBarChart
          key={variant}
          data={data}
          config={chartConfig}
          className="h-48 w-full p-2"
        >
          <XAxis dataKey="month" />
          <Bar dataKey="sales" variant={variant} />
        </EvilBarChart>
      ))}
    </div>
  )
}
```

<h2 id="faq">常见问题</h2>

### 如何配置颜色？

通过 `config` 属性为每个系列指定 `label` 和 `colors`。`colors` 支持单色或多色渐变（自动均分）:

```tsx
const chartConfig = {
  sales: {
    label: "销售额",
    colors: {
      light: ["#2563eb"],
      dark: ["#60a5fa"],
    },
  },
} satisfies ChartConfig
```

### 如何做堆叠柱状图？

设置 `stackType="stacked"`:

```tsx
<EvilBarChart data={data} config={chartConfig} stackType="stacked">
  <Bar dataKey="revenue" />
  <Bar dataKey="cost" />
</EvilBarChart>
```

### 如何做百分比堆叠？

设置 `stackType="percent"`:

```tsx
<EvilBarChart data={data} config={chartConfig} stackType="percent">
  <Bar dataKey="revenue" />
  <Bar dataKey="cost" />
</EvilBarChart>
```

### 水平柱状图如何配置？

设置 `layout="horizontal"`，同时 XAxis 需显式指定 `type="number"`，YAxis 需 `dataKey` 和 `type="category"`:

```tsx
<EvilBarChart data={data} config={chartConfig} layout="horizontal">
  <XAxis type="number" />
  <YAxis dataKey="month" type="category" />
  <Bar dataKey="revenue" />
</EvilBarChart>
```

### 如何关闭柱条动画？

设置 `animationType="none"`:

```tsx
<EvilBarChart data={data} config={chartConfig} animationType="none">
  <Bar dataKey="revenue" />
</EvilBarChart>
```

### 如何实现点击柱条高亮其余变暗？

为 `Bar` 设置 `isClickable`，并通过 `onSelectionChange` 或 `defaultSelectedDataKey` 控制选中状态。未选中系列的柱条会自动半透明化（opacity: 0.3）。

```tsx
<EvilBarChart data={data} config={chartConfig}>
  <Bar dataKey="revenue" isClickable />
  <Bar dataKey="cost" isClickable />
</EvilBarChart>
```

### loading 状态的工作机制？

设置 `isLoading` 后，所有子组件（XAxis、Tooltip 等）自动隐藏，替换为骨架柱条和 shimmer 动画。`loadingBars` 控制骨架柱条数量（默认 12）。shimmer 使用 motion.dev 的 `onUpdate` 回调精确同步，避免 setTimeout 时序漂移。

### 如何自定义样式？

优先使用 `variant`（default、hatched 等）和布局属性组合。需调整圆角时使用 `barRadius`。需完全自定义时通过 `barProps` 透传 Recharts Bar 原始属性。不推荐在调用侧大量覆写 `className`。

### Buffer Bar 的适用场景？

`bufferBar` 适用于需要展示"实际值 + 目标值/预期值"的场景。实际值使用标准柱条，目标值使用 `bufferBar` 渲染为斜线填充的柱条，视觉上自然区分。

<h2 id="llm-rules">LLM 使用规则</h2>

AI 生成 EvilBarChart 代码时：

1. 从 `@/components/charts/evil-charts` 导入 `EvilBarChart`
2. 子组件从 `@/components/charts/evil-charts/components/bar-chart` 导入：`Bar`、`XAxis`、`YAxis`、`Grid`、`Tooltip`、`Legend`
3. `ChartConfig` 类型从 `@/components/charts/evil-charts/components/chart` 导入
4. `config` 使用 `satisfies ChartConfig` 确保类型安全
5. 每个系列必须同时在 `config` 和 `data` 中存在对应字段
6. 必传 `children`，至少包含一个 `Bar` 和一个 `XAxis`
7. 使用 `variant` 选择视觉样式，不要手动写 CSS 填充
8. 使用 `stackType` 而非手动计算堆叠
9. `layout="horizontal"` 时，XAxis 和 YAxis 需互换 type
10. 加载态使用 `isLoading`，不要在外部自行实现骨架屏
11. 不要直接使用 Recharts 的 BarChart/Bar，始终通过 EvilBarChart 组合
