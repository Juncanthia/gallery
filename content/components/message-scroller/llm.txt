# Message 全局提示

全局展示操作反馈信息。

本组件借鉴 Ant Design Message 的使用心智，但不是 AntD 兼容层。实现以本地源码为准，不引入 AntD 的 ConfigProvider、css-in-js、Design Token 运行时或历史兼容语法糖。

## 何时使用 {#when-to-use}

适合在需要 全局提示 相关能力的界面中使用。优先通过组件自身 props 表达状态和语义，避免在业务侧堆叠重复 className。

常见场景：

- 需要复用统一设计语言、状态样式和可访问性约定。
- 需要和本项目 Tailwind 主题变量、暗色模式、Radix/Base UI primitive 或本地交互行为保持一致。
- 需要让 AI 或业务代码稳定地从 `@/components/ui/message-scroller` 引用组件，而不是直接引用第三方底层库。

不推荐：

- 为了照搬 Ant Design API 而新增兼容别名。
- 在调用侧绕过组件 API，直接依赖内部 DOM 结构完成业务逻辑。
- 引入 `antd`、`@ant-design/icons`、`antd-style` 或 rc-component 作为使用该组件的前置条件。

## 代码演示 {#examples}

### 基础用法

```tsx
import { MessageScroller } from "@/components/ui/message-scroller"

export default function Demo() {
  return (
    <MessageScroller>
      全局提示
    </MessageScroller>
  )
}
```

> 说明：这是面向 LLM 的最小导入示例。复杂组件应结合下方 API 摘要和源码类型补充必需 props，例如 `options`、`items`、`columns`、`dataSource` 等。

## API {#api}

### MessageScrollerProvider

包装 `@shadcn/react/message-scroller` 的 Provider，提供滚动上下文。支持 `React.ComponentProps<typeof MessageScrollerPrimitive.Provider>` 的所有属性。

### MessageScroller

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 自定义样式类名 | `string` | - |

同时支持 `React.ComponentProps<typeof MessageScrollerPrimitive.Root>` 的所有属性。

### MessageScrollerViewport

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 自定义样式类名 | `string` | - |

同时支持 `React.ComponentProps<typeof MessageScrollerPrimitive.Viewport>` 的所有属性。

### MessageScrollerContent

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| className | 自定义样式类名 | `string` | - |

同时支持 `React.ComponentProps<typeof MessageScrollerPrimitive.Content>` 的所有属性。

### MessageScrollerItem

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| scrollAnchor | 是否作为滚动锚点 | `boolean` | `false` |
| className | 自定义样式类名 | `string` | - |

同时支持 `React.ComponentProps<typeof MessageScrollerPrimitive.Item>` 的所有属性。

### MessageScrollerButton

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| direction | 滚动方向 | `"start" \| "end"` | `"end"` |
| variant | 按钮样式变体，透传至 Button | `"filled" \| ...` | `"filled"` |
| size | 按钮尺寸，透传至 Button | `"small" \| ...` | `"small"` |
| render | 自定义渲染元素 | `ReactNode` | `<Button variant={variant} size={size} shape="square" />` |
| children | 按钮内容（默认为箭头图标 + sr-only 文本） | `ReactNode` | 箭头图标 |
| className | 自定义样式类名 | `string` | - |

同时支持 `React.ComponentProps<typeof MessageScrollerPrimitive.Button>` 的其他属性。

### Hooks

- `useMessageScroller` — 返回 MessageScroller 上下文
- `useMessageScrollerScrollable` — 返回是否可滚动及滚动方向
- `useMessageScrollerVisibility` — 返回按钮显示/隐藏状态

## Semantic DOM {#semantic-dom}

源码中出现的稳定语义槽位：

- `message-scroller`
- `message-scroller-button`
- `message-scroller-content`
- `message-scroller-item`
- `message-scroller-viewport`

额外状态属性：

- `data-direction`
- `data-size`
- `data-slot`
- `data-variant`

生成测试、样式定位或自动化脚本时，优先使用这些语义属性；不要依赖易变的 Tailwind class 拼接顺序。

## 样式模型 {#styling-model}

- 样式主要由 Tailwind CSS className 与本地 `cn` 合并函数控制。
- 如果源码使用 CVA，则变体样式集中在组件文件或相邻 `*-variants.ts` 中。
- 如果源码包装 Radix/Base UI primitive，则交互状态、键盘行为和 ARIA 基础能力来自底层 primitive，本地组件负责视觉和 API 收敛。
- 主题颜色优先使用 `bg-background`、`text-foreground`、`border-border`、`text-muted-foreground`、`ring-ring` 等项目级语义变量。

## 依赖与实现说明 {#implementation-notes}

核心文件：

- `src/components/ui/message-scroller.tsx`

直接依赖摘要：

- `@shadcn/react/message-scroller`
- `@/components/ui/button`

实现注意：

1. 组件 API 以本地 TypeScript 类型为准。
2. 样式应优先通过组件 props、variant 或本地 className 扩展点表达。
3. 如果需要新增能力，应先修改该组件源码和相邻 variant 文件，再更新本 llm 文档。
4. 不要把第三方文档中的全量 API 视为本地已支持能力。

## 可访问性 {#accessibility}

- 保留原生元素或底层 primitive 的键盘语义。
- 交互型组件应提供可访问名称，例如 `aria-label`、`title` 或可见文本。
- 禁用、加载、展开、选中等状态应优先使用组件已有 props，而不是只改视觉 class。
- 纯图标触发器必须提供明确的可访问名称。

## 与 Ant Design 的取舍 {#antd-differences}

| 方向 | 本地决策 |
| --- | --- |
| API 来源 | 以 `src/components/ui/message-scroller.tsx` 的 TypeScript 类型为准 |
| 视觉系统 | 使用 Tailwind CSS 和项目主题变量 |
| 全局配置 | 不依赖 AntD ConfigProvider |
| 样式运行时 | 不引入 AntD css-in-js 或 token 运行时 |
| 兼容目标 | 借鉴 AntD 使用心智，但不承诺 API 兼容 |

## LLM 使用规则 {#llm-rules}

当 AI 生成本项目 Message 代码时，应遵守：

1. 从 `@/components/ui/message-scroller` 导入，不要从 `antd` 导入。
2. 只使用本文件 API 摘要和源码中存在的 props。
3. 不要生成 AntD 专属的 `classNames`、`styles`、`ConfigProvider`、Design Token 配置，除非本地源码明确支持。
4. 需要自定义样式时，优先使用组件暴露的 props；最后才使用 `className`。
5. 交互组件必须保留键盘和 ARIA 语义，纯图标操作必须补 `aria-label`。
6. 如果不确定某个 AntD API 是否存在，先检查 `src/components/ui/message-scroller.tsx`，不要凭 AntD 记忆生成。
